# <a name="develop-with-visual-studio"></a>Разработка с помощью Visual Studio

Вы можете значительно повысить функциональность форм InfoPath, расширив их с помощью управляемого кода, разработанного Visual Studio 2012 г. Затем можно опубликовать формы с кодом для формирования библиотек на SharePoint Server 2013.
  
Для начала программирования и развертывания форм InfoPath с управляемым кодом необходимо выполнить три основных действия:
  
1. Установите Visual Studio 2012 с [надстройки 2012 набор средств Microsoft Visual Studio Tools для работы с приложениями 2012.](https://www.microsoft.com/en-us/download/details.aspx?id=38807) 
    
2. Установите язык программирования, а затем напишите и отлаговите код в редакторе Visual Studio 2012 г.
    
3. По завершению разработки формы и разработки кода шаблон формы может быть опубликован в SharePoint Server 2013.
    
Вот несколько причин, по которой необходимо создать формы, совместимые с SharePoint Server 2013:
  
- Формы, развернутые SharePoint Server 2013 с InfoPath Forms Services могут быть заполнены в браузере. Это позволяет пользователям, у которых нет установки InfoPath, открывать и использовать ваши формы.
    
- Вы проектировать только одну версию формы. Формы, совместимые с Microsoft SharePoint Server, также совместимы с заполнитель infoPath, но формы, совместимые только с наполнительом InfoPath, не могут быть открыты в браузере.
    
Существует два способа публикации формы для SharePoint: SharePoint в песочнице и решений, развернутых администратором. Дополнительные сведения о каждом методе публикации и предложениях о том, какой метод наиболее оптимальный для вашего сценария, см. в публикации [Forms with Code.](publishing-forms-with-code.md) Например, решения для решений с песочницами см. в [примере "Решения из песочницы".](sample-sandboxed-solutions.md)
  
## <a name="developing-with-visual-studio"></a>Разработка с Visual Studio

После Visual Studio 2012 года и установки надстройки 2012 набор средств Microsoft Visual Studio Tools для работы с приложениями 2012 г. вы готовы приступить к разработке решений кода с управляемым кодом InfoPath.
  
### <a name="choosing-a-programming-language"></a>Выбор языка программирования

InfoPath предоставляет параметры для программирования с помощью четырех версий объектной модели InfoPath на двух языках: Visual Basic и C#. Четыре версии объектной модели обеспечивают совместимость с InfoPath 2013, InfoPath, Office InfoPath 2007 и Microsoft InfoPath 2003.
  
### <a name="to-specify-the-programming-language-and-object-model"></a>Указание языка программирования и объектной модели

1. Открыв проект шаблона формы в конструкторе InfoPath, щелкните **Язык** на **вкладке Developer.** 
    
2. В категории **Программирование** диалогового окна **Параметры формы** выберите используемый язык из раскрывающегося списка **Язык кода шаблона формы**. Затем выберите версию объектной модели из выпадаемого списка **целевой** версии. Параметр **Целевой версии,** совместимый только с InfoPath 2013, не имеет года версии, следующего за **именем InfoPath.** 
    
    > [!NOTE]
    > Код поддерживается не всеми типами шаблонов форм. Например, типы шаблона формы **Список SharePoint** и **Блоки шаблона** не поддерживают код формы. При разработке типа шаблона, не поддерживающего код, вкладка **Разработчик** не будет доступна. Кроме того, только некоторые типы шаблонов форм поддерживают все четыре версии объектной модели. Например, тип шаблона Blank **Form (InfoPath Filler)** поддерживает все четыре версии объектной модели (и создает шаблон формы, совместимый только с наполнительом InfoPath в этих версиях), но шаблон **Blank Form** поддерживает только InfoPath 2013 и InfoPath (и создает шаблоны форм, совместимые как с наполнительом InfoPath, так и браузером). 
  
    Вы можете настроить язык программирования по умолчанию, чтобы конструктор форм InfoPath всегда начал с версии языковой и объектной модели по вашему выбору.
    
### <a name="to-set-the-default-programming-language"></a>Настройка языка программирования по умолчанию

1. Щелкните вкладку **Файл**, а затем щелкните элемент **Параметры**.
    
2. В разделе **Общие** диалогового окна **Параметры InfoPath** щелкните элемент **Дополнительные параметры**.
    
3. На вкладке **Разработка** диалогового окна **Параметры** выберите язык программирования по умолчанию в разделе **Параметры программирования по умолчанию**.  
    
### <a name="writing-code"></a>Написание кода

Теперь вы можете начать разработку с InfoPath 2013 и Visual Studio 2012. 
  
### <a name="to-start-the-visual-studio-code-editor"></a>Запуск редактора Visual Studio Code

1. Откройте шаблон формы в конструкторе InfoPath.
    
2. Щелкните элемент **Редактор кода** на вкладке **Разработчик**. 
    
> [!TIP]
> Вы также можете запустить редактор кода и автоматически добавить обработчики событий для событий формы и управления с помощью команд на вкладке **Разработчик,** меню контекста и других методов пользовательского интерфейса. Дополнительные сведения см. в [добавлении обработи](how-to-add-an-event-handler.md)
