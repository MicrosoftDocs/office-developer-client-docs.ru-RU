### <a name="ref"></a><span data-ttu-id="c7af7-189">#REF!</span><span class="sxs-lookup"><span data-stu-id="c7af7-189">#REF!</span></span>

<span data-ttu-id="c7af7-p117">Excel возвращает ошибку **#ССЫЛКА!** в выражении, если оно копируется в расположение, где получившаяся относительная ссылка выходит за границы. <span data-ttu-id="c7af7-p118">Если функция надстройки принимает в качестве аргументов ссылки, то эту ошибку рекомендуется возвращать при получении недействительных ссылок или передаче ошибок со ссылками. В разделе статьи [Управление памятью в Excel](memory-management-in-excel.md), посвященном XLOPER/XLOPER12, описано, как создавать функции, которые могут принимать и возвращать ссылки в качестве аргументов.</span><span class="sxs-lookup"><span data-stu-id="c7af7-p118">If your add-in function takes reference arguments, you should consider returning this error if the references are invalid, or if you are passed a reference error. ### <a name="num"></a><span data-ttu-id="c7af7-201">#NUM!</span><span class="sxs-lookup"><span data-stu-id="c7af7-201">#NUM!</span></span>

<span data-ttu-id="c7af7-p120">Многие из встроенных числовых и математических функций в Excel возвращают ошибку **#ЧИСЛО!**, если числовые входные данные выходят за пределы допустимого диапазона. Рекомендуем возвращать эту ошибку из функции надстройки, чтобы указать, что числовые входные данные недействительны или выходят за пределы допустимого диапазона.</span><span class="sxs-lookup"><span data-stu-id="c7af7-p120">Many of the built-in numerical and mathematical functions in Excel return the **#NUM!** error when a numerical input is out of the permitted range, for example,  `LN(0)`. ### <a name="na"></a><span data-ttu-id="c7af7-205">#N/A</span><span class="sxs-lookup"><span data-stu-id="c7af7-205">#N/A</span></span>

<span data-ttu-id="c7af7-p121">Ошибку **#Н/Д** часто возвращают, чтобы указать, что успешный или осмысленный результат недоступен.
