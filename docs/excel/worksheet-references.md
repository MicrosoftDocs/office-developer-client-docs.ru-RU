<span data-ttu-id="9f992-106">Ссылка в Microsoft Excel — это тип данных, который ссылается на прямоугольный блок ячеек (который может быть только одной ячейкой), или в некоторых случаях несколько разроных блоков ячеек.</span><span class="sxs-lookup"><span data-stu-id="9f992-106">A reference in Microsoft Excel is a data type that refers to a rectangular block of cells (which can be just one cell), or in some cases, a number of disjoint blocks of cells.</span></span> <span data-ttu-id="9f992-107">Внутри Excel используется один тип ссылок для ячеек на текущем листе, известный как внутренняя ссылка.</span><span class="sxs-lookup"><span data-stu-id="9f992-107">Internally, Excel uses one reference type for cells on the current sheet, known as an internal reference.</span></span> <span data-ttu-id="9f992-108">Любая ячейка, не на текущем листе, описывается другим типом ссылки, известной как внешняя ссылка.</span><span class="sxs-lookup"><span data-stu-id="9f992-108">Any cell that is not on the current sheet is described by another type of reference known as an external reference.</span></span> <span data-ttu-id="9f992-109">В следующем разделе см. определение активного и текущего.</span><span class="sxs-lookup"><span data-stu-id="9f992-109">See the next section for the definition of active and current.</span></span>
  
## <a name="active-vs-current"></a><span data-ttu-id="9f992-110">Active vs. Current</span><span class="sxs-lookup"><span data-stu-id="9f992-110">Active vs. Current</span></span>

<span data-ttu-id="9f992-111">В Excel термин active относится к просмотру пользователя.</span><span class="sxs-lookup"><span data-stu-id="9f992-111">In Excel, the term active refers to what the user is viewing.</span></span> <span data-ttu-id="9f992-112">Активная книга и таблица — это те книги, которые пользователь в настоящее время ищет или, если Excel потерял фокус на другое приложение, смотрел, когда Excel последний раз был фокус.</span><span class="sxs-lookup"><span data-stu-id="9f992-112">The active workbook and worksheet are those that the user is currently looking at, or, if Excel has lost focus to another application, was looking at when Excel last had focus.</span></span> <span data-ttu-id="9f992-113">Активный лист всегда находится в активной книге.</span><span class="sxs-lookup"><span data-stu-id="9f992-113">The active sheet is always in the active workbook.</span></span> <span data-ttu-id="9f992-114">Одна или несколько ячеек, выбранных в активном листе, называются активными ячейками.</span><span class="sxs-lookup"><span data-stu-id="9f992-114">The one or more cells that are selected in the active sheet are known as the active cells.</span></span> <span data-ttu-id="9f992-115">Если встроенный объект имеет фокус, последние выбранные ячейки по-прежнему активны.</span><span class="sxs-lookup"><span data-stu-id="9f992-115">If an embedded object has focus, the last-selected cells are still active.</span></span> 
  
<span data-ttu-id="9f992-116">Текущий термин относится к Excel пересчета.</span><span class="sxs-lookup"><span data-stu-id="9f992-116">The term current refers to what Excel is recalculating.</span></span> <span data-ttu-id="9f992-117">Текущая книга и таблица — это те, которые в настоящее время пересчитываются.</span><span class="sxs-lookup"><span data-stu-id="9f992-117">The current workbook and worksheet are those that are currently being recalculated.</span></span> <span data-ttu-id="9f992-118">Текущий лист всегда находится в текущей книге.</span><span class="sxs-lookup"><span data-stu-id="9f992-118">The current sheet is always in the current workbook.</span></span> <span data-ttu-id="9f992-119">Пересчитаемая ячейка называется текущей ячейкой или в случае пересчета формулы массива текущими ячейками.</span><span class="sxs-lookup"><span data-stu-id="9f992-119">The cell being recalculated is known as the current cell, or, in the case of an array formula being recalculated, the current cells.</span></span> 
  
<span data-ttu-id="9f992-120">Важные моменты, которые следует помнить:</span><span class="sxs-lookup"><span data-stu-id="9f992-120">The important points to remember are as follows:</span></span>
  
- <span data-ttu-id="9f992-121">Активная книга/таблица/ячейка обычно не является текущей, хотя она может быть.</span><span class="sxs-lookup"><span data-stu-id="9f992-121">The active workbook/worksheet/cell is not generally the current one, although it can be.</span></span>
    
- <span data-ttu-id="9f992-122">Функция надстройки, будь то в модуле Visual Basic для приложений (VBA) или DLL или XLL, всегда вызвана из текущей ячейки на текущем листе или одна из них в случае многоуровневого пересчета (MTR).</span><span class="sxs-lookup"><span data-stu-id="9f992-122">An add-in function, whether in a Visual Basic for Applications (VBA) module or a DLL or XLL, is always called from the current cell on the current sheet, or one of them in the case of multithreaded recalculation (MTR).</span></span>
    
<span data-ttu-id="9f992-123">Многие Excel, которые предоставляют сведения о ячейке, диапазоне ячеек или листе в книге, различают активную книгу, лист или ячейку и текущую книгу, лист или ячейку.</span><span class="sxs-lookup"><span data-stu-id="9f992-123">Many Excel functions that provide information about a cell, a range of cells, or a sheet in a workbook distinguish between the active workbook, sheet, or cell and the current workbook, sheet, or cell.</span></span> <span data-ttu-id="9f992-124">Это различие отражается в типах данных, используемых для описания ссылок на блоки ячеек, как описано в следующем разделе.</span><span class="sxs-lookup"><span data-stu-id="9f992-124">This difference is reflected in the data types used to describe references to blocks of cells, as described in the following section.</span></span>
  
## <a name="internal-and-external-worksheet-references"></a><span data-ttu-id="9f992-125">Ссылки на внутренние и внешние таблицы</span><span class="sxs-lookup"><span data-stu-id="9f992-125">Internal and External Worksheet References</span></span>

<span data-ttu-id="9f992-126">Главное различие между внутренними и внешними ссылками состоит в том, что внешний тип справочных данных содержит ID для таблицы, а также описание, на которые ссылаются ячейки.</span><span class="sxs-lookup"><span data-stu-id="9f992-126">The key difference between internal and external references is that the external reference data type contains an ID for the worksheet and also a description of which cells are referred to.</span></span> <span data-ttu-id="9f992-127">Внутренняя ссылка не содержит ссылки на лист, неявно подразумевается, что лист является текущим листом.</span><span class="sxs-lookup"><span data-stu-id="9f992-127">An internal reference contains no reference to the sheet—it is implicit that the sheet is the current sheet.</span></span> 
  
<span data-ttu-id="9f992-128">Многие функции API C возвращают ссылки или принимают справочные аргументы.</span><span class="sxs-lookup"><span data-stu-id="9f992-128">Many C API functions return references or take reference arguments.</span></span> <span data-ttu-id="9f992-129">Любая функция API C, которая принимает справочные аргументы, принимает внутренние или внешние ссылки, за исключением **функции xlSheetNm,** которая требует внешней ссылки.</span><span class="sxs-lookup"><span data-stu-id="9f992-129">Any C API function that takes reference arguments accepts either internal or external references, except the **xlSheetNm** function, which requires an external reference.</span></span> <span data-ttu-id="9f992-130">Некоторые функции возвращают только внутренние или внешние ссылки.</span><span class="sxs-lookup"><span data-stu-id="9f992-130">Some functions only return either internal or external references.</span></span> <span data-ttu-id="9f992-131">Например, функция C API [xlfCaller](xlfcaller.md) возвращает ссылку на ячейки вызова по определению на текущем листе.</span><span class="sxs-lookup"><span data-stu-id="9f992-131">For example, the C API function [xlfCaller](xlfcaller.md) returns a reference to the calling cells, by definition, on the current sheet.</span></span> <span data-ttu-id="9f992-132">Возвращаемая ссылка всегда является внутренней ссылкой, хотя функция может возвращать нестандартные типы, в которых функция не вызвана из ячейки таблицы.</span><span class="sxs-lookup"><span data-stu-id="9f992-132">The returned reference is always an internal reference, although the function can return non-reference types where the function is not called from a worksheet cell.</span></span> <span data-ttu-id="9f992-133">Функция C API [xlSheetId](xlsheetid.md) всегда возвращает ID таблицы, содержаной в внешнем типе справочных данных.</span><span class="sxs-lookup"><span data-stu-id="9f992-133">The C API function [xlSheetId](xlsheetid.md) always returns the ID of a worksheet contained within an external reference data type.</span></span> 
  
<span data-ttu-id="9f992-134">Другое ключевое различие между внутренними и внешними типами ссылок состоит в том, что тип внешних эталонных данных может описывать несколько несоединяемых блоков ячеек на одном листе.</span><span class="sxs-lookup"><span data-stu-id="9f992-134">The other key difference between the internal and external reference types is that the external reference data type can describe multiple disjoint blocks of cells on the same sheet.</span></span> <span data-ttu-id="9f992-135">Внутренние ссылки могут описывать только один блок на текущем листе.</span><span class="sxs-lookup"><span data-stu-id="9f992-135">Internal references can describe only a single block on the current sheet.</span></span> <span data-ttu-id="9f992-136">Несоединяемые ссылки можно передать любой функции, которая принимает аргумент диапазона.</span><span class="sxs-lookup"><span data-stu-id="9f992-136">Disjoint references can be passed to any function that takes a range argument.</span></span>
