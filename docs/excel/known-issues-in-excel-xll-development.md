# <a name="known-issues-in-excel-xll-development"></a>Известные проблемы в Excel XLL

 **Область применения:** Excel 2013 | Office 2013 | Visual Studio 
  
В этом разделе описываются известные проблемы в Microsoft Excel, с которыми вы можете столкнуться в разработке XLL.
  
## <a name="unregistering-xll-commands-and-functions"></a>Незарегистрированные команды и функции XLL

Когда XLL регистрирует функцию или команду, Excel создает новое имя для ресурса и связывает ссылку на функцию DLL с этим именем. Имя взято из четвертого аргумента *pxFunctionText* функции [xlfRegister.](xlfregister-form-1.md) Это имя скрыто от обычных диалогов для управления именами таблиц. Чтобы удалить функцию или команду, необходимо удалить имя с помощью [функции xlfSetName,](xlfsetname.md) передав имя, но не передав определение. Однако ошибка не позволяет удалить имя из списков мастера функций. 
  
## <a name="argument-description-string-truncation-in-the-function-wizard"></a>Argument Description String Truncation in the Function Wizard

Параметр  *pxArgumentHelp1*  и все последующие параметры функции **xlfRegister** являются необязательными строками, соответствующими аргументам функции XLL. Мастер функции отображает их для оказания помощи в диалоговом окне конструкции аргументов. Иногда Excel строку, соответствующую заключительному аргументу одного или двух символов при ее отображке в диалоговом окне. Этого можно избежать, добавив дополнительную "пустую строку" в качестве последнего параметра "справка аргумента" при регистрации функции.
  
## <a name="binary-name-scope-limitation"></a>Двоичное ограничение области имен

Двоичные имена и их данные можно получить только тогда, когда таблица, которая была активна во время их создания, снова активна. Поскольку функции таблицы не могут активировать таблицы в книге (это могут делать только команды), приложения двоичных имен очень ограничены. Если у вас есть команда, вызываемая только из определенного таблицы, например, можно использовать двоичное имя для хранения данных, связанных с командой.
  
## <a name="xlset-and-workbooks-with-array-formulas"></a>xlSet и книги с формулами Array

В Excel 2003 и более ранних версиях функция [xlSet](xlset.md) не работает, если попытаться назначить значения диапазону на листе, который не является активным листом, где эквивалентный диапазон на активном листе содержит формулу массива. В этом Excel отображается сообщение "Нельзя изменить часть массива". Это было исправлено в Excel 2007 г. 
  
## <a name="circular-references-are-tolerated-in-data-tables"></a>Циркулярные ссылки допускаются в таблицах данных

Excel в настоящее время не вызывает ошибку, если расчет, основанный на таблице данных, относится к чему-то в самой таблице. При создании или изменении формул, используемых для вычисления значений таблицы данных, следует быть осторожными.
  
## <a name="converting-an-integer-xloper12-to-an-xloper"></a>Преобразование integer XLOPER12 в XLOPER

Поскольку **xltypeInt** типа integer — это 32-битный подписанный набор в типе **данных XLOPER12,** но это всего лишь 16-битный подписанный набор в типе данных **XLOPER,** не исключено, что некоторые значения **XLOPER12** не могут содержаться в integer **XLOPER**. Если Excel преобразует этот тип внутри страны, он объехает эту проблему, преобразуя тип в **xltypeNum XLOPER** с плавающей **точкой.**
  
Функция Framework [XLOper12ToXLOper](xloper12toxloper.md) зеркально отражает такое поведение, чтобы соответствовать Excel внутренне. При вызове этой функции не следует предполагать, что возвращенный **XLOPER** всегда будет **xltypeInt;** чтение **my_xloper.val.w** дает ложное значение, **если** тип my_xloper **xltypeNum**.
  
## <a name="returning-xloper-or-xloper12-by-modifying-arguments-in-place"></a>Возвращение XLOPER или XLOPER12 путем изменения аргументов на месте

Excel позволяет регистрацию функций, возвращающих **XLOPER** или **XLOPER12,** путем изменения аргумента на месте. Однако если аргумент **XLOPER XLOPER12** указывает на память, а указатель перезаписывается возвращаемой стоимостью функции DLL, Excel может /   утечь память. Excel не отображает ошибку, но в конечном итоге она может привести к сбою. Кроме того, если DLL выделяет память для возвращаемой величины, Excel может попытаться освободить эту память, что может привести к немедленному сбою приложения. Поэтому не следует изменять **аргументы** /  **XLOPER XLOPER12** на месте. Все **аргументы XLOPER** или **XLOPER12** следует рассматривать как строго только для чтения. 
  
Дополнительные сведения см. в статье [Управление памятью в Excel](memory-management-in-excel.md).
