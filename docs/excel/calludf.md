# <a name="calludf"></a>CallUDF

**Относится к**: Excel 2013 | Office 2013 | Visual Studio 
  
Вызывает пользовательскую функцию в высокопроизводительной вычислительной среде.
  
```cpp
int CallUDF(int SessionId, WCHAR *XllName, WCHAR *UDFName, LPXLOPER12 pxAsyncHandle, int (*CallBackAddr)(), int ArgCount, LPXLOPER12 Parameter1, ...)
```

## <a name="parameters"></a>Параметры

_SessionId_
  
> Идентификатор сеанса, в котором необходимо совершить вызов.
    
_Ксллнаме_
  
> Имя XLL, которое содержит определяемую пользователем функцию.
    
_Удфнаме_
  
> Имя пользовательской функции.
    
_Каллбаккаддр_
  
> Функция, которая должна вызываться соединителем при завершении пользовательской функции.
    
_Пксасинчандле_
  
> Асинхронный дескриптор, используемый Excel и соединителем для отслеживания незавершенного вызова пользовательской функции. Соединитель использует его позже, когда вызов завершается, при обратном вызове в Excel с помощью указателя на функцию, переданного в аргументе _каллбаккаддр_ . 
    
_Аргкаунт_
  
> Число аргументов, передаваемых в определенную пользователем функцию. Максимально допустимое значение — 255.
    
_Параметр1_
  
> Значение, передаваемое пользовательской функции. Повторите этот аргумент для каждого параметра, указанного с помощью _аргкаунт_.
    
## <a name="return-value"></a>Возвращаемое значение

**кслхпкретсукцесс** при успешной инициации вызова UDF; **кслхпкретинвалидсессионид** , если аргумент _SessionID_ является недопустимым; **кслхпкреткаллфаилед** для других сбоев, включая время ожидания. Если вызов возвращает код ошибки (все, кроме **кслхпкретсукцесс**), Excel считает, что не удалось выполнить вызов UDF, делает недействительным _пксасинчандле_и не ожидает выполнения обратного вызова.
  
## <a name="remarks"></a>Замечания

Эта функция выполняется асинхронно.
