# <a name="xlfregister-form-1"></a>xlfRegister (форма 1)

**Область применения:** Excel 2013 | Office 2013 | Visual Studio 
  
Может быть вызвано из команды DLL или XLL, которая сама была вызвана Microsoft Excel. Это равно вызову **REGISTER** из макрос Excel XLM. 
  
**xlfRegister** можно назвать в двух формах: 
  
- xlfRegister (Форма 1): регистрирует одну команду или функцию.
    
- [xlfRegister (Форма 2)](xlfregister-form-2.md): Загружает и активирует XLL.
    
Вызванная в форме 1, эта функция делает функцию DLL доступной для Excel, задает ее количество использования до 1 и возвращает ее регистрационный ID, который можно использовать для вызова функции позже с помощью [функции xlUDF](xludf.md) или **xlfCall.** Регистрационный номер также используется для отрегистрации функции [с помощью xlfUnregister (Form 1).](xlfunregister-form-1.md) Если функция зарегистрирована, вызов **xlfRegister** снова приумношает количество ее использования. 
  
Эта форма функции также определяет скрытое имя, которое является текстовым аргументом функции  _pxFunctionText_ и которое оценивается в регистрационный ID функции или команды. При отрегистрации функции удалите это имя с помощью [xlfSetName.](xlfsetname.md) Дополнительные сведения см. в статье [Известные проблемы, возникающие при разработке XLL для Excel](known-issues-in-excel-xll-development.md).
  
```cs
Excel12(xlfRegister, LPXLOPER12 pxRes, int iCount,
    LPXLOPER12 pxModuleText,   LPXLOPER12 pxProcedure,
    LPXLOPER12 pxTypeText,     LPXLOPER12 pxFunctionText,
    LPXLOPER12 pxArgumentText, LPXLOPER12 pxMacroType,
    LPXLOPER12 pxCategory,     LPXLOPER12 pxShortcutText,
    LPXLOPER12 pxHelpTopic,    LPXLOPER12 pxFunctionHelp,
    LPXLOPER12 pxArgumentHelp1, LPXLOPER12 pxArgumentHelp2,
        ...);
```

## <a name="parameters"></a>Parameters

_pxModuleText_ **(xltypeStr)**
  
Имя DLL, содержатую функцию. Это можно получить путем вызова функции [xlGetName](xlgetname.md) только для XLL, если зарегистрированная функция также находится в настоящее время выполняется DLL. 
  
_pxProcedure_ **(xltypeStr** или **xltypeNum)**
  
Если строка, имя функции, вызываемой в коде DLL. Если номер, ordinal export number of the function to call. Для ясности всегда используйте строковую форму.
  
_pxTypeText_ **(xltypeStr)**
  
Необязательная строка, которая указывает типы аргументов для функции и тип возвращаемого значения функции. Дополнительные сведения см. в разделе "Примечания". Этот аргумент может быть опущен для отдельной DLL (XLL), которая включает функцию [xlAutoRegister](xlautoregister-xlautoregister12.md) или **xlAutoRegister12**. 
  
> [!NOTE]
> **xlAutoRegister12** поддерживается только Excel 2007 г. 
  
Если **xlfRegister** вызывается с этим аргументом отсутствует, Excel **вызывает xlAutoRegister** или **xlAutoRegister12**, если либо существует в указанной DLL, которая должна правильно зарегистрировать функцию, предоставив эту информацию.
  
_pxFunctionText_ **(xltypeStr)**
  
Имя функции, которое появится в мастере функции. Этот аргумент необязателен; Если она не будет опущена, функция недоступна в мастере функции и может быть вызвана только с помощью функции **CALL** с помощью ИД регистрации функций из макрос листа XLM. Поэтому для обычного использования таблицы этот аргумент следует обрабатывать по мере необходимости. 
  
_pxArgumentText_ **(xltypeStr)**
  
Необязательная текстовая строка, описываемая аргументами для функции. Пользователь видит это в мастере функции. Если оно не будет пропущено, Excel основные описания из _pxTypeText_.

Начиная с Excel 2007 г., **xlfRegister** может принимать до 255 аргументов, чтобы вы могли предоставить эту помощь для 245 параметров функции.

Если регистрация прошла успешно, эта функция возвращает регистрационный номер функции **(xltypeNum),** который можно использовать в вызовах **xlUDF** и **xlfUnregister** в DLL или с **CALL** и **UNREGISTER** в макрослете XLM. В противном случае возвращается #VALUE! ошибка.

## <a name="additional-data-type-information"></a>Дополнительные сведения о типе данных

В этом разделе содержатся подробные сведения о типах данных **E,** **F,** **F%**, **G**, **G%,** **K,** **O,** **P,** **Q,** **R** и **U,** а также другие сведения о _аргументе pxTypeText._ 
  
### <a name="e-data-type"></a>Тип данных E

Excel ожидает, что DLL, использующий тип данных E, передает указатели на номера с плавающей точкой в стеке. Это может привести к проблемам с некоторыми языками (например, Borland C++), которые ожидают, что номер будет передан в стек эмулятора coprocessor. Обходное решение — передать указатель на номер в стеке coprocessor. В следующем примере показано, как вернуть дубль из Borland C++.
  
```cpp
typedef double * lpDbl;
extern "C" lpDbl __stdcall AddDbl(double D1,
    double D2, WORD npDbl)
{
    lpDbl Result;
    Result = (lpDbl)MK_FP(_SS, npDbl);
    *Result = D1 + D2;
    return (Result);
}
```

### <a name="f-f-g-and-g-data-types"></a>Типы данных F, F%, G и G%

С **типами данных F,** **F%,** **G** и **G%** функция может изменять буфер строки, выделенный Excel. Если код типа возвращаемого значения является одним из этих типов, Excel игнорирует возвращаемую функцией значение. Вместо этого Excel список аргументов функций для первого соответствующего типа данных **(F,** **F%**, **G** или **G%),** а затем принимает текущее содержимое выделенного буфера строки в качестве возвращаемого значения. Все версии Excel выделяют 256 bytes для строк **F** и **G** ASCII, а начиная с Excel 2007 г. выделяется 65 536 битов, что достаточно для 32 768 символов Unicode для строк Юникод **F%** и **G%** . Помните, что буферы должны включать символ графа (типы **G** и **G%)** или null-termination (типы **F** и **F%),** чтобы фактическая максимальная длина строки была 255 и 32 767. Строки Юникод и, следовательно, аргументы **типа F%** и **G%** доступны только через API C в Excel. 
  
### <a name="k-and-k-data-types"></a>Типы данных K и K%

Типы **данных K** и **K%** используют указатели для структур FP и FP12 размеров переменной соответственно. Эти структуры определены в XLLCALL.H. Структуры FP12 и, следовательно, аргументы **типа K%** поддерживаются только с Excel 2007 г. 
  
### <a name="o-and-o-data-types"></a>Типы данных O и O%

Типы **данных O** и **O%** можно использовать только для аргументов, а не для возвращающихся значений, хотя значения можно вернуть, изменяя аргумент типа **O** или **O%** на месте. Каждый из них передает три пункта: указатель на количество строк в массиве, указатель на количество столбцов в массиве и указатель на двумерный массив номеров с плавающей точкой. 
  
Чтобы изменить массив, переданный типом данных O или O% на месте, в качестве аргумента  _pxTypeText_ можно использовать ">O" или ">O%". Дополнительные сведения об изменении массива см. в разделе "Изменение на месте: функции, объявленные недействительными". 
  
Тип **данных O** был создан для прямой совместимости с DLLs Fortran, которые передают аргументы по ссылке. 
  
**O%** поддерживается начиная с Excel 2007 г. и вмещает большее число строк, Excel поддерживается. 
  
### <a name="p-and-q-data-types"></a>Типы данных P и Q

Когда аргументы функции DLL регистрируются как принимая тип **P** XLOPERs или тип **Q** XLOPER12, Excel преобразует одноклеточные ссылки на простые значения и многоклеточные ссылки на массивы при подготовке этих аргументов. Другими словами, типы **P** и **Q** всегда будут поступать в функцию как один из этих типов: **xltypeNum**, **xltypeStr**, **xltypeBool**, **xltypeErr**, **xltypeMulti**, **xltypeMissing**, или **xltypeNil**, но не **xltypeRef** или **xltypeSRef,** поскольку они всегда dereferenced. **Аргументы XLOPER12** и, следовательно, аргументы **типа Q** поддерживаются только с Excel 2007 г. 
  
Если типы **xltypeMissing** или **xltypeNil** используются для возвращающихся значений, они интерпретируются Excel как числовая ноль. **xltypeMissing** передается, когда вызывающий недопустит аргумент. **xltypeNil** передается, когда вызываемая передает ссылку на пустую ячейку. Когда диапазон ячеек преобразуется в **xltypeMulti,** который передается как тип **P** или **Q,** все пустые ячейки в диапазоне преобразуются в **элементы массива xltypeNil.** Отсутствующие элементы в буквальном массиве также передаются в **качестве элементов xltypeNil.** 
  
### <a name="volatile-functions-and-recalculation"></a>Летучие функции и пересчет

На таблице можно сделать функцию DLL или ресурс кода волатильной, чтобы она пересчитала каждый раз при пересчете таблицы. Для этого добавьте восклицательный знак (!) после последнего кода аргументов в _аргументе pxTypeText._ 
  
> [!NOTE]
> По умолчанию функции, которые принимают **тип R** XLOPERs или тип **U** XLOPER12 и регистрируются в качестве эквивалентов макросовых листов (тип, см. следующий раздел), обрабатываются как неустойчивые в **#** Excel. 
  
### <a name="functions-declared-as-void"></a>Функции, объявленные недействительными

Существует два случая, когда функция объявляется недействительной. В обоих случаях функция возвращает результат другими средствами.
  
#### <a name="modifying-in-place"></a>Изменение на месте

Для кода возвращаемого типа  _в_  _pxTypeText_ можно использовать одну цифру n, где  _n_ — это число от 1 до 9. Это предписывает Excel значение переменной в расположении, на который указывает аргумент __n_th pxTypeText_ в качестве возвращаемого значения. Это также называется изменением на месте. Аргумент _n_th должен быть типом данных для передачи данных (C, D, E, F, F%, G, G%, K, K%, L, M, N, O, O%, P, Q, R или U). Функция DLL или ресурс кода также  должны быть объявлены с пустым ключевым  словом на языках C/C++ (или ключевое слово процедуры на паскальском языке). 
  
Например, функция DLL, которая принимает строку с null-terminated и две указатели для integers, так как аргументы могут изменить строку на месте. Используйте "1FMM" в качестве  _аргумента pxTypeText_ и объявите функцию недействительной. 
  
Предыдущие версии Excel, используемые в начале **\>** _pxTypeText,_ чтобы уликать, что функция была объявлена недействительной и что первый аргумент должен быть изменен на месте, не было способа изменить любой аргумент, кроме первого. Эквивалент n = 1 в текущих версиях Excel и это использование синхронных функций поддерживается только для обратной **\>**  **\>** совместимости. 
  
#### <a name="asynchronous-functions"></a>Асинхронные функции

Асинхронная функция, обозначаемая с помощью параметра типа X в **pxTypeText,** не возвращает результат от начального вызова функции. Вместо этого необходимо объявить асинхронную функцию недействительной, а затем надстройка возвращает результат с помощью обратного вызова. Асинхронная функция должна быть зарегистрирована с помощью в **\>** начале **pxTypeText.** В асинхронных функциях указывается, что функция объявляется недействительной, но не указывает, что первый аргумент **\>** изменен на месте. Дополнительные сведения об асинхронных функциях см. в [User-Defined.](asynchronous-user-defined-functions.md) 

### <a name="registering-worksheet-functions-as-macro-sheet-equivalents-handling-uncalculated-cells"></a>Регистрация функций листа в качестве эквивалентов макрос листа (обработка нерасчетных ячеек)

Размещение **#** символа после последнего кода параметров  _в pxTypeText_ дает функции те же разрешения на вызов, что и функции на макросхеме. Эти данные следуют следующим образом: 
  
- Функция может получить значения ячеек, которые еще не были рассчитаны в этом цикле пересчета.
    
- Функция может вызывать любые функции XLM-информации (класс 2), например **xlfGetCell.**
    
- Если знака номера (#) нет: оценка нерасчитаемой ячейки приводит к ошибке **xlretUncalced,** и текущая функция снова вызвана после вычисления ячейки; вызов любой функции XLM, кроме **xlfCaller,** приводит к ошибке **xlretInvXlfn.** 
    
### <a name="registering-worksheet-functions-as-thread-safe"></a>Регистрация функций таблицы в качестве безопасной для потоков

Начиная с Excel 2007 г. Excel могут выполнять многоуровневый пересчет книг. Это означает, что он может назначать различные экземпляры функции, безопасной для потоков, для одновременного переозначения потоков. Начиная с Excel 2007 г. большинство встроенных функций таблицы безопасны для потоков. Начиная с Excel 2007 г. Excel XLLs также позволяет регистрировать функции таблицы как безопасные потоки. Для этого включите **$** символ после последнего кода параметров _в pxTypeText._ 
  
> [!NOTE]
> Только функции таблицы можно объявлять безопасными для потоков. Excel не считает функцию, эквивалентную макросам, безопасной для потока, так что нельзя примесить и символы к аргументу **#** **$** _pxTypeText._ 
  
Если вы зарегистрировали функцию в качестве безопасной для потоков, необходимо убедиться, что она ведет себя безопасно, хотя Excel отклоняет любые небезопасные для потоков вызовы через API C. Например, если функция, безопасная для потоков, пытается вызвать **xlfGetCell,** вызов не удается с **ошибкой xlretNotThreadSafe.** 
  
### <a name="registering-worksheet-functions-as-cluster-safe"></a>Регистрация функций таблицы в качестве безопасного кластера

Начиная с Excel 2010 г., Excel может разгрузить вызовы функций назначенного поставщика вычислительных кластеров. Дополнительные сведения см. в [Сейф Cluster.](cluster-safe-functions.md) Все функции таблицы XLL, зарегистрированные как безопасные для кластера, принимают участие в разгрузке, если кластер доступен. Функции, безопасные для кластера, регистрируются путем включив символ после последнего кода **&amp;** параметров в _аргументе pxTypeText._ 
  
Если вы зарегистрировали функцию в качестве безопасной для кластера, необходимо убедиться, что она ведет себя безопасно кластерно. Дополнительные сведения см. в [Сейф Cluster.](cluster-safe-functions.md)
  
> [!NOTE]
> Только функции таблицы можно объявлять кластерными. Excel не считает функцию, эквивалентную макрос листам, безопасной для кластера, так что к аргументу **#** **&amp;** _pxTypeText_ нельзя примесить как символы, так и символы. Функции таблицы можно объявлять как кластеробезопасные, так и нитки. В этом случае Excel эти функции будут принимать участие в многопрофиленном пересчете при отключении отгрузки кластера. 
  
### <a name="category-names"></a>Имена категорий

Чтобы определить, в какую категорию помещались функции XLL, используйте следующие рекомендации.
  
- Если функция делает что-то, что может быть сделано пользователем в составе пользовательского интерфейса надстройки, следует поместить функцию в категорию **Команд.** 
    
- Если функция возвращает сведения о состоянии надстройки или любых других полезных сведений, следует поместить функцию в **категорию "Информация".** 
    
- Надстройка никогда не должна добавлять функции или команды в **категорию User Defined.** Эта категория используется для исключительного использования конечных пользователей. 
    
Категория указывается с помощью параметра _pxCategory_ **в xlfRegister.** Это может быть номер или текст, соответствующий одной из стандартных категорий с жесткой кодом, или текст новой категории, заданной DLL. Если данный текст еще не существует, Excel создает новую категорию с этим именем.
