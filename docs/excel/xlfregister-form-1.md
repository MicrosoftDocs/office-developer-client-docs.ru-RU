# <a name="xlfregister-form-1"></a>xlfRegister (форма 1)

**Область применения:** Excel 2013 | Office 2013 | Visual Studio 
  
Может вызываться из команды DLL или XLL, которая вызывается Microsoft Excel. Это значение соответствует **регистру** звонков из листа макросов Microsoft Office XLM. 
  
**xlfRegister** можно вызывать в двух формах: 
  
- xlfRegister (форма 1): регистрирует одну команду или функцию.
    
- [xlfRegister (форма 2)](xlfregister-form-2.md): Загрузка и активация XLL-модуля.
    
Функция, вызываемая в форме 1, делает функцию или команду DLL доступной для Excel, устанавливает значение 1 и возвращает идентификатор регистрации, который можно использовать для вызова функции позже с помощью функции [xlUDF](xludf.md) или **кслфкалл** . ИДЕНТИФИКАТОР регистрации также используется для отмены регистрации функции с помощью [xlfUnregister (форма 1)](xlfunregister-form-1.md). Если функция была зарегистрирована, вызов **xlfRegister** еще раз увеличивает значение счетчика использования. 
  
Эта форма функции также определяет скрытое имя, которое является аргументом Text функции, _пксфунктионтекст_и определяет идентификатор регистрации функции или команды. При отмене регистрации функции удалите это имя с помощью [xlfSetName](xlfsetname.md). Дополнительные сведения см. в статье [Известные проблемы, возникающие при разработке XLL для Excel](known-issues-in-excel-xll-development.md).
  
```cs
Excel12(xlfRegister, LPXLOPER12 pxRes, int iCount,
    LPXLOPER12 pxModuleText,   LPXLOPER12 pxProcedure,
    LPXLOPER12 pxTypeText,     LPXLOPER12 pxFunctionText,
    LPXLOPER12 pxArgumentText, LPXLOPER12 pxMacroType,
    LPXLOPER12 pxCategory,     LPXLOPER12 pxShortcutText,
    LPXLOPER12 pxHelpTopic,    LPXLOPER12 pxFunctionHelp,
    LPXLOPER12 pxArgumentHelp1, LPXLOPER12 pxArgumentHelp2,
        ...);
```

## <a name="parameters"></a>Параметры

_пксмодулетекст_ (**кслтипестр**)
  
Имя библиотеки DLL, которая содержит функцию. Его можно получить, вызвав функцию [кслжетнаме](xlgetname.md) только XLL, если функция зарегистрирована также в текущей ИСПОЛНЯЕМОЙ библиотеке DLL. 
  
_пкспроцедуре_ (**кслтипестр** или **кслтипенум**)
  
Если строка, имя вызываемой функции отображается в коде DLL. Если число, порядковый номер экспорта вызываемой функции. Для ясности всегда используйте форму String.
  
_пкстипетекст_ (**кслтипестр**)
  
Необязательная строка, указывающая типы аргументов функции и тип возвращаемого значения функции. Дополнительные сведения см. в разделе "Примечания". Этот аргумент можно опустить для автономной библиотеки DLL, которая включает [функцию xlAutoRegister](xlautoregister-xlautoregister12.md) или **xlAutoRegister12**. 
  
> [!NOTE]
> **xlAutoRegister12** поддерживается только в Excel 2007. 
  
Если **xlfRegister** вызывается с отсутствующим аргументом, Excel вызывает **xlAutoRegister** или **xlAutoRegister12**, если он существует в указанной библиотеке DLL, который затем правильно зарегистрирует функцию, предоставив эту информацию.
  
_пксфунктионтекст_ (**кслтипестр**)
  
Имя функции, как оно будет отображаться в мастере функций. Этот аргумент является необязательным; Если он не указан, функция недоступна в мастере функций и может вызываться только с помощью функции **вызвать** с помощью идентификатора регистрации функций на листе макросов XLM. Таким образом, для обычной работы с листом этот аргумент должен обрабатываться по мере необходимости. 
  
_пксаргументтекст_ (**кслтипестр**)
  
Необязательная текстовая строка, описывающая аргументы для функции. Пользователь видит это в мастере функций. Если он не указан, Excel создает основные описания из _пкстипетекст_.
  
_пксмакротипе_ (**кслтипенум** или **кслтипеинт**)
  
Необязательный аргумент, указывающий тип точки входа XLL. Значение по умолчанию, если оно опущено, равно 1.

При использовании формы URL-адреса Excel открывает только файл справки, на который указывает ссылка. 
  
_пксфунктионхелп_ (**кслтипестр**)
  
Необязательная строка, описывающая настраиваемую функцию при ее выборе в мастере функций.
  
_pxArgumentHelp1_ (**кслтипестр**)
  
Необязательное. Первая из строк, описывающих пользовательские аргументы функции при выборе функции в мастере функций. Начиная с Excel 2007, **xlfRegister** может задействовать до 255 аргументов, чтобы можно было предоставить эту справку до 245 параметров функции. 
  
## <a name="property-valuereturn-value"></a>Значение свойства и возвращаемое значение

Если регистрация прошла успешно, эта функция возвращает идентификатор регистра функции (**кслтипенум**), который можно использовать в вызовах **xlUDF** и **xlfUnregister** в DLL, а также с **вызовом** и **отменой регистрации** на листе макросов XLM. В противном случае возвращается #VALUE! ошибкой.

## <a name="additional-data-type-information"></a>Дополнительные сведения о типах данных

В этом разделе содержатся подробные сведения о типах данных **E**, **f**, **f%**, **g**, **g%**, **K**, **O**, **P**, **Q**, **R**и **U** , а также другие сведения об аргументе _пкстипетекст_ . 
  
### <a name="e-data-type"></a>Тип данных E

Excel ожидает библиотеку DLL, использующую тип данных E, для передачи указателей на числа с плавающей точкой в стеке. Это может привести к проблемам с некоторыми языками (например, Borland C++), которые ожидают, что число передается в стек эмуляторов сопроцессора. Временное решение заключается в передаче указателя на номер в стеке сопроцессора. В приведенном ниже примере показано, как вернуть удвоенное значение из Borland C++.
  
```cpp
typedef double * lpDbl;
extern "C" lpDbl __stdcall AddDbl(double D1,
    double D2, WORD npDbl)
{
    lpDbl Result;
    Result = (lpDbl)MK_FP(_SS, npDbl);
    *Result = D1 + D2;
    return (Result);
}
```

### <a name="f-f-g-and-g-data-types"></a>Типы данных f, F%, G и G%

С помощью типов данных **f**, **f%**, **g**и **g%** функция может изменять буфер строк, выделенный Excel. Если код типа возвращаемого значения является одним из этих типов, Excel игнорирует значение, возвращенное функцией. Вместо этого Excel ищет в списке аргументов функции первый соответствующий тип данных (**f**, **f%**, **g**или **G%**), а затем получает текущее содержимое выделенного буфера строк в качестве возвращаемого значения. Все версии Excel распределяют 256 **байт для строк** ASCII и **G** ASCII, а затем начиная с версии Excel 2007 65 536 байт выделяется, достаточно для 32 768 символов Юникода для строк **F%** и **G%** Юникод. Помните, что буферы должны включать символ подсчета (Types **g** и **g%**) или символ завершения (Types **f** и **f%**), чтобы максимальная длина строк 255 и 32 767. Строки Юникода, поэтому типы **F%** и **G%** , доступны только через API C в Excel. 
  
### <a name="k-and-k-data-types"></a>Типы данных k и K%

Типы данных **k** и **k%** используют указатели на структуры FP и FP12 с переменным размером, соответственно. Эти структуры определены в КСЛЛКАЛЛ. Высоты. Структуры FP12, поэтому Types **K%** аргументов, поддерживаются только начиная с Excel 2007. 
  
### <a name="o-and-o-data-types"></a>Типы данных o и O%

Типы данных **o** и **o%** можно использовать только для аргументов, а не возвращаемых значений, несмотря на то, что значения могут возвращать значение аргумента типа **o** или **o%** на месте. Каждый из них передает три элемента: указатель на число строк в массиве, указатель на число столбцов в массиве и указатель на двухмерный массив чисел с плавающей запятой. 
  
Чтобы изменить массив, передаваемый с помощью типа данных O или O%, можно использовать в качестве аргумента _пкстипетекст_ ">O" или ">o%". Для получения дополнительных сведений об изменении массива обратитесь к разделу "изменение на месте: функции, объявленные как void" этой статьи. 
  
Тип данных **O** был создан для непосредственной совместимости с библиотеками Fortran, которые передают аргументы по ссылке. 
  
Параметр **O%** поддерживается начиная с версии Excel 2007 и поддерживает большее количество строк, поддерживаемых Excel. 
  
### <a name="p-and-q-data-types"></a>Типы данных P и Q

Когда аргументы функции DLL регистрируются как принимающий тип **P** XLOPER или тип **Q** XLOPER12, Excel преобразует ссылки на отдельные ячейки в простые значения и ссылки на несколько ячеек в массивах при подготовке этих аргументов. Другими словами, типы **P** и **Q** всегда будут присвоены функции в качестве одного из следующих типов: **кслтипенум**, **кслтипестр**, **кслтипебул**, **кслтипирр**, **xltypeMulti**, **кслтипемиссинг**или **кслтипенил**, но не **кслтипереф** или **кслтипесреф** , так как они всегда разыменованы. **XLOPER12**s, поэтому аргументы **Q** поддерживаются только начиная с Excel 2007. 
  
Если для возвращаемых значений используются типы **кслтипемиссинг** или **кслтипенил** , они интерпретируются в Excel как цифры нуля. **кслтипемиссинг** передается, когда вызывающий объект опускает аргумент. **кслтипенил** передается, когда вызывающий абонент передает ссылку на пустую ячейку. Когда диапазон ячеек преобразуется в **xltypeMulti** для передачи в качестве типа **P** или **Q**, все пустые ячейки в диапазоне преобразуются в элементы массива **кслтипенил** . Отсутствующие элементы в массиве литералов аналогично передаются в виде элементов **кслтипенил** . 
  
### <a name="volatile-functions-and-recalculation"></a>Временные функции и пересчет

На листе можно сделать функцию DLL или ресурс кода временной, чтобы она пересчиталась каждый раз, когда лист пересчитывается. Для этого добавьте восклицательный знак (!) после кода последнего аргумента в аргументе _пкстипетекст_ . 
  
> [!NOTE]
> По умолчанию функции, которые принимают тип " **R** XLOPER" или " **U** XLOPER12" и регистрируются как эквиваленты листа макросов **#**(тип; см. следующий раздел), обрабатываются как переменные в Excel. 
  
### <a name="functions-declared-as-void"></a>Функции, объявленные как void

При объявлении функции как возвращающей значение void существует два случая. В обоих случаях функция возвращает результат другими способами.
  
#### <a name="modifying-in-place"></a>Изменение на месте

Можно использовать одну цифру _n_ для кода типа возврата в _пкстипетекст_, где _n_ — число от 1 до 9. Это указывает Excel на получение значения переменной в расположении, на которое указывает аргумент _n_th в _пкстипетекст_ в качестве возвращаемого значения. Это также называется изменением на месте. Аргументом _n_th должен быть тип данных, который передается по ссылке (C, D, E, F, F%, G, G%, K, K%, L, M, N, O, O%, P, Q, R или U). Функция DLL или ресурс кода также должны быть объявлены с помощью ключевого слова **void** в языках C/C++ (или в ключевом слове **PROCEDURE** языка Pascal). 
  
Например, функция DLL, которая принимает строку, заканчивающуюся нулем, и два указателя на целые числа в качестве аргументов, могут изменять строку на месте. Используйте "1FMM" в качестве аргумента _пкстипетекст_ и объявите функцию как void. 
  
Предыдущие версии Excel использовались **\>** в начале _пкстипетекст_ , чтобы показать, что функция была объявлена как void и что первый аргумент был изменен на месте, не существовал способ изменить любой аргумент, кроме первого. Эквивалентно _n_ = 1 в текущей версии Excel, и это использование **\>** в синхронных функциях поддерживается только для обратной совместимости. **\>** 
  
#### <a name="asynchronous-functions"></a>Асинхронные функции

Асинхронная функция, обозначенная с помощью параметра типа X в **пкстипетекст**, не возвращает результат из начального вызова функции. Вместо этого необходимо объявить асинхронную функцию как void, а затем надстройка возвращает результат через обратный вызов. Асинхронную функцию необходимо зарегистрировать с помощью **\>** параметра в начале **пкстипетекст**. В асинхронных функциях **\>** указывает, что функция объявлена как void, но не указывает на то, что первый аргумент изменяется на месте. Более подробную информацию об асинхронных функциях можно узнать в статье [Асинхронные пользовательские функции](asynchronous-user-defined-functions.md). 

### <a name="registering-worksheet-functions-as-macro-sheet-equivalents-handling-uncalculated-cells"></a>Регистрация функций листа в качестве эквивалентов листа макросов (обработка невычисляемых ячеек)

Помещение **#** символа после последнего кода параметра в _пкстипетекст_ дает функции те же разрешения вызова, что и функции на листе макросов. Ниже приведены некоторые из них. 
  
- Функция может извлекать значения ячеек, которые еще не были вычислены в цикле пересчета.
    
- Функция может вызывать любую из функций информации о XLM (класс 2), например **например xlfGetCell**.
    
- Если знак номера (#) отсутствует: Оценка невычисляемой ячейки приводит к ошибке **кслретункалцед** , а текущая функция вызывается повторно после вычисления ячейки; вызов любой функции данных XLM, отличной от **xlfCaller** , приводит к ошибке **кслретинвкслфн** . 
    
### <a name="registering-worksheet-functions-as-thread-safe"></a>Регистрация функций листа в качестве потокобезопасных

Начиная с Excel 2007, Excel может выполнять пересчет многопотоковых книг. Это означает, что он может назначить различные экземпляры потокобезопасной функции параллельным потокам для повторной оценки. Начиная с Excel 2007, большая часть встроенных функций листа является потокобезопасным. Начиная с Excel 2007, Excel также позволяет Xlls регистрировать функции листа как потокобезопасные. Для этого добавьте **$** символ после последнего кода параметра в _пкстипетекст_. 
  
> [!NOTE]
> Только функции листа могут объявляться как потокобезопасные. Приложение Excel не считает эквивалентную функцию листа макросов потокобезопасной, поэтому в аргумент **#** _пкстипетекст_ нельзя добавить оба символа и **$** . 
  
Если функция была зарегистрирована как безопасная для потоков, необходимо убедиться, что она работает потокобезопасным способом, хотя Excel отклоняет вызовы с небезопасными потоками через API C. Например, если потокобезопасная функция пытается вызвать **например xlfGetCell**, вызов завершается с ошибкой **кслретнотсреадсафе** . 
  
### <a name="registering-worksheet-functions-as-cluster-safe"></a>Регистрация функций электронной таблицы в качестве надежного кластера

Начиная с Excel 2010, Excel может разгрузить вызовы функций на назначенный поставщик вычислительных кластеров. Дополнительные сведения можно найти в статье [безопасные функции кластера](cluster-safe-functions.md). Все функции листа XLL, зарегистрированные как безопасные кластеры, участвуют в разгрузке, если кластер доступен. Функции, защищенные кластером, регистрируются **&amp;** путем включения знака после последнего кода параметра в аргументе _пкстипетекст_ . 
  
Если вы зарегистрировали функцию как безопасную для кластеризации, необходимо убедиться в том, что она работает в кластерной безопасном способе. Дополнительные сведения можно найти в статье [безопасные функции кластера](cluster-safe-functions.md).
  
> [!NOTE]
> Только функции листа можно объявить безопасными в кластере. Excel не рассматривает функцию, эквивалентную кластеризации, для безопасного кластера, поэтому в аргумент _пкстипетекст_ нельзя добавить **#** оба **&amp;** символа и. Функции листов могут быть объявлены как безопасные как кластеризация, так и потокобезопасные. В этом случае Excel позволит этим функциям принимать участие в многопоточном пересчете при отключении разгрузки кластера. 
  
### <a name="category-names"></a>Имена категорий

Чтобы определить категорию, в которой нужно разместить функции XLL, следуйте приведенным ниже рекомендациям.
  
- Если функция выполняет действия, выполняемые пользователем в составе пользовательского интерфейса надстройки, следует поместить функцию в категорию **Commands (команды** ). 
    
- Если функция возвращает сведения о состоянии надстройки или другие полезную информацию, следует поместить функцию в категорию **Information** . 
    
- Надстройка никогда не должна добавлять функции или команды в **определенную пользователем** категорию. Эта категория предназначена для монопольного использования конечных пользователей. 
    
Категория указывается с помощью параметра _пкскатегори_ для **xlfRegister**. Это может быть число или текст, соответствующий одной из жестких запрограммированных стандартных категорий, или тексту новой категории, заданной библиотекой DLL.
