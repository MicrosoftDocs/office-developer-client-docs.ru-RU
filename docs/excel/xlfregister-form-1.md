# <a name="xlfregister-form-1"></a>xlfRegister (форма 1)

**Область применения:** Excel 2013 | Office 2013 | Visual Studio 
  
Может быть вызван из команды DLL или XLL, которая сама была вызвана Microsoft Excel. Это равноценно вызову **REGISTER** из листа макроса XLM в Excel. 
  
**XlfRegister** может быть вызван в двух формах: 
  
- xlfRegister (форма 1): регистрирует одну команду или функцию.
    
- [xlfRegister (форма 2):](xlfregister-form-2.md)загружает и активирует XLL.
    
Вызываемая в форме 1, эта функция делает функцию или команду DLL доступной для Excel, устанавливает для нее количество использования 1 и возвращает ее ид регистрации, который можно использовать для вызова функции позже с помощью [функции xlUDF](xludf.md) или **xlfCall.** Кроме того, для отрегистрации функции с помощью [xlfUnregister (форма 1)](xlfunregister-form-1.md)используется регистрационный ИД. Если функция зарегистрирована, вызов **xlfRegister** еще раз добавит количество ее использования. 
  
Эта форма функции также определяет скрытое имя, которое является текстовым аргументом функции  _pxFunctionText_ и оценивается по зарегистрированным ИД функции или команды. При отостановке регистрации функции удалите это имя с помощью [xlfSetName.](xlfsetname.md) Дополнительные сведения см. в статье [Известные проблемы, возникающие при разработке XLL для Excel](known-issues-in-excel-xll-development.md).
  
```cs
Excel12(xlfRegister, LPXLOPER12 pxRes, int iCount,
    LPXLOPER12 pxModuleText,   LPXLOPER12 pxProcedure,
    LPXLOPER12 pxTypeText,     LPXLOPER12 pxFunctionText,
    LPXLOPER12 pxArgumentText, LPXLOPER12 pxMacroType,
    LPXLOPER12 pxCategory,     LPXLOPER12 pxShortcutText,
    LPXLOPER12 pxHelpTopic,    LPXLOPER12 pxFunctionHelp,
    LPXLOPER12 pxArgumentHelp1, LPXLOPER12 pxArgumentHelp2,
        ...);
```

## <a name="parameters"></a>Параметры

_pxModuleText_ (**xltypeStr)**
  
Имя DLL-имени, содержа имени функции. Это можно получить путем вызова функции [xlGetName](xlgetname.md) только xLL, если зарегистрированная функция также находится в исполняемой DLL. 
  
_pxProcedure_ (**xltypeStr** или **xltypeNum)**
  
Если строка, имя вызываемой функции, которая отображается в коде DLL. Если номер, порядковая экспортируемая номер вызываемой функции. Для ясности всегда используйте строковую форму.
  
_pxTypeText_ (**xltypeStr)**
  
Необязательная строка, которая указывает типы аргументов функции и тип возвращаемого значения функции. Дополнительные сведения см. в разделе "Примечания". Этот аргумент можно о пропущен для отдельной DLL(XLL), которая включает функцию [xlAutoRegister](xlautoregister-xlautoregister12.md) или **xlAutoRegister12.** 
  
> [!NOTE]
> **XlAutoRegister12** поддерживается только в Excel 2007. 
  
Если **вызывается xlfRegister** с отсутствием этого аргумента, Excel вызывает **xlAutoRegister** или **xlAutoRegister12,** если в указанной DLL есть функция, которая затем должна правильно зарегистрировать функцию, предоставив эти сведения.
  
_pxFunctionText_ (**xltypeStr)**
  
Имя функции, которое будет отображаться в мастере функций. Этот аргумент является необязательным; Если он опущен, функция недоступна в мастере функций и может вызываться только с помощью функции **CALL,** используя ИД регистрации функций из листа макроса XLM. Поэтому для обычного использования таблицы этот аргумент следует обрабатывать по мере необходимости. 
  
_pxArgumentText_ (**xltypeStr)**
  
Необязательная текстовая строка, описываемая аргументы функции. Пользователь видит это в мастере функций. Если он опущен, Excel составляет основные описания _из pxTypeText._

Необязательный аргумент, который указывает тип точки входа XLL. Значение по умолчанию, если оно опущено, — 1.

Необязательный параметр. Первая строка, описывая настраиваемые аргументы функции при выборе функции в мастере функций. Начиная с Excel 2007, **xlfRegister** может принимать до 255 аргументов, чтобы вы могли предоставлять эту справку для 245 параметров функций.

## <a name="additional-data-type-information"></a>Дополнительные сведения о типе данных

В этом разделе содержатся подробные сведения о типах данных **E,** **F,** **F%**, **G,** **G%**, **K,** **O,** **P,** **Q,** **R** и **U,** а также другие сведения о аргументе _pxTypeText._ 
  
### <a name="e-data-type"></a>Тип данных E

Excel ожидает, что DLL с использованием типа данных E будет передавать указатели на числа с плавающей точкой в стеке. Это может вызвать проблемы с некоторыми языками (например, Beland C++), которые ожидают, что номер будет передан в стек эмулятора coprocessor. Обходным решением является передать указатель на номер в стеке coprocessor. В следующем примере показано, как вернуть двойной номер из Острова C++.
  
```cpp
typedef double * lpDbl;
extern "C" lpDbl __stdcall AddDbl(double D1,
    double D2, WORD npDbl)
{
    lpDbl Result;
    Result = (lpDbl)MK_FP(_SS, npDbl);
    *Result = D1 + D2;
    return (Result);
}
```

### <a name="f-f-g-and-g-data-types"></a>Типы данных F, F%, G и G%

С помощью **типов данных F,** **F%**, **G** и **G%** функция может изменять буфер строк, выделенный Excel. Если код типа возвращаемого значения является одним из этих типов, Excel игнорирует значение, возвращаемого функцией. Вместо этого Excel выполняет поиск первого соответствующего типа данных в списке аргументов функции **(F,** **F%**, G или **G%),** а затем принимает текущее содержимое выделенного буфера строк в качестве возвращаемого значения.  Все версии Excel выделяют 256 bytes для **строк F** и **G** ASCII, а начиная с Excel 2007 выделяется 65 536 bytes, достаточно для 32 768 символов Юникода, для **строк F%** и **G%** Юникода. Помните, что буферы должны включать знак подсчета (типы **G** и **G%)** или знак завершения null (типы **F** и **F%),** чтобы фактическая максимальная длина строки составляет 255 и 32 767. Строки Юникода и, следовательно, введите **аргументы F%** и **G%,** доступны только через API C в Excel. 
  
### <a name="k-and-k-data-types"></a>Типы данных K и K%

Типы **данных K** и **K%** используют указатели на структуры FP и FP12 переменного размера соответственно. Эти структуры определены в XLLCALL.H. Структуры FP12 и, следовательно, типы **аргументов K%** поддерживаются только начиная с Excel 2007. 
  
### <a name="o-and-o-data-types"></a>Типы данных O и O%

Типы **данных O** и **O%** можно использовать только для аргументов, а не для возвращаемого значения, хотя значения могут быть возвращены при изменении аргумента типа **O** или **O%** на месте. Каждый из них передает три пункта: указатель на количество строк в массиве, указатель на число столбцов в массиве и указатель на двумерный массив чисел с плавающей за точкой. 
  
Чтобы изменить массив, переданный типом данных O или O%, можно использовать в качестве аргумента  _pxTypeText_ ">O" или ">O%". Дополнительные сведения об изменении массива см. в разделе "Изменение на месте: функции, объявленные как void" этой статьи. 
  
Тип **данных O** был создан для прямой совместимости с DLL Для Fortran, которые передают аргументы по ссылке. 
  
**O%** поддерживается начиная с Excel 2007 и поддерживает большее количество строк, поддерживаемых Excel. 
  
### <a name="p-and-q-data-types"></a>Типы данных P и Q

Когда аргументы функции DLL регистрируются как принимая тип **P** XLOPERs или **Q** XLOPER12s, Excel преобразует ссылки на одно ячейки в простые значения и многоуровневые ссылки на массивы при подготовке этих аргументов. Другими словами, типы **P** и **Q** всегда поступают в функцию как один из этих типов: **xltypeNum,** **xltypeStr**, **xltypeBool**, **xltypeErr**, **xltypeMulti,** **xltypeMissing** или **xltypeNil**, но не **xltypeRef** или **xltypeSRef,** так как они всегда dereferenced. **Аргументы XLOPER12** и, следовательно, типы **аргументов Q** поддерживаются только начиная с Excel 2007. 
  
Если типы **xltypeMissing** или **xltypeNil** используются для возвращаемого значения, они интерпретируются Excel как числовая ноль. **XltypeMissing** передается, когда вызывающий опустить аргумент. **XltypeNil** передается, когда звонящие передают ссылку на пустую ячейку. Когда диапазон ячеек преобразуется в **XltypeMulti,** который передается в виде типа **P** или **Q,** все пустые ячейки в диапазоне преобразуются в элементы массива **xltypeNil.** Отсутствующие элементы в массиве литералов аналогично передаются в качестве **элементов xltypeNil.** 
  
### <a name="volatile-functions-and-recalculation"></a>Переменные функции и пересчет

На этом сайте можно сделать функцию DLL или ресурс кода переменной, чтобы она пересчитывался каждый раз при пересчете. Для этого добавьте восклицательный знак (!) после кода последнего аргумента в _аргументе pxTypeText._ 
  
> [!NOTE]
> По умолчанию функции, которые принимают тип **R** XLOPERs или U XLOPER12 и зарегистрированы как эквиваленты листа макроса (тип ; см. следующий раздел), обрабатываются в Excel как  переменные. **#** 
  
### <a name="functions-declared-as-void"></a>Функции, объявленные как void

Существует два случая, когда функция объявляется как возвращаемая void. В обоих случаях функция возвращает результат другими способами.
  
#### <a name="modifying-in-place"></a>Изменение на месте

Вы можете использовать одну цифру  _n_ для кода возвращаемого типа в  _pxTypeText,_ где  _n_ — это число от 1 до 9. Это указывает Excel принимать значение переменной в расположении, на который указывает аргумент _n_th  _в pxTypeText_ в качестве возвращаемого значения. Это также называется изменением на месте. Аргументом _n_th должен быть тип данных "по ссылке" (C, D, E, F, F%, G, G%, K, K%, L, M, N, O, O%, P, Q, R или U). Функцию DLL или ресурс кода также необходимо объявлять с помощью ключевого  слова **void** на языках C/C++ (или ключевого слова процедуры на языке Pascal). 
  
Например, функция DLL, которая принимает строку с завершенным нулом и два указателя на integers в качестве аргументов, может изменять строку на месте. Используйте "1FMM" в качестве  _аргумента pxTypeText_ и объявите функцию как void. 
  
Предыдущие версии Excel, которые использовались в начале **\>**  _pxTypeText_ для знака того, что функция объявлена как void и что первый аргумент должен был быть изменен на месте, не существовало способа изменить любой аргумент, кроме первого. Эквивалент n = 1 в текущих версиях Excel, и это использование синхронных функций поддерживается только для обратной **\>**  **\>** совместимости. 
  
#### <a name="asynchronous-functions"></a>Асинхронные функции

Асинхронная функция, обозначаемая с помощью параметра типа X в **pxTypeText,** не возвращает результат исходного вызова функции. Вместо этого необходимо объявить асинхронную функцию как void, а затем надстройка возвращает результат с помощью обратного вызова. Асинхронная функция должна быть зарегистрирована с помощью в начале **\>** **pxTypeText.** В асинхронных функциях означает, что функция объявлена как void, но не указывает, что первый аргумент изменен **\>** на месте. Дополнительные сведения об асинхронных функциях см. в [User-Defined.](asynchronous-user-defined-functions.md) 

### <a name="registering-worksheet-functions-as-macro-sheet-equivalents-handling-uncalculated-cells"></a>Регистрация функций листа в качестве эквивалентов листа макроса (обработка невычисляемых ячеек)

Размещение знака после кода последнего **#** параметра  _в pxTypeText_ предоставляет функции те же разрешения вызова, что и функции на листе макроса. Эти данные следуют следующим образом: 
  
- Функция может получить значения ячеек, которые еще не были рассчитаны в этом цикле пересчета.
    
- Функция может вызывать любые функции сведений XLM (класс 2), например **xlfGetCell.**
    
- Если знака номера (#) нет: оценка невычисляемой ячейки приводит к ошибке **xlretUncalced,** и текущая функция снова будет вызвана после вычисления ячейки; Вызов любой информационной функции XLM, кроме **xlfCaller,** приводит к ошибке **xlretInvXlfn.** 
    
### <a name="registering-worksheet-functions-as-thread-safe"></a>Регистрация функций таблицы в качестве потокобезопасной

Начиная с Excel 2007, Excel может выполнять многопочитанные пересчеты книг. Это означает, что она может назначать разные экземпляры потокобезопасной функции для параллельного потока для переоверки. Начиная с Excel 2007, большинство встроенных функций листа потокобезопасны. Начиная с Excel 2007, Excel также позволяет XLL регистрировать функции листа как потокобезопасные. Для этого включите **$** символ после кода последнего параметра _в pxTypeText._ 
  
> [!NOTE]
> Потокобезопасные можно объявлять только функции таблиц. Excel не считает функцию, эквивалентную листу макроса, потокобезопасной, поэтому к аргументу **#** **$**  _pxTypeText_ нельзя одновременно и символы. 
  
Если вы зарегистрировали функцию как потокобезопасную, необходимо убедиться, что она работает потокобезопасно, хотя Excel отклоняет все потокобезопасные вызовы через API C. Например, если потокобезопасная функция пытается вызвать **xlfGetCell,** вызов не удастся с ошибкой **xlretNotThreadSafe.** 
  
### <a name="registering-worksheet-functions-as-cluster-safe"></a>Регистрация функций таблицы как безопасных в кластере

Начиная с Excel 2010, Excel может разгружать вызовы функций для назначенного поставщика вычислительных кластеров. Дополнительные сведения см. в [теме "Безопасные функции кластера".](cluster-safe-functions.md) Все функции таблиц XLL, зарегистрированные как безопасные для кластера, принимают участие в разгрузке, если кластер доступен. Функции, безопасные для кластера, регистрируются с помощью символа после последнего кода **&amp;** параметра в _аргументе pxTypeText._ 
  
Если вы зарегистрировали функцию как кластеробезопасную, необходимо убедиться, что она работает безопасным для кластера образом. Дополнительные сведения см. в [теме "Безопасные функции кластера".](cluster-safe-functions.md)
  
> [!NOTE]
> Только функции таблицы можно объявлять как безопасные в кластере. Excel не считает функцию, эквивалентную листу макроса, кластеробезопасной, поэтому к аргументу **#** **&amp;**  _pxTypeText_ нельзя одновременно и символы. Функции таблицы можно объявлять как кластеробезопасные, так и потокобезопасные. В этом случае Excel позволит этим функциям принимать участие в многопочитаемом пересчете, если разгрузка кластера отключена. 
  
### <a name="category-names"></a>Имена категорий

Чтобы определить категорию для применения функций XLL, воспользуйтесь следующими рекомендациями.
  
- Если функция делает что-то, что может сделать пользователь в составе пользовательского интерфейса надстройки, следует поместить эту функцию в категорию команд.  
    
- Если функция возвращает сведения о состоянии надстройки или любые другие полезные  сведения, следует поместить функцию в категорию "Сведения". 
    
- Надстройка никогда не должна добавлять функции или команды в категорию **User Defined.** Эта категория используется исключительно для конечных пользователей.
