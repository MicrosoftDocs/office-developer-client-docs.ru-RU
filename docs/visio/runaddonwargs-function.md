# <a name="runaddonwargs-function"></a>Функция RUNADDONWARGS

Выполняет _строку_ и передает _аргументы_ командной строки программы как строку. 
  
## <a name="syntax"></a>Синтаксис

RUNADDONWARGS ("** *строки* **","** *аргументы* **") 
  
### <a name="parameters"></a>Параметры

|**Имя**|**Обязательный или необязательный**|**Тип данных**|**Описание**|
|:-----|:-----|:-----|:-----|
| _string_ <br/> |Обязательный  <br/> |**Строка** <br/> | Имя дополнительного компонента.  <br/> |
| _аргументы_ <br/> |Обязательный  <br/> |**Строка** <br/> |Аргументы для передачи в программу.  <br/> |
   
## <a name="remarks"></a>Замечания

На практике _аргументы_ должны быть не более 50 знаков. Используйте функцию RUNADDONWARGS для привязки программы, например дополнительного компонента для ячейки, например, действия или события ячейки. 
  
Функция RUNADDONWARGS можно выполнить только надстройки, которые являются участниками семейства **надстроек** приложения. Должны присутствовать в этой коллекции, дополнительный компонент должен быть EXE-файл или (VSL) для файла, который является: 
  
- Установлен в пути приложения **при запуске** или **надстройки** . 
    
- Добавлен программными средствами с помощью метода **Add** коллекции **надстроек** . 
    
Дополнительные сведения о запуске кода в Visio содержатся в разделе [о параметров безопасности и выполнить код в Visio](about-security-settings-and-running-code-in-visio-shapesheet.md) этот справочник по таблице свойств фигуры. 
  
В более ранних версиях Visio эта функция отображается в виде _RUNADDONWARGS. Версии приложения Visio 4.0 и более поздних версий принимать любого из стилей.
  
## <a name="example"></a>Пример

RUNADDONWARGS ("GRAPHMKR. Exe-ФАЙЛ», "/ GraphMaker = стек») 
  
Запускает Graphmkr.exe дополнительный компонент и передает его /GraphMaker аргумент = стека.
