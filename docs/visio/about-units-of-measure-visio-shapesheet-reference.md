# <a name="about-units-of-measure-visio-shapesheet-reference"></a>Об единицах измерения (справочник по таблицам фигур Visio)

При вставке полей в текст или формулы сборки часто указываются единицы измерения для введенных значений.
  
Microsoft Visio оценивает результат формулы по-разному в зависимости от ячейки, в которой вы вводите формулу. Как правило, для ячеек, которые представляют положение фигуры, измерение или угол, требуется пара "число-единица", состоящая из числа и единиц, необходимых для интерпретации числа. Многие другие ячейки не требуют единиц и не оцениваются как строка, true, FALSE или индекс. Например, та же формула, что в ячейке **FillForegnd** означает цвет 5 из цветовой палитры рисунка, означает TRUE (и блокирует ширину фигуры) в ячейке LockWidth. 
  
Всегда укажите единицу измерения при вводе формулы в ячейке, которая ожидает измерения. Если единица измерения не указана, Visio использует для этой ячейки единицу по умолчанию, которая может быть единицами измерения страницы, единицами рисования, единицами типа, единицами длительности или угловой единицей измерения.

## <a name="implicit-units-of-measure"></a>Неявные единицы измерения

Когда Visio разсмеяет и сохраняет пару "число-единица", он может использовать явные единицы или неявные единицы. Число, выраженного явными единицами измерения, всегда отображается в единицах измерения, которые были введены изначально. Число, выраженное в неявных единицах, всегда преобразуется в эквивалентное значение в единицах рисования, страницы или угловых единиц, соответствующих ячейке.
  
Например, предположим, что вы вводите эквивалент 1 дюйма в ячейке A с использованием явных единиц и в ячейке B с использованием неявных единиц, а ячейки A и ячейки B используют единицы рисования. Затем измените единицы по умолчанию для страницы на смайлики. Ячейка A по-прежнему отображает значение "1 in", так как она использует явные единицы, которые не изменяются с использованием по умолчанию. Ячейка B теперь отображает 2,54 cm, эквивалентное значению в единицах по умолчанию.
  
Чтобы ввести единицы неявно, используйте следующий синтаксис.
  
```vb
number  [unit , flag ]
```

|||
|:-----|:-----|
| _число_ <br/> |Исходное значение, например 3,7, 1,7E-4 или 5 1/2.  <br/> |
| _unit_ <br/> |Единицы, в  _которых изначально_ выражается число.  <br/> |
| _flag_ <br/> |Система показателей, используемая при отобра использовании неявного значения. Значения см. ниже.  <br/> |
   
Флаг  _параметра —_ это одна из следующих букв (в верхнем или нижнем регистре), обозначающее систему показателей, которая должна использоваться при отображке единицы неявного значения. 
  
|**_Flag_**|**Система показателей**|**Пример**|
|:-----|:-----|:-----|
| a, A  <br/> | Angular  <br/> | =5[deg,A]  <br/> |
| d, D  <br/> | Drawing  <br/> | =5[in,D]  <br/> |
| e, E  <br/> | Duration  <br/> | =5[eh,E]  <br/> |
| p, P  <br/> | Page  <br/> | =5[in,P]  <br/> |
| t, T  <br/> | Type  <br/> | =5[pt,T]  <br/> |
   
Кроме того, неявные единицы DL, DP, DT, DA, DE можно использовать для неявных единиц рисования, страниц, текста, угловых и единиц времени соответственно. Эти единицы предполагают, что связанное значение является внутренними единицами. Например, если текущая система показателей имеет размер в см см,  *=2 DL*  интерпретируется как 2 внутренних единицы (в дюймах) и отображается как 5,08 cm. 
  
С помощью неявного синтаксиса, описанного выше, это выражение (=2 DL) эквивалентно 2[in,d]. Неявный синтаксис позволяет интерпретировать значение, поэтому можно также указать значение 2[ft,d], которое будет интерпретироваться как 2 м и отображаться как 60,96 cm. Неявные единицы DL, DP, DT, DA и DE являются универсальными и не имеют локализованных аналогов.
