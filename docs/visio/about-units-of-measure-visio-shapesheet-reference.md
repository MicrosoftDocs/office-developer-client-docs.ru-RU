# <a name="about-units-of-measure-visio-shapesheet-reference"></a>Сведения об единицах измерения (ссылка на таблицу свойств фигур для Visio)

Когда вы вставляете поля в текст или создаете формулы, часто указываются единицы измерения для введенных значений.
  
Microsoft Visio оценивает результат формулы по-разному в зависимости от ячейки, в которой вводится формула. Как правило, ячейки, представляющие позицию фигуры, размерность или угол, требуют использования в качестве аргумента числовой единицы, состоящего из числа и единиц измерения, необходимых для интерпретации этого числа. Во многих других ячейках не требуются единицы измерения и значение TRUE или FALSE или индексу. Например, та же формула в ячейке **FillForegnd** означает, что цвет 5 из цветовой палитры документа означает true (и заблокирует ширину фигуры) в ячейке LockWidth. 
  
Всегда указывайте единицу измерения при вводе формулы в ячейку, в которой ожидается значение измерения. Если вы не укажете единицу измерения, Visio использует единицу измерения по умолчанию для этой ячейки, которая может быть единицей измерения страниц, единицами измерения, единицами измерения, единицами измерения длительности и угловыми единицами.

## <a name="implicit-units-of-measure"></a>НеЯвные единицы измерения

Когда Visio выполняет синтаксический анализ и сохраняет комбинацию числовых единиц, она может использовать явные единицы или неявные единицы. Число, выраженное в явных единицах, всегда отображается в единицах измерения, которые были введены изначально. Число, выраженное в неявных единицах, всегда преобразуется в эквивалентное значение на изображении, странице или угловых единицах, соответствующих ячейке.
  
Например, предположим, что вы вводите эквивалент 1 дюйма в ячейке A с помощью явных единиц и в ячейке B с использованием неявных единиц, а ячейка A и ячейка B используют единицы рисования. Затем измените единицы измерения для страницы по умолчанию на сантиметры. В ячейке A по-прежнему отображается 1., так как в ней используются явные единицы, которые не изменяются по умолчанию. В ячейке B теперь отображается 2,54 cm, эквивалентное значение в единицах по умолчанию.
  
Для неявного ввода устройств используется следующий синтаксис.
  
```vb
number  [unit , flag ]
```

|||
|:-----|:-----|
| _число_ <br/> |Исходное значение (например, 3,7, 1.7 E – 4 или 5 1/2).  <br/> |
| _устройств_ <br/> |Единицы, в которых выражается первоначальное _число_ .  <br/> |
| _flag_ <br/> |Система измерения, используемая при отображении единицы неявного значения. Ниже приведены значения.  <br/> |
   
_Флаг_ параметра — это одна из следующих букв (прописные или строчные буквы), указывающих систему измерения, которая должна использоваться при отображении единицы измерения неявного значения. 
  
|**_Флаг_**|**Система единиц измерения**|**Пример**|
|:-----|:-----|:-----|
| a, A  <br/> | Angular  <br/> | = 5 [град, A]  <br/> |
| d, D  <br/> | Drawing  <br/> | = 5 [in, D]  <br/> |
| e, E  <br/> | Duration (Длительность)  <br/> | = 5 [EH, E]  <br/> |
| p, P  <br/> | Page  <br/> | = 5 [in, P]  <br/> |
| t, T  <br/> | Тип  <br/> | = 5 [pt, T]  <br/> |
   
Кроме того, вы можете использовать неявные единицы DL, DP, DT, DA, DE для неявной прорисовки, страницы, текста, угловых и посторонних единиц времени соответственно. Эти единицы предполагают, что связанное значение — внутренние единицы. Например, если текущая система измерения — сантиметры, то *= 2 DL* интерпретируются как 2 внутренних единицы (дюймы) и отображаются как 5,08 см. 
  
При использовании неявного синтаксиса, описанного выше, это выражение (= 2 DL) эквивалентно 2 [in, d]. Неявный синтаксис позволяет выбрать способ интерпретации значения, поэтому вы также можете указать 2 [ФТ, d], которые интерпретируются как 2 метра и отображаются как 60,96 см. Неявные единицы DL, DP, DT, DA и DE являются универсальными и не имеют локализованных аналогов.
