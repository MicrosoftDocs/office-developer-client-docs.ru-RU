# <a name="runaddon-function"></a>Функция RUNADDON

Выполняет надстройку или макрос в проекте Microsoft Visual Basic для приложений (VBA). 
  
## <a name="syntax"></a>Синтаксис

RUNADDON (" *строка* ") 
  
### <a name="parameters"></a>Параметры

|**Имя**|**Необходимость**|**Тип данных**|**Описание**|
|:-----|:-----|:-----|:-----|
| _string_ <br/> |Обязательный  <br/> |**String** <br/> | Имя надстройки в коллекции **надстроек** или макросе в проекте VBA.  <br/> |
   
## <a name="remarks"></a>Примечания

Если проект документа, который содержит вызов функции RUNADDON (или другой проект, если он указан), не имеет именованной _строки_(процедура без аргументов), Microsoft Visio выполняет именованную _строку_надстройки. Если не удается найти _строку_ с именованной надстройкой, Visio ничего не делает и не сообщает об ошибке. (Вы можете использовать свойство **трацефлагс** для отслеживания процедур и надстроек, которые Visio пытается запустить.) 
  
При вызове процедуры в стандартном модуле рекомендуется помечать строку именем модуля, который содержит процедуру (например, *ModuleName., CNAME*), так как несколько модулей могут содержать процедуру с одинаковыми именами. 
  
Чтобы вызвать процедуру в проекте, отличном от проекта документа, содержащего вызов функции RUNADDON, используйте синтаксис *projName. моднаме., CNAME* (необходимо явно задать ссылку на *ProjName* в проекте VBA). 
  
> [!NOTE]
>  Начиная с Visio 2002, функция RUNADDON не может выполнить строку, содержащую произвольный код VBA. Код, который ранее был передан в функцию RUNADDON, можно переместить в процедуру в проекте VBA документа, вызываемой из функции RUNADDON. 
  
Дополнительные сведения о запуске кода в Visio можно найти в статье [о параметрАх безопасности и выполнении кода в Visio](about-security-settings-and-running-code-in-visio-shapesheet.md) в этой ссылке на таблицу свойств фигуры. 
  
В более ранних версиях Visio Эта функция отображается как _РУНАДДОН. Visio версии 4,0 и более поздних принимают любой из этих стилей. 
  
## <a name="example-1"></a>Пример 1

RUNADDON ("Calendar. exe")
  
Запускает надстройку с именем Calendar. exe.
  
## <a name="example-2"></a>Пример 2

RUNADDON ("фигуры массива")
  
Запускает надстройку с (с расширением VSL), имя которой состоит из фигур массива.
  
## <a name="example-3"></a>Пример 3

RUNADDON ("ThisDocument. Репортстатистикс")
  
Вызывает макрос Репортстатистикс в модуле **ThisDocument** в проекте документа, содержащем вызов этой функции. 
  
> [!NOTE]
>  Чтобы вызвать макрос в модуле **ThisDocument** , необходимо перед строкой ввести "ThisDocument", как показано ниже. 
  
## <a name="example-4"></a>Пример 4

RUNADDON (" *ModuleName* . Репортстатистикс ") 
  
Вызывает макрос Репортстатистикс в *ModuleName* в проекте документа, содержащем вызов этой функции.
