# <a name="objtype-cell-miscellaneous-section"></a>ObjType Cell (Miscellaneous Section)

Определяет, можно ли разместить объекты или маршрутизать  их на схемах при использовании диалоговых окна "Настройка макета" для разметки фигур. 
  
|**Значение**|**Описание**|**Константа автоматизации**|
|:-----|:-----|:-----|
|&amp;H0  <br/> |Значение, используемое по умолчанию. Приложение принимает решение на основе контекста рисования.  <br/> |**visLOFlagsVisDecides** <br/> |
|&amp;H1  <br/> |Фигуру можно разместить.  <br/> |**visLOFlagsPlacable** <br/> |
|&amp;H2  <br/> |Фигура является маршрутивной. Должна быть одномерной (одномерной) фигурой.  <br/> |**visLOFlagsRoutable** <br/> |
|&amp;H4  <br/> |Фигура не может быть замеенна, не маршрутивна.  <br/> |**visLOFlagsDont** <br/> |
|&amp;H8  <br/> |Группа содержит фигуры, которые можно разместить или маршрутить.  <br/> |**visLOFlagsPNRGroup** <br/> |
   
## <a name="remarks"></a>Примечания

По умолчанию ячейка ObjType имеет значение No Formula для фигуры, значение которой оценивается как 0, то есть приложение определяет, можно ли разместить фигуру в зависимости от ее контекста. Например, если нарисовать простой прямоугольник, значение его ячейки ObjType будет 0. Если затем использовать  соединитель для подключения прямоугольника к другой фигуре, Visio сбрасывает значение ячейки ObjType прямоугольника до 1 (место). 
  
Значение ячейки ObjType может быть комбинацией значений. Если задается неустанавливаемый бит ( H4), то он имеет приоритет перед другими значениями, кроме значения &amp; группы ( &amp; H8).
  
Чтобы получить ссылку на ячейку ObjType по имени из другой формулы или из программы с помощью свойства **CellsU,** используйте: 
  
|||
|:-----|:-----|
|Имя ячейки:  <br/> |ObjType  <br/> |
   
Чтобы получить ссылку на ячейку ObjType по индексу из программы, используйте свойство **CellsSRC** со следующими аргументами: 
  
|||
|:-----|:-----|
|Индекс раздела:  <br/> |**visSectionObject** <br/> |
|Индекс строки:  <br/> |**visRowMisc** <br/> |
|Индекс ячейки:  <br/> |**visLOFlags** <br/> |
