# <a name="lookup-function"></a>Функция LOOKUP

Возвращает отсчитываемый от нуля индекс, который указывает расположение подстроки _ключа_ в виде _списка_, или возвращает значение -1, если конечная строка содержит _разделитель_.
  
## <a name="syntax"></a>Синтаксис

ПОДСТАНОВКИ ("** *ключ* **","** *списка* **"[,"** *разделитель* **"]) 
  
### <a name="parameters"></a>Параметры

|**Имя**|**Обязательный или необязательный**|**Тип данных**|**Описание**|
|:-----|:-----|:-----|:-----|
| _ключ_ <br/> |Обязательный  <br/> |**Строка** <br/> |Строка, которую требуется найти.  <br/> |
| _списки_ <br/> |Обязательный  <br/> |**Строка** <br/> | Список, в котором требуется выполнить поиск.  <br/> |
| _delimiter_ <br/> |Optional  <br/> |**Строка** <br/> | Строка для использования в качестве разделителя в _списке_. Строка _разделителя_ может быть несколько символов в длину и может включать многобайтовых символов. Значение по умолчанию — точка с запятой.  <br/> |
   
### <a name="return-value"></a>������������ ��������

Числовой
  
## <a name="remarks"></a>Замечания

Функция поиска использует поиск без учета регистра. Если список начинается или заканчивается на разделителя, предполагается, что строка значение null, существующих до или после списка. Последовательные разделители между подразумевает пустая строка. 
  
Все аргументы должны быть строки или выражения, которые могут быть преобразованы в строки. Если это не так, вызвавшей ошибку аргумент заменяется пустую строку. 
  
## <a name="example-1"></a>Пример 1

ПОДСТАНОВКИ ("rat", "кошка; rat; goat»)
  
Возвращает 1.
  
## <a name="example-2"></a>Пример 2

ПОДСТАНОВКИ ("","; кошка; rat; goat»)
  
Возвращает 0.
  
## <a name="example-3"></a>Пример 3

ПОДСТАНОВКИ ("t", "кошка; rat; goat», «»)
  
Возврат значения 3.
