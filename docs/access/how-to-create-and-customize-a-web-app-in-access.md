# <a name="create-and-customize-a-web-app-in-access"></a>Создание и настройка веб-приложения в Access

> [!IMPORTANT]
> Корпорация Майкрософт больше не рекомендует создавать и использовать веб-приложения для Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/), чтобы создавать бизнес-решения без кода для Интернета и мобильных устройств. 
  
Access 2013 компонентов новая модель приложения, которая позволяет экспертов для быстрого создания веб-приложения. Входит в состав доступа — это набор шаблонов, которые можно использовать для перехода приступить к созданию приложения.

<a name="ac15_CreateAndCustomizeWebApp_Prerequisites"> </a>

## <a name="prerequisites-for-building-an-app-with-access-2013"></a>Предварительные требования для создания приложения с помощью Access 2013

Для выполнения действий, описанных в этом примере, вам потребуется следующее:
  
- Access
    
- Среда разработки SharePoint
    
Дополнительные сведения о настройке среды разработки SharePoint видеть [настроить среду разработки, общие для SharePoint](https://docs.microsoft.com/en-us/sharepoint/dev/general-development/set-up-a-general-development-environment-for-sharepoint). 
  
Дополнительные сведения о получении доступа и SharePoint в разделе [файлов для загрузки](http://msdn.microsoft.com/en-US/office/apps/fp123627).

## <a name="create-the-app"></a>Создание приложения

Предположим, что вы хотите создать приложение Access, который отслеживает проблемы в бизнесе. Прежде чем начинать создание таблиц и представление «с нуля», необходимо найти шаблона схемы, который соответствует потребностям.
  
### <a name="to-create-the-issue-tracking-app"></a>Чтобы создать приложение отслеживания вопросов

1. Откройте доступ и выберите пункт **настраиваемых веб-приложения**.
    
2. Введите имя и расположение веб-приложения. Можно выбрать расположение из **списка** и выберите команду **Создать**.
    
3. **Проблемы** в **как для отслеживания?** поле и нажмите клавишу ВВОД. 
    
   На рисунке 1 отображается список шаблонов, которые могут оказаться полезными для отслеживания проблем.
    
   **На рисунке 1. Шаблоны, соответствующие поиску проблем**

   ![Шаблоны, соответствующие поиску проблем] (media/odc_Access15_CreateAndCustomizeWebApp_Figure01.JPG "Шаблоны, соответствующие поиску проблем")
  
4. Выберите **проблемы**.
    
Access создает набор таблиц и представлений.
  
## <a name="explore-the-app"></a>Изучение приложения
<a name="ac15_CreateAndCustomizeWebApp_ExploreTheApp"> </a>

Чтобы понять, будет ли схемы и представления соответствии со своими потребностями, их следует проверить.
  
Таблицы, созданные путем выбора схемы "проблемы" отображаются в области заголовков. В таблицах проблемы, клиентов и сотрудников — это основное внимание приложения. В таблице ошибок хранятся сведения о каждой проблемы. Каждая проблема открыто с и сотруднику от имени клиента. В таблицах связанные вопросы и комментарии проблему играют вспомогательные роль в приложении. В таблице проблем, связанных с ними позволяет связать одна проблема в другую. В таблице комментариев проблему сохранение нескольких комментариев для одного проблемы.
  
В базе данных Microsoft Access рабочего стола (.accdb) связи между таблицами осуществляется в окне **схемы** . Приложения Access 2013 управление отношениями с помощью поля, установите тип данных **подстановки** . Давайте рассмотрим связи для таблицы проблемы, щелкнув правой кнопкой мыши плитку **проблемы** и выбрав пункт **Изменение таблицы**.
  
Поле « **клиент** » относится к таблице **Customers** . Для проверки связи, выберите поле **клиента** и затем выберите **Изменить операции поиска**. Появится **Мастер подстановок** , как показано на рисунке 2. 
  
**На рисунке 2. Мастер подстановок, отображающий отношение к таблице Customers**

![Мастер подстановок, отображающий связь] (media/odc_Access15_CreateAndCustomizeWebApp_Figure02.jpg "Мастер подстановок, отображающий связь")
  
Диалоговое окно Мастер подстановок показывает, поле **клиента** связана к таблице **Customers** и возвращает **Отображаемое имя и фамилия** поля из таблицы **Customers** . 
  
**Открывается**, **Назначено**и **Кем изменено** относятся к таблице **Employees** . Несколько полей, также присваивается тип данных **поиска** . В таких случаях тип данных поиска используется для указания конкретного значения о предоставлении в соответствующем поле. 
  
Закройте таблица **вопросов** и просмотрите область плиток. Верхняя трех плиток для **проблемы**, **клиентами**и таблицы **сотрудников** , отображаются иначе, чем двух заголовков нижней для таблицы **Связанные вопросы** и **Комментарии проблему** , как показано на рисунке 3. 
  
**На рисунке 3. Область плиток для схемы "проблемы"**

![Область плиток для схемы проблем] (media/odc_Access15_CreateAndCustomizeWebApp_Figure03.jpg "Область плиток для схемы проблем")
  
В таблицах **Связанные вопросы** и **Комментарии проблему** недоступны, поскольку они, должны быть скрыты от пользователя в веб-браузере. 
  
Для отслеживания некоторые проблемы воспользуемся приложения. Для этого нажмите кнопку **Запуск приложения** , чтобы открыть его в веб-браузере. 
  
Приложение откроется представление **Списка проблем** в таблице проблемы. Перед добавлением проблему, будет полезно определить для добавления некоторых клиентов и сотрудников. Щелкните плитку **клиентов** , чтобы начать добавление клиентов. 
  
Выбор представления используйте для выберите один из трех представления, доступные в таблице **Customers** , обозначенный **списка**, **таблицы**и **группы** , как показано на рисунке 4. 
  
**На рисунке 4. Выбор представления**

![Выбор представления] (media/odc_Access15_CreateAndCustomizeWebApp_Figure04.jpg "Выбор представления")
  
Выбор **списка** активирует представления **Списка клиентов** , который является представлением списка сведений. Сведения о списке является одним из представлений, которые автоматически создается при создании таблицы. Основной компонент, который отличает Просмотр сведений о списке является области списка, который отображается в левой части представления. В области списка используется для фильтрации и перемещаться по записям, содержащихся в представлении. Базы данных доступа к рабочему столу реализация представления списка для поиска требуется написании пользовательского кода. 
  
Выбор **таблицы** откроется представление **Таблицы данных клиентов** . Таблицы данных является другой тип представления, которое автоматически создается при создании таблицы. Представления таблицы данных полезны для тех, кому удобным для ввода, отсортировать, и фильтрация данных в виде наподобие электронных таблиц. 
  
Выбор групп откроется представление сводки. Сводного представления можно использовать для группировки записей на основе поля и при необходимости вычисление суммы или среднего.
  
При добавлении пользователей, используйте панель действий для добавления записей, изменение записей, сохраните записей, удалить записи и отменить изменения. Панель действий — настраиваемая панель инструментов, который отображается в верхней части каждого представления, как показано на рисунке 5.
  
**На рисунке 5. Панель действий**

![Панель действий] (media/odc_Access15_CreateAndCustomizeWebApp_Figure05.jpg "Панель действий")
  
Один раз вы добавили некоторые клиенты и сотрудники открыть представление списка проблем и начать добавление проблемы. При вводе имени клиента в в поле клиента одно или несколько имен клиентов будет отображаться, как показано на рисунке 6.
  
**На рисунке 6. Элемент управления AutoComplete**

## <a name="customize-the-app"></a>Настройка приложения
<a name="ac15_CreateAndCustomizeWebApp_CustomizeTheApp"> </a>

Теперь, когда вы извлекли тур приложения, вы Обратите внимание на то, что представление списка проблемы не содержит контактные данные для клиента. Настроим приложение, чтобы добавить рабочий телефон клиента в таблицу проблемы при создании проблему.
  
### <a name="to-add-a-field-to-the-issues-table"></a>Добавление поля в таблице проблемы

1. Откройте приложение в Access.
    
2. Выберите плитку **проблемы** , выберите значок **Параметры и действия** и выберите **Изменение таблицы**.
    
3. Введите **Номер контакта** в ячейку в столбце **Имя поля** . 
    
4. Выберите **Короткий текст** в столбце **Тип данных** . 
    
5. Нажмите кнопку **Сохранить**.
    
6. Закройте таблица вопросов.
    
Теперь, когда у нас есть поля, в которых хранятся номер телефона, давайте создадим макрос данных, чтобы найти контактные сведения.
  
### <a name="to-create-the-data-macro-to-look-up-contact-information"></a>Чтобы создать макрос данных, чтобы найти контактные сведения

1. В группе **Создание** нажмите кнопку **Дополнительно**и выберите **Макрос данных**.
    
2. Выберите **параметр Создать**.
    
3. В поле **имя** введите **CustID**. В раскрывающемся списке **Тип** выберите **(с плавающей десятичных знаков).**
    
4. В раскрывающемся списке **Добавить действие** выберите **макрокомандой НайтиЗапись, после**. 
    
5. В раскрывающемся списке **Поиск копирование записи в** выберите **клиентов**. 
    
6. В поле **Условие отбора** введите **[клиенты]. [ Идентификатор] = [CustID]**. 
    
7. Выберите **SetReturnVar** из раскрывающегося списка **Добавить действие** . 
    
    > [!NOTE]
    > Вы увидите два **Добавления нового действия** раскрывающихся списков, один внутри блока **макрокомандой НайтиЗапись, после** и другой вне блока **макрокомандой НайтиЗапись, после** . Следует выбрать в раскрывающемся списке **Добавить действие** в блоке **макрокомандой НайтиЗапись, после** , как показано на рисунке 7. 
  
   **На рисунке 7. Добавление нового действия раскрывающийся список**

   ![Добавление нового действия раскрывающийся список] (media/odc_Access15_CreateAndCustomizeWebApp_Figure07.jpg "Добавление нового действия раскрывающийся список")
  
8. В поле **имя** введите **ContactPhone**. 
    
9. Введите в поле **выражение** **[клиенты]. [ Номер рабочего телефона]**. 
    
10. Нажмите кнопку **Сохранить**. В поле **Имя** введите **GetContactPhone** и нажмите кнопку **ОК**.
    
    Макрос должен выглядеть макрос, показано на рисунке 8.
    
    **На рисунке 8. Макрос данных getcontactphone**

    ![Макрос данных getcontactphone] (media/odc_Access15_CreateAndCustomizeWebApp_Figure08.jpg "Макрос данных getcontactphone")
  
11. Закрытие конструктора макроса.
    
Теперь мы готовы для добавления в поле **Номер контакта** к форме списка вопросов. 
  
### <a name="to-add-the-contact-number-field-to-the-issues-list-form"></a>Чтобы добавить поле номер контакта к форме списка вопросов

1. Выберите таблицу **проблемы** . Это выбирает формы списка проблем. 
    
2. Выбор представления в выберите **список**, выберите значок **Параметры и действия** и выберите команду **Изменить**.
    
3. Перетащите элемент **Контактов номер** поля формы области **Список полей** расположения в форме место номер контакта для отображения. 
    
4. Выберите **Номер контакта** текстовое поле и нажмите кнопку **данных**. 
    
5. В поле **Имя элемента управления** ввода **CustomerContact** , а затем закройте меню " **данные** ". 
    
6. Нажмите кнопку **Сохранить**.
    
Теперь следует написать макрос пользовательского интерфейса пользователя, который копируется в поле **Рабочий телефон** из таблицы **Customers** в поле **Телефон контакта** таблицы **проблемы** . **После обновления** события элемента управления **CustomerAutocomplete** — это удобное место для макроса. 
  
### <a name="to-create-the-afterupdate-macro"></a>Чтобы создать макрос после обновления

1. Выберите элемент управления **CustomerAutocomplete** , нажмите кнопку **действия** и выберите **После обновления**. 
    
    Пустой макрос открывается в режиме конструктора макросов.
    
2. В раскрывающемся списке **Добавить действие** выберите **ЗапускМакросаДанных**. 
    
3. Выберите в раскрывающемся списке **Имя макроса** **GetContactPhone**. 
    
4. В поле **CustID** введите **[CustomerAutocomplete]**. 
    
5. В поле **SetLocalVar** введите **Телефон**. 
    
    При выборе данных getcontactphone, который был создан ранее, Access автоматически введено имя параметра и возврат переменной для макроса.
    
    Номер телефона для клиента, хранящиеся в переменной с именем телефона.
    
6. В раскрывающемся списке **Добавить действие** выберите **SetProperty**. 
    
7. В поле **Имя элемента управления** введите **CustomerContact**. 
    
8. В раскрывающемся списке **Свойства** выберите **значение**. 
    
9. В поле **значение** введите **= [Телефон]**. 
    
10. Нажмите кнопку **Сохранить**.
    
    Макрос должен выглядеть макрос, показано на рисунке 9.
    
    **На рисунке 9. Макрос после обновления**

    ![Макрос после обновления] (media/odc_Access15_CreateAndCustomizeWebApp_Figure09.jpg "Макрос после обновления")
  
11. Закрытие конструктора макроса.
    
12. Закройте представление списка проблем. Когда вам будет предложено сохранить изменения, нажмите кнопку **Да** . 
    
Теперь мы готовы к тексту настройки. Нажмите кнопку **Запуск приложения** откройте приложение в веб-браузере и добавьте новый вопрос. Обновления поля **Номер контакта** , автоматически после того, как введено имя клиента, как показано на рисунке 10. 
  
**На рисунке 10. Представление проблем с обновленным номером телефона**

![Представление проблем с обновленным номером телефона] (media/odc_Access15_CreateAndCustomizeWebApp_Figure10.jpg "Представление проблем с обновленным номером телефона")
