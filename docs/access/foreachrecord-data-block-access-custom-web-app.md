# <a name="foreachrecord-data-block-access-custom-web-app"></a>Блок данных ForEachRecord (Пользовательский доступ к веб-приложению)

Блок **данных ForEachRecord** повторяет набор заявлений для каждой записи в домене. 
  
> [!IMPORTANT]
> Корпорация Майкрософт в настоящее время не рекомендует создавать и использовать веб-приложения Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/) для создания бизнес-решений без кода для Интернета и мобильных устройств. 
  
> [!NOTE]
> Блок **данных ForEachRecord** доступен только в макросах данных. 
  
## <a name="setting"></a>Setting

Действие **ForEachRecord имеет** следующие аргументы. 
  
|**Имя аргумента**|**Обязательный**|**Описание**|
|:-----|:-----|:-----|
|**В** <br/> |Да  <br/> |Строка, определяемая доменом записей для работы. Аргумент *In* может содержать имя таблицы, выбранный запрос или SQL.  <br/> **ПРИМЕЧАНИЕ.** Указанный домен не может включать данные, хранимые в связанной таблице или источнике данных ODBC.           |
|**Где состояние** <br/> |Нет  <br/> |Строковая экспрессия, используемая для ограничения диапазона данных, на которых выполняется блок данных **ForEachRecord.** Например, критерии часто эквивалентны пункту WHERE в SQL, без слова WHERE. Если критерии не указаны, блок **данных ForEachRecord** работает на всем домене, указанном аргументом *In.* Любое поле, включенное в критерии, также должно быть полем  *в In*  .  <br/> |
|**Alias** <br/> |Нет  <br/> |Строка, которая предоставляет альтернативное имя для домена, указанного аргументом *In.* Часто используется для сокращения имени таблицы для последующих ссылок, чтобы предотвратить возможные неоднозначные ссылки. Если  *псевдоним не*  указан, в качестве псевдонима будет использоваться таблица или имя запроса.  <br/> |
   
## <a name="remarks"></a>Примечания

Чтобы немедленно выйти из блока данных **ForEachRecord, используйте действие ExitForEachRecord.** **[](exitforeachrecord-macro-action-access-custom-web-app.md)**
