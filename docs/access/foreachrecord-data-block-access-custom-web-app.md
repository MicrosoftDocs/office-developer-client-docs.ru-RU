# <a name="foreachrecord-data-block-access-custom-web-app"></a>Блок данных ForEachRecord (пользовательское веб-приложение Access)

Блок **данных ForEachRecord** повторяет набор записей для каждой записи в домене. 
  
> [!IMPORTANT]
> Корпорация Майкрософт в настоящее время не рекомендует создавать и использовать веб-приложения Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/) для создания бизнес-решений без кода для Интернета и мобильных устройств. 
  
> [!NOTE]
> Блок **данных ForEachRecord** доступен только в макросах данных. 
  
## <a name="setting"></a>Setting

Действие **ForEachRecord** имеет следующие аргументы. 
  
|**Имя аргумента**|**Обязательный**|**Описание**|
|:-----|:-----|:-----|
|**In** <br/> |Да  <br/> |Строка, определяя домен записей для работы. Аргумент  *In*  может содержать имя таблицы, запрос выбора или SQL.  <br/> **ПРИМЕЧАНИЕ.** Указанный домен не может включать данные, хранимые в связанной таблице или источнике данных ODBC.           |
|**Where Condition** <br/> |Нет  <br/> |Строка выражения, используемая для ограничения диапазона данных, для которого выполняется блок данных **ForEachRecord.** Например, критерии часто эквивалентны предложению WHERE в SQL выражения без слова WHERE. Если критерии не указаны, блок данных **ForEachRecord** работает на всем домене, указанном аргументом *In.* Любое поле, включенное в условия, также должно быть полем в  *in*  .  <br/> |
|**Alias** <br/> |Нет  <br/> |Строка, которая предоставляет альтернативное имя для домена, указанного аргументом *In.* Часто используется для сокращения имени таблицы для последующих ссылок, чтобы предотвратить возможные неоднозначные ссылки. Если  *псевдоним не*  указан, в качестве псевдонима будет использоваться таблица или имя запроса.  <br/> |
   
## <a name="remarks"></a>Примечания

Используйте действие **[ExitForEachRecord,](exitforeachrecord-macro-action-access-custom-web-app.md)** чтобы немедленно выйти из блока данных **ForEachRecord.**
