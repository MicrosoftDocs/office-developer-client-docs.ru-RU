# <a name="chapter-6-error-handling"></a>Глава 6. Обработка ошибок

**Область применения**: Access 2013, Office 2013

ADO использует несколько различных способов уведомления о возможном возникновении ошибок. В этой главе обсуждаются типы ошибок, которые могут возникать при использовании ADO и способ уведомления приложения. Она завершается путем предоставления предложений об обработке этих ошибок.

## <a name="how-does-ado-report-errors"></a>Как ADO сообщать об ошибках?

ADO уведомляет об ошибках несколькими способами:

- Ошибки во время выполнения генерируют ошибки ADO. Обработка ошибки ADO аналогично любой другой ошибке во время выполнения, например, при использовании оператора **On Error** в Visual Basic.

- Программа может получать ошибки из OLE DB. Ошибка во время выполнения также возникает при возникновении ошибки OLE DB.

- Если ошибка относится к поставщику данных, в коллекцию Errors объекта **Connection** , который использовался для **** доступа к хранилищу данных при возникновении ошибки, помещается один или несколько объектов **Error** .

- Если процесс, вызвавший событие, также вызвал ошибку, сведения об ошибке помещается в объект **Error** и передаются в качестве параметра для события. Дополнительные сведения о событиях можно найти в [главе 7: обработка событий ADO](chapter-7-handling-ado-events.md) .

- Проблемы, возникающие при обработке пакетных обновлений или других массовых операций с использованием объекта **Recordset** , могут быть указаны в свойстве **Status** объекта **Recordset**. Например, значения **RecordStatusEnum** могут задаваться нарушениями ограничений схемы или недостаточными разрешениями.

- Проблемы, возникающие при возникновении определенного **поля** в текущей записи, также указываются в свойстве **Status** каждого **поля** в коллекции **Fields** **записи** или **набора записей**. Например, значения **фиелдстатусенум** могут указывать на обновления, которые не удалось выполнить или с несовместимыми типами данных.
