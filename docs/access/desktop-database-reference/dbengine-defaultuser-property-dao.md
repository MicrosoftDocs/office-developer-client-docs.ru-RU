# <a name="dbenginedefaultuser-property-dao"></a>Свойство DBEngine. DefaultUser (DAO)


**Область применения**: Access 2013, Office 2013

Задает имя пользователя, используемое для создания **рабочей области** по умолчанию при ее инициализации. Для чтения и записи, **String**.

## <a name="syntax"></a>Синтаксис

*Expression* . DefaultUser

*выражение*: выражение, возвращающее объект **DBEngine**.

## <a name="remarks"></a>Примечания

Параметр для **DefaultUser** имеет тип данных String. Это может быть длина от 1 до 20 символов в рабочих областях Microsoft Access, которая может содержать буквенные символы, диакритические знаки, цифры, пробелы и символы, кроме: "(кавычки),/(косая черта), \\ ( \[ \] обратная косая черта),: (двоеточие), | (pipe), \< (знак "меньше"), \> (знак "больше"), + (знак плюс), = (знак равенства),; (точка с запятой), (запятая),? (вопросительный знак) \* , (звездочка), начальные пробелы и управляющие символы (ASCII 00 — ASCII 31).


> [!NOTE]
> Используйте надежные пароли, содержащие строчные и прописные буквы, цифры и знаки. В ненадежных паролях не используются сочетания таких элементов. Надежный пароль: Y6dh!et5. Слабый пароль: House27. Используйте надежный пароль, который можно запомнить, чтобы не пришлось его записывать.

По умолчанию для свойства **DefaultUser** задано значение "admin", а для свойства **DefaultPassword** задана строка нулевой длины ("").

Имена пользователей обычно не учитывают регистр; Однако при повторном создании учетной записи пользователя, которая была удалена или создана в другой рабочей группе, имя пользователя должно быть точным совпадением с учетом регистра с учетом исходного имени. В паролях учитывается регистр.

Как правило, метод **CreateWorkspace** используется для создания объекта **Workspace** с заданными именем пользователя и паролем. Однако для обеспечения обратной совместимости с более ранними версиями и удобства при отсутствии защищенной базы данных ядро СУБД Microsoft Access автоматически создает объект **рабочей области** по умолчанию, если он еще не открыт. В этом случае значения свойств **DefaultUser** и **DefaultPassword** определяют пользователя и пароль для объекта **рабочей области** по умолчанию.

Чтобы это свойство вступило в силу, необходимо задать его перед вызовом методов DAO.
