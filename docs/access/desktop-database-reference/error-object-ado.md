# <a name="error-object-ado"></a>Объект ошибки (ADO)


**Область применения**: Access 2013, Office 2013

Содержит сведения об ошибках доступа к данным, относящихся к одной операции с участием поставщика.

## <a name="remarks"></a>Примечания

Любая операция с объектами ADO может создавать одну или несколько ошибок поставщика. По мере возникновения каждой ошибки один или несколько объектов **ошибки** помещаются в коллекцию [Ошибок](errors-collection-ado.md) объекта [Подключения.](connection-object-ado.md) Когда другая операция ADO создает  ошибку, коллекция ошибок очищается  и новый набор объектов Ошибки помещается в коллекцию **Ошибок.**

> [!NOTE]
> Каждый **объект Ошибки** представляет собой определенную ошибку поставщика, а не ошибку ADO. Ошибки ADO подвергаются воздействию механизма обработки исключений во время работы. Например, в microsoft Visual Basic, возникновение ошибки, определенной для ADO, вызовет событие **On Error** и появится в **объекте Error.** Полный список ошибок ADO см. в разделе [ErrorValueEnum.](errorvalueenum.md)

Вы можете прочитать свойства **объекта Ошибки,** чтобы получить конкретные сведения о каждой ошибке, включая следующие:

- Свойство [Description,](description-property-ado.md) в котором содержится текст ошибки. Это свойство по умолчанию.

- Свойство [Number,](number-property-ado.md) которое содержит значение **Long** integer константы ошибок.

- Свойство [Source,](source-property-ado-error.md) которое определяет объект, который поднял ошибку. Это особенно полезно, если в коллекции **Ошибок** имеется несколько объектов **ошибки** после запроса в источник данных.

- Свойства [SQLState](sqlstate-property-ado.md) и [NativeError,](nativeerror-property-ado.md) которые предоставляют информацию из SQL источников данных.

При ошибке поставщика она помещается в коллекцию **Ошибок** объекта **Подключения.** ADO поддерживает возврат нескольких ошибок одной операцией ADO для получения сведений об ошибках, определенных поставщику. Чтобы получить эту богатую информацию обработщика ошибок, используйте соответствующие функции улавливания ошибок языка или среды, с которой вы работаете, а затем используйте вложенные циклы, чтобы скомпромить свойства каждого объекта **Error** в коллекции **Ошибок.**

**Пользователи microsoft Visual Basic и VBScript** Если допустимый объект **Подключения** не существует, необходимо получить сведения об ошибках из **объекта Error.**

Так же, как и поставщики, ADO очищает объект **OLE Error Info** перед вызовом, который потенциально может привести к новой ошибке поставщика. Однако коллекция **ошибок** на объекте **Подключение** очищается и заполняется только тогда, когда поставщик создает новую ошибку или когда вызывается метод [Clear.](clear-method-ado.md)

Некоторые свойства и методы возвращают предупреждения, которые отображаются как объекты ошибки в коллекции **Errors,** но не останавливают выполнение программы.  Перед вызовом [методов Resync,](resync-method-ado.md) [UpdateBatch](updatebatch-method-ado.md)или [CancelBatch](cancelbatch-method-ado.md) на [объекте Recordset;](recordset-object-ado.md) метод [Open](open-method-ado-connection.md) на **объекте Подключение;** или установите свойство [Filter](filter-property-ado.md) на **объекте Recordset,** вызывай **метод Clear** в коллекции **Ошибок.** Таким образом, вы можете прочитать свойство [Count](count-property-ado.md) из коллекции **Ошибок** для проверки на возврат предупреждений.
