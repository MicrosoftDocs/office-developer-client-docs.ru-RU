# <a name="indexclustered-property-dao"></a>Свойство index. Clustered (DAO)

**Область применения**: Access 2013, Office 2013

Задает или возвращает значение, которое указывает, представляет ли объект **index** кластеризованный индекс для таблицы (только для рабочих областей Microsoft Access). Для чтения и записи, **Boolean**.

## <a name="syntax"></a>Синтаксис

*Expression* . Сгруппирован

*Expression (выражение* ) Выражение, возвращающее объект **индекса** .

## <a name="remarks"></a>Примечания

Параметр или возвращаемое значение — это логический тип данных, который имеет значение **true** , если объект **index** представляет кластеризованный индекс.

В некоторых форматах баз данных IISAM для рабочих станций используются кластеризованные индексы. Кластеризованный индекс состоит из одного или нескольких неключевых полей, которые, взятые вместе, упорядочивают все записи в таблице в предварительно заданном порядке. Кластеризованный индекс обеспечивает эффективный доступ к записям в таблице, в которой значения индекса не могут быть уникальными.

Свойство **Clustered** доступно для чтения и записи для нового объекта **index** , который еще не добавлен в коллекцию и доступен только для чтения для существующего объекта **index** в коллекции **indexes** .

> [!NOTE]
> - Базы данных ядра СУБД Microsoft Access игнорируют свойство **Clustered** , так как ядро СУБД Microsoft Access не поддерживает кластеризованные индексы.
> - Для источников данных ODBC свойство **Clustered** всегда возвращает **значение false**; Он не определяет, имеется ли кластеризованный индекс в источнике данных ODBC.
