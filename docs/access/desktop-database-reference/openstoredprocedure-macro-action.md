# <a name="openstoredprocedure-macro-action"></a>Макрокоманда OpenStoredProcedure

**Область применения**: Access 2013, Office 2013

В проекте Access можно использовать действие **опенсторедпроцедуре** для открытия хранимой процедуры в режиме таблицы, конструктора хранимых процедур или в режиме предварительного просмотра. Это действие запускает именованную хранимую процедуру при открытии в представлении таблицы данных. Вы можете выбрать режим ввода данных для хранимой процедуры и ограничить записи, отображаемые хранимой процедурой.

> [!NOTE]
> Эта макрокоманда доступна только для доверенных баз данных.

## <a name="remarks"></a>Замечания

Это действие аналогично двойному щелчку хранимой процедуры в области навигации или щелчку правой кнопкой мыши сохраненной процедуры в области навигации и выбора нужной команды.

Переключение в режим конструктора в то время, когда хранимая процедура открыта, удаляет значение аргумента **режим данных** для хранимой процедуры. Этот параметр не действует, даже если пользователь возвращается в режим таблицы.

> [!TIP]
> - Вы можете перетащить хранимую процедуру из области навигации в строку макрокоманды. При этом автоматически создается действие **опенсторедпроцедуре** , которое открывает хранимую процедуру в представлении таблицы данных.
> - Если вы не хотите отображать системные сообщения, которые обычно отображаются при запуске хранимой процедуры (что указывает на то, что это хранимая процедура и показывает, сколько записей будет затронуто), можно использовать действие **сетварнинг** для отключения отображения этих писем.

Чтобы запустить действие **опенсторедпроцедуре** в модуле Visual Basic для приложений (VBA), используйте метод **опенсторедпроцедуре** объекта **DoCmd** .
