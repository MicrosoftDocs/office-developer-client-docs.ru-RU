# <a name="openstoredprocedure-macro-action"></a>Макрокоманда OpenStoredProcedure

**Область применения**: Access 2013, Office 2013

В проекте Access вы можете использовать действие **OpenStoredProcedure** для открытия сохраненной процедуры в представлении Таблицы данных, в представлении "Дизайн процедуры" или "Предварительный просмотр печати". Это действие выполняет именоваемую сохраненную процедуру при запуске в представлении Datasheet. Вы можете выбрать режим ввода данных для сохраненной процедуры и ограничить записи, отображаемые в сохраненной процедуре.

> [!NOTE]
> Эта макрокоманда доступна только для доверенных баз данных.

## <a name="remarks"></a>Примечания

Это действие аналогично дважды щелкнуть сохраненную процедуру в области навигации или щелкнуть правой кнопкой мыши сохраненную процедуру в области навигации и выбрать команду, которая вам нужна.

Переход на представление "Дизайн" во время открытия сохраненной процедуры удаляет параметр **аргумента "Режим** данных" для сохраненной процедуры. Этот параметр не действует, даже если пользователь возвращается в представление Таблицы данных.

> [!TIP]
> - Можно перетаскивать сохраненную процедуру из области навигации в строку действия макроса. Это автоматически создает действие **OpenStoredProcedure,** которое открывает сохраненную процедуру в представлении Datasheet.
> - Если вы не хотите отображать системные сообщения, которые обычно отображаются при запуске сохраненной процедуры (указывающее, что это сохраненная процедура и показывая, сколько записей будет затронуто), вы можете использовать действие **SetWarning** для подавления отображения этих сообщений.

Чтобы выполнить **действие OpenStoredProcedure** в модуле Visual Basic для приложений (VBA), используйте метод **OpenStoredProcedure** объекта **DoCmd.**
