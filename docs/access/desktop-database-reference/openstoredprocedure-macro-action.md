# <a name="openstoredprocedure-macro-action"></a>Макрокоманда OpenStoredProcedure

**Область применения**: Access 2013, Office 2013

В проекте Access можно использовать действие **OpenStoredProcedure,** чтобы открыть хранимую процедуру в представлении таблицы, представлении конструктора хранимой процедуры или предварительного просмотра печати. Это действие выполняет именоваемую хранимую процедуру при запуске в представлении таблицы. Можно выбрать режим ввода данных для хранимой процедуры и ограничить отображаемую запись.

> [!NOTE]
> Эта макрокоманда доступна только для доверенных баз данных.

## <a name="remarks"></a>Заметки

Это действие аналогично двойному щелчку хранимой процедуры в области навигации или щелчку правой кнопкой мыши хранимой процедуры в области навигации и выбору нужной команды.

При переключении в режим конструктора во время  открытия хранимой процедуры удаляется параметр аргумента режима данных для хранимой процедуры. Этот параметр не действует, даже если пользователь возвращается в представление таблицы.

> [!TIP]
> - Вы можете перетащить хранимую процедуру из области навигации в строку макроса. При этом автоматически создается действие **OpenStoredProcedure,** которое открывает хранимую процедуру в представлении таблицы.
> - Если вы не хотите отображать системные сообщения, которые обычно отображаются при запуске хранимой процедуры (указывая, что это хранимая процедура и показывает, сколько записей будет затронуто), можно использовать действие **SetWarning,** чтобы запретить отображение этих сообщений.

Чтобы запустить действие **OpenStoredProcedure** в модуле Visual Basic для приложений (VBA), используйте метод **OpenStoredProcedure** объекта **DoCmd.**
