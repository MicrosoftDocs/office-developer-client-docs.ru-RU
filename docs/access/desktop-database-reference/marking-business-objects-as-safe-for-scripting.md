# <a name="marking-business-objects-as-safe-for-scripting"></a>Пометка бизнес-объектов как безопасных для написания скриптов

**Область применения**: Access 2013, Office 2013

Чтобы обеспечить безопасную интернет-среду, необходимо пометить любые бизнес-объекты, с помощью [RDS. Метод](dataspace-object-rds.md) [CreateObject](createobject-method-rds.md) объекта DataSpace как "безопасный для сценариев". Чтобы их можно было использовать в DCOM, необходимо убедиться, что они помечены как такие в области лицензий системного реестра.

Чтобы вручную пометить бизнес-объект как безопасный для создания сценариев, создайте текстовый файл с расширением REG, содержа своим текстом ниже. Следующие два числа позволяют использовать функцию безопасных сценариев:

```vb 
 
[HKEY_CLASSES_ROOT\CLSID\<MyActiveXGUID>\Implemented 
Categories\{7DD95801-9882-11CF-9FA9-00AA006C42C4}] 
[HKEY_CLASSES_ROOT\CLSID\<MyActiveXGUID>\Implemented 
Categories\{7DD95802-9882-11CF-9FA9-00AA006C42C4}] 
```

где \< *MyActiveXGUID* — это \> деxadecimal GUID вашего бизнес-объекта. Сохраните его и объедините в реестре с помощью редактора реестра или дважды щелкните REG-файл в проводнике Windows.

Бизнес-объекты, созданные в Microsoft Visual Basic могут автоматически помечаться как безопасные для сценариев с помощью мастера упаковки и развертывания. Когда мастер попросит указать параметры безопасности, выберите "Безопасный **для** инициализации" и **"Безопасный для сценариев".**

На последнем этапе мастер установки приложений создает HTM-файл и CAB-файл. Затем можно скопировать эти два файла на целевой компьютер и дважды щелкнуть HTM-файл, чтобы загрузить страницу и правильно зарегистрировать сервер.

Так как бизнес-объект будет установлен в каталоге Windows System32 Occache по умолчанию, переместите его в каталог Windows System32 и измените корневой \\ \\ \\ **\_ \_ \\ CLSID \\** \< *MYActiveXGUID* \> \\ **InprocServer32** в соответствие с правильным путем.


> [!NOTE]
> Бизнес-объекты, помеченные как безопасные для сценариев или безопасные для инициализации, могут быть инициализированы любыми по сети. Любой настраиваемый бизнес-объект не должен быть разработан и реализован случайно. Крайне важно, чтобы такие объекты не представляют угрозу безопасности, которую могут исследовать злоумышленники для получения доступа к конфиденциальной области сервера размещения и ущерба.
