# <a name="understanding-the-customization-file"></a>Understanding the Customization File


**Область применения**: Access 2013, Office 2013

Каждый загон раздела в файле настройки состоит из квадратных скобок ( **\[\]** ), содержащих тип и параметр. Четыре типа разделов обозначены литеральными строками **подключения,** **sql,** **списком пользователей** или **журналами.** Параметром является литеральная строка, значение по умолчанию, указанный пользователем идентификатор или ничего.

Поэтому каждый раздел помечен одним из следующих разделов:

```text 
 
[ connect   default     ]
[ connect   identifier  ]
[ sql       default     ]
[ sql       identifier  ]
[ userlist  identifier  ]
[ logs                  ]
```

**DataFactory** вызывает обработчик, передавая параметры клиента. Обработчик ищет целые строки в параметрах клиента, соответствующие идентификаторам в соответствующих разделах. Если найдено совпадение, содержимое этого раздела применяется к параметру клиента.

Конкретный раздел используется при следующих обстоятельствах:

  - Раздел **подключения** используется, если значение части ключевого слова строки подключения клиента, "* Data  *Source=***value",* соответствует идентификатору раздела *подключения.*

  - Раздел **sql** используется, если строка команд клиента содержит строку, которая соответствует идентификатору раздела **sql.**

  - Если **нет** совпадающих идентификаторов, используется раздел подключения или **sql** с параметром по умолчанию.

  - Раздел **списка пользователей** используется, если идентификатор раздела **списка** пользователей совпадает с **идентификатором раздела** подключения. При совпадении содержимое раздела списка  пользователей применяется к подключению, управляемому разделом **подключения.**

  - Если строка в строке подключения или команды не  соответствует идентификатору в заголовке  раздела подключения или **sql** и нет заголовка раздела подключения или **sql** с параметром по умолчанию, то строка клиента используется без изменений.

  - Раздел **"Журналы"** используется при работе **DataFactory.**
