# <a name="understanding-the-customization-file"></a>Understanding the Customization File


**Применимо к**: Access 2013 | Office 2013

Каждый раздел заголовка в файле параметров состоит из квадратные скобки (**\[**) содержит тип и параметр. Типы четыре раздела указанный в параметре буквенные строки **подключения**, **sql**, **userlist**или **Журналы**. Параметр — это строковый литерал, по умолчанию, определенные пользователем идентификатор или ничего.

Таким образом каждый раздел помечен с одним из следующих разделов:

```text 
 
[ connect   default     ]
[ connect   identifier  ]
[ sql       default     ]
[ sql       identifier  ]
[ userlist  identifier  ]
[ logs                  ]
```

**DataFactory** вызывает обработчик, передача параметров клиента. Обработчик выполняет поиск полных строк в параметрах клиента, соответствующие идентификаторы в соответствующий раздел заголовков. Если совпадение найдено, содержимое этого раздела применяются к параметра клиента.

Конкретный раздел используется в следующих случаях:

  - Раздел **подключение** используется, если значение частью клиента подключиться ключевое слово string «**источник данных = *** значение*«, соответствует **подключения** идентификатор раздела *.*

  - Раздел **sql** используется, если строка команды клиента содержит строку, которая сопоставляет идентификатор раздела **sql** .

  - Раздел **подключение** или **sql** с помощью параметра по умолчанию используется, если соответствующий идентификатор отсутствует.

  - Раздел **userlist** используется, если идентификатор раздела **userlist** совпадает с идентификатором **подключения** раздела. Если совпадение, содержимое раздела **userlist** применяются к подключения, в соответствии с разделом **подключиться** .

  - Если строка в подключении или командной строки не соответствует идентификатор в заголовок раздела любого **подключения** или **sql** , и нет заголовка без **подключения** или **sql** с помощью параметра по умолчанию, то клиент используется строка без изменение.

  - В разделе **Журналы** используется всякий раз, когда **DataFactory** в операции.
