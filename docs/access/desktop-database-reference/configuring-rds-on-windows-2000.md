# <a name="configuring-rds-on-windows-2000"></a>Настройка RDS в Windows 2000


**Область применения**: Access 2013, Office 2013

Если у вас возникают проблемы при попытке правильной работы RDS после обновления до Windows 2000, выполните указанные ниже действия, чтобы устранить эту проблему.

1.  Убедитесь, что служба веб-публикации запущена первым путем перехода на*сервер* HTTPS://с помощью Internet Explorer. Если вы не можете получить доступ к веб-серверу таким образом, перейдите в окно командной строки и введите следующую команду: "NET START W3SVC".

2.  В меню Пуск выберите команду выполнить. Введите мсдфмап. ini и нажмите кнопку ОК, чтобы открыть файл мсдфмап. ini в блокноте. Установите флажок \[подключиться к\] разделу по умолчанию, а если параметру Access присвоено значение "недоступно", измените его на ReadOnly.

3.  С помощью программы regedit перейдите к разделу\_"\_программное\\обеспечение\\локального компьютера\\(\\Microsoft Хандлеринфо, Microsoft. **хандлеррекуиред** ") и убедитесь, что для задано значение 0, а **дефаулсандлер** — "" (нулевая строка).
    
    > [!NOTE]
    > Если вы вносите изменения в этот раздел реестра, необходимо остановить и перезапустить службу публикации в Интернете, введя следующие команды в командной строки: "NET STOP W3SVC" и "NET START W3SVC".

4.  С помощью программы Regedit переместитесь в реестре "CurrentControlSet\_Local\_Machine\\System\\Services\\\\Services\\\\адклаунч" и убедитесь, что существует ключ с именем **рдссервер.. факт**. Если это не так, создайте его.

5.  С помощью диспетчера служб Интернета перейдите на веб-сайт по умолчанию и просмотрите свойства виртуального корневого каталога MSADC. Проверьте ограничения на безопасность/IP-адрес и доменное имя каталога. Если флажок "отказано в доступе" установлен, выберите пункт "предоставлено".

Не забудьте попробовать перегрузить сервер, если изменения, которые не будут отображаться, не помогли в начале проблемы.
