# <a name="unique-table-unique-schema-unique-catalog-dynamic-properties-ado"></a>Уникальная таблица, уникальная схема, динамические свойства уникального каталога (ADO)


**Область применения**: Access 2013, Office 2013

Позволяет тщательно контролировать изменения определенной базовой таблицы [](recordset-object-ado.md) в наборе записей, который был создан операцией JOIN на нескольких базовых таблицах.

  - **В уникальной** таблице указывается имя базовой таблицы, на которую допускаются обновления, вставки и удаления.

  - **Уникальная схема** указывает *схему* или имя владельца таблицы.

  - **Уникальный каталог** указывает *каталог* или имя базы данных, содержащей таблицу.

## <a name="settings-and-return-values"></a>Параметры и значения возврата

Задает или возвращает значение **String,** которое является именем таблицы, схемы или каталога.

## <a name="remarks"></a>Примечания

Желаемая базовая таблица однозначно определена по каталогу, схеме и именам таблиц. При **наборе** свойства Unique Table для поиска базовой  таблицы используются значения свойств **Уникальной** схемы или Уникального каталога. Предполагается, но не обязательно, чтобы либо свойства **Уникальной** схемы, так и уникального каталога были задатки перед набором свойства **Unique Table.** 

Основной ключ **уникальной таблицы** рассматривается как основной ключ всего **Набор записей.** Это ключ, который используется для любого метода, требующего основного ключа.

Несмотря **на то,** что установлена уникальная таблица, метод [Delete](delete-method-ado-recordset.md) влияет только на именоваемую таблицу. Методы [AddNew,](addnew-method-ado.md) [Resync,](resync-method-ado.md) [Update](update-method-ado.md)и [UpdateBatch](updatebatch-method-ado.md) влияют на любые соответствующие базовые таблицы **Recordset.**

**Уникальная** таблица должна быть указана, прежде чем делать какие-либо настраиваемые ресинхронизации. Если **уникальная** таблица не указана, свойство [Команды Resync](resync-command-property-dynamic-ado.md) не будет иметь эффекта.

Ошибка во время работы приводит к невозможности найти уникальную базовую таблицу.

Все эти динамические свойства примыкают к коллекции свойств объектов **Recordset,** когда свойство [CursorLocation](cursorlocation-property-ado.md) задано **для adUseClient.** [](properties-collection-ado.md)
