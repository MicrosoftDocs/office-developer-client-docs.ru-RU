# <a name="errors"></a>Ошибки

**Область применения**: Access 2013, Office 2013

Любая операция, включаемая объекты ADO, может создавать одну или несколько ошибок поставщика. При каждой ошибке один или несколько объектов **Error** помещаются в коллекцию **Errors** объекта **Connection.** Подробные сведения об обработке предупреждений и ошибок в приложении ADO см. в главе [6 "Обработка ошибок".](chapter-6-error-handling.md)

Ошибки приложения могут быть и вызываются отдельным механизмом. Например, в Visual Basic объект **Err** будет содержать ошибки уровня приложения.
