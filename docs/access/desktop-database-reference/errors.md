# <a name="errors"></a>Ошибки

**Область применения**: Access 2013, Office 2013

Любая операция с объектами ADO может создавать одну или несколько ошибок поставщика. По мере возникновения каждой ошибки один или несколько объектов **ошибки** помещаются в коллекцию **Ошибок** объекта **Подключения.** Сведения об обработке предупреждений и ошибок в приложении ADO см. в главе [6. Обработка ошибок.](chapter-6-error-handling.md)

Ошибки приложения могут быть подняты отдельным механизмом. Например, в Visual Basic **объекте Err** будут содержаться ошибки на уровне приложений.
