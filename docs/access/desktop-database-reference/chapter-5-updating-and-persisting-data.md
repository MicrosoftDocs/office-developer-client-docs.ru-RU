# <a name="chapter-5-updating-and-persisting-data"></a><span data-ttu-id="329e8-102">Глава 5. Обновление и сохранение данных</span><span class="sxs-lookup"><span data-stu-id="329e8-102">Chapter 5: Updating and persisting data</span></span>

<span data-ttu-id="329e8-103">**Область применения**: Access 2013, Office 2013</span><span class="sxs-lookup"><span data-stu-id="329e8-103">**Applies to**: Access 2013, Office 2013</span></span>

<span data-ttu-id="329e8-104">В предыдущих главах было рассмотрено, как использовать ADO для получения данных в источнике данных, как перемещаться по данным и даже как редактировать данные.</span><span class="sxs-lookup"><span data-stu-id="329e8-104">The preceding chapters have discussed how to use ADO to get to data in a data source, how to move around in the data, and even how to edit the data.</span></span> <span data-ttu-id="329e8-105">Конечно, если цель приложения — разрешить пользователям вносить изменения в данные, необходимо понимать, как сохранить эти изменения.</span><span class="sxs-lookup"><span data-stu-id="329e8-105">Of course, if the goal of your application is to allow users to make changes to the data, you will need to understand how to save those changes.</span></span> <span data-ttu-id="329e8-106">Вы можете сохранить изменения **recordset** в файле с помощью метода **Save** или отправить изменения обратно в источник данных для хранения с помощью методов **Update** или **UpdateBatch.**</span><span class="sxs-lookup"><span data-stu-id="329e8-106">You can either persist the **Recordset** changes to a file using the **Save** method, or you can send the changes back to the data source for storage using the **Update** or **UpdateBatch** methods.</span></span>

<span data-ttu-id="329e8-107">В предыдущих главах данные были изменены в нескольких строках **наборов записей.**</span><span class="sxs-lookup"><span data-stu-id="329e8-107">In the preceding chapters, you changed the data in several rows of the **Recordset**.</span></span> <span data-ttu-id="329e8-108">ADO поддерживает два основных понятия, связанных с добавлением, удалением и изменением строк данных.</span><span class="sxs-lookup"><span data-stu-id="329e8-108">ADO supports two basic notions relating to the addition, deletion, and modification of rows of data.</span></span>

<span data-ttu-id="329e8-109">Первое понятие состоит в том, что изменения не сразу внося в **набор записей;** вместо этого они будут сделаны во внутренний *буфер копирования.*</span><span class="sxs-lookup"><span data-stu-id="329e8-109">The first notion is that changes aren't immediately made to the **Recordset**; instead, they are made to an internal *copy buffer*.</span></span> <span data-ttu-id="329e8-110">Если вы решите, что не хотите вносить изменения, изменения в буфере копирования будут отменены.</span><span class="sxs-lookup"><span data-stu-id="329e8-110">If you decide you don't want the changes, the modifications in the copy buffer are discarded.</span></span> <span data-ttu-id="329e8-111">Если вы решили сохранить изменения, изменения в буфере копирования применяются к **набору записей.**</span><span class="sxs-lookup"><span data-stu-id="329e8-111">If you decide to keep the changes, the changes in the copy buffer are applied to the **Recordset**.</span></span>

<span data-ttu-id="329e8-112">Во-вторых, изменения распространяются в источник данных, как только вы объявляете, что  работа над строкой завершена (то есть в немедленном режиме), или все изменения набора строк собираются до тех пор, пока вы не объявите работу для набора завершенных (то есть *пакетный* режим).</span><span class="sxs-lookup"><span data-stu-id="329e8-112">The second notion is that changes are either propagated to the data source as soon as you declare the work on a row complete (that is, *immediate* mode), or all changes to a set of rows are collected until you declare the work for the set complete (that is, *batch* mode).</span></span> <span data-ttu-id="329e8-113">Свойство **LockType** определяет время внесения изменений в исходный источник данных.</span><span class="sxs-lookup"><span data-stu-id="329e8-113">The **LockType** property determines when the changes are made to the underlying data source.</span></span> <span data-ttu-id="329e8-114">**AdLockOptimistic** или **adLockPessimistic** указывает непосредственный режим, а **adLockBatchOptimistic** указывает режим пакета.</span><span class="sxs-lookup"><span data-stu-id="329e8-114">**adLockOptimistic** or **adLockPessimistic** specifies immediate mode, while **adLockBatchOptimistic** specifies batch mode.</span></span> <span data-ttu-id="329e8-115">Свойство **CursorLocation** может влиять на то, какие параметры **LockType** доступны.</span><span class="sxs-lookup"><span data-stu-id="329e8-115">The **CursorLocation** property can affect which **LockType** settings are available.</span></span> <span data-ttu-id="329e8-116">Например, параметр **adLockPessimistic** не поддерживается, если свойство **CursorLocation** имеет свойство **adUseClient.**</span><span class="sxs-lookup"><span data-stu-id="329e8-116">For instance, the **adLockPessimistic** setting is not supported if the **CursorLocation** property is set to **adUseClient**.</span></span>

<span data-ttu-id="329e8-117">В немедленном режиме каждое вызов метода **Update** распространяет изменения в источник данных.</span><span class="sxs-lookup"><span data-stu-id="329e8-117">In immediate mode, each invocation of the **Update** method propagates the changes to the data source.</span></span> <span data-ttu-id="329e8-118">В пакетном режиме каждое вызов обновления или перемещение текущей позиции строки сохраняет изменения в буфере копирования, но только метод **UpdateBatch** распространяет изменения в источник данных. </span><span class="sxs-lookup"><span data-stu-id="329e8-118">In batch mode, each invocation of **Update** or movement of the current row position saves the changes to the copy buffer, but only the **UpdateBatch** method propagates the changes to the data source.</span></span>
