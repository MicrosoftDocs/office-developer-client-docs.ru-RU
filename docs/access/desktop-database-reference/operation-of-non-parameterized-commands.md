# <a name="operation-of-non-parameterized-commands"></a>Operation of Non-Parameterized Commands


**Применимо к**: Access 2013 | Office 2013

Для команды без параметров выполняются все команды поставщика и **наборов записей** , созданных во время выполнения команды. Если команда выполняется синхронно, всех наборов **записей** будет полностью заполнена. Выделенный режим асинхронное заполнение, заполненный состояние **наборов записей** будут зависеть от режима заполнения и размера **набора записей**.

Например *родительский command* может вернуть **записей** клиентов для компании из таблицы клиентов и *дочерние команда* может вернуть **записей** заказов для всех клиентов из таблицы заказов.

```vb 
 
SHAPE {SELECT * FROM Customers} 
 APPEND ({SELECT * FROM Orders} AS chapOrders 
 RELATE customerID TO customerID) 
```

Для связи без параметров родитель потомок каждого объекта **набора записей** родительских и дочерних должна иметь столбца Общие, чтобы связать их. Столбцы с именем в предложении ссылка *родительского столбца* , а затем *дочерний столбец*. Столбцы могут иметь разные имена в соответствующими объектами **набора записей** , но должны ссылаться на те же данные для задания удобной для восприятия отношения. Например объекты **клиентов** и **Заказы** **записей** может имеют поле customerID. Так как членство дочерних **записей** определяет, какие команды поставщика, возможна для дочерних **записей** для хранения потерянных строк. Эти потерянных строк недоступны без дальнейшего изменения формы.

Формирование данных добавляет столбец родительского **набора записей**. Значения в столбце главы: ссылки на строки в дочерних **записей**, которые удовлетворяют предложение ссылка. Такое же значение является в *родительский столбец* указанной родительской строки в *столбец дочерних* всех строк дочерних главы. При использовании нескольких предложений кому в том же предложении ссылка они неявно объединяются с помощью оператора AND. Если родительский столбцы в предложении ссылка не приводят ключ для родительского **набора записей**, строки одного дочернего может иметь несколько родительских строк.

При доступе к ссылку в столбце главы ADO автоматически извлекает **записей** представлены ссылки на. Обратите внимание на то, что в команде без параметров, хотя извлек всей дочерних **записей** главы только представляет подмножество строк.

Если добавленный столбец не *главы псевдоним*, имя будут создаваться автоматически для него. Объект [поля](field-object-ado.md) для столбца добавляется к коллекции [полей](fields-collection-ado.md) объекта **набора записей** и его тип данных будет **adChapter**.
