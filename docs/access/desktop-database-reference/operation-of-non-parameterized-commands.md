# <a name="operation-of-non-parameterized-commands"></a>Выполнение непараметризованных команд


**Область применения**: Access 2013, Office 2013

Для непараметровизированных команд выполняются все команды поставщика и  создаются записи во время выполнения команд. Если команда выполняется синхронно, все  записи будут заполнены полностью. Если выбран асинхронный режим популяции, то  состояние населенных пунктов записей будет зависеть от режима популяции и размера **записей.**

Например,   *родительская* команда может возвращать набор записей клиентов для компании из  таблицы Клиенты, а детская команда может возвращать набор заказов для всех клиентов из таблицы Заказов.

```vb 
 
SHAPE {SELECT * FROM Customers} 
 APPEND ({SELECT * FROM Orders} AS chapOrders 
 RELATE customerID TO customerID) 
```

Для непараметровных отношений родителей и детей каждый объект **recordset** для родителей и детей должен иметь общий столбец, чтобы связать их. Столбцы называются в пункте RELATE, сначала в *родительском* столбце, а затем *в столбце child-column.* Столбцы могут иметь разные имена в соответствующих объектах **Recordset,** но должны ссылаться на ту же информацию, чтобы указать значимое отношение. Например, **объекты Recordset Клиентов** и **Заказов**  могут иметь поле customerID. Так как членство  в наборе записей ребенка определяется командой  поставщика, ребенок может содержать осиротевших строк. Эти осиротевших строки недоступны без дальнейшего изменения.

Формирование данных придает столбец главы родительскому **набору записей.** Значения в столбце главы — это ссылки на строки в наборе записей **ребенка,** удовлетворяющие пункту RELATE. То есть в родительском  столбце родительской строки то же  значение, что и в детской колонке всех строк ребенка главы. При использовании нескольких пунктов TO в одном и том же пункте RELATE они неявно объединяются с помощью оператора AND. Если родительские столбцы в статье relate не являются ключом к родительскому набору **записей,** одна детская строка может иметь несколько родительских строк.

При доступе к ссылке в столбце главы ADO автоматически получает **набор записей,** представленный ссылкой. Обратите внимание, что в непараметровной команде, хотя весь набор записей ребенка был извлечен, в главе представлен только подмножество строк. 

Если в приложении столбца нет *псевдонима* главы, для него будет автоматически сгенерировано имя. Объект [Field](field-object-ado.md) для столбца будет примещаться к [](fields-collection-ado.md) коллекции Полей объекта **Recordset,** а его тип данных **будет adChapter**.

Сведения о навигации по иерархическому **набору записей** см. в сайте [Accessing Rows in a Hierarchical Recordset.](accessing-rows-in-a-hierarchical-recordset.md)
