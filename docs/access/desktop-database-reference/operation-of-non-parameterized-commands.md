# <a name="operation-of-non-parameterized-commands"></a>Выполнение непараметризованных команд


**Область применения**: Access 2013, Office 2013

Для непараметризованных команд все команды поставщика выполняются, а **наборы записей** создаются во время выполнения команды. Если команда выполняется синхронно, все **наборы записей** будут заполнены полностью. Если выбран режим асинхронного заполнения, заполненное состояние **наборов записей** будет зависеть от режима заполнения и размера **наборов записей**.

Например, родительская *команда* может возвращать **набор записей** клиентов для компании из таблицы Customers, а дочерняя *команда* может возвращать **набор записей** заказов для всех клиентов из таблицы Orders.

```vb 
 
SHAPE {SELECT * FROM Customers} 
 APPEND ({SELECT * FROM Orders} AS chapOrders 
 RELATE customerID TO customerID) 
```

Для связей с непараметризованными родительскими и дочерними объектами каждый родительский и дочерний объект **Recordset** должен иметь общий столбец, чтобы связать их. Столбцы именуются в предложении reLATE, First *— Column* , а потом дочерний *столбец*. Столбцы могут иметь разные имена в соответствующих объектах **Recordset** , но должны ссылаться на одну и ту же информацию, чтобы указать осмысленное отношение. Например, объекты **Recordset** для **Customers** и **Orders** могут иметь поле CustomerID. Так как членство дочернего **набора записей** определяется с помощью команды provider, доЧерний **набор записей** может содержать потерянные строки. Эти потерянные строки недоступны без дальнейшей изменения формы.

Формирование данных добавляет столбец главы в родительский **набор записей**. Значения в столбце Chapter — это ссылки на строки в дочернем **наборе записей**, которые удовлетворяют предложенИЮ Relate. То есть одно и то же значение находится в *родительском столбце* заданной родительской строки, что и в *дочернем столбце* всех строк дочернего столбца главы. Если несколько предложений TO используются в одном предложении СВЯЗЫВАНИЯ, они неявно комбинируются с помощью оператора AND. Если родительские столбцы в предложении RELATE не составляют ключ для родительского объекта **Recordset**, одна дочерняя строка может иметь несколько родительских строк.

При доступе к ссылке в столбце Chapter, ADO автоматически получает **набор записей** , представленный ссылкой. Обратите внимание, что в непараметризованной команде, несмотря на то, что получен весь дочерний **набор записей** , в главе представлен только подмножество строк.

Если добавленный столбец не имеет *псевдонима Chapter*, для него будет создано имя. Объект [поля](field-object-ado.md) для столбца будет добавлен в коллекцию Fields объекта **Recordset** , а его [](fields-collection-ado.md) тип данных будет **адчаптер**.

Сведения о переходе на иерархический **набор записей**содержатся [в разделе доступ к строкам в иерархическом наборе записей](accessing-rows-in-a-hierarchical-recordset.md).
