# <a name="operation-of-non-parameterized-commands"></a>Выполнение непараметризованных команд


**Область применения**: Access 2013, Office 2013

Для команд без параметров выполняются все команды поставщика,  а во время выполнения команды создаются записи. Если команда выполняется синхронно, все  записи будут полностью заполнены. Если выбран асинхронный режим заполнения, заполнение  записей будет зависеть от режима заполнения и размера **записей.**

Например, *родительская* команда может  вернуть набор записей клиентов для компании из  таблицы Customers, а потомок может вернуть набор заказов **recordset** для всех клиентов из таблицы "Заказы".

```vb 
 
SHAPE {SELECT * FROM Customers} 
 APPEND ({SELECT * FROM Orders} AS chapOrders 
 RELATE customerID TO customerID) 
```

Для непараметровизированных отношений родительский-потомок каждый родительский и потомок объекта **Recordset** должен иметь общий столбец, чтобы связать их. Столбцы именуются в предложении RELATE, *сначала в родительском* столбце, а затем *в столбце child-column.* У столбцов могут быть разные имена в соответствующих объектах **Recordset,** но они должны ссылаться на те же сведения, чтобы указать осмысленное отношение. Например, **объекты Customers** и **Orders** **Recordset** могут иметь поле customerID. Поскольку членство в  этом наборе записей определяется командой поставщика, он может содержать потерянные строки.  Эти потерянные строки недоступны без дальнейшего изменения их области.

Формирование данных привносим столбец главы в родительский **набор записей.** Значения в столбце главы являются ссылками на строки в наборе записей, которые удовлетворяют предложению RELATE. То есть в родительском  столбце заданной родительской строки находится то же значение, что и в столбце *child* всех строк в родительской главе. Если в одном предложении RELATE используется несколько предложений TO, они неявно объединяются с помощью оператора AND. Если родительские столбцы в предложении relate не составляют ключ к родительскому набору записей, одна родительская строка может иметь несколько родительских строк.

При доступе к ссылке в столбце главы ADO автоматически получает набор **записей,** представленный ссылкой. Обратите внимание, что в команде без параметров, хотя был извлечен весь набор записей, в главе представлено только подмножество строк. 

Если в столбце с приложением нет псевдонима *главы,* для него автоматически создается имя. Объект [Field](field-object-ado.md) для столбца будет примещаться к коллекции [Fields](fields-collection-ado.md) объекта **Recordset,** а его типом данных будет **adChapter.**

Сведения о навигации по иерархическому набору **записей** см. в подсети "Доступ к [строкам в иерархическом наборе записей".](accessing-rows-in-a-hierarchical-recordset.md)
