# <a name="refreshrecord-macro-action"></a>Макрокоманда RefreshRecord


**Область применения**: Access 2013, Office 2013

С помощью действия **RefreshRecord** вы можете обновить источник записей для активной формы или таблицы, чтобы отразить изменения, внесенные в записи в текущем наборе.

## <a name="remarks"></a>Заметки

Действие **RefreshRecord** отображает только изменения, внесенные в записи в текущем наборе. Так как действие **RefreshRecord** фактически не повторно затребовывало базу данных, текущий набор не будет включать в себя записи, которые были добавлены или исключены из нее с момента последнего повторного взвода базы данных; Также не будут исключены записи, которые больше не удовлетворяют условиям запроса или фильтра. Чтобы повторно запросить базу данных, используйте метод **[Requery](requery-macro-action.md)**. При повторном запросе источника записей текущий набор записей будет точно отражать все данные в источнике записей.

Поведение этого макроса зависит от того, вызывается ли оно в клиентской или веб-базе данных.

## <a name="client-database"></a>База данных клиента

В клиентской базе данных можно использовать действие **RefreshRecord,** чтобы обновить исходный источник записей для активной формы или таблицы данных, чтобы отразить изменения, внесенные в данные в текущем наборе. Изменения включают изменения, внесенные текущим пользователем или другими пользователями в среде с несколькими пользователями. Эквивалентно методу **[Refresh.](https://docs.microsoft.com/office/vba/api/Access.Form.Refresh)**

**Макрокод RefreshRecord** в клиентской базе данных делает следующее:

1.  Обновляет источник записей для активной формы или таблицы данных, чтобы отразить изменения, внесенные в строки в текущем наборе. Для связанных таблиц ODBC извлекает изменения записей в текущем наборе из источника данных.

2.  Обновляет текущий набор, чтобы отразить изменения. Если строка в источнике записей была удалена, она меняется на \# "Удалено".

3.  Обновляет активную или таблицу данных для отображения всех измененных и удаленных записей \# в текущем наборе.

4.  Повторно затребовывать все в подчиненные формы и в подрепорты в активной форме или таблице данных.

## <a name="web-database"></a>Веб-база данных

В веб-базе данных можно использовать действие **RefreshRecord** для обновления источника баз данных для активной формы или таблицы данных в соответствии с изменениями, внесенными в записи в текущем наборе. Изменения включают изменения, внесенные текущим пользователем или другими пользователями.

**Макрокод RefreshRecord** в веб-базе данных делает следующее:

1.  Извлекает изменения с сервера для всех базовых таблиц в текущем наборе. Для связанных таблиц ODBC извлекает изменения записей в текущем наборе из источника данных.

2.  Обновляет текущий набор, чтобы отразить изменения. Если строка в текущем наборе удалена, она меняется на \# "Удалено".

3.  Обновляет активную форму или таблицу данных для отображения всех измененных и удаленных записей \# в текущем наборе.

4.  Повторно затребовывать все в подчиненные формы и в подрепорты в активной форме или таблице данных.
