# <a name="customization-file-sql-section"></a>Раздел SQL в файле настройки


**Область применения**: Access 2013, Office 2013

Раздел **SQL** может содержать новую строку SQL, заменяющую командную строку клиента. Если в разделе нет строки SQL, раздел будет проигнорирован.

Может быть параметризована новая строка ** SQL. То есть параметры в строке SQL **** строки SQL (обозначенные символом "_км_") могут быть заменены соответствующими аргументами в идентификаторе ** в строке команд клиента (с разделителями-запятыми в скобках). Поведение списка идентификаторов и аргументов аналогично вызову функции.

Например, предположим, что Командная строка клиента имеет значение "Кустомербид (4)", заголовок раздела SQL \[— SQL\] кустомербид, а новая строка раздела SQL — "выбрать \* из клиентов, где CustomerID = _км_". Обработчик создастся, заголовок раздела SQL — \[SQL кустомербид\] , а новая строка раздела SQL — "выбрать \* из клиентов, где CustomerID = _км_". Обработчик создаст запрос "выберите \* из клиентов, где CustomerID = 4", и используйте эту строку для запроса к источнику данных.

Если новая инструкция SQL представляет собой пустую строку (""), раздел игнорируется.

Если новая строка оператора SQL является недопустимой, выполнение оператора завершится с ошибками. Параметр Client фактически игнорируется. Это можно сделать, чтобы отключить все команды SQL клиента, указав следующее:

```sql 
 
[SQL default] 
SQL = " " 
```
