# <a name="using-ado-for-internet-publishing"></a>Использование ADO для публикации в Интернете


**Применимо к**: Access 2013, Office 2013



[Поставщик OLE DB для публикации Интернет](the-ole-db-provider-for-internet-publishing.md) показывает конкретном примере доступа к разнородных данных с помощью ADO. Во время примеров, описанных в этом разделе, относится только к с помощью службу публикации в Интернете, принципы показано должно быть одинаковым при использовании ADO с другими поставщиками для разнородных данных, таких как поставщика в хранилище электронной почты.

## <a name="urls"></a>URL-адреса

Указатели универсальный код ресурса (URL-адреса) можно использовать в качестве альтернативы для строки подключения и текст команды для указания источников данных и папки для файлов и папок. URL-адреса, можно использовать с существующими объектами [подключения](connection-object-ado.md) и **записей** , а также с объектами **потока** и **записи** .

Дополнительные сведения об использовании URL-адреса можно [абсолютного и относительных URL-адресов](absolute-and-relative-urls.md).

## <a name="record-fields"></a>Поля записей

Отличительные различие между разнородных данных и однородных данных, для предыдущих версий, каждой строки данных или **записи**, а может иметь другой набор столбцы или **поля**. Для однородных данных каждая строка имеет тот же набор столбцов. Дополнительные сведения о полях конкретных службу публикации в Интернете можно [записей и полей Provider-Supplied](records-and-provider-supplied-fields.md).

## <a name="appending-new-fields"></a>Добавление новых полей

Несколько объектов ADO были улучшены для использования в сочетании с объектами **потока** и **записи** .

  - Коллекции [полей](fields-collection-ado.md) [Добавить](append-method-ado.md) метод, который создает и добавляет в коллекцию объекта [поля](field-object-ado.md) , можно также указать значение **поля**.

  - Метод [Update](update-method-ado.md) завершает добавления или удаления полей в коллекцию.

  - Ярлык и альтернативу метод **Append** можно создать поля, просто значение присваивается значение undefined или ранее удаленного поля.
