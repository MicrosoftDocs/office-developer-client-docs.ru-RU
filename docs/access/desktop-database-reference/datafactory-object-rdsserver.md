# <a name="datafactory-object-rdsserver"></a>Объект фактического объекта (Рдссервер)


**Область применения**: Access 2013, Office 2013

Этот рабочий объект на стороне сервера по умолчанию реализует методы, обеспечивающие доступ к данным для чтения и записи к указанным источникам данных для клиентских приложений.

## <a name="remarks"></a>Замечания

Объект **рдссервер.** DataObject создан как объект автоматизации на стороне сервера, который получает запросы клиентов. При реализации в Интернете он размещается на веб-сервере и создается компонентом АДИСАПИ. Объект **рдссервер.** DataObject предоставляет доступ для чтения и записи к указанным источникам данных, но не содержит логики проверки или бизнес-правил.

При использовании метода, доступного как в **рдссервер.** , так и в [RDS. Объекты управления](datacontrol-object-rds.md) данными, служба удаленных данных использует **RDS. **По умолчанию используется версия элемента управления. По умолчанию предполагается базовый сценарий программирования, в котором **рдссервер.** DataObject выступает в качестве универсального серверного бизнес-объекта.

Если требуется, чтобы веб-приложение обрабатывал обработку на стороне сервера, относящееся к задаче, вы можете заменить **рдссервер.** DataObject на настраиваемый бизнес-объект.

Вы можете создавать серверные бизнес-объекты, вызывающие методы **рдссервер.** CreateRecordset, такие как [Query](query-method-rds.md) и [](createrecordset-method-rds.md). Это полезно, если вы хотите добавить функции к бизнес-объектам, но воспользоваться преимуществами существующих технологий удаленной службы данных.

Идентификатор класса для объекта **рдссервер.** DataObject для объекта факта — 9381D8F5-0288-11D0-9501-00AA00B911A5.
