# <a name="stream-object-ado"></a>Объект Stream (ADO)


**Применимо к**: Access 2013, Office 2013

Представляет поток двоичных данных или текст.

## <a name="remarks"></a>Примечания

В структурированной дерева иерархии в файловой системе или систему электронной почты, [записи](record-object-ado.md) могут иметь по умолчанию двоичный поток битов, связанные с ним, содержащий содержимое файла или по электронной почте. Объект **потока** можно использовать для управления полей или записей, содержащих эти потоки данных. Объект **потока** можно получить следующими способами:

  - URL-адреса, с указанием объекта (обычно-файл), содержащего двоичные или текстовые данные. Этот объект может быть простой документ объект **записи** , представляющий структурированных документов или папку.

  - Открыв объект **Stream** , связанной с объектом **записи** по умолчанию. Вы можете получить потока по умолчанию, связанного с объектом **записи** при открытии **записи** , чтобы избежать работа с форматом только для открытия в потоке.

  - Путем создания экземпляра объекта **потока** . Эти объекты **потока** можно использовать для хранения данных в рамках приложения. В отличие от **потока** связанного с URL-адрес или **потока** **записи**по умолчанию инициализированный **потока** не связан ни с базовым источником по умолчанию.

С помощью методов и свойств объекта **потока** сделайте следующее:

  - Откройте объект **потока** из **записи** или URL-адрес с помощью метода [Open](open-method-ado-stream.md) .

  - Закройте **потока** с помощью метода [Close](close-method-ado.md) .

  - Ввод: байт или текст **потока** с методами [создания](write-method-ado.md) и [WriteText](writetext-method-ado.md) .

  - Чтение байт **потока** с помощью методов [чтения](read-method-ado.md) и [ReadText](readtext-method-ado.md) .

  - Написание любой **поток** данных ADO буфера базового объекта с помощью метода [очистки](flush-method-ado.md) .

  - Скопируйте содержимое **потока** с помощью метода [CopyTo](copyto-method-ado.md) другого **потока** .

  - Определяют, как читать строки из исходного файла с помощью метода [SkipLine](skipline-method-ado.md) и свойство [LineSeparator](lineseparator-property-ado.md) .

  - Определение конечных положение потока с [EOS](eos-property-ado.md) свойство и метод [SetEOS](seteos-method-ado.md) .

  - Сохранения и восстановления данных в файлах с помощью методов [SaveToFile](savetofile-method-ado.md) и [LoadFromFile](loadfromfile-method-ado.md) .

  - Укажите набор знаков, используемый для хранения **потока** со свойством [набор символов](charset-property-ado.md) .

  - Остановка асинхронной операции **потока** с помощью метода [Отменить](cancel-method-ado.md) .

  - Определение числа байтов **потока** с помощью свойства [размера](https://msdn.microsoft.com/library/jj250128\(v=office.15\)) .

  - Элемент управления текущей позиции в **потоке** со свойством [положение](position-property-ado.md) .

  - Определение типа данных в **поток** со свойством [типа](type-property-ado-stream.md) .

  - Определите текущее состояние **потока** (закрытой, open или выполнения) с помощью свойства [состояний](state-property-ado.md) .

  - Задать режим доступа для **потока** с помощью свойства [режима](mode-property-ado.md) .

> [!NOTE]
> URL-адреса, с помощью схемы http автоматически вызывает [Поставщик Microsoft OLE DB для публикации Интернет](microsoft-ole-db-provider-for-internet-publishing.md). Для получения дополнительных сведений см [абсолютных и относительных URL-адресов](absolute-and-relative-urls.md).
