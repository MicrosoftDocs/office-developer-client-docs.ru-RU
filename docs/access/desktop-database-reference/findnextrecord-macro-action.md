# <a name="findnextrecord-macro-action"></a>Макрокоманда FindNextRecord


**Область применения**: Access 2013, Office 2013

Вы можете использовать действие **FindNextRecord,** чтобы найти следующую запись, которая соответствует критериям, заданным предыдущим действием **FindRecord** или значением в диалоговом окне Найти и заменить (на вкладке Главная нажмите кнопку **Найти).**   Вы можете использовать **действие FindNextRecord** для многократного поиска записей. Например, можно последовательно перемещать все записи для конкретного клиента.

## <a name="setting"></a>Параметр

Действие **FindNextRecord** не имеет аргументов. Действие **FindNextRecord** находит следующую запись, которая соответствует критериям, установленным действием **FindRecord** или диалоговом окне **Find and Replace.** Аргументы для действия **FindRecord** делятся с вариантами в диалоговом окне Найти и **заменить.**

Чтобы установить критерии поиска, используйте **действие FindRecord.** Как правило, вы вводите действие **FindRecord** в макросе, а затем используете **действие FindNextRecord,** чтобы найти последующие записи, которые соответствуют тем же критериям. Чтобы искать записи только при определенном условии, можно ввести условное выражение в столбце **Condition** строки действий для действия **FindNextRecord.**

## <a name="remarks"></a>Примечания

Это действие имеет тот же эффект, что и использование кнопки **Find Next** в диалоговом окне Найти и **заменить.**

> [!NOTE]
> Хотя действие **FindRecord** соответствует команде **Find** на вкладке **Главная** для таблиц, запросов и форм,  оно не соответствует команде Find в меню **Edit** в окне Code. Вы не можете использовать действие **FindRecord** или **FindNextRecord** для поиска текста в модулях.

> [!TIP]
> Если для действия  **FindRecord** задавалось только текущее **поле,** может потребоваться использовать действие **GoToControl,** чтобы переместить фокус на управление, содержащее данные, которые вы ищете, прежде чем использовать действие **FindNextRecord.**

Если выбранный в настоящее время текст является таким же, как текст поиска во время действия **макрос FindNextRecord,** поиск начинается сразу после выбора, в том же поле, что и выбор, и в той же записи. В противном случае поиск начинается в начале текущей записи. Это позволяет находить несколько экземпляров тех же критериев поиска, которые могут отображаться в одной записи.

Однако обратите внимание, что при использовании кнопки команды для запуска макроса, содержащего действие **FindNextRecord,** первый экземпляр критериев поиска будет найден повторно. Такое поведение возникает из-за того, что нажатие кнопки команды удаляет фокус из поля, содержащего совпадающее значение. После **этого действие FindNextRecord** начнет поиск с самого начала записи. Чтобы избежать этой проблемы, запустите макрос с помощью метода, который не меняет фокус, например настраиваемой кнопки панели инструментов или сочетания ключей, определенных в макрос AutoKeys. Кроме того, перед проведением действия **FindNextRecord** установите фокус в макросе на поле, содержащее критерии поиска.

Такое же поведение также происходит, если вы используете кнопку команды для запуска макроса, содержащего действие **FindRecord** с набором аргумента **Find First** для **No**.

Чтобы выполнить **действие FindNextRecord** в Visual Basic для приложений, используйте метод **FindNext** объекта **DoCmd.**
