# <a name="findnextrecord-macro-action"></a>Макрокоманда FindNextRecord


**Область применения**: Access 2013, Office 2013

С помощью действия **FindNextRecord** можно найти следующую запись, которая соответствует критериям, указанным предыдущим  действием **FindRecord,**  или значение в диалоговом окне "Найти и заменить" (на вкладке "Главная" нажмите кнопку "Найти").  Вы можете использовать действие **FindNextRecord** для многократного поиска записей. Например, можно последовательно перемещаться по всем записям для определенного клиента.

## <a name="setting"></a>Setting

Действие **FindNextRecord** не имеет аргументов. Действие **FindNextRecord** находит следующую запись, которая соответствует критериям, установленным действием **FindRecord** или в диалоговом окне "Найти и заменить".  Аргументы для действия **FindRecord** совместно с вариантами в диалоговом окне "Найти **и** заменить".

Чтобы установить условия поиска, используйте действие **FindRecord.** Как правило, вы вводите действие **FindRecord** в макросе, а затем с помощью действия **FindNextRecord** находите записи, которые соответствуют тем же критериям. Чтобы искать записи только при определенном условии, можно ввести условное выражение в столбце **Condition** строки действий для действия **FindNextRecord.**

## <a name="remarks"></a>Заметки

Это действие имеет тот же эффект, что и использование кнопки "Найти **далее"** в диалоговом окне "Найти **и** заменить".

> [!NOTE]
> Хотя действие **FindRecord** соответствует  команде "Найти" на вкладке "Главная" для таблиц, запросов и форм,  оно не соответствует команде "Найти" в меню "Правка" в окне "Код".   Для поиска текста в модулях нельзя использовать действие **FindRecord** или **FindNextRecord.**

> [!TIP]
> Если для аргумента **"Только** текущее поле" действия **FindRecord** установлено **"Да",** может потребоваться использовать действие **GoToControl,** чтобы переместить фокус на тот или иной контроль, содержащий данные, которые вы ищете, прежде чем использовать действие **FindNextRecord.**

Если выбранный в данный момент текст является тем же, что и текст поиска во время макрокоде **FindNextRecord,** поиск начинается сразу после выделения, в том же поле, что и выделение, и в той же записи. В противном случае поиск начинается с начала текущей записи. Это позволяет найти несколько экземпляров одного условия поиска, которые могут отображаться в одной записи.

Однако обратите внимание, что при использовании командной кнопки для запуска макроса, содержащего действие **FindNextRecord,** первый экземпляр условия поиска будет найден несколько раз. Это происходит потому, что нажатие кнопки команды удаляет фокус с поля, содержащего совпадающее значение. После **этого действие FindNextRecord** начнет поиск с начала записи. Чтобы избежать этой проблемы, запустите макрос с помощью метода, который не меняет фокус, например настраиваемой кнопки панели инструментов или сочетания клавиш, определенных в макросах AutoKeys. Кроме того, перед действием **FindNextRecord** установите фокус макроса на поле, содержащее условия поиска.

Такое же поведение происходит, если вы используете кнопку команды для запуска макроса, содержащего действие **FindRecord** с аргументом **"Найти** первый" с установленным **no**.

Чтобы запустить **действие FindNextRecord** в модуле Visual Basic для приложений, используйте метод **FindNext** объекта **DoCmd.**
