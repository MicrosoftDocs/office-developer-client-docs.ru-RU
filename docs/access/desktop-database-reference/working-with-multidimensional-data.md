# <a name="working-with-multidimensional-data"></a>Работа с многомерными данными

**Область применения**: Access 2013, Office 2013

Ячейки *—* это результат запроса многомерных данных. Он состоит из коллекции осей, обычно не более четырех осей и, как правило, только двух или трех. *Ось* — это коллекция членов из одного или более измерений, которая используется для поиска или фильтрации определенных значений в кубе.

Положение *—* это точка вдоль оси. Для оси, состоящей из одного измерения, эти позиции являются подмножество членов измерения. Если ось состоит из нескольких измерений, каждое положение является составной сущностью, которая имеет *n* частей, где *n* — это число измерений, ориентированных вдоль этой оси. Каждая часть позиции является членом из одного составляющих измерения.

Например, если измерения "География" и "Продукт" куба, содержащего данные о продажах, ориентированы вдоль оси X ячеек, положение вдоль этой оси может содержать элементы "USA" и "Computers". В этом примере определение положения вдоль оси x требует, чтобы члены из каждого измерения были ориентированы вдоль оси.

*Ячейка* — это объект, который находится на пересечении координат оси. Каждая ячейка имеет несколько элементов данных, связанных с ней, включая сами данные, форматированную строку (отображаемую форму данных ячейки) и порядковое значение ячейки. (Каждая ячейка — это уникальное порядковые значения в ячейках. Порядковые значения первой ячейки в ячейках — нуль, а самая левая ячейка во второй строке ячейки с восемью столбцами будет иметь порядковые значения 8.)

Например, куб имеет следующие шесть измерений (обратите внимание, что эта схема куба немного отличается от примера, представленного в обзоре многомерных схем [и данных):](overview-of-multidimensional-schemas-and-data.md)

- Salesperson
- География (естественные иерархии) — континенты, страны, состояния и так далее
- Кварталы — кварталы, месяцы, дни
- Годы
- Меры — продажи, PercentChange, BudgetedSales
- Продукты

> [!NOTE]
> Значения ячеей в примере можно рассматривать как упорядоченные пары порядков положения оси, где первая цифра представляет положение оси X, а вторая — положение оси Y.

Характеристики этого ячеек:

- Измерения оси: Кварталы, Salesperson, География

- Размеры фильтра: "Показатели", "Годы", "Продукты"

- Две оси: COLUMN (x или Axis 0) и ROW (y или Axis 1)

- Ось x: два вложенных измерения: Salesperson и Geography

- Y-axis: измерение кварталов

Ось x имеет два вложенных измерения: Salesperson и Geography. В области "География" выбраны четыре члена: Сиэтл, Глоу, США-Южная и Япония. В salesperson выбраны два члена: "Иван" и "Вайс". Это дает в общей сложности восемь позиций на этой оси (8 = 4 \* 2).

Каждая координата представлена в качестве позиции с двумя членами : один из измерения Salesperson, а другой из измерения "География":

```vb 
 
(Valentine, Seattle), (Valentine, Boston), (Valentine, USA_North), 
(Valentine, Japan), (Nash, Seattle), (Nash, Boston), (Nash, USA_North), 
(Nash, Japan) 
```

Ось Y имеет только одно измерение, содержащее следующие восемь позиций:

```vb 
 
Jan, Feb, Mar, Qtr2, Qtr3, Oct, Nov, Dec 
```

Ячейки, ячейки, оси и положения представлены в ADO MD соответствующими объектами: [Cellset,](cellset-object-ado-md.md) [Cell,](cell-object-ado-md.md) [Axis](axis-object-ado-md.md)и [Position.](position-object-ado-md.md)
