# <a name="recordset-positioning"></a>Размещение наборов записей

**Область применения**: Access 2013, Office 2013

Используйте свойство **AbsolutePosition** для перемещения к записи на основе его расположения в объекте **Recordset** или для определения координат текущей записи. Поставщик должен поддерживать соответствующие функции, чтобы это свойство было доступно.

**AbsolutePosition** основан на 1 и равен 1, когда текущая запись является первой записью в **Наборе записей.** Как упоминалось ранее, общее число записей в объекте **Recordset** можно получить из свойства **RecordCount.**

При задании свойства **AbsolutePosition,** даже если оно записано в текущем кэше, ADO перезагружает кэш новой группой записей, начиная с указанной записи. Свойство **CacheSize** определяет размер этой группы.

> [!NOTE]
> Свойство **AbsolutePosition** не следует использовать в качестве номера записи суррогата. Положение данной записи меняется при удалении предыдущей записи. Кроме того, нет уверенности в том, что в случае повторного или повторного открытия объекта **Recordset** у данной записи будет одинаковый **absolutePosition.** Закладки — это рекомендуемый способ сохранения и возвращения в заданную позицию и единственный способ позиционирования во всех типах объектов **Recordset.**
