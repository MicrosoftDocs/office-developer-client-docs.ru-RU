# <a name="relationpartialreplica-property-dao"></a>Свойство relation. Партиалреплика (DAO)

**Область применения**: Access 2013, Office 2013

Задает или возвращает значение объекта **relation** , указывающее, следует ли учитывать это отношение при заполнении частичной реплики из полной реплики. (Только базы данных ядра СУБД Microsoft Access). Для чтения и записи, **Boolean**.

## <a name="syntax"></a>Синтаксис

*Expression* . партиалреплика

*Expression (выражение* ) Выражение, возвращающее объект **relation** .

## <a name="remarks"></a>Примечания

Параметр или возвращаемое значение — это тип данных Boolean, который имеет **значение true** , если отношение должно применяться во время синхронизации.

Это свойство позволяет реплицировать данные из полной реплики в частичную реплику на основе связей между таблицами. Свойство **партиалреплика** можно использовать, если установка свойства **репликафилтер** не может адекватно указывать, какие данные следует реплицировать в частичный. Например, предположим, что у вас есть база данных, в которой таблица Customers имеет связь "один ко многим" с таблицей "заказы", и вы хотите настроить частичную реплику, которая реплицирует заказы только от клиентов в регионе Калифорния (а не во всех заказах). Невозможно задать для свойства **репликафилтер** таблицы Orders значение Region = ' Ca ', так как поле область находится в таблице клиентов, а не в таблице Orders.

Для репликации всех заказов из региона Калифорния необходимо указать, что связь между таблицами Orders и Customers будет активна во время репликации. После создания частичной реплики приведенные ниже действия будут заполняться всеми заказами из региона Калифорния:

1.  Задайте для свойства **репликафилтер** объекта Customers **tabledef** значение "Region = ' Ca '".

2.  Присвойте свойству **партиалреплика** значение **true** для объекта **relation** , соответствующего связи между Orders и Customers.

3.  Вызовите метод **популатепартиал** .
    

> [!NOTE]
> При задании фильтра реплики или связи реплики следует учитывать, что записи в частичной реплике, которые не удовлетворяют условиям ограничения, будут удалены из частичной реплики, но не из полной реплики. Например, предположим, что вы установили свойство **репликафилтер** для клиентов **tabledef** в частичной реплике на "Region = ' Ca '", а затем повторно заполните базу данных. При этом будут вставлены или обновлены все записи для клиентов с учетом Калифорнии. 
> 
> Если затем сбросить свойство **репликафилтер** в значение "Region = ' FL '" и повторно заполнить базу данных, все записи областей штата Калифорния в частичной реплике будут удалены, а все записи из клиентов на основе Флорида будут вставлены из полной реплики. Никакие записи в полной реплике не будут удалены. 
>
> Перед настройкой свойства **репликафилтер** или **партиалреплика** рекомендуется синхронизировать частичную реплику, в которой вы настраиваете эти свойства с помощью полной реплики. Это гарантирует, что ожидающие изменения частичной реплики будут объединены в полную реплику перед удалением записей в частичной реплике.
