# <a name="relationpartialreplica-property-dao"></a>Свойство Relation.PartialReplica (DAO)

**Область применения**: Access 2013, Office 2013

Задает или возвращает значение объекта **Relation,** указывающее, следует ли учитывать это отношение при заполнении частичной реплики из полной реплики. (Только базы данных баз данных Microsoft Access). Для чтения и записи, **Boolean**.

## <a name="syntax"></a>Синтаксис

*выражения* . PartialReplica

*выражение* Выражение, возвращаемая **объекту Relation.**

## <a name="remarks"></a>Примечания

Значение параметра или возврата — это тип данных Boolean, который является **true,** если при синхронизации необходимо применять отношение.

Это свойство позволяет реплицировать данные из полной реплики в частичную реплику на основе взаимосвязей между таблицами. Свойство **PartialReplica** можно использовать при настройке свойства **ReplicaFilter** только при недостаточном указании, какие данные следует реплицировать в частичную. Например, предположим, что у вас есть база данных, в которой таблица Клиентов имеет отношение один к многим со таблицей Заказы, и необходимо настроить частичную реплику, которая реплицирует заказы только от клиентов в регионе Калифорния (вместо всех заказов). Невозможно задать свойство **ReplicaFilter** в таблице Заказы к Региону = "CA", так как поле Region находится в таблице Клиенты, а не в таблице Заказы.

Чтобы реплицировать все заказы из региона Калифорния, необходимо указать, что связь между таблицами "Заказы" и "Клиенты" будет активна во время репликации. После создания частичной реплики следующие действия заполнят ее всеми заказами из региона Калифорния:

1.  Установите свойство **ReplicaFilter** в объекте **Customers TableDef** на "Region = "CA".

2.  Установите значение свойства **PartialReplica** **true** на **объекте Relation,** соответствующем отношениям между заказами и клиентами.

3.  Вызов метода **PopulatePartial.**
    

> [!NOTE]
> При наборе отношения реплики или фильтра реплики следует помнить, что записи частичной реплики, не удовлетворяющие критериям ограничения, будут удалены из частичной реплики, но не из полной реплики. Например, предположим, что вы установите свойство **ReplicaFilter** в таблице Customers **TableDef** в частичной реплике на "Region = "CA", а затем перенаселенность базы данных. Это позволит вставить или обновить все записи для клиентов из Калифорнии. 
> 
> Если затем сбросить свойство **ReplicaFilter** в "Region = "FL" и перенаселить базу данных, все записи региона Калифорнии в частичной реплике будут удалены, а все записи клиентов из Флориды будут вставлены из полной реплики. Записи в полной реплике не будут удалены. 
>
> Перед установкой свойства **ReplicaFilter** или **PartialReplica** лучше синхронизировать частичную реплику, в которой вы устанавливаете эти свойства, с полной репликой. Это гарантирует, что ожидающих изменений частичной реплики будут объединены в полную реплику до удаления каких-либо записей в частичной реплике.
