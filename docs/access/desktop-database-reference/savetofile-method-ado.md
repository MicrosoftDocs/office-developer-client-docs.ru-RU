# <a name="savetofile-method-ado"></a>Метод SaveToFile (ADO)

**Область применения**: Access 2013, Office 2013

Сохраняет двоичное содержимое [потока](stream-object-ado.md) в файл.

## <a name="syntax"></a>Синтаксис

*Stream*. *Имя файла*SaveToFile, *савеоптионс*

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|:--------|:----------|
|*FileName* |**Строковое** значение, содержащее полное имя файла, в котором будет сохранено содержимое **потока** . Вы можете сохранить данные в любом допустимом локальном расположении или в любом расположении, к которому у вас есть доступ через значение UNC.|
|*савеоптионс* |Значение [савеоптионсенум](saveoptionsenum.md) , которое указывает, должен ли новый файл создаваться с помощью **SaveToFile**, если он еще не существует. Значение по умолчанию — **адсавекреатенотексистс**. С помощью этих параметров можно указать, что произойдет ошибка, если указанный файл не существует. Вы также можете указать, что **SaveToFile** перезаписывает текущее содержимое существующего файла.|

> [!NOTE]
> При перезаписи существующего файла (когда **адсавекреатеоверврите** задается) **SaveToFile** усекает все байты из исходного существующего файла, которые следуют за новым [EOS](eos-property-ado.md).

## <a name="remarks"></a>Примечания

**SaveToFile** можно использовать для копирования содержимого объекта **Stream** в локальный файл. Содержимое или свойства объекта **Stream** не изменяются. Перед вызовом **SaveToFile**объект **Stream** должен быть открыт.

Этот метод не изменяет связь объекта **Stream** с базовым источником. Объект **Stream** по-прежнему будет связан с исходным URL-адресом или **записью** , которая была источником при открытии.

После операции **SaveToFile** текущая позиция ([позиция](position-property-ado.md)) в потоке задается в начале потока (0).
