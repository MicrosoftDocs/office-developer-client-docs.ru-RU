# <a name="data-shaping"></a><span data-ttu-id="36385-102">Формирование данных</span><span class="sxs-lookup"><span data-stu-id="36385-102">Data shaping</span></span>

<span data-ttu-id="36385-103">**Область применения**: Access 2013, Office 2013</span><span class="sxs-lookup"><span data-stu-id="36385-103">**Applies to**: Access 2013, Office 2013</span></span>

<span data-ttu-id="36385-104">Представление данных позволяет определять столбцы в объекте **Recordset**в форме, отношения между сущностями, представленные столбцами, и способ заполнения **набора** данных данными.</span><span class="sxs-lookup"><span data-stu-id="36385-104">Data shaping enables you to define the columns of a shaped **Recordset**, the relationships between the entities represented by the columns, and the manner in which the **Recordset** is populated with data.</span></span>

<span data-ttu-id="36385-105">В столбцах объекта **Recordset** в форме могут содержаться данные от поставщика данных, например Microsoft SQL Server, ссылки на другой **набор записей**, производные от вычисления по одной строке **набора записей**, значения, полученные в результате операции над столбцом всего **набора записей**. или они могут быть созданным пустым столбцом.</span><span class="sxs-lookup"><span data-stu-id="36385-105">Columns of a shaped **Recordset** may contain data from a data provider such as Microsoft SQL Server, references to another **Recordset**, values derived from a calculation on a single row of a **Recordset**, values derived from an operation over a column of an entire **Recordset**; or they may be a newly fabricated, empty column.</span></span>

<span data-ttu-id="36385-106">При получении значения столбца, содержащего ссылку на другой **набор записей**, ADO автоматически возвращает реальный **набор записей** , представленный ссылкой.</span><span class="sxs-lookup"><span data-stu-id="36385-106">When you retrieve the value of a column that contains a reference to another **Recordset**, ADO automatically returns the actual **Recordset** represented by the reference.</span></span> <span data-ttu-id="36385-107">Объект **Recordset** , содержащий другой **набор записей** , называется иерархическим набором записей.</span><span class="sxs-lookup"><span data-stu-id="36385-107">A **Recordset** that contains another **Recordset** is called a hierarchical recordset.</span></span> <span data-ttu-id="36385-108">В иерархических наборах записей представлено отношение " *родители-потомки* ", в котором *родительским объектом* является содержащийся набор записей, а *дочерним* — контейнерный набор.</span><span class="sxs-lookup"><span data-stu-id="36385-108">Hierarchical recordsets exhibit a *parent-child* relationship, in which the *parent* is the containing recordset and the *child* is the contained recordset.</span></span> <span data-ttu-id="36385-109">Ссылка на **набор записей** фактически является ссылкой на подмножество дочерних элементов, называемых *главой*.</span><span class="sxs-lookup"><span data-stu-id="36385-109">The reference to a **Recordset** is actually a reference to a subset of the child, called a *chapter*.</span></span> <span data-ttu-id="36385-110">Один родительский элемент может ссылаться на более чем один дочерний **набор записей**.</span><span class="sxs-lookup"><span data-stu-id="36385-110">A single parent may reference more than one child **Recordset**.</span></span>

<span data-ttu-id="36385-111">Синтаксис команды Shape позволяет программным способом создавать **набор записей**в форме.</span><span class="sxs-lookup"><span data-stu-id="36385-111">The shape command syntax enables you to programmatically create a shaped **Recordset**.</span></span> <span data-ttu-id="36385-112">После этого вы можете получить доступ к компонентам **набора записей** программным способом или с помощью соответствующего визуального элемента управления.</span><span class="sxs-lookup"><span data-stu-id="36385-112">You can then access the components of the **Recordset** programmatically or through an appropriate visual control.</span></span> <span data-ttu-id="36385-113">Команда Shape выполняется как любой другой текст команды ADO.</span><span class="sxs-lookup"><span data-stu-id="36385-113">A shape command is issued like any other ADO command text.</span></span>

<span data-ttu-id="36385-114">С помощью синтаксиса команды Shape можно создавать иерархические объекты **Recordset** двумя способами.</span><span class="sxs-lookup"><span data-stu-id="36385-114">You can make hierarchical **Recordset** objects in two ways with the shape command syntax.</span></span> <span data-ttu-id="36385-115">Первый добавляет дочерний объект **Recordset** в родительский **набор записей**.</span><span class="sxs-lookup"><span data-stu-id="36385-115">The first appends a child **Recordset** to a parent **Recordset**.</span></span> <span data-ttu-id="36385-116">Родительский и дочерний, как правило, имеют по крайней мере один столбец: значение столбца в строке родителя совпадает со значением столбца во всех строках дочернего элемента.</span><span class="sxs-lookup"><span data-stu-id="36385-116">The parent and child typically have at least one column in common: the value of the column in a row of the parent is the same as the value of the column in all rows of the child.</span></span>

<span data-ttu-id="36385-117">Второй способ создает родительский **набор записей** из дочернего объекта **Recordset**.</span><span class="sxs-lookup"><span data-stu-id="36385-117">The second way generates a parent **Recordset** from a child **Recordset**.</span></span> <span data-ttu-id="36385-118">Записи в дочернем **наборе записей** группируются, как правило, с помощью предложения by, и одна строка добавляется в родительский **набор записей** для каждой результирующей группы в дочернем элементе.</span><span class="sxs-lookup"><span data-stu-id="36385-118">The records in the child **Recordset** are grouped, typically using the BY clause, and one row is added to the parent **Recordset** for each resulting group in the child.</span></span> <span data-ttu-id="36385-119">Если предложение BY опущено, дочерний **набор записей** будет состоять из одной группы, а родительский **набор записей** будет содержать ровно одну строку.</span><span class="sxs-lookup"><span data-stu-id="36385-119">If the BY clause is omitted, the child **Recordset** will form a single group and the parent **Recordset** will contain exactly one row.</span></span> <span data-ttu-id="36385-120">Это полезно для вычисления статистических данных "общий итог" для всего дочернего объекта **Recordset**.</span><span class="sxs-lookup"><span data-stu-id="36385-120">This is useful for computing "grand total" aggregates over the entire child **Recordset**.</span></span>

<span data-ttu-id="36385-121">Независимо от способа формирования родительского **набора записей** он будет содержать столбец Chapter, который используется для связывания с дочерним **набором записей**.</span><span class="sxs-lookup"><span data-stu-id="36385-121">Regardless of which way the parent **Recordset** is formed, it will contain a chapter column that is used to relate it to a child **Recordset**.</span></span> <span data-ttu-id="36385-122">При желании у родительского **набора записей** также могут быть столбцы, содержащие статистические выражения (SUM, min, Max и т. д.), для дочерних строк.</span><span class="sxs-lookup"><span data-stu-id="36385-122">If you wish, the parent **Recordset** may also have columns that contain aggregates (SUM, MIN, MAX, and so on) over the child rows.</span></span> <span data-ttu-id="36385-123">Как родительский, так и дочерний **набор записей** могут иметь столбцы, которые содержат выражение для строки в **наборе записей**, а также столбцы, которые являются новыми и изначально пусты.</span><span class="sxs-lookup"><span data-stu-id="36385-123">Both the parent and the child **Recordset** may have columns which contain an expression on the row in the **Recordset**, as well as columns which are new and initially empty.</span></span>

<span data-ttu-id="36385-124">Вы можете вкладывать иерархические объекты **Recordset** в любую глубину (то есть, создавать дочерние объекты **Recordset** для дочерних объектов **Recordset** и т. д.).</span><span class="sxs-lookup"><span data-stu-id="36385-124">You can nest hierarchical **Recordset** objects to any depth (that is, create child **Recordset** objects of child **Recordset** objects, and so on).</span></span>

<span data-ttu-id="36385-125">Можно получить доступ к **компонентам наборов записей** в **форме** программным способом или с помощью соответствующего визуального элемента управления.</span><span class="sxs-lookup"><span data-stu-id="36385-125">You can access the **Recordset** components of the shaped **Recordset** programmatically or through an appropriate visual control.</span></span>
