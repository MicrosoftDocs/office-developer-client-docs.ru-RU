# <a name="runmacro-macro-action"></a>Макрокоманда RunMacro

**Область применения**: Access 2013, Office 2013

Для запуска макроса можно использовать макрокоманду **ЗапускМакроса** . Макрос может находиться в группе макросов.

Вы можете использовать это действие:

- Для запуска макроса в другом макросе.

- Для запуска макроса на основе определенного условия.

- Присоединение макроса к команде настраиваемого меню.

## <a name="remarks"></a>Примечания

Если ввести имя группы макросов для аргумента **имя макроса** , Access выполняет первый макрос в группе макросов.

Это действие аналогично нажатию кнопки **выполнить макрос** на вкладке **Работа с базами данных** , выбором макроса и нажатием кнопки **ОК**. Тем не менее, эта команда выполняет макрос только один раз, а Макрокоманда **ЗапускМакроса** может выполнять макрос произвольное количество раз.

> [!TIP]
> Вы можете использовать аргументы **Repeat Count** и **Repeat Expression** , чтобы определить, сколько раз будет выполняться макрос:
> - Если оставить оба аргумента пустыми, макрос будет выполняться один раз.
> - Если ввести число **повторных** попыток, но оставить **выражение Repeat** пустым, макрос будет выполняться указанное число раз.
> - Если оставить **число повторов** пустым, но ввести выражение для **выражения Repeat**, макрос будет выполняться до тех пор, пока выражение не станет равно **false**.
> - Если вы вводите значения для обоих аргументов, макрос запустится количество раз, указанное в поле **число повторов** или пока **выражение Repeat** не будет равно **false**, что происходит в первую очередь.

При запуске макроса, содержащего макрокоманду **ЗапускМакроса** , и достижении макрокоманды **ЗапускМакроса** , Access запускает вызываемый макрос. После завершения вызванного макроса Access возвращается к исходному макросу и выполняет следующее действие.

> [!NOTE]
> - Вы можете вызвать макрос в той же группе макросов или в другой группе макросов.
> - Вы можете вложить макросы. То есть вы можете запустить макрос A, который, в свою очередь, вызывает макрос B и т. д. В каждом случае, когда вызванный макрос закончился, доступ возвращается к макросу, который вызывает его, и выполняет следующее действие в этом макросе.

Чтобы выполнить макрокоманду **ЗапускМакроса** в модуле Visual Basic для приложений (VBA), используйте метод **RunMacro** объекта **DoCmd** .
