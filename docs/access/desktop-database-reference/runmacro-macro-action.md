# <a name="runmacro-macro-action"></a>Макрокоманда RunMacro

**Область применения**: Access 2013, Office 2013

Вы можете использовать действие **RunMacro** для запуска макроса. Макрос может быть в макросовой группе.

Вы можете использовать это действие:

- Запуск макроса из другого макроса.

- Запуск макроса на основе определенного условия.

- Присоединение макроса к настраиваемой команде меню.

## <a name="remarks"></a>Примечания

Если введите имя группы макроса для **аргумента "Имя** макроса", Access запускает первый макрос в группе макроса.

Это действие аналогично нажатию **выполнить** макрос на вкладке **Средства** базы данных, выбрать макрос и щелкнуть **ОК**. Однако эта команда выполняет макрос только один раз, в то время как действие **RunMacro** может запускать макрос столько раз, сколько вам нужно.

> [!TIP]
> Для определения, сколько  раз выполняется макрос, можно использовать аргументы **Повторный** подсчет и Выражение повтора.
> - Если оставить оба аргумента пустыми, макрос выполняется один раз.
> - Если ввести номер для **повторного** подсчета, но оставить **повторяемую** экспрессию пустым, макрос выполняет указанное количество раз.
> - Если оставить **повторный отсчет** пустым, но введите выражение для **выражения repeat Expression,** макрос будет проходить до тех пор, пока выражение не будет оцениваться как **False**.
> - Если вы вводите значения для обоих аргументов, макрос  запускает количество  раз, заданное в повторе, или до тех пор, пока повторное выражение не будет оценено до **False**, в зависимости от того, что произойдет в первую очередь.

При запуске макроса, содержащего **действие RunMacro** и достигаемого действия **RunMacro,** Access запускает называемый макрос. После завершения называемого макроса Access возвращается к исходному макросу и выполняет следующее действие.

> [!NOTE]
> - Вы можете вызвать макрос в той же макросной группе или в другой макросной группе.
> - Можно вложенные макрос. То есть можно запустить макрос A, который в свою очередь вызывает макрос B и так далее. В каждом случае по завершению называемого макроса Access возвращается к макросу, который его называл, и выполняет следующее действие в этом макросе.

Чтобы выполнить **действие RunMacro** в Visual Basic для приложений (VBA), используйте метод **RunMacro** объекта **DoCmd.**
