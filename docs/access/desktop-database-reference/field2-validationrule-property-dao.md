# <a name="field2validationrule-property-dao"></a>Свойство Field2.ValidationRule (DAO)


**Применимо к**: Access 2013, Office 2013

Задает или возвращает значение, которое проверяет данные в поля, как она изменен или добавлен в таблицу (только для рабочих областей Microsoft Access). Для чтения и записи, **String**.

## <a name="syntax"></a>Синтаксис

*выражение* . Значение

*выражение* Выражение, возвращающее объект **поле2** .

## <a name="remarks"></a>Примечания

Параметры или возвращаемые значения — это строка, описывающая сравнения в виде SQL, ГДЕ предложения WHERE без зарезервированным словом. Для объекта еще не добавляется в конец коллекции **[полей](fields-collection-dao.md)** это свойство соответствует чтения и записи.

**Значение** определяет, содержит ли поле допустимых данных. Если данные не является допустимым, возникает перехватываемые ошибки времени выполнения. Сообщения об ошибке — текст свойства **сообщение об ошибке** , если указан или текст заданное **значение**.

Для объекта **поле2** использование **значение** зависит от объекта, которое содержит коллекцию **полей** , к которому добавляется объект **поле2** .

Проверка поддерживается только для баз данных, которые используют ядро базы данных Microsoft Access.

Строковое выражение, заданное свойством **значение** объекта **поле2** можно обратиться только к этой **поле2**. Выражение не может ссылаться на пользовательские функции, статистические функции SQL и запросы. Чтобы задать **значение объекта **поле2** ** при его **Проверка набора** свойство имеет значение **True**, выражение успешно необходимо проанализировать (с именем подразумеваемых операнда) и значение **True**. Если значение свойства его **Проверка набора** присвоено **значение False**, параметр **значение** свойства игнорируется.
