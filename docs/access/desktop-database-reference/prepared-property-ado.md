# <a name="prepared-property-ado"></a>Свойство Prepared (ADO)


**Область применения**: Access 2013, Office 2013

Указывает, следует ли сохранить скомпилированную версию команды перед выполнением.

## <a name="settings-and-return-values"></a>Параметры и возвращаемые значения

Задает или возвращает **логическое** значение, которое, если задано значение **true**, указывает, что команда должна быть подготовлена.

## <a name="remarks"></a>Замечания

Используйте **подготовленное** свойство, чтобы поставщик сохранял подготовленную (или скомпилированную) версию запроса, указанного в свойстве [CommandText](commandtext-property-ado.md) перед первым выполнением объекта [Command](command-object-ado.md) . Это может замедлить выполнение команды, но когда поставщик компилирует команду, поставщик будет использовать скомпилированную версию команды для всех последующих выполнений, что приведет к увеличению производительности.

Если свойство имеет **значение false**, то поставщик будет выполнять **Командный** объект напрямую, не создавая скомпилированную версию.

Если поставщик не поддерживает подготовку команды, он может вернуть сообщение об ошибке, как только это свойство будет иметь значение **true**. Если он не возвращает ошибку, он просто игнорирует запрос на подготовку команды и задает для свойства **Prepared** **значение false**.
