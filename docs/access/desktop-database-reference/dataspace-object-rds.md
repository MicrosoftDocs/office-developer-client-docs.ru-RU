# <a name="dataspace-object-rds"></a>Объект Space (RDS)

**Область применения**: Access 2013, Office 2013

Создает клиентские прокси-серверы для настраиваемых бизнес-объектов, расположенных на среднем уровне.

## <a name="remarks"></a>Замечания

Для удаленной службы данных необходимы прокси-серверы бизнес-объектов, чтобы клиентский компонент мог связываться с бизнес-объектами, расположенными на среднем уровне. Прокси-серверы упрощают упаковку, распаковку и транспортировку (маршалинг) данных [набора записей](recordset-object-ado.md) приложения в пределах одного процесса или компьютера.

Служба удаленных данных использует **RDS. **Метод [CreateObject](createobject-method-rds.md) объекта Space для создания прокси бизнес-объектов. Прокси-сервер бизнес-объектов создается динамически при создании экземпляра своего бизнес-объекта среднего уровня. Служба удаленных данных поддерживает следующие протоколы: HTTP, HTTPS (SSL Secure Sockets), DCOM и внутрипроцессный (клиентские компоненты и бизнес-объект находятся на одном компьютере).

> [!NOTE]
> При использовании RDS в качестве "неопределенного состояния" RDS перейдет в состояние "без состояния" **. Объект Space** использует протоколы HTTP или HTTPS. То есть все внутренние сведения о клиентском запросе удаляются после того, как сервер возвратит ответ.

Несмотря на то, что бизнес-объект существует в течение всего времени существования прокси бизнес-объекта, бизнес-объект фактически существует только до отправки ответа в запрос. Когда выдается запрос (то есть метод вызывается в бизнес-объекте), прокси открывает новое подключение к серверу, а сервер создает новый экземпляр бизнес-объекта. После того как бизнес-объект ответит на запрос, сервер уничтожает бизнес-объект и закрывает подключение.

Это означает, что вы не можете передавать данные из одного запроса в другой, используя свойство или переменную бизнес-объекта. Для сохранения данных состояния необходимо использовать другой механизм, например файл или аргумент метода.
