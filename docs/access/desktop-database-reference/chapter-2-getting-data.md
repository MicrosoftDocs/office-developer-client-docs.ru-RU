# <a name="chapter-2-getting-data"></a>Глава 2. Получение данных

**Область применения**: Access 2013, Office 2013

Предыдущая глава предоставила четыре основных операции, которые используются при создании приложения ADO: извлечение данных, изучение данных, редактирование данных и обновление данных. В этой главе рассматриваются сведения о концепциях, связанных с первой операцией: получения данных.

В этой операции может воспроизводиться несколько объектов ADO. Сначала подключитесь к источнику данных с помощью объекта **подключения** ADO (который иногда создается неявным образом). Затем вы передаете инструкции источнику данных о том, что нужно сделать с помощью объекта **команды** ADO (который также можно создать неявно). Результат передачи команды источнику данных и получения его ответа обычно будет представлен в объекте **Recordset** объекта ADO.

Для получения данных ваше приложение должно находиться в связи с источником данных, таким как СУБД, хранилищем файлов или текстовым файлом, разделенным запятыми. Это взаимодействие представляет собой *Подключение* — среду, необходимую для обмена данными.

Объектная модель ADO представляет концепцию подключения с помощью объекта **Connection** — фундамента, на котором создаются многие функциональные возможности ADO. Объект **Connection** предназначен для:

- Определение сведений, необходимых ADO для связи с источниками данных и создания сеансов.

- Определите возможности сеанса для транзакций.

- Позволяет создавать и выполнять команды для источника данных.

- Предоставьте сведения о структуре базового источника данных в виде наборов строк схемы. Дополнительные сведения о наборах строк схемы приведены в разделе [метод OpenSchema](openschema-method-ado.md).
