# <a name="chapter-2-getting-data"></a>Глава 2. Получение данных

**Область применения**: Access 2013, Office 2013

В предыдущей главе были представлены четыре основные операции, связанные с созданием приложения ADO: получение данных, изучение данных, редактирование данных и обновление данных. В этой главе основное внимание будет уделяться сведениям о понятиях, которые имеют отношение к первой операции: получение данных.

Несколько объектов ADO могут играть роль в этой операции. Сначала вы подключаете к источнику данных с помощью объекта ADO **Connection** (который иногда создается неявно). Затем вы передаете инструкции источнику данных о том, что нужно сделать с помощью объекта ADO **Command** (который также можно создать неявно). Результат передачи команды источнику данных и получения ответа обычно будет представлен в объекте ADO **Recordset.**

Чтобы получить данные, приложение должно быть в связи с источником данных, таким как DBMS, файловый магазин или текстовый файл с запятой. Это сообщение представляет собой *подключение* — среду, необходимую для обмена данными.

Объектная модель ADO представляет концепцию подключения к объекту **Connection** — основа, на которой построено большое количество функций ADO. Целью объекта **Подключения** является:

- Определите сведения, необходимые ADO для связи с источниками данных и создания сеансов.

- Определите возможности транзакций сеанса.

- Разрешить создание и выполнение команд в отношении источника данных.

- Предоставление сведений о разработке источника данных в виде строк схемы. Дополнительные сведения о строках схемы см. в [методе OpenSchema.](openschema-method-ado.md)
