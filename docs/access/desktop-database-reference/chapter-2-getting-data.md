# <a name="chapter-2-getting-data"></a>Глава 2. Получение данных

**Область применения**: Access 2013, Office 2013

В предыдущей главе представлены четыре основные операции, связанные с созданием приложения ADO: получение данных, изучение данных, изменение данных и обновление данных. В этой главе основное внимание будет уделяться концепциям, относянымся к первой операции: получение данных.

Несколько объектов ADO могут играть определенную роль в этой операции. Сначала необходимо подключиться к источнику данных с помощью объекта подключения **ADO** (который иногда создается неявно). Затем вы передаете в источник данных инструкции о том, что нужно сделать с помощью объекта команды **ADO** (который также можно создать неявно). Результат передачи команды в источник данных и получения ее ответа обычно будет представлен в объекте ADO **Recordset.**

Чтобы получить данные, приложение должно быть в связи с источником данных, таким как DBMS, хранилище файлов или текстовый файл с запятой. Это сообщение представляет собой *подключение —* среду, необходимую для обмена данными.

Объектная модель ADO представляет концепцию подключения к объекту **Connection** — основа, на которой построена функциональность ADO. Объект **Connection** предназначен для:

- Определите сведения, необходимые ADO для связи с источниками данных, и создайте сеансы.

- Определите транзакционные возможности сеанса.

- Позволяет создавать и выполнять команды для источника данных.

- Предоставление сведений о проектировании источника данных в форме наборов строк схемы. Дополнительные сведения о строках схемы см. в [методе OpenSchema.](openschema-method-ado.md)
