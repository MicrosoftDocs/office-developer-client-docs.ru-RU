# <a name="recordset2eof-property-dao"></a>Свойство Recordset2.EOF (DAO)


**Применимо к**: Access 2013, Office 2013

Возвращает значение, указывающее, является ли положение текущей записи после последней записи в объекте **набора записей** . Только для чтения, **Boolean**.

## <a name="syntax"></a>Синтаксис

*выражение* . ФУНКЦИЯ EOF

*выражение* Переменная, которая представляет собой объект- **Recordset2** .

## <a name="remarks"></a>Примечания

Свойства **BOF** и **EOF** можно использовать для определения того, содержит ли объект **набора записей** ли вы уменьшилось за пределы ограничения объекта **набора записей** , при перемещении по записям или записи.

Определяет расположение указатель текущей записи **BOF** и **EOF** возвращаемые значения.

Если параметр **BOF** или **EOF** свойство имеет **значение True**, нет нет текущей записи.

При открытии объекта **набора записей** нет записей, содержащий **BOF** и **EOF** было задано значение **True**, и объекта **набора записей** **RecordCount** свойство имеет значение 0. При открытии объекта **набора записей** , который содержит по крайней мере одна запись первой записи является текущей записи и свойства **BOF** и **EOF** — это **значение False**; они остаются **значение False** , пока не перемещать за пределы начала или окончания объекта **набора записей** с помощью **MovePrevious** или метод **MoveNext** , соответственно. При перемещении за пределы начале или конце **записей**нет текущей записи или запись не существует.

При удалении последней оставшиеся записи в объекте **набора записей** , свойства **BOF** и **EOF** может оставаться **значение False,** пока вы пытаетесь изменить положение текущей записи.

При использовании метода **MoveLast** на объект **набора записей** , содержащий записи Последняя запись становится текущей; При использовании метода **MoveNext** текущей записи становится недействительным и свойство **EOF** имеет значение **True**. С другой стороны при использовании метода **MoveFirst** на объект **набора записей** , содержащий записи первая запись становится текущей; При использовании метода **MovePrevious** нет текущей записи и **BOF** задано значение **True**.

Как правило при работе с всех записей в объекте **набора записей** код перебирает записей с помощью метода **MoveNext** , пока свойство **EOF** имеет значение **True**.

Если вы используете метод **MoveNext** , хотя свойство **EOF** установлено значение **True,** либо метод **MovePrevious** во время **BOF** задано значение **True**, возникает ошибка.

Метод **OpenRecordset** во внутренней сети вызывает метод **MoveFirst** . Таким образом с помощью метода **OpenRecordset** на пустой набор записей задает свойства **EOF** и **BOF** значение **True**. (См. в следующей таблице поведение неудачных метод **MoveFirst** .)
