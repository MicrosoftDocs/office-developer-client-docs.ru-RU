<span data-ttu-id="e9204-109">Например, предположим, что пользователь хочет запустить запрос.</span><span class="sxs-lookup"><span data-stu-id="e9204-109">For example, suppose a user wants to run a query.</span></span> <span data-ttu-id="e9204-110">Рассмотрим следующие сценарии:</span><span class="sxs-lookup"><span data-stu-id="e9204-110">Consider the following scenarios:</span></span>

  - <span data-ttu-id="e9204-111">Данные находятся в реляционной базе данных, для которой в настоящее время существует драйвер ODBC, но не является родным поставщиком OLE DB: приложение использует ADO для беседы с поставщиком OLE DB для ODBC, который загружает соответствующий драйвер ODBC.</span><span class="sxs-lookup"><span data-stu-id="e9204-111">The data resides in a relational database for which there currently exists an ODBC driver but no native OLE DB provider: The application uses ADO to talk to the OLE DB Provider for ODBC, which then loads the appropriate ODBC driver.</span></span> <span data-ttu-id="e9204-112">Водитель передает SQL в DBMS, который извлекает данные.</span><span class="sxs-lookup"><span data-stu-id="e9204-112">The driver passes the SQL statement to the DBMS, which retrieves the data.</span></span>

  - <span data-ttu-id="e9204-116">Данные находятся в файловой системе Microsoft NTFS в виде документов. <span data-ttu-id="e9204-117">Во всех предыдущих примерах приложение может запрашивать данные.</span><span class="sxs-lookup"><span data-stu-id="e9204-117">In all of the preceding examples, the application can query the data.</span></span> <span data-ttu-id="e9204-118">Потребности пользователя удовлетворены минимальным количеством компонентов.</span><span class="sxs-lookup"><span data-stu-id="e9204-118">The user's needs are met with a minimum number of components.</span></span> <span data-ttu-id="e9204-119">В каждом случае дополнительные компоненты используются только при необходимости, и вызываются только необходимые компоненты.</span><span class="sxs-lookup"><span data-stu-id="e9204-119">In each case, additional components are used only if needed, and only the required components are invoked.</span></span> <span data-ttu-id="e9204-120">Такая загрузка многоразовых и совместно используемых компонентов значительно способствует высокой производительности при работе с OLE DB.</span><span class="sxs-lookup"><span data-stu-id="e9204-120">This demand-loading of reusable and shareable components greatly contributes to high performance when OLE DB is used.</span></span>
