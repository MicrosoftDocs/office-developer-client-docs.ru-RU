<span data-ttu-id="9b302-109">Например, предположим, что пользователь хочет выполнить запрос.</span><span class="sxs-lookup"><span data-stu-id="9b302-109">For example, suppose a user wants to run a query.</span></span> <span data-ttu-id="9b302-110">Рассмотрим следующие сценарии:</span><span class="sxs-lookup"><span data-stu-id="9b302-110">Consider the following scenarios:</span></span>

  - <span data-ttu-id="9b302-111">Данные находятся в реляционной базе данных, для которой в настоящее время существует драйвер ODBC, но нет нативного поставщика OLE DB: приложение использует ADO для связи с поставщиком OLE DB для ODBC, который затем загружает соответствующий драйвер ODBC.</span><span class="sxs-lookup"><span data-stu-id="9b302-111">The data resides in a relational database for which there currently exists an ODBC driver but no native OLE DB provider: The application uses ADO to talk to the OLE DB Provider for ODBC, which then loads the appropriate ODBC driver.</span></span> <span data-ttu-id="9b302-112">Драйвер передает SQL dBMS, которая извлекает данные.</span><span class="sxs-lookup"><span data-stu-id="9b302-112">The driver passes the SQL statement to the DBMS, which retrieves the data.</span></span>

  - <span data-ttu-id="9b302-116">Данные находятся в файловой системе Microsoft NTFS в виде документов: доступ к данным обеспечивается с помощью поставщика OLE DB через службу индексации Майкрософт, который индексирует контент и свойства документов в файловой системе, чтобы обеспечить эффективный поиск контента.</span><span class="sxs-lookup"><span data-stu-id="9b302-116">The data resides in the Microsoft NTFS file system in the form of documents: Data is accessed by using a native OLE DB provider over Microsoft Indexing Service, which indexes the content and properties of documents in the file system to enable efficient content searches.</span></span>

<span data-ttu-id="9b302-117">Во всех предыдущих примерах приложение может запрашивать данные.</span><span class="sxs-lookup"><span data-stu-id="9b302-117">In all of the preceding examples, the application can query the data.</span></span> <span data-ttu-id="9b302-118">Потребности пользователя совмеются с минимальным количеством компонентов.</span><span class="sxs-lookup"><span data-stu-id="9b302-118">The user's needs are met with a minimum number of components.</span></span> <span data-ttu-id="9b302-119">В каждом случае дополнительные компоненты используются только при необходимости, и вызываются только необходимые компоненты.</span><span class="sxs-lookup"><span data-stu-id="9b302-119">In each case, additional components are used only if needed, and only the required components are invoked.</span></span> <span data-ttu-id="9b302-120">Такая загрузка повторно используемых и совместно используемых компонентов значительно способствует высокой производительности при работе с OLE DB.</span><span class="sxs-lookup"><span data-stu-id="9b302-120">This demand-loading of reusable and shareable components greatly contributes to high performance when OLE DB is used.</span></span>
