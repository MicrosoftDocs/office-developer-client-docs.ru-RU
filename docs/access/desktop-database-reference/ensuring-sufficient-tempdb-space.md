# <a name="ensuring-sufficient-tempdb-space"></a>Обеспечение наличия достаточного места в TempDB


**Область применения**: Access 2013, Office 2013

Если при обработке объектов Recordset, для обработки объектов [recordset,](recordset-object-ado.md) для обработки Microsoft SQL Server 6,5, могут возникнуть ошибки, может потребоваться увеличить размер TempDB. (Для некоторых запросов требуется временное пространство обработки; например, для запроса с предложением ORDER BY требуется определенный набор записей, для которого требуется некоторое временное пространство.)

> [!IMPORTANT]
> Прочитайте эту процедуру перед выполнением действий, так как сжать устройство после его расширения не так просто.

> [!NOTE]
> По умолчанию в Microsoft SQL Server 7.0 и более поздних, tempDB автоматически растет по мере необходимости. Следовательно, эта процедура может потребоваться только на серверах с более ранними версиями, чем 7.0.



**Увеличение пространства TempDB на SQL Server 6.5**

1.  Запустите Microsoft SQL Server Enterprise Manager, откройте дерево сервера, а затем откройте дерево "Устройства базы данных".

2.  Выберите (физическое) устройство для расширения, например "Мастер", и  дважды щелкните устройство, чтобы открыть диалоговое окно "Изменение устройства базы данных". В этом диалоговом окне показано, сколько места используется текущими базами данных.

3.  В поле **"Размер"** увеличите устройство до нужного объема (например, 50 мегабайт (МБ) места на жестком диске).

4.  Нажмите **кнопку "Изменить",** чтобы увеличить объем пространства, до которого может развернуться (логический) TempDB.

5.  Откройте дерево баз данных на сервере и дважды щелкните  **TempDB,** чтобы открыть диалоговое окно "Изменение базы данных". На **вкладке "База** данных" указан объем пространства, выделенного для TempDB **(размер данных).** По умолчанию это 2 МБ.

6.  В группе **"Размер"** нажмите кнопку **"Развернуть".** На графиках покажите доступное и выделенное пространство на каждом физическом устройстве. Полосы в цвете маруна представляют доступное пространство.

7.  Выберите устройство **журнала,** например "Хозяин", чтобы отобразить доступный размер в поле **"Размер ( МБ)".**

8.  Нажмите **кнопку "Развернуть сейчас",** чтобы выделить это пространство для базы данных TempDB. В **диалоговом** окне "Изменение базы данных" отображается новый выделенный размер для TempDB.

Дополнительные сведения об этой теме можно найти в Microsoft SQL Server справки enterprise Manager по запросу "Диалоговое окно "Развернуть базу данных".
