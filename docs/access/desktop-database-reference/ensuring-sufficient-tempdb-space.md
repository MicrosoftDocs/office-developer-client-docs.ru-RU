# <a name="ensuring-sufficient-tempdb-space"></a>Обеспечение наличия достаточного места в TempDB


**Область применения**: Access 2013, Office 2013

Если при обработке объектов [Recordset,](recordset-object-ado.md) для обработки Microsoft SQL Server 6.5, могут возникнуть ошибки, возможно, потребуется увеличить размер TempDB. (Некоторые запросы требуют временного пространства обработки; например, запрос с оговоркой ORDER BY требует своего рода **набор** записей, который требует некоторого временного пространства.)

> [!IMPORTANT]
> Перед выполнением действий ознакомьтесь с этой процедурой, так как сжать устройство после расширения не так просто.

> [!NOTE]
> По умолчанию в Microsoft SQL Server 7.0 и более поздних значениях tempDB будет автоматически расти по мере необходимости. Поэтому эта процедура может потребоваться только на серверах, работающих с версиями до 7.0.



**Увеличение пространства TempDB на SQL Server 6,5**

1.  Запустите Microsoft SQL Server Enterprise manager, откройте дерево для сервера и откройте дерево устройств базы данных.

2.  Выберите (физическое) устройство для расширения, например Master, и дважды щелкните устройство, чтобы открыть диалоговое окно **Edit Database Device.** В этом диалоговом окне показано, сколько места используют текущие базы данных.

3.  В **поле Размер** увеличите устройство до нужного количества (например, 50 мегабайт (МБ) пространства жесткого диска.

4.  Нажмите **кнопку Изменить сейчас,** чтобы увеличить количество пространства, на которое может расширяться (логический) TempDB.

5.  Откройте дерево Баз данных на сервере, а затем дважды щелкните **TempDB,** чтобы открыть диалоговое окно **Edit Database.** Вкладка **Database** перечисляет количество пространства, выделенного в настоящее время tempDB **(Размер данных).** По умолчанию это 2 МБ.

6.  В группе **Размер** нажмите кнопку **Расширение**. На графиках покажите доступное и выделенное пространство на каждом физическом устройстве. Полосы в бордовом цвете представляют доступное пространство.

7.  Выберите **устройство журнала**, например Master, для отображения доступных размеров в поле **Размер (МБ).**

8.  Нажмите **кнопку Развяжье** время, чтобы выделить это пространство в базу данных TempDB. Диалоговое **окно Edit Database** отображает новый выделенный размер для TempDB.

Дополнительные сведения по этому вопросу можно найти в файле справки Microsoft SQL Server Enterprise диспетчера в разделе "Расширение диалогового окна базы данных".
