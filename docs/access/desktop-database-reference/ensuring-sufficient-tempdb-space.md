# <a name="ensuring-sufficient-tempdb-space"></a>Обеспечение наличия достаточного места в TempDB


**Область применения**: Access 2013, Office 2013

Если при обработке объектов [Recordset](recordset-object-ado.md) , требующих обработки пространства в Microsoft SQL Server 6,5, возникли ошибки, может потребоваться увеличить размер базы данных tempdb. (В некоторых запросах требуется временная область обработки; например, для запроса с предложением ORDER BY требуется сортировка объекта **Recordset**, для которого требуется некоторое временное пространство.)

> [!IMPORTANT]
> ПроЧтите эту процедуру перед выполнением действий, так как это не просто сокращает устройство после его развертывания.

> [!NOTE]
> По умолчанию в Microsoft SQL Server 7,0 и более поздних версиях для базы данных TempDB задано автоматическое увеличение по мере необходимости. Таким образом, эта процедура может потребоваться только на серверах с более ранними версиями, чем 7,0.



**Увеличение пространства TempDB на сервере SQL Server 6,5**

1.  Запустите Microsoft SQL Server Enterprise Manager, откройте дерево для сервера, а затем откройте дерево устройств баз данных.

2.  Выберите (физическое) устройство, которое нужно развернуть, например master, и дважды щелкните устройство, чтобы открыть диалоговое окно **Изменение устройства базы данных** . В этом диалоговом окне показано, сколько места используется текущими базами данных.

3.  В поле **Размер** увеличьте устройство до нужного значения (например, 50 мегабайт (МБ) места на жестком диске).

4.  Нажмите кнопку **Изменить сейчас** , чтобы увеличить пространство, в течение которого (логическая) база данных tempdb может быть расширена.

5.  Откройте дерево базы данных на сервере, а затем дважды щелкните базу данных **tempdb** , чтобы открыть диалоговое окно **изменение базы данных** . На вкладке **база данных** отображается объем пространства, выделенный на данный момент для базы данных tempdb (**Размер данных**). По умолчанию это 2 МБ.

6.  В группе **Размер** нажмите кнопку **развернуть**. На графиках показан доступный и выделенный объем на каждом из физических устройств. Полосы в Марун Color представляют доступное место.

7.  Выберите **устройство для ведения журнала**, например master, чтобы отобразить доступный размер в поле **Размер (МБ)** .

8.  Нажмите **развернуть** , чтобы выделить пространство для базы данных tempdb. В диалоговом окне **изменение базы данных** отображается новый выделенный размер базы данных tempdb.

Для получения дополнительных сведений об этом разделе выполните поиск в файле справки Microsoft SQL Server Enterprise Manager для "диалоговое окно развертывания базы данных".
