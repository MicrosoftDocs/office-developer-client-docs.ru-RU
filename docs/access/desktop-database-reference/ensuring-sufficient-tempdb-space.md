# <a name="ensuring-sufficient-tempdb-space"></a>Обеспечение достаточно места для базы данных TempDB


**Применимо к**: Access 2013, Office 2013

При возникновении ошибок во время обработки объектов [наборов записей](recordset-object-ado.md) , которые должны обработки места на Microsoft SQL Server 6.5, может потребоваться увеличение размера базы данных TempDB. (Некоторые запросы требуют пространства временные обработки; например запрос с предложение ORDER BY требует сортировку набора **записей**, что требует временного место).

> [!IMPORTANT]
> Прочитайте эту процедуру перед выполнением действий, так как он не так просто уменьшения размера устройства, когда она развернута.

> [!NOTE]
> По умолчанию в Microsoft SQL Server 7.0 и более поздних версий базы данных TempDB присваивается автоматически увеличиваться по мере необходимости. Таким образом эта процедура может быть только необходимые на серверах под управлением версии меньше 7.0.



**Чтобы увеличить размер базы данных TempDB на SQL Server 6.5**

1.  Запустите Microsoft SQL Server Enterprise Manager, откройте дерево для сервера и откройте дерево устройств базы данных.

2.  Выберите устройство (физических) для развертывания, такие как главные и дважды щелкните устройство, чтобы открыть диалоговое окно **Изменение устройства базы данных** . Это диалоговое окно показывает объем пространства, используя текущей базы данных.

3.  В поле **размер** увеличение устройства требуемое число (например, 50 мегабайт (МБ) пространства жесткого диска).

4.  Нажмите кнопку **Изменить сейчас** , чтобы увеличить объем пространства, к которому можно развернуть (логическое) базы данных TempDB.

5.  Открытие дерева баз данных на сервере и затем дважды щелкните значок **базы данных TempDB** , чтобы открыть диалоговое окно **Изменение базы данных** . На вкладке **базы данных** перечислены дискового пространства, выделенный в текущий момент для базы данных TempDB (**Размер данных**). По умолчанию это 2 МБ.

6.  В разделе **размер** группы нажмите кнопку **Развернуть**. На графиках показано доступны и выделенного места на каждом из физических устройств. Столбцы в Темно-бордовая цвета представляют доступное место.

7.  Выберите **Журнала устройств**, такие как главные, чтобы отобразить доступные размер в поле **Размер (МБ)** .

8.  Нажмите кнопку **Развернуть теперь** распределения этого пространства для базы данных TempDB. Диалоговое окно **Изменение базы данных** отображаются новые выделенный размер для базы данных TempDB.

Дополнительные сведения по этому разделу найти в файле справки Microsoft SQL Server Enterprise Manager «Развернуть диалоговое окно базы данных.»
