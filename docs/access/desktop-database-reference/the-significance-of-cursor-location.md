# <a name="significance-of-cursor-location"></a>Важность положения курсора

**Область применения**: Access 2013, Office 2013

Каждый курсор использует временные ресурсы для хранения своих данных. Эти ресурсы могут быть памятью, файлом подкатки на диске, временными файлами дисков или даже временным хранилищем в базе данных. Курсор называется курсором на стороне клиента, если эти ресурсы находятся на клиентском компьютере.  Курсор называется курсором на стороне сервера, если эти ресурсы находятся на сервере. 

## <a name="client-side-cursors"></a>Client-Side курсоров

В ADO вызовите клиентский курсор с помощью **adUseClient** **CursorLocationEnum.** С помощью курсора на стороне клиента, не относяшегося к ключам, сервер отправляет весь набор результатов по сети на клиентский компьютер. Клиентский компьютер предоставляет временные ресурсы, необходимые курсору и набору результатов, и управляет ими. Клиентские приложения могут просматривать весь набор результатов, чтобы определить, какие строки ему нужны.

Статические курсоры на стороне клиента и на основе наборов ключей могут значительно загружать рабочие станции, если они содержат слишком много строк. Хотя все библиотеки курсоров могут строить курсоры с тысячами строк, приложения, предназначенные для получения таких больших строк, могут работать плохо. Конечно, существуют исключения. В некоторых приложениях большой курсор на стороне клиента может быть идеально подходящим, а производительность может не оказаться проблемой.

Одно из очевидных преимуществ курсора на стороне клиента — быстрое отклик. После загрузки набора результатов на клиентский компьютер просмотр строк будет очень быстрым. Как правило, ваше приложение более масштабируемо с помощью курсоров на стороне клиента, так как требования к ресурсам курсора размещаются на каждом отдельном клиенте, а не на сервере.

## <a name="server-side-cursors"></a>Server-Side курсоров

В ADO вызовите серверный курсор с помощью **adUseServer** **CursorLocationEnum.** С помощью курсора на стороне сервера сервер управляет набором результатов, используя ресурсы, предоставляемые серверным компьютером. Курсор на стороне сервера возвращает только запрашиваемую информацию по сети. Этот тип курсора иногда может обеспечить более качествевую производительность, чем курсор на стороне клиента, особенно в ситуациях, когда чрезмерное количество сетевого трафика является проблемой.

Однако важно отметить, что курсор на стороне сервера (по крайней мере временно) потребляет ресурсы сервера-сервера для каждого активного клиента. Необходимо соответствующим образом спланировать, чтобы серверное оборудование было способно управлять всеми серверными курсорами, запрашивающимися активными клиентами. Кроме того, курсор на стороне сервера может быть медленным, так как он обеспечивает доступ только к одной строке — пакетный курсор не доступен.

Курсоры на стороне сервера полезны при вставке, обновлении или удалении записей. С помощью курсоров на стороне сервера можно использовать несколько активных действий для одного подключения.
