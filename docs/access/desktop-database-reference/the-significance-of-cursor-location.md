# <a name="significance-of-cursor-location"></a>Важность положения курсора

**Область применения**: Access 2013, Office 2013

Каждый курсор использует временные ресурсы для хранения данных. Этими ресурсами могут быть объем памяти, файл подкачки диска, временные файлы на диске или даже временное хранилище в базе данных. Курсор называется *клиентским* курсором, если эти ресурсы находятся на клиентском компьютере. Курсор называется курсором на *стороне сервера* , когда эти ресурсы находятся на сервере.

## <a name="client-side-cursors"></a>Курсоры на стороне клиента

В ADO вызовите курсор на стороне клиента с помощью **адусеклиент** **курсорлокатионенум**. С помощью клиентского курсора, не относящегося к набору ключей, сервер отправляет весь набор результатов по сети на клиентский компьютер. Клиентский компьютер предоставляет и управляет временными ресурсами, необходимыми для курсора и набора результатов. Клиентское приложение может просматривать весь набор результатов, чтобы определить, какие строки требуются.

Статические и клиентские курсоры, управляемые набором ключей, могут помещать на рабочую станцию значительную нагрузку, если они содержат слишком много строк. Несмотря на то, что все библиотеки курсоров могут создавать курсоры с тысячами строк, приложения, предназначенные для извлечения таких больших наборов строк, могут работать плохо. Разумеется, есть исключения. Для некоторых приложений большой курсор на стороне клиента может быть вполне соответствующим, и производительность может быть недостаточной.

Одним из очевидных преимуществ клиентского курсора является быстрое реагирование. После загрузки набора результатов на клиентский компьютер Просмотр строк выполняется очень быстро. Приложение, как правило, лучше масштабируется с помощью клиентских курсоров, так как требования к ресурсам курсора размещаются на каждом отдельном клиенте, а не на сервере.

## <a name="server-side-cursors"></a>Курсоры на стороне сервера

В ADO вызовите курсор на стороне сервера с помощью **адусесервер** **курсорлокатионенум.** С помощью курсора на стороне сервера сервер управляет результирующим набором с помощью ресурсов, предоставляемых компьютером сервера. Серверный курсор возвращает только запрошенные данные в сети. Этот тип курсора иногда предоставляет лучшую производительность по сравнению с курсором на стороне клиента, особенно в ситуациях, когда проблема с избыточным сетевым трафиком.

Тем не менее, важно отметить, что серверный курсор — хотя бы временно, занимать ценные ресурсы сервера для каждого активного клиента. Необходимо спланировать соответствующим образом, чтобы оборудование сервера способно управлять всеми курсорами на стороне сервера, запрошенными активными клиентами. Кроме того, серверный курсор может быть медленным, так как он предоставляет только доступ к отдельным строкам, но нет доступного пакетного курсора.

Курсоры на стороне сервера удобно использовать при вставке, обновлении или удалении записей. С помощью курсоров на стороне сервера можно использовать несколько активных операторов в одном соединении.
