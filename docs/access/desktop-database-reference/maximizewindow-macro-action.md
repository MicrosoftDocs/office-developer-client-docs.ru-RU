# <a name="maximizewindow-macro-action"></a>Макрокоманда MaximizeWindow

**Область применения**: Access 2013, Office 2013

Если приложение Access настроено на использование перекрывающихся окон вместо документов с вкладками, можно использовать действие **максимизевиндов** , чтобы увеличить активное окно, чтобы оно заполнило окно Access. Это действие позволит увидеть максимально возможный объем объекта в активном окне.

> [!NOTE]
> Это действие невозможно применить к окнам кода в редакторе Visual Basic. Сведения о том, как повлиять на окна кода, представлены в разделе свойство **WindowState** .

## <a name="setting"></a>Параметр

Макрокоманда **максимизевиндов** не содержит аргументов.

## <a name="remarks"></a>Замечания

Это действие имеет тот же результат, что и нажатие кнопки **развернуть** в правом верхнем углу окна, или с помощью команды **развернуть** в меню **** оконного меню.

Вы можете использовать действие **ресторевиндов** для восстановления предыдущего размера развернутого окна.

> [!TIP]
> - Если окно, которое вы хотите развернуть, не является активным, может потребоваться использовать макрокоманду "ВыделитьОбъект" ( **SelectObject** ).
> - При развертывании окна в Access все остальные окна также размещаются в развернутом состоянии при открытии или переключении на них. Однако всплывающие формы не развернуты. Если вы хотите, чтобы форма состроила свой размер при разворачивании других окон, задайте для свойства **Popup свойство Popup** значение **Да**.

Чтобы запустить действие **максимизевиндов** в модуле Visual Basic для приложений, используйте метод **Maximize** объекта **DoCmd** .
