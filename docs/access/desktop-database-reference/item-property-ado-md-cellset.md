# <a name="item-property-ado-md-cellset"></a>Свойство Item (Cellset в ADO MD)

**Область применения**: Access 2013, Office 2013

Извлекает ячейку из ячейки с помощью ее координат.

## <a name="syntax"></a>Синтаксис

Set *Cell*  =  *Cellset*. Item (*Positions*)

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|:--------|:----------|
|*Positions* |Массив **значений Variant,** однозначно определяя ячейку. *Позиции могут* быть одним из следующих:<br/><br/>- Массив номеров позиции<br/>- Массив имен членов<br/>- Порядковая позиция |

## <a name="remarks"></a>Заметки

Используйте свойство **Item** для возврата объекта [Cell](cell-object-ado-md.md) в [объекте Cellset.](cellset-object-ado-md.md) Если **свойству Item** не удается найти ячейку, соответствующую аргументу *Positions,* возникает ошибка.

Свойство **Item** является свойством по умолчанию для объекта **Cellset.** Следующие синтаксис-формы являются взаимозаменяемыми:

```vb
    Cellset.Item ( Positions )
    Cellset ( Positions )
```

Аргумент *Positions* указывает возвращаемую ячейку. Ячейку можно указать по порядковому положению или по позиции вдоль каждой оси. При указании ячейки по позиции вдоль каждой оси можно указать числовую позицию или имена членов для каждой позиции.

Порядковая позиция — это число, уникальным образом идентифицирует одну ячейку в **cellset.** Концептуально ячейки нумеруются в  ячейках, как если бы ячейки были массивом *p-dimensional,* *где p* — это число осей.  Ячейки адресованы в порядке с основными строками.

Если имена членов передаются в качестве строк в **item,** они должны быть указаны в порядке увеличения числимого идентификатора оси. Внутри оси члены должны быть указаны в порядке увеличения вложенности измерений, то есть на первое место идет член внешнего измерения, за которым следуют члены внутренних измерений. Каждое измерение должно быть представлено отдельной строкой, а список строк членов должен разделяться запятой.

> Сбор ячеек по имени участника может не поддерживаться поставщиком данных. Дополнительные сведения см. в документации к поставщику.
