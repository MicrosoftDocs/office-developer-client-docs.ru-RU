# <a name="types-of-events"></a>Типы событий


**Применимо к**: Access 2013, Office 2013



Существует два основных типа событий. «Будет события,» вызываемые до начала операции, обычно включает «Будет» в именах — например, **WillChangeRecordset** или **WillConnect**. События, которые вызываются после выполнения события, обычно включают «Полная» в именах — например, **RecordChangeComplete** или **ConnectComplete**. Существуют исключения, например **InfoMessage** —, но это происходит после завершения операции.

## <a name="will-events"></a>Будет событий

Обработчики событий вызывается перед началом операции предлагают возможность проверить или изменить параметры операции и отменить операцию или разрешить его завершения. Эти процедуры обработчика событий обычно есть имена формы **будет*события ***.

## <a name="complete-events"></a>Завершить событий

Обработчики событий, вызывается после завершения операции может уведомлять приложение, которое предполагает операции. Такой обработчик событий также уведомления, когда обработчик события будет показано, как отменить ожидающие операции. Эти процедуры обработчика событий обычно есть имена формы ***событий * полная**.

Будет и завершения события обычно используются в пары.

## <a name="other-events"></a>Другие события

Обработчики событий — то есть, события, чьи имена не формы **будет * события*** или ***событий * полная —** вызываются только после завершения операции. Эти события, **Отключение**, **EndOfRecordset**и **InfoMessage**.
