# <a name="workspaceisolateodbctrans-property-dao"></a>Свойство Workspace. Исолатеодбктранс (DAO)


**Область применения**: Access 2013, Office 2013

Задает или возвращает значение, указывающее, изолированы ли множественные транзакции, использующие один и тот же источник данных ODBC, подключенный к ядру СУБД Microsoft Access (только для рабочих областей Microsoft Access).

## <a name="syntax"></a>Синтаксис

*Expression* . исолатеодбктранс

*expression*: переменная, представляющая объект **Workspace**.

## <a name="remarks"></a>Примечания

В некоторых случаях требуется несколько одновременных транзакций для одного и того же подключения ODBC. Для этого необходимо открыть отдельную **рабочую область** для каждой транзакции. Несмотря на то, что каждая **Рабочая область** может иметь собственное подключение ODBC к базе данных, производительность системы снижается. Так как изолированная изоляция транзакций обычно не обязательна, подключения ODBC из нескольких объектов **рабочей области** , открытых одним и тем же пользователем, совместно используются по умолчанию.

Некоторые серверы ODBC, например Microsoft SQL Server, не разрешают одновременные транзакции для одного подключения. Если для такой базы данных требуется более одной транзакции, установите для свойства **исолатеодбктранс** значение **true** для каждой **рабочей области** сразу после ее открытия. Это приводит к выделению отдельного подключения ODBC для каждой **рабочей области**.
