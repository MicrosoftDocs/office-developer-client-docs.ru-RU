# <a name="transfersqldatabase-macro-action"></a>Действия макроса TransferSQLDatabase


**Применимо к**: Access 2013 | Office 2013

В проекте Microsoft Access **«ПереносБазыДанныхSQL»** можно использовать для передачи Microsoft SQL Server 7.0 или более поздняя версия базы данных на другой SQL Server 7.0 или более поздняя версия базы данных. Дополнительные сведения о переносе базы данных SQL Server см.


> [!NOTE]
> 
		Эта действие не разрешено, если база данных не является доверенной. Дополнительные сведения о включении макросов см. по ссылкам в разделе See Also этой статьи.

## <a name="remarks"></a>Замечания

Нельзя выполнять другие операции во время передачи базы данных.

**«ПереносБазыДанныхSQL»,** по умолчанию, копирование данных, определений данных, объектов базы данных и расширенные свойства, такие как значения по умолчанию, текстовые ограничения и значений подстановки.

Существует требования для переноса базы данных.

  - Необходимо быть членом роли sysadmin на конечном сервере (нет специальных роль необходима на исходном сервере).

<!-- end list -->

  - Текущий сервер SQL, подключенный к проекту доступа и на целевой сервер, на которого переключается базы данных должен быть SQL Server версии 7.0 или более поздней версии.


> [!NOTE]
> <P>Связанные серверы не переносятся во время операции переноса базы данных.</P>



Чтобы запустить **«ПереносБазыДанныхSQL»** в Visual Basic для приложений (VBA) модуль, используйте метод **TransferSQLDatabase** **объекта** .
