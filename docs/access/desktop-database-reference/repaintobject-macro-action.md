# <a name="repaintobject-macro-action"></a>Макрокоманда RepaintObject

**Область применения**: Access 2013, Office 2013

Действие **RepaintObject** можно использовать для завершения любых ожидающих обновлений экрана для указанного объекта базы данных или активного объекта базы данных, если не задан ни один из них. Такие обновления включают любые ожидающих пересчетов для элементов управления объекта.

## <a name="remarks"></a>Заметки

Microsoft Access ожидает завершения ожидающих обновлений экрана до завершения других ожидающих задач. С помощью этого действия можно принудительно принудительно переопределять элементы управления в указанном объекте. Вы можете использовать это действие:

- При использовании действия **SetValue** для изменения значений в ряде элементов управления. Доступ может не показывать изменения немедленно, особенно если другие элементы управления (например, вычисляются элементы управления) зависят от значений измененных элементов управления.

- Если вы хотите убедиться, что просматриваемая форма отображает данные во всех своих средствах управления. Например, элементы управления, содержащие объекты OLE, не отображают свои данные сразу после открытия формы.

> [!NOTE]
> - Это действие не вызывает запрос базы данных, поэтому оно не показывает новые и измененные записи или не удаляет удаленные записи из таблицы или запроса объекта. Используйте действие **Requery** для повторного иска источника объекта или одного из его элементов управления. Используйте действие **ShowAllRecords** для отображения последних записей и удаления примененных фильтров.
> - Действие **RepaintObject** не имеет того же эффекта, что  и нажатие кнопки "Обновить" в группе "Записи" на вкладке "Главная", где отображаются все изменения, внесенные вами или другими пользователями в текущий момент отображаемой записи в формах и таблицах данных.  

Чтобы запустить действие **RepaintObject** в модуле Visual Basic для приложений (VBA), используйте метод **RepaintObject** объекта **DoCmd.**
