Вы можете использовать действие **SendKeys** для отправки клавиш непосредственно в Microsoft Access или в Windows приложение.

> [!NOTE]
> Эта макрокоманда доступна только для доверенных баз данных.

## <a name="remarks"></a>Примечания

Доступ обрабатывает нажатие клавиш, которые он получает с помощью действия **SendKeys,** точно так же, как если бы вы впечатали их непосредственно в окне Access.

Чтобы указать клавиши, используйте тот же синтаксис, что и для заявления **SendKeys.**

> [!NOTE]
> Ошибка может возникнуть, если аргумент **Keystrokes** содержит неправильный синтаксис, неправильный текст или другие значения, которые не подходят для окна, в которое отправляются клавиши.

Это действие можно использовать для ввода сведений в диалоговом окне, особенно если вы не хотите прерывать макрос, чтобы вручную ответить на диалоговое окно. Некоторые действия Access, такие как **PrintOut** и **FindRecord,** автоматически выбирают параметры в некоторых часто используемых диалоговом окнах. Вы можете использовать **действие SendKeys** для выбора параметров в менее часто используемых диалоговом окнах.

> [!NOTE]
> - Так как диалоговое окно приостанавливать макрос, необходимо поместить действие **SendKeys** перед действием, из-за которое диалоговое окно откроется, и заставьте аргумент **Wait** **no**.
> - Сроки нажатия клавиш до доступа или другого приложения могут быть сложными. В результате рекомендуется использовать другой метод (например, действие **FindRecord)** для достижения нужной задачи, а не использовать действие **SendKeys** для заполнения параметров в диалоговом окне.

Если вы хотите отправить более 255 символов в access или другое Windows на основе приложения, вы можете использовать несколько действий **SendKeys** последовательно в макрос.

Использование действия **SendKeys** для отправки нажатия клавиш вызывает соответствующие события **KeyDown,** **KeyUp** и **KeyPress.** Отправка клавиш, не в которые не велись anSI (например, клавиша функции), не вызывает **событие KeyPress.**

Это действие не доступно из модуля Visual Basic для приложений (VBA). Вместо этого используйте **утверждение SendKeys.**
