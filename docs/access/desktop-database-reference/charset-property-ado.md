# <a name="charset-property-ado"></a>Свойство Charset (ADO)


**Область применения**: Access 2013, Office 2013

Указывает набор символов, в который содержимое текстового [потока](stream-object-ado.md) необходимо преобразовать для хранения в внутреннем буфере объектов потока.

## <a name="settings-and-return-values"></a>Параметры и возвращаемые значения

Задает или возвращает **строковое** значение, задающее набор символов, в который будет переводиться содержимое **потока** . Значение по умолчанию — "Unicode". Допустимые значения — типичные строки, передаваемые через интерфейс в виде строк набора символов Интернета (например, "ISO-8859-1", "Windows-1252" и т. д.). Список строк набора символов, который называется системой, представлен в разделе\_подразделы набора символов\_корневой\\базы данных\\\\MIME в реестре Windows.

## <a name="remarks"></a>Примечания

В объекте текстового **потока** текстовые данные хранятся в наборе символов, указанном в свойстве **CharSet** . Значение по умолчанию: Unicode. Свойство **CharSet** используется для преобразования данных, поступающих в **поток** или выходя из **потока**. Например, если **поток** содержит данные ISO-8859-1, и эти данные будут скопированы в строку BSTR, объект **Stream** преобразует данные в Юникод. Обратное также имеет значение true.

Для открытого **потока**текущая [позиция](position-property-ado.md) должна находиться в начале **потока** (0), чтобы можно было задать набор **символов**.

**CharSet** используется только с объектами текстового **потока** ([Type](type-property-ado-stream.md) — **адтипетекст**). Это свойство игнорируется, если **тип** — **адтипебинари**.
