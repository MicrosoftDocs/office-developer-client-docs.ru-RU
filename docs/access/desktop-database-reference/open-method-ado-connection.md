# <a name="open-method-ado-connection"></a>Метод Open (Connection в ADO)

**Область применения**: Access 2013, Office 2013
 
Открывает подключение к источнику данных.

## <a name="syntax"></a>Синтаксис

*подключение*. Open *ConnectionString*, *UserID*, *Пароль*, *Параметры*

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|:--------|:----------|
|*ConnectionString* |Необязательный параметр. Значение **String,** содержаное сведения о подключении. Сведения о допустимом параметре см. в свойстве [ConnectionString.](connectionstring-property-ado.md)|
|*UserID* |Необязательный параметр. Значение **String,** которое содержит имя пользователя, которое необходимо использовать при создании подключения.|
|*Password* |Необязательный параметр. Значение **String,** которое содержит пароль, который необходимо использовать при создании подключения.|
|*Options* |Необязательный параметр. Установлено [значение ConnectOptionEnum,](connectoptionenum.md) которое определяет, должен ли этот метод возвращаться после (синхронно) или до (асинхронно) подключения.|

## <a name="remarks"></a>Примечания

Использование метода **Open** на [объекте Подключение](connection-object-ado.md) устанавливает физическое подключение к источнику данных. После успешного завершения этого метода подключение будет подключено в прямом эфире, и вы сможете выдавать ему команды и обрабатывать результаты.

Используйте необязательный аргумент *ConnectionString,* чтобы указать строку подключения, содержащую серию аргументов  *=* операторов значений, разделенных зайколонами, или файл или ресурс каталога, идентифицированный с URL-адресом. Свойство **ConnectionString** автоматически наследует значение, используемую для *аргумента ConnectionString.* Поэтому перед открытием объекта ConnectionString можно  либо задать свойство **ConnectionString,** либо использовать аргумент *ConnectionString* для задания или переопределения текущих параметров подключения во время вызова метода **Open.**

Если вы передаете сведения о пользователях и паролях как в *аргументе ConnectionString,* так и в необязательных аргументах *UserID* и *Password,* аргументы *UserID* и *Password* переопределяют значения, указанные в *ConnectionString.*

Когда вы завершили операции по открытому подключению, используйте метод [Close,](close-method-ado.md) чтобы освободить все связанные ресурсы системы. Закрытие объекта не удаляет его из памяти; Вы можете изменить его параметры свойств и использовать метод **Open,** чтобы открыть его снова позже. Чтобы полностью исключить объект из памяти, установите переменную объекта *в Nothing*.

**Удаленное использование службы данных** При этом метод **Open** не устанавливает подключение к серверу до открытия [recordset](recordset-object-ado.md) на  **объекте Подключение.**
