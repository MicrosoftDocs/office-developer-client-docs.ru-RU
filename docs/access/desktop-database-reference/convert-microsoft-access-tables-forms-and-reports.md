# <a name="convert-microsoft-access-tables-forms-and-reports"></a>Преобразование таблиц, форм и отчетов Microsoft Access

**Область применения**: Access 2013, Office 2013

Некоторые изменения, внесенные в Microsoft Access 2002, могут повлиять на поведение вашей версии 1. *приложения x* или 2.0; В следующих разделах приводится дополнительные сведения об этих изменениях.

## <a name="indexes-and-relationships"></a>Индексы и связи

Таблица Microsoft Access может содержать до 32 индексов. Очень сложные таблицы, которые являются частью многих связей, могут превысить ограничение индекса, и вы не сможете преобразовать базу данных, которая содержит эти таблицы. Яд баз данных Microsoft Access создает индексы с обеих сторон отношений между таблицами. Если база данных не будет преобразована, удалите некоторые связи и попробуйте еще раз преобразовать базу данных.

## <a name="the-limittolist-property-of-combo-boxes"></a>Свойство LimitToList полей со списком

В Microsoft Access 2002 или более поздней, поля со списком принимают значения **NULL,** если свойство **LimitToList** имеет значение **True** (–1), независимо от того, содержит ли список значения **NULL.** В версии 2.0 поле со списком со свойством **LimitToList** со значением **True** не будет принимать значение **NULL,** если список не содержит значение **NULL.** Если вы хотите запретить пользователям вводить значение **NULL** с помощью  поля со значением "Да", установите для обязательного свойства поля в **таблице значение "Да".**

## <a name="menus-and-in-place-activation-of-ole-objects"></a>Меню и активация объектов OLE на месте

Чтобы сделать дополнительные функции доступными при активации объектов OLE на месте, некоторые команды меню могли быть перемещены в меню, которое не заменяется при активации сервера OLE.

Макрос в преобразованном приложении, который использует действие DoMenuItem для выполнения команды меню версии 2.0 при активации компонента, не будут затронуты изменениями. Команды версии 2.0 созначяются с их эквивалентами в более поздних версиях Microsoft Access.

## <a name="referencing-a-control-on-a-read-only-form"></a>Ссылка на управление в форме только для чтения

В Microsoft Access 2002 или более поздней форме нельзя использовать выражение для ссылки на значение управления в форме, доступной только для чтения, привязанной к пустому источнику записей. В предыдущих версиях выражение возвращает значение **NULL.** Прежде чем ссылаться на управление в форме только для чтения, убедитесь, что источник записей формы содержит записи.

## <a name="date-fields-and-data-entry"></a>Поля даты и запись данных

При вводе **3/3** в поле типа Date в форме или таблице таблицы текущий год автоматически добавляется в Microsoft Access 2002 или более поздней. Однако если ввести **3/3/** в том же поле, Microsoft Access возвращает сообщение об ошибке. Необходимо опустить последний разграничив дату, чтобы Microsoft Access можно было перевести дату в правильный формат.

## <a name="buttons-created-with-the-command-button-wizard"></a>Кнопки, созданные с помощью мастера командной кнопки

Если вы использовали мастер командной кнопки в версии 2.0 или 7.0 Microsoft Access для создания кода, который вызвал другое приложение, необходимо удалить кнопку и повторно создать ее с помощью мастера командной кнопки в Microsoft Access 2002 или более поздней версии.

## <a name="form-and-report-class-modules"></a>Модули классов форм и отчетов

В версиях Microsoft Access до 2002 года объекты **Form** и **Report** имеют связанные модули классов, даже если за объектом нет кода. В Microsoft Access 2002 или более поздней можно установить для свойства **HasModule** формы или отчета false **.** Если свойству **HasModule** задано **false,** форма или отчет займет меньше места на диске и будет загружаться быстрее, так как у нее больше нет связанного модуля класса.

## <a name="converted-version-20-report-has-different-margins"></a>Преобразованный отчет версии 2.0 имеет разные поля

При попытке распечатать или просмотреть отчет в Microsoft Access 2002 или более поздней версии, преобразованный из Microsoft Access 2.0, могут возникнуть проблемы, если в отчете есть поля 0. При преобразовании отчета Microsoft Access 2.0 поля не устанавливаются на 0; вместо этого им устанавливается минимальное поле, допустимый для принтера по умолчанию. Это предотвращает печать данных в непечатаемой области принтера.

Чтобы устранить эту проблему, уменьшите ширину столбца, интервал столбцов или число столбцов в отчете, чтобы ширина столбцов и ширина полей по умолчанию были равны или меньше ширины бумаги.

## <a name="cant-use-the-format-property-to-distinguish-null-values-and-zero-length-strings"></a>Не позволяет использовать свойство Format, чтобы различать значения NULL и строки нулевой длины

В версиях 1. *X* и 2,0 можно использовать свойство **Format** для отображения различных значений для значений **NULL** и строк нулевой длины (" "). В Microsoft Access 2002 или более поздней, чтобы различать значения **NULL** и строки нулевой длины в объекте управления в  форме, установите для свойства **controlSource** выражение, которое проверяет на нулевое значение. Например, чтобы отобразить "Null" или "ZLS" в объекте управления, установите его свойство **ControlSource** следующим выражением:

`=IIf(IsNull([MyControl]), "Null", Format([MyControl], "@;ZLS"))`
