# <a name="requery-macro-action"></a>Макрокоманда Requery

**Область применения**: Access 2013, Office 2013

Вы можете использовать действие **Requery** для обновления данных в указанном объекте, затевев источник этого объекта. Если ни один из них не указан, это действие повторно затмеет источник самого объекта. Это действие позволяет убедиться, что активный объект или один из его элементов управления отображает самые актуальные данные.

## <a name="remarks"></a>Заметки

Действие **Requery** делает одно из следующих действий:

- Повторно передает запрос, на котором основан этот объект или этот объект.

- Отображает новые или измененные записи и удаляет все удаленные записи из таблицы, на которой основан этот объект или объект.

> [!NOTE]
> Действие **Requery** не влияет на положение указателя записи.

Элементы управления, основанные на запросе или таблице:

- Списки и поля со списком.

- Подформатные элементы управления.

- Объекты OLE, например диаграммы.

- Элементы управления, содержащие агрегатные функции домена, такие как **DSum.**

Если указанный контроль не основан на запросе или таблице, это действие привнося в него пересчет.

Если оставить аргумент **Control Name** пустым, действие **Requery** будет действовать так же, как нажатие shift+F9, когда объект находится в фокусе. Если фокус имеется на подформе, это действие повторно заимещает только источник подформы (так же, как нажатие shift+F9).

> [!NOTE]
> Действие **Requery** повторно затевеет источник объекта или управления. С другой стороны, действие **RepaintObject** переопректирует элементы управления в указанном объекте, но не повторно заискирует базу данных и не отображает новые записи. Действие **ShowAllRecords** не только повторно затребует активный объект, но и удаляет все примененные фильтры, чего действие **Requery** не делают.

Если вы хотите повторно заиметь объект, который не находится в активном объекте, необходимо использовать метод **Requery** в модуле Visual Basic для приложений (VBA), а не действие **Requery** или соответствующий метод **Requery** объекта **DoCmd.** Метод **Requery** в VBA быстрее, чем действие **Requery** или **метод DoCmd.Requery.** Кроме того, при использовании действия **Requery** или **метода DoCmd.Requery** Microsoft Access закрывает запрос и перезагружает его из базы данных, но при использовании метода **Requery** Access повторно передает запрос, не закрывая и не перезагружая его. Обратите внимание, что ActiveX **Requery** объекта данных (ADO) работает так же, как метод Access **Requery.**
