# <a name="requery-macro-action"></a>Повторный запрос действия макроса


**Применимо к**: Access 2013, Office 2013

Действие **запроса** можно использовать для обновления данных в указанный элемент управления для текущего объекта повторным запросом источника элемента управления. Если элемент не указан, это действие вызывает повторный запрос источника самого объекта. Используйте это действие для убедитесь, что объекте или в элементе управления отображает текущих данных.

## <a name="remarks"></a>Замечания

Действие **повторный запрос** выполняет одно из следующих действия.

  - Повторно выполняет запрос, на котором основан элемент управления или объекта.

  - Отображает все новые или измененные записи и удаляет все удаленные записи из таблицы, на котором основан элемент управления или объекта.


> [!NOTE]
> <P>Действие <STRONG>повторный запрос</STRONG> не влияет на положение указателя записи.</P>



Следующие элементы управления на основе запроса или в таблице:

  - Списки и поля со списками.

  - Подчиненная форма элементов управления.

  - Объекты OLE, такие как диаграммы.

  - Элементы управления, содержащий домен агрегатных функций, таких как **DSum**.

Если указанный элемент управления не на основе запроса или таблицы, приводит пересчета элемента управления.

Если аргумент **Имя элемента управления** не заполнено, действие **повторный запрос** имеет тот же эффект, как при нажатии клавиши SHIFT + F9, когда объект имеет фокус. Если элемент управления подчиненной формы имеет фокус, это действие вызывает повторный запрос только источника данных подчиненной формы (так же, как при нажатии клавиши SHIFT + F9).


> [!NOTE]
> <P>Действие <STRONG>повторный запрос</STRONG> вызывает повторный запрос источника элемента управления или объекта. С другой стороны <STRONG>ОбновитьОбъект</STRONG> перерисует элементов управления в указанном объекте, но не повторный запрос базы данных или отображения новых записей. <STRONG>ПоказатьВсеЗаписи</STRONG> не только вызывает повторный запрос активного объекта, но оно также удаляет все примененные фильтры, которых не выполняет действие <STRONG>повторный запрос</STRONG> .</P>



Если требуется повторный запрос элемент управления, который не для текущего объекта, необходимо использовать метод **запроса** в Visual Basic для приложений (VBA) модуль, не действие **повторный запрос** или его соответствующий метод **повторный запрос** **объекта** . Выполняется быстрее, чем действие **повторный запрос** или метод **DoCmd.Requery** **повторного VBA** . Кроме того при использовании действие **повторный запрос** или метод **DoCmd.Requery** Microsoft Access закрывает запрос и перезагружает из базы данных, но при использовании метода **повторный запрос** повторно выполняет запрос без закрытия и перезагрузки. Обратите внимание на то, что метод объектов данных ActiveX (ADO) **повторный запрос** работает так же, как метод доступа **повторный запрос** .
