# <a name="name-property-ado"></a>Свойство Name (ADO)


**Область применения**: Access 2013, Office 2013

Указывает имя объекта.

## <a name="settings-and-return-values"></a>Параметры и возвращаемые значения

Задает или возвращает **строковое** значение, указывающее имя объекта.

## <a name="remarks"></a>Замечания

Используйте свойство **Name** для присвоения имени или извлечения имени **команды**, **Свойства**, **поля**или объекта **параметра** .

Значение доступно для чтения и записи в объекте **Command** и доступно только для чтения для объекта **Property** .

Для объекта **поля** **имя** обычно имеет значение "только для чтения". Однако для новых объектов **field** , добавленных в коллекцию [Fields](fields-collection-ado.md) [записи](record-object-ado.md), **имя** доступно для чтения и записи только после того, как было указано свойство [value](value-property-ado.md) для **поля** и у поставщика данных новое **поле** успешно добавлено путем вызова метода [Update](update-method-ado.md) коллекции Fields. ****

Для объектов **Parameter** , еще не добавленных в коллекцию [Parameters](parameters-collection-ado.md) , свойство **Name** доступно для чтения и записи. Для добавленных объектов **Parameter** и всех других объектов свойство **Name** доступно только для чтения. Имена не обязательно должны быть уникальными в пределах коллекции.

Вы можете получить свойство **Name** объекта по порядковому номеру, после чего вы можете ссылаться на объект напрямую по имени. Например, если Рстмаин. Properties (20). Name дает возможность обновления, поэтому вы можете ссылаться на это свойство в качестве результата обновления, поэтому вы можете ссылаться на это свойство как Рстмаин. Properties ("обновление").
