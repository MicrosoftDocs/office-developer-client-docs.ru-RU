# <a name="movefirst-movelast-movenext-and-moveprevious-methods-ado"></a>Методы MoveFirst, MoveLast, MoveNext и MovePrevious (ADO)


**Область применения**: Access 2013, Office 2013

Перемещается к первой, последней, следующей или предыдущей записи в указанном объекте [Recordset](recordset-object-ado.md) и записи текущей записи.

## <a name="syntax"></a>Синтаксис

*набор записей.* { MoveFirst | MoveLast | MoveNext | MovePrevious}

## <a name="remarks"></a>Примечания

Используйте метод **MoveFirst,** чтобы переместить текущее положение записи в первую запись **в Наборе записей.**

Используйте метод **MoveLast,** чтобы переместить текущее положение записи до последней записи в **Наборе записей.** Объект **Recordset** должен поддерживать закладки или движение обратного курсора; в противном случае вызов метода создает ошибку.

Вызов в **MoveFirst** или **MoveLast** при пустом наборе записей (как **BOF,** так и **EOF** являются true) создает ошибку. 

Используйте метод **MoveNext,** чтобы переместить текущее положение записи на одну запись вперед (в нижней части **наборов записей).** Если последняя запись — это текущая запись, и вы вызываете метод **MoveNext,** ADO задает текущую запись позиции после последней записи в **Recordset** [(EOF](bof-eof-properties-ado.md) — **true).** Попытка двигаться вперед, когда **свойство EOF** уже **является True,** создает ошибку.

В случаях, когда **набор записей** фильтруется или сортироваться, а данные текущей записи изменяются, может также измениться положение. В таких случаях метод **MoveNext** работает нормально, но следует помнить, что позиция перемещается на одну запись вперед из новой позиции, а не из старой позиции. Например, изменение данных в текущей записи, например, перемещение записи до конца отсортировали **Recordset,** будет означать, что вызов **MoveNext** приводит к тому, что ADO устанавливает текущую запись на позицию после последней записи в **Recordset** **(EOF**  =  **True).**

Используйте метод **MovePrevious,** чтобы переместить текущее положение записи на одну запись назад (в верхней части **наборов записей).** Объект **Recordset** должен поддерживать закладки или движение обратного курсора; в противном случае вызов метода создает ошибку. Если первая запись — это текущая запись, и вы вызываете метод **MovePrevious,** ADO задает текущую запись позиции перед первой записью в **Recordset** [(BOF](bof-eof-properties-ado.md) **true).** Попытка переместить назад, когда **свойство BOF** уже **является True,** создает ошибку. Если объект **Recordset** не поддерживает ни закладки, ни движение курсора назад, метод **MovePrevious** создает ошибку.

Если только **набор записей** будет только вперед и вы хотите поддерживать как вперед, так и назад прокрутку, вы можете использовать свойство [CacheSize](cachesize-property-ado.md) для создания кэша записей, который будет поддерживать движение обратного курсора с помощью метода [Move.](move-method-ado.md) Так как кэшировать записи загружаются в память, следует избегать кэшировать больше записей, чем это необходимо. Метод **MoveFirst** можно вызвать в объекте **Recordset** только для нападающих; это может привести к тому, что поставщик повторно выполнит команду, сгенерированную **объектом Recordset.**
