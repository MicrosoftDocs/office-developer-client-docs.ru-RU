# <a name="movefirst-movelast-movenext-and-moveprevious-methods-ado"></a>Методы MoveFirst, MoveLast, MoveNext и MovePrevious (ADO)


**Область применения**: Access 2013, Office 2013

Перемещается к первой, последней, следующей или предыдущей записи в указанном объекте [Recordset](recordset-object-ado.md) и делает запись текущей.

## <a name="syntax"></a>Синтаксис

*recordset*. { MoveFirst | MoveLast | MoveNext | MovePrevious}

## <a name="remarks"></a>Заметки

Используйте метод **MoveFirst** для перемещения текущей записи в первую запись в **наборе записей.**

Используйте метод **MoveLast** для перемещения текущей записи в последнюю запись в **наборе записей.** Объект **Recordset должен** поддерживать закладки или движение курсора назад; в противном случае вызов метода создает ошибку.

Вызов **MoveFirst** или **MoveLast,** когда набор **recordset** пуст (и **BOF,** и **EOF** имеют true) генерирует ошибку.

Используйте метод **MoveNext,** чтобы переместить текущую запись на одну запись вперед (в нижнюю часть **recordset).** Если последняя запись является текущей и вызовите метод **MoveNext,** ADO устанавливает текущую запись в положение после последней записи в **наборе записей** [(EOF](bof-eof-properties-ado.md) имеет **true).** Попытка двигаться вперед, когда свойство **EOF** уже имеет свойство **True,** вызывает ошибку.

Если набор **записей** был отфильтрован или отсортован и данные текущей записи изменены, положение также может измениться. В таких случаях метод **MoveNext** работает нормально, но следует помнить, что положение перемещается на одну запись вперед от новой позиции, а не от старой позиции. Например, изменение данных в текущей записи таким образом, чтобы она была перемещена в конец отсортировали набор **записей, означает,** что вызов **MoveNext** приводит к тому, что ADO устанавливает текущую запись на позицию после последней записи в наборе записей **(** **EOF**  =  **True).**

Используйте метод **MovePrevious,** чтобы переместить текущую запись на одну запись назад (в верхнюю часть **recordset).** Объект **Recordset должен** поддерживать закладки или движение курсора назад; в противном случае вызов метода создает ошибку. Если первая запись является текущей записью и вызовите метод **MovePrevious,** ADO устанавливает текущую запись в положение перед первой записью в **наборе записей** [(BOF](bof-eof-properties-ado.md) **имеет** true). Попытка перейти назад, когда свойство **BOF** уже имеет свойство **True,** вызывает ошибку. Если объект **Recordset** не поддерживает закладки или движение курсора в обратном направлении, метод **MovePrevious** создает ошибку.

Если набор **записей** поддерживает только прокрутку вперед и назад, можно использовать свойство [CacheSize](cachesize-property-ado.md) для создания кэша записей, который будет поддерживать перемещение курсора назад с помощью метода [Move.](move-method-ado.md) Так как кэшные записи загружаются в память, не следует кэшировать больше записей, чем необходимо. Вы можете вызвать метод **MoveFirst** в объекте **Recordset** только для переад. Это может привести к повторному выполнению поставщиком команды, которая вызвала **объект Recordset.**
