# <a name="parameterized-commands-with-intervening-compute-commands"></a>Параметризованные команды с промежуточными командами COMPUTE


**Область применения**: Access 2013, Office 2013

В обычной команде С параметром APPEND ДОБАВЛЯЕТся предложение, которое создает родительский **набор записей** с помощью команды запроса и другое предложение, которое создает доЧерний **набор записей** с помощью команды параметризованного запроса, то есть команды, содержащей параметр заполнитель (вопросительный знак, "_Км_"). Полученный в форме **набор записей** имеет два уровня, в которых родитель находится на верхнем уровне, а дочерний — на более низком.

Предложение, которое создает дочерний **набор записей** , теперь может быть произвольным числом вложенных команд COMPUTE Shape, где самая глубокая вложенная команда содержит параметризованный запрос. Полученный в форме **набор записей** имеет несколько уровней, в которых родитель находится на самом верхнем уровне, дочерний элемент занимает уровень ловермост, а с помощью небольшого количества **записей**, создаваемых командами вычисления фигуры, занимают промежуточный тонов.

Как правило, эта функция используется для вызова функции статистической функции и группирования команд Shape COMPUTE, позволяющих создавать промежуточные объекты **Recordset** с помощью аналитических сведений о доЧернем **наборе записей**. Кроме того, поскольку это команда параметризованной фигуры, к которой обращается столбец главы родительского объекта, может быть извлечен новый дочерний **набор записей** . Так как промежуточные уровни являются производными от дочернего, они также будут пересчитаны.
