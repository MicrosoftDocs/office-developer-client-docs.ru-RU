# <a name="role-of-ado-in-microsoft-data-access"></a>Роль ADO в Microsoft Data Access

**Область применения**: Access 2013, Office 2013

Компоненты microsoft Data Access (MDAC) предоставляют доступ к данным, который не зависит от хранилищ данных, средств и языков. Он предоставляет высокоуровневый, простой в использовании интерфейс и низкоуровневый высокоскоростной интерфейс для практически любого доступного хранения данных. Вы можете использовать эту гибкость для интеграции разнообразных хранилищ данных и выбора инструментов, приложений и служб платформы для создания правильных решений для ваших потребностей. Эти технологии предоставляют базовую структуру для доступа к данным общего назначения в операционных системах Microsoft Windows.

В MDAC существует три основных технологии. ActiveX data Objects (ADO) — это высокоуровневый, простой в использовании интерфейс для OLE DB. OLE DB — это низкоуровневый высокоуровневый интерфейс для различных хранилищ данных. ADO и OLE DB могут работать с реляционными (табличные) и нерациональными (иерархическими или потоками) данными. Наконец, open Database Connectivity (ODBC) — это еще один низкоуровневый высокоуровневый интерфейс, разработанный специально для реляционных хранилищ данных.

ADO предоставляет уровень абстракции между клиентом или приложением среднего уровня и низкоуровневым интерфейсом OLE DB. ADO использует небольшой набор объектов автоматизации для предоставления простого и эффективного интерфейса OLE DB. Этот интерфейс делает ADO идеальным выбором для разработчиков на языках более высокого уровня, таких как Visual Basic и даже VBScript, которые хотят получить доступ к данным, не изучив тонкости COM и OLE DB.
