# <a name="role-of-ado-in-microsoft-data-access"></a>Роль ADO в Microsoft Data Access

**Область применения**: Access 2013, Office 2013

Компоненты доступа к данным MDAC предоставляют доступ к данным независимо от хранилищ, средств и языков. Он предоставляет высокоуровневый, простой в использовании интерфейс и высокоуровневый высокопроизводительный интерфейс для практически любого доступного хранилища данных. Эту гибкость можно использовать для интеграции различных хранилищ данных и выбора средств, приложений и служб платформы для создания правильных решений. Эти технологии предоставляют базовую среду для общего доступа к данным в операционных системах Microsoft Windows.

В MDAC существует три основных технологии. Объекты данных ActiveX (ADO) — это высокоуровневый, простой в использовании интерфейс для OLE DB. OLE DB — это высокоуровневая высокопроизводительный интерфейс для различных хранилищ данных. ADO и OLE DB могут работать с реляционными (табличными) и нереляционными (иерархическими и потоками) данными. Наконец, открытое подключение к базам данных (ODBC) — это другой высокопроизводительный интерфейс, предназначенный специально для хранилищ реляционных данных.

ADO предоставляет уровень абстракции между клиентским приложением или приложением среднего уровня и интерфейсами низкоуровневых баз данных OLE DB. ADO использует небольшой набор объектов автоматизации для создания простого и эффективного интерфейса OLE DB. Этот интерфейс делает ADO идеальным выбором для разработчиков на более высоких языках, таких как Visual Basic и даже VBScript, которые хотят получить доступ к данным без необходимости изучать структуры COM и OLE DB.
