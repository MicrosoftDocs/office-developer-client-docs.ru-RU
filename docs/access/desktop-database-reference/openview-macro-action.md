# <a name="openview-macro-action"></a>Действия OpenView макроса


**Применимо к**: Access 2013, Office 2013

В проекте Microsoft Access можно использовать **ОткрытьПредставление** , чтобы открыть представление в режиме таблицы, конструктора или просмотра. Это действие выполняется именованные представления при открытии в представлении таблицы данных. Можно выбрать ввода данных для представления и ограничить записи, которые отображаются в представлении.


> [!NOTE]
> <P>
		Эта действие не разрешено, если база данных не является доверенной. Дополнительные сведения о включении макросов см. по ссылкам в разделе See Also этой статьи.
</P>

## <a name="remarks"></a>Замечания

Это действие аналогично дважды щелкнув представления в области переходов или щелкнув правой кнопкой мыши представления в области переходов и затем выбрав команду.

**Советы **

  - Представления можно перетаскивать из области переходов в строку действие в макросе. Это автоматически создает **OpenView** действие, которое открывается представление в режиме таблицы.

  - Если не требуется отображать системные сообщения, которые появляются при запуске представления (это представление и показывают количество записей, которые будут затронуты), можно использовать действие **УстановитьСообщения** отображение этих сообщений.

Чтобы запустить **ОткрытьПредставление** в Visual Basic для приложений (VBA) модуль, используйте метод **OpenView** **объекта** .
