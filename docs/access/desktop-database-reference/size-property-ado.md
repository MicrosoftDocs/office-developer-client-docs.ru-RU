# <a name="size-property-ado"></a>Свойство Size (ADO)


**Область применения**: Access 2013, Office 2013

Указывает максимальный размер в bytes или characters объекта [Parameter.](parameter-object-ado.md)

## <a name="settings-and-return-values"></a>Параметры и значения возврата

Задает или возвращает значение **Long,** которое указывает максимальный размер в bytes или символах значения в **объекте Параметр.**

## <a name="remarks"></a>Примечания

Используйте свойство **Size** для определения максимального размера значений, написанных для чтения или чтения из свойства [Value](value-property-ado.md) объекта **Parameter.**

Если указать тип данных переменной длины для объекта **Parameter** (например, любого типа **String,** например **adVarChar),** необходимо задать свойство Size объекта, прежде чем примкните его к коллекции [Параметры;](parameters-collection-ado.md)  в противном случае возникает ошибка.

Если вы уже пристроили объект **Parameter** к коллекции **Параметры** объекта [Command](command-object-ado.md) и измените его тип на тип данных  переменной длины, перед выполнением объекта **Command** необходимо задать свойство **Size** объекта Параметр. в противном случае возникает ошибка.

Если метод [Обновления](refresh-method-ado.md) используется для получения сведений о параметрах от поставщика и  возвращает один или несколько объектов типа параметра переменной длины, ADO может выделять память для параметров в зависимости от их максимального потенциального размера, что может привести к ошибке во время выполнения. Чтобы предотвратить ошибку, перед выполнением команды необходимо явно задано свойство **Size** для этих параметров.

Свойство **Size** — это чтение и написание.
