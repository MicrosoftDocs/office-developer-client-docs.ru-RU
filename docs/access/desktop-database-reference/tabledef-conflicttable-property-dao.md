# <a name="tabledefconflicttable-property-dao"></a>TableDef.ConflictTable Property (DAO)


**Применимо к**: Access 2013, Office 2013

Возвращает имя таблицы конфликта, содержащий записи базы данных, которые конфликт во время синхронизации двух реплик (только для рабочих областей Microsoft Access). Только для чтения, **String**.

## <a name="syntax"></a>Синтаксис

*выражение* . ConflictTable

*выражение* Выражение, возвращающее объект **TableDef** .

## <a name="remarks"></a>Замечания

Возвращаемое значение имеет тип данных **String** , который является строкой нулевой длины ("») Если нет конфликтов таблицы или базы данных не является репликой.

Если двух пользователей в нескольких репликах каждого внести изменения в той же записи в базу данных, изменения, внесенные одним пользователем завершится с ошибкой для применения к другой реплики. Следовательно пользователю с неудачных изменений необходимо разрешить конфликты.

Конфликты возникают на уровне записей, не между полями. Например если один пользователь изменяет поле адреса и другую обновляет поле телефон в той же записи, затем одно изменение отклоняется. Так как конфликтов на уровне записей, отклонения возникает несмотря на то, что успешное изменение и отклоненные изменение скорее всего привести значение true, конфликта информации.

Механизм синхронизации обрабатывает записи конфликтов путем создания таблицы конфликта, содержащие сведения, которые определены в таблице, если изменения была успешно. Можно проверить эти таблицы конфликтов и работать с ними по строкам, исправление все соответствующие.

Все таблицы конфликтов называются таблице\_конфликта, где в таблице — исходное имя таблицы, усекается до максимального таблицы длина имени.
