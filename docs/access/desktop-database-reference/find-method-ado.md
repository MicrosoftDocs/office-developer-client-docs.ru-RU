# <a name="find-method-ado"></a>Метод Find (ADO)

**Область применения**: Access 2013, Office 2013

Выполняет поиск в [наборе записей](recordset-object-ado.md) для строки, удовлетворяющей заданным условиям. При необходимости можно указать направление поиска, начальную строку и смещение из начальной строки. При соблюдении условий для найденной записи задается текущая позиция строки; в противном случае в качестве положения задается конец (или начало) объекта **Recordset**.

## <a name="syntax"></a>Синтаксис

Find (*условия_отбора*, *скипровс*, *сеарчдиректион*, *Start*)

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|:--------|:----------|
|*Критерий* |**Строковое** значение, содержащее оператор, указывающий имя столбца, оператор сравнения и значение, используемое при поиске.|
|*Скипровс* |Необязательный атрибут. **Длинное** значение, значение по умолчанию которого равно нулю, которое указывает смещение строки от текущей строки или *открывающую* закладку, чтобы начать поиск. По умолчанию поиск начнется с текущей строки.|
|*SearchDirection* |Необязательный атрибут. Значение [сеарчдиректионенум](searchdirectionenum.md) , которое указывает, должен ли поиск начинаться с текущей строки или из следующей доступной строки в направлении поиска. Неудачный поиск заканчивается в конце **набора записей** , если значение равно **адсеарчфорвард**. Неудачный поиск заканчивается в начале **набора записей** , если значение равно **адсеарчбакквард**.|
|*Start* |Необязательный атрибут. Закладка **Variant** , которая выступает в качестве начальной позиции для поиска.|

## <a name="remarks"></a>Замечания

В *критерии*можно указать только одно имя столбца. Этот метод не поддерживает поиск по нескольким столбцам.

Оператор сравнения в *критерии* может иметь значение "**\>**" (больше, чем),**\<**"" (меньше), "=" (равно), "\>=" (больше или равно), "\<=" (меньше или равно), "\<\>" (не равно) или "Like" (сопоставление с шаблоном).

Значение в поле *условие* может быть строкой, числом с плавающей запятой или датой. Строковые значения разделяются символами одинарных\#кавычек или "" (знак номера) (например, "State = ' WA '" или "State \#=\#WA"). значения даты разделяются знаками\#"" (число знаков ") (например," дата\_ \> \#начала 7/22/97\#") и могут содержать часы, минуты и секунды для обозначения меток времени, которые не должны содержать миллисекунд .

Если оператор сравнения "Like", строковое значение может содержать звездочку (\*), чтобы найти один или несколько вхождений любого знака или подстроки. Например, "State Like m\*" соответствует Мэн и Массачусетс. Кроме того, можно использовать ведущие и замыкающие звездочки, чтобы найти подстроку, содержащуюся в значениях. Например, "состояние, например"\*как\*", соответствует" Аляска "," Арканзас "и Массачусетс.

Звездочки можно использовать только в конце строки условий, а также в начале и конце строки условий, как показано выше. Вы не можете использовать звездочку в качестве начального подстановочного\*знака ("str") или внедренного\*подстановочного знака ("r"). Это приведет к ошибке.

> [!NOTE]
> Если положение текущей строки не задано до вызова **Find**, произойдет ошибка. Перед вызовом **Find**должен быть вызван любой метод, который задает положение строки, например [MoveFirst](movefirst-movelast-movenext-and-moveprevious-methods-ado.md).

> [!NOTE]
> Если вызвать метод **Find** для объекта Recordset, а текущая позиция в наборе записей находится в последней записи или конце файла (EOF), ничего не будет найдено. Для установки текущей позиции или курсора в начало набора записей необходимо вызвать метод **MoveFirst** .
