# <a name="setproperty-macro-action"></a>Макрокоманда SetProperty

**Область применения**: Access 2013, Office 2013

Макрокоманду **SetProperty** можно использовать для задания свойства элемента управления в форме или отчете.

## <a name="setting"></a>Параметр

Макрокоманда **SetProperty** имеет следующие аргументы.

<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<thead>
<tr class="header">
<th><p>Аргумент макрокоманды</p></th>
<th><p>Описание</p></th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td><p>Контрольное имя</p></td>
<td><p>Введите имя поля или элемента управления, для которого требуется задать значение свойства. Используйте только имя элемента управления, а не полный синтаксис. Оставьте этот аргумент пустым, чтобы задать свойство для текущей формы или отчета.</p></td>
</tr>
<tr class="even">
<td><p>Свойство</p></td>
<td><p>Выберите свойство, которое требуется задать. Список свойств, которые можно задать с помощью этого действия, представлен в разделе <strong>"Примечания</strong> " этой статьи.</p></td>

## <a name="remarks"></a>Замечания

- Макрокоманду **SetProperty** можно использовать для задания следующих свойств элемента управления: **Enabled**, **Visible**, **locked**, **Left**, **Top**, **Width**, **Height**, **Color переднего плана**, **цвета фона**или **Подпись**.

- Если ввести недопустимое значение аргумента ***value*** , ошибка не возникнет, но Access может изменить значение свойства на другое, в зависимости от того, как он интерпретирует аргумент.

- Макрокоманду **SetProperty** можно использовать в изолированном макросе, только если перед ней выполняется действие, которое выбирает форму или отчет, содержащие элемент управления, для которого задается свойство. Если форма или отчет не открыты, можно открыть и выбрать его с помощью макрокоманды " **ОткрытьФорму** " или " **ОткрытьОтчет** ". Если форма или отчет уже открыты, можно использовать макрокоманду "ВыделитьОбъект" ( **ВыделитьОбъект** ), чтобы выбрать ее. Затем можно использовать действие **SetProperty** , чтобы задать свойство. Выбор объекта не требуется, если вы используете макрокоманду **SetProperty** в макросе, внедренном в элемент управления в той же форме или отчете, что и элемент управления, для которого задается свойство.

- Чтобы выполнить макрокоманду **SetProperty** в модуле VBA, используйте метод **SetProperty** объекта DoCmd. ****

## <a name="example"></a>Пример

В приведенном ниже примере показано, как использовать действие SetProperty для переключения видимости текстового поля **митекстбокс** .

**Пример кода из** [справочника программиста Microsoft Access 2010](https://www.amazon.com/Microsoft-Access-2010-Programmers-Reference/dp/8126528125).

```vb
    Submacro: TestVisible
        SetProperty
            Control Name Text40
            Property Visible
            Value =Not[Text40].[Visible]
    End Submacro
```
