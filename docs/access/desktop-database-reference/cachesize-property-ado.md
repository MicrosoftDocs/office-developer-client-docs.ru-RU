# <a name="cachesize-property-ado"></a>Свойство CacheSize (ADO)


**Область применения**: Access 2013, Office 2013

Указывает количество записей из объекта [Recordset,](recordset-object-ado.md) кэшироваться локально в памяти.

## <a name="settings-and-return-values"></a>Параметры и значения возврата

Задает или возвращает значение **Long,** которое должно быть больше 0. Значение по умолчанию  1.

## <a name="remarks"></a>Примечания

Используйте **свойство CacheSize** для управления количеством записей, которые можно получить одновременно в локальной памяти поставщика. Например, если **CacheSize** — 10, после первого открытия объекта **Recordset** поставщик извлекает первые 10 записей в локализованную память. При переходе через **объект Recordset** поставщик возвращает данные из локального буфера памяти. После перемещения последней записи в кэше поставщик извлекает следующие 10 записей из источника данных в кэш.

> [!NOTE]
> **CacheSize** основан на свойстве **поставщика Maximum Open Rows** (в коллекции **Свойств** объекта **Recordset).** Невозможно установить **значение CacheSize,** большее, чем **максимально открытые строки.** Чтобы изменить количество строк, которые может открыть поставщик, установите **максимальные открытые строки.**

Значение **CacheSize** может быть скорректировано в течение жизни объекта **Recordset,** но изменение этого значения влияет только на количество записей в кэше после последующих искавалов из источника данных. Только изменение значения свойства не изменит текущее содержимое кэша.

Если записей для получения меньше, чем указывает **CacheSize,** поставщик возвращает оставшиеся записи, и ошибка не возникает.

Параметр **CacheSize** с нолем не допускается и возвращает ошибку.

Записи, извлеченные из кэша, не отражают одновременно внесенные другими пользователями изменения в исходные данные. Для принудительного обновления всех кэшных данных используйте [метод Resync.](resync-method-ado.md)

Если **значение CacheSize** превышает одно, методы навигации [(Move,](move-method-ado.md) [MoveFirst, MoveLast, MoveNext и MovePrevious)](movefirst-movelast-movenext-and-moveprevious-methods-ado.md)могут привести к переходу на удаленную запись, если удаление происходит после получения записей. После первоначального получения последующие удаления не будут отражаться в кэше данных, пока вы не попытайтесь получить доступ к значению данных из удаленной строки. Однако настройка **CacheSize** для одного устраняет эту проблему, так как удаленные строки не могут быть извлечены.
