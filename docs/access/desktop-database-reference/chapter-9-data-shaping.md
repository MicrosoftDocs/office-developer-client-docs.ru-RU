# <a name="chapter-9-data-shaping"></a>Глава 9. Формирование данных

**Область применения**: Access 2013, Office 2013

*Формирование данных позволяет* запрашивать источник данных и возвращать [набор записей,](recordset-object-ado.md) представляюющий связь между двумя или более логическими сущностями (иерархией). 

Классическим примером иерархических отношений являются клиенты и заказы. Для каждого клиента в базе данных может быть ноль или больше заказов. Регулярные SQL предоставляют средства для сбора данных с помощью синтаксиса JOIN, но это может быть неэффективным и громоздким, так как избыточные родительские данные повторяются в каждой записи, возвращаемой для данного отношения между родителем и ребенком. Формирование данных может связать запись одного  родителя в родительском наборе записей с несколькими записями ребенка в детском наборе записей, избегая избыточности JOIN. Большинство людей считают модель программирования с несколькими **записями** родительского ребенка более естественной и простой в работе, чем с одной моделью **Join Recordset.**

Синтаксис формирования данных также предоставляет другие возможности. Разработчики могут создавать новые объекты **Recordset** без источника данных, используя ключевое слово NEW для описания полей родительских и детских **наборов записей.** Новый объект **Recordset** может быть заполнен данными и постоянно храниться. Разработчики также могут выполнять различные вычисления или агрегации (например, SUM, AVG и MAX) на детских полях. Формирование данных также может создать родительский набор записей из детского **recordset** путем группировки записей в ребенке и размещения одной строки в родительской группе для каждой группы в ребенке.
