# <a name="openvisualbasicmodule-macro-action"></a>Макрокоманда OpenVisualBasicModule

**Область применения**: Access 2013, Office 2013

Вы можете использовать действие **OpenVisualBasicModule,** чтобы открыть указанный модуль Visual Basic для приложений (VBA) в указанной процедуре. Это может быть процедура Sub, процедура function или процедура события.

> [!NOTE]
> Эта макрокоманда доступна только для доверенных баз данных. 

## <a name="setting"></a>Параметр

Действие **OpenVisualBasicModule** имеет следующие аргументы.

<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />

<tr class="header">

<th><p>Описание</p></th>

</thead>
<tbody>
<tr class="odd">
<td><p><strong>Имя модуля</strong></p></td>
<td><p>Имя модуля, который нужно открыть. Этот аргумент можно оставить пустым, если необходимо найти процедуру во всех стандартных модулях в базе данных и открыть соответствующий модуль. При запуске макроса, содержащего действие <strong>OpenVisualBasicModule</strong> в базе данных библиотеки, Microsoft Access сначала ищет модуль с этим именем в базе данных библиотеки, а затем в текущей базе данных.</p></td>

> [!NOTE]
> Необходимо ввести допустимые имена в **аргументе "Имя модуля"** или **"Имя** процедуры".


## <a name="remarks"></a>Заметки

Это действие можно использовать для открытия процедуры события, указав аргумент **"Имя** модуля" и **"Имя** процедуры". Например, чтобы открыть процедуру события **Click** кнопки PrintInvoice в форме "Заказы", установите для аргумента **"Имя** модуля" имя **Form.Orders,** а для аргумента **"Имя** процедуры" — **"PrintInvoice \_ Click".** Чтобы просмотреть процедуру события для формы или отчета, форма или отчет должны быть открыты.

Аналогично, чтобы открыть процедуру в модуле класса, необходимо указать имя модуля, хотя модуль класса не нужно открывать.

Чтобы открыть частную процедуру, необходимо открыть модуль, содержащий ее.

Это действие действует так же, как щелкнув правой кнопкой мыши модуль в области навигации и нажав кнопку **"Конструктор".** Это действие также позволяет указать имя процедуры и найти стандартные модули в базе данных для поиска процедур.

> [!TIP]
> Можно выбрать модуль в области навигации и перетащить его в строку макроса. При этом автоматически создается действие **OpenVisualBasicModule,** которое открывает модуль в разделе объявлений.

Чтобы запустить действие **OpenVisualBasicModule** в модуле VBA, используйте метод **OpenModule** объекта **DoCmd.**
