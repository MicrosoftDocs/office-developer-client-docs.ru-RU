# <a name="openvisualbasicmodule-macro-action"></a>Макрокоманда OpenVisualBasicModule

**Область применения**: Access 2013, Office 2013

Вы можете использовать действие **Макрокоманда openvisualbasicmodule** , чтобы открыть указанный модуль Visual Basic для приложений (VBA) в указанной процедуре. Это может быть процедура Sub, Function или процедура обработки события.

> [!NOTE]
> Эта макрокоманда доступна только для доверенных баз данных. 

## <a name="setting"></a>Параметры

Макрокоманда **Макрокоманда openvisualbasicmodule** имеет следующие аргументы.

<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<thead>
<tr class="header">
<th><p>Аргумент макрокоманды</p></th>
<th><p>Описание</p></th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td><p><strong>Имя модуля</strong></p></td>
<td><p>Имя модуля, который необходимо открыть. Если требуется выполнить поиск по всем стандартным модулям в базе данных для процедуры и открыть соответствующий модуль в этой процедуре, можно оставить этот аргумент пустым. При запуске макроса, содержащего действие <strong>Макрокоманда openvisualbasicmodule</strong> , в библиотечной базе данных Microsoft Access сначала ищет модуль с этим именем в библиотечной базе данных, а затем в текущей базе данных.</p></td>
</tr>
<tr class="even">
<td><p><strong>Имя процедуры</strong></p></td>
<td><p>Имя процедуры, в которую нужно открыть модуль. Если оставить этот аргумент пустым, модуль откроется в разделе "объявления".</p></td>
</tr>
</tbody>
</table>

> [!NOTE]
> Необходимо ввести допустимое имя в аргументе **имя модуля** или **имя процедуры** .


## <a name="remarks"></a>Замечания

С помощью этого действия можно открыть процедуру обработки события, указав аргумент **имени модуля** и аргумент **имени процедуры** . Например, чтобы открыть процедуру обработки события **Click** кнопки принтинвоице в формах Orders, задайте для аргумента **имя модуля** значение **Form. Orders** и присвойте аргументу **имя процедуры** значение **принтинвоице\_нажмите кнопку**. Чтобы просмотреть процедуру обработки события для формы или отчета, необходимо открыть форму или отчет.

Аналогично, чтобы открыть процедуру в модуле класса, необходимо указать имя модуля, несмотря на то что модуль класса не обязательно должен быть открыт.

Чтобы открыть частную процедуру, необходимо открыть модуль, содержащий его.

Это действие аналогично щелчку правой кнопкой мыши модуля в области навигации и нажатию кнопки **режим конструктора**. Это действие также позволяет указать имя процедуры и выполнить поиск процедур в стандартных модулях базы данных.

> [!TIP]
> Вы можете выбрать модуль в области навигации и перетащить его в строку макрокоманды. При этом автоматически создается действие **Макрокоманда openvisualbasicmodule** , которое открывает модуль в разделе "объявления".

Чтобы выполнить действие **Макрокоманда openvisualbasicmodule** в модуле VBA, используйте метод **опенмодуле** объекта **DoCmd** .
