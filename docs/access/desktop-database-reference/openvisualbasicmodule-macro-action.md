# <a name="openvisualbasicmodule-macro-action"></a>Макрокоманда OpenVisualBasicModule

**Область применения**: Access 2013, Office 2013

Вы можете использовать **действие OpenVisualBasicModule** для открытия определенного модуля Visual Basic для приложений (VBA) в указанной процедуре. Это может быть процедура Sub, процедура Function или процедура события.

> [!NOTE]
> Эта макрокоманда доступна только для доверенных баз данных. 

## <a name="setting"></a>Параметр

Действие **OpenVisualBasicModule** имеет следующие аргументы.

<col style="width: 50%" />
<col style="width: 50%" />

</thead>
<tbody>
<tr class="odd">
<td><p><strong>Имя модуля</strong></p></td>
<td><p>Имя модуля, который необходимо открыть. Этот аргумент можно оставить пустым, если вы хотите найти все стандартные модули в базе данных для процедуры и открыть соответствующий модуль в этой процедуре. Если вы запустите макрос, содержащий действие <strong>OpenVisualBasicModule</strong> в базе данных библиотеки, Microsoft Access сначала ищет модуль с этим именем в базе данных библиотеки, а затем в текущей базе данных.</p></td>

> [!NOTE]
> Необходимо ввести допустимые имена в **аргументе "Имя модуля"** или **"Имя** процедуры".


## <a name="remarks"></a>Примечания

Это действие можно использовать для открытия процедуры события, указав аргумент **"Имя** модуля" и аргумент **"Имя** процедуры". Например, чтобы открыть  процедуру события Нажмите кнопку PrintInvoice в  форме Заказы, установите аргумент Имя модуля **в Form.Orders** и установите аргумент **Имя** процедуры в **PrintInvoice \_ Click**. Чтобы просмотреть процедуру события для формы или отчета, форма или отчет должны быть открыты.

Аналогично, чтобы открыть процедуру в модуле класса, необходимо указать имя модуля, хотя модуль класса не должен открываться.

Чтобы открыть частную процедуру, модуль, содержащий ее, должен быть открыт.

Это действие имеет тот же эффект, что и щелкнув правой кнопкой мыши модуль в области навигации, а затем щелкнув **Представление дизайна.** Это действие также позволяет указать имя процедуры и искать стандартные модули в базе данных для процедур.

> [!TIP]
> Вы можете выбрать модуль в области навигации и перетаскивать его в строку действий макроса. Это автоматически создает действие **OpenVisualBasicModule,** которое открывает модуль в раздел Объявления.

Чтобы выполнить **действие OpenVisualBasicModule** в модуле VBA, используйте метод **OpenModule** объекта **DoCmd.**
