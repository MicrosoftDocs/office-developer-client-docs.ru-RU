# <a name="importexportdata-macro-action"></a>Макрокоманда ImportExportData

**Область применения**: Access 2013, Office 2013

С помощью действия **ImportExportData** можно импортировать или экспортировать данные между текущей базой данных Access (MDB или ACCDB) или проектом Access (ADP) и другой базой данных. Для баз данных Microsoft Access можно также связать таблицу с текущей базой данных Access из другой базы данных. Связь таблицы дает возможность получать доступ к данным в ней, хотя сама таблица остается в другой базе данных.

> [!NOTE]
> Эта макрокоманда доступна только для доверенных баз данных.

## <a name="remarks"></a>Заметки

Можно импортировать и экспортировать таблицы между Access и базами данных другого типа. Вы также можете экспортировать запросы на выборку Access к другим типам баз данных. Access экспортирует результирующий набор запроса в виде таблицы. Если обе базы данных являются базами данных Access, можно импортировать и экспортировать любые объекты базы данных Access.

При импорте таблицы из другой базы данных Access (MDB или ACCDB), которая является связанной таблицей в этой базе данных, она будет по-прежнему связана после импорта. То есть импортируется связь, а не сама таблица.

Если для базы данных, к которой вы хотите получить доступ, требуется пароль, при запуске макроса появляется диалоговое окно. Введите пароль в этом диалоговом окне.

Действие **ImportExportData** аналогично командам на вкладке **"Внешние** данные" в **области** "Импорт или **экспорт".** Эти команды можно использовать для выбора источника данных, например базы данных Access или другого типа базы данных, электронной таблицы или текстового файла. При выборе базы данных появляется одно или несколько диалогов, в которых вы выбираете тип объекта для импорта или экспорта (для баз данных Access), имя объекта и другие параметры в зависимости от базы данных, из которой импортируется, экспортируется или связывается. Аргументы для действия **ImportExportData** отражают параметры в этих диалоговом окнах.

Если вы хотите предоставить сведения об индексе для связанной таблицы dBASE, сначала привяжете таблицу:

1.  Щелкните **dBASE-файл.**

2.  В **диалоговом** окне "Получить внешние данные" введите путь к файлу dBASE в поле **"Имя файла".**

3.  Щелкните **ссылку на источник данных, создав связанную** таблицу, а затем нажмите кнопку **"ОК".**

4.  Укажите индексы в диалоговом окле для этой команды. Access сохраняет данные индекса в файле специальной информации (INF), расположенном в Microsoft Office папке.

5.  Затем можно удалить ссылку на связанную таблицу.

При следующем использовании действия **ImportExportData** для привязки таблицы dBASE Access использует указанные вами данные индекса.

> [!NOTE]
> При запросе или фильтрации связанной таблицы запрос или фильтр чувствительны к делу.

Чтобы запустить действие **ImportExportData** в модуле Visual Basic для приложений (VBA), используйте метод **TransferDatabase** объекта **DoCmd.**
