# <a name="importexportdata-macro-action"></a>Действия макроса ImportExportData

**Применимо к**: Access 2013 | Office 2013

Действие **ImportExportData** можно использовать для импорта или экспорта данных между текущей базы данных Access (MDB- или .accdb) или проект Microsoft Access (файлы с расширением ADP) и другой базы данных. Для баз данных Microsoft Access можно связать таблицы в текущей базе данных из другой базы данных. Связанные таблицы имеют доступ к данным таблицы оставив всю таблицу в другой базе данных.

> [!NOTE]
> 
		Эта действие не разрешено, если база данных не является доверенной. Дополнительные сведения о включении макросов см. по ссылкам в разделе See Also этой статьи.

## <a name="remarks"></a>Замечания

Можно импортировать и экспортировать таблицы между доступа и другие типы баз данных. Также можно экспортировать запросах доступа к другим типам баз данных. Microsoft Access экспортирует набор результатов запроса в виде таблицы. Можно импортировать и экспортировать все объекты базы данных Access, если обе базы данных, базы данных Access.

Если Импорт таблицы из другой базы данных Access (MDB- или .accdb), который является связанной таблицы в этой базе данных будет по-прежнему связан после его импорта. То есть, что импортируется связь, не всю таблицу.

Если базы данных, доступ к которым осуществляется требуется пароль, диалоговое окно отображается, когда запустить макрос. В этом диалоговом окне введите пароль.

Действие **ImportExportData** аналогична команды на вкладке " **Внешние данные** " в разделе **импорта** или **экспорта**. Эти команды можно использовать для выбора источника данных, например базы данных Access или другой тип базы данных, электронную таблицу или текстовый файл. Если выбрать базу данных, один или несколько диалоговых окон отображаются выберите тип объекта для импорта или экспорта (для баз данных Access), имя объекта и другие варианты, в зависимости от базы данных при импорте из или экспорте или создание ссылок на. Аргументы для действия **ImportExportData** соответствуют параметрам этих диалоговых окон.

Если вы хотите предоставить данные индекса для связанного dBASE таблицы, сначала связи с таблицей:

1.  Щелкните **файл dBASE**.

2.  В диалоговом окне **Внешние данные** введите путь к файлу dBASE в поле **имя файла** .

3.  Щелкните **ссылку на источник данных путем создания связанной таблицы**, а затем нажмите **кнопку ОК**.

4.  Укажите индексы в диалоговых окнах для этой команды. Для сохранения индекса данных в специальных INF-файл, находящийся в папке Microsoft Office.

5.  Вы можете удалить ссылки на связанные таблицы.

Следующий раз использовать действие **ImportExportData** для связи между этой таблицей dBASE используются сведения о индекса, заданного.

> [!NOTE]
> Если запроса или фильтра в связанной таблице запроса или фильтра с учетом регистра.

Чтобы выполнить действие **ImportExportData** в Visual Basic для приложений (VBA) модуль, используйте метод **ПреобразоватьБазуДанных** **объекта** .
