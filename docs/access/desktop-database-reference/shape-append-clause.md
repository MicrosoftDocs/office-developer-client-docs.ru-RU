# <a name="shape-append-clause"></a>Предложение Append команды Shape


**Область применения**: Access 2013, Office 2013

Положение appEND команды фигуры примыкает столбец или столбцы к **набору записей.** Часто эти столбцы являются столбцами глав, которые относятся к детскому **набору записей.**

## <a name="syntax"></a>Синтаксис

```vb 
 
SHAPE [parent-command [[AS] parent-alias]] APPEND column-list
```

## <a name="description"></a>Описание

Части этого положения:

- *родительская команда*

- Ноль или один из следующих (можно полностью отопустить *родительную* команду):
    
  - Команда поставщика в фигурных скобки ("), которая {} возвращает **объект Recordset.** Команда выдана основному поставщику данных, и ее синтаксис зависит от требований этого поставщика. Обычно это будет язык SQL, хотя ADO не требует какого-либо конкретного языка запросов.
    
  - Еще одна команда фигуры, встроенная в скобки.
    
  - Ключевое слово TABLE с именем таблицы в поставщике данных.

- *родительский псевдоним*

  - Необязательный псевдоним, который относится к родительскому **набору записей.**

- *столбец-список*

  - Один или несколько из следующих ниже.
    
    - Сводный столбец.
    
    - Вычислимый столбец.
    
    - Новый столбец, созданный с новым предложением.
    
    - Столбец главы. Определение столбца главы заключено в скобки ("()"). См. синтаксис ниже:


        ```vb 
        
        SHAPE [parent-command [[AS] parent-alias]] 
        APPEND (child-recordset [ [[AS] child-alias] 
        RELATE parent-column TO child-column | PARAMETER param-number, ... ]) 
        [[AS] chapter-alias] 
        [, ... ] 
        ```

- *набор записей для детей*

  - Команда поставщика в фигурных скобки ("), которая {} возвращает **объект Recordset.** Команда выдана основному поставщику данных, и ее синтаксис зависит от требований этого поставщика. Обычно это будет язык SQL, хотя ADO не требует какого-либо конкретного языка запросов.
    
  - Еще одна команда фигуры, встроенная в скобки.
    
  - Имя существующего формы **Recordset**.
    
  - Ключевое слово TABLE с именем таблицы в поставщике данных.

- *псевдоним ребенка*

  - Псевдоним, который относится к детскому **набору записей.**

- *родитель-столбец*

  - Столбец в **наборе записей,** возвращаемом *родительской командой.*

- *столбец child-column*

  - Столбец в **Наборе записей,** возвращенный *детской командой.*

- *param-number*

  - См. [операцию параметризированных команд.](operation-of-parameterized-commands.md)

- *псевдоним главы*

  - Псевдоним, который ссылается на столбец главы, приданный родительскому.


> [!NOTE]
> - Положение _"родительский столбец для ребенка-столбца"_ на самом деле является списком, где каждое определенное отношение разделено запятой.
> - Пункт после ключевого слова APPEND фактически является списком, где каждый пункт разделяется запятой и определяет другой столбец, который должен быть придан родительскому.



## <a name="remarks"></a>Примечания

При построении команд поставщика из входных данных пользователя в составе команды SHAPE SHAPE будет рассматривать команду поставщика, предоставленную пользователем, как непрозрачной строки, и передавать их поставщику. Например, в следующей команде SHAPE

```vb 
 
SHAPE {select * from t1} APPEND ({select * from t2} RELATE k1 TO k2) 
```

SHAPE выполнит две команды: выберите из t1 и (выберите из \* \* t2 RELATE k1 TO k2). Если пользователь поставляет комплексную команду, состоящую из нескольких команд поставщика, разделенных зайколонами, SHAPE не сможет различить разницу. Итак, в следующей команде SHAPE

```vb 
 
SHAPE {select * from t1; drop table t1} APPEND ({select * from t2} RELATE k1 TO k2) 
```

SHAPE выполняет выбор из t1; drop table t1 и (выберите из \* t2 RELATE k1 to k2), не понимая, что таблица drop table t1 является отдельной и в этом случае опасной \* командой поставщика. Приложения всегда должны проверять вход пользователя, чтобы предотвратить такие потенциальные хакерские атаки.
