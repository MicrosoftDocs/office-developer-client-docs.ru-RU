# <a name="shape-append-clause"></a>Предложение Append команды Shape


**Область применения**: Access 2013, Office 2013

Предложение APPEND команды Shape добавляет столбец или столбцы в **набор записей**. Часто эти столбцы являются столбцами главы, которые ссылаются на дочерний **набор записей**.

## <a name="syntax"></a>Синтаксис

```vb 
 
SHAPE [parent-command [[AS] parent-alias]] APPEND column-list
```

## <a name="description"></a>Описание

Ниже приведен список частей этого предложения.

- *родительская — команда*

- Ноль или один из следующих элементов ( *команда* может полностью пропускаться):
    
  - Команда поставщика в фигурных скобках ("{}"), которая возвращает объект **Recordset** . Команда выдается базовому поставщику данных, а его синтаксис зависит от требований этого поставщика. Как правило, это язык SQL, хотя для ADO не требуется определенный язык запросов.
    
  - Еще одна команда "Фигура", внедренная в круглые скобки.
    
  - Ключевое слово TABLE, за которым следует имя таблицы в поставщике данных.

- *псевдоним родителя*

  - Необязательный псевдоним, ссылающийся на родительский **набор записей**.

- *Column (список)*

  - Один или несколько из следующих вариантов:
    
    - Сводный столбец.
    
    - Вычисляемый столбец.
    
    - Новый столбец, созданный с помощью оператора NEW.
    
    - Столбец главы. Определение столбца главы заключено в круглые скобки ("()"). Ниже приведен синтаксис:


        ```vb 
        
        SHAPE [parent-command [[AS] parent-alias]] 
        APPEND (child-recordset [ [[AS] child-alias] 
        RELATE parent-column TO child-column | PARAMETER param-number, ... ]) 
        [[AS] chapter-alias] 
        [, ... ] 
        ```

- *дочерний набор записей*

  - Команда поставщика в фигурных скобках ("{}"), которая возвращает объект **Recordset** . Команда выдается базовому поставщику данных, а его синтаксис зависит от требований этого поставщика. Как правило, это язык SQL, хотя для ADO не требуется определенный язык запросов.
    
  - Еще одна команда "Фигура", внедренная в круглые скобки.
    
  - Имя существующего **набора записей**в форме.
    
  - Ключевое слово TABLE, за которым следует имя таблицы в поставщике данных.

- *дочерний псевдоним*

  - Псевдоним, ссылающийся на дочерний **набор записей**.

- *родительский столбец*

  - Столбец в **наборе записей** , возвращенный *родительской командой.*

- *дочерний столбец*

  - Столбец в **наборе записей** , возвращенный *дочерней командой*.

- *Param — Number*

  - Просмотр [действий с параметризованными командами](operation-of-parameterized-commands.md).

- *Глава — псевдоним*

  - Псевдоним, ссылающийся на столбец Chapter, добавляемый к родительскому элементу.


> [!NOTE]
> - Предложение _"родительский столбец для дочернего столбца"_ фактически является списком, где каждое определенное отношение отделяется запятой.
> - В действительности предложение после ключевого слова APPEND представляет собой список, в котором каждое предложение отделяется запятой и определяет другой столбец, добавляемый к родительскому элементу.



## <a name="remarks"></a>Примечания

Когда вы создаете команды поставщика из входных данных пользователя в составе команды SHAPE, SHAPE будет рассматривать указанную пользователем команду поставщика как непрозрачную строку и передавать их в поставщике. Например, в следующей команде SHAPE

```vb 
 
SHAPE {select * from t1} APPEND ({select * from t2} RELATE k1 TO k2) 
```

Фигура будет выполнять две команды: выбрать \* из T1 и (выбрать \* из T2 связать K1 с K2). Если пользователь предоставляет составную команду, состоящую из нескольких команд поставщика, разделенных точкой с запятой, фигура не может различать различия. Поэтому в следующей команде SHAPE

```vb 
 
SHAPE {select * from t1; drop table t1} APPEND ({select * from t2} RELATE k1 TO k2) 
```

ФИГУРА выполняет SELECT \* из T1; Drop Table T1 и (Select \* from T2 K1 to K2), не зная, что инструкция DROP TABLE T1 является отдельным и в данном случае опасной команды поставщика. Приложения всегда должны проверять входные данные пользователя, чтобы предотвратить возникновение подобных потенциальных атак хакеров.
