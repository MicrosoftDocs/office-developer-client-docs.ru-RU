# <a name="workflowtasks-macro-action"></a>Макрокоманда WorkflowTasks


**Область применения**: Access 2013, Office 2013

Вы можете использовать **действие WorkflowTasks** для отображения **диалогового** окна задачи рабочего процесса.

## <a name="setting"></a>Параметр

Действие **WorkflowTasks имеет** следующий аргумент.

<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<thead>
<tr class="header">
<th><p>Аргумент макрокоманды</p></th>
<th><p>Описание</p></th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td><p><strong>Номер записи</strong></p></td>
<td><p>Положение элемента в списке Microsoft SharePoint, начиная с <strong>1</strong> для первого элемента в списке, <strong>2</strong> для второго элемента и так далее. Вы также можете ввести выражение для этого аргумента.</p></td>
</tr>
</tbody>
</table>


## <a name="remarks"></a>Примечания

  - Действие **WorkflowTasks** открывает диалоговое окно **Workflow Tasks.** В этом диалоговом окне отображаются все задачи, доступные для указанного элемента. Рабочий процесс должен быть определен для списка в SharePoint Foundation.

  - Действие **WorkflowTasks** можно использовать только после открытия и выбора связанного списка SharePoint Foundation. Чтобы открыть и выбрать связанный список, используйте **действие OpenTable.** Если список уже открыт, выберите действие **SelectObject.**

  - Действие **WorkflowTasks** имеет тот же эффект, что и щелкнуть правой кнопкой мыши любую ячейку в связанном списке SharePoint Foundation, пока она открыта в представлении таблицы данных, указав на рабочий **процесс,** а затем щелкнув задачи рабочего процесса **.**

  - Чтобы выполнить **действие WorkflowTasks** в модуле VBA, используйте метод **WorkflowTasks** объекта **DoCmd.**
