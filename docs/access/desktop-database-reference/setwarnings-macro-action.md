# <a name="setwarnings-macro-action"></a>SetWarnings Macro Action


**Применимо к**: Access 2013 | Office 2013

Действие **УстановитьСообщения** можно использовать для включения системные сообщения включено или отключено.


> [!NOTE]
> <P>
		Эта действие не разрешено, если база данных не является доверенной. Дополнительные сведения о включении макросов см. по ссылкам в разделе See Also этой статьи.
</P>



## <a name="setting"></a>Параметр

Действие **УстановитьСообщения** использует следующий аргумент.

## <a name="remarks"></a>Замечания

Это действие можно использовать для предотвращения модальные окна предупреждения и окнах сообщений в каждом окне. Тем не менее всегда отображаются сообщения об ошибках. Кроме того, Microsoft Access отображает все диалоговые окна, требующие ввода данных, отличных от простого нажатия кнопок (например, **кнопку ОК**, **Отменить**, **Да**или **Нет**), к примеру, любое диалоговое окно, которое требуется для ввода текста или выберите один из несколько вариантов .

Выполнение этого действия с **В** аргументе значение **Нет** имеет тот же эффект, как при нажатии клавиши ВВОД каждый раз, когда отображается предупреждение или окно сообщения. Как правило **ОК** или **Да** кнопка выбирается в ответ на предупреждения или сообщения.

После завершения макроса Microsoft Access автоматически включает вывод системных сообщений обратно на.

Часто используется это действие с действием **эхо** , который позволяет скрыть результаты макроса до его завершения. Чтобы скрыть предупреждения и окнах сообщений также можно использовать действие **УстановитьСообщения** .


> [!WARNING]
> <P>Несмотря на то, что действие <STRONG>УстановитьСообщения</STRONG> упрощает взаимодействие с макросами, необходимо соблюдать осторожность отключение системные сообщения. В некоторых случаях не требуется для продолжения макроса при определенных предупреждение отображается. Если вы не уверены результата все действия макроса, следует использовать это действие.</P>



Чтобы выполнить действие **УстановитьСообщения** в Visual Basic для приложений (VBA) модуль, используйте метод **УстановитьСообщения** **объекта** .
