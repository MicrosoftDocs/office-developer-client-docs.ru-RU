# <a name="setwarnings-macro-action"></a>Макрокоманда SetWarnings

**Область применения**: Access 2013, Office 2013

Вы можете использовать действие **SetWarnings,** чтобы включить или отключить системные сообщения.

> [!NOTE]
> Эта макрокоманда доступна только для доверенных баз данных. 

## <a name="setting"></a>Параметр

## <a name="remarks"></a>Заметки

Это действие можно использовать для предотвращения остановки макроса модальными предупреждениями и окнами сообщений. Однако сообщения об ошибках всегда отображаются. Кроме того, Microsoft Access отображает все диалоговые окна, которые требуют ввода, кроме простого нажатия кнопки (например, "ОК", "Отмена", **"Да"** или "Нет"), например любое диалоговое окно, в которое требуется ввести текст или выбрать один из нескольких параметров. 

Это действие с аргументом **Warnings On,** для которого установлено **"Нет",** имеет тот же эффект, что и нажатие на ввод при отображке предупреждения или окна сообщения. Обычно в **ответ**  на предупреждение или сообщение выбирается кнопка "ОК" или "Да".

После завершения макроса Access автоматически включает отображение системных сообщений.

Часто это действие используется с действием **Echo,** которое скрывает результаты макроса, пока оно не будет завершено. Действие **SetWarnings** также можно использовать для скрытие предупреждений и полей сообщений.

> [!WARNING]
> Хотя действие **SetWarnings может** упростить взаимодействие с макросами, необходимо быть осторожным при отключлении системных сообщений. В некоторых случаях не нужно продолжать макрос, если отображается определенное предупреждение. Если вы не уверены в результате всех макро действий, не следует использовать это действие.

Чтобы запустить действие **SetWarnings** в модуле Visual Basic для приложений (VBA), используйте метод **SetWarnings** объекта **DoCmd.**
