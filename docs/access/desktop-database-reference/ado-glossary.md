# <a name="ado-glossary"></a>Глоссарий ADO

**Область применения**: Access 2013, Office 2013

## <a name="a"></a>A

**абсолютный URL-адрес**

Полностью квалифицированный URL-адрес, который указывает расположение ресурса, расположенного в Интернете или интрасети. См. **также URL-адрес** **и относительный URL-адрес.**

**Элемент ActiveX**

Самозарегистравающийся компонент COM, который часто имеет визуальный элемент во время разработки или во время запуска. ActiveX элементы управления также могут взаимодействовать с контейнером Active Document, например Microsoft Internet Explorer.

**ADISAPI (интерфейс программирования приложений для приложений с расширенными данными в Интернете)**

DLL ISAPI, которая обеспечивает разметку, управление автоматизацией, маршалинг **Recordset** и упаковку MIME. Компонент ADISAPI работает через API, предоставляемый службы IIS (IIS). См. **также ISAPI**.

**агрегатная функция**

В запросе используется функция COUNT, AVG или STDEV, которая вычисляет значение, используя все строки в столбце таблицы. При написании выражений и программировании можно использовать SQL (в том числе три перечисленных выше) и агрегированные функции домена для определения различных статистических данных.

**alias**

Альтернативное имя, которое вы даете столбце или выражению в заявлении SELECT SQL, часто более короткое или более значимое. Например, BobSales — это псевдоним в следующем заявлении SELECT: "Выберите wr-Sales как BobSales из SalesDB". Псевдоним можно использовать для динамического назначения столбцов для управления привязками объекта **DataControl.**

**резьба по квартире**

Модель потоков com, в которой все вызовы объекта происходят в одном потоке. В потоке квартир com синхронизирует и вызывает маршалов. См. также **com**.

**асинхронная операция**

Операция, возвращаемая управление в вызываемую программу, не дожидаясь завершения операции. До завершения операции выполнение кода продолжается. См. также **синхронную операцию.**

## <a name="b"></a>B

**запись привязки**

Сопоставление между полем в таблице и переменной. В расширениях ADO Visual C++ поля **Recordset** соеди-ются с переменными C/C++.

**bitmask**

Числовое значение, предназначенное для сопоставления по-битным значениям с другими числовые значения, как правило, для флага параметров или значений возврата. Обычно это сравнение делается с логическими операторами, такими как **And** and and **Or** в Visual Basic, а также **&** в **|** C++.

Например, значения ADO **FieldAttributeEnum** можно использовать как битмаски для определения атрибутов поля. Предположим, необходимо определить, обновлялось ли поле. Вы можете проверить это с помощью следующего выражения в Visual Basic:  
  

Если результат true, то поле обновляется.

**bookmark**

Маркер, который однозначно идентифицирует строку в наборе строк, чтобы пользователь быстро переходить к ней.

**бизнес-объект**

Объект, который выполняет определенный набор операций, например проверку данных или логику бизнес-правил. Бизнес-объекты обычно находятся на среднем уровне.

**бизнес-правило**

Сочетание изменений проверки, проверки логотипов, поиска баз данных, политик и алгоритмических преобразований, составляющих способ ведения бизнеса предприятия. Также известная как *бизнес-логика.*

## <a name="c"></a>C

**вычисляемая экспрессия**

Выражение, которое не является постоянным, но значение которого зависит от других значений. Для оценки вычисляемого выражения необходимо получать и вычислять значения из других источников, как правило, в других полях или строках.

**глава**

Ссылка на ряд строк из источника данных. В ADO глава обычно является ссылкой на другой **набор записей.**

Столбцы глав делают возможным  определение отношения между  родителем и ребенком, в  котором родитель — набор записей, содержащий столбец главы, а ребенок — набор **записей,** представленный главой. 

**псевдоним главы**

Псевдоним, который ссылается на столбец, приданный родительскому.

**кодировка**

Сопоставление набора символов с их числными значениями. Например, Unicode — это 16-битный набор символов, способный кодировать все известные символы и использоваться в качестве стандарта кодирования символов во всем мире.

**ребенок**

Зависимая сторона иерархического отношения. Ребенок — это узел в иерархической структуре, который имеет другой узел над ней (ближе к корневому). См. **также псевдоним child-alias,** **родительские и родительские** **отношения, родительские**.

**псевдоним ребенка**

Псевдоним, который относится к ребенку. См. **также псевдоним**, **ребенок**.

**CLSID (идентификатор класса)**

Универсальный уникальный идентификатор (UUID), который идентифицирует компонент COM. Каждый компонент COM имеет свой CLSID в Windows реестре, чтобы он был загружен другими приложениями. См. также **ProgID**, **COM**.

**уровень клиента**

Логический слой распределенной системы, которая обычно представляет данные пользователю и обрабатывает входные данные пользователя, иногда именуемых *передней частью.* Обычно клиентский уровень запрашивает данные с сервера на основе ввода, а затем форматы и отображает результат. См. **также средний уровень**, уровень источника **данных**, **распределенного приложения**.

**COM (Компонентная объектная модель)**

Двоичный стандарт, позволяющий объектам скооперироваться в сетевой среде независимо от языка, на котором они были разработаны или на каких компьютерах они находятся. Технологии, основанные на com, включают ActiveX, автоматизацию и привязку объектов и их встраивку (OLE). COM позволяет объекту подвергать свои функции другим компонентам и принимать приложения. Он определяет как то, как объект предоставляет себя, так и то, как эта экспозиция работает в процессах и в сетях. Com также определяет жизненный цикл объекта.

**Компонент COM**

Двоичный файл , например .dll, .ocx и некоторые .exe, который поддерживает стандарт COM для предоставления объектов. Такой файл содержит код для одного или нескольких заводов класса, классов COM, механизмов записи реестра, кода загрузки и так далее.

**оператор сравнения**

Оператор, который сравнивает два выражения и возвращает значение Boolean.

Параметр критериев, который может быть выражен как "(больше), \> \< "(меньше), "=" (равно), \> "=" (больше или равно), \< "=" (меньше или равно), \< \> "(не равно) или "like" (соответствие шаблонам).

**component**

Объект, инкапсулирует данные и код и предоставляет определенный набор общедоступных служб.

**сложный файл**

Реализация структурированного хранилища COM для файлов. Сложный файл хранит отдельные объекты в одном структурированных файлах, состоящих из двух основных элементов: объектов хранения и объектов потока. Вместе они функционируют как файловая система в файле. Дополнительные сведения см. в материалах Compound Files в SDK Microsoft Platform.

Несколько отдельных файлов, связанных между собой в одном физическом файле. К каждому отдельному файлу в сложном файле можно получить доступ как к одному физическому файлу.

**constant**

Числовая или строковая величина, которая не меняется. В коде вместо фактических значений можно использовать именуемые переумывления ADO (перезамещенные константы), например **adUseClient** — это константа с значением 3. (Const adUseClient = 3). См. **также переумежение**.

**курсор**

Элемент базы данных, который управляет навигацией записи, возможностью обновления данных и наглядностью изменений, внесенных в базу данных другими пользователями.

В начало

## <a name="d"></a>D

**привязка данных**

Процесс связывать объекты или элементы управления приложения с источником данных. Управление, связанное с источником данных, называется *управлением, связанным с данными.*

Содержимое управления, связанного с данными, связано со значениями из базы данных. Например, управление сеткой, привязанное к объекту **Recordset,** может быть обновлено при обновлении строк **в Наборе** записей. При извлечении новых значений **в Наборе записей** в сетке отображаются новые значения.

**поставщик данных**

Программное обеспечение, которое предоставляет данные приложению ADO напрямую или через поставщика услуг. См. **также поставщик услуг.**

**формирование данных**

Метод, который использует формализованный синтаксис (называемый языком **Shape)** для определения специализированного объекта **Recordset** (называемого фигурным набором записей), который содержит не только данные, но и ссылки на другие объекты **Recordset** и/или вычисляемые значения на основе этих других объектов **Recordset.**

**Уровень источника данных**

Логический уровень распределенной системы, представляю которой представляет компьютер с DBMS, например SQL Server базы данных. См. также **уровень клиента,** **средний уровень,** **распределенные приложения**.

**DCOM**

Протокол провода, который позволяет компонентам COM напрямую взаимодействовать друг с другом по сети. См. также **com**, **компонент**.

**DDL (язык определения данных)**

Эти утверждения в SQL, которые определяют, а не манипулируют данными. Схема базы данных создается или изменена с помощью DDL. Например, **CREATE TABLE,** **CREATE INDEX,** **GRANT** и **REVOKE** SQL DDL.

**поток по умолчанию**

Текстовый или двоичный поток (представленный объектом **Stream),** связанный с объектами **Record** или **Recordset** при использовании определенных поставщиков OLE DB, таких как поставщик DB Microsoft OLE для интернет-публикации. Поток по умолчанию обычно содержит содержимое файла, например HTML-код для корневого сайта.

**распределенные приложения**

Программа, написанная таким образом, чтобы обработка была разделена на несколько компьютеров по сети. Обычно распределенное приложение делится на уровни презентаций, бизнес-логики и хранения *данных.* См. также **уровень клиента,** **средний уровень,** **уровень источника данных.**

**отключенный Recordset**

Объект **Recordset** в клиентском кэше, который больше не имеет живого подключения к серверу. Если исходный источник данных по какой-либо причине необходимо получить доступ к нему, например к обновлению данных, подключение необходимо заново установить. Однако к коллекциям, свойствам и методам отключенного набору **записей** по-прежнему можно получить доступ.

**DLL (библиотека динамических ссылок)**

Файл, содержащий одну или несколько функций, которые компилировать, связывать и хранить отдельно от процессов, которые их используют. Операционная система сопопосает DLLs в адресное пространство процесса вызова при запуске или во время его запуска.

**DML (язык обработки данных)**

Эти утверждения в SQL, которые манипулируют, а не определяют данные. Значения в базе данных выбираются и модифицироваться с помощью DML. Например, **INSERT,** **UPDATE,** **DELETE** и **SELECT** являются SQL DML.

**поставщик исходных документов**

Особый класс поставщиков, которые управляют папками и документами. Если документ представлен объектом **Record** или папка документов представлена объектом **Recordset,** поставщик источников документов заполняет эти объекты уникальным набором полей, описывая характеристики документа, а не самим документом. См. также **запись ресурса.**

**DSN (имя источника данных)**

Коллекция сведений, используемых для подключения приложения к определенной базе данных ODBC. Диспетчер драйверов ODBC использует эти сведения для создания подключения к базе данных. DSN можно хранить в файле (файл DSN) или в Windows реестре (машина DSN).

**динамическое свойство**

Свойство, определенное поставщику данных или службе курсоров. Коллекция **свойств** объекта заполняется этими автоматически ("динамически"). Объект не имеет динамических свойств, пока не подключен к источнику данных через конкретного поставщика данных. См. **также поставщик данных,** **курсор**.

Также именуется *как константа с переумехой.* См. также **постоянную**.

**event**

Действие, распознается объектом, для которого можно написать код для ответа. События могут быть созданы выполнением команд, завершением транзакций, навигацией по набору записей и обновлениями данных, а также другими действиями. См. также **обработник событий**.

**обработчик событий**

Обработник событий — это код, который выполняется при событии. См. **также событие**.

**handler**

Рутина, которая управляет общим и относительно простым состоянием или операцией, например восстановлением ошибок или управлением данными.

**иерархический Набор записей**

Набор **записей,** содержащий другой **набор записей.** См. **также формирование данных**, **глава**.

Дополнительные сведения см. в [рублях Accessing Rows in a Hierarchical Recordset.](accessing-rows-in-a-hierarchical-recordset.md)

**иерархия**

В ADO иерархические **записи** используются для представления отношения между записью и главой между родителем и ребенком. Кроме того, в объектах ADO, **Record** и **Stream** можно использовать для доступа к иерархическим структурам дерева, таким как папка и документы. ADO MD также включает **объекты Иерархия,** которые представляют связь между уровнями измерения в кубе OLAP. **ISAPI (интерфейс программирования приложений на интернет-сервере)**

Набор функций для интернет-серверов, таких как Windows NT Server/Windows 2000 Server Microsoft IIS (IIS).

## <a name="k-m"></a>K-M

**key**

Столбец или столбцы в таблице, которые однозначно идентифицируют строку; часто используется для индексации таблицы.

**маршалинг**

Процесс упаковки, отправки и распаковки параметров метода интерфейса через границы потоков или процессов.

**средний уровень**

Логический уровень в распределенной системе между пользовательским интерфейсом или веб-клиентом и базой данных. Это обычно, когда бизнес-объекты мгновенно. Средний уровень — это набор бизнес-правил и функций, которые создают и работают при получении информации. Они выполняют это с помощью бизнес-правил, которые могут часто меняться и таким образом инкапсулироваться в компоненты, физически отделяются от самой логики приложения. Также известный как *уровень сервера приложений*. См. **также распределенные приложения,** **клиентский уровень,** **уровень источника данных.**

**MIME (многоцелевая расширение интернет-почты)**

Изначально разработан протокол Интернета, позволяющий обмениваться сообщениями электронной почты с богатым контентом в разнородной среде сети, машины и электронной почты. На практике MIME также был принят и расширен приложениями, не относяными к почте.

MIME — это стандарт, позволяющий публиковать и читать двоичные данные в Интернете. В загонах файла с двоичными данными содержится тип данных MIME; это информирует клиентские программы (например, веб-браузеры и почтовые пакеты), что они должны обрабатывать данные по-другому, чем с прямым текстом. Например, заглушек веб-документа, содержащего графику JPEG, содержит тип MIME, определенный формату файла JPEG. Это позволяет браузеру отображать файл с его просмотра JPEG, если он присутствует.

**node**

Переменная, которая содержит ссылку на объект. Например, objCustomObject — это переменная, которая указывает на объект типа CustomObject:

**ODBC (Open Database Connectivity)**

Стандартный языковой интерфейс программирования, используемый для подключения к различным источникам данных. Обычно к этому можно получить доступ через панель управления, где для использования определенных драйверов ODBC можно на назначены имена источников данных (DSNs).

**OLE DB**.

Набор интерфейсов, которые раскрывают данные из различных источников с помощью COM. Интерфейсы OLE DB предоставляют приложениям единый доступ к данным, хранимым в различных источниках информации. Эти интерфейсы поддерживают количество функций DBMS, соответствующих источнику данных, что позволяет ему обмениваться данными. См. также **com**.

**оптимистичная блокировка**

Тип блокировки, в котором страница данных, содержащая одну или несколько записей, включая редактируемую запись, недоступна для других пользователей только во время обновления записи методом **Update,** но доступна до и после вызова обновления **.**

Оптимистичная блокировка используется, когда объект **Recordset** открывается с параметром **LockType** или свойством **adLockOptimistic** или **adLockBatchOptimistic.** См. **также пессимистичные блокировки**.

**ordinal value**

Числовая расположение элемента в порядке. В коллекции ADO ordinal value of the first item is zero (0). Следующий элемент — один (1) и так далее.

## <a name="p"></a>P

**параметризованная команда**

Запрос или команда, которая позволяет заданы значения параметров до выполнения команды. Например, строка SQL может быть параметризирована путем встраив маркеры параметров в строку SQL (обозначенную символом '?'). Затем приложение указывает значения для каждого параметра и выполняет команду.

**родительский**

Контролирующая сторона иерархических отношений. В иерархической структуре родитель имеет один или несколько детских узлов непосредственно под ним в иерархии. См. **также псевдоним parent-alias**, **родительский и детский отношения**, **ребенок**.

**родительский псевдоним**

Псевдоним, который относится к родительскому. См. **также псевдоним**, **родитель**.

**отношения между родителем и ребенком**

Отношение в иерархической структуре, в которой родитель на один уровень выше и напрямую связан с одним или более детьми. Ребенок на один уровень ниже и должен иметь одного родителя. См. также **родительский** **, ребенок**.

**упорствовать**

Сохранение данных в постоянном состоянии, например сохранение **наборов записей** в файл.

**пессимистическая блокировка**

Тип блокировки, в котором страница, содержащая одну или несколько записей, включая редактируемую, недоступна для других пользователей, чтобы убедиться в том, что обновление будет сделано. Пессимистическое поведение блокировки определяется поставщиком OLE DB. Как правило, записи блокированы при редактировании и остаются недоступными до завершения метода **Обновления.**

Пессимистическая блокировка включена, когда объект **Recordset** открывается с параметром **LockType** или свойством **adLockPessimistic.** См. также **оптимистичную блокировку**.

**объединение**

Оптимизация производительности на основе использования коллекций предварительно выделенных ресурсов, таких как объекты или подключения к базе данных. Более эффективно использовать существующий ресурс из пула, чем создавать новый ресурс.

**ProgID (программный идентификатор)**

Уникальное имя, которое Windows реестру с помощью приложения COM. ProgID для ADO Connection — это "ADODB. Подключение". См. **также CLSID**, **COM**.

**прокси**

Объект, определенный интерфейсом, который обеспечивает маршалинг параметров и связь, необходимые для вызова клиентом объекта приложения, запущенного в другой среде выполнения, например в другом потоке или в другом процессе. Прокси-сервер расположен с клиентом и взаимодействует с соответствующим забоем, расположенным с объектом приложения, который называется. См. **также заблок.**

В начало

## <a name="r"></a>R

**относительный URL-адрес**

Частично квалифицированный URL-адрес, который указывает ресурс в Интернете или интрасети, расположение которого относительно начальной точки, указанной абсолютным URL-адресом или эквивалентным объектом ADO Connection. По сути, примитивные абсолютные и относительные URL-адреса с полным URL-адресом. См. **также URL-адрес** **и абсолютный URL-адрес.**

**удаленный источник данных**

Источник данных, который существует на другом компьютере, а не в локальной системе (где выполняется клиентская заявка).

**запись ресурса**

Запись поставщика исходных документов, который содержит поля для определения и описания папки или документа. Сам документ не содержится в записи ресурса, но обычно к нему можно получить доступ в потоке по умолчанию или поле в записи ресурса, содержащей URL-адрес. См. **также источник документов,** **поток по умолчанию**, **URL-адрес**.

**root**

Верхний уровень иерархической структуры дерева. Корневой узел не имеет родителей, но может иметь детей. См. **также иерархию**, **дерево**, **родитель**, **ребенок**.

**rowset**

Набор строк из источника данных с одинаковой схемой поля. Набор строк может представлять все или некоторые поля из таблицы. Набор строк также может представлять виртуальную таблицу, созданную запросом или соединив две или более таблиц. В ADO строки представлены **объектами Recordset.**

В начало

## <a name="s"></a>S

**schema**

Описание базы данных для системы управления базами данных (DBMS), обычно генерируемой с помощью языка определения данных, предоставленного DBMS. Схема определяет атрибуты базы данных, такие как таблицы, столбцы и свойства.

**scope**

Диапазон ссылок для объекта или переменной или диапазона записей в представлении или таблице. Например, локальные переменные можно ссылаться только в рамках процедуры, в которой они были определены. Общедоступные переменные доступны из любой точки приложения. Объекты, например текущая база данных, находятся в области, если они находятся в определенном пути поиска. Диапазоны записей можно укаменеть с оговоркой Область во многих командах.

**поставщик услуг**

Программное обеспечение, которое инкапсулирует службу путем создания и потребления данных, расширения функций в приложениях ADO. Это поставщик, который не предоставляет данные напрямую, а предоставляет службу, например обработку запросов. Поставщик услуг может обрабатывать данные, предоставляемые поставщиком данных. См. **также поставщик данных**.

**shaped Recordset**

Набор **записей,** столбцы которого были специально определены, чтобы содержать не только данные, но и ссылки (называемые главами) на другие объекты Recordset и/или вычислительные значения на основе других объектов **Recordset.** 

**sibling**

Любые два или несколько узлов иерархической структуры, которые находятся на одном уровне в иерархии. Корневой узел в иерархии не имеет братьев и сестер.

**хранимая процедура**

Предкомпиловый набор кода, например SQL и необязательных отчетов о контроле потока, хранимых под именем и обрабатываемых в качестве единицы. Сохраненные процедуры хранятся в базе данных; они могут выполняться одним вызовом из приложения и позволяют использовать объявленные пользователем переменные, условное выполнение и другие мощные функции программирования.

**stub**

Объект, определенный интерфейсом, который обеспечивает маршалинг параметров и связь, необходимые для объекта приложения для получения вызовов от клиента, который работает в другой среде выполнения, например в другом потоке или в другом процессе. Заблок расположен с объектом приложения и взаимодействует с соответствующим прокси-сервером, который находится с клиентом, который вызывает его. См. также **прокси.**

**sub-node**

См. **в .**

**синхронная операция**

Операция, инициированная кодом, завершаемой до начала следующей операции. См. **также асинхронную операцию.**

## <a name="t-w"></a>T-W

**дерево**

Структура, представляющая иерархическую связь между элементами (узлами). Существует один узел на верхнем уровне дерева (корень). Под корнем может быть несколько детей. Каждый ребенок может, в свою очередь, быть родителем других детей, таким образом ветвях, как дерево. Папка, содержащая документы и другие папки, является типичным примером структуры дерева. См. **также иерархию,** **узел**, **корень**, **ребенок**, **родитель**.

**URL-адрес (Единый локатор ресурсов)**

Указывает расположение ресурса, расположенного в Интернете или интрасети. Полный URL-адрес состоит из схемы (например, FTP, HTTP, mailto, file и так далее), за которой следуют двоеточие, имя сервера и полный путь ресурса (например, документ, графический или другой файл). Некоторые примеры URL-адресов:  
  
- https://www.domain.com/default.html  
- ftp://ftp.server.somewhere/ftp.file  
  
- ftp://ftp.server.somewhere/ftp.file  
- file://Server/Share/File.doc

См. **также абсолютный URL-адрес** **и относительный URL-адрес.**

**веб-сервер**

Компьютер, который предоставляет веб-службы и страницы пользователям интрасети и Интернета.
