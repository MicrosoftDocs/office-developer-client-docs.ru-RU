# <a name="ado-glossary"></a>Глоссарий ADO

**Область применения**: Access 2013, Office 2013

## <a name="a"></a>A

**абсолютный URL-адрес**

Полное URL-адрес, который указывает расположение ресурса, который находится в Интернете или интрасети. См. также **URL-адрес** **и относительный URL-адрес.**

**Элемент ActiveX**

Самозаверяющий in-process COM-компонент, который часто имеет визуальный элемент во время разработки или во время работы. ActiveX элементы управления также могут взаимодействовать с контейнером Active Document, например Microsoft Internet Explorer.

**ADISAPI (расширенный интерфейс программирования приложений Internet Server для данных)**

IsAPI DLL, которая обеспечивает разметку, управление автоматизацией, маршалинг **наборов** записей и упаковку MIME. Компонент ADISAPI работает через API, предоставляемый службами IIS. См. также **ISAPI.**

**агрегатная функция**

В запросе функция, например COUNT, AVG или STDEV, которая вычисляет значение, используя все строки в столбце таблицы. При написании выражений и программировании можно использовать SQL (включая три перечисленных выше) и агрегатные функции домена для определения различных статистических данных.

**alias**

Альтернативное имя, которое вы даете столбцу или выражению в SQL SELECT, часто короче или осмысленнее. Например, BobSales — это псевдоним в следующем заявлении SELECT: "Select wr-Sales as BobSales from SalesDB". Псевдоним можно использовать для динамического назначения столбцов для управления привязками объекта **DataControl.**

**потоковая цепочка apartment**

Модель потоков COM, в которой все вызовы объекта происходят в одном потоке. В потоке в многоквартирных домах COM синхронизирует и маршалирует вызовы. См. также **COM.**

**асинхронная операция**

Операция, которая возвращает управление вызываемой программе, не дожидаясь завершения операции. Перед завершением операции выполнение кода продолжается. См. также **синхронную операцию.**

## <a name="b"></a>B

**запись привязки**

Сопоставление между полем в таблице и переменной. В расширениях ADO Visual C++ поля **Recordset** соединит с переменными C/C++.

**битоваяmask**

Числовые значения, предназначенные для по битового сравнения значений с другими числовые значения, как правило, для пометки параметров в параметрах или возвращаемом значении. Обычно это сравнение происходит с помощью по битовой логики операторов, таких как **And** и **Or** в Visual Basic, а также **&** на **|** C++.

Например, значения ADO **FieldAttributeEnum** можно использовать в качестве битовыхmasks для определения атрибутов поля. Предположим, вы хотите определить, можно ли обновить поле. Вы можете проверить это с помощью следующего выражения в Visual Basic:  
  

Если результат имеет true, поле можно обновить.

**bookmark**

Маркер, который уникальным образом идентифицирует строку в наборе строк, чтобы пользователь быстро переходить к ней.

**бизнес-объект**

Объект, который выполняет определенный набор операций, например проверку данных или логику бизнес-правил. Бизнес-объекты обычно находятся на среднем уровне.

**бизнес-правило**

Сочетание проверочных изменений, проверок при выходе в систему, поиска базы данных, политик и алгоритмических преобразований, составляющих способ ведения бизнеса на предприятии. Также называется *бизнес-логикой.*

## <a name="c"></a>C

**вычисляемая выражение**

Выражение, которое не является постоянным, но значение которого зависит от других значений. Для оценки вычисленное выражение должно получать и вычислять значения из других источников, как правило, в других полях или строках.

**chapter**

Ссылка на диапазон строк из источника данных. В ADO глава обычно является ссылкой на другой **набор записей.**

Столбцы главы по возможности определяют отношение    "родитель-потомок", где родительским  является набор записей, содержащий столбец главы, а потомок — набор записей, представленный главой. 

**chapter-alias**

Псевдоним, который ссылается на столбец, который был придан родительскому столбце.

**кодировка**

Сопоставление набора символов с их числами. Например, Юникод — это 16-битный набор символов, способный кодировать все известные символы и использоваться в качестве всемирного стандарта кодирования символов.

**child**

Зависимая сторона иерархической связи. Child — это узел в иерархической структуре, который находится над другим узлом (ближе к корню). См. **также псевдоним "родитель-родитель",** **"родительский".** 

**child-alias**

Псевдоним, который ссылается на child. См. **также псевдоним**, **child**.

**CLSID (идентификатор класса)**

Универсальный уникальный идентификатор (UUID), определяющие com-компонент. Каждый компонент COM имеет свой CLSID в реестре Windows, чтобы его могли загружать другие приложения. См. также **ProgID**, **COM**.

**уровень клиента**

Логический уровень распределенной системы, которая обычно представляет данные пользователю и обрабатывает входные данные пользователя, иногда называется *интерфейсным.* Обычно клиентский уровень запрашивает данные с сервера на основе входных данных, а затем форматирование и отображение результата. См. также **средний уровень,** **уровень источника данных,** **распределенные приложения.**

**COM (объектная модель компонента)**

Двоичный стандарт, позволяющий объектам вести межсеансовую среду независимо от языка, на котором они были разработаны или на каких компьютерах они находятся. Технологии на основе COM включают ActiveX элементы управления, автоматизацию и связывание объектов и их встраивку (OLE). COM позволяет объекту раскрыть свои функции для других компонентов и для работы с приложениями. Он определяет, как объект предоставляет себя и как это экспозиция работает в процессах и в сетях. COM также определяет жизненный цикл объекта.

**Компонент COM**

Двоичный файл , например DLL, OCX и некоторые EXE-файлы, который поддерживает стандарт COM для предоставления объектов. Такой файл содержит код для одного или нескольких фабрик классов, классов COM, механизмов записи реестра, загрузки кода и так далее.

**оператор сравнения**

Оператор, который сравнивает два выражения и возвращает значение boolean.

Параметр условия, который может быть выражен как \> " (больше), \< " " (меньше), "=" (равно), " \> =" (больше или равно), " \< =" (меньше или равно), \< \> " " (не равно) или "like" (соответствие шаблона).

**component**

Объект, который инкапсулирует данные и код и предоставляет хорошо определенный набор общедоступных служб.

**составной файл**

Реализация структурированного хранилища COM для файлов. Составной файл хранит отдельные объекты в одном структурированный файл, состоящий из двух основных элементов: объектов хранилища и потоковых объектов. Вместе они функционируют как файловая система в файле. Дополнительные сведения см. в составных файлах в Microsoft Platform SDK.

Несколько отдельных файлов, связанных в одном физическом файле. К каждому отдельному файлу в составных файлах можно получить доступ как к одному физическому файлу.

**constant**

Числовая или строковая строка, которая не меняется. Именуемые enumerations ADO (enumerated constants) могут использоваться в коде вместо фактических значений, например **adUseClient** — это константа, значение которой 3. (Const adUseClient = 3). См. **также enumeration**.

**курсор**

Элемент базы данных, который управляет навигацией, возможностью обновления данных и видимостью изменений, внесенных в базу данных другими пользователями.

## <a name="d"></a>D

**привязка данных**

Процесс связывание объектов или элементов управления приложения с источником данных. Связанный с источником данных контроль называется привязанным *к данным.*

Содержимое привязанного к данным управления связано со значениями из базы данных. Например, при обновлении строк в объекте **Recordset** можно обновить сетку, привязанную к объекту **Recordset.** Когда набор записей извлекает новые **значения,** в сетке отображаются новые значения.

**поставщик данных**

Программное обеспечение, которое предоставляет данные приложению ADO напрямую или через поставщика услуг. См. также **поставщик услуг.**

**формирование данных**

Метод, который использует формализованный синтаксис (называемый языком **фигуры)** для определения специализированного объекта **Recordset** (называемого фигурным набором записей), который содержит не только данные, но и ссылки на другие объекты **Recordset** и/или вычисленные значения на основе этих других объектов **Recordset.**

**уровень источника данных**

Логический уровень распределенной системы, который представляет компьютер под управлением DBMS, например SQL Server данных. См. также **уровень клиента,** **средний уровень,** **распределенные приложения.**

**DCOM**

Проводной протокол, позволяющий com-компонентам напрямую взаимодействовать друг с другом по сети. См. также **COM**, **компонент**.

**DDL (язык определения данных)**

Эти заявления в SQL, которые определяют, в отличие от управления данными. Схема базы данных создается или изменена с помощью DDL. Например, **CREATE TABLE,** **CREATE INDEX,** **GRANT** и **REVOKE** SQL DDL.

**поток по умолчанию**

Текстовый или двоичный поток (представленный объектом **Stream),** связанный с объектами **Record** или **Recordset** при использовании определенных поставщиков OLE DB, таких как поставщик Microsoft OLE DB для публикации в Интернете. Поток по умолчанию обычно содержит содержимое файла, например HTML-код для корня веб-сайта.

**распределенные приложения**

Программа, написанная для разделения обработки на несколько компьютеров по сети. Как правило, распределенное приложение делится на уровни представления, бизнес-логики и хранения *данных.* См. также **уровень клиента,** **средний уровень,** **уровень источника данных.**

**disconnected Recordset**

Объект **Recordset** в клиентском кэше, который больше не имеет прямого подключения к серверу. Если исходный источник данных по какой-либо причине требуется получить доступ к исходному источнику данных, например к обновлению данных, подключение необходимо повторно установить. Однако доступ к коллекциям, свойствам и методам отключенного объекта **Recordset** по-прежнему можно получить.

**DLL (библиотека динамической ссылки)**

Файл, содержащий одну или несколько функций, которые компилются, связываются и хранятся отдельно от процессов, которые их используют. Операционная система сопоирует DLL в адресное пространство вызываемого процесса при запуске процесса или во время его работы.

**DML (язык обработки данных)**

Эти заявления в SQL, которые изменяют данные, а не определяют их. Значения в базе данных выбираются и меняются с помощью DML. Например, **INSERT,** **UPDATE,** **DELETE** и **SELECT** SQL DML.

**поставщик источника документов**

Особый класс поставщиков, которые управляют папками и документами. Когда документ представлен объектом **Record** или папка документов представлен объектом **Recordset,** поставщик источника документов заполняет эти объекты уникальным набором полей, описывая характеристики документа, а не сам документ. См. также **запись ресурса.**

**DSN (имя источника данных)**

Коллекция сведений, используемых для подключения приложения к определенной базе данных ODBC. Диспетчер драйверов ODBC использует эти сведения для создания подключения к базе данных. DSN может храниться в файле (DSN файла) или в реестре Windows (DSN компьютера).

**динамическое свойство**

Свойство, специфиственное для поставщика данных или службы курсора. Коллекция **свойств** объекта заполняется этими свойствами автоматически ("динамически"). У объекта нет динамических свойств, пока он не будет подключен к источнику данных через определенного поставщика данных. См. **также поставщик данных**, **курсор**.

## <a name="e-i"></a>E-I

**enumeration**

Список именуемой константы. Enumerated values need not be unique. Однако имя каждого значения должно быть уникальным в пределах области, в которой определено значение. В ADO, enumerations are used for numeric parameter and return values, to add meaning to ADO code and to shield the developer from the numeric values (which may change from version to version). Например, чтобы открыть статический **набор записей,** используйте **enumerated значение adOpenStatic:**  
  

Также называется *enumerated constant*. См. также **константы.**

**event**

Действие, распознано объектом, для которого можно написать код для ответа. События могут быть созданы в результате выполнения команды, завершения транзакций, навигации по набору записей и обновления данных, помимо других действий. См. также **обработок событий.**

**обработчик событий**

Обработок события — это код, который выполняется при выполнении события. См. также **событие**.

**handler**

Процедура, управляющая общим и относительно простым условием или операцией, например восстановлением ошибок или управлением данными.

**hierarchical Recordset**

Набор **записей,** содержащий другой **набор записей.** См. также **формирование данных**, **глава**.

Дополнительные сведения [см. в подсети "Доступ к строкам в иерархическом наборе записей".](accessing-rows-in-a-hierarchical-recordset.md)

**hierarchy**

В общем, иерархия — это ранжированные структуры с верхним и подчиненным уровнями. В ADO иерархические записи используются для представления родительско-родительской связи между записью и главой.  Кроме того, в ADO **объекты Record** и **Stream** можно использовать для доступа к иерархическим структурам дерева, таким как папка и документы. ADO MD также включает **объекты Иерархии,** которые представляют отношение между уровнями измерения в кубе OLAP. См. **также иерархические записи,** **родительские и родительские** отношения, **глава,** **дерево.**

**ISAPI (интерфейс программирования приложений Internet Server)**

Набор функций для интернет-серверов, таких как Windows NT Server/Windows 2000 Server с Microsoft IIS (IIS).

## <a name="k-m"></a>K-M

**key**

Столбец или столбцы в таблице, однозначно идентифицируют строку; часто используется для индексации таблицы.

**marshaling**

Процесс упаковки, отправки и распаковки параметров метода интерфейса через границы потока или процесса.

**средний уровень**

Логический уровень в распределенной системе между пользовательским интерфейсом или веб-клиентом и базой данных. В таких случаях обычно используются бизнес-объекты. Средний уровень — это коллекция бизнес-правил и функций, которые создают и работают при получении информации. Для этого используются бизнес-правила, которые могут меняться часто и поэтому инкапсулированы в компоненты, физически отделенные от самой логики приложения. Также известный как *уровень сервера приложений.* См. также **распределенные приложения,** **уровень клиента,** **уровень источника данных.**

**MIME (Multi-purpose Internet Mail Extension)**

Протокол Интернета, изначально разработанный для обмена электронными сообщениями электронной почты с форматом содержимого в разнородных сетях, на компьютерах и в средах электронной почты. На практике MIME также был принят и расширен приложениями, не относяжимся к почте.

MIME — это стандарт, позволяющий публиковать и считывать двоичные данные в Интернете. Заглавная папка файла с двоичными данными содержит тип MIME данных; Это сообщает клиентских программам (например, веб-браузерам и почтовым пакетам), что им потребуется обрабатывать данные не так, как при прямом тексте. Например, загон веб-документа, содержащего графику JPEG, содержит тип MIME, относящееся к формату файла JPEG. Это позволяет браузеру отобразить файл с помощью просмотра JPEG, если он присутствует.

## <a name="n-o"></a>N-O

**node**

Элемент в иерархической структуре дерева. Узел может быть корневым или дитя другого узла. Узел также может быть родительским для нескольких детей. См. **также иерархию,** **дерево,** **корневой,** **родительский,** **родительский.**

**переменная объекта**

Переменная, которая содержит ссылку на объект. Например, objCustomObject — это переменная, которая указывает на объект типа CustomObject:

**ODBC (Open Database Connectivity)**

Стандартный интерфейс языка программирования, используемый для подключения к различным источникам данных. Обычно доступ к этой функции можно получить с помощью панели управления, где имена источников данных (DSNs) могут быть назначены для использования определенных драйверов ODBC.

**OLE DB**.

Набор интерфейсов, которые выставляют данные из различных источников с помощью COM. Интерфейсы OLE DB предоставляют приложениям единый доступ к данным, хранимым в различных источниках информации. Эти интерфейсы поддерживают функциональные возможности DBMS, соответствующие источнику данных, позволяя им обмениваться данными. См. также **COM.**

**оптимистическая блокировка**

Тип блокировки, в котором страница данных, содержащая одну или несколько записей, в том числе редактируемую, недоступна другим пользователям только во время обновления записи с помощью метода **Update,** но доступна до и после вызова **update.**

Оптимистичная блокировка используется, когда объект **Recordset** открывается с параметром **LockType** или свойством **adLockOptimistic** или **adLockBatchOptimistic.** См. **также пессимистическую блокировку.**

**порядковая величина**

Числовое расположение элемента в заказе. В коллекции ADO порядковая величина первого элемента — ноль (0). Следующий элемент — один (1) и так далее.

В начало

## <a name="p"></a>P

**параметризованная команда**

Запрос или команда, которая позволяет установить значения параметров перед выполнением команды. Например, строку SQL можно задать с помощью встраив маркеры параметров в строку SQL (обозначенную символом "?"). Затем приложение указывает значения для каждого параметра и выполняет команду.

**родитель**

Контрольная сторона иерархической связи. В иерархической структуре родительский узел имеет один или несколько непосредственных узлов в иерархии. См. **также псевдоним "родитель-псевдоним",** **"родительский-родительский",** **"child".**

**parent-alias**

Псевдоним, который ссылается на родительский. См. **также псевдоним**, **родительский**.

**связь "родительский-родительский"**

Связь в иерархической структуре, в которой родительский уровень выше на один уровень и напрямую связан с одним или более потомком. Потомок на один уровень ниже и должен иметь один родительский уровень. См. также **родительский**, **родительский**.

**persist**

Сохранение данных в постоянном состоянии, например сохранение **наборов записей** в файл.

**пессимистическая блокировка**

Тип блокировки, в которой страница, содержащая одну или несколько записей, в том числе редактируемую, недоступна другим пользователям для обновления. Пессимистическое поведение блокировки определяется поставщиком OLE DB. Как правило, записи блокированы при редактировании и остаются недоступными до завершения работы метода **Update.**

Пессимистическая блокировка включена, когда объект **Recordset** открывается с помощью параметра **LockType** или свойства **adLockPessimistic.** См. также **оптимистичную блокировку.**

**pooling**

Оптимизация производительности на основе использования коллекций предварительно выделенных ресурсов, таких как объекты или подключения к базе данных. Эффективнее отрисовка существующего ресурса из пула, чем создание нового ресурса.

**ProgID (программный идентификатор)**

Уникальное имя, которое приложение COM соедино реестру Windows. ProgID для подключения ADO — "ADODB. Подключение". См. также **CLSID,** **COM.**

**proxy**

Объект для определенного интерфейса, который обеспечивает маршалинг параметров и связь, необходимые клиенту для вызова объекта приложения, запущенного в другой среде выполнения, например в другом потоке или другом процессе. Прокси-сервер находится с клиентом и взаимодействует с соответствующей загона, которая расположена с объектом приложения, который будет вызван. См. также **загона.**

В начало

## <a name="r"></a>R

**относительный URL-адрес**

Частично соответствующий URL-адрес, который указывает ресурс в Интернете или интрасети, расположение которого относительно начальной точки, указанной абсолютным URL-адресом или эквивалентным объектом подключения ADO. По сути, совмещенные абсолютные и относительные URL-адреса являются полными URL-адресами. См. также **URL-адрес** **и абсолютный URL-адрес.**

**удаленный источник данных**

Источник данных, который существует на другом компьютере, а не в локальной системе (где выполняется клиентские приложения).

**запись ресурса**

Запись от поставщика источника документов, которая содержит поля для определения и описания папки или документа. Сам документ не содержится в записи ресурса, но обычно к нему можно получить доступ с помощью потока по умолчанию или поля в записи ресурса, содержащей URL-адрес. См. также **поставщик источника документов,** **поток по умолчанию,** **URL-адрес.**

**root**

Верхний уровень иерархической структуры дерева. Корневой узел не имеет родительских узлов, но может иметь детей. См. **также иерархию,** **дерево,** **родительский**, **child**.

**rowset**

Набор строк из источника данных с одинаковой схемой поля. Набор строк может представлять все или некоторые поля из таблицы. Набор строк также может представлять виртуальную таблицу, созданную запросом или присоединив две или более таблиц. В ADO наборы строк представлены **объектами Recordset.**

В начало

## <a name="s"></a>S

**schema**

Описание базы данных в системе управления базами данных (DBMS), которое обычно создается с использованием языка определения данных, предоставляемого DBMS. Схема определяет атрибуты базы данных, такие как таблицы, столбцы и свойства.

**scope**

Диапазон ссылок для объекта или переменной или диапазона записей в представлении или таблице. Например, на локальные переменные можно ссылаться только в рамках процедуры, в которой они были определены. Общедоступные переменные доступны из любого места приложения. Объекты, например текущая база данных, находятся в области, если они находятся в определенном пути поиска. Диапазоны записей могут быть заданы с помощью предложения Scope во многих командах.

**поставщик услуг**

Программное обеспечение, которое инкапсулирует службу, выдав и потребляя данные, дополнив функции в приложениях ADO. Это поставщик, который не предоставляет данные напрямую, а предоставляет службу, например обработку запросов. Поставщик услуг может обрабатывать данные, предоставленные поставщиком данных. См. также **поставщик данных.**

**shaped Recordset**

Набор **записей,** столбцы которого были специально определены как содержащие не только данные, но и ссылки (называемые главами) на другие объекты **Recordset** и/или вычисленные значения на основе других объектов **Recordset.**

**sibling**

Любые два или более узлов в иерархической структуре, которые находятся на одном уровне иерархии. Корневой узел в иерархии не имеет ни одного уровня.

**хранимая процедура**

Предварительно компилировать коллекцию кода, например SQL и необязательные утверждения управления потоком, хранимые под именем и обрабатываемых как единица. Хранимые процедуры хранятся в базе данных; их можно выполнять одним вызовом из приложения и разрешить объявленные пользователем переменные, условное выполнение и другие мощные функции программирования.

**stub**

Объект для определенного интерфейса, который обеспечивает маршалинг параметров и связь, необходимые объекту приложения для получения вызовов от клиента, запущенного в другой среде выполнения, например в другом потоке или другом процессе. Загона расположена с объектом приложения и взаимодействует с соответствующим прокси-сервером, который находится с клиентом, который его вызывает. См. также **прокси-сервер.**

**sub-node**

См. **child**.

**синхронная операция**

Операция, инициированная кодом, которая завершается до запуска следующей операции. См. также **асинхронную операцию.**

В начало

## <a name="t-w"></a>T-W

**tree**

Структура, представляющая иерархическую связь между элементами (узлами). На верхнем уровне дерева (корневой) имеется один узел. Под корнем может быть несколько детей. Каждый из них, в свою очередь, может быть родительским для других детей, поэтому ветвяется как дерево. Папка, содержащая документы и другие папки, является типичным примером структуры дерева. См. **также иерархию,** **узел,** **корневой,** **родительский,** **родительский.**

**URL-адрес (Унифицированный поиск ресурсов)**

Указывает расположение ресурса, расположенного в Интернете или интрасети. Полный URL-адрес состоит из схемы (например, FTP, HTTP, mailto, file и так далее), двоеточия, имени сервера и полного пути к ресурсу (например, документу, графике или другому файлу). Вот несколько примеров URL-адресов:  
  
- https://www.domain.com/default.html  
- ftp://ftp.server.somewhere/ftp.file  
  
- ftp://ftp.server.somewhere/ftp.file  
- file://Server/Share/File.doc

См. также **абсолютный URL-адрес** **и относительный URL-адрес.**

**веб-сервер**

Компьютер, который предоставляет веб-службы и страницы пользователям интрасети и Интернета.
