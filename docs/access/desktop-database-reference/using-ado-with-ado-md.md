# <a name="using-ado-with-ado-md"></a>Использование ADO с ADO MD


**Область применения**: Access 2013, Office 2013

ADO и ADO MD являются связанными, но отдельными объектными моделями. ADO предоставляет объекты для подключения к источникам данных, выполнения команд, извлечения табличных данных и метаданных схемы в табличном формате и просмотра сведений об ошибках поставщика. ADO MD предоставляет объекты для получения многомерных данных и просмотра метаданных многомерных схем.

При работе с МДП вы можете использовать ADO, ADO MD или и то, и другое вместе с приложением. Ссылаясь на обе библиотеки в проекте, вы получите полный доступ к функциональным возможностям, предоставляемым вашим МДП.

Часто пользователи могут получить плоское табличное представление многомерного набора данных. Это можно сделать с помощью объекта [Recordset](recordset-object-ado.md) объекта ADO. Укажите источник для набора [ячеек](cellset-object-ado-md.md) в качестве ***исходного*** параметра для метода [Open](open-method-ado-recordset.md) объекта Recordset, а не источника для **набора** **ячеек**ADO MD.

Также может быть полезен Просмотр метаданных схемы в табличном представлении, а не в виде иерархии объектов. Метод ADO [OpenSchema](openschema-method-ado.md) для объекта [Connection](connection-object-ado.md) позволяет пользователю открыть объект **Recordset** , содержащий сведения о схеме. Параметр ***куеритипе*** метода **OpenSchema** имеет несколько значений [счемаенум](schemaenum.md) , которые относятся только к мдпс. Эти значения:

  - **Адсчемакубес**

  - **Адсчемадименсионс**

  - **Адсчемахиерарчиес**

  - **Адсчемалевелс**

  - **Адсчемамеасурес**

  - **Адсчемамемберс**

Чтобы использовать значения перечисления ADO с помощью свойств или методов ADO MD, проект должен ссылаться на библиотеки ADO и ADO MD. Например, вы можете использовать значения перечисления ADO **адстате** с помощью свойства ADO MD [State](state-property-ado-md.md) . Для получения дополнительных сведений об установке ссылок на библиотеки, ознакомьтесь с документацией по средству разработки.

Дополнительные сведения об объектах и методах ADO приведены в статье [Справочник по API ADO](ado-api-reference.md).
