# <a name="service-providers-and-components"></a>Компоненты и поставщики служб


**Область применения**: Access 2013, Office 2013

Поставщики услуг — это компоненты, расширяющие функциональность поставщиков данных, реализуя расширенные интерфейсы, которые не поддерживаются в хранилище данных.

Microsoft Data Access  предоставляет архитектуру компонентов, которая позволяет отдельным специализированным компонентам реализовывать дискретные наборы функциональных возможностей базы данных или "службы" на основе менее способных магазинов. Таким образом, вместо того, чтобы заставить каждый хранилище данных обеспечить собственную реализацию расширенных функций или принудить универсальные приложения к внутренней реализации функциональных возможностей баз данных, компоненты службы обеспечивают общую реализацию, которую любое приложение может использовать при доступе к любому хранилище данных. Тот факт, что некоторые функции реализуются на основе хранения данных, а некоторые — с помощью общих компонентов, является прозрачным для приложения.

Например, курсорный двигатель, например служба Microsoft Cursor для OLE DB, является компонентом службы, который может потреблять данные из последовательного хранения данных только для передачи прокрутки. Другие поставщики услуг, обычно используемые ADO, включают поставщика сохраняемости Microsoft OLE DB (для сохранения данных в файле), Службу формирования данных Майкрософт для OLE DB (для иерархических наборов записей) и поставщика remoting Microsoft OLE DB (для ссылок поставщиков данных на удаленном компьютере).

Дополнительные сведения о поставщиках услуг и данных см. в [приложении A: Providers.](appendix-a-providers.md)
