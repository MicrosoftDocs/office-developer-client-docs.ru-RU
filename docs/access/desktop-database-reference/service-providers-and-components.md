# <a name="service-providers-and-components"></a>Компоненты и поставщики служб


**Область применения**: Access 2013, Office 2013

Поставщики услуг — это компоненты, расширяющие функциональные возможности поставщиков данных путем реализации расширенных интерфейсов, которые не поддерживаются хранилищем данных.

Microsoft Data Access  предоставляет архитектуру компонентов, которая позволяет отдельным специализированным компонентам реализовать отдельные наборы функций базы данных или "службы" поверх менее функциональных хранилищ. Таким образом, вместо того чтобы заставить каждое хранилище данных обеспечить собственную реализацию расширенных функций или заставить универсальные приложения реализовать функции базы данных внутри организации, компоненты службы обеспечивают общую реализацию, которую может использовать любое приложение при доступе к любому окну данных. Тот факт, что некоторые функции реализованы в основном хранилищем данных, а некоторые — универсальными компонентами, является прозрачным для приложения.

Например, механизм курсора, например служба курсоров Майкрософт для OLE DB, — это компонент службы, который может использовать данные из последовательного, только однонаправленного хранения данных для создания прокручиваемых данных. К другим поставщикам услуг, часто используемым ADO, относятся поставщик сохраняемости Microsoft OLE DB (для сохранения данных в файле), служба формирования данных Майкрософт для OLE DB (для иерархических наборов записей) и поставщик удаленного управления Microsoft OLE DB (для передачи поставщиков данных на удаленном компьютере).

Дополнительные сведения о службах и поставщиках данных см. в приложении [А. Поставщики.](appendix-a-providers.md)
