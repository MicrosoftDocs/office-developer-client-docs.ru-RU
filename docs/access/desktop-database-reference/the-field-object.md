# <a name="field-object"></a>Объект поля


**Применимо к**: Access 2013, Office 2013

Каждый объект **поля** обычно соответствует столбец таблицы базы данных. Тем не менее, **поле** также может представлять указатель на другой **набор записей**, называется главу. Исключения, такие как главы столбцы будут описаны позже в этом руководстве.

Свойство **значение** **поля** объектов или возвращает данные для текущей записи. В зависимости от функциональности предоставляет поставщик, некоторые коллекции методов, или свойства объекта **поля** могут быть недоступны.

С помощью семейства сайтов, методы и свойства объекта **поля** сделайте следующее:

  - Возвращает имя поля, с помощью свойства **Name** .

  - Просмотр или изменение данных в поле с помощью свойства **Value** . **Значение** является свойством по умолчанию объекта **поля** .

  - Возвращает основные характеристики поля с помощью свойства **типа**, **точность**и **NumericScale** .

  - Возвращает размер объявленные поля с помощью свойства **DefinedSize** .

  - Возвращает фактический размер данных в этом поле, используя свойство **ActualSize** .

  - Определите, какие функциональные возможности поддерживаются для данного поля с помощью свойства **атрибуты** и **Свойства** коллекции.

  - Работа с элементами управления значения полей, содержащие данные длинный двоичные или много символов с помощью методов **AppendChunk** и **GetChunk** .

Разрешения несоответствий в значения полей во время пакета обновления с помощью свойства **OriginalValue** и **UnderlyingValue** , если поставщик поддерживает пакетные обновления.

## <a name="describing-a-field"></a>Описание поля

Разделы, следуйте будет обсудить свойства объекта [поля](field-object-ado.md) , которые представляют сведения, описывающие самого объекта **поля** — метаданные о поле. Эти сведения можно использовать для определения значительная о схеме **набора записей**. Эти свойства включают **Тип**, **DefinedSize** и **ActualSize**, **имя**и **NumericScale** и **точность**.

## <a name="discovering-the-data-type"></a>Обнаружение тип данных

Свойство **Type** указывает тип данных поля. Константы типа перечисления данных, поддерживаемых приложением ADO, описаны в [DataTypeEnum](datatypeenum.md) в *Справочник программиста ADO*.

Для типы с плавающей запятой числовое таких **adNumeric**, можно получить дополнительные сведения. Свойство **NumericScale** указывает количество цифр справа от десятичной запятой будет использоваться для представления значений для **поля**. Свойство **точности** указывает максимальное число знаков, используемых для представления значений для **поля**.

## <a name="determining-field-size"></a>Определение размера поля

Свойство **DefinedSize** используется для определения емкости данных объекта **поля** .

Свойство **ActualSize** возвращает фактический длины значения объекта **поля** . Для всех полей **ActualSize** свойство доступно только для чтения. Если ADO не может быть определен значение **поля** объекта, свойство **ActualSize** возвращает **adUnknown**.

Свойства **DefinedSize** и **ActualSize** имеют различных целей. Например рассмотрим объект **Field** с типом объявленные **adVarChar** и значение свойства **DefinedSize** 50, содержащий один символ. **ActualSize** свойство возвращает значение длины в байтах один символ.

## <a name="determining-field-contents"></a>Определение полей

Идентификатор столбца из источника данных, представленного **имя** свойства **поля**. Свойство **Value** объекта **Field** Возвращает или задает фактические данные содержимое поля. Это свойство по умолчанию.

Для изменения данных в поле, задайте **значение** свойства новое значение соответствующего типа. Вашей текущей позиции тип должен поддерживать обновлений для изменения значения поля. База данных не проверяются здесь в пакетном режиме, поэтому требуется проверка ошибок при вызове **UpdateBatch** в таком случае. Некоторые поставщики также поддерживают свойства объекта ADO **поля** **UnderlyingValue** и **OriginalValue** для помощи по Устранение конфликтов при попытке выполнить пакетные обновления. Подробные сведения о способах устранения таких конфликтов в разделе [раздел 4: редактирование данных](chapter-4-editing-data.md).

## <a name="getting-more-field-information"></a>Получение дополнительных сведений о поле

Объекты ADO имеют два типа свойств: встроенные и динамические. На данный момент рассматриваются только встроенные свойства объекта **поля** .

Встроенные свойства — это свойства, реализованные в ADO и сразу же доступны для каждого нового объекта, используя следующий синтаксис. Они не отображаются как **свойство** объектов в коллекции **свойств** объекта.

Динамические свойства определяются базового поставщика данных и отображаются в коллекцию **свойств** подходящий объект ADO. Например свойство поставщика может указывать, поддерживает ли объект **набора записей** транзакций или обновление. Эти дополнительные свойства отображаются как **свойство** объектов в коллекции **свойств** объекта **набора записей** . Динамические свойства можно ссылаться только с помощью коллекции, используя синтаксис MyObject.Properties(0) или или MyObject.Properties("Name").

Не удается удалить любой тип свойства.

Объект динамического **Свойства** имеет четыре встроенных свойства свои собственные.

  - Свойство **Name** — это строка, которая определяет свойство.

  - Свойство **Type** — целое число, определяющее тип данных свойства.

  - Свойство **Value** имеет тип variant, содержащее значение свойства. **Значение** является свойством по умолчанию для **Свойства** объекта.

  - Свойство **Attributes** — это значение типа **Long** , указывающее характеристики свойства поставщика.

Коллекции **свойств** для объекта **поля** содержит дополнительные метаданные о поле. Содержимое коллекции различаться в зависимости от поставщика. В следующем примере кода проверяет коллекции **свойств** пример, который **записей** введена в начале статьи. Сначала он просматривает содержимое коллекции. Этот код использует [Поставщик OLE DB для SQL Server](microsoft-ole-db-provider-for-sql-server.md), поэтому коллекции **свойств** содержит сведения, относящиеся к этому поставщику.

```vb 
 
'BeginFieldProps 
 Dim objProp As ADODB.Property 
 
 For intLoop = 0 To (objFields.Count - 1) 
 Debug.Print objFields.Item(intLoop).Name 
 
 For Each objProp In objFields(intLoop).Properties 
 Debug.Print vbTab & objProp.Name & " = " & objProp.Value 
 Next objProp 
 Next intLoop 
'EndFieldProps 
```

## <a name="dealing-with-binary-data"></a>Работа с двоичные данные

Используйте метод [AppendChunk](appendchunk-method-ado.md) для объекта **поля** для наполнения данными длинный двоичные или знак. В случаях, когда системной памяти ограниченный можно использовать метод **AppendChunk** для работы с длинные значения в части, а не полностью.

Если бит **adFldLong** в свойство **Attributes** объекта **поля** задано значение **True**, можно использовать метод **AppendChunk** для этого поля.

Первый вызов **AppendChunk** для объекта **поля** записывает данные в поле перезапись существующих данных. Последующие вызовы **AppendChunk** добавьте существующих данных. При добавлении данных в одном поле, а затем задайте или читать значение другого поля в текущей записи, ADO предполагается, что окончании добавления данных до первого поля. При вызове метода **AppendChunk** в первом поле еще раз, ADO интерпретирует звонок как операции создания **AppendChunk** и перезапись существующих данных. Доступ к полей в другие объекты **набора записей** , которые не копирует первый объект **набора записей** не нарушить **AppendChunk** операций.

Используйте метод **GetChunk** для объекта **поля** для извлечения или часть ее длинные двоичный или данных символов. В случаях, когда системной памяти ограниченный можно использовать метод **GetChunk** для работы с длинные значения в частях, а не полностью.

Данные, вызов **GetChunk** возвращает назначается *переменной*. Если *размер* больше, чем остальные данные, метод **GetChunk** возвращает только оставшихся данных без заполнения *переменной* пробелами. Если поле не заполнено, метод **GetChunk** возвращает значение null.

Каждый последующий вызов **GetChunk** возвращает данные, начиная с предыдущего звонка **GetChunk** же места. Тем не менее если извлекаются данные из одного поля и задайте его или читать значение другого поля в текущей записи, ADO предполагается, что Вы завершили извлечения данных из первого поля. При вызове метода **GetChunk** в первом поле еще раз, ADO интерпретирует звонок как операции создания **GetChunk** и запускает чтения с самого начала данных. Доступ к полей в другие объекты **набора записей** , которые не копирует первый объект **набора записей** не нарушить **GetChunk** операций.

Если бит **adFldLong** в свойство **Attributes** объекта **поля** задано значение **True**, можно использовать метод **GetChunk** для этого поля.

Если при использовании **GetChunk** или метод **AppendChunk** объекта **поля** нет текущей записи, возникает ошибка 3021 (текущая запись).

Пример использования эти методы для работы с двоичных данных в примерах [Методы GetChunk](getchunk-method-ado.md) и [AppendChunk](appendchunk-method-ado.md) в *Справочник программиста ADO*.
