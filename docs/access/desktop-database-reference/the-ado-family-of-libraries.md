# <a name="the-ado-family-of-libraries"></a>Семейство библиотек ADO

**Область применения**: Access 2013, Office 2013

Семейство ADO состоит из трех основных библиотек: ADO (включая RDS), ADO MD и ADOX.

## <a name="ado"></a>ОБЪЕКТЫ

ADO позволяет клиентским приложениям получать данные с сервера баз данных и управлять ими с помощью поставщика OLE DB. Основные преимущества ADO — простота использования, высокая скорость, недостаток памяти и небольшое место на диске. ADO поддерживает основные функции для создания клиентских и серверных и веб-приложений.

Кроме того, ADO поддерживает удаленную службу данных (RDS), с помощью которой можно перемещать данные с сервера в клиентское приложение или веб-страницу, управлять данными на клиенте и возвращать обновления на сервер в едином цикле обработки.

## <a name="ado-md"></a>ADO MD

Объекты данных Microsoft ActiveX (многомерные) (ADO MD) обеспечивают простой доступ к многомерным данным из таких языков, как Microsoft Visual Basic, Microsoft Visual C++ и Microsoft Visual J++. ADO MD расширяет ADO, чтобы включить объекты, относящиеся к многомерным данным, например объектам **CubeDef** и набору **ячеек** . С помощью ADO MD можно просматривать многомерную схему, запрашивать куб и получать результаты.

Как и в ADO, ADO MD использует поставщика OLE DB для получения доступа к данным. Для работы с ADO MD поставщик должен быть многомерным поставщиком данных (МДП), как определено в спецификации OLE DB для OLAP. Мдпс представляют данные в многомерных представлениях, а не в табличных поставщиках данных (Тдпс), которые представляют данные в табличных представлениях. Для получения более подробных сведений о синтаксисе и возМожностях, поддерживаемых поставщиком, обратитесь к документации по поставщику OLE DB для поставщика OLAP.

## <a name="adox"></a>ADOX

Расширения объектов данных Microsoft ActiveX для языка определения данных и безопасности (ADOX) являются расширением объектов ADO и программной модели. ADOX включает объекты для создания и изменения схемы, а также для обеспечения безопасности. Так как это подход, основанный на объектах, для управления схемой, вы можете написать код, который будет работать с разными источниками данных независимо от различий в их исходных синтаксисах.

ADOX — вспомогательная библиотека для базовых объектов ADO. Он предоставляет доступ к дополнительным объектам для создания, изменения и удаления объектов схемы, таких как таблицы и процедуры. Он также включает объекты безопасности для поддержки пользователей и групп, а также для предоставления и отзыва разрешений на объекты.
