<span data-ttu-id="8d48e-104">Для Microsoft Visual J++ 6,0 объекты ADO были расширены для работы с классами Windows Foundation (WFC) следующими способами.</span><span class="sxs-lookup"><span data-stu-id="8d48e-104">For Microsoft Visual J++ 6.0, ADO has been extended to work with Windows Foundation Classes (WFC) in the following ways.</span></span> <span data-ttu-id="8d48e-105">В первую очередь реализован набор классов Java, расширяющий интерфейсы ADO и создающий уведомления, представляющие интерес для программиста Java; классы Java также предоставляют функции, возвращающие для пользователя типы Java.</span><span class="sxs-lookup"><span data-stu-id="8d48e-105">First, a set of Java classes has been implemented that extends the ADO interfaces and creates notifications interesting to the Java programmer; the Java classes also expose functions that return Java types to the user.</span></span> <span data-ttu-id="8d48e-106">Для повышения производительности класс Java напрямую получает доступ к собственным типам данных в объекте набора строк OLE DB и возвращает их программистам Java как типы Java без предварительного преобразования их в тип Variant и обратно.</span><span class="sxs-lookup"><span data-stu-id="8d48e-106">To improve performance, the Java class directly accesses the native data types in the OLE DB rowset object, and returns them to the Java programmer as Java types without first converting them to and from a variant.</span></span> <span data-ttu-id="8d48e-107">Кроме того, ADO был расширен для работы с уведомлениями о событиях в платформе WFC Framework.</span><span class="sxs-lookup"><span data-stu-id="8d48e-107">ADO has also been extended to work with event notifications in the WFC framework.</span></span>

<span data-ttu-id="8d48e-108">Классы ADO для Windows Foundation (ADO/WFC) поддерживают все стандартные методы, свойства, объекты и события ADO.</span><span class="sxs-lookup"><span data-stu-id="8d48e-108">ADO for Windows Foundation Classes (ADO/WFC) supports all the standard ADO methods, properties, objects, and events.</span></span> <span data-ttu-id="8d48e-109">Тем не менее, операции, требующие использования Variant в качестве параметра и показывающие отличное быстродействие на языке, например Microsoft Visual Basic, отображают меньшую производительность на таком языке, как Visual J++.</span><span class="sxs-lookup"><span data-stu-id="8d48e-109">However, operations that require a variant as a parameter and show excellent performance in a language such as Microsoft Visual Basic, display lesser performance in a language such as Visual J++.</span></span> <span data-ttu-id="8d48e-110">По этой причине ADO/WFC также предоставляет функции для доступа к объекту [field](field-object-ado.md) , которые используют собственные типы данных Java вместо типа данных Variant.</span><span class="sxs-lookup"><span data-stu-id="8d48e-110">For that reason, ADO/WFC also provides accessor functions on the [Field](field-object-ado.md) object that take native Java data types instead of the variant data type.</span></span>
