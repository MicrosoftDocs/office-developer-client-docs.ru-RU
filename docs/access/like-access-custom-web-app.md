# <a name="like-access-custom-web-app"></a>LIKE (пользовательское веб-приложение Access)

Определяет, соответствует ли определенная строка символов указанному шаблону. Шаблон может включать обычные символы и под шаблоны. При совпадении шаблонов обычные символы должны точно соответствовать символам, указанным в строке символов. Тем не менее, поддиадрочные знаки можно соединить с произвольными фрагментами строки символов. Использование поддиапных знаков делает **оператор LIKE** более гибким, чем использование операторов сравнения строк = и != . 
  
> [!IMPORTANT]
> Корпорация Майкрософт в настоящее время не рекомендует создавать и использовать веб-приложения Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/) для создания бизнес-решений без кода для Интернета и мобильных устройств. 
  
## <a name="syntax"></a>Синтаксис

 *Выражение*  [ NOT ] **LIKE** *Pattern*  [ ESCAPE  *EscapeChar*  ] 
  
Оператор **LIKE** содержит следующие аргументы 
  
|**Имя аргумента**|**Обязательный**|**Описание**|
|:-----|:-----|:-----|
| *Expression*  <br/> |Да  <br/> |Допустимые выражения.  <br/> |
| *Pattern*  <br/> |Да  <br/> |Определенная строка символов для поиска в  *Expression*  . Может включать поддиаными знаками. Список допустимого поддеревного знака можно найти в примечаний.  <br/> |
| *EscapeChar*  <br/> |Нет  <br/> |Символ, который находится перед поддиаными знаками, чтобы указать, что под ним следует интерпретировать как обычный символ, а не как поддианом.  *EscapeChar*  — это выражение символа, которое не имеет значения по умолчанию и должно иметь только один символ.  <br/> |
   
## <a name="remarks"></a>Примечания

В следующей таблице содержатся под шаблонные знаки, допустимые для использования в *аргументе Pattern.* 
  
|**Под wildcard character**|**Описание**|**Пример**|
|:-----|:-----|:-----|
|%  <br/> |Любая строка ноль или больше символов.  <br/> | *WHERE title LIKE '%computer%'*  finds all book titles with the word 'computer' anywhere in the book title.  <br/> |
|_ (знак подчеркивания)  <br/> |Любой отдельный знак.  <br/> | *Where au_fname LIKE '_ean'*  finds all four-letter first names that end with ean (Айт, Юли и так далее).  <br/> |
|[]  <br/> |Любой символ в указанном диапазоне ([a-f]) или набор ([abcdef]).  <br/> | *WHERE au_lname LIKE '[C-P]karn'*  находит фамилии авторов, заканчивающийся на вален и начинающийся с любого отдельного символа между C и P, например Карен, Larsen, Karsen и так далее.  <br/> |
|[^]  <br/> |Любой символ, не в пределах указанного диапазона ([^a-f]) или установленный ([^abcdef]).  <br/> | *WHERE au_lname LIKE 'de[^l]%'*  all author last names starting with de and where the following letter is not l.  <br/> |
   
При выполнении сравнения строк с помощью **LIKE** все символы в строке шаблона имеют значительное значение. К ним относятся пробелы в первой или вехи. Если сравнение в запросе возвращает все строки со строкой **LIKE** "abc" (abc, за которой следует одно пространство), строка, в которой значение этого столбца — abc (abc без пробела) не возвращается. Однако пустые в конце выражения, с которым совпадает шаблон, игнорируются. Если сравнение в запросе возвращает все строки со строкой **LIKE** "abc" (abc без пробела), возвращаются все строки, начинающиеся с abc и пустые в конце строки с нулем или более. 
  
Если какой-либо из аргументов не является типом строки данных, он по возможности преобразуется в тип данных строки.
