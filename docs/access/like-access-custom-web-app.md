# <a name="like-access-custom-web-app"></a>КАК (приложение настраиваемых web Access)

Определяет, соответствует ли заданная строка символов заданному шаблону. Шаблон может включать в себя обычные символы и подстановочные знаки. При обработке шаблона обычные символы должен полностью совпадать с символами, указанными в строке. Тем не менее подстановочные знаки может совпадать с произвольной фрагментов строку знаков. Использование подстановочных знаков делает оператор **LIKE** более гибкий, чем использование = и! = операторов сравнения строк. 
  
> [!IMPORTANT]
> Корпорация Майкрософт больше не рекомендует создавать и использовать веб-приложения для Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/), чтобы создавать бизнес-решения без кода для Интернета и мобильных устройств. 
  
## <a name="syntax"></a>Синтаксис

 *Выражение*  [НЕ] **Как** *Шаблон*  [ESCAPE- *EscapeChar* ] 
  
Оператор **LIKE** содержит следующие аргументы 
  
|**Имя аргумента**|**Обязательное**|**Описание**|
|:-----|:-----|:-----|
| *Выражение*  <br/> |Да  <br/> |Допустимое выражение.  <br/> |
| *Шаблон*  <br/> |Да  <br/> |Определенные строковые символов для поиска в *выражении* . Может включать подстановочные знаки. Обратитесь к примечания список допустимый подстановочные знаки.  <br/> |
| *EscapeChar*  <br/> |Нет  <br/> |Символ, который размещается перед подстановочный знак, чтобы указать, что подстановочный знак следует интерпретировать как обычный символ, а не как подстановочный знак.  *EscapeChar* является выражением символ, который не имеет стандартного и должен быть только один символ.  <br/> |
   
## <a name="remarks"></a>Замечания

В следующей таблице приведены подстановочные знаки, которые можно использовать в качестве аргумента *шаблона* . 
  
|**Подстановочный знак**|**Описание**|**Пример**|
|:-----|:-----|:-----|
|%  <br/> |Любая строка ноль или более символов.  <br/> | *ГДЕ title LIKE '% компьютер %'* находит все названия книг, содержащие слово «компьютер» в любом месте в название книги.  <br/> |
|_ (знак подчеркивания)  <br/> |Любой отдельный знак.  <br/> | *ГДЕ au_fname как «_етров»* поиск всех четырех имена которых оканчиваются ean (Дин, Sean и т. д.).  <br/> |
|[]  <br/> |Любой отдельный символ в заданном диапазоне ([a-f]) или наборе ([abcdef]).  <br/> | Находит *WHERE Фамилия_автора как «[C-С] омов»* фамилии, заканчивающихся омов и начиная с любой отдельный знак между C и P, например Ломов, Ларсен Сомов и т. д.  <br/> |
|[^]  <br/> |Любой отдельный знак, не в заданном диапазоне ([^ a-f]) или задать ([^ abcdef]).  <br/> | *WHERE Фамилия_автора как "de [^ l] %'* все фамилии, начинающиеся с de и букву не является l.  <br/> |
   
При выполнении сравнения строк с помощью **как**высоки все символы в строке шаблона. Этот компонент включает начальных и конечных пробелов. Если операция сравнения в запросе для возврата всех строк, содержащих **как** 'abc' (abc следуют пробел), строку, в котором значение этого столбца — abc (без должен быть пробел) не возвращается. Тем не менее конечные пробелы в выражении, к которому сравнивается шаблон, учитываются. Если операция сравнения в запросе для возврата все строки, содержащие строки **LIKE** 'abc' (без пробела), будут возвращены все строки, начинающиеся с abc и иметь ноль или больше конечные пробелы. 
  
Если любой из аргументов не относится к строковому типу данных, преобразуется в тип данных string, если это возможно.
