# <a name="stopmacro-macro-action-access-custom-web-app"></a>Макрокоманда Макрокоманда StopMacro (пользовательское веб-приложение для Access)

Чтобы остановить выполняемый в данный момент макрос, можно использовать действие **Макрокоманда StopMacro** . 
  
> [!IMPORTANT]
> Корпорация Майкрософт в настоящее время не рекомендует создавать и использовать веб-приложения Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/) для создания бизнес-решений без кода для Интернета и мобильных устройств. 
  
## <a name="setting"></a>Параметр

Макрокоманда **Макрокоманда StopMacro** не содержит аргументов. 
  
## <a name="remarks"></a>Примечания

Обычно это действие используется при условии, что необходимо остановить макрос. Например, вы можете создать макрос ПОЛЬЗОВАТЕЛЬСКОГО интерфейса, который открывает представление, в котором отображаются итоговые значения ежедневных заказов для даты, введенной в текущем представлении. Можно использовать условное выражение, чтобы убедиться, что элемент управления "Дата заказа" в диалоговом окне содержит допустимую дату. В противном случае действие **MessageBox** может отобразить сообщение об ошибке, а действие **Макрокоманда StopMacro** может остановить макрос пользовательского интерфейса.
