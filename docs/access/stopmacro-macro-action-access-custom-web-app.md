# <a name="stopmacro-macro-action-access-custom-web-app"></a>StopMacro Macro Action (Доступ к настраиваемой веб-приложению)

Вы можете использовать действие **StopMacro,** чтобы остановить текущий макрос. 
  
> [!IMPORTANT]
> Корпорация Майкрософт в настоящее время не рекомендует создавать и использовать веб-приложения Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/) для создания бизнес-решений без кода для Интернета и мобильных устройств. 
  
## <a name="setting"></a>Параметр

Действие **StopMacro** не имеет аргументов. 
  
## <a name="remarks"></a>Примечания

Обычно это действие используется, когда из-за условия необходимо остановить макрос. Например, можно создать макрос пользовательского интерфейса (пользовательского интерфейса), который откроет представление, показывающую ежедневные итоги заказа для даты, введенной в текущем представлении. Можно использовать условное выражение, чтобы убедиться, что контроль даты заказа в диалоговом окне содержит допустимую дату. Если этого не происходит, действие **MessageBox** может отображать сообщение об ошибке, а действие **StopMacro** может остановить макрос пользовательского интерфейса.
