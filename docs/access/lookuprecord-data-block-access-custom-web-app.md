# <a name="lookuprecord-data-block-access-custom-web-app"></a>LookupRecord Data Block (Access custom web app)

Блок **данных LookupRecord** выполняет набор действий с определенной записью. 
  
> [!IMPORTANT]
> Корпорация Майкрософт в настоящее время не рекомендует создавать и использовать веб-приложения Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/) для создания бизнес-решений без кода для Интернета и мобильных устройств. 
  
> [!NOTE]
> Блок **данных LookupRecord** доступен только в макросах данных. 
  
## <a name="setting"></a>Setting

Действие **SetField** имеет следующие аргументы. 
  
|**Аргумент**|**Обязательный**|**Описание**|
|:-----|:-----|:-----|
| _In_ <br/> |Да  <br/> |Строка, идентифицирует запись, с которую необходимо работать. Аргумент  *In*  может содержать имя таблицы, запрос выбора или SQL.  <br/> |
| _Where Condition_ <br/> |Нет  <br/> |Строка выражения, используемая для ограничения диапазона данных, для которого выполняется блок данных **LookupRecord.** Например, критерии часто эквивалентны предложению WHERE в SQL выражения без слова WHERE. Если критерии не указаны, блок данных **LookupRecord** работает на всем домене, указанном аргументом *In.* Любое поле, включенное в условия, также должно быть полем в  *in*  .  <br/> |
| _Alias_ <br/> |Нет  <br/> |Строка, которая предоставляет альтернативное имя для записи, указанной в *аргументе In.* Часто используется для сокращения имени таблицы для последующих ссылок, чтобы предотвратить возможные неоднозначные ссылки. Если  *псевдоним не*  указан, в качестве псевдонима будет использоваться имя таблицы или запроса.  <br/> |
   
## <a name="remarks"></a>Примечания

Если в аргументах  условия  "В" и "Где" указано несколько записей, блок данных **LookupRecord** будет работать только с первой записью. 
  
Если ни одна запись не удовлетворяет  условию *where или* если in не содержит записей, **То LookupRecord** создает пустую запись, в которой все поля содержат значение **Null.**
