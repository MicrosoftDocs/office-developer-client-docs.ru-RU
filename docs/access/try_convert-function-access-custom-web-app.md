# <a name="try_convert-function-access-custom-web-app"></a>Try_Convert Function (Доступ к настраиваемой веб-приложению)

Преобразует значение в указанный тип данных или возвращает Null, если преобразование не допустимо.
  
> [!IMPORTANT]
> Корпорация Майкрософт в настоящее время не рекомендует создавать и использовать веб-приложения Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/) для создания бизнес-решений без кода для Интернета и мобильных устройств. 
  
## <a name="syntax"></a>Синтаксис

 **Try_Convert** *(DataType*, *Expression*) 
  
Функция **Try_Convert** содержит следующие аргументы. 
  
|**Имя аргумента**|**Описание**|
|:-----|:-----|
| *DataType*  <br/> |Тип данных, в который необходимо преобразовать *Expression.*  <br/> |
| *Expression*  <br/> |Значение, необходимое для преобразования.  <br/> |
   
## <a name="remarks"></a>Примечания

 **Try_Convert** принимает переданные ему значения и пытается преобразовать его в указанный *DataType.* Если преобразование успешно, **Try_Convert** возвращает значение в качестве указанного  *DataType;*  если возникает ошибка, возвращается null. Однако если вы запросили преобразование, которое  явно не разрешено, Try_Convert с ошибкой.
