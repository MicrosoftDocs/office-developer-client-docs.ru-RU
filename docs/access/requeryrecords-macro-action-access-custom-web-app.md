# <a name="requeryrecords-macro-action-access-custom-web-app"></a>Действия макроса RequeryRecords (приложение настраиваемых web Access)

Действие **RequeryRecords** можно использовать для обновления, сортировки и фильтрации данных в активном представлении повторным запросом источника представления. 
  
> [!IMPORTANT]
> Корпорация Майкрософт больше не рекомендует создавать и использовать веб-приложения для Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/), чтобы создавать бизнес-решения без кода для Интернета и мобильных устройств. 
  
## <a name="setting"></a>Параметр

Действие **RequeryRecords** состоит из следующих аргументов. 
  
|**Параметр**|**Обязательное**|**Описание**|
|:-----|:-----|:-----|
|**Где =** <br/> |Нет  <br/> |SQL предложение WHERE, которое ограничивает число записей в представлении. По умолчанию этот аргумент не задан.  <br/> |
|**OrderBy** <br/> |Нет  <br/> |Строковое выражение, которое включает в себя имя поля или полей, по которому выполняется сортировка записей и необязательные ASC или DESC ключевых слов. По умолчанию этот аргумент не задан.  <br/> |
   
## <a name="remarks"></a>Замечания

При вызове действие **RequeryRecords** сортировки или фильтрации применяется пользователем снят. 
  
Аргумент *OrderBy* — это имя в поле или поля, в которых используется для сортировки записей. При использовании более одного имени поля, разделяйте имена запятыми (,). 
  
Если значение аргумента *OrderBy* , записи сортируются в порядке возрастания по умолчанию. 
  
Чтобы отсортировать записи в убывающем порядке, введите DESC в конце выражения аргумента *OrderBy* . Например для сортировки записей клиентов в порядке убывания имя контакта, задайте *OrderBy* аргумент «ContactName DESC». 
  
Чтобы отсортировать имена, LastName по убыванию и FirstName по возрастанию, задайте *OrderBy* «Фамилия Desc, имя ASC».
