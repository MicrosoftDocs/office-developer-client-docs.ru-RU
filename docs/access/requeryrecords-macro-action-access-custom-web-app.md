# <a name="requeryrecords-macro-action-access-custom-web-app"></a>Макрокоманда RequeryRecords (пользовательское веб-приложение для Access)

Вы можете использовать действие **RequeryRecords** для обновления, сортировки и фильтрации данных в активном представлении, запрашивая источник представления. 
  
> [!IMPORTANT]
> Корпорация Майкрософт больше не рекомендует создавать и использовать веб-приложения для Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/), чтобы создавать бизнес-решения без кода для Интернета и мобильных устройств. 
  
## <a name="setting"></a>Параметр

Макрокоманда **RequeryRecords** имеет следующие аргументы. 
  
|**Параметр**|**Обязательный**|**Описание**|
|:-----|:-----|:-----|
|**Где =** <br/> |Нет  <br/> |Предложение WHERE (SQL), которое позволяет ограничить записи в представлении. По умолчанию этот аргумент не задан.  <br/> |
|**OrderBy** <br/> |Нет  <br/> |Строковое выражение, включающее имя поля или полей, по которым сортируются записи, а также необязательные ключевые слова ASC и DESC. По умолчанию этот аргумент не задан.  <br/> |
   
## <a name="remarks"></a>Замечания

При вызове действия **RequeryRecords** удаляется любая сортировка или фильтрация, примененные пользователем. 
  
Аргумент *OrderBy* — это имя поля или полей, для которых требуется отсортировать записи. При использовании нескольких имен полей разделяйте их запятыми (,). 
  
При задании аргумента *OrderBy* записи сортируются по умолчанию в порядке возрастания. 
  
Чтобы отсортировать записи в убывающем порядке, введите DESC в конце выражения аргумента *OrderBy* . Например, чтобы отсортировать записи клиентов по убыванию по имени контакта, задайте для аргумента *OrderBy* значение "ContactName DESC". 
  
Чтобы сортировать имена по убыванию и по возрастанию, задайте для аргумента *OrderBy* значение "фамилия DESC, имя ASC".
