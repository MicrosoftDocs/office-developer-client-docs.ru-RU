# <a name="requeryrecords-macro-action-access-custom-web-app"></a>RequeryRecords Macro Action (Access custom web app)

Вы можете использовать действие **RequeryRecords** для обновления, сортировки и фильтрации данных в активном представлении путем повторного представления источника представления. 
  
> [!IMPORTANT]
> Корпорация Майкрософт в настоящее время не рекомендует создавать и использовать веб-приложения Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/) для создания бизнес-решений без кода для Интернета и мобильных устройств. 
  
## <a name="setting"></a>Параметр

Действие **RequeryRecords имеет** следующие аргументы. 
  
|**Параметр**|**Обязательный**|**Описание**|
|:-----|:-----|:-----|
|**Where=** <br/> |Нет  <br/> |Пункт SQL WHERE, ограничивающий записи в представлении. По умолчанию этот аргумент является пустым.  <br/> |
|**OrderBy** <br/> |Нет  <br/> |Строковая фраза, включаемая имя поля или полей для сортировки записей, а также необязательные ключевые слова ASC или DESC. По умолчанию этот аргумент является пустым.  <br/> |
   
## <a name="remarks"></a>Примечания

Любая сортировка или фильтрация, применяемая пользователем, очищается при наложении действия **RequeryRecords.** 
  
Аргумент  *OrderBy*  — это имя поля или полей, на которых необходимо сортировать записи. При использовании более чем одного имени поля разделять имена с запятой (,). 
  
При наборе  *аргумента OrderBy*  записи сортироваться по умолчанию в порядке восходящей. 
  
Чтобы сортировать записи в порядке убывления, введите DESC в конце выражения *аргумента OrderBy.* Например, чтобы сортировать записи клиентов в порядке убывания по имени контакта, установите аргумент  *OrderBy*  на "ContactName DESC". 
  
Чтобы сортировать имена по убыванию LastName и по возрастанию FirstName, установите аргумент  *OrderBy*  на "LastName DESC, FirstName ASC".
