# <a name="setproperty-macro-action-access-custom-web-app"></a>SetProperty Macro Action (Доступ к настраиваемой веб-приложению)

Действие **SetProperty можно использовать** для набора свойства для управления представлением. 
  
> [!IMPORTANT]
> Корпорация Майкрософт в настоящее время не рекомендует создавать и использовать веб-приложения Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/) для создания бизнес-решений без кода для Интернета и мобильных устройств. 
  
## <a name="setting"></a>Параметр

Действие **SetProperty** имеет аргументы, перечисленные в следующей таблице. 
  
|**Аргумент макрокоманды**|**Описание**|
|:-----|:-----|
| _Имя элемента управления_ <br/> |Введите имя поля или управления, для которого необходимо установить значение свойства. Оставьте этот аргумент пустым, чтобы установить свойство для представления.  <br/> |
| _Property_ <br/> |Выберите свойство, которое необходимо установить. Список  свойств, которые можно установить с помощью этого действия, см. в разделе Замечания в этой статье.  <br/> |
| _Значение_ <br/> |Введите значение, за которое должно быть установлено свойство. Для свойств, значения которых либо да, либо нет, используйте **-1** для да и **0** для No.  <br/> |
   
## <a name="remarks"></a>Примечания

Вы можете использовать **действие SetProperty** для набора следующих свойств управления: 
  
- Подпись
    
- Включено
    
- ForeColor
    
- Значение
    
- Visible
    
Если вы вводите недействительное значение для аргумента ** *Value* ** , ошибки не возникает, но Access может изменить свойство на другое, в зависимости от того, как он интерпретирует аргумент.
