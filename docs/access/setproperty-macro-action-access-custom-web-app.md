# <a name="setproperty-macro-action-access-custom-web-app"></a>SetProperty Macro Action (пользовательское веб-приложение Access)

Вы можете использовать **действие SetProperty,** чтобы установить свойство для управления в представлении. 
  
> [!IMPORTANT]
> Корпорация Майкрософт в настоящее время не рекомендует создавать и использовать веб-приложения Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/) для создания бизнес-решений без кода для Интернета и мобильных устройств. 
  
## <a name="setting"></a>Параметр

Действие **SetProperty** имеет аргументы, перечисленные в следующей таблице. 
  
|**Аргумент макрокоманды**|**Описание**|
|:-----|:-----|
| _Имя элемента управления_ <br/> |Введите имя поля или управления, для которого необходимо установить значение свойства. Оставьте этот аргумент пустым, чтобы установить свойство для представления.  <br/> |
| _Property_ <br/> |Выберите свойство, которое нужно установить. Список **свойств,** которые можно настроить с помощью этого действия, см. в разделе "Примечаний" этой статьи.  <br/> |
| _Значение_ <br/> |Введите значение, которое должно быть установлено для свойства. Для свойств со значениями "Да" или "Нет" используйте **-1** для "Да" и **0** для "Нет".  <br/> |
   
## <a name="remarks"></a>Примечания

С помощью действия **SetProperty можно** настроить следующие свойства управления: 
  
- Подпись
    
- Включено
    
- ForeColor
    
- Значение
    
- Visible
    
Если ввести недопустимое значение для аргумента ** *Value* **, ошибка не возникает, но Access может изменить свойство на другое значение в зависимости от того, как он интерпретирует аргумент.
