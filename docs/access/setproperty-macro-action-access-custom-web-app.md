# <a name="setproperty-macro-action-access-custom-web-app"></a>Макрокоманда SetProperty (пользовательское веб-приложение для Access)

Макрокоманду **SetProperty** можно использовать для задания свойства элемента управления в представлении. 
  
> [!IMPORTANT]
> Корпорация Майкрософт в настоящее время не рекомендует создавать и использовать веб-приложения Access в SharePoint. В качестве альтернативы можно использовать [Microsoft PowerApps](https://powerapps.microsoft.com/en-us/) для создания бизнес-решений без кода для Интернета и мобильных устройств. 
  
## <a name="setting"></a>Параметр

Макрокоманда **SetProperty** содержит аргументы, приведенные в следующей таблице. 
  
|**Аргумент макрокоманды**|**Описание**|
|:-----|:-----|
| _Имя элемента управления_ <br/> |Введите имя поля или элемента управления, для которого требуется задать значение свойства. Оставьте этот аргумент пустым, чтобы задать свойство для представления.  <br/> |
| _Свойство_ <br/> |Выберите свойство, которое требуется задать. Список свойств, которые можно задать с помощью этого действия, представлен в разделе **"Примечания** " этой статьи.  <br/> |
| _Значение_ <br/> |Введите значение, которое нужно присвоить свойству. Для свойств, значения которых имеют значение "Да" или "нет", используйте значение **– 1** для "Да" и **0** для "нет".  <br/> |
   
## <a name="remarks"></a>Примечания

Макрокоманду **SetProperty** можно использовать для задания следующих свойств элемента управления: 
  
- Подпись
    
- Включено
    
- ForeColor
    
- Значение
    
- Visible
    
Если вы введете недопустимое значение для аргумента * * *value* * * * * * * * * * * * * * *, то ошибка не возникнет, но Access может изменить значение свойства на другое, в зависимости от того, как он интерпретирует аргумент.
