# <a name="application-interface-onenote"></a>Интерфейс Application (OneNote)

Интерфейс **приложения** включает в себя методы Получение справки, редактирования и обновления данных OneNote и контента. Методы, в четыре общие категории: 
  
- **Структура записная книжка** &ndash; Методы для работы со структурой записной книжки, в том числе для обнаружения, открытие, изменение, закрытие и удаление записные книжки, группы разделов и разделы. 
    
- **Контент страницы** &ndash; Методы для работы со страницы и страницы содержимого, в том числе для обнаружения, изменение, сохранение и удаление содержимого страницы. Контент страницы включает в себя двоичные объекты, такие как рукописного ввода и изображения, а текстовые объекты, такие как файлов в режиме структуры. 
    
- **Навигация** &ndash; Методы для поиска, создание ссылок на и переход к страницам и объекты. 
    
- **Функциональные** &ndash; Все другие методы, которые выполнения определенных действий или задать параметры в OneNote. 
    
Кроме того интерфейс **приложения** включает в себя ряд *свойств* и *событий* . 
  
## <a name="notebook-structure-methods"></a>Методы структур записная книжка
<a name="ON14DevRef_Application_NotebookStructure"> </a>

Методы, описанные в этом разделе позволяют обнаружения, открытие, изменение, закройте и удаление записных книжек OneNote, группы разделов и разделы.
  
### <a name="gethierarchy-method"></a>Метод GetHierarchy

|||
|:-----|:-----|
|**Описание** <br/> |Получает записной книжки узел структуру иерархии, начиная с узла указан (всех записных книжек или одной записной книжки, раздел группы или раздел) и расширение вниз на всех потомков указан на уровне.  <br/> |
|**Синтаксис** <br/> | `HRESULT GetHierarchy(`<br/>`[in]BSTR bstrStartNodeID,`<br/>`[in]HierarchyScope hsScope,`<br/>`[out]BSTR * pbstrHierarchyXmlOut,`<br/>`[in,defaultvalue(xs2013)]XMLSchema xsSchema);` <br/> |
|**Параметры** <br/> | _bstrStartNodeID_ &ndash; Узел (записной книжки, группы или раздела) которого вы хотите потомков. Если передать пустую строку ("»), метод возвращает все узлы ниже корневого узла (то есть, все записные книжки, группы разделов и разделы). При указании записной книжки, группе или узел раздела, метод возвращает только потомки этого узла.  <br/><br/>_hsScope_ &ndash; Наименьший уровень дочернего узла требуется. Например при указании страниц, метод возвращает все узлы как можно ближе как уровне страницы. При указании разделов, метод возвращает только узлы раздел под записной книжки. Дополнительные сведения см в перечислении **HierarchyScope** в разделе [перечисления](enumerations-onenote-developer-reference.md#odc_HierarchyScope) .  <br/><br/>_pbstrHierarchyXmlOut_ &ndash; (Выходной параметр) A указатель на строку, в которой вы не нужно записывать выходные данные XML.  <br/><br/>_xsSchema_ &ndash; (Необязательно) версии OneNote XML-схему, тип **XMLSchema**, нужно быть выходные данные. Можно указать, следует ли XML-схему версии 2013, 2010, 2007, или текущей версии.  <br/><br/>**Примечание**: рекомендуется использовать указание версии OneNote (например, **xs2013**), а не с помощью **xsCurrent** или оставить пустым, так как это позволит надстройки для работы с будущими версиями OneNote.           |
   
Метод GetHierarchy возвращает строку в формате OneNote 2013 XML по умолчанию, либо можно задать основной версии схемы XML с помощью параметра необязательно _xsSchema_ . 
  
В зависимости от параметров, передаваемого метод **GetHierarchy** может возвращать списки (например, всех записных книжек, все разделы в всех записных книжек, все страницы в данном разделе или всех страниц в пределах указанной записной книжки). Для каждого узла XML строки, возвращаемой предоставляет свойства (например, заголовок раздела или страницы, идентификатор и времени последнего изменения). 
  
Не все сочетания начальный узел и области являются допустимыми. Например при указании раздел запуска узел и область записной книжки, **GetHierarchy** возвращает значение null, так как записной книжке в иерархии узлов выше раздел. 
  
В следующем примере C# показано, как использовать метод **GetHierarchy** для получения всей иерархии OneNote, включая все записные книжки, до уровня страницы. Строка выходные данные копируются в буфер обмена, откуда можно вставить строку в текстовый редактор для ознакомления. 
  
```cs
static void GetEntireHierarchy()
    {
        String strXML;
        OneNote.Application onApplication = new OneNote.Application();
        onApplication.GetHierarchy(null, 
            OneNote.HierarchyScope.hsPages, out strXML);
        Clipboard.SetText(strXML);
        MessageBox.Show("The XML has been copied to the clipboard");
    }

```

### <a name="updatehierarchy-method"></a>Метод UpdateHierarchy

|||
|:-----|:-----|
|**Описание**|Изменяет или обновляет иерархии записных книжек. Например можно добавить разделы и группы разделов в записной книжке Добавление новой записной книжки, Перемещение разделов в записной книжке, измените имя раздела, добавьте в раздел страницы или изменить порядок страниц в разделах.|
|**Синтаксис**| `HRESULT UpdateHierarchy(`<br/>`[in]BSTR bstrChangesXmlIn,`<br/>`[in,defaultvalue(xsCurrent)] XMLSchema xsSchema);`|
|**Параметры**| _bstrChangesXmlIn_ &ndash; Строка, содержащая код OneNote XML, указывающее, чтобы сделать изменения иерархии. Например если вы хотите добавить новый раздел, можно добавить элемент **раздела** в XML-строку, чтобы указать, где будет новый раздел для добавления. Кроме того Если вы хотите изменить имя существующего раздела, можно прежнего идентификатора раздела и измените его атрибут **имени** в XML-код.<br/><br/>_xsSchema_ &ndash; The OneNote (необязательно) версия схемы строка _bstrChangesXmln_. Это необязательный значение используется для указания версии OneNote XML-схему, _bstrChangesXmlIn_ строки. Если это значение не указано, OneNote будет предполагать, что XML-код находится в версии схемы _xsCurrent_. <br/><br/>**Примечание**: рекомендуется использовать указание версии OneNote (например, **xs2013**), а не с помощью **xsCurrent** или оставить пустым, так как это позволит надстройки для работы с будущими версиями OneNote.           |
   
Если передать только частичный строку OneNote XML для параметра _bstrChangesXmlIn_ OneNote пытается определить необходимые изменения. Например если указан элемент **записной книжки** , который содержит только один раздел OneNote добавляет разделе после любые существующие разделы. Тем не менее если операция указываемые является неоднозначным, результат может быть трудно определить. Например если существующие записная книжка содержит четыре раздела и XML-строка, передаваемого включает записной книжки и только четвертый и первый раздел (в таком порядке), OneNote поместить второго и третьего разделов до четвертый раздел или после первого раздела . 
  
Метод **UpdateHierarchy** нельзя использовать для удаления части записной книжки. То есть передав XML-строка, которая включает в себя только часть из существующей иерархии не удаляет разделы, которые не включены в строке. Чтобы удалить частью иерархии, используйте метод **DeleteHierarchy** . 
  
Следующий код C# показано одним из способов использования метода **UpdateHierarchy** для изменения иерархии OneNote с помощью изменения имени существующий раздел. Считывает XML-код из примера файла с именем ChangeSectionName.xml в корне диска C, загружает в XML-документ и затем передает XML структуры документа в метод. 
  
```cs
static void UpdateExistingHierarchy()
    {
        OneNote.Application onApplication = new OneNote.Application();
        
        // Get the XML from the file.
        XmlTextReader reader = new XmlTextReader("C:\\ChangeSectionName.xml");
        reader.WhitespaceHandling = WhitespaceHandling.None;
        XmlDocument xmlDocIn = new XmlDocument();
        xmlDocIn.Load(reader);
        
        // Update the hierarchy.
        onApplication.UpdateHierarchy(xmlDocIn.OuterXml,
        OneNote.XMLSchema.xs2007);   
    }

```

Следующий XML-код приведен фрагмент файла ChangeSectionName.xml, который передает предыдущего кода C# метод. Когда XML-код передается методу **UpdateHierarchy** , он изменяется имя одного из разделов в существующую иерархию (, изменив значение атрибута **name** «Мои переименовать раздел»). Затем удаляет все разделы за исключением того, имя которого был изменен. Кроме того код удаляет ненужные атрибуты из целевого **раздел** элемента, включая атрибуты **lastModifiedTime**, **isCurrentlyViewed**и **цвет** , оставив только **имя**, **идентификатор**и ** путь** атрибуты без изменений. 
  
```XML
<?xml version="1.0" ?> 
    <one:Notebooks xmlns:one="http://schemas.microsoft.com/office/onenote/12/2004/onenote"> 
        <one:Notebook name="My Notebook" nickname="My Notebook" ID="{0B8E7305-AC2C-4BCB-8651-1CDA55AAE14C}{1}{B0}"> 
            <one:Section name="My Renamed Section" ID="{5F4E2908-44BA-4C02-91FE-49FC665E9A33}{1}{B0}" path="C:\My Section.one" /> 
        </one:Notebook> 
    </one:Notebooks>
```

С помощью кода, показано в примере для метода **GetHierarchy** , который изменяется, как показано ниже, для ограничения области к разделам был получен приведенный выше XML-код. 
  
```cs
static void GetAllSections()
    {
        String strXML;
        OneNote.Application onApplication = new OneNote.Application();
        onApplication.GetHierarchy(System.String.Empty, 
            OneNote.HierarchyScope.hsSections, out strXML);
        Clipboard.SetText(strXML.ToString());
        MessageBox.Show("The XML has been copied to the Clipboard");
    }

```

В следующем примере C# показано полное консольное приложение, которое выполняет поиск раздела с именем " `Sample_Section`«, запрос на ввод нового имени для того раздела, а затем использует метод **UpdateHierarchy** Чтобы изменить имя раздела имя, которое пользователь типизированный. Перед выполнением кода, измените " `Sample_Section`" к имени раздел, в котором существует в иерархии OneNote.
  
```cs
    static void Main(string[] args)
    {
        
        // OneNote 2013 Schema namespace.
        string strNamespace = "http://schemas.microsoft.com/office/onenote/2013/onenote";
        string outputXML;
        Application onApplication = new Application();
        onApplication.GetHierarchy(null, HierarchyScope.hsSections, out outputXML);
        // Load a new XmlDocument.
        XmlDocument xmlDoc = new XmlDocument();
        xmlDoc.LoadXml(outputXML);
        XmlNamespaceManager nsmgr = new XmlNamespaceManager(xmlDoc.NameTable);
            nsmgr.AddNamespace("one", strNamespace);
        // Search for the section named "Sample_Section".
        XmlNode xmlNode = xmlDoc.SelectSingleNode("//one:Section[@name='Sample_Section']", nsmgr);
        // Prompt for a new section title.
        System.Console.Write("Please enter a new title for the section: ");
        string input = System.Console.ReadLine();
        xmlNode.Attributes["name"].Value = input; 
        // Update the section with the new title.
        onApp.UpdateHierarchy(xmlNode.OuterXml);
        System.Console.Write("Done!\n");
    }

```

### <a name="openhierarchy-method"></a>Метод OpenHierarchy

|||
|:-----|:-----|
|**Описание** <br/> |Откроется раздел группы или раздела, который указан.  <br/> |
|**Синтаксис** <br/> | `HRESULT OpenHierarchy(`<br/>`[in]BSTR bstrPath,`<br/>`[in]BSTR bstrRelativeToObjectID,`<br/>`[out]BSTR * pbstrObjectID,`<br/>`[in,defaultvalue(cftNone)]CreateFileType cftIfNotExist);` <br/> |
|**Параметры** <br/> | _bstrPath_ &ndash; Пути, который необходимо открыть. Записной книжке или группы разделов в записной книжке _bstrPath_ может быть путь к папке или путь к файлу раздела .one. Если указать путь к файлу раздела .one необходимо включить расширение .one в строке путь к файлу.  <br/><br/>_bstrRelativeToObjectID_ &ndash; OneNote идентификатор родительского объекта, с которой будет выполняться новый объект, чтобы открыть (записная книжка или раздел группы). Если параметр _bstrPath_ абсолютный путь, вы можете передать пустую строку ("») для _bstrRelativeToObjectID_. Кроме того вы можете передать идентификатор объекта группы записная книжка или раздел, который должен содержать объект (раздел или группу разделов), который вы хотите создать, а затем укажите имя файла (например, section1.one) объекта, который вы хотите создать в группе родительский объект.  <br/><br/>_pbstrObjectID_ &ndash; (Выходной параметр), которое возвращает OneNote для записной книжки, группы или раздела, метод **OpenHierarchy** открывает идентификатор объекта. Этот параметр является указатель на строку, в которую необходимо метода для записи с идентификатором.  <br/><br/>_cftlfNotExist_ &ndash; Перечислимые значение из перечисления [CreateFileType](enumerations-onenote-developer-reference.md#odc_CreateFileType) (не обязательно). При передаче значения для _cftIfNotExist_метод **OpenHierarchy** создает группу разделов или раздел файла по указанному пути только в том случае, если файл не существует.  <br/> |
   
При указании группы раздела, который не является в open записной книжки, метод **OpenHierarchy** открывает группу разделов в записной книжке. При указании раздела, который не является в open записной книжки, метод **OpenHierarchy** открывает разделе в группе раздел последних открытых разделах. При указании группы раздела или раздела, который уже присутствует в open записной книжки, ничего не происходит, так как группа разделов или раздел уже открыт, а также. В любом случае **OpenHierarchy** возвращает идентификатор объекта для группы разделов, записной книжки или раздела, который требуется указать, которые можно использовать в других операций. 
  
Можно также использовать метод **OpenHierarchy** для создания новых раздела, вместо этого путем импорта XML. 
  
Приведенный ниже код показано, как использовать метод **OpenHierarchy** для откройте раздел собраний в записной книжке работы и получить идентификатор для раздела. Раздел еще не существует, OneNote создает в указанном вами месте. 
  
```cs
static void OpenSection()
    {
        String strID;
        OneNote.Application onApplication = new OneNote.Application();
        onApplication.OpenHierarchy("C:\\Documents and Settings\\user\\My Documents\\OneNote Notebooks\\Work\\Meetings.one", 
        System.String.Empty, out strID, OneNote.CreateFileType.cftSection);
    }

```

### <a name="closenotebook-method"></a>Метод CloseNotebook

|||
|:-----|:-----|
|**Описание** <br/> |Закрывает указанного записной книжки.  <br/> |
|**Синтаксис** <br/> | `HRESULT CloseNotebook(`<br/>`[in]BSTR bstrNotebookID,`<br/>`[in,defaultvalue(false)]VARIANT_BOOL force);` <br/> |
|**Параметры** <br/> | _bstrNotebookID_ &ndash; OneNote идентификатор записной книжки, которую нужно закрыть.  <br/><br/>_Принудительное_ &ndash; (Необязательно) **значение true** для закрытия записной книжки, даже в том случае, если есть изменения в записной книжке, который не удается синхронизировать перед закрытием; в противном случае — **значение false** (значение по умолчанию).  <br/> |
   
Чтобы закрыть записной книжки, указываемые можно использовать метод **CloseNotebook** . ### <a name="getpagecontent-method"></a>Метод GetPageContent

Чтобы получить двоичных объектов, вызовите метод **GetBinaryPageContent** и передайте ему идентификатор OneNote, вы получаете из метода **GetPageContent** . Метод **GetPageContent** используется в том случае, когда нет необходимости сразу же двоичных данных. 
  
### <a name="updatepagecontent-method"></a>Метод UpdatePageContent

|||
|:-----|:-----|
|**Описание**|Обновление или изменение содержимого на странице.|
|**Синтаксис**| `HRESULT UpdatePageContent(`<br/>`[in]BSTR bstrPageChangesXmlIn,`<br/>`[in,defaultvalue(0)]DATE dateExpectedLastModified,`<br/>`[in,defaultvalue(xsCurrent)]XMLSchema xsSchema,`<br/>`[in,defaultvalue(false)]VARIANT_BOOL force);`|
|**Параметры**| _bstrPageChangesXmlIn_ &ndash; Строка, содержащая код OneNote XML, которая включает в себя изменения, который требуется сделать на страницу.<br/><br/>_dateExpectedLastModified_ &ndash; (Необязательно) Дата и время, которые вы думаете, страницы, которую требуется обновить Дата и время последнего изменения. Если передать ненулевое значение для этого параметра, OneNote продолжается с пакетом обновления, только в том случае, если значение, передаваемого соответствует фактическая дата и время последнего изменения страницы. Передача значения для этого параметра предотвращает случайной перезаписи изменения пользователей, с момента последнего изменения страницы.<br/><br/>_xsSchema_ &ndash; (Необязательно) версии OneNote XML-схему, тип **XMLSchema**, нужно быть выходные данные. Можно указать, следует ли XML схемы версии 2013, 2010, 2007, или текущей версии. <br/><br/>**Примечание**: рекомендуется использовать указание версии OneNote (например, **xs2013**), а не с помощью **xsCurrent** или оставить пустым, так как это позволит надстройки для работы с будущими версиями OneNote.<br/><br/>_Принудительное_ (Необязательно) **значение true,** чтобы обновить контент страницы, даже при наличии неизвестные данные на странице в будущих версиях OneNote; в противном случае — **значение false** (значение по умолчанию). |
   
Этот метод можно использовать для изменения страницы разными способами. Например можно использовать метод **UpdatePageContent** Добавление структуры на страницу, изменять содержимое структуры, добавление изображений, добавлять рукописного ввода, Перемещение содержимого или изменение текста в файлов в режиме структуры. 
  
В качестве примера более конкретные может использование метода **GetPageContent** для экспорта существующей страницы, внести изменения в XML-коде для страницы и затем использовать метод **UpdatePageContent** попытку импорта всей страницы. Или можно использовать этот метод для добавления новых объектов страницы, такие как изображения, в нижней части существующей страницы. 
  
Только объекты, которые необходимо включить в XML-код, передайте методу **UpdatePageContent** являются объектов на уровне страницы (например, файлов в режиме структуры, изображений на странице или рукописного ввода на странице ""), которые были изменены. Этот метод не изменять или удалять объекты уровня страницы, которые не указан в параметре _bstrPageChangesXmlIn_ . Метод полностью заменяет объектов на уровне страницы, например файлов в режиме структуры, чьи идентификаторы совпадают с объектами, передаваемого. Следовательно необходимо полностью указать все объекты уровня страницы в коде, включая их существующего контента и изменения, которые необходимо внести в них. 
  
К примеру Если страница содержит структуру и фонового изображения страницы, можно замените структуры и оставьте без изменений, полностью указание структуры в XML-код, с помощью идентификатора существующей структуры и без учета изображения в коде изображение. Так как оптимизированная структуры, включаемых в коде полностью заменяет существующий структуры, необходимо включить все содержимое структуры.
  
Кроме того метод **UpdatePageContent** изменяет только свойства элемента, указанные в параметре _bstrPageChangesXmlIn_ . Например при указании некоторые, но не все свойства элемента **PageSettings** свойства, которые не указан не изменяются. 
  
Следующем примере показано, как использовать метод **UpdatePageContent** для изменения заголовка страницы и добавьте некоторые образец текста на страницу. Перед выполнением кода, замените допустимый код идентификатор страницы, показано в примере кода, чтобы код работает на вашем компьютере. Вы можете получить идентификатор страницы для страницы, с помощью метода **GetHierarchy** и анализ результатов. 
  
```cs
static void UpdatePageContent()
    {
        OneNote.Application onApplication = new OneNote.Application();
        String strImportXML;
        strImportXML = "<?xml version=\"1.0\"?>" +
            "<one:Page xmlns:one=\"http://schemas.microsoft.com/office/onenote/12/2004/onenote\" 
            ID=\"{3428B7BB-EF39-4B9C-A167-3FAE20630C37}{1}{B0}\">" +
            "    <one:PageSettings RTL=\"false\" color=\"automatic\">" +
            "        <one:PageSize>" +
            "            <one:Automatic/>" +
            "        </one:PageSize>" +
            "        <one:RuleLines visible=\"false\"/>" +
            "    </one:PageSettings>" +
            "    <one:Title style=\"font-family:Calibri;
                 font-size:17.0pt\" lang=\"en-US\">" +
            "        <one:OE alignment=\"left\">" +
            "            <one:T>" +
            "                <![CDATA[My Sample Page]]>" +
            "            </one:T>" +
            "        </one:OE>" +
            "    </one:Title>" +
            "    <one:Outline >" +
            "        <one:Position x=\"120\" y=\"160\"/>" +
            "        <one:Size width=\"120\" height=\"15\"/>" +
            "        <one:OEChildren>" +
            "            <one:OE alignment=\"left\">" +
            "                <one:T>" +
            "                    <![CDATA[Sample Text]]>" +
            "                </one:T>" +
            "            </one:OE>" +
            "        </one:OEChildren>" +
            "    </one:Outline>" +
            "</one:Page>";
        // Update the page content.
        onApplication.UpdatePageContent(strImportXML, System.DateTime.MinValue);
    }

```

### <a name="findpages-method"></a>Метод FindPages

|||
|:-----|:-----|
|**Описание**|Возвращает список страниц, соответствующие указанному запросу терминов.|
|**Синтаксис**| `HRESULT FindPages(`<br/>`[in]BSTR bstrStartNodeID,`<br/>`[in]BSTR bstrSearchBSTR,`<br/>`[out]BSTR * pbstrHierarchyXmlOut,`<br/>`[in,defaultvalue(#)]VARIANT_BOOL fIncludeUnindexedPages,`<br/>`[in,defaultvalue(0)]VARIANT_BOOL fDisplay,`<br/>`[in,defaultvalue(#)]XMLSchema xsSchema);`|
|**Параметры**| _bstrStartNodeID_ &ndash; Узел (корневой, записной книжки, раздел группы или раздела) ниже для поиска содержимого. Этот параметр задает область поиска.<br/><br/>_bstrSearchString_ &ndash; Строке поиска. Передайте именно то же, следует ввести в поле «Поиск» в пользовательском Интерфейсе OneNote. Можно использовать битовые операторы, такие как **и** и **или**, должна быть прописными буквами.<br/><br/>_pbstrHierarchyXmlOut_ &ndash; (Выходной параметр) A указатель на строку, в которую необходимо OneNote для записи выходного XML-строка.  **FindPages** работает только в том случае, если у вас есть Microsoft Search 3.0 или 4.0, установленный на компьютере. ОС Windows Vista и Windows 7 включают этот компонент.
