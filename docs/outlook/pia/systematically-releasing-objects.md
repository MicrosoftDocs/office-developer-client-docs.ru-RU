<span data-ttu-id="5047b-106">Начиная с Outlook 2010, Outlook по умолчанию не сообщает надстройкам о завершении работы.</span><span class="sxs-lookup"><span data-stu-id="5047b-106">Starting in Outlook 2010, Outlook, by default, does not signal add-ins that it is shutting down.</span></span> <span data-ttu-id="5047b-107">В частности, в приложении Outlook в процессе быстрого завершения работы больше не вызываются методы **OnBeginShutdown(Array)** и **OnDisconnection(ext\_DisconnectMode, Array)** интерфейса **IDTExtensibility2**.</span><span class="sxs-lookup"><span data-stu-id="5047b-107">Specifically, Outlook no longer calls the **OnBeginShutdown(Array)** and **OnDisconnection(ext\_DisconnectMode, Array)** methods of the **IDTExtensibility2** interface during fast shutdown.</span></span> <span data-ttu-id="5047b-108">Также в надстройках Outlook, написанных с помощью инструментов разработчика Office в Visual Studio 2010 или более поздней версии, при завершении работы приложения Outlook больше не вызывается метод ThisAddin\_Shutdown.</span><span class="sxs-lookup"><span data-stu-id="5047b-108">Similarly, an Outlook add-in, written with Office development tools in Visual Studio 2010 or a later version, no longer calls the ThisAddin\_Shutdown method when Outlook is shutting down.</span></span>

<span data-ttu-id="5047b-112">Разработчикам настоятельно рекомендуется соблюдать следующие рекомендации по завершению работы надстроек. - <span data-ttu-id="5047b-114">Разработчикам надстроек следует выполнять только те задачи, которые абсолютно необходимы при завершении работы.</span><span class="sxs-lookup"><span data-stu-id="5047b-114">Add-in developers should only perform tasks that absolutely must take place during shutdown.</span></span>
