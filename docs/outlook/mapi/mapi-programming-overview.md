# <a name="mapi-programming-overview"></a>Общие сведения о программировании MAPI

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Эта Справка по ИНТЕРФЕЙСу API Microsoft Outlook (MAPI) предназначена для разработчиков C и C++ с различными потребностями и опытом. Для тех разработчиков, которые хотят использовать MAPI для расширения своих приложений, у которых есть функции обмена сообщениями, не требуется никаких сведений о предварительных требованиях. Вам нужен фоновый рисунок в обмене сообщениями и модель компонентных объектов (COM), чтобы использовать MAPI для создания крупномасштабных приложений рабочей группы или драйверов для специализированных служб системы обмена сообщениями.
  
Перед началом разработки следует рассмотреть следующие сведения о том, как использовать MAPI, процесс входа в систему и как создаются и настраиваются профили и службы сообщений.
  
Интерфейс MAPI (MAPI) — это обширный набор функций, которые разработчики могут использовать для создания приложений с поддержкой почты. Библиотека функций целиком называется MAPI. MAPI обеспечивает полный контроль над системой обмена сообщениями на клиентском компьютере, создание и управление сообщениями, управление почтовым ящиком клиента, поставщиками услуг и т. д.
  
> [!NOTE]
> Расширенный MAPI такой же, как и MAPI, и просто называется MAPI в документации по MAPI. 
  
 **Simple MAPI**
  
Simple MAPI предоставляет набор функций, позволяющих добавлять основной уровень функции обмена сообщениями в приложения Microsoft Windows.
  
> [!IMPORTANT]
> Функция Simple MAPI MAPISendMail поддерживается в Microsoft Outlook 2013 и Microsoft Outlook 2010. Другие простые функции MAPI не поддерживаются в Windows.
