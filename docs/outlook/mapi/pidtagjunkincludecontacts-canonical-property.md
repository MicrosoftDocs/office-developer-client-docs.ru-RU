# <a name="pidtagjunkincludecontacts-canonical-property"></a>Каноническое свойство PidTagJunkIncludeContacts

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Указывает, обрабатываются ли адреса электронной почты контактов в папке Contacts специально в отношении фильтра нежелательной почты.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_JUNK_INCLUDE_CONTACTS  <br/> |
|Идентификатор:  <br/> |0x6100  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Спам  <br/> |
   
## <a name="remarks"></a>Примечания

Если установлено "0x00000001", эти адреса электронной почты должны заполнять "доверенный" адрес электронной почты в части ограничения правил нежелательной почты, чтобы почта с этих адресов относилась к "не нежелательной". Если установлено "0x00000000", адреса электронной почты из папки "Контакты" не должны быть добавлены в правило нежелательной почты, а раздел правила должен быть NULL.
  
Если это свойство имеет значение "0x00000001" и если у добавленного контакта есть адреса электронной почты, которые еще не включены в раздел доверенных контактов правила нежелательной электронной почты, эти адреса электронной почты должны быть добавлены в ограничение. Если это свойство "0x00000000", никаких действий не требуется.
