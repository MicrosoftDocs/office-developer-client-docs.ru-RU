# <a name="download-hierarchy-state"></a>Состояние загрузки иерархии

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время загрузки состояния иерархии для конечного автомата репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояния:  <br/> |**ЛР_СИНК_ДОВНЛОАД_ХИЕРАРЧИ** <br/> |
|Связанная структура данных:  <br/> |**[DNHIER](dnhier.md)** <br/> |
|Из этого состояния:  <br/> |[Состояние синхронизации](synchronize-state.md) <br/> |
|В это состояние:  <br/> |Состояние синхронизации  <br/> |
   
> [!NOTE]
> Конечный автомат репликации — это детерминированный конечный автомат. Клиент, который отменяет от одного состояния к другому, должен, в конечном итоге, вернуться к предыдущему из последнего. 
  
## <a name="description"></a>Описание

Это состояние инициирует загрузку иерархической структуры папок с сервера в локальное хранилище. 
  
Outlook инициализирует связанную структуру данных **днхиер** с помощью указателя на иерархию. Клиент загружает иерархию и вставляет новые папки или изменения в папки в локальном хранилище. В процессе загрузки выполняется добавочная синхронизация изменений Microsoft Exchange (ICS). Дополнительные сведения об ICS см. в статье [Критерии оценки ICS](https://msdn.microsoft.com/library/aa579252%28EXCHG.80%29.aspx).
  
При завершении этого состояния локальное хранилище возвращается в состояние Synchronize.
