# <a name="download-hierarchy-state"></a>Состояние иерархии загрузки

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время состояния иерархии загрузки машины состояния репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояния:  <br/> |**LR_SYNC_DOWNLOAD_HIERARCHY** <br/> |
|Связанная структура данных:  <br/> |**[DNHIER](dnhier.md)** <br/> |
|Из этого состояния:  <br/> |[Состояние синхронизации](synchronize-state.md) <br/> |
|Для этого состояния:  <br/> |Состояние синхронизации  <br/> |
   
> [!NOTE]
> Машина состояния репликации — это детерминистический автомат состояния. Клиент, отправляющийся из одного состояния в другое, должен в конечном итоге вернуться в первое из последнего. 
  
## <a name="description"></a>Описание

Это состояние инициирует загрузку иерархии папок дерева с сервера в локальный магазин. 
  
Outlook инициализирует связанную структуру **данных DNHIER** с указателем на иерархию. Клиент загружает иерархию и вставляет новые папки или изменения в папки в локальном магазине. Процесс загрузки принимает microsoft Exchange синхронизации изменений (ICS). Дополнительные сведения об ICS см. в статье [Критерии оценки ICS](https://msdn.microsoft.com/library/aa579252%28EXCHG.80%29.aspx).
  
После окончания этого состояния локальный магазин возвращается в состояние синхронизации.
