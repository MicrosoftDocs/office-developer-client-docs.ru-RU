# <a name="implementing-security"></a>Реализация безопасности

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Если система обмена сообщениями требует этого, поставщик транспорта отвечает за реализацию соответствующего уровня безопасности для доступа к системе обмена сообщениями. Каждое входящие или исходяющие сообщения, отправленные через поставщика транспорта с помощью пула MAPI обрабатывается в контексте сеанса для работы с поставщиком. Поставщик транспорта может отобразить диалоговое окно входа для пользователя, которое запросит учетные данные пользователя, прежде чем устанавливать такое подключение. Кроме того, поставщик транспорта может хранить ранее введенные учетные данные пользователя в защищенном диапазоне свойств в разделе профиля и использовать их для доступа без запроса.
  
При реализации безопасности поставщика транспорта учитывайте следующее:
  
- При установке нескольких поставщиков служб с пользователем может быть связано множество имен и паролей.
    
- MAPI позволяет использовать несколько сеансов с несколькими удостоверениями. Поставщикам рекомендуется поддерживать несколько сеансов, но это не требуется.
    
- Каждый сеанс с поставщиком транспорта связан mapI с дискретным разделом в профиле пользователя. Поставщик транспорта может использовать метод [IMAPISupport::OpenProfileSection](imapisupport-openprofilesection.md) для получения доступа к этому разделу, который можно использовать для хранения любой информации, связанной с этим сеансом, включая учетные данные. 
    
- Если установлено несколько поставщиков транспорта, не обязательно, чтобы у пользователя был только один адрес электронной почты. У пользователя может быть отдельный адрес электронной почты для каждого установленного поставщика транспорта или отдельный адрес для каждого сеанса для одного поставщика.
    
Дополнительные сведения о хранении учетных данных в разделах профиля см. в разделах ["Службы сообщений и](message-services-and-profiles.md) профили и [IProfSect : IMAPIProp"](iprofsectimapiprop.md).
