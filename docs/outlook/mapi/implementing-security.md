# <a name="implementing-security"></a>Реализация безопасности

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Если система обмена сообщениями требует этого, поставщик транспорта отвечает за реализацию соответствующего уровня безопасности для доступа к системе обмена сообщениями. Каждое входящий или исходящая сообщение, отправленное через поставщика транспорта пулером MAPI, обрабатывается в контексте сеанса логотипа поставщика. Поставщик транспорта может отобразить диалоговое окно с логотипом пользователю, который запросит учетные данные пользователя перед установлением такого подключения. Кроме того, поставщик транспорта может хранить ранее вписаные учетные данные пользователя в диапазоне безопасных свойств в разделе профилей и использовать их для доступа без запроса.
  
При реализации безопасности поставщика транспорта рассмотрите следующие вопросы:
  
- С несколькими установленными поставщиками услуг может быть множество имен и паролей, связанных с пользователем.
    
- MAPI позволяет несколько сеансов с несколькими удостоверениями. Поставщикам рекомендуется поддерживать несколько сеансов, но от них не требуется.
    
- Каждый сеанс с поставщиком транспорта связан MAPI с дискретным разделом в профиле пользователя. Поставщик транспорта может использовать метод [IMAPISupport::OpenProfileSection](imapisupport-openprofilesection.md) для получения доступа к этому разделу, который можно использовать для хранения любых сведений, связанных с этим сеансом, включая учетные данные. 
    
- С несколькими установленными поставщиками транспорта необязательно, что у пользователя есть только один адрес электронной почты. Пользователь может иметь отдельный адрес электронной почты для каждого установленного поставщика транспорта или иметь другой адрес для каждого сеанса на одном поставщике.
    
Дополнительные сведения о хранении учетных данных в разделах профилей см. в разделах [Message Services and Profiles and](message-services-and-profiles.md) [IProfSect: IMAPIProp](iprofsectimapiprop.md).
