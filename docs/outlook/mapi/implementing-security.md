# <a name="implementing-security"></a>Реализация безопасности

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Если это требуется для системы обмена сообщениями, поставщик транспорта несет ответственность за реализацию соответствующего уровня безопасности для доступа к системе обмена сообщениями. Каждое входящее или исходящее сообщение, отправляемое через поставщик транспорта с помощью диспетчера очереди MAPI, обрабатывается в контексте сеанса входа поставщика. Поставщик транспорта может отобразить диалоговое окно входа в систему для пользователя, который запрашивает учетные данные пользователя, прежде чем устанавливать такое подключение. Кроме того, поставщик транспорта может хранить введенные ранее учетные данные пользователя в диапазоне свойств безопасности в разделе профиля и использовать их для доступа без запросов.
  
При реализации безопасности поставщика транспорта учитывайте следующие возможности.
  
- С несколькими установленными поставщиками услуг может быть множество имен и паролей, связанных с пользователем.
    
- MAPI разрешает несколько сеансов с несколькими удостоверениями. Поставщики должны поддерживать несколько сеансов, но это необязательно.
    
- Каждый сеанс с поставщиком транспорта связан MAPI с дискретным разделом в профиле пользователя. Поставщик транспорта может использовать метод [имаписуппорт:: опенпрофилесектион](imapisupport-openprofilesection.md) для получения доступа к этому разделу, который можно использовать для хранения любых сведений, связанных с этим сеансом, в том числе учетных данных. 
    
- Если у вас есть несколько установленных поставщиков транспорта, это не обязательно означает, что у пользователя есть только один адрес электронной почты. У пользователя может быть отдельный адрес электронной почты для каждого установленного поставщика транспорта или другой адрес для каждого сеанса на одном поставщике.
    
Для получения дополнительных сведений о хранении учетных данных в разделах профилей, ознакомьтесь со статьей [службы и профили сообщений](message-services-and-profiles.md) и [ипрофсект: IMAPIProp](iprofsectimapiprop.md).
