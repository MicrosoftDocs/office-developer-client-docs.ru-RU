# <a name="form-configuration-file-properties-section"></a>Файл конфигурации формы: раздел [Свойства]

  
  
**Относится к**: Outlook 
  
В разделе **[Свойства]** представлен полный набор свойств, которые использует формы и публикует; то есть свойства, он создает в его настраиваемого сообщения, клиент MAPI приложений можно отображения столбцов, фильтрация содержимого таблицы, Настройка папки результатов поиска и т.д. Каждая запись в этом списке свойство ссылается на последующих **[свойство.** _строка_ раздел **]** , как показано ниже. 
  
 **[Свойства]**
  
 **Свойство.** _Строка_ =  _строки_
  
Формат [ **свойство.** _string_] — Это раздел: 
  
 **[Свойство.** _строка_ **]**
  
 **Тип** =  _целое число_
  
 **NmidPropset** =  _guid_
  
 **NmidString** =  _строки_
  
 **NmidInteger** =  _целое число_
  
 **Отображаемое имя** =  _строки_
  
 **Флаги** =  _целое число_
  
 **SpecialType** = 0 | 1 
  
 **Enum1** =  _строки_
  
Каждый **[свойство.** _строка_ **]** разделе описываются отдельное свойство. Запись **типа** указывает тип свойства MAPI, например 3 (PT_I4), свойство. Запись **NmidPropset** является необязательным; вместе с **NmidString** запись или запись **NmidInteger** запись **NmidPropset** предоставляет имя свойства. **NmidString** предоставляет имя свойства, в то время как **NmidInteger** предоставляет идентификатор свойства. **NmidString** и **NmidInteger** являются взаимоисключающими. 
  
Если задан, **NmidPropset** должны содержать имя набора свойств; Если отсутствует, **NmidPropset** задано значение по умолчанию, на основе следующие правила: Если присутствует **NmidInteger** его значение равно, меньше, чем 0x8000 **NmidPropset** задано значение PS_MAPI. Если значение **NmidInteger** имеет значение целое число больше, чем 0x8000 или он отсутствует, **NmidPropset** задано значение PS_PUBLIC_STRINGS. 
  
Запись **DisplayName** содержит метку для свойства. Запись **SpecialType** , если этот параметр указан и ненулевое значение указывает, что это свойство специальные свойства. В настоящее определенного типа только специальные свойства — это **SpecialType** = 1 задает свойства строки перечисления. Если **SpecialType** задано значение 1, ссылается на запись **Enum1** **[Enum1.** _строка_ раздел **]** . 
  
Ниже приведен пример раздела **[Свойства]** и **[свойств.** _строка_ раздел **]** . 
  
```cpp
[Properties]
Property.1 = Fire Hazard
Property.2 = Safe
[Property.Fire Hazard]
Type = 1
NmidPropSet = {E47F4480-8400-101B-934D-04021C007002]
NmidString = FireHazard
DisplayName = Fire Hazard
SpecialType = 1
Enum1 = HazardEnum

```

Запись **Enum1** в предыдущем примере ссылки на последующем **[Enum1.** _строка_ **]** раздела, описывающего перечисления определенного типа. Такие перечисления связывает к первому свойству **[свойство.** _строка_ раздел **]** с помощью свойства целого числа, называется индекса. Такие перечисления также содержит список возможных значений, которые можно допустить пару отображения индекса. Указание типа свойства для перечисления необязательно, так как по определению запись **Enum1** всегда имеет тип PT_I4. Формат для **[Enum1.** _строка_ раздел **]** : 
  
 **[Enum1.** _строка_ **]**
  
 **NmidPropset** =  _guid_
  
 **NmidString** =  _строки_
  
 **NmidInteger** =  _целое число_
  
 **EnumCount** =  _целое число_
  
 **Функция Val.** _целое число_ **. Отображение** =  _строки_
  
 **Функция Val.** _целое число_ **. Индекс** =  _целое число_
  
Ниже приводится определение примере свойство для перечислимых свойств, с именем пожарной опасности с возможными значениями из низкий, средний и высокий.
  
```cpp
[Properties]
Property1 = Fire Hazard
[Enum1.HazardEnum]
IdxNmidPropset={E47F4480-8400-101B-934D-04021C007002]
IdxNmidString=FireHazardEnum
EnumCount = 3
Val.1.Display = Low
Val.1.Index = 1
Val.2.Display = Medium
Val.2.Index = 2
Val.3.Display = High
Val.3.Index = 3

```

 **[Enum1.** _строка_ разделы **]** можно использовать в приложениях двух целях: ускорить фильтрацию свойств с помощью индекса вместо строки и сортировка в порядке, отличном от буквенно-цифровой порядок строковых значений. Например сортировку можно осуществить зависимости от порядка Low Умеренно высокий вместо порядке высоким среднего.
