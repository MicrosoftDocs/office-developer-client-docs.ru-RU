# <a name="form-configuration-file-properties-section"></a>Файл конфигурации формы — раздел [Свойства]

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Раздел **[Properties]** содержит полный набор свойств, которые форма использует и публикует; то есть свойства, которые он создает в настраиваемых сообщениях, которые клиентские приложения MAPI могут использовать при отображении столбцов, фильтрации таблиц содержимого, настройке папок результатов поиска и т. д. Каждая запись в этом списке свойств ссылается на последующее **[Property.** _строка_ **]** , как показано ниже. 
  
 **Параметры**
  
 **Свойств.** _string_ =  _строка_ String
  
Формат **свойства [.** _строка_] раздел: 
  
 **Свойств.** _строка_ **]**
  
 **Тип** =  _Integer_
  
 **NmidPropset** =  _GUID_ нмидпропсет
  
 **NmidString** =  _Строка_ нмидстринг
  
 **NmidInteger** =  _Целое число_ нмидинтежер
  
 **DisplayName** =  _Строка_ DisplayName
  
 **Flags** =  _Целое число_ флагов
  
 **СпеЦиалтипе** = 0 | 1 
  
 **Enum1** =  _Строка_ Enum1
  
Каждое **свойство [.** _строка_ **]** описание одиночного свойства. Запись **Type** указывает тип свойства MAPI, например 3 (PT_I4) свойства. Запись **нмидпропсет** является необязательной; Вместе с записью **нмидстринг** или **Нмидинтежер** , запись **нмидпропсет** предоставляет имя свойства. **Нмидстринг** предоставляет имя свойства, в то время как **нмидинтежер** задает идентификатор свойства. **Нмидстринг** и **нмидинтежер** являются взаимоисключающими. 
  
Если этот параметр установлен, **нмидпропсет** должен содержать имя набора свойств; Если отсутствует, для **нмидпропсет** устанавливается значение по умолчанию, основанное на следующем правиле: Если **нмидинтежер** присутствует, а его значение меньше 0x8000, **нмидпропсет** задано значение PS_MAPI. Если для параметра **нмидинтежер** задано целое число больше 0x8000 или отсутствует, **нмидпропсет** имеет значение PS_PUBLIC_STRINGS. 
  
Запись **DisplayName** содержит метку для свойства. Запись **спеЦиалтипе** , если она есть, и ненулевое значение указывает на то, что это свойство является особым. В настоящее время определен единственный тип специального свойства: **спеЦиалтипе** = 1, который указывает строковые свойства перечисления. Если для **спеЦиалтипе** задано значение 1, запись **Enum1** ссылается на **[Enum1.** _строка_ **]** . 
  
Ниже приведен пример раздела **[Properties]** и **свойства [Properties.** _строка_ **]** . 
  
```cpp
[Properties]
Property.1 = Fire Hazard
Property.2 = Safe
[Property.Fire Hazard]
Type = 1
NmidPropSet = {E47F4480-8400-101B-934D-04021C007002]
NmidString = FireHazard
DisplayName = Fire Hazard
SpecialType = 1
Enum1 = HazardEnum

```

В предыдущем примере запись **Enum1** ссылается на последующий **[Enum1.** _строка_ **]** , описывающая перечисление определенного типа. Такое перечисление связывает первое свойство в **свойстве [.** _строка_ **]** с целочисленным свойством, называемым индексом. Такое перечисление также содержит список возможных значений, которые может предполагать набор для отображения индекса Display. Указание типа свойства для перечисления не требуется, так как в определении **Enum1** запись всегда имеет тип PT_I4. Формат для параметра **[Enum1.** _строка_ **]** раздел: 
  
 **[Enum1.** _строка_ **]**
  
 **NmidPropset** =  _GUID_ нмидпропсет
  
 **NmidString** =  _Строка_ нмидстринг
  
 **NmidInteger** =  _Целое число_ нмидинтежер
  
 **EnumCount** =  _Целое число_ енумкаунт
  
 **Стоимость.** _целое число_ **. Отображаемая** =  _строка_
  
 **Стоимость.** _целое число_ **. ** =  _Целое число_ индекса
  
Ниже приведен пример определения свойства для перечислимого свойства с именем Fire опасные с возможными значениями "минимум", "средний" и "высокий".
  
```cpp
[Properties]
Property1 = Fire Hazard
[Enum1.HazardEnum]
IdxNmidPropset={E47F4480-8400-101B-934D-04021C007002]
IdxNmidString=FireHazardEnum
EnumCount = 3
Val.1.Display = Low
Val.1.Index = 1
Val.2.Display = Medium
Val.2.Index = 2
Val.3.Display = High
Val.3.Index = 3

```

 **[Enum1.** _строка_ **]** разделы могут использоваться приложениями для двух целей: для ускорения фильтрации свойств с помощью индекса вместо строки, а также для сортировки по порядку, отличному от буквенно-цифрового порядка строковых значений. Например, сортировку можно выполнить на основе минимального среднего значения, а не высокого среднего.
