# <a name="form-configuration-file-properties-section"></a>Файл конфигурации формы [свойства] Раздел

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
В **разделе [Свойства]** перечислены все свойства, которые форма использует и публикует; т. е. свойства, которые оно создает в настраиваемом сообщении, которое клиентские приложения MAPI могут использовать при отображке столбцов, фильтрации таблиц содержимого, настройке папок результатов поиска и т. д. Каждая запись в этом списке свойств ссылается на **последующее [Свойство.** _строка_ **]** раздела, как показано ниже. 
  
 **[Свойства]**
  
 **Свойство.** _string_  =   _string_
  
Формат [ **Свойство.** _string_] раздел: 
  
 **[Свойство.** _string_ **]**
  
 **Тип**  =   _integer_
  
 **NmidPropset**  =   _guid_
  
 **NmidString**  =   _string_
  
 **NmidInteger**  =   _integer_
  
 **DisplayName**  =   _string_
  
 **Флаги**  =   _integer_
  
 **SpecialType** = 0|1 
  
 **Enum1**  =   _string_
  
Каждое **[Свойство.** _в_ разделе **строки ]** описывается одно свойство. Запись **Type** указывает тип свойства MAPI, например 3 (PT_I4), свойства. Запись **NmidPropset** является необязательной; Вместе с **записью NmidString** или **NmidInteger** запись **NmidPropset** предоставляет имя свойства. **NmidString** предоставляет имя свойства, а **NmidInteger** — идентификатор свойства. **NmidString и** **NmidInteger** являются взаимоисключающими. 
  
Если этот набор за установлен, **NmidPropset** должен содержать имя набора свойств; Если значение отсутствует, **для NmidPropset** задано значение по умолчанию на основе следующего правила: если **NmidInteger** присутствует и его значение меньше 0x8000, **NmidPropset** имеет значение PS_MAPI. Если для **NmidInteger** задано значение, большее 0x8000 или если оно отсутствует, **NmidPropset** имеет значение PS_PUBLIC_STRINGS. 
  
Запись **DisplayName** содержит метку свойства. Запись **SpecialType** ,если она присутствует и не является zero, указывает на то, что это свойство является специальным свойством. В настоящее время определен единственный специальный тип свойства **SpecialType** = 1, который указывает строковые свойства, указанные в этом свойстве. Если **для SpecialType** установлено 1, запись **Enum1 ссылается** **на [Enum1.** _строка_ **]** раздела. 
  
Ниже приводится пример раздела **[Properties]** и **[Properties.** _строка_ **]** раздела. 
  
```cpp
[Properties]
Property.1 = Fire Hazard
Property.2 = Safe
[Property.Fire Hazard]
Type = 1
NmidPropSet = {E47F4480-8400-101B-934D-04021C007002]
NmidString = FireHazard
DisplayName = Fire Hazard
SpecialType = 1
Enum1 = HazardEnum

```

Запись **Enum1** в предварительном примере ссылается на **последующий [Enum1.** _строка_ **],** описывающий enumeration определенного типа. Такое enumeration связывает первое свойство в **[Property.** _строка_ **]** с свойством, называемым индексом. Такое перечисляние также содержит список возможных значений, которые может использовать пара display-index. Указывать тип свойства для этого PT_I4 не нужно, так как по определению запись **Enum1** всегда имеет тип PT_I4. Формат для **[Enum1.** _строка_ **]** раздел: 
  
 **[Enum1.** _string_ **]**
  
 **NmidPropset**  =   _guid_
  
 **NmidString**  =   _string_
  
 **NmidInteger**  =   _integer_
  
 **EnumCount**  =   _integer_
  
 **Вэл.** _integer_ **. Отображаемая**  =   _строка_
  
 **Вэл.** _integer_ **.**  =   _Integer индекса_
  
Ниже приводится пример определения свойства для нумеруемого свойства Fire Hazard с возможными значениями Low, Medium и High.
  
```cpp
[Properties]
Property1 = Fire Hazard
[Enum1.HazardEnum]
IdxNmidPropset={E47F4480-8400-101B-934D-04021C007002]
IdxNmidString=FireHazardEnum
EnumCount = 3
Val.1.Display = Low
Val.1.Index = 1
Val.2.Display = Medium
Val.2.Index = 2
Val.3.Display = High
Val.3.Index = 3

```

 **[Enum1.** _разделы_ строки **]** могут использоваться приложениями для двух целей: для ускорения фильтрации свойств с помощью индекса вместо строки и сортировки по порядку, отличаемого от буквно-цифрового порядка строки. Например, сортировка может быть сделана на основе низкого-среднего и высокого порядка, а не на основе высокого-среднего и низкого порядка.
