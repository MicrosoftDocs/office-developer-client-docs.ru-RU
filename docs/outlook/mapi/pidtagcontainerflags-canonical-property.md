# <a name="pidtagcontainerflags-canonical-property"></a>Каноническое свойство PidTagContainerFlags

  
  
**Относится к**: Outlook 
  
Содержит битовую маску флаги, описывающие возможности контейнер адресной книги. 
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_CONTAINER_FLAGS  <br/> |
|Идентификатор:  <br/> |0x3600  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Адресная книга  <br/> |
   
## <a name="remarks"></a>Замечания

Для Битовая маска, которая может быть установлено одно или несколько из следующих флагов:
  
AB_FIND_ON_OPEN 
  
> Отображает диалоговое окно для запроса ограничение перед отображением любое содержимое контейнера. 
    
AB_MODIFIABLE 
  
> Записи могут быть добавлены в и удаляется из контейнера. Этот флаг указывает, можно ли изменить все записи из контейнера.
    
AB_RECIPIENTS 
  
> Получатели может содержаться в контейнере. Этот флаг указывает, ли все получатели действительно присутствует в контейнере или ли они могут добавлять или удалять. 
    
AB_SUBCONTAINERS 
  
> Контейнер может содержать дочерних контейнеров. Этот флаг не указывает, являются ли все подконтейнеры действительно присутствует в контейнере, а также является ли они могут добавлять или удалять. AB_SUBCONTAINERS должен иметь значение для контейнера для поддержки [IMAPIContainer::GetHierarchyTable](imapicontainer-gethierarchytable.md). 
    
AB_UNMODIFIABLE 
  
> Невозможно добавить для записи или удаляется из контейнера. Этот флаг указывает, можно ли изменить все записи из контейнера. 
    
Флаг AB_FIND_ON_OPEN настоятельно рекомендуется для контейнеры, используемые с Интернет-служб или медленных подключений к серверам. При открытии контейнера, который имеет значение AB_FIND_ON_OPEN, диалоговое окно **Найти** представлены пользователю для ограничения отображаемых пользователи обмена мгновенными сообщениями. Даже частичное спецификацию ограничение числа пользователей, обмена мгновенными сообщениями можно значительно ускорить отображения содержимого. 
  
Должен быть установлен флаг AB_MODIFIABLE или AB_UNMODIFIABLE. Чтобы указать, что контейнер не знает ли его можно изменить или нет, например, если изменения зависит от прав доступа можно задать обоих флагов. В этом случае клиентского приложения должен попытаться звонка и проанализировать код возврата для определения возможности контейнера. Клиент обычно начинает проверять AB_MODIFIABLE. Если оно установлено, клиент вызывает метод, пытается изменить контейнер и проверяет возвращаемое значение. 
  
Флаг AB_MODIFIABLE Указывает типы записей, которые могут быть добавлены в контейнер. Для определения этого клиента следует использовать соответствующий метод [OpenProperty](imapiprop-openproperty.md) для открытия контейнера свойств **PR_CREATE_TEMPLATES** ([PidTagCreateTemplates](pidtagcreatetemplates-canonical-property.md)). Открытие **PR_CREATE_TEMPLATES** причины раза контейнера таблицы должно быть возвращено список типов записей, которые могут быть созданы в контейнере.
