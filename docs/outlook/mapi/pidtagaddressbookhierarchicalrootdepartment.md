# <a name="pidtagaddressbookhierarchicalrootdepartment"></a>Каноническое свойство PidTagAddressBookHierarchicalRootDepartment

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
 Содержит различающееся имя (DN) иерархического корня адресов (ИЕРАРХИческой адресной книги). 
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |ПР_ЕМС_АБ_ХАБ_РУТ_ДЕПАРТМЕНТ, ПР_ЕМС_АБ_ХАБ_РУТ_ДЕПАРТМЕНТ_А  <br/> |
|Набор свойств:  <br/> |Адресная книга  <br/> |
|Длинный идентификатор (крышка):  <br/> |0x8C98  <br/> |
|Тип данных:  <br/> |PT_STRING8  <br/> |
|Область:  <br/> |Адресная книга Exchange  <br/> |
   
## <a name="remarks"></a>Замечания

Это свойство в контейнере глобального списка адресов (GAL), представляющее различающееся имя иерархического корня адреса. Это свойство присутствует только в автономной адресной книге и никогда в доменных службах Active Directory (AD DS). Вызывающие абоненты должны передавать МАПИ_КАЧЕ_ОНЛИ в вызове метода Props, чтобы избежать вызова удаленной процедуры. Если этот параметр отсутствует, вызывающим абонентам следует использовать ПР_ЕМС_АБ_ХАБ_РУТ_ДЕПАРТМЕНТ, имеющий тип ПТ_ОБЖЕКТ, чтобы найти корневой отдел. 
  
После получения корневого отдела он может иметь тип объекта МАПИ_МАИЛУСЕР или МАПИ_ДИСТЛИСТ. Если типом объекта является МАПИ_ДИСТЛИСТ, используется новая схема. Если типом объекта является МАПИ_МАИЛУСЕР, используется Предыдущая схема. 
  
- Microsoft Office Outlook 2007 с пакетом обновления 2 поддерживает обе схемы. 
    
- Microsoft Outlook 2010 и Microsoft Outlook 2013 поддерживают новую схему.
    
В новой схеме все группы отделов также являются списками рассылки и относятся к типу МАПИ_ДИСТЛИСТ. Члены групп отделов и отделов в группах отделов получаются с помощью ПР_ЕМС_АБ_МЕМБЕР, точно так же, как члены списка рассылки.
  
После получения корневого отдела он может иметь тип объекта МАПИ_МАИЛУСЕР или МАПИ_ДИСТЛИСТ. Если типом объекта является МАПИ_ДИСТЛИСТ, используется новая схема. Если типом объекта является МАПИ_МАИЛУСЕР, используется старая схема. 
  
В новой схеме все группы отделов также являются разделами и относятся к типу МАПИ_ДИСТЛИСТ.
