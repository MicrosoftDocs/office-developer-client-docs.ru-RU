# <a name="required-and-optional-interfaces-for-message-store-providers"></a>Необходимые и необязательные интерфейсы для поставщиков store сообщений

 
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
MAPI определяет набор интерфейсов, которые связаны с поставщиками store сообщений. Из-за широкого спектра функций, которые хранилище сообщений может выбрать для реализации, некоторые из этих интерфейсов являются обязательной, а некоторые нет. В следующей таблице перечислены интерфейсы MAPI, связанные с поставщиками store сообщений, указывается, являются ли интерфейсы обязательными или необязательными, а также описывается их назначение.
  
|**Интерфейс**|**Состояние**|**Описание**|
|:-----|:-----|:-----|
|[IMSProvider](imsprovideriunknown.md) <br/> |Обязательна  <br/> |Войдите в хранилище сообщений и выйдите из нее.  <br/> |
|[IMSLogon](imslogoniunknown.md) <br/> |Обязательна  <br/> |Открывает папки или сообщения, проверяет удостоверение хранилище сообщений и обрабатывает уведомления.  <br/> |
|[IMsgStore](imsgstoreimapiprop.md) <br/> |Обязательна  <br/> |Открывает папки или сообщения, находит специальные папки и обрабатывает отправки сообщений.  <br/> |
|[IMAPIFolder](imapifolderimapicontainer.md) <br/> |Обязательна  <br/> |Находит и управляет сообщениями и вудерями.  <br/> |
|[IMessage](imessageimapiprop.md) <br/> |Обязательна  <br/> |Управляет вложениями и задает некоторые свойства сообщения.  <br/> |
|[IMAPITable](imapitableiunknown.md) <br/> |Обязательна  <br/> |Позволяет другим объектам представлять коллекции данных различным компонентам MAPI.  <br/> |
|[IMAPIStatus](imapistatusimapiprop.md) <br/> |Обязательна  <br/> |Позволяет клиентам проверять состояние хранения сообщений и выполнять некоторые задачи настройки.  <br/> |
|[IAttach](iattachimapiprop.md) <br/> |Необязательна  <br/> |Доступ к свойствам вложений сообщений, если поставщик магазина поддерживает вложения файлов.  <br/> |
|**IStorage** <br/> |Необязательна  <br/> |Управляет объектами структурированного хранилища, если поставщик хранилища поддерживает вложения объектов OLE.  <br/> |
|**IStream** <br/> |Необязательна  <br/> |Позволяет объектам сообщений и вложений считывать и записывать данные для потоковой передачи объектов.  <br/> |
|**IStreamDocfile** <br/> |Необязательна  <br/> |Позволяет некоторым поставщикам услуг открывать объект хранилища, например составной файл в формате OLE 2.0.  <br/> |
   
Основные сведения, необходимые для реализации **IMAPIFolder,** **IMessage,** **IMAPIStatus** и **IMAPITable,** задокументированы в справочных темах для этих интерфейсов. В этом разделе содержатся дополнительные сведения, которые более непосредственно связаны с поставщиками хранения сообщений. Остальные интерфейсы MAPI должны быть реализованы в соответствии с информацией в этом разделе и в соответствующих справочных разделах. Дополнительные сведения о реализации **IStorage,** **IStream** и **IStreamDocFile** см. в разделе com и ActiveX Object Services в Windows SDK.
