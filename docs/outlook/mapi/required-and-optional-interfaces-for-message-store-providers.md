# <a name="required-and-optional-interfaces-for-message-store-providers"></a>Необходимые и необязательные интерфейсы для поставщиков магазинов сообщений

 
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
MAPI определяет набор интерфейсов, которые относятся к поставщикам хранения сообщений. Из-за широкого спектра функций, которые может реализовать хранилище сообщений, некоторые из этих интерфейсов необходимы, а некоторые — нет. В следующей таблице перечислены интерфейсы MAPI, связанные с поставщиками хранения сообщений, указывается, являются ли интерфейсы обязательными или необязательными, и описывается их назначение.
  
|**Интерфейс**|**Состояние**|**Описание**|
|:-----|:-----|:-----|
|[IMSProvider](imsprovideriunknown.md) <br/> |Обязательный  <br/> |Вход в хранилище сообщений и его отключение.  <br/> |
|[IMSLogon](imslogoniunknown.md) <br/> |Обязательный  <br/> |Открывает папки или сообщения, проверяет удостоверение магазина сообщений и обрабатывает уведомления.  <br/> |
|[IMsgStore](imsgstoreimapiprop.md) <br/> |Обязательный  <br/> |Открывает папки или сообщения, находит специальные папки и обрабатывает отправки сообщений.  <br/> |
|[IMAPIFolder](imapifolderimapicontainer.md) <br/> |Обязательный  <br/> |Находит и манипулирует сообщениями и подмостки.  <br/> |
|[IMessage](imessageimapiprop.md) <br/> |Обязательный  <br/> |Манипулирует вложениями и задает некоторые свойства сообщения.  <br/> |
|[IMAPITable](imapitableiunknown.md) <br/> |Обязательный  <br/> |Позволяет другим объектам представлять коллекции данных для различных компонентов MAPI.  <br/> |
|[IMAPIStatus](imapistatusimapiprop.md) <br/> |Обязательный  <br/> |Позволяет клиентам проверять состояние магазина сообщений и выполнять некоторые задачи конфигурации.  <br/> |
|[IAttach](iattachimapiprop.md) <br/> |Необязательный  <br/> |Доступ к свойствам вложений сообщений, если поставщик магазина поддерживает вложения файлов.  <br/> |
|**IStorage** <br/> |Необязательный  <br/> |Управляет структурированными объектами хранения, если поставщик магазинов поддерживает вложения объектов OLE.  <br/> |
|**IStream** <br/> |Необязательный  <br/> |Позволяет объектам сообщений и вложений читать и записывать данные для потоковой передачи объектов.  <br/> |
|**IStreamDocfile** <br/> |Необязательный  <br/> |Позволяет некоторым поставщикам услуг открывать объект хранения, например сложный файл в формате файлов OLE 2.0.  <br/> |
   
Основные сведения, необходимые для реализации **IMAPIFolder,** **IMessage,** **IMAPIStatus** и **IMAPITable,** описаны в справочных темах для этих интерфейсов. В этом разделе содержатся дополнительные сведения, более непосредственно связанные с поставщиками хранения сообщений. Остальные интерфейсы MAPI должны быть реализованы в соответствии с информацией в этом разделе и в соответствующих справочных разделах. Дополнительные сведения о реализации **IStorage,** **IStream** и **IStreamDocFile** см. в разделе Com и ActiveX object Services в Windows SDK.
