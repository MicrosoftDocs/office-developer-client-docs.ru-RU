# <a name="disk-instances-and-cache-tables"></a>Экземпляры диска и таблицы кэша

**Область применения**: Outlook 2013 | Outlook 2016 
  
Чтобы активировать форму, ее исполняемые файлы должны быть доступны на компьютере пользователя. Если они недоступны, их необходимо скопировать из библиотеки форм на локальный диск. Для этого диспетчер форм по умолчанию создает подтексто в каталоге Windows пользователя, чтобы содержать исполняемые файлы формы (. EXEs. HLP). Этот каталог называется экземпляром диска формы.
  
Диспетчер форм по умолчанию поддерживает таблицу всех экземпляров диска, чтобы если экземпляр диска уже существует, его можно использовать без копирования файлов из библиотеки форм на диск пользователя. Таблица экземпляров дисков управляется как наименее часто используемый кэш. Если требуется новый экземпляр диска, он копируется на компьютер пользователя, заменяя наименее часто используемый экземпляр диска. Затем таблица кэша экземпляра диска обновляется с учетом последней конфигурации. Размер дискового кэша — это настраиваемый пользователем параметр, позволяющий пользователям балансировать скорость с доступной емкостью диска.
  
Помимо кэша экземпляра диска диспетчер форм по умолчанию поддерживает таблицу запущенных экземпляров, в которую перечислены все запущенные экземпляры серверов форм на компьютере пользователя. При этом используется возможность MAPI сохранять бездействуют экземпляры форм, работающие в невидимом состоянии, пока не активируется форма класса сообщений этого сервера формы. Другими словами, серверы форм можно кэшируют в оперативной памяти, чтобы свести к минимуму количество раз, когда исполняемая форма должна располагаться в библиотеке форм и загружаться в память с диска или по сети. Как и кэш экземпляра диска, кэш запущенного экземпляра ведет себя наименее часто используемым способом, чтобы можно было счидить экземпляр запущенной формы из кэша, чтобы сделать место для другого экземпляра формы. Этот кэш будет искать запущенный экземпляр сервера форм перед поиском библиотек форм для сервера форм.
  
> [!NOTE]
> Диспетчер форм по умолчанию отображает индикатор прогресса при установке формы на рабочей станции пользователя, что позволяет пользователю отменить операцию. Это особенно полезно, если подключение пользователя к исполняемого файла сервера форм находится в сети с низкой пропускной способностью.
