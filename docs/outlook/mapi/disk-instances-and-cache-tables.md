# <a name="disk-instances-and-cache-tables"></a>Дисковые экземпляры и таблицы кэша

**Относится к**: Outlook 2013 | Outlook 2016 
  
Чтобы активировать форму, ее исполняемые файлы должны быть доступны на компьютере пользователя. Если они недоступны, их необходимо скопировать из библиотеки форм на локальный диск. Для этого диспетчер форм по умолчанию создает в каталоге Windows пользователя подкадиректор, содержащий исполняемые файлы формы (. EXEs,. HLP). Этот каталог называется экземпляром диска формы.
  
Диспетчер форм по умолчанию поддерживает таблицу всех экземпляров диска, чтобы, если экземпляр диска уже существует, его можно использовать без копирования файлов из библиотеки форм на диск пользователя. Таблица экземпляров диска управляется как кэш с наименьшей регулярной таблицей. Если требуется новый экземпляр диска, он копируется на компьютер пользователя, заменяя наименее часто используемый экземпляр диска. Затем таблица кэша экземпляров диска обновляется с учетом последней конфигурации. Размер дискового кэша настраивается пользователем, позволяя пользователям сбалансировать скорость и доступную емкость диска.
  
Помимо кэша экземпляров диска диспетчер форм по умолчанию поддерживает запущенную таблицу экземпляров, в которую перечислены все запущенные экземпляры серверов форм на компьютере пользователя. При этом используется возможность MAPI сохранять бездействовать экземпляры форм, работающие в невидимом состоянии, пока не будет активирована форма класса сообщений этого сервера форм. Другими словами, серверы форм можно кэшируют в ОЗУ, чтобы свести к минимуму количество раз, когда исполняемый исполняемый фрагмент формы должен располагаться в библиотеке форм и загружаться в память с диска или по сети. Как и кэш экземпляра диска, кэш запущенных экземпляров работает в наименее часто используемом способе, что делает экземпляр запущенной формы очисткой из кэша, чтобы уместить его для другого экземпляра формы. Поиск в этом кэше запущенного экземпляра сервера форм перед поиском библиотеки форм на сервере форм.
  
> [!NOTE]
> Диспетчер форм по умолчанию отображает индикатор хода выполнения при установке формы на рабочей станции пользователя, позволяя пользователю отменить операцию. Это особенно полезно, если подключение пользователя к исполняемом файлу сервера форм выполняется по сети с низкой пропускной способностью.
