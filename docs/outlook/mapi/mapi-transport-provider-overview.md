# <a name="mapi-transport-provider-overview"></a>Обзор поставщика транспорта MAPI

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Поставщики транспорта обрабатывают передачу и прием сообщений и реализуют безопасность при необходимости. Они также заботятся о любых необходимых задачах по подготовке и постпроцессации. Для каждой активной системы обмена сообщениями обычно используется один поставщик транспорта.
  
Клиентские приложения взаимодействуют с поставщиком транспорта через поставщика магазина сообщений. 
  
Поставщики транспорта регистрируются в MAPI для обработки одного или более определенных типов записей получателей. Когда сообщение готово к отправлению, MAPI должен определить, какой поставщик транспорта должен обрабатывать передачу. В зависимости от типа получателя MAPI может даже вызвать несколько поставщиков транспорта. Если недоступный поставщик транспорта является единственным поставщиком транспорта, который может обрабатывать получателя, передача сообщений будет отложена до повторного подключения к этому поставщику.
  
Некоторые системы обмена сообщениями являются безопасными системами; все потенциальные пользователи должны ввести набор допустимых учетных данных, прежде чем будет разрешен доступ. MAPI предотвращает несанкционированный доступ к таким защищенным системам обмена сообщениями с помощью проверки учетных данных поставщика транспорта во время логоса.
