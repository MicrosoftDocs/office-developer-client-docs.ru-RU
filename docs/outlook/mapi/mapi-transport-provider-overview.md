# <a name="mapi-transport-provider-overview"></a>Обзор поставщика транспорта MAPI

  
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
Поставщики транспорта обработки передачи сообщений и прием и обеспечения безопасности, если это необходимо. Они также взяла на себя любые необходимые предварительной обработки и постобработки задачи. Отсутствует поставщик обычно один транспорта для каждой активной системы обмена сообщениями.
  
Клиентские приложения общаться с помощью поставщика транспорта через поставщика хранилища сообщений. 
  
Зарегистрируйте поставщиками транспорта MAPI для обработки одного или нескольких отдельных типов получателей. Если сообщение не готово к отправке, MAPI необходимо определить, какие поставщика транспорта должен обрабатывать передачи. В зависимости от типа получателя MAPI даже могут использовать более одного поставщика транспорта. Если поставщик недоступен транспорта — единственный, который может обрабатывать получателя, передачи сообщений будет отложено до можно заново установить соединение с этого поставщика.
  
Некоторые системы обмена сообщениями, безопасных систем; все потенциальные клиенты придется ввести набор допустимые учетные данные перед разрешением доступа. MAPI не позволяет несанкционированного доступа для таких безопасных систем обмена сообщениями благодаря использованию поставщика транспорта проверки учетных данных во время входа в систему.
