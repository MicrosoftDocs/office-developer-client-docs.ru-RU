# <a name="pidtagstatus-canonical-property"></a>Каноническое свойство PidTagStatus

  
  
**Относится к**: Outlook 
  
Содержит Битовая 32-разрядная версия флаги, описывающие состояние папки.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_STATUS  <br/> |
|Идентификатор:  <br/> |0x360B  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Контейнер MAPI  <br/> |
   
## <a name="remarks"></a>Замечания

Это свойство для папок является аналогом свойство **PR_MSG_STATUS** ([PidTagMessageStatus](pidtagmessagestatus-canonical-property.md)) для сообщений. Флаги предоставляются для в клиентском приложении и не влияют на хранилище сообщений. Клиенты могут использовать или эти параметры игнорируются. Клиент можно также определить собственные значения для клиента определенный бит этого свойства.
  
Для Битовая маска, которая может быть установлено одно или несколько из следующих флагов:
  
FLDSTATUS_DELMARKED 
  
> Папка помечается для удаления. Клиентское приложение задает этот флаг.
    
FLDSTATUS_HIDDEN 
  
> Папка является скрытым.
    
FLDSTATUS_HIGHLIGHTED 
  
> Папка будет выделена, например, показано в обратном порядке видео.
    
FLDSTATUS_TAGGED 
  
> Папка тегами.
    
Этому свойству присвоено поставщиков хранилища сообщений в папке один или несколько из следующих значений и клиентов интерпретации состояния в зависимости от своих приложений. Например клиент может использовать состояние папки позволяет визуально различать папок в иерархии таблицы, отображение папок в одном состоянии так же, как. Выделенный папок можно будет отображаться в обратном видео, помеченных папки и папки, помеченные для удаления могут отображаться со значком удобной для восприятия и скрытые папки можно скрытых.
  
Бит 16 до 31 («0x10000» через «0x80000000») этого свойства доступны для использования клиентским приложением IPM. Все другие биты, зарезервированы для использования с MAPI; не определенных в предыдущем списке следует сначала задать нулевое значение и не изменяется.
