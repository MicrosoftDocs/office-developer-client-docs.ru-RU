# <a name="pidtagstatus-canonical-property"></a>Каноническое свойство PidTagStatus

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Содержит 32 — битовую маску для флагов, определяющих состояние папки.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_STATUS  <br/> |
|Идентификатор:  <br/> |0x360B  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Контейнер MAPI  <br/> |
   
## <a name="remarks"></a>Примечания

Это свойство для папок аналогично свойству **PR_MSG_STATUS** ([PidTagMessageStatus](pidtagmessagestatus-canonical-property.md)) для сообщений. Его флаги предоставляются только для клиентского приложения и не влияют на хранилище сообщений. Клиенты могут использовать или игнорировать эти параметры. Клиент также может определить собственные значения для задаваемых клиентом битов этого свойства.
  
Для битовой маски можно задать один или несколько из следующих флагов:
  
FLDSTATUS_DELMARKED 
  
> Папка помечена для удаления. Клиентское приложение устанавливает этот флаг.
    
FLDSTATUS_HIDDEN 
  
> Папка скрыта.
    
FLDSTATUS_HIGHLIGHTED 
  
> Папка выделяется, например, в обратном видеоролике.
    
FLDSTATUS_TAGGED 
  
> Папка размечена.
    
Поставщики хранилища сообщений присвойте этому свойству папки одно или несколько из этих значений, и клиенты интерпретируют состояние в соответствии с их приложениями. Например, клиент может использовать состояние папки для визуального различения папок в иерархической таблице, отображая папки с одинаковым состоянием аналогичным образом. Выделенные папки можно отобразить в обратном видеоролике, папках с тегами и папках, помеченных для удаления, можно отобразить с осмысленным значком, а скрытые папки можно скрыть.
  
Биты с 16 по 31 ("0x10000", "0x80000000") этого свойства доступны для использования клиентским приложением IPM. Все остальные биты зарезервированы для использования MAPI; для тех, которые не определены в предыдущем списке, необходимо сначала установить нулевое значение и не изменять.
