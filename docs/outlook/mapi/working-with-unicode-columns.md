# <a name="working-with-unicode-columns"></a>Работа со столбцами в Юникоде

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Строки символов в таблицах могут быть представлены с помощью стандартных 8 – разрядных символов, которые являются типом свойства PT_STRING8 или 16 – разрядными символами Юникода, которые являются типом свойства PT_UNICODE. Разработчики таблиц могут свободно выбирать, могут ли их таблицы поддерживать строки Юникода. Так как Юникод добавляет значение для клиентов и поставщиков услуг, расширяя набор функций, везде, где это возможно, рекомендуется поддерживать Юникод. 
  
Многие методы таблиц принимают флаг, определяющий, должны ли значения свойства String быть Юникод. При вводе с указанием флага MAPI_UNICODE указывает на то, что все значения строковых свойств, переданные с вызовом, являются строками Юникода и имеют типы свойств PT_UNICODE. В выходных данных этот флаг указывает на то, что все возвращенные значения свойств строки должны быть строками Юникода, если это возможно. Зависит ли значение флага для ввода или вывода от метода. Разработчики таблиц, не поддерживающие Юникод и передаваемые флагом MAPI_UNICODE, возвращают MAPI_E_BAD_CHAR_WIDTH значение.
