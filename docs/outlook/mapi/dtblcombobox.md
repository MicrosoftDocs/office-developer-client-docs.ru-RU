# <a name="dtblcombobox"></a>DTBLCOMBOBOX

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Описывает элемент управления "поле со списком", который будет использоваться в диалоговом окне, созданном из таблицы отображения.
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |MAPIDEFS. h  <br/> |
|Связанный макрос:  <br/> |[SizedDtblComboBox](sizeddtblcombobox.md) <br/> |
   
```cpp
typedef struct _DTBLCOMBOBOX
{
  ULONG ulbLpszCharsAllowed;
  ULONG ulFlags;
  ULONG ulNumCharsAllowed;
  ULONG ulPRPropertyName;
  ULONG ulPRTableName;
} DTBLCOMBOBOX, FAR *LPDTBLCOMBOBOX;

```

## <a name="members"></a>Members

 **Улблпсзчарсалловед**
  
> Смещение от начала структуры **дтблкомбобокс** к фильтру строк символов, описывающему ограничения (при их наличии) до символов, которые можно ввести в поле редактирования поля со списком. Фильтр не интерпретируется как регулярное выражение, и к каждому введенному символу применяется один и тот же фильтр. Фильтр имеет следующий формат: 
    
|**Символ**|**Описание**|
|:-----|:-----|
| `*` <br/> |Разрешен любой символ (например, `"*"`).  <br/> |
| `[ ]` <br/> |Определяет набор символов (например, `"[0123456789]"`).  <br/> |
| `-` <br/> |Указывает диапазон символов (например, `"[a-z]"`).  <br/> |
| `~` <br/> |Указывает, что эти символы не допускаются. (например, `"[~0-9]"`).  <br/> |
| `\` <br/> |Используется для заключения в кавычки любого из предыдущих символов (например `"[\-\\\[\]]"` , допустимые \, символы-, [, и]).  <br/> |
   
 **ulFlags**
  
> Битовая маска флагов, используемых для обозначения формата фильтра строк символов. Можно задать следующий флаг:
    
MAPI_UNICODE 
  
> Фильтр имеет формат Юникод. Если флаг МАПИ_УНИКОДЕ не установлен, фильтр имеет формат ANSI.
    
 **Улнумчарсалловед**
  
> Максимальное число символов, которое можно ввести в текстовом поле поля со списком.
    
 **Улпрпропертинаме**
  
> Тег свойства для свойства типа ПТ_ТСТРИНГ. 
    
 **Улпртабленаме**
  
> Тег свойства для свойства типа ПТ_ОБЖЕКТ, в котором можно открыть интерфейс **IMAPITable** с помощью вызова **опенпроперти** . У таблицы должен быть один столбец со свойством, имеющим тот же тип, что и свойство, определяемое элементом **улпрпропертинаме** . Строки таблицы используются для заполнения списка. 
    
## <a name="remarks"></a>Замечания

Структура **дтблкомбобокс** описывает элемент управления "поле со списком", состоящий из списка и поля выбора. В списке представлены сведения, которые пользователь может выбрать, а в поле Выбор отображается текущий выделенный фрагмент. Поле Selection является элементом управления редактирования, который можно использовать для ввода текста, которого еще нет в списке. 
  
Два элемента тега свойства взаимодействуют для координации отображения списка с помощью элемента управления "поле ввода". При первом отображении поля со списком с помощью MAPI вызывается метод **опенпроперти** реализации **IMAPIProp** , связанный с таблицей отображения для получения таблицы, представленной элементом **улпртабленаме** . В этой таблице есть один столбец, содержащий значения для свойства, представленного элементом **улпрпропертинаме** . Таким образом, этот столбец должен иметь тот же тип, что и свойство **улпрпропертинаме** , и оба столбца должны быть строками символов. 
  
Значения для столбца отображаются в разделе "список" в поле со списком. Таким образом, **пр_нулл** ([PidTagNull](pidtagnull-canonical-property.md)) не является допустимым тегом свойства для **улпрпропертинаме**. Когда пользователь выбирает одну из строк или вводит новые данные в текстовое поле, свойству **улпрпропертинаме** присваивается выбранное или введенное значение. 
  
Для отображения начального значения для элемента управления "поле ввода" MAPI вызывает [IMAPIProp::](imapiprop-getprops.md) /PROPS для получения значений свойств для таблицы отображения. Если одно из извлеченных свойств соответствует свойству, представленному элементом **улпрпропертинаме** , его значение становится начальным. 
  
Общие сведения о отображаемых таблицах приведены в разделе [Display Tables](display-tables.md). Сведения о том, как реализовать таблицу отображения, можно найти [в статье реализация таблицы отображения](display-table-implementation.md).
