# <a name="dtblcombobox"></a>DTBLCOMBOBOX

  
  
**Относится к**: Outlook 
  
Поле со списком, который будет использоваться в диалоговое окно, построенная на основе таблицы Отображаемое описание.
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapidefs.h  <br/> |
|Связанные макрос:  <br/> |[SizedDtblComboBox](sizeddtblcombobox.md) <br/> |
   
```cpp
typedef struct _DTBLCOMBOBOX
{
  ULONG ulbLpszCharsAllowed;
  ULONG ulFlags;
  ULONG ulNumCharsAllowed;
  ULONG ulPRPropertyName;
  ULONG ulPRTableName;
} DTBLCOMBOBOX, FAR *LPDTBLCOMBOBOX;

```

## <a name="members"></a>Members

 **ulbLpszCharsAllowed**
  
> Смещение от начала структуры **DTBLCOMBOBOX** в фильтр символ строка, описывающая ограничения, если оно нужно символов, которое можно ввести в поле со списком редактирования элемента управления. Фильтр не обрабатывается как регулярное выражение и тот же фильтр применяется к каждый символ, введенный. Формат фильтр выглядит следующим образом: 
    
|**Символ**|**Описание**|
|:-----|:-----|
| `*` <br/> |Допускается любой символ (например, `"*"`).  <br/> |
| `[ ]` <br/> |Определяет набор символов (например, `"[0123456789]"`).  <br/> |
| `-` <br/> |Указывает диапазон символов (например, `"[a-z]"`).  <br/> |
| `~` <br/> |Указывает, что эти символы не допускается. (например, `"[~0-9]"`).  <br/> |
| `\` <br/> |Используется для любой из предыдущих символы кавычек (например, `"[\-\\\[\]]"` означает, что-, \, [, и] могут знаков).  <br/> |
   
 **ulFlags**
  
> Битовая маска флаги, используемые для обозначения формата фильтра строку знаков. Можно задать следующий флаг:
    
MAPI_UNICODE 
  
> Фильтр — в формате Юникод. Если флаг MAPI_UNICODE не установлен, в формате ANSI — фильтра.
    
 **ulNumCharsAllowed**
  
> Максимальное число символов, которые можно ввести в поле со списком текстовое поле.
    
 **ulPRPropertyName**
  
> Свойство tag для свойства типа PT_TSTRING. 
    
 **ulPRTableName**
  
> Свойство тег для свойства PT_OBJECT тип, на котором можно открыть интерфейс **IMAPITable** путем вызова **OpenProperty** . Таблица должна иметь один столбец со свойством, которое совпадает с типом свойство, определяемое члена **ulPRPropertyName** . Строки таблицы используются для заполнения списка. 
    
## <a name="remarks"></a>Замечания

Структура **DTBLCOMBOBOX** описывает элемент управления, содержащий список и поле выбора поле со списком. В списке представлены сведения, с которого пользователь может выбрать, и поле выбора отображает текущее выделение. В поле выбора является элементом управления, который также можно использовать для ввода текста еще не в списке. 
  
Элементы тега два свойства работают вместе для координации отображения списка с помощью элемента управления поля ввода. Когда MAPI сначала отображает поле со списком, он вызывает метод **OpenProperty** реализации **IMAPIProp** , связанный с таблицей отображения для получения таблицы, представленного элементом **ulPRTableName** . Эта таблица имеет столбец, который содержит значения для свойства, представленного элементом **ulPRPropertyName** один столбец. Таким образом этот столбец должен быть того же типа как свойство **ulPRPropertyName** и обоих столбцов должно быть строк символов. 
  
Значения для столбца отображаются в разделе список поле со списком. Таким образом, **PR_NULL** ([PidTagNull](pidtagnull-canonical-property.md)) не является допустимым свойством тег для **ulPRPropertyName**. Если пользователь выбирает один из строки или ввод новых данных в текстовом поле, свойство **ulPRPropertyName** задано значение выбранного или введенная. 
  
Чтобы отобразить начальное значение для элемента управления поля ввода, MAPI вызывает [IMAPIProp::GetProps](imapiprop-getprops.md) для извлечения значения свойств для отображения таблицы. Если один из извлеченные свойства соответствует свойству, представленному член **ulPRPropertyName** , его значение становится начального значения.
