# <a name="mapi-service-provider-objects"></a>Объекты поставщика услуг MAPI

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Поставщики услуг реализуют множество объектов. Некоторые из них в основном используются MAPI, а другие — клиентские приложения. Несколько объектов реализованы всеми типами поставщиков услуг; остальные типы являются специфическими для одного типа поставщика. В следующей таблице описаны все объекты поставщика услуг.
  
|**Объект поставщика услуг**|**Описание**|
|:-----|:-----|
|Контейнер адресной книги  <br/> |Содержит сведения о получателях для одного поставщика адресной книги в активном профиле; Поставщики адресных книг могут иметь один или несколько контейнеров адресной книги.  <br/> |
|Вложение  <br/> |Содержит дополнительные данные, например файл или объект OLE, которые должны быть связаны с сообщением.  <br/> |
|Элемент управления  <br/> |Включает или отключает кнопку и инициирует обработку при нажатии кнопки.  <br/> |
|Список рассылки  <br/> |Описывает группу отдельных получателей сообщений.  <br/> |
|Folder  <br/> |Содержит сообщения и другие контейнеры сообщений.  <br/> |
|Вход в систему  <br/> |Обработка уведомлений о событиях поставщика услуг и запросов клиентов.  <br/> |
|Пользователь системы обмена сообщениями  <br/> |Описывает отдельного получателя сообщения.  <br/> |
|Сообщение  <br/> |Содержит сведения, которые могут быть отправлены одному или более получателям.  <br/> |
|Хранилище сообщений  <br/> |Действует как иерархически уорганизованная база данных сообщений.  <br/> |
|Поставщик  <br/> |Обрабатывает запуск и завершение работы поставщика услуг.  <br/> |
|Обселок пула  <br/> |Выполняет специальную обработку входящие и исходящие сообщения.  <br/> |
|Состояние  <br/> |Предоставляет доступ к состоянии поставщика услуг.  <br/> |
|Table  <br/> |Предоставляет доступ к сводным представлениям данных объектов в формате строк и столбцов, аналогично таблице базы данных.  <br/> |
   
Все поставщики услуг реализуют объект поставщика и объект для логотипа. Объекты поставщика исключительно для регистрации; ОНИ используются MAPI для управления процессами запуска и завершения работы. Объекты для эмблемы неявно обслуживания некоторых клиентских запросов. Например, объект для регистрации поставщика для хранения сообщений обрабатывает регистрацию уведомлений и запросы на открытие объектов, хранимых в хранилище сообщений. 
  
Объекты поставщиков и объектов для логотипа реализуют другой интерфейс в зависимости от типа поставщика услуг, который обеспечивает реализацию. Поставщик хранения сообщений реализует [интерфейсы IMSProvider : IUnknown](imsprovideriunknown.md) и [IMSLogon: интерфейсы IUnknown](imslogoniunknown.md) в своих объектах поставщика и объектах для логотипа, поставщик адресной книги реализует [IABProvider : IUnknown](iabprovideriunknown.md) и [IABLogon : интерфейсы IUnknown,](iablogoniunknown.md) а поставщик транспорта реализует [интерфейсы IXPProvider : IUnknown](ixpprovideriunknown.md) и [IXPLogon : IUnknown.](ixplogoniunknown.md) 
  
Поставщики обработок сообщений реализуют объекты обработки пулов или объекты, фильтруя входящие и исходящие сообщения.
  
Поставщики услуг обычно используют только несколько объектов. Чаще всего они используют объект поддержки, который ПРЕДОСТАВЛЯЕТ MAPI для реализации клиентских запросов. Объект поддержки настраивается для типа используемого поставщика. Для всех поставщиков услуг объект поддержки включает методы для обработки уведомлений о событиях, отображения свойств конфигурации, открытия объектов и обработки ошибок. Остальные методы специфически для его использования; существуют настроенные версии для адресной книги, магазина сообщений и поставщиков транспорта, а также для поддержки конфигурации. Например, объект поддержки адресной книги отображает сведения и настраиваемые диалоги получателей. Объект поддержки для хранения сообщений поддерживает операции копирования и перемещения папок и сообщений. Объект поддержки поставщика транспорта включает методы для облегчения взаимодействия с пулом MAPI. 
  
Некоторые поставщики услуг используют табличные данные и объекты данных свойств — объекты-служние, которые реализует MAPI. Объекты табличные данные позволяют поставщикам услуг управлять данными, которые находятся в сфере таблицы. Объекты данных свойств позволяют поставщикам услуг устанавливать доступ к объектам и свойствам. 
  
Поставщики транспорта, которые поддерживают формат TNEF для передачи свойств, используют объект TNEF, реализованный MAPI для поддержки интерфейса [ITnef : IUnknown.](itnefiunknown.md) Дополнительные сведения см. в TNEF-Enabled [поставщика транспорта.](developing-a-tnef-enabled-transport-provider.md)
