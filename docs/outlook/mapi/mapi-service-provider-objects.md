# <a name="mapi-service-provider-objects"></a>Объекты поставщика службы MAPI

  
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
Поставщики услуг реализовать множество объектов. Некоторые используются в основном MAPI и некоторые используются с клиентскими приложениями. Несколько объектов, реализуются посредством всех типов поставщиков услуг; остальные, специфичные для типа одного поставщика. В следующей таблице описываются все объекты поставщика службы.
  
|**Объект, предоставляющий службы**|**Описание**|
|:-----|:-----|
|Контейнер адресной книги  <br/> |Содержит сведения о получателях для один поставщик адресной книги в текущей конфигурации; поставщиками адресной книги может иметь один или несколько контейнеров адресной книги.  <br/> |
|Attachment  <br/> |Содержит дополнительные данные, например, файл или объект OLE, который будет связан с сообщением.  <br/> |
|Control  <br/> |Включает или отключает кнопку и инициирует обработки при нажатии кнопки.  <br/> |
|Список рассылки  <br/> |Описывает группировку получателей отдельных сообщений.  <br/> |
|Folder  <br/> |Содержит сообщения и другие контейнеры сообщения.  <br/> |
|Вход в систему  <br/> |Поставщик событий уведомлений и клиентских запросы в службу маркеров.  <br/> |
|Обмена сообщениями пользователя  <br/> |Описание отдельного получателя сообщения.  <br/> |
|Message  <br/> |Содержит сведения, которые могут быть отправлены для одного или нескольких получателей.  <br/> |
|Хранение сообщений  <br/> |Действует как иерархически организованы базы данных сообщений.  <br/> |
|Поставщик  <br/> |Маркеры службы поставщика запуска и завершения.  <br/> |
|Обработчик очереди  <br/> |Выполняет специальная обработка входящих и исходящих сообщений.  <br/> |
|Status  <br/> |Предоставляет доступ к состоянию поставщика услуг.  <br/> |
|Таблица  <br/> |Предоставляет доступ к сводное представление данных объекта в формате строк и столбцов, аналогичную таблицы базы данных.  <br/> |
   
Все поставщики услуг реализации поставщика объектов и входа в систему. Поставщик объектов можно использовать только для регистрации; они используются для MAPI для управления процессов запуска и завершения. Объекты входа службы косвенно некоторые запросы клиентов. Например сообщение хранения регистрацию поставщика входа в систему объекта значками уведомлений и запросы на открытие объектов хранилища сообщений. 
  
Объекты поставщика и входа в систему реализовать различные интерфейс в зависимости от типа поставщика услуг, который предоставляет реализацию. Реализует поставщика хранилища сообщений [IMSProvider: IUnknown](imsprovideriunknown.md) и [IMSLogon: IUnknown](imslogoniunknown.md) интерфейсы в его поставщика и объекты входа в систему, адрес книги реализует поставщик [IABProvider: IUnknown](iabprovideriunknown.md) и [IABLogon: IUnknown](iablogoniunknown.md) реализует интерфейсы и поставщика транспорта [IXPProvider: IUnknown](ixpprovideriunknown.md) и [IXPLogon: IUnknown](ixplogoniunknown.md) интерфейсов. 
  
Поставщики обработчик сообщения реализовать очереди обработчик объектов или объектов, которые фильтрации входящих и исходящих сообщений.
  
Поставщиков услуг обычно используется только несколько объектов. Наиболее часто используются объект поддержки, который предоставляет MAPI, чтобы реализовать запросов клиентов. Объект поддержки настроенный тип поставщика, с помощью его. Для всех поставщиков услуг поддержки объект содержит методы для обработки уведомлений о событиях, отображения свойства конфигурации, открывающий объекты и обработка ошибок. Остальных методов предназначенные специально для ее использования; существует настроенной версии для адресной книги, хранилища сообщений и поставщиками транспорта, а также для настройки поддержки. К примеру объект поддержки адресной книги отображает сведения о и получателей диалоговых окон. Хранилище сообщений поддерживают объект поддерживает копирование и перемещение операции для сообщений и папок. Объект поддержки поставщика транспорта содержит методы, позволяющие упрощение взаимодействия с диспетчер очереди MAPI. 
  
Некоторые поставщики услуг используйте таблицу данных и свойства объектов данных — служебная программа объекты, внедряемые MAPI. В таблице данных объекты позволяют поставщиков услуг для управления данным таблицы. Объекты данных свойство включите поставщиков услуг установить доступ объектов и свойств. 
  
Поставщики транспорта, которые поддерживают транспорта Neutral Encapsulation формата TNEF для передачи свойств используйте объект TNEF, реализуемые программным интерфейсом MAPI для поддержки [ITnef: IUnknown](itnefiunknown.md) интерфейса. Для получения дополнительных сведений см [Разработка поставщика транспорта TNEF-Enabled](developing-a-tnef-enabled-transport-provider.md).
