# <a name="pidtagentryid-canonical-property"></a>Каноническое свойство PidTagEntryId

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит идентификатор записи MAPI, используемый для открытия и изменения свойств определенного объекта MAPI. 
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_ENTRYID  <br/> |
|Идентификатор:  <br/> |0x0FFF  <br/> |
|Тип данных:  <br/> |PT_BINARY  <br/> |
|Область:  <br/> |Свойства идентификатора  <br/> |
   
## <a name="remarks"></a>Комментарии

Это свойство идентифицирует объект для **OpenEntry** , который создает экземпляр, и предоставляет доступ ко всем его свойствам с помощью соответствующего производного интерфейса **IMAPIProp**. 
  
Это свойство является одним из свойств базового адреса для всех пользователей системы обмена сообщениями. 
  
Это свойство может содержать либо долгосрочный, либо краткосрочный идентификатор. Краткосрочные идентификаторы проще и быстрее строятся, но ограничены их областью и длительностью, обычно к текущему сеансу и рабочей станции. Они обычно используются для объектов временной природы, таких как строки таблицы или элементы диалоговых окон, а затем отброшены. Долгосрочные идентификаторы используются для объектов более широкого и долгосрочного характера. 
  
Это свойство всегда доступно с помощью метода [IMAPIProp::](imapiprop-getprops.md) -Props, следующего за первым вызовом метода [IMAPIProp:: SaveChanges](imapiprop-savechanges.md) . Некоторые поставщики услуг могут сделать их доступными сразу же после создания экземпляра. Поставщик должен всегда возвращать долгосрочный идентификатор записи из PROPS ****. Таким образом, чтобы преобразовать краткосрочный идентификатор в долгосрочный, просто откройте объект и получите его свойство через **PROPS**. 
  
В следующей таблице приведены важные различия между этим свойством, **пр_рекорд_кэй** ([PidTagRecordKey](pidtagrecordkey-canonical-property.md)) и **пр_сеарч_кэй** ([PidTagSearchKey](pidtagsearchkey-canonical-property.md)).
