<span data-ttu-id="f9962-105">Клиенты могут завершать свои сеансы в ответ на запрос пользователя либо сразу же, либо после того, как все исходящие сообщения будут обработаны, а также при возникновении критической ошибки.</span><span class="sxs-lookup"><span data-stu-id="f9962-105">Clients can end their sessions in response to a user's request, either immediately or after all outbound messages have been processed, and when a critical error occurs.</span></span> <span data-ttu-id="f9962-106">Для некоторых клиентов необходимо оставаться в системе, чтобы ожидающие исходящие сообщения могли получить доступ к поставщику транспорта и конечной системе обмена сообщениями.</span><span class="sxs-lookup"><span data-stu-id="f9962-106">Some clients need to stay logged on so that pending outbound messages can reach the transport provider and the destination messaging system.</span></span> <span data-ttu-id="f9962-107">Если такой клиент отправляет сообщение и сразу же выходит из нее, сообщение может находиться в исходящей очереди до тех пор, пока пользователь не войдет в систему и не войдет в систему достаточно долго для передачи сообщения.</span><span class="sxs-lookup"><span data-stu-id="f9962-107">If such a client sends a message and immediately logs off, the message may remain in the outgoing queue until a user logs back on and stays logged on long enough for the message to be transmitted.</span></span>
  
 <span data-ttu-id="f9962-108">**Когда необходимо завершить сеанс с подсистемой MAPI**</span><span class="sxs-lookup"><span data-stu-id="f9962-108">**When you need to terminate your session with the MAPI subsystem**</span></span>
  
1. <span data-ttu-id="f9962-109">Отмените регистрацию для всех уведомлений, вызвав метод **unadvise** каждого зарегистрированного объекта.</span><span class="sxs-lookup"><span data-stu-id="f9962-109">Cancel the registrations for all notifications by calling the **Unadvise** method of every registered object.</span></span> 
    
2. <span data-ttu-id="f9962-110">Освободите все открытые объекты, вызвав их методы [IUnknown:: Release](https://msdn.microsoft.com/library/ms682317%28VS.85%29.aspx) .</span><span class="sxs-lookup"><span data-stu-id="f9962-110">Release all open objects by calling their [IUnknown::Release](https://msdn.microsoft.com/library/ms682317%28VS.85%29.aspx) methods.</span></span> <span data-ttu-id="f9962-111">Типы открытых объектов могут включать приемники уведомлений, таблицу состояния, папку "Исходящие", одно или несколько хранилищ сообщений и адресную книгу.</span><span class="sxs-lookup"><span data-stu-id="f9962-111">The types of open objects can include advise sinks, the status table, the Outbox folder, one or more message stores, and the address book.</span></span> 
    
3. <span data-ttu-id="f9962-112">Вызовите [мапифрибуффер](mapifreebuffer.md) , чтобы освободить память для всех кэшированных идентификаторов записей, таких как **PR_IPM_SUBTREE_ENTRYID** ([PidTagIpmSubtreeEntryId](pidtagipmsubtreeentryid-canonical-property.md)).</span><span class="sxs-lookup"><span data-stu-id="f9962-112">Call [MAPIFreeBuffer](mapifreebuffer.md) to free the memory for any cached entry identifiers, such as **PR_IPM_SUBTREE_ENTRYID** ([PidTagIpmSubtreeEntryId](pidtagipmsubtreeentryid-canonical-property.md)).</span></span>
    
4. <span data-ttu-id="f9962-113">Call [IMAPISession:: logoff](imapisession-logoff.md), установка флага MAPI_LOGOFF_UI, если вы разрешите пользовательский интерфейс и флаг MAPI_LOGOFF_SHARED, если вы владеете текущим общим сеансом.</span><span class="sxs-lookup"><span data-stu-id="f9962-113">Call [IMAPISession::Logoff](imapisession-logoff.md), setting the MAPI_LOGOFF_UI flag if you allow a user interface and the MAPI_LOGOFF_SHARED flag if you own the current shared session.</span></span> <span data-ttu-id="f9962-114">**Выход из системы** уведомляет всех остальных клиентов, использующих текущий общий сеанс, которые должны выйти из системы, отправив уведомление об ошибке.</span><span class="sxs-lookup"><span data-stu-id="f9962-114">**Logoff** notifies all other clients that are using the current shared session that they should log off by sending an error notification.</span></span> 
    
5. <span data-ttu-id="f9962-115">Освободите указатель сеанса, вызвав метод **IUnknown:: Release** сеанса.</span><span class="sxs-lookup"><span data-stu-id="f9962-115">Release the session pointer by calling the session's **IUnknown::Release** method.</span></span> 
    
6. <span data-ttu-id="f9962-116">Если вы вызвали [олеинитиализе](https://msdn.microsoft.com/library/ms690134%28v=VS.85%29.aspx) во время запуска сеанса для инициализации библиотек OLE, теперь вызовите их, вызвав [олеунинитиализе](https://msdn.microsoft.com/library/ms691326%28VS.85%29.aspx).</span><span class="sxs-lookup"><span data-stu-id="f9962-116">If you called [OleInitialize](https://msdn.microsoft.com/library/ms690134%28v=VS.85%29.aspx) during session startup to initialize the OLE libraries, uninitialize them now by calling [OleUninitialize](https://msdn.microsoft.com/library/ms691326%28VS.85%29.aspx).</span></span> <span data-ttu-id="f9962-117">Только клиенты, вызываемые **олеинитиализе** , должны вызывать **олеунинитиализе**.</span><span class="sxs-lookup"><span data-stu-id="f9962-117">Only clients that have called **OleInitialize** must call **OleUninitialize**.</span></span> 
    
7. <span data-ttu-id="f9962-118">Инициализируйте библиотеки MAPI, вызвав [мапиунинитиализе](mapiuninitialize.md).</span><span class="sxs-lookup"><span data-stu-id="f9962-118">Uninitialize the MAPI libraries by calling [MAPIUninitialize](mapiuninitialize.md).</span></span> <span data-ttu-id="f9962-119">Если вы вызвали **олеинитиализе** в некоторый момент, убедитесь, что вызов **олеунинитиализе** происходит перед этим вызовом **мапиунинитиализе**.</span><span class="sxs-lookup"><span data-stu-id="f9962-119">If you called **OleInitialize** at some point, make sure that a call to **OleUninitialize** occurs before this call to **MAPIUninitialize**.</span></span> <span data-ttu-id="f9962-120">Время имеет решающее значение.</span><span class="sxs-lookup"><span data-stu-id="f9962-120">The timing is crucial.</span></span> <span data-ttu-id="f9962-121">Если при вызове **олеунинитиализе** вызывается метод **мапиунинитиализе**, клиент может завершиться некорректно.</span><span class="sxs-lookup"><span data-stu-id="f9962-121">If the call to **OleUninitialize** follows the call to **MAPIUninitialize**, your client might terminate ungracefully.</span></span> 
    
8. <span data-ttu-id="f9962-122">Если во время запуска сеанса вызывается [сЦинитмапиутил](scinitmapiutil.md) для инициализации библиотеки служебной программы MAPI, его можно деинициализировать, вызвав [деинитмапиутил](deinitmapiutil.md).</span><span class="sxs-lookup"><span data-stu-id="f9962-122">If you called [ScInitMapiUtil](scinitmapiutil.md) during session startup to initialize the MAPI utility library, uninitialize it now by calling [DeinitMapiUtil](deinitmapiutil.md).</span></span> <span data-ttu-id="f9962-123">Только клиенты, вызываемые **сЦинитмапиутил** , должны вызывать **деинитмапиутил**.</span><span class="sxs-lookup"><span data-stu-id="f9962-123">Only clients that have called **ScInitMapiUtil** must call **DeinitMapiUtil**.</span></span>
