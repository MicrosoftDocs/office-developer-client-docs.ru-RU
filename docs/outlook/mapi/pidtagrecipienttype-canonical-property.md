# <a name="pidtagrecipienttype-canonical-property"></a>Каноническое свойство PidTagRecipientType

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Содержит тип получателя сообщения.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_RECIPIENT_TYPE  <br/> |
|Идентификатор:  <br/> |0x0C15  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Получатель MAPI  <br/> |
   
## <a name="remarks"></a>Примечания

Тип получателя, содержащийся в этом свойстве, состоит из одного обязательного значения и одного необязательного флага.
  
Это свойство должно содержать только одно из следующих значений:
  
MAPI_TO 
  
> Получатель является основным (to) получателем. Для обработки основных получателей требуются клиенты. ��� ��������� ���� �������� ���������������.
    
MAPI_CC 
  
> Получатель — это получатель копии (CC), а получатель, который получает сообщение в дополнение к основным получателям.
    
MAPI_BCC 
  
> Получатель — это получатель скрытой копии (BCC). Основной получатель и получатель копии не знают о существовании получателей СКРЫТой копии. 
    
MAPI_P1 
  
> Получателю не удалось получить сообщение о предыдущей попытке. Это повторная отправка более ранней передачи.
    
Кроме того, можно задать следующий флаг:
  
MAPI_SUBMITTED 
  
> Получатель уже получил сообщение и не требует его повторного получения. Это повторная отправка более ранней передачи. Этот флаг задается в сочетании со значениями **MAPI_TO**, **MAPI_CC**и **MAPI_BCC** . 
    
Значение MAPI_P1 и флаг **MAPI_SUBMITTED** используются, когда сообщение передается в связи с доставкой одного или нескольких указанных получателей. Для такой повторной передачи клиент устанавливает **MAPI_SUBMITTED** для каждого получателя, который не требует повторного сообщения, но должен отображаться в списке получателей. Для каждого получателя, который не получал сообщение ранее, клиент сохраняет исходный получатель со своим **PR_RECIPIENT_TYPEным** значением без изменений, но дополнительно отправляет копию получателя с MAPI_P1 вместо исходного значения. Эта копия, которая отбрасывается до фактической доставки, приводит получателя к конверту P1 и гарантирует физическую повторную передачу для этого получателя. Для свойства **PR_RESPONSIBILITY** ([PidTagResponsibility](pidtagresponsibility-canonical-property.md)) для MAPI_P1 получателей задано значение "false".
  
Когда клиент отображает форму повторной отправки, отображаются только получатели MAPI_P1. Если пользователь не введет дополнительных получателей, то при доставке сообщения список получателей отображается точно так же, как при первом отправке сообщения. 
  
Свойства **PR_DISPLAY_TO** ([PidTagDisplayTo](pidtagdisplayto-canonical-property.md)), **PR_DISPLAY_CC** ([PidTagDisplayCc](pidtagdisplaycc-canonical-property.md)) и **PR_DISPLAY_BCC** ([PidTagDisplayBcc](pidtagdisplaybcc-canonical-property.md)) относятся к типу получателя. Когда клиент вызывает **IMAPIProp сообщения:: SaveChanges** и есть по крайней мере один получатель в списке получателей, поставщик хранилища сообщений задает эти свойства следующим образом: 
  
|**Свойство**|**Описание**|
|:-----|:-----|
|PR_DISPLAY_TO  <br/> |Задайте значение TRUE, если один или несколько получателей **MAPI_TO** получателей.  <br/> |
|PR_DISPLAY_CC  <br/> |Задайте значение TRUE, если один или несколько получателей **MAPI_CC** получателей.  <br/> |
| PR_DISPLAY_BCC  <br/> |Задайте значение TRUE, если один или несколько получателей **MAPI_BCC** получателей.  <br/> |
   
В X. 400 конверт P1 или Delivery является сведениями, необходимыми для доставки сообщения, включая свойства адреса получателя и флаги, управляющие доставкой и ответами. В качестве конверта P2 или Display отображаются сведения, которые обычно отображаются для каждого получателя, кроме самого текста сообщения. Обычно она включает тему, важность, приоритет, чувствительность и время отправки, а также имя основного и скопированного получателей.
