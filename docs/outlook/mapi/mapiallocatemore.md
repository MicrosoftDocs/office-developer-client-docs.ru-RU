# <a name="mapiallocatemore"></a>MAPIAllocateMore

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Выделяет буфер памяти, связанный с другим буфером, выделенным ранее с помощью функции [мапиаллокатебуффер](mapiallocatebuffer.md) . 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Мапикс. h  <br/> |
|Реализовано в:  <br/> |MAPI  <br/> |
|Вызывающая сторона:  <br/> |Клиентские приложения и поставщики услуг  <br/> |
   
```cpp
SCODE MAPIAllocateMore(
  ULONG cbSize,
  LPVOID lpObject,
  LPVOID FAR * lppBuffer
);
```

## <a name="parameters"></a>Параметры

 _кбсизе_
  
> возврата Размер нового выделяемого буфера (в байтах). 
    
 _лпобжект_
  
> возврата Указатель на существующий буфер MAPI, выделенный с помощью **мапиаллокатебуффер**.
    
 _лппбуффер_
  
> вышли Указатель на возвращаемый, вновь выделенный буфер.
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Вызов выполнен успешно и возвращен указатель на запрошенный объем памяти.
    
## <a name="remarks"></a>Примечания

Во время обработки вызовов **мапиаллокатеморе** вызывающая реализация получает блок памяти из операционной системы. Буфер памяти выделяется на байтовый адрес с четным номером. На платформах, где доступ к долговременному целому числу эффективнее, операционная система выделяет буфер на адресе, размер которого в байтах равен четырем. 
  
Единственный способ освободить буфер, выделенный с помощью **мапиаллокатеморе** , — передать указатель буфера, указанный в параметре _лпобжект_ , в функцию [мапифрибуффер](mapifreebuffer.md) . Связь между буферами памяти, выделенными с помощью [мапиаллокатебуффер](mapiallocatebuffer.md) и **Мапиаллокатеморе** , позволяет **мапифрибуффер** освобождать оба буфера с помощью одного вызова.
