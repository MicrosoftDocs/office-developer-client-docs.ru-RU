# <a name="transport-neutral-encapsulation-format-tnef"></a>Формат TNEF

 
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Формат TNEF — это формат, служащий для преобразования набора параметров MAPI — сообщения MAPI — в поток последовательных данных. Функции TNEF в основном используются поставщиками транспорта, которые должны кодировать свойства сообщений MAPI для передачи через систему обмена сообщениями, которая не поддерживает эти свойства напрямую. Например, транспорт на основе SMTP использует TNEF для кодирования таких **свойств,** как PR_SENT_REPRESENTING_NAME [(PidTagSentRepresentingName),](pidtagsentrepresentingname-canonical-property.md)которые не имеют прямых представлений в структуре сообщения SMTP.
  
Реализация TNEF определяет несколько атрибутов, определенных для TNEF, каждый из которых соответствует определенному свойству MAPI. Эти атрибуты используются для кодирования соответствующих свойств MAPI в поток TNEF. Кроме того, определен специальный атрибут, который можно использовать для инкапсулации любого свойства MAPI, не соответствующего ему. Причина, по которой определяются эти атрибуты, а не просто использование единообразного метода кодификатора для всех свойств MAPI, заключается в том, чтобы включить обратную совместимость с программным обеспечением, не совместимым с MAPI, использующим TNEF.
  
В остальной части этого раздела описывается структура и синтаксис потока TNEF, сопоставление свойств MAPI и атрибутов TNEF, а также важные соображения для определенных атрибутов TNEF.
