# <a name="transport-neutral-encapsulation-format-tnef"></a>Формат TNEF

 
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Формат TNEF — это формат, служащий для преобразования набора параметров MAPI — сообщения MAPI — в поток последовательных данных. Функции TNEF в основном используются поставщиками транспорта, которые нуждаются в кодировании свойств сообщений MAPI для передачи через систему обмена сообщениями, которая не поддерживает эти свойства напрямую. Например, транспорт на основе SMTP использует TNEF для кодирования свойств, таких как **PR_SENT_REPRESENTING_NAME** ([PidTagSentRepresentingName](pidtagsentrepresentingname-canonical-property.md)), которые не имеют прямого представления в структуре SMTP-сообщения.
  
Реализация TNEF определяет несколько атрибутов TNEF, каждый из которых соответствует определенному свойству MAPI. Эти атрибуты используются для кодирования соответствующих свойств MAPI в поток TNEF. Кроме того, определен специальный атрибут, который можно использовать для инкапсуляции любого свойства MAPI, для которого не определен соответствующий атрибут. Причина, по которой эти атрибуты определены, вместо простого использования единообразного метода кодирования для всех свойств MAPI — это обеспечение обратной совместимости с программным обеспечением, не совместимым с MAPI, которое использует формат TNEF.
  
В оставшейся части этого раздела описывается структура и синтаксис потока TNEF, сопоставление между свойствами MAPI и атрибутами TNEF, а также важные рекомендации по некоторым атрибутам TNEF.
