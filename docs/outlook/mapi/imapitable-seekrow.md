# <a name="imapitableseekrow"></a>IMAPITable::SeekRow

  
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
Перемещение курсора в определенное место в таблице.
  
```cpp
HRESULT SeekRow(
BOOKMARK bkOrigin,
LONG lRowCount,
LONG FAR * lplRowsSought
);
```

## <a name="parameters"></a>Параметры

 _bkOrigin_
  
> [in] Закладки, определяющее позицию для операции поиска. Закладки можно создать с помощью метода [IMAPITable::CreateBookmark](imapitable-createbookmark.md) или один из следующих предварительно заданных значений может быть передан. 
    
BOOKMARK_BEGINNING 
  
> Запускает операции поиска с начала таблицы. 
    
BOOKMARK_CURRENT 
  
> Запускает операцию поиска из строки в таблице, в котором находится курсор. 
    
BOOKMARK_END 
  
> Запускает операции поиска с конца таблицы. 
    
 _lRowCount_
  
> [in] Счетчик со знаком число строк для перемещения, начиная с закладки определенного параметром _bkOrigin_ . 
    
 _lplRowsSought_
  
> [out] Если _lRowCount_ допустимый указатель на ввода, _lplRowsSought_ указывает на количество строк, обработанных в операции поиска, знак из которых указывает направление поиска, вперед или назад. Если _lRowCount_ является отрицательным, _lplRowsSought_ является отрицательным.

ЗНАЧЕНИЕ S_OK 
  
> Операции поиска выполнена успешно.
    
MAPI_E_BUSY 
  
> В, не позволяющей операции поиска строки запуск выполняется другой операции. В процессе выполнения операции должны выполнить либо должна быть остановлена.
    
MAPI_E_INVALID_BOOKMARK 
  
> Закладки, указанный в параметре _bkOrigin_ является недопустимым, так как он был удален или находится за последней строки запроса. 
    
MAPI_W_POSITION_CHANGED 
  
> Вызов завершился успешно, но закладку, указанный в параметре _bkOrigin_ больше не устанавливаются в той же строке, что его последнего использования. Если не использовалась закладки, больше не в том же месте, которая была при его создании. При возвращении этого предупреждения вызова необходимо обрабатывать об успешном завершении. Чтобы проверить это предупреждение, используйте **HR_FAILED** макрос. Дополнительные сведения можно [С помощью макросов для обработки ошибок](using-macros-for-error-handling.md).
    
## <a name="remarks"></a>Замечания

Метод **IMAPITable::SeekRow** устанавливает новую BOOKMARK_CURRENT позицию курсора. Параметр _lRowCount_ указывает число строк, которые перемещает курсор и направление перемещения. 
  
Если итоговый позицию за пределами последней строки в таблице, курсор расположен после последней строки. Если итоговый положение перед первой строки в таблице, курсор расположен в начале первой строки. 
  
## <a name="notes-to-implementers"></a>Примечания для исполнителей

Если не удается установить новую позицию закладки строки, на который указывает _bkOrigin_ больше не существует в таблице, возвращают MAPI_E_INVALID_BOOKMARK. Если строка, на который указывает _bkOrigin_ больше не существует, можно создать новую позицию закладки возврата MAPI_W_POSITION_CHANGED. 
  
Строка, в которой сворачивается из представления таблицы с указанием закладку по-прежнему можно использовать. Попытка вызывающего абонента, переместите курсор к такой как закладка переместите курсор к следующей строке видимым и возвращают MAPI_W_POSITION_CHANGED. 
  
Вы можете переместить закладки для положения свернуты экрана, во время использования или во время скрытую строку. Если закладки перемещается во время скрытую строку, помните, немного закладки, которое указывает, имеет ли закладки перемещен с момента ее последнего использования или, если он имеет никогда не использовался, с момента его создания.
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

Чтобы указать обратной перемещения для **SeekRow**, передайте отрицательное значение _lRowCount_. Чтобы выполнить поиск в начало таблицы, передайте нулевое значение в _lRowCount_ и значение BOOKMARK_BEGINNING в _bkOrigin_. 
  
Если большое количество строк в таблице, операция **SeekRow** может быть медленно. Может также повлиять на производительность при необходимости число строк должно быть возвращено содержимое параметр _lplRowsSought_ . 
  
 **SeekRow** возвращает количество строк, фактически осуществляется через, положительное или отрицательное, указанному в переменной, на который указывает _lRowCount_. В обычные операции он должен возвращать такое же значение для _lplRowsSought_ как передано в качестве _lRowCount_, если не достигнут начала или окончания в таблице. 
  
Не следует устанавливать _lRowCount_ значение больше, чем 50. Чтобы поиск с помощью большое число строк, используйте метод [IMAPITable::SeekRowApprox](imapitable-seekrowapprox.md) .
