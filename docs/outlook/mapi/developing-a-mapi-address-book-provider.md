# <a name="developing-a-mapi-address-book-provider"></a>Разработка поставщика адресной книги MAPI

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Поставщик адресной книги передает сведения о получателях в клиентские приложения, в хранилище сообщений и поставщикам транспорта, а также в MAPI. Сведения о получателях иерархически организованы в хранилища, известные как контейнеры. Каждая адресная книга в профиле вносит один или несколько контейнеров верхнего уровня или родительских контейнеров в адресную книгу MAPI, интегрированное представление сведений о получателях от всех поставщиков адресных книг в сеансе. Клиенты и другие поставщики услуг получают доступ к данным поставщика адресной книги через адресную книгу MAPI.
  
MAPI создает интегрированную адресную книгу с помощью:
  
1. Искомые контейнеры верхнего уровня от каждого поставщика адресной книги.
    
2. Искомая таблица иерархии каждого контейнера. 
    
3. Копирование каждой таблицы иерархии в интегрированную таблицу иерархии. Это интегрированная таблица иерархии, которая является для клиента. 
    
MAPI предъявляет несколько требований к авторам поставщиков адресных книг. Диапазон возможных функций, которые можно реализовать в качестве автора адресной книги, является разнообразным и гибким. Например, поставщик может быть ограничен представлением сведений о получателях определенного типа только для чтения или реализовать полный набор функций, что, возможно, позволит клиентам или поставщикам внести дополнения или изменения в данные получателя и назначить условия поиска для определения настраиваемых представлений. 
  
Данные поставщика могут находиться локально в файле или базе данных или на удаленном сервере. Некоторые поставщики адресных книг предназначены для работы с определенной системой обмена сообщениями, тесной связи с поставщиком транспорта, а другие могут работать с любой системой обмена сообщениями.
  
MAPI определяет особый тип поставщика адресной книги, который называется личной адресной книгой, или PAB, который реализует один из измояемых контейнеров и может удерживать сведения о получателях, скопированные из других контейнеров, а также сведения, созданные напрямую. Хотя любой поставщик адресной книги может реализовать PAB, и в профиль можно добавить несколько таких адресных книг, только один из этих поставщиков может быть назначен для работы в качестве PAB во время одного сеанса.
