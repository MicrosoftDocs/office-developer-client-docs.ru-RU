# <a name="upload-folder-state"></a>Состояние отправки папки

  
  
**Относится к**: Outlook 
  
 В этом разделе описываются двоичных файлов состояние папки загрузки конечного автомата репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояний:  <br/> |**LR_SYNC_UPLOAD_FOLDER** <br/> |
|Структура связанных данных:  <br/> |**[UPFLD](upfld.md)** <br/> |
|Из этого состояния:  <br/> |[Отправка иерархия состояний](upload-hierarchy-state.md) <br/> |
|Это состояние:  <br/> |Отправка иерархия состояний  <br/> |
   
> [!NOTE]
> Конечный автомат репликации является детерминированным конечного автомата. Клиент отправления одного состояния в другое должны в конечном счете возврата на первый из последний. 
  
## <a name="description"></a>Описание

Это состояние означает инициирует отправку папки в иерархии, который был указан в предыдущем состоянии иерархии отправить. Во время этого состояния Outlook предоставляет объект папки (если она не была удалена) и флаги, указывающее состояние папки (новый, перемещены, изменены или удалены) как часть соответствующего структуры данных **UPFLD** . Затем клиент передает эти сведения на сервер. 
  
Если загрузка прошла успешно, клиент задает *ulFlags* в **UPFLD** для **UPF_OK**. Затем Outlook очищает внутренние сведения о запросе для передачи в папку. 
  
После завершения загрузки папки локального хранилища возвращается в состояние иерархии отправки. В зависимости от структуры **[UPHIER](uphier.md)** , соответствующий в предыдущем состояние иерархии отправки Outlook определяет, следует ли продолжить отправку пользователями к следующей папке и для подготовки к следующее состояние папки загрузки. 
  
> [!NOTE]
> Если клиент должен только одна папка отправки, клиент может отправить репликации через [синхронизировать состояние](synchronize-state.md) без ввода состояние иерархии отправки. Клиент задает некоторые члены **[СИНХРОНИЗАЦИИ](sync.md)** — *ulFlags* **UPS_UPLOAD_ONLY** и **UPS_ONE_FOLDER** и *feid* на идентификатор папки — чтобы сообщить Outlook, только одна папка будет отправлен.
