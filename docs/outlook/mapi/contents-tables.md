<span data-ttu-id="d1c97-113">Таблицы содержимого большинства контейнеров адресной книги и многих папок не поддерживают сортировку по категориям.</span><span class="sxs-lookup"><span data-stu-id="d1c97-113">The contents tables of most address book containers and many folders do not support categorized sorting.</span></span>

    - <span data-ttu-id="d1c97-116">Или -</span><span class="sxs-lookup"><span data-stu-id="d1c97-116">Or -</span></span>

<span data-ttu-id="d1c97-118">Поставщики хранения сообщений и поставщики адресных книг должны поддерживать оба метода для ирисовки свойств таблицы.</span><span class="sxs-lookup"><span data-stu-id="d1c97-118">Message store and address book providers must support both techniques for retrieving table properties.</span></span> <span data-ttu-id="d1c97-119">Для поставщиков недопустимо поддерживать только один способ доступа к этим таблицам, так как клиенты ожидают выбора.</span><span class="sxs-lookup"><span data-stu-id="d1c97-119">It is unacceptable for providers to support only one way for accessing these tables because clients expect to have the choice.</span></span>
