<span data-ttu-id="50a5d-105">Методы в [интерфейсе IXPLogon : IUnknown](ixplogoniunknown.md) используются spooler MAPI при вызове поставщика транспорта.</span><span class="sxs-lookup"><span data-stu-id="50a5d-105">The methods in the [IXPLogon : IUnknown](ixplogoniunknown.md) interface are used by the MAPI spooler when calling the transport provider.</span></span> <span data-ttu-id="50a5d-106">Большинство типов транспортных поставщиков должны реализовать большинство этих методов, чтобы они быстро возвращались.</span><span class="sxs-lookup"><span data-stu-id="50a5d-106">It should be possible for most types of transport providers to implement most of these methods so that they return quickly.</span></span> <span data-ttu-id="50a5d-107">Это желательно, так как если метод занимает много времени, чтобы вернуться, его следует разбить с вызовами обратно в пульпер MAPI, чтобы освободить ЦП для других задач.</span><span class="sxs-lookup"><span data-stu-id="50a5d-107">This is desirable because if a method takes a long time to return then it should be broken up with calls back to the MAPI spooler to release the CPU for other tasks.</span></span> 
  
<span data-ttu-id="50a5d-108">Spooler MAPI делает свою работу и вызывает поставщиков транспорта, когда приложения переднего плана простаивают.</span><span class="sxs-lookup"><span data-stu-id="50a5d-108">The MAPI spooler does its work and makes its calls to transport providers when foreground applications are idle.</span></span> <span data-ttu-id="50a5d-109">После необязательного отображения диалоговых ящиков, когда поставщик транспорта впервые вошел в систему (управляется флагами, переданными из MAPI поставщику транспорта), поставщики транспорта работают в фоновом режиме, если клиент не вызвал их для очистки отправки и получения очередей.</span><span class="sxs-lookup"><span data-stu-id="50a5d-109">After optionally displaying dialog boxes when the transport provider is first logged on (governed by flags passed from MAPI to the transport provider), transport providers operate in the background unless called by the client to flush send and receive queues.</span></span> <span data-ttu-id="50a5d-110">Очистка очередей — это единственный раз, когда поставщику транспорта не требуется освободить ЦП, а затем только в том случае, если пользователю будет известно о возможном длительном действии.</span><span class="sxs-lookup"><span data-stu-id="50a5d-110">Flushing queues is the only time that a transport provider need not release the CPU, and then only if the user is informed that a potentially long action is in progress.</span></span> <span data-ttu-id="50a5d-111">Пульпер MAPI обычно запрашивает, чтобы поставщик транспорта смыл очереди в ответ на действия пользователя, поэтому поставщику транспорта обычно не нужно ничего делать, чтобы убедиться, что пользователь проинформирован.</span><span class="sxs-lookup"><span data-stu-id="50a5d-111">The MAPI spooler typically requests that a transport provider flush its queues in response to a user action, so the transport provider typically does not need to do anything to ensure that the user is informed.</span></span>
  
<span data-ttu-id="50a5d-112">Поставщик транспорта может самостоятельно принять решение смыть очередь и использовать биты STATUS_INBOUND_FLUSH и STATUS_OUTBOUND_FLUSH в свойстве **PR_STATUS_CODE** [(PidTagStatusCode)](pidtagstatuscode-canonical-property.md)строки состояния, чтобы сообщить пульперу MAPI о том, что он хочет обратить внимание, чтобы получить задание.</span><span class="sxs-lookup"><span data-stu-id="50a5d-112">A transport provider can independently decide to flush a queue and use the STATUS_INBOUND_FLUSH and STATUS_OUTBOUND_FLUSH bits in the **PR_STATUS_CODE** ([PidTagStatusCode](pidtagstatuscode-canonical-property.md)) property of its status row to inform the MAPI spooler that it wants attention so that it can get the job done.</span></span> <span data-ttu-id="50a5d-113">Строка состояния обновляется с помощью [метода IMAPISupport::ModifyStatusRow.](imapisupport-modifystatusrow.md)</span><span class="sxs-lookup"><span data-stu-id="50a5d-113">The status row is updated using the [IMAPISupport::ModifyStatusRow](imapisupport-modifystatusrow.md) method.</span></span> <span data-ttu-id="50a5d-114">В этом случае поставщик транспорта, вероятно, должен отображать индикатор прогресса или другой интерфейс, чтобы сообщить пользователю о том, что происходит длительное действие.</span><span class="sxs-lookup"><span data-stu-id="50a5d-114">In this case the transport provider should probably display a progress indicator or other interface to inform the user that a long action is occuring.</span></span>
