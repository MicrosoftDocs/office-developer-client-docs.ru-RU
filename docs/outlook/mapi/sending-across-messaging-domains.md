# <a name="sending-across-messaging-domains"></a><span data-ttu-id="48c1d-103">Отправка между доменами обмена сообщениями</span><span class="sxs-lookup"><span data-stu-id="48c1d-103">Sending Across Messaging Domains</span></span>

<span data-ttu-id="48c1d-105">Домен обмена сообщениями представляет одну или несколько систем обмена сообщениями, которые имеют общий формат адресов.</span><span class="sxs-lookup"><span data-stu-id="48c1d-105">A messaging domain represents one or more messaging systems that share a common address format.</span></span> <span data-ttu-id="48c1d-106">Обмен сообщениями между несколькими доменами обмена сообщениями включает перевод сообщения, отправленного в формате исходного домена обмена сообщениями, в формат конечного домена обмена сообщениями.</span><span class="sxs-lookup"><span data-stu-id="48c1d-106">Communication across multiple messaging domains involves translating a message sent in the format of the original messaging domain into the format of the destination messaging domain.</span></span> <span data-ttu-id="48c1d-107">Так как не все форматы адресов совместимы, шлюз необходим для перевода адресных данных из формата источника в формат назначения.</span><span class="sxs-lookup"><span data-stu-id="48c1d-107">Because not all address formats are compatible, a gateway is needed to translate the addressing information from the source format into the destination format.</span></span> <span data-ttu-id="48c1d-108">Чтобы обеспечить действительность для доменов обмена сообщениями, клиентские приложения хранят важные сведения об адресовке в свойствах MAPI.</span><span class="sxs-lookup"><span data-stu-id="48c1d-108">To ensure validity across messaging domains, client applications store important addressing information in MAPI properties.</span></span> <span data-ttu-id="48c1d-109">Кроме того, шлюзы выполняют перевод, проверяя свойства, которые, как известно, требуют перевода, и меняя их на формат, который может использовать домен обмена сообщениями назначения.</span><span class="sxs-lookup"><span data-stu-id="48c1d-109">In addition, gateways perform the translation, examining the properties known to need <span data-ttu-id="48c1d-110">Ранее MAPI позволял связывать эту адресную информацию только с пользователями, составляющими текущий список получателей сообщения.</span><span class="sxs-lookup"><span data-stu-id="48c1d-110">Previously, MAPI allowed this addressing information to be associated with only the users who comprise a message's current recipient list.</span></span> <span data-ttu-id="48c1d-111">Свойства, описывающие каждого участника списка получателей, были переведены шлюзом, чтобы обеспечить правильность их работы в доменах обмена сообщениями.</span><span class="sxs-lookup"><span data-stu-id="48c1d-111">The properties describing each member of the recipient list underwent the required translation by the gateway to ensure validity across messaging domains.</span></span> <span data-ttu-id="48c1d-112">Однако некоторые приложения требуют, чтобы их сообщения включали адресную информацию о пользователях, которые, возможно, были получателями в приложения маршрутинга, которые отправляют сообщения группе пользователей в указанном порядке, встраив сведения об этих пользователях в сообщения.</span><span class="sxs-lookup"><span data-stu-id="48c1d-113">For example, routing applications, which send messages in a specified order to a group of users, embed addressing information about these users in the messages.</span></span> <span data-ttu-id="48c1d-114">Внедренные сведения обычно включают адрес и тип адреса будущих получателей, а также, возможно, их роли и позиции в порядке маршрутки, их имена и один или несколько <span data-ttu-id="48c1d-115">Чтобы сообщения включали сведения об этих нереципличных пользователях, MAPI теперь включает стратегию для обеспечения правильного перевода этих нерефициентных сведений между доменами обмена сообщениями.</span><span class="sxs-lookup"><span data-stu-id="48c1d-115">To enable messages to include information about these nonrecipient users, MAPI now includes a strategy for ensuring that this nonrecipient information is also translated correctly across messaging domains.</span></span> <span data-ttu-id="48c1d-116">Эта стратегия основана на концепции свойств, которые можно смоблить шлюзом.</span><span class="sxs-lookup"><span data-stu-id="48c1d-116">This strategy is based on the concept of gateway-mappable properties.</span></span>
