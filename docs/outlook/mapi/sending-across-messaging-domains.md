# <a name="sending-across-messaging-domains"></a><span data-ttu-id="cf23d-103">Отправка через домены обмена сообщениями</span><span class="sxs-lookup"><span data-stu-id="cf23d-103">Sending Across Messaging Domains</span></span>

  
  
<span data-ttu-id="cf23d-104">**Область применения**: Outlook 2013 | Outlook 2016</span><span class="sxs-lookup"><span data-stu-id="cf23d-104">**Applies to**: Outlook 2013 | Outlook 2016</span></span> 
  
<span data-ttu-id="cf23d-105">Домен обмена сообщениями представляет одну или несколько систем обмена сообщениями, которые имеют общий адресный формат.</span><span class="sxs-lookup"><span data-stu-id="cf23d-105">A messaging domain represents one or more messaging systems that share a common address format.</span></span> <span data-ttu-id="cf23d-106">Связь между несколькими доменами обмена сообщениями включает перевод сообщения, отправленного в формате исходного домена обмена сообщениями, в формат домена обмена сообщениями назначения.</span><span class="sxs-lookup"><span data-stu-id="cf23d-106">Communication across multiple messaging domains involves translating a message sent in the format of the original messaging domain into the format of the destination messaging domain.</span></span> <span data-ttu-id="cf23d-107">Поскольку не все форматы адресов совместимы, шлюз необходим для перевода адресных сведений из формата источника в формат назначения.</span><span class="sxs-lookup"><span data-stu-id="cf23d-107">Because not all address formats are compatible, a gateway is needed to translate the addressing information from the source format into the destination format.</span></span> <span data-ttu-id="cf23d-108">Чтобы обеспечить действительность в доменах обмена сообщениями, клиентские приложения хранят важные сведения о адресов в свойствах MAPI.</span><span class="sxs-lookup"><span data-stu-id="cf23d-108">To ensure validity across messaging domains, client applications store important addressing information in MAPI properties.</span></span> <span data-ttu-id="cf23d-109">Кроме того, шлюзы выполняют перевод, проверяя свойства, которые, как известно, нуждаются в переводе, и меняя их на формат, который может использовать домен обмена сообщениями назначения.</span><span class="sxs-lookup"><span data-stu-id="cf23d-109">In addition, gateways perform the translation, examining the properties known to need translation and changing them to a format that the destination messaging domain can use.</span></span>
  
<span data-ttu-id="cf23d-110">Ранее MAPI позволяла связывать эту адресную информацию только с пользователями, которые составляют текущий список получателей сообщения.</span><span class="sxs-lookup"><span data-stu-id="cf23d-110">Previously, MAPI allowed this addressing information to be associated with only the users who comprise a message's current recipient list.</span></span> <span data-ttu-id="cf23d-111">Свойства, описывающие каждого члена списка получателей, были переведены шлюзом для обеспечения достоверности в доменах обмена сообщениями.</span><span class="sxs-lookup"><span data-stu-id="cf23d-111">The properties describing each member of the recipient list underwent the required translation by the gateway to ensure validity across messaging domains.</span></span> <span data-ttu-id="cf23d-112">Однако некоторые приложения требуют, чтобы их сообщения включали сведения о пользователях, которые, возможно, были получателями в прошлом, будут получателями в будущем или никогда не будут получателями.</span><span class="sxs-lookup"><span data-stu-id="cf23d-112">However, some applications require that their messages include addressing information about users that perhaps were recipients in the past, will be recipients in the future, or will never be recipients.</span></span> <span data-ttu-id="cf23d-113">Например, маршрутные приложения, которые отправляют сообщения в указанном порядке группе пользователей, встраив сведения об этих пользователях в сообщения.</span><span class="sxs-lookup"><span data-stu-id="cf23d-113">For example, routing applications, which send messages in a specified order to a group of users, embed addressing information about these users in the messages.</span></span> <span data-ttu-id="cf23d-114">Встраиваемая информация обычно включает тип адресов и адресов будущих получателей, а также, возможно, их роли и позиции в порядке маршрутиза, их имена и один или несколько двоичных идентификаторов на каждого получателя.</span><span class="sxs-lookup"><span data-stu-id="cf23d-114">The embedded information typically includes the address and address type of the future recipients, and perhaps also their roles and positions in the routing order, their names, and one or more binary identifiers per recipient.</span></span>
  
<span data-ttu-id="cf23d-115">Для того чтобы сообщения включали сведения об этих нереципентных пользователях, MAPI теперь включает стратегию обеспечения правильного перевода этих нереципентных сведений в доменах обмена сообщениями.</span><span class="sxs-lookup"><span data-stu-id="cf23d-115">To enable messages to include information about these nonrecipient users, MAPI now includes a strategy for ensuring that this nonrecipient information is also translated correctly across messaging domains.</span></span> <span data-ttu-id="cf23d-116">Эта стратегия основана на концепции свойств, которые могут быть недоступны для шлюза.</span><span class="sxs-lookup"><span data-stu-id="cf23d-116">This strategy is based on the concept of gateway-mappable properties.</span></span>
