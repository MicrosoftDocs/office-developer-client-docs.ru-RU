# <a name="sending-across-messaging-domains"></a>Отправка между доменами обмена сообщениями

  
  
**Относится к**: Outlook 
  
Обмена сообщениями домена представляет один или несколько систем обмена сообщениями, совместно использующих общий формат адреса. Связи через несколько доменов обмена сообщениями включает перевод сообщение, отправленное в формате исходного домена обмена сообщениями в формат обмена сообщениями целевом домене. Так как не все форматы адресов совместимы, шлюз необходимых для перевода сведения об адресах из исходного формата в формат назначения. Для обеспечения допустимости между доменами обмена мгновенными сообщениями, клиентские приложения хранить важные сведения об адресах в свойства MAPI. Кроме того, шлюзы выполнить миграцию, проверки свойств ожидающих перевода и их изменения в формат, который можно использовать в целевом домене обмена мгновенными сообщениями.
  
Ранее MAPI разрешены в этом сведения об адресах необходимо связать с пользователями, которые составляют текущего списка получателей сообщения. Свойства, описывающие каждого элемента списка получателей были внесены необходимые перевода с требованиями шлюза для их проверки доменов обмена мгновенными сообщениями. Тем не менее некоторых приложений требуется, что сообщения, помещенные включают сведения о пользователях, которые возможно были получателей в прошлом-адресов, будут иметь получателей или никогда не будут получателей. Например маршрутизации приложений, которые отправлять сообщения в указанном порядке для группы пользователей, внедрить адресации сведения об этих пользователей в сообщениях. Внедренные данные обычно включает в себя, адрес и тип адреса будущих получателей и возможно также их ролей и положения в маршрутизации порядке, их имена и один или несколько двоичных идентификаторов отдельно для каждого получателя.
  
Чтобы включить сообщения со сведениями о nonrecipient пользователей, MAPI теперь включает стратегию проверка того, что эти nonrecipient сведения также преобразуется правильно между доменами системы обмена сообщениями. Эта стратегия основана на концепции шлюза сопоставляемые свойства.
