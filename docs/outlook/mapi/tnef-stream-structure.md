# <a name="tnef-stream-structure"></a>Структура потока TNEF

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Поток TNEF начинается с 32-битной подписи, которая определяет поток как поток TNEF. После подписи используется 16-битное неподписаенное значение, которое используется в качестве ключа для перекрестной ссылки на вложения в их расположение в тексте помеченного сообщения. Остальная часть потока — это последовательность атрибутов TNEF. Атрибуты сообщений отображаются первыми в потоке TNEF, а атрибуты вложения следуют за ними. Атрибуты, принадлежащие определенному вложению, группются вместе, начиная с атрибута **attAttachRenddata.** 
  
Большинство постоянных значений, используемых в потоках TNEF, определяются в TNEF. Файл h-загона. В частности, **TNEF_SIGNATURE,** **LVL_MESSAGE,** **LVL_ATTACHMENT** и все идентификаторы атрибутов TNEF определены в этом файле. Другие константы имеют значения, указанные в их интерпретации компилятору языка C. Как правило, такие константы используются для придаия размерам следующего элемента. Например, **sizeof(ULONG)** в определении элемента указывает на то, что в этом месте в потоке TNEF должно быть указано, что в этом месте должно быть указано следующее длинное без подписи. 
  
Все integers in a TNEF stream are stored in little-endian binary form, but are shown in hexadecimal throughout this section. Значения контрольной суммы — это просто 16-битовые неподписаные значения, которые являются суммой(модулятором 65536) данных, к которые применяется контрольная сумма. Все длины атрибутов — это неподписаные длинные знаки, включая любые завершающие символы null.
  
Ключ — это 16-битное 16-битное неподписамое значение, которое означает начальное значение ссылок на ключи вложения. Эталонные ключи вложений последовательно начинаются с начального значения, которое передается в функцию [OpenTnefStream](opentnefstream.md) поставщиком услуг, который использует TNEF. Поставщик услуг должен создать случайное начальное значение для ключа, чтобы свести к минимуму вероятность того, что два сообщения используют один и тот же ключ. 
  
Реализация TNEF использует идентификаторы атрибутов для связываия атрибутов с соответствующими свойствами MAPI. Идентификатор атрибута — это 32-битное неподписаное значение, которое состоит из двух значений слов. Слово высокого порядка указывает тип данных, например строку или двоичный файл, а слово низкого порядка определяет определенный атрибут. Типы данных в слове высокого порядка:
  
|**Тип**|**Значение**|
|:-----|:-----|
|atpTriples  <br/> |0x0000  <br/> |
|atpString  <br/> |0x0001  <br/> |
|atpText  <br/> |0x0002  <br/> |
|atpDate  <br/> |0x0003  <br/> |
|atpShort  <br/> |0x0004  <br/> |
|atpLong  <br/> |0x0005  <br/> |
|atpByte  <br/> |0x0006  <br/> |
|atpWord  <br/> |0x0007  <br/> |
|atpDword  <br/> |0x0008  <br/> |
|atpMax  <br/> |0x0009  <br/> |
   
Значения слов низкого порядка для каждого атрибута определяются в TNEF. Файл h-загона.
