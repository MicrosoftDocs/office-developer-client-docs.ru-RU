# <a name="tnef-stream-structure"></a>Структура потока TNEF

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Поток TNEF начинается с 32-битной подписи, которая определяет поток как поток TNEF. После подписи следует 16-битный неподписаный компонент, который используется в качестве ключа для перекрестных ссылок вложений к их расположению в помеченном тексте сообщения. Остальная часть потока — это последовательность атрибутов TNEF. Атрибуты сообщений сначала отображаются в потоке TNEF, а затем следуют атрибуты вложения. Атрибуты, которые относятся к определенному вложению, группуются вместе, начиная с **атрибута attAttachRenddata.** 
  
Большинство постоянных значений, используемых в потоках TNEF, определяются в TNEF. Файл заглавной папки H. Примечательно, **TNEF_SIGNATURE,** **LVL_MESSAGE,** **LVL_ATTACHMENT** и все идентификаторы атрибутов TNEF определяются в этом файле. Другие константы имеют значения, указанные их интерпретацией компилятору языка C. Как правило, такие константы используются для придаия размеров следующего элемента. Например, в определении **элемента sizeof (ULONG)** указывается, что в этом месте в потоке TNEF должен происходить integer, представляющий размер следующего неподписаного длинного многоугольник. 
  
Все integers в потоке TNEF хранятся в маленькой двоичной форме, но показаны в hexadecimal во всем этом разделе. Значения Checksum — это просто 16-битные неподписаные наборы, которые являются суммой, modulo 65536, из битов данных, к которые применяется контрольный реестр. Все длины атрибутов — это неподписаные длинные integers, включая любые завершающие null символы.
  
Ключ — это нецерон, 16-битный неподписаный компонент, который означает начальное значение ссылок на вложения. Клавиши ссылки на вложения назначены каждому вложению последовательно, начиная с начального значения, которое передается функции [OpenTnefStream](opentnefstream.md) поставщиком услуг, использующим TNEF. Поставщик служб должен создать случайное начальное значение для ключа, чтобы свести к минимуму вероятность того, что два сообщения используют один и тот же ключ. 
  
Реализация TNEF использует идентификаторы атрибутов для картографии атрибутов с соответствующими свойствами MAPI. Идентификатор атрибута — это 32-битный неподписаный ряд, который состоит из двух значений слов. Слово высокого порядка указывает тип данных, например строку или двоичный, и слово низкого порядка определяет определенный атрибут. Типы данных в слове высокого порядка:
  
|**Тип**|**Значение**|
|:-----|:-----|
|atpTriples  <br/> |0x0000  <br/> |
|atpString  <br/> |0x0001  <br/> |
|atpText  <br/> |0x0002  <br/> |
|atpDate  <br/> |0x0003  <br/> |
|atpShort  <br/> |0x0004  <br/> |
|atpLong  <br/> |0x0005  <br/> |
|atpByte  <br/> |0x0006  <br/> |
|atpWord  <br/> |0x0007  <br/> |
|atpDword  <br/> |0x0008  <br/> |
|atpMax  <br/> |0x0009  <br/> |
   
Значения слов низкого порядка для каждого атрибута определяются в TNEF. Файл заглавной папки H.
