## <a name="overview-of-message-store-providers"></a><span data-ttu-id="1d21d-105">Обзор поставщиков хранилищ сообщений</span><span class="sxs-lookup"><span data-stu-id="1d21d-105">Overview of Message Store Providers</span></span>

<span data-ttu-id="1d21d-106">Поставщики хранилищ сообщений обрабатывают хранение и извлечение сообщений и другие сведения для пользователей клиентских приложений.</span><span class="sxs-lookup"><span data-stu-id="1d21d-106">Message store providers handle the storage and retrieval of messages and other information for the users of client applications.</span></span> <span data-ttu-id="1d21d-107">Сведения о сообщении организованы с помощью иерархической системы, называемой хранилищем сообщений.</span><span class="sxs-lookup"><span data-stu-id="1d21d-107">The message information is organized by using a hierarchical system known as a message store.</span></span> <span data-ttu-id="1d21d-108">Хранилище сообщений реализуется на нескольких уровнях с контейнерами, которые называются папками, в которых хранятся сообщения разных типов.</span><span class="sxs-lookup"><span data-stu-id="1d21d-108">The message store is implemented in multiple levels, with containers called folders that hold messages of different types.</span></span> <span data-ttu-id="1d21d-109">Количество уровней в хранилище сообщений не ограничено; папки могут содержать множество вложенных папок.</span><span class="sxs-lookup"><span data-stu-id="1d21d-109">There is no limit to the number of levels in a message store; folders can contain many subfolders.</span></span>
  
<span data-ttu-id="1d21d-110">Данные хранилища сообщений можно использовать различными способами.</span><span class="sxs-lookup"><span data-stu-id="1d21d-110">Message store data can be used in a variety of ways.</span></span> <span data-ttu-id="1d21d-111">Кроме типичного использования электронной почты, папки можно использовать как форум для общедоступных обсуждений, как репозиторий для справочных документов или как контейнер для сведений о доске объявлений.</span><span class="sxs-lookup"><span data-stu-id="1d21d-111">In addition to the typical email usage, folders can be used as a forum for public discussion, as a repository for reference documents, or as a container for bulletin board information.</span></span> <span data-ttu-id="1d21d-112">Одно хранилище сообщений может содержать множество типов информации, некоторые из них, которые можно изменять.</span><span class="sxs-lookup"><span data-stu-id="1d21d-112">A single message store can hold many types of information, some modifiable and some not.</span></span> <span data-ttu-id="1d21d-113">Несколько клиентов могут устанавливать одно и то же хранилище сообщений, что упрощает и ускоряет совместное использование данных.</span><span class="sxs-lookup"><span data-stu-id="1d21d-113">Multiple clients can install the same message store, making it easy and fast to share data.</span></span>
  
<span data-ttu-id="1d21d-114">Папки хранилища сообщений позволяют сортировать и фильтровать сообщения, а также настраивать представление в пользовательском интерфейсе.</span><span class="sxs-lookup"><span data-stu-id="1d21d-114">Message store folders allow you to sort and filter messages and to customize the view in a user interface (UI) display.</span></span> <span data-ttu-id="1d21d-115">Ссылки на фильтруемые сообщения хранятся в специальных папках, которые называются папками результатов поиска.</span><span class="sxs-lookup"><span data-stu-id="1d21d-115">Links to filtered messages are held in special folders called search-results folders.</span></span> <span data-ttu-id="1d21d-116">Пользователь клиентского приложения вводит условия фильтрации, которые относятся к ограничению, а критерии применяются к сообщениям, хранящимся в одной или нескольких папках.</span><span class="sxs-lookup"><span data-stu-id="1d21d-116">The user of a client application enters filtering criteria, which MAPI refers to as a restriction, and the criteria is applied to the messages stored in one or more folders.</span></span> <span data-ttu-id="1d21d-117">Например, пользователю может потребоваться просмотреть только те сообщения, которые имеют определенную тему с датами прибытия, предшествующими прошлой неделе.</span><span class="sxs-lookup"><span data-stu-id="1d21d-117">For example, a user might want to view only those messages dealing with a particular subject with arrival dates that are more recent than last week.</span></span> <span data-ttu-id="1d21d-118">Ссылки на сообщения, соответствующие условиям, указаны в папке результатов поиска, а реальные сообщения хранятся в обычных папках.</span><span class="sxs-lookup"><span data-stu-id="1d21d-118">References to the messages that match the criteria are listed in the search-results folder and the real messages remain in their regular folders.</span></span>
  
<span data-ttu-id="1d21d-119">Сообщения — это единицы данных, передаваемых от одного пользователя или приложения другому пользователю или приложению.</span><span class="sxs-lookup"><span data-stu-id="1d21d-119">Messages are the units of data transferred from one user or application to another user or application.</span></span> <span data-ttu-id="1d21d-120">Каждое сообщение содержит текст сообщения и сведения о конверте сообщения, которые используются для передачи данных.</span><span class="sxs-lookup"><span data-stu-id="1d21d-120">Every message contains some message text and message envelope information that is used for transmission.</span></span> <span data-ttu-id="1d21d-121">Некоторые сообщения включают одно или несколько вложений, а также дополнительные данные, связанные с сообщением в виде файла, другого сообщения или объекта OLE.</span><span class="sxs-lookup"><span data-stu-id="1d21d-121">Some messages include one or more attachments, or additional data related to and transported with a message in the form of a file, another message, or an OLE object.</span></span>
