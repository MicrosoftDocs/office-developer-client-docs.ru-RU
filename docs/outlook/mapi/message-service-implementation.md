# <a name="message-service-implementation"></a>Реализация службы сообщений

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Служба сообщений — это один или несколько связанных поставщиков служб, сгруппных в целях упрощения установки и настройки. Все поставщики услуг должны быть включены в службу сообщений.
  
Чтобы реализовать службу сообщений с одним или более поставщиками, используйте следующую процедуру:
  
1. Разработать службу сообщений, определив количество и тип поставщиков услуг, которые необходимо включить. Дополнительные сведения о проектировании службы сообщений см. в подпроекте "Разработка [службы сообщений".](designing-a-message-service.md)
    
2. Создайте программу установки для установки поставщиков услуг в службе сообщений. Дополнительные сведения о написании программы установки службы сообщений см. в [подпрограмме "Поддержка установки службы сообщений".](supporting-message-service-installation.md) 
    
3. Создайте функцию точки входа для выполнения настройки. Дополнительные сведения о написании функции точки входа службы сообщений см. в вспомогательных настройках службы сообщений [и](supporting-message-service-configuration.md) [MSGSERVICEENTRY.](msgserviceentry.md) 
    
4. Создайте файл открытого загона, содержащий теги свойств и описания допустимого значения для любых настраиваемого свойства, поддерживаемого службой сообщений.
