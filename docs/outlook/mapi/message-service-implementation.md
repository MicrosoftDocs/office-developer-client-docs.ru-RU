# <a name="message-service-implementation"></a>Реализация службы сообщений

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Служба сообщений — это один или несколько связанных поставщиков услуг, сгруппив их для упрощения установки и конфигурации. Все поставщики услуг должны быть включены в службу сообщений.
  
Чтобы реализовать службу сообщений с одним или более поставщиками, используйте следующую процедуру:
  
1. Разработка службы сообщений, определение числа и типа поставщиков услуг, которые будут включены. Дополнительные сведения о разработке службы сообщений см. в [сообщении Designing a Message Service.](designing-a-message-service.md)
    
2. Создайте программу установки для установки поставщиков услуг в службе сообщений. Дополнительные сведения о написании программы установки службы сообщений см. в [статьи Поддержка установки службы сообщений.](supporting-message-service-installation.md) 
    
3. Создайте функцию точки входа для выполнения конфигурации. Дополнительные сведения о написании функции точки входа службы сообщений см. в [статьи Поддержка](supporting-message-service-configuration.md) конфигурации службы сообщений и [MSGSERVICEENTRY.](msgserviceentry.md) 
    
4. Создайте общедоступный файл загона, содержащий теги свойств и описания допустимого значения для любых настраиваемого свойства, поддерживаемого службой сообщений.
