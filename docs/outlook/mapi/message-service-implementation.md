# <a name="message-service-implementation"></a>Реализация службы сообщений

  
  
**Относится к**: Outlook 
  
Служба сообщений — один или несколько поставщиков услуг, связанных с ними, сгруппированная для упрощения процесса установки и настройки. Все поставщики услуг должно быть включено в службе сообщений.
  
Для реализации службы сообщений с помощью одного или нескольких поставщиков, используйте следующую процедуру:
  
1. Разработка служба сообщений, определение числа и типа поставщиков услуг, которые требуется включить. Дополнительные сведения о создании службы сообщений [Службы сообщений](designing-a-message-service.md)см.
    
2. Создайте программу установки для поставщиков услуг службы сообщений. Дополнительные сведения о записи программы установки службы сообщений можно [Поддержки установки службы сообщений](supporting-message-service-installation.md). 
    
3. Создайте функцию точки входа для выполнения конфигурации. Дополнительные сведения о создании записи службы сообщений точки функция см [MSGSERVICEENTRY](msgserviceentry.md)и [Поддержки конфигурации службы сообщений](supporting-message-service-configuration.md) . 
    
4. Создайте файл общедоступных заголовка, который содержит свойство теги и описания допустимых значений для все настраиваемые свойства, которые поддерживает службы сообщений.
