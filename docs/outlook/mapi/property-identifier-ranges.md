# <a name="property-identifier-ranges"></a>Диапазоны идентификаторов свойств

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
В следующей таблице описываются различные диапазоны идентификаторов свойств, описывающие владельца свойств в каждом диапазоне.
  
|**Диапазон идентификаторов**|**Описание**|
|:-----|:-----|
|0000  <br/> |Зарезервировано MAPI для специального **значения PR_NULL**.  <br/> |
|0001 - 0BFF  <br/> |Свойства конверта сообщений, определенные MAPI.  <br/> |
|0C00 - 0DFF  <br/> |Свойства получателей, определенные MAPI.  <br/> |
|0E00 - 0FFF  <br/> |Свойства непередаваемого сообщения, определенные MAPI.  <br/> |
|1000 - 2FFF  <br/> |Свойства контента сообщений, определенные MAPI.  <br/> |
|3000 — 3FFF  <br/> |Свойства для объектов, помимо сообщений и получателей, определенных MAPI.  <br/> |
|4000 - 57FF  <br/> |Свойства конверта сообщений, определенные поставщиками транспорта.  <br/> |
|5800 - 5FFF  <br/> |Свойства получателей, определенные поставщиками транспортных и адресных книг.  <br/> |
|6000 - 65FF  <br/> |Свойства непередаваемого сообщения, определенные клиентами.  <br/> |
|6600 - 67FF  <br/> |Свойства, которые не передаются, определенные поставщиком услуг. Эти свойства могут быть видимыми или невидимыми для пользователей.  <br/> |
|6800 - 7BFF  <br/> |Свойства контента сообщений для пользовательских классов сообщений, определенных создателями этих классов.  <br/> |
|7C00 - 7FFF  <br/> |Непередаваемые свойства для настраиваемых классов сообщений, определенных создателями этих классов.  <br/> |
|8000 - FFFE  <br/> |Свойства, определенные клиентами и иногда поставщиками услуг, которые определяются по имени с помощью [методов IMAPIProp::GetNamesFromIDs](imapiprop-getnamesfromids.md) и [IMAPIProp::GetIDsFromNames.](imapiprop-getidsfromnames.md)  <br/> |
|FFFF  <br/> |Зарезервировано MAPI для специального значения ошибки PROP_ID_INVALID.  <br/> |
   
Диапазон между 3000 и 3FFF зарезервирован для свойств, не связанных ни с сообщениями, ни с получателями. MAPI делит этот диапазон на подстанцию по типам объектов; В следующей таблице показана следующая разбивка. 
  
|**Диапазон идентификаторов**|**Тип свойства**|
|:-----|:-----|
|3000 - 33FF  <br/> |Общие свойства, которые отображаются на нескольких объектах, таких как **PR_DISPLAY_NAME** и **PR_ENTRYID.**  <br/> |
|3400 - 35FF  <br/> |Свойства магазина сообщений  <br/> |
|3600 - 36FF  <br/> |Свойства контейнера папки и адресной книги  <br/> |
|3700 - 38FF  <br/> |Свойства вложения  <br/> |
|3900 - 39FF  <br/> |Свойства адресной книги  <br/> |
|3A00 - 3BFF  <br/> |Свойства пользователей обмена сообщениями  <br/> |
|3C00 - 3CFF  <br/> |Свойства списка рассылки  <br/> |
|3D00 - 3DFF  <br/> |Свойства профиля  <br/> |
|3E00 - 3FFF  <br/> |Свойства объекта status  <br/> |
