# <a name="property-identifier-ranges"></a>Диапазоны идентификаторов свойств

  
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
В следующей таблице приведены различные диапазонов для идентификаторов свойств, описывающее владельца для свойств в каждом диапазоне.
  
|**Идентификатор диапазона**|**Описание**|
|:-----|:-----|
|0000  <br/> |Зарезервировано для MAPI для особое значение **PR_NULL**.  <br/> |
|0001 - 0BFF  <br/> |Свойства конверт сообщения, определенные в MAPI.  <br/> |
|0C 00 - 0DFF  <br/> |Получателей свойств, определенных в MAPI.  <br/> |
|0E00 - 0FFF  <br/> |Свойства не передаваемого сообщения, определенные в MAPI.  <br/> |
|1000 - 2FFF  <br/> |Сообщение контента свойств, определенных в MAPI.  <br/> |
|3000 - 3FFF  <br/> |Свойства для объектов, отличные от сообщений и получателей, определенных для MAPI.  <br/> |
|4000 - 57FF  <br/> |Свойства конверт сообщения, определенные поставщиками транспорта.  <br/> |
|5800 - 5FFF  <br/> |Получателей свойств, определенных в транспортный сервер и адресной книги поставщиков.  <br/> |
|6000 - 65FF  <br/> |Свойства не передаваемого сообщения, определенные клиентами.  <br/> |
|6600 - 67FF  <br/> |Не передаваемого свойств, определенных поставщиком услуг. Эти свойства могут быть видимой или невидимой для пользователей.  <br/> |
|6800 - 7BFF  <br/> |Свойства содержимого сообщения для пользовательских классов сообщений определенные создателями классов.  <br/> |
|7C 00 - 7FFF  <br/> |Не передаваемого свойства для пользовательских классов сообщений определенные создателями классов.  <br/> |
|8000 - FFFE  <br/> |Свойства иногда службы поставщиков, которые определяются по имени с помощью методов [IMAPIProp::GetNamesFromIDs](imapiprop-getnamesfromids.md) и [IMAPIProp::GetIDsFromNames](imapiprop-getidsfromnames.md) и определенных клиентами.  <br/> |
|FFFF  <br/> |Зарезервировано для MAPI для PROP_ID_INVALID значение специальные ошибки.  <br/> |
   
Диапазон между 3000 и 3FFF зарезервирован для свойств, которые не связаны с сообщения или получателей. MAPI делит этот диапазон в подменю диапазонов на тип объектов; в следующей таблице показаны в этом тщательного анализа. 
  
|**Идентификатор диапазона**|**Тип свойства**|
|:-----|:-----|
|3000 - 33FF  <br/> |Общие свойства, отображаемые на несколько объектов, таких как **PR_DISPLAY_NAME** и **PR_ENTRYID**.  <br/> |
|3400 - 35FF  <br/> |Свойства хранилища сообщений  <br/> |
|3600 - 36FF  <br/> |Папка и адресной книги контейнер свойств  <br/> |
|3700 - 38FF  <br/> |Свойства вложения  <br/> |
|3900 - 39FF  <br/> |Свойств адресной книги  <br/> |
|3A00 - 3BFF  <br/> |Система обмена сообщениями свойств пользователя  <br/> |
|3C 00 - 3CFF  <br/> |Свойства списка рассылки  <br/> |
|3D 00 - 3DFF  <br/> |Свойства профиля  <br/> |
|3E00 - 3FFF  <br/> |Свойства объекта состояния  <br/> |
