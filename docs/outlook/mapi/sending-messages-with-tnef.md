# <a name="sending-messages-with-tnef"></a>Отправка сообщений с помощью TNEF

  
  
**Относится к**: Outlook 
  
Многие поставщики транспорта автоматически отправлять все исходящие сообщения с транспорта Neutral Encapsulation формата TNEF. Формат TNEF используется для передачи форматированный текст, который поддерживает множество клиентов и поставщиков хранилища сообщений в их сообщения, вложения из различных типов и настраиваемые свойства для пользовательских классов сообщений. Хотя режим по умолчанию для большинства поставщиков транспорта на отправку сообщений с TNEF, некоторые поставщики транспорта поддерживает его. Отсутствие поддержки формата TNEF проблема не возникает для стандартного обмена сообщениями клиентов, которые отправляют и получают сообщения IPM. Тем не менее для клиентов на основе форм или клиентов, которым требуются настраиваемые свойства, важно использование TNEF. Конструкторы клиентов, основанные на формах или настраиваемые свойства необходимо иметь представление о возможностях поставщиками транспорта, которые они используют.
  
Получателей сообщения можно управлять ли поставщика транспорта передает сообщения с TNEF путем установки свойства **PR_SEND_RICH_INFO** . Для получения дополнительных сведений см **PR_SEND_RICH_INFO** ([PidTagSendRichInfo](pidtagsendrichinfo-canonical-property.md)). Когда получатель **PR_SEND_RICH_INFO** задано значение TRUE, поставщика транспорта, поддерживающего TNEF передает его с сообщением. Если задано значение FALSE, форматирование удаляются. Когда **PR_SEND_RICH_INFO** не существует, это до поставщика транспорта выбор по умолчанию порядок действий. 
  
При создании настраиваемых получателя, клиентов и поставщиков услуг, они могут повлиять на значение свойства **PR_SEND_RICH_INFO** , передав флаг MAPI_SEND_NO_RICH_INFO с помощью параметра _ulFlags_ **IAddrBook::CreateOneOff** или ** IMAPISupport::CreateOneOff** вызова. Для получения дополнительных сведений см [IAddrBook::CreateOneOff](iaddrbook-createoneoff.md) и [IMAPISupport::CreateOneOff](imapisupport-createoneoff.md). Передача MAPI_SEND_NO_RICH_INFO вызывает MAPI для настраиваемых получатель свойства **PR_SEND_RICH_INFO** значение FALSE; в большинстве случаев не передав флаг вызывает MAPI для свойства значение TRUE. Одно исключение — если адрес получателя настраиваемых интерпретируется быть адрес в Интернете. В этом случае один MAPI задает **PR_SEND_RICH_INFO** значение FALSE.
