# <a name="sending-messages-with-tnef"></a>Отправка сообщений с помощью TNEF

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Многие поставщики транспорта автоматически отправляют все исходяющие сообщения с помощью формата инкапсуляции с нейтральным транспортом (TNEF). TNEF используется для передачи отформатированный текст, поддерживаемый многими клиентами и поставщиками сообщений в сообщениях, вложениях различных типов и настраиваемые свойства для настраиваемого класса сообщений. Хотя для большинства поставщиков транспорта по умолчанию существует режим отправки исходяющих сообщений с помощью TNEF, некоторые поставщики транспорта не поддерживают его. Отсутствие поддержки TNEF не является проблемой для стандартных клиентов обмена сообщениями, которые отправляют и получают сообщения IPM. Однако для клиентов или клиентов на основе форм, которые требуют настраиваемого свойства, использование TNEF имеет важное значение. Дизайнеры клиентов, которые полагаются на формы или настраиваемые свойства, должны быть осведомлены о возможностях транспортных поставщиков, которые они используют.
  
Получатели сообщений могут контролировать, передает ли поставщик транспорта сообщения с помощью TNEF, **PR_SEND_RICH_INFO** свойства. Дополнительные сведения см. **в PR_SEND_RICH_INFO** [(PidTagSendRichInfo).](pidtagsendrichinfo-canonical-property.md) Если свойство PR_SEND_RICH_INFO  установлено true, поставщик транспорта, который поддерживает TNEF, передает его с сообщением. Если свойство настроено на FALSE, форматирование отбрасывается. Если **PR_SEND_RICH_INFO** не существует, поставщик транспорта должен выбрать курс действий по умолчанию. 
  
Когда клиенты и поставщики услуг создают настраиваемого  получателя, они могут повлиять на значение его свойства PR_SEND_RICH_INFO, передав флаг MAPI_SEND_NO_RICH_INFO в параметре _ulFlags_ на **вызов IAddrBook::CreateOneOff** или **IMAPISupport::CreateOneOff.** Дополнительные сведения см. [в книге IAddrBook::CreateOneOff](iaddrbook-createoneoff.md) и [IMAPISupport::CreateOneOff](imapisupport-createoneoff.md). Передача MAPI_SEND_NO_RICH_INFO заставляет MAPI устанавливать свойство пользовательского **PR_SEND_RICH_INFO** false; в большинстве случаев не проход флага вызывает MAPI, чтобы установить свойство TRUE. Исключением является, если пользовательский адрес получателя интерпретируется как интернет-адрес. В этой ситуации MAPI задает PR_SEND_RICH_INFO **false.**
