# <a name="processing-a-sent-message"></a>Обработка отправленного сообщения

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
ИсХодящие сообщения, после их отправки, можно оставить в папке "исХодящие", переместить в папку, предназначенную для хранения отправленных сообщений, или удалить. Тип обработки зависит от того, настроены ли свойства хранилища сообщений:
  
- **Пр_делете_афтер_субмит** ([PidTagDeleteAfterSubmit](pidtagdeleteaftersubmit-canonical-property.md)) 
    
- **Пр_сентмаил_ентрид** ([PidTagSentMailEntryId](pidtagsentmailentryid-canonical-property.md)) 
    
 **Пр_делете_афтер_субмит** является логическим свойством, имеет значение true, если после отправки сообщения необходимо удалить, а в противном случае — значение false. **Пр_сентмаил_ентрид** — идентификатор элемента папки. Если это свойство задано, вы должны переместить отправленные сообщения в папку, представленную этим идентификатором записи. Сохраненные сообщения обычно имеют идентификатор последнего хранилища сообщений или поставщика транспорта для их отправки. 
  
Не должно быть задано ни одно из этих свойств либо ни то, ни другое. Тем не менее, если вы настроили **пр_сентмаил_ентрид**, он должен содержать допустимый идентификатор записи. 
  
В следующей таблице показано, как эти свойства влияют на действия, выполняемые с отправленными сообщениями.
  
|||
|:-----|:-----|
|Если не задано ни одно свойство:  <br/> |Оставьте сообщение в папке, из которой оно было отправлено (обычно это папка "исХодящая").  <br/> |
|Если задано значение **пр_сентмаил_ентрид** :  <br/> |Переместить сообщение в указанную папку.  <br/> |
|Если задано значение **пр_делете_афтер_субмит** :  <br/> |Удаление сообщения.  <br/> |
