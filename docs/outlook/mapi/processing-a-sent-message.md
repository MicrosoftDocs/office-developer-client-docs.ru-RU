# <a name="processing-a-sent-message"></a>Обработка отправленного сообщения

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Исходящие сообщения после их передачи можно оставить в папке "Исходящие", переложить в папку, предназначенную для удержания отправленных сообщений, или удалить. Тип обработки зависит от того, установили ли вы свойства магазина сообщений:
  
- **PR_DELETE_AFTER_SUBMIT** [(PidTagDeleteAfterSubmit)](pidtagdeleteaftersubmit-canonical-property.md) 
    
- **PR_SENTMAIL_ENTRYID** [(PidTagSentMailEntryId)](pidtagsentmailentryid-canonical-property.md) 
    
 **PR_DELETE_AFTER_SUBMIT** является свойством Boolean, заданным true, если сообщения должны быть удалены после их удаления, и FALSE в противном случае. **PR_SENTMAIL_ENTRYID** является идентификатором записи папки. Когда это свойство заданной, следует переместить отправленные сообщения в папку, представленную этим идентификатором записи. Сохраненные сообщения обычно имеют удостоверение последнего магазина сообщений или поставщика транспорта для их отправки. 
  
Либо одно, либо другое, либо ни одно из этих свойств не должно быть за установлено, но не оба. Однако, если вы **PR_SENTMAIL_ENTRYID,** он должен содержать допустимый идентификатор записи. 
  
В следующей таблице описывается, как эти свойства влияют на работу с отправленными сообщениями.
  
|||
|:-----|:-----|
|Если не установлено ни одно свойство:  <br/> |Оставьте сообщение в папке, из которой оно было отправлено (как правило, в папке Outbox).  <br/> |
|Если **PR_SENTMAIL_ENTRYID** установлено:  <br/> |Перемещение сообщения в указанную папку.  <br/> |
|Если **PR_DELETE_AFTER_SUBMIT** установлено:  <br/> |Удаление сообщения.  <br/> |
