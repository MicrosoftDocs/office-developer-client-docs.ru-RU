# <a name="processing-a-sent-message"></a>Обработка отправленного сообщения

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Исходящие сообщения после их передачи можно оставить в папке "Исходящие", в папку, предназначенную для удержания отправленных сообщений, или удалить. Тип обработки зависит от того, установлены ли свойства хранения сообщений:
  
- **PR_DELETE_AFTER_SUBMIT** ([PidTagDeleteAfterSubmit)](pidtagdeleteaftersubmit-canonical-property.md) 
    
- **PR_SENTMAIL_ENTRYID** ([PidTagSentMailEntryId)](pidtagsentmailentryid-canonical-property.md) 
    
 **PR_DELETE_AFTER_SUBMIT** является boolean свойством, заданным как TRUE, если сообщения должны быть удалены после их и в противном случае FALSE. **PR_SENTMAIL_ENTRYID** является идентификатором записи папки. Если за установлено это свойство, необходимо переместить отправленные сообщения в папку, представленную идентификатором этой записи. Сохраненные сообщения обычно имеют удостоверение последнего поставщика личных сообщений или поставщика транспорта для их отправки. 
  
Либо одно, либо другое, либо ни одно из этих свойств не должно быть установлено, но не оба. Однако если вы **PR_SENTMAIL_ENTRYID,** он должен содержать допустимый идентификатор записи. 
  
В следующей таблице описано, как эти свойства влияют на работу с отправленными сообщениями.
  
|||
|:-----|:-----|
|Если ни одно из свойств не за установлено:  <br/> |Оставьте сообщение в папке, из которой оно было отправлено (как правило, в папке "Из папки").  <br/> |
|Если **PR_SENTMAIL_ENTRYID** установлено:  <br/> |Переместит сообщение в указанную папку.  <br/> |
|Если **PR_DELETE_AFTER_SUBMIT** установлено:  <br/> |Удалите сообщение.  <br/> |
