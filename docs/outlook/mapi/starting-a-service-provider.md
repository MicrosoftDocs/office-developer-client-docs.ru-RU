# <a name="starting-a-service-provider"></a>Запуск поставщика услуг

 
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
В определенный момент после того, как клиент начнет сеанс с ПОМОЩЬЮ MAPI, будет запущен ваш поставщик услуг. Поставщики транспорта запущены, когда клиент запрашивает свои службы. Поставщики адресной книги и магазина сообщений запущены во время процесса работы клиента.
  
Клиент вызывает [IMAPISession::OpenAddressBook,](imapisession-openaddressbook.md) чтобы загрузить каждого поставщика адресной книги, включенных в профиль и [IMAPISession::OpenMsgStore,](imapisession-openmsgstore.md) для загрузки определенного поставщика хранилищ сообщений. Поставщики адресных книг, которые являются частью службы сообщений, запущены перед любым из других поставщиков в службе. 
  
MAPI запускает каждого поставщика услуг в активном профиле, выступая следующим образом:
  
- Поиск имени DLL в профиле. Необходимо зарегистрировать имя DLL поставщика в файле конфигурации Mapisvc.inf, чтобы убедиться, что оно отображается в профиле. Когда поставщик услуг добавляется в профиль по отдельности или как часть службы сообщений, все разделы **[Поставщик услуг]** из Mapisvc.inf, применимые к поставщику, копируется в профиль. Дополнительные сведения о структуре Mapisvc.inf см. в формате файла [MapiSvc.inf.](file-format-of-mapisvc-inf.md)
    
- Вызов функции API Windows **LoadLibrary** для загрузки DLL. Так как MAPI вызывает **LoadLibrary** каждый раз, когда использует DLL поставщика услуг (независимо от того, загружена ли она) или только в первый раз, поставщик услуг не должен делать предположений о том, сколько раз он будет загружен. Для каждого вызова **LoadLibrary** MAPI совершает вызов функции API Windows **FreeLibrary,** когда DLL поставщика услуг больше не требуется. 
    
- Вызов функции точки входа для поставщика. MAPI вызывает функцию точки входа поставщика для инициации процесса входа. Функции точки входа гарантируют использование версии интерфейса поставщика услуг (SPI), совместимой с версией, используемой MAPI. Эти функции также возвращают указатели на вновь созданные объекты поставщика. Дополнительные сведения о создании функции точки входа для поставщика см. в реализации функции точки [входа поставщика услуг.](implementing-a-service-provider-entry-point-function.md)
