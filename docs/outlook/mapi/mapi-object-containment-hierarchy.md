# <a name="mapi-object-containment-hierarchy"></a>Иерархия сдерживания объектов MAPI
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Связь сдерживания между объектами указывает зависимости, которые есть у некоторых объектов для доступа к другим объектам. Для клиентского приложения доступ к определенным объектам позволяет получить доступ к другим. В некоторых случаях связь сдерживания между объектами, реализуемая поставщиком услуг, следует логической иерархии. В других случаях это произвольно. 
  
Клиент должен получить доступ к объекту сеанса MAPI, прежде чем он сможет использовать многие другие объекты (например, поставщики услуг и адресная книга MAPI).
  
Сдерживание магазина сообщений основано на иерархической связи между объектами в хранилище сообщений: самим объектом магазина сообщений, папок, сообщений и вложений. Логически вложения содержатся в сообщениях, сообщениях в папках и папках в хранилище сообщений. Связь с сдерживанием соответствует этой логической иерархии. Например, чтобы получить доступ к сообщению, клиент должен сначала получить доступ к папке, в которой содержится сообщение. Профили и объекты состояния являются примерами более произвольного отношения сдерживания. Оба этих объекта доступны в ходе сеанса. 
  
С некоторыми объектами контейнеры предоставляют единственный доступ. Вложения и получатели — это примеры объектов, полностью зависящих от контейнеров. Единственный доступ к вложениям или получателю — это сообщение, которому оно принадлежит. Другие объекты имеют альтернативные пути доступа. Эти объекты назначены двоичными идентификаторами, известными как идентификаторы записей, создателями служб. Идентификаторы записей можно использовать для непосредственного доступа к своим объектам, что позволяет клиентам обойти дерево сдерживания. 
  
На следующем рисунке показана иерархия сдерживания MAPI. Сеанс находится в верхней части дерева, так как именно через сеанс клиент получает доступ ко всем другим объектам. Следующий уровень включает в себя таблицу хранения сообщений, объект таблицы, в который перечислены свойства для всех поставщиков магазина сообщений в текущем сеансе, и адресную книгу для обеспечения доступа ко всем поставщикам адресной книги. Таблица хранения сообщений и адресная книга используются для доступа к объектам, реализованным определенными поставщиками услуг, показанным далее, в порядке сдерживания.
