# <a name="mapi-object-containment-hierarchy"></a>Иерархия сужению объектов MAPI
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Отношение совмества между объектами определяет зависимости, которые некоторые объекты имеют от других объектов для доступа. Для клиентского приложения доступ к определенным объектам обеспечивает доступ к другим. В некоторых случаях отношение совмества между объектами, реализованными поставщиком услуг, следует логической иерархии. В других случаях это произвольно. 
  
Клиент должен получить доступ к объекту сеанса MAPI, прежде чем использовать множество других объектов (например, поставщиков услуг и адресную книгу MAPI).
  
Хранение сообщений основано на иерархической связи между объектами в хранилище сообщений: сам объект хранения сообщений, папки, сообщения и вложения. Логически вложения содержатся в сообщениях, сообщениях в папках и папках в хранилище сообщений. Отношение с соединилось этой логической иерархии. Например, чтобы получить доступ к сообщению, клиент должен сначала получить доступ к папке, в которой находится сообщение. Профили и объекты состояния являются примерами более произвольных отношений с сопроивлением. Оба этих объекта доступны в сеансе. 
  
С некоторыми объектами контейнеры предоставляют единственный доступ. Вложения и получатели — это примеры объектов, полностью зависящих от их контейнеров. Единственный доступ к вложению или получателю — через сообщение, к которому оно принадлежит. Другие объекты имеют альтернативные пути доступа. Эти объекты создаются поставщиками служб, которые их создают. Идентификаторы записей можно использовать для прямого доступа к своим объектам, позволяя клиентам обходить дерево блокировки. 
  
На следующем рисунке показана иерархия реализации MAPI. Сеанс находится в верхней части дерева, так как клиент получает доступ ко всем другим объектам через сеанс. На следующем уровне содержится таблица хранения сообщений, объект таблицы, включающая свойства всех поставщиков службы хранения сообщений в текущем сеансе, а также адресная книга для доступа ко всем поставщикам адресных книг. Таблица и адресная книга для хранения сообщений используются для доступа к объектам, реализованным определенными поставщиками услуг, как показано далее, в порядке хранения.
