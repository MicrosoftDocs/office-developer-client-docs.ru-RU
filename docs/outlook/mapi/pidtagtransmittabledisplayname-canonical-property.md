# <a name="pidtagtransmittabledisplayname-canonical-property"></a>Каноническое свойство PidTagTransmittableDisplayName

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит отображаемое имя получателя в защищенной форме, которое не может быть изменено.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |ПР_ТРАНСМИТАБЛЕ_ДИСПЛАЙ_НАМЕ, ПР_ТРАНСМИТАБЛЕ_ДИСПЛАЙ_НАМЕ_А, ПР_ТРАНСМИТАБЛЕ_ДИСПЛАЙ_НАМЕ_В  <br/> |
|Идентификатор:  <br/> |0x3A20  <br/> |
|Тип данных:  <br/> |ПТ_УНИКОДЕ, PT_STRING8  <br/> |
|Область:  <br/> |Address  <br/> |
   
## <a name="remarks"></a>Комментарии

Эти свойства должны быть реализованы всеми поставщиками адресных книг. Они содержат версию отображаемого имени получателя, которая передается вместе с сообщением. Для большинства поставщиков адресных книг эти свойства имеют то же значение, что и свойство **пр_дисплай_наме** ([PidTagDisplayName](pidtagdisplayname-canonical-property.md)). Поставщики, не имеющие безопасного отображаемого имени, возвращают ПТ_ЕРРОР, а MAPI изменяет отображаемое имя, добавляя кавычки вокруг имени.
  
Клиентское приложение может использовать это свойство для запрета изменения или подмены записей. Пример подмены имени Иван Петров (специалиста) Петров.
