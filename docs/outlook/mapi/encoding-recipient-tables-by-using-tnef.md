# <a name="encoding-recipient-tables-by-using-tnef"></a>Кодирование таблиц получателей с помощью TNEF

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Кодирование таблицы получателей в потоке TNEF редко требуется, так как большинство систем обмена сообщениями напрямую поддерживают списки получателей. В общем случае свойства получателей передаются в заголовке сообщения. При включении таблицы получателей TNEF может кодировать таблицу получателей в рамках обычной обработки. Это выполняется на начальном этапе обработки TNEF. Поставщик транспорта может включать таблицу получателей сообщения, вызывая метод [итнеф:: аддпропс](itnef-addprops.md) со свойством **пр_мессаже_реЦипиентс** ([PidTagMessageRecipients](pidtagmessagerecipients-canonical-property.md)), указанным в списке включений. TNEF получает таблицу получателей из сообщения, запрашивает набор столбцов и обрабатывает каждую строку таблицы в поток TNEF.
  
Альтернативный метод доступен, если поставщик транспорта должен изменить таблицу получателей, прежде чем она будет закодирована. Поставщик транспорта может создать необходимую таблицу и затем вызвать метод [итнеф:: енкодереЦипс](itnef-encoderecips.md) . Если в параметре _лпреЦиптабле_ переДАЕТСЯ значение null, то таблица получателей берется непосредственно из сообщения, как описано в **Итнеф:: аддпропс**.
