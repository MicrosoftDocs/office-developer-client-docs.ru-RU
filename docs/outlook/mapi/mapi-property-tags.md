# <a name="mapi-property-tags"></a>Теги свойств MAPI
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Тег свойства — это 32-битный номер, содержащий уникальный идентификатор свойства в битах от 16 до 31 и тип свойства в битах от 0 до 15, как показано на следующем рисунке. 
  
**Элементы тега свойства**
  
![Элементы тега свойства Элементы](media/amapi_10.gif "тегов Свойства")
  
Теги свойств используются для определения свойств MAPI, и каждое свойство должно иметь одно, независимо от того, определяется ли свойство MAPI, клиентом или поставщиком услуг. MAPI определяет набор констант тегов свойств для его свойств в файле загона Mapitags.h; эти свойства называются "свойствами, определенными mapI". 
  
Константы тегов свойств следуют конвенции именования для последовательности и простоты использования. В названии каждого тега свойств есть две части: префикс PR_ и одна или несколько строк символов, описывающие содержимое свойства. Несколько строк символов разделены подчеркнутами. Например, тег свойства для типа адреса получателя сообщения — PR **\_ ADDRTYPE** [(PidTagOrgAddrtype),](https://msdn.microsoft.com/library/d40b5707-e4d5-4746-88d4-8616a3789789%28Office.15%29.aspx)а идентификатором входа для папки, назначенной для получения копии каждого исходящие сообщения, является **PR_IPM_SENTMAIL_ENTRYID** [(PidTagIpmSentMailEntryId).](pidtagipmsentmailentryid-canonical-property.md)
  
Для работы с тегами свойств доступны несколько макрос, среди которых [PROP_TYPE,](prop_type.md) [PROP_ID](prop_id.md)и [PROP_TAG.](prop_tag.md) **PROP \_ TYPE** извлекает тип свойства из тега свойства; **PROP \_ Идентификатор извлекает** идентификатор. **PROP_TAG** создает тег свойства из типа свойства и идентификатора.
