# <a name="dismissmodeless"></a>DISMISSMODELESS

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Определяет функцию обратного вызова, которая вызывает MAPI при закрытии диалогового окна с немодальной адресной книгой. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |MAPIDEFS. h  <br/> |
|Определенная функция реализована следующим образом:  <br/> |Клиентские приложения  <br/> |
|Определенная функция, вызываемая:  <br/> |MAPI  <br/> |
   
```cpp
void (STDMETHODCALLTYPE DISMISSMODELESS)(
  ULONG_PTR ulUIParam,
  LPVOID lpvContext
);
```

## <a name="parameters"></a>Параметры

 _улуипарам_
  
> возврата Зависящее от реализации значение, которое обычно используется для передачи сведений о пользовательском интерфейсе в функцию. Например, в Microsoft Windows этот параметр является дескриптором родительского окна для диалогового окна и имеет тип HWND, приведенный к **ULONG_PTR**. Нулевое значение указывает, что родительское окно отсутствует. 
    
 _лпвконтекст_
  
> возврата Указатель на произвольное значение, переданное функции обратного вызова при ее вызове MAPI. Это значение может представлять собой адрес, значимый клиентскому приложению. Как правило, для кода C++ _лпвконтекст_ является указателем на адрес экземпляра объекта c++. 
    
## <a name="return-value"></a>Возвращаемое значение

Нет
  
## <a name="remarks"></a>Примечания

Когда клиентское приложение вызывает немодальное диалоговое окно адресной книги, оно включает в себя цикл обработки сообщений Windows, который вызывает функцию на основе прототипа [акцелератеабсди](accelerateabsdi.md) , который проверяет и обрабатывает сочетания клавиш. Когда диалоговое окно закрывается, MAPI вызывает функцию на основе **дисмиссмоделесс** , чтобы клиентское приложение не вызывало вызов функции на основе **акцелератеабсди** .
