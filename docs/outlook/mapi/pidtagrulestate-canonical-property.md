# <a name="pidtagrulestate-canonical-property"></a>Каноническое свойство PidTagRuleState

  
  
**Относится к**: Outlook 
  
Значение, которые интерпретируются как битовая маска сочетание флаги, определяющие состояние правила.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_RULE_STATE  <br/> |
|Идентификатор:  <br/> |0x6677  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Правила со стороны сервера  <br/> |
   
## <a name="remarks"></a>Замечания

Следующая таблица определяет возможные значения этого свойства.
  
EN (ST_ENABLED, битовая маска 0x00000001)
  
> Правило будет включено для выполнения. Если этот флаг не установлен, сервер должен пропускать это правило при оценке правила.
    
Аварийного восстановления (st_Сообщения об, битовая маска 0x00000002)
  
> Сервер обнаружил ошибку при обработке правила.
    
ИЗ (ST_ONLY_WHEN_OOF, битовая маска 0x00000004)
  
> Правило выполняется только в том случае, когда пользователь устанавливает состояние об отсутствии на работе Office (OOF) для почтового ящика. Этот флаг не должен задаваться в правиле общей папки.
    
Высокая (ST_KEEP_OOF_HIST, битовая маска 0x00000008)
  
> Этот флаг не должен задаваться в правиле общей папки.
    
EL (ST_EXIT_LEVEL, битовая маска 0x00000010)
  
> Правило оценки приведет к сбросу после выполнения этого правила, за исключением оценки правил об отсутствии на работе.
    
Вероятности нежелательной почты (ST_SKIP_IF_SCL_IS_SAFE, битовая маска 0x00000020)
  
> Ознакомительная версия этого правила могут быть пропущены.
    
Среда Предустановки (ST_RULE_PARSE_ERROR, битовая маска 0x00000040)
  
> Сервер обнаружил ошибку синтаксического анализа данных правил, заданных клиентом.
    
X 
  
> Не используется, этот протокол. Этот бит не должно изменяться клиентом.
    
Обратите внимание на взаимодействие между ST_ONLY_WHEN_OOF и ST_EXIT_LEVEL флаги: 
  
Когда установлено состояние «Нет на работе» для почтового ящика и правила условие имеет значение TRUE, 
  
И:
  
- Правило имеет флаг ST_EXIT_LEVEL и не имеет флаг ST_ONLY_WHEN_OOF. Затем, сервер не должен оценить последующих правил, у которых нет ST_ONLY_WHEN_OOF флаг, set и необходимо оценить последующих правил, которые имеют ST_ONLY_WHEN_OOF флаг.
    
OR:
  
- Данное правило соответствует ST_EXIT_LEVEL и ST_ONLY_WHEN_OOF набора флагов. Затем сервер не должен оценить все последующие правила.
