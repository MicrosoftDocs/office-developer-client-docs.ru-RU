# <a name="pidtagrulestate-canonical-property"></a>Каноническое свойство PidTagRuleState

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Значение, которое интерпретируется как битовая маска флагов, указывающих состояние правила.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |ПР_РУЛЕ_СТАТЕ  <br/> |
|Идентификатор:  <br/> |0x6677  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Правила на стороне сервера  <br/> |
   
## <a name="remarks"></a>Комментарии

В следующей таблице определены возможные значения этого свойства.
  
EN (СТ_ЕНАБЛЕД, битовая маска 0x00000001)
  
> Правило включается для выполнения. Если этот флаг не установлен, сервер должен пропускать это правило при оценке правил.
    
ER (СТ_ЕРРОР, битовая маска 0x00000002)
  
> Сервер обнаружил ошибку при обработке правила.
    
OF (СТ_ОНЛИ_ВХЕН_УФ, битовая маска 0x00000004)
  
> Правило выполняется только в том случае, если пользователь настраивает в почтовом ящике состояние "нет на месте" (отсутствие на работе). Этот флаг не должен задаваться в правиле общедоступных папок.
    
HI (СТ_КИП_УФ_ХИСТ, битовая маска 0x00000008)
  
> Этот флаг не должен задаваться в правиле общедоступных папок.
    
EL (СТ_ЕКСИТ_ЛЕВЕЛ, битовая маска 0x00000010)
  
> После выполнения этого правила вычисление правила завершится, за исключением оценки правил "нет на месте".
    
SCL (СТ_СКИП_ИФ_СКЛ_ИС_САФЕ, битовая маска 0x00000020)
  
> Оценка этого правила может быть пропущена.
    
PE (СТ_РУЛЕ_ПАРСЕ_ЕРРОР, битовая маска 0x00000040)
  
> Сервер обнаружил ошибку при синтаксическом анализе данных правила, предоставленных клиентом.
    
X
  
> Не используется этим протоколом. Этот бит не должен изменяться клиентом.
    
Примечание о взаимодействии между флагами СТ_ОНЛИ_ВХЕН_УФ и СТ_ЕКСИТ_ЛЕВЕЛ: 
  
Если для почтового ящика задано состояние "нет на месте", а условие правила имеет значение TRUE, 
  
С
  
- Правило имеет установленный флаг СТ_ЕКСИТ_ЛЕВЕЛ и не имеет установленного флага СТ_ОНЛИ_ВХЕН_УФ. Затем сервер не должен оценивать последующие правила, для которых не установлен флаг СТ_ОНЛИ_ВХЕН_УФ, и при необходимости оценить последующие правила с установленным флагом СТ_ОНЛИ_ВХЕН_УФ.
    
ТАКЖЕ
  
- Для правила заданы оба флага СТ_ЕКСИТ_ЛЕВЕЛ и СТ_ОНЛИ_ВХЕН_УФ. Затем сервер не должен оценивать последующие правила.
