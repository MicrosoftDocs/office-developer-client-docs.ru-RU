# <a name="dtblbutton"></a>DTBLBUTTON

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит сведения о элементе управления "Кнопка" для диалогового окна, созданного из таблицы отображения.
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |MAPIDEFS. h  <br/> |
|Связанный макрос:  <br/> |[SizedDtblButton](sizeddtblbutton.md) <br/> |
   
```cpp
typedef struct _DTBLBUTTON
{
  ULONG ulbLpszLabel;
  ULONG ulFlags;
  ULONG ulPRControl;
} DTBLBUTTON, FAR *LPDTBLBUTTON;

```

## <a name="members"></a>Members

 **Улблпсзлабел**
  
> Позиция в памяти строки символов, отображаемой на кнопке.
    
 **ulFlags**
  
> Битовая маска флагов, используемых для обозначения формата метки, на который указывает элемент **улблпсзлабел** . Можно задать следующий флаг: 
    
MAPI_UNICODE 
  
> Метка имеет формат Юникод. Если флаг МАПИ_УНИКОДЕ не установлен, метка имеет формат ANSI.
    
 **Улпрконтрол**
  
> Тег свойства для свойства типа ПТ_ОБЖЕКТ, реализующего интерфейс [имапиконтрол](imapicontroliunknown.md) . При нажатии кнопки MAPI вызывает метод [IMAPIProp:: опенпроперти](imapiprop-openproperty.md) для реализации [IMAPIProp](imapipropiunknown.md) таблицы отображения, чтобы получить это свойство. 
    
## <a name="remarks"></a>Комментарии

Структура **дтблбуттон** описывает кнопку элемента управления, которая при нажатии позволяет пользователю начать операцию. Как правило, нажатие кнопки приводит к отображению модального диалогового окна или программной задачи для вызова. Поставщики услуг могут реализовать что-либо с помощью элемента управления "Кнопка". Если кнопка должна выполнять задачу на основе значений других элементов управления, эти элементы управления должны иметь установленный флаг ДТ_СЕТ_ИММЕДИАТЕ. 
  
Элемент **улблпсзлабел** — это позиция в памяти строки символов, которая отображается на кнопке. Поставщики услуг могут добавить символ амперсанда (&amp;), чтобы обозначить ускоритель Windows в метке кнопки. Нажатие клавиши вызова эквивалентно нажатию кнопки. 
  
Элемент **улпрконтрол** описывает свойство Object, которое при открытии с помощью метода **IMAPIProp:: опенпроперти** возвращает указатель на объект элемента управления. Реализация объекта элемента управления, поддерживающего интерфейс **имапиконтрол** , позволяет расширить набор функций MAPI и определить операцию или задачу, выполняемую при нажатии кнопки. **Имапиконтрол** предоставляет два метода для управления кнопками: [](imapicontrol-getstate.md) "с выходом" и "включить кнопки" и " [активировать](imapicontrol-activate.md) для обработки нажатий кнопок". 
  
Общие сведения о отображаемых таблицах приведены в разделе [Display Tables](display-tables.md). Сведения о том, как реализовать таблицу отображения, можно найти [в статье реализация таблицы отображения](display-table-implementation.md).
