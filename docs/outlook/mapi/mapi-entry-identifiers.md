## <a name="entry-identifiers"></a><span data-ttu-id="008ce-109">Идентификаторы записей</span><span class="sxs-lookup"><span data-stu-id="008ce-109">Entry Identifiers</span></span>

<span data-ttu-id="008ce-110">Поставщики хранилища сообщений назначить идентификаторы записей хранилищ сообщений, папок и сообщений. поставщиками адресных книг их присваивания адресной книги контейнеров, списков рассылки и обмена мгновенными сообщениями пользователи.</span><span class="sxs-lookup"><span data-stu-id="008ce-110">Message store providers assign entry identifiers to message stores, folders, and messages; address book providers assign them to address book containers, distribution lists, and messaging users.</span></span> <span data-ttu-id="008ce-111">Идентификаторы входа также используется для открытия объекта, представленного строка в таблице, такие как объект состояния в таблице состояния.</span><span class="sxs-lookup"><span data-stu-id="008ce-111">Entry identifiers are also used to open an object represented by a row in a table, such as a status object in the status table.</span></span> <span data-ttu-id="008ce-112">Объекты хранят свои идентификаторы записей в своем свойстве **PR_ENTRYID** ([PidTagEntryId](pidtagentryid-canonical-property.md)).</span><span class="sxs-lookup"><span data-stu-id="008ce-112">Objects store their entry identifiers in their **PR_ENTRYID** ([PidTagEntryId](pidtagentryid-canonical-property.md)) property.</span></span> 
  
<span data-ttu-id="008ce-113">В то время как поставщиков услуг создание, назначение и проверьте идентификаторы записей, клиентские приложения их использовать только в качестве средства для открытия объектов.</span><span class="sxs-lookup"><span data-stu-id="008ce-113">Whereas service providers create, assign, and examine entry identifiers, client applications use them only as tools for opening objects.</span></span> <span data-ttu-id="008ce-114">Для клиентов идентификаторы записей — непрозрачный части двоичных данных и не работают с базовой системы обмена сообщениями.</span><span class="sxs-lookup"><span data-stu-id="008ce-114">To clients, entry identifiers are opaque pieces of binary data and have nothing to do with the underlying messaging system.</span></span> 
  
<span data-ttu-id="008ce-115">Клиенты вызывать метод [IMAPIProp::GetProps](imapiprop-getprops.md) объекта для извлечения его **PR_ENTRYID** свойство или метод [IMAPITable::QueryColumns](imapitable-querycolumns.md) таблицы для получения столбца, в котором содержатся свойства **PR_ENTRYID** .</span><span class="sxs-lookup"><span data-stu-id="008ce-115">Clients call an object's [IMAPIProp::GetProps](imapiprop-getprops.md) method to retrieve its **PR_ENTRYID** property or a table's [IMAPITable::QueryColumns](imapitable-querycolumns.md) method to retrieve the column that holds the **PR_ENTRYID** property.</span></span> 
  
<span data-ttu-id="008ce-116">Идентификаторы записей, передаются как параметры методов **OpenEntry** и **CompareEntryIDs** .</span><span class="sxs-lookup"><span data-stu-id="008ce-116">Entry identifiers are passed as parameters to the **OpenEntry** and **CompareEntryIDs** methods.</span></span> <span data-ttu-id="008ce-117">Несколько объектов MAPI реализовать методы **OpenEntry** и **CompareEntryIDs** .</span><span class="sxs-lookup"><span data-stu-id="008ce-117">Several MAPI objects implement the **OpenEntry** and **CompareEntryIDs** methods.</span></span> <span data-ttu-id="008ce-118">С помощью **OpenEntry**клиенты могут открывать объекта.</span><span class="sxs-lookup"><span data-stu-id="008ce-118">With **OpenEntry**, clients can open an object.</span></span> <span data-ttu-id="008ce-119">С помощью **CompareEntryIDs**клиенты могут сравнить два идентификаторы записей для определения, является ли они ссылаются на тот же объект.</span><span class="sxs-lookup"><span data-stu-id="008ce-119">With **CompareEntryIDs**, clients can compare two entry identifiers to determine whether they refer to the same object.</span></span> <span data-ttu-id="008ce-120">Поскольку идентификаторы записей не обязательно содержит двоичные сравнимые, клиенты необходимо сравнить их с помощью метода **CompareEntryIDs** .</span><span class="sxs-lookup"><span data-stu-id="008ce-120">Because entry identifiers are not necessarily binary comparable, clients must compare them by the **CompareEntryIDs** method.</span></span> 
  
<span data-ttu-id="008ce-121">Клиенты должны всегда передать идентификаторы естественно выровненные записей в соответствующих вызовов для поставщиков услуг, так как хотя поставщиков услуг должен обрабатывать идентификаторы записей, выровненные произвольно, не всегда регистр.</span><span class="sxs-lookup"><span data-stu-id="008ce-121">Clients should always pass naturally aligned entry identifiers in their calls to service providers, because although service providers should handle entry identifiers that are arbitrarily aligned, this is not always the case.</span></span> <span data-ttu-id="008ce-122">Адрес естественно выровненные памяти позволяет получить доступ к любой тип данных, поддерживаемых по этому адресу без создания ошибку выравнивание.</span><span class="sxs-lookup"><span data-stu-id="008ce-122">A naturally aligned memory address enables the computer to access any data type it supports at that address without generating an alignment fault.</span></span> <span data-ttu-id="008ce-123">Коэффициент естественным выравниванием обычно же фактором выравнивание, используемых распределителя памяти системы и обычно является 8 байтов.</span><span class="sxs-lookup"><span data-stu-id="008ce-123">The natural alignment factor is typically the same alignment factor used by the system memory allocator and is usually 8 bytes.</span></span>
  
<span data-ttu-id="008ce-124">Идентификаторы записей бывают двух типов: краткосрочной и долгосрочной.</span><span class="sxs-lookup"><span data-stu-id="008ce-124">Entry identifiers come in two types: short-term and long-term.</span></span> <span data-ttu-id="008ce-125">Краткосрочные идентификаторы записей, выполняются быстрее, для создания, но их уникальность гарантируется только в течение текущего сеанса на текущей рабочей станции.</span><span class="sxs-lookup"><span data-stu-id="008ce-125">Short-term entry identifiers are faster to construct, but their uniqueness is guaranteed only over the life of the current session on the current workstation.</span></span> <span data-ttu-id="008ce-126">Долгосрочные идентификаторы записей имеют более длительного времени существования.</span><span class="sxs-lookup"><span data-stu-id="008ce-126">Long-term entry identifiers have a more prolonged lifespan.</span></span> <span data-ttu-id="008ce-127">Краткосрочные идентификаторы записей используются в основном для строк в таблицах и записей в диалоговых окнах, тогда как долгосрочного идентификаторы записей, используемых для многих объекты, такие как сообщения, папки, и списки рассылки.</span><span class="sxs-lookup"><span data-stu-id="008ce-127">Short-term entry identifiers are used primarily for rows in tables and entries in dialog boxes, whereas long-term entry identifiers are used for many objects such as messages, folders, and distribution lists.</span></span>
