# <a name="mapi-entry-identifiers"></a><span data-ttu-id="75c50-103">Идентификаторы записей MAPI</span><span class="sxs-lookup"><span data-stu-id="75c50-103">MAPI Entry Identifiers</span></span>

  
  
<span data-ttu-id="75c50-104">**Область применения**: Outlook 2013 | Outlook 2016</span><span class="sxs-lookup"><span data-stu-id="75c50-104">**Applies to**: Outlook 2013 | Outlook 2016</span></span> 
  
<span data-ttu-id="75c50-105">Идентификаторы входа — это части двоичных данных, хранимые в структуре [ENTRYID,](entryid.md) которые используются для уникальной идентификации и открытия объекта MAPI.</span><span class="sxs-lookup"><span data-stu-id="75c50-105">Entry identifiers are pieces of binary data stored in an [ENTRYID](entryid.md) structure that are used to uniquely identify and open a MAPI object.</span></span> <span data-ttu-id="75c50-106">Большинство объектов MAPI имеют идентификаторы входа.</span><span class="sxs-lookup"><span data-stu-id="75c50-106">Most MAPI objects have entry identifiers.</span></span> <span data-ttu-id="75c50-107">Идентификаторы входа для объектов аналогичны именам файлов.</span><span class="sxs-lookup"><span data-stu-id="75c50-107">Entry identifiers for objects are analogous to file names for files.</span></span> <span data-ttu-id="75c50-108">Однако они не передаются и не могут использоваться в системах, помимо системы, на основе системы, в которая они были зародились.</span><span class="sxs-lookup"><span data-stu-id="75c50-108">However, they are not transmittable and cannot be used on systems other than the system they originated on.</span></span> 
  
## <a name="entry-identifiers"></a><span data-ttu-id="75c50-109">Идентификаторы записи</span><span class="sxs-lookup"><span data-stu-id="75c50-109">Entry Identifiers</span></span>

<span data-ttu-id="75c50-110">Поставщики хранилища сообщений назначают идентификаторы записей в хранилищах сообщений, папках и сообщениях; Поставщики адресных книг назначают их для адресных контейнеров книг, списков рассылки и пользователей сообщений.</span><span class="sxs-lookup"><span data-stu-id="75c50-110">Message store providers assign entry identifiers to message stores, folders, and messages; address book providers assign them to address book containers, distribution lists, and messaging users.</span></span> <span data-ttu-id="75c50-111">Идентификаторы записи также используются для открытия объекта, представленного строкой в таблице, например объекта состояния в таблице состояния.</span><span class="sxs-lookup"><span data-stu-id="75c50-111">Entry identifiers are also used to open an object represented by a row in a table, such as a status object in the status table.</span></span> <span data-ttu-id="75c50-112">Объекты хранят идентификаторы записей в **свойстве PR_ENTRYID** [(PidTagEntryId).](pidtagentryid-canonical-property.md)</span><span class="sxs-lookup"><span data-stu-id="75c50-112">Objects store their entry identifiers in their **PR_ENTRYID** ([PidTagEntryId](pidtagentryid-canonical-property.md)) property.</span></span> 
  
<span data-ttu-id="75c50-113">Если поставщики услуг создают, назначают и изучают идентификаторы записей, клиентские приложения используют их только в качестве средств для открытия объектов.</span><span class="sxs-lookup"><span data-stu-id="75c50-113">Whereas service providers create, assign, and examine entry identifiers, client applications use them only as tools for opening objects.</span></span> <span data-ttu-id="75c50-114">Для клиентов идентификаторы входа являются непрозрачными частями двоичных данных и не имеют ничего общего с системой обмена сообщениями.</span><span class="sxs-lookup"><span data-stu-id="75c50-114">To clients, entry identifiers are opaque pieces of binary data and have nothing to do with the underlying messaging system.</span></span> 
  
<span data-ttu-id="75c50-115">Клиенты звонят по методу [IMAPIProp::GetProps](imapiprop-getprops.md) объекта для получения свойства **PR_ENTRYID** или метода [IMAPITable::QueryColumns](imapitable-querycolumns.md) для  получения столбца, на который PR_ENTRYID свойства.</span><span class="sxs-lookup"><span data-stu-id="75c50-115">Clients call an object's [IMAPIProp::GetProps](imapiprop-getprops.md) method to retrieve its **PR_ENTRYID** property or a table's [IMAPITable::QueryColumns](imapitable-querycolumns.md) method to retrieve the column that holds the **PR_ENTRYID** property.</span></span> 
  
<span data-ttu-id="75c50-116">Идентификаторы записи передаются в качестве параметров методам **OpenEntry** и **CompareEntryIDs.**</span><span class="sxs-lookup"><span data-stu-id="75c50-116">Entry identifiers are passed as parameters to the **OpenEntry** and **CompareEntryIDs** methods.</span></span> <span data-ttu-id="75c50-117">Несколько объектов MAPI реализуют методы **OpenEntry** и **CompareEntryIDs.**</span><span class="sxs-lookup"><span data-stu-id="75c50-117">Several MAPI objects implement the **OpenEntry** and **CompareEntryIDs** methods.</span></span> <span data-ttu-id="75c50-118">С **помощью OpenEntry** клиенты могут открыть объект.</span><span class="sxs-lookup"><span data-stu-id="75c50-118">With **OpenEntry**, clients can open an object.</span></span> <span data-ttu-id="75c50-119">С **помощью compareEntryID клиенты** могут сравнить два идентификатора записи, чтобы определить, относятся ли они к одному объекту.</span><span class="sxs-lookup"><span data-stu-id="75c50-119">With **CompareEntryIDs**, clients can compare two entry identifiers to determine whether they refer to the same object.</span></span> <span data-ttu-id="75c50-120">Так как идентификаторы записей не обязательно являются двоичными, клиенты должны сравнивать их с помощью метода **CompareEntryIDs.**</span><span class="sxs-lookup"><span data-stu-id="75c50-120">Because entry identifiers are not necessarily binary comparable, clients must compare them by the **CompareEntryIDs** method.</span></span> 
  
<span data-ttu-id="75c50-121">Клиенты всегда должны передавать поставщику услуг идентификаторы записей с естественным соответствием, так как, хотя поставщики услуг должны обрабатывать идентификаторы записей, которые произвольно выровнены, это не всегда так.</span><span class="sxs-lookup"><span data-stu-id="75c50-121">Clients should always pass naturally aligned entry identifiers in their calls to service providers, because although service providers should handle entry identifiers that are arbitrarily aligned, this is not always the case.</span></span> <span data-ttu-id="75c50-122">Естественно выровненный адрес памяти позволяет компьютеру получать доступ к любому типу данных, который он поддерживает по этому адресу, без создания ошибки выравнивания.</span><span class="sxs-lookup"><span data-stu-id="75c50-122">A naturally aligned memory address enables the computer to access any data type it supports at that address without generating an alignment fault.</span></span> <span data-ttu-id="75c50-123">Естественный коэффициент выравнивания обычно является одним и тем же фактором выравнивания, используемым в системе, и обычно составляет 8 bytes.</span><span class="sxs-lookup"><span data-stu-id="75c50-123">The natural alignment factor is typically the same alignment factor used by the system memory allocator and is usually 8 bytes.</span></span>
  
<span data-ttu-id="75c50-124">Идентификаторы записи приходят в двух типах: краткосрочный и долгосрочный.</span><span class="sxs-lookup"><span data-stu-id="75c50-124">Entry identifiers come in two types: short-term and long-term.</span></span> <span data-ttu-id="75c50-125">Краткосрочные идентификаторы записей быстрее создавать, но их уникальность гарантируется только в течение текущего сеанса на текущей рабочей станции.</span><span class="sxs-lookup"><span data-stu-id="75c50-125">Short-term entry identifiers are faster to construct, but their uniqueness is guaranteed only over the life of the current session on the current workstation.</span></span> <span data-ttu-id="75c50-126">Долгосрочные идентификаторы записей имеют более длительный срок службы.</span><span class="sxs-lookup"><span data-stu-id="75c50-126">Long-term entry identifiers have a more prolonged lifespan.</span></span> <span data-ttu-id="75c50-127">Краткосрочные идентификаторы записей используются в основном для строк в таблицах и записей в диалоговых окнах, в то время как долгосрочные идентификаторы записей используются для многих объектов, таких как сообщения, папки и списки рассылки.</span><span class="sxs-lookup"><span data-stu-id="75c50-127">Short-term entry identifiers are used primarily for rows in tables and entries in dialog boxes, whereas long-term entry identifiers are used for many objects such as messages, folders, and distribution lists.</span></span>
