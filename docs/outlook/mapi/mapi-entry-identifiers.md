# <a name="mapi-entry-identifiers"></a>Идентификаторы записей MAPI

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Идентификаторы записей — это части двоичных данных, хранимые в структуре [ENTRYID,](entryid.md) которые используются для уникальной идентификации и открытия объекта MAPI. У большинства объектов MAPI есть идентификаторы записей. Идентификаторы записей объектов аналогичны именам файлов. Однако они не передаются и не могут использоваться в системах, кроме систем, в которые они исходили. 
  
## <a name="entry-identifiers"></a>Идентификаторы записей

Поставщики хранилища сообщений назначают идентификаторы записей хранилищам сообщений, папок и сообщений; Поставщики адресных книг назначают их контейнерам адресных книг, спискам рассылки и пользователям сообщений. Идентификаторы записей также используются для открытия объекта, представленного строкой в таблице, например объекта состояния в таблице состояния. Идентификаторы записей объектов хранятся в свойстве **PR_ENTRYID** ([PidTagEntryId).](pidtagentryid-canonical-property.md) 
  
В то время как поставщики услуг создают, назначают и проверяют идентификаторы записей, клиентские приложения используют их только в качестве инструментов для открытия объектов. Для клиентов идентификаторы записей являются непрозрачными двоичными данными и не имеют ничего общего с системой обмена сообщениями. 
  
Клиенты вызывают метод [IMAPIProp::GetProps](imapiprop-getprops.md) объекта, чтобы получить его свойство **PR_ENTRYID** или метод [IMAPITable::QueryColumns](imapitable-querycolumns.md) таблицы для получения столбца, который **содержит свойство PR_ENTRYID.** 
  
Идентификаторы записей передаются в качестве параметров методам **OpenEntry** и **CompareEntryIDs.** Несколько объектов MAPI реализуют методы **OpenEntry** и **CompareEntryIDs.** С **помощью OpenEntry** клиенты могут открывать объект. С **помощью CompareEntryIDs** клиенты могут сравнить два идентификатора записей, чтобы определить, ссылаются ли они на один и тот же объект. Так как идентификаторы записей не обязательно являются двоичными, клиенты должны сравнивать их с помощью метода **CompareEntryIDs.** 
  
Клиенты должны всегда передавать естественно выровненные идентификаторы записей в своих вызовах к поставщикам услуг, так как хотя поставщики услуг должны обрабатывать идентификаторы записей, которые произвольно выровнены, это не всегда так. Естественно выровненный адрес памяти позволяет компьютеру получать доступ к любому типу данных, который он поддерживает по этому адресу, без сбоя выравнивания. Коэффициент естественного выравнивания обычно является тем же коэффициентом выравнивания, который используется для системного памяти и обычно составляет 8байт.
  
Идентификаторы записей могут быть двух типов: краткосрочные и долгосрочные. Краткосрочные идентификаторы записей быстрее создавать, но их уникальность гарантируется только в течение срока действия текущего сеанса на текущей рабочей станции. Долгосрочные идентификаторы записей имеют более длительный срок действия. Краткосрочные идентификаторы записей в основном используются для строк в таблицах и записей в диалоговых окнах, тогда как долгосрочные идентификаторы записей используются для многих объектов, таких как сообщения, папки и списки рассылки.
