# <a name="hrcomposeeid"></a>HrComposeEID

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Создает составной идентификатор для объекта, обычно является сообщением в хранилище сообщений. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Мапиутил. h  <br/> |
|Реализовано в:  <br/> |MAPI  <br/> |
|Вызывающая сторона:  <br/> |Клиентские приложения  <br/> |
   
```cpp
HrComposeEID(
  LPMAPISESSION psession,
  ULONG cbStoreRecordKey,
  LPBYTE pStoreRecordKey,
  ULONG cbMsgEID,
  LPENTRYID pMsgEID,
  ULONG FAR * pcbEID,
  LPENTRYID FAR * ppEID
);
```

## <a name="parameters"></a>Параметры

 _псессион_
  
> возврата Указатель на сеанс, используемый клиентским приложением. 
    
 _Кбсторерекордкэй_
  
> возврата Размер (в байтах) ключа записи хранилища сообщений, содержащего сообщение или другой объект. Если в параметре _кбсторерекордкэй_ передается нулевое значение, параметр _ппеид_ указывает на копию идентификатора записи объекта. 
    
 _Псторерекордкэй_
  
> возврата Указатель на ключ записи хранилища сообщений, содержащего сообщение или другой объект. 
    
 _Кбмсжеид_
  
> возврата Размер (в байтах) идентификатора записи сообщения или другого объекта. 
    
 _Пмсжеид_
  
> возврата Указатель на идентификатор записи объекта. 
    
 _Пкбеид_
  
> вышли Указатель на размер возвращенного идентификатора (в байтах). 
    
 _Ппеид_
  
> вышли Указатель на указатель на возвращенный идентификатор записи. Если значение параметра _кбсторерекордкэй_ больше нуля, параметр _ппеид_ указывает на указатель на созданный идентификатор составной записи. Если _кбсторерекордкэй_ имеет значение 0, _ппеид_ указывает на указатель на копию идентификатора записи объекта. 
    
## <a name="return-value"></a>Возвращаемое значение

Нет.
  
## <a name="remarks"></a>Примечания

Если сообщение или другой объект, для которого создается создаваемый идентификатор составного элемента, находится в хранилище сообщений, идентификатор создается из идентификатора записи объекта и ключа записи в хранилище. Если объект не находится в хранилище, то есть, если количество байт для ключа записи хранилища, переданное в _кбсторерекордкэй_ , равно нулю, то идентификатор записи объекта просто копируется. 
  
Функция **хркомпосиид** позволяет приложениям работать с объектами в нескольких хранилищах с помощью составных идентификаторов записей. Приложение может вызвать функцию [хрдекомпосиид](hrdecomposeeid.md) , чтобы разделить идентификатор составной записи на исходные составляющие.
