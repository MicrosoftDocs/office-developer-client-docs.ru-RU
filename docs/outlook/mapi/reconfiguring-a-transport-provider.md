# <a name="reconfiguring-a-transport-provider"></a>Перенастройка поставщика транспорта

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Вы можете использовать объект состояния поставщика транспорта, чтобы изменить некоторые свойства поставщика. Диапазон свойств, которые можно изменить, зависит от свойств, которые включены в страницу свойств поставщика, и от того, как эти свойства определены. 
  
 **Повторная настройка активного поставщика транспорта**
  
1. Call [IMAPISession:: жетстатустабле](imapisession-getstatustable.md) для доступа к таблице состояния. 
    
2. Укажите строку для поставщика транспорта, которую необходимо изменить, создав ограничение свойства, которое соответствует **пр_дисплай_наме** ([PidTagDisplayName](pidtagdisplayname-canonical-property.md)) с именем целевого поставщика. 
    
3. Call [IMAPITable:: FindRow](imapitable-findrow.md) для получения соответствующей строки. 
    
4. Убедитесь, что флаги СТАТУС_СЕТТИНГС_ДИАЛОГ и СТАТУС_ВАЛИДАТЕ_СТАТЕ установлены в свойстве **пр_ресаурце_месодс** ([PidTagResourceMethods](pidtagresourcemethods-canonical-property.md)) целевого поставщика транспорта. Если СТАТУС_СЕТТИНГС_ДИАЛОГ не задано, то поставщик транспорта не отображает страницу свойств конфигурации. Если СТАТУС_ВАЛИДАТЕ_СТАТЕ не задано, невозможно выполнить динамическую перенастройку.
    
5. Если задано значение СТАТУС_СЕТТИНГС_ДИАЛОГ, вызовите [имапистатус:: сеттингсдиалог](imapistatus-settingsdialog.md) , чтобы отобразить лист свойств поставщика транспорта и разрешить пользователю вносить изменения. 
    
6. Когда пользователь завершит перенастройку, вызовите метод [имапистатус:: валидатестате](imapistatus-validatestate.md) , если статус_валидате_стате задано, передает конфиг_чанжед.
