# <a name="creating-display-tables-and-related-structures"></a>Создание таблиц отображения и соответствующих структур
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Создание таблицы отображения аналогично написанию программы на языке сценариев. Вы можете создать таблицу отображения, вызывая [BuildDisplayTable](builddisplaytable.md) или написав пользовательский код для заполнения строк и столбцов таблицы. В общем, следует использовать метод **BuildDisplayTable,** так как он проще. 
  
Перед вызовом **BuildDisplayTable** для запроса MAPI на создание отображаемой таблицы необходимо создать иерархию структур. Структура верхнего уровня [DTPAGE](dtpage.md)описывает одну страницу свойств с вкладками. В каждой **структуре DTPAGE** имеется [структура DTCTL,](dtctl.md) описываемая один из них, например поле редактирования или кнопка параметра. Каждая **структура DTCTL** содержит структуру, которая характерна для типа управления. Например, если структура **DTCTL** описывает управление полем редактирования, она будет содержать **структуру DTBLEDIT.** Структура **DTCTL** для кнопки выбора содержит **структуру DTBLRADIOBUTTON.** 
  
Эти структуры напрямую связаны с **BuildDisplayTable;** они не имеют смысла вне контекста этой функции. При вызове **BuildDisplayTable** передается одна или несколько **структур DTPAGE** в качестве входных параметров. Структуры **DTPAGE** содержат массив структур **DTCTL** и число структур **DTCTL** в массиве. Каждый из них должен отображаться в диалоговом окне по одной структуре. **Структуры DTPAGE** также имеют строку символов, которая представляет имя соответствующего файла справки и ресурса диалоговых окна. 
  
Каждая **структура DTCTL** в **структуре DTPAGE** содержит следующие данные, используемые для набора свойств для управления: 
  
- Тип управления для **настройки** PR_CONTROL_TYPE ([PidTagControlType).](pidtagcontroltype-canonical-property.md)
    
- Флаги управления для **настройки PR_CONTROL_FLAGS** ([PidTagControlFlags).](pidtagcontrolflags-canonical-property.md)
    
- Данные уведомления для настройки **PR_CONTROL_ID** ([PidTagControlId](pidtagcontrolid-canonical-property.md)).
    
- Структура управления для **настройки** PR_CONTROL_STRUCTURE ([PidTagControlStructure).](pidtagcontrolstructure-canonical-property.md)
    
**Структуры DTCTL** также содержат идентификатор ресурса и фильтр символов для элементов управления полем редактирования и сомеда. 
  
Член структуры управления структуры **DTCTL** описывает данные, уникальные для типа управления. MAPI определяет разную структуру для каждого типа управления. Например, данные редактируемого управления представлены структурой **DTBLEDIT;** данные списка представлены структурой **DTBLLBX.** 
  
Связь между тремя типами структур таблицы отображения показана на следующем рисунке. Диалоговое окно, описанное в этой таблице отображения, имеет два средства управления: метку и элемент управления редактированием. Структура **DTBLLBX** имеет член смещения метки, как и некоторые структуры управления, которые описывают, с чего начинается строка символов для метки. Строки символов меток обычно помещаются в память сразу после структуры.
