# <a name="property-names-and-property-sets"></a>Имена свойств и наборы свойств

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Имя каждого имени свойства имеет две части:
  
- Глобальный уникальный идентификатор или GUID, который указывает набор свойств.
    
- Строка символов Unicode или 32-битное числовое значение. 
    
Имена имен имен описаны с помощью [структуры MAPINAMEID.](mapinameid.md) В этой структуре содержится член набора свойств, член для указания имени в числовом или строковом формате, а также член для определения используемой формы. Так как набор свойств является частью имени свойства, он не является необязательным. MAPI определила несколько наборов свойств для использования клиентами и поставщиками услуг, но если существующий набор свойств не подходит, можно определить новый набор свойств. Клиенты и поставщики услуг могут определять собственные наборы свойств, вызывая [функцию CoCreateGUID.](https://msdn.microsoft.com/library/ms688568.aspx) Обычно эти наборы свойств создаются для пользовательских клиентских приложений. 
  
Наборы свойств MAPI представлены следующими константами:
  
PS_MAPI
  
PS_PUBLIC_STRINGS
  
PS_ROUTING_EMAIL_ADDRESSES
  
PS_ROUTING_ADDRTYPE
  
PS_ROUTING_SEARCH_KEY
  
PS_ROUTING_DISPLAY_NAME
  
PS_ROUTING_ENTRYID
  
Зарезервируется PS_MAPI свойства; он используется поставщиками служб для создания имен для свойств с идентификаторами ниже названного диапазона свойств. Набор PS_PUBLIC_STRINGS используется клиентами для именных свойств сообщений IPM. Так как в наборе PS_PUBLIC_STRINGS свойства отображаются в пользовательском интерфейсе клиента, незаметные сообщения, такие как те, которые относятся к классу сообщений IPC, должны избегать создания именных свойств с этим набором свойств. Вместо этого они должны создавать свойства в диапазоне классов сообщений, 0x6800 через 0x7FFF.
  
Другие наборы свойств удерживают именные свойства, описывающие получателей, которые обычно являются членами списка маршрутов. Содержащие те же сведения, что и свойства, связанные со свойствами списка получателей, свойства в этих наборах свойств понимаются шлюзами для необходимости сопоставления для целевой системы обмена сообщениями. Так как для описания свойств имеется пять типов сведений, MAPI определила пять различных наборов свойств. Клиент, отправляющий сообщение, которое должно включать адрес и тип адреса для членов списка маршрутов, назначает именитое свойство для каждого участника в PS_ROUTING_EMAIL_ADDRESSES и PS_ROUTING_ADDRTYPE свойств. Это гарантирует, что тип адресов и адресов будет оставаться жизнеспособным при отправке в иностранную систему обмена сообщениями.
