# <a name="property-names-and-property-sets"></a>Имена свойств и наборы свойств

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Имя каждого именоваемого свойства имеет две части:
  
- Глобальный уникальный идентификатор или ИДЕНТИФИКАТОР GUID, который указывает набор свойств.
    
- Строка символов Юникода или 32-битное числовая величина. 
    
Имена именуемого свойства описываются с помощью структуры [MAPINAMEID.](mapinameid.md) Эта структура содержит член набора свойств, член для указания имени в числовом или строковом формате, а также член для определения используемой формата. Так как набор свойств является частью имени свойства, он не является необязательным. MAPI определил несколько наборов свойств для использования клиентами и поставщиками услуг, но если существующий набор свойств не подходит, можно определить новый набор свойств. Клиенты и поставщики услуг могут определять собственные наборы свойств, вызывая [функцию CoCreateGUID.](https://msdn.microsoft.com/library/ms688568.aspx) Обычно эти наборы свойств создаются для пользовательских клиентских приложений. 
  
Наборы свойств MAPI представлены следующими константами:
  
PS_MAPI
  
PS_PUBLIC_STRINGS
  
PS_ROUTING_EMAIL_ADDRESSES
  
PS_ROUTING_ADDRTYPE
  
PS_ROUTING_SEARCH_KEY
  
PS_ROUTING_DISPLAY_NAME
  
PS_ROUTING_ENTRYID
  
Набор PS_MAPI зарезервирован; Он используется поставщиками служб для создания имен свойств с идентификаторами под именовамом диапазоном свойств. Набор PS_PUBLIC_STRINGS используется клиентами для именных свойств сообщений IPM. Так как именные свойства в наборе свойств PS_PUBLIC_STRINGS отображаются в пользовательском интерфейсе клиента, не видны такие сообщения, как сообщения, принадлежащие классу сообщений IPC, не следует создавать именные свойства с этим набором свойств. Вместо этого они должны создавать свойства в диапазоне для определенного класса сообщений, 0x6800 через 0x7FFF.
  
Другие наборы свойств имеют именовые свойства, описывающие получателей, которые обычно являются членами списка маршрутов. Содержащие сведения того же типа, что и свойства, связанные со свойствами списка получателей, шлюзы понимают, что свойства в этих наборах свойств требуют сопоставления для целевой системы обмена сообщениями. Так как существует пять типов сведений для описания свойств, MAPI определил пять различных наборов свойств. Клиент, отправляющий сообщение, которое должно включать адрес и тип адреса для членов списка маршрутов, назначает именоватое свойство каждому участнику в PS_ROUTING_EMAIL_ADDRESSES и PS_ROUTING_ADDRTYPE свойств. Это гарантирует, что адрес и тип адреса останутся веская при отправке во внешней системе обмена сообщениями.
