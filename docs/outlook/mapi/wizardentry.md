# <a name="wizardentry"></a>WIZARDENTRY

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Определяет точку функцию запись службы поставщика, которая вызывает мастер профиля для извлечения достаточно сведений для отображения страницы свойств конфигурации поставщика. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapiwz.h  <br/> |
|Функция реализован:  <br/> |Поставщики служб  <br/> |
|Вызывается функция:  <br/> |Мастер профилей MAPI  <br/> |
   
```cpp
ULONG WIZARDENTRY(
  HINSTANCE hProviderDLLInstance,
  LPSTR FAR * lpcsResourceName,
  DLGPROC FAR * lppDlgProc,
  LPMAPIPROP lpMAPIProp,
  LPMAPISUPPORTOBJECT lpMapiSupportObject
);
```

## <a name="parameters"></a>Параметры

 _hProviderDLLInstance_
  
> [in] Дескриптор экземпляра DLL поставщика услуг. 
    
 _lpcsResourceName_
  
> [out] Указатель на строку, содержащую полное имя ресурса диалоговое окно, которое должно отображаться в мастере профилей во время настройки. Максимальный размер строки, включая символ конца строки составляет 32 символов. 
    
 _lppDlgProc_
  
> [out] Указатель на стандартный Windows процедуру диалогового окна, который будет вызываться мастер профиля для уведомления поставщика различные события. 
    
 _lpMAPIProp_
  
> [in] Указатель на реализацию интерфейса свойства, который предоставляет доступ к свойствам конфигурации. 
    
 _lpMapiSupportObject_
  
> [in] Указатель на объект поддержки MAPI, которые применяются для данного сеанса.
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Функция поставщика услуг **WIZARDENTRY** вызван успешно. 
    
MAPI_E_CALL_FAILED 
  
> Ошибка непредвиденные или неизвестный происхождения запрещено выполнение операции.
    
## <a name="remarks"></a>Замечания

Мастер профиля вызывает функцию **WIZARDENTRY** на основе, когда он будет готов для отображения пользовательского интерфейса конфигурации поставщика услуг. После завершения настройки всех поставщиков мастера профилей, он записывает данные свойства конфигурации профиле путем вызова [IMsgServiceAdmin::ConfigureMsgService](imsgserviceadmin-configuremsgservice.md). 
  
## <a name="notes-to-implementers"></a>Примечания для реализующих

Имя функции **WIZARDENTRY** на основе должны находиться в запись WIZARD_ENTRY_NAME в MAPISVC.INF. 
  
Имя ресурса является, что ресурса диалогового окна, которое будет отображаться в области мастер профилей. Ресурс, который передается обратная должны содержать все страницы в одном диалоговом окне ресурса. Когда мастер профиля получает этот ресурс, игнорирует диалоговое окно стиль, но не стилей элементов управления и создает все элементы управления в качестве дочерних элементов страница мастера профилей. Все элементы управления изначально являются скрытыми. Поставщики следует сделать уверены в том, что координаты для элементов управления нулевой или с отсчетом от нуля и они не превышать Максимальная ширина 200 единиц диалогового окна и максимальная высота 150 единиц диалогового окна. Идентификаторы элементов управления под 400 зарезервированы для мастера профилей. Мастер профиля отображает название поставщика полужирным шрифтом выше поставщика пользовательского интерфейса. 
  
Указатель интерфейса свойство, указанное в параметре _lpMAPIProp_ должны храниться поставщиком для последующего использования. Мастер профиля работает с самого базового набора свойств, и поставщик может использовать свойство реализации интерфейса, чтобы включить дополнительные свойства. Во время настройки поставщики следует добавить объект, реализующий интерфейс свойство их свойства конфигурации. После настройки всех поставщиков профилей мастер добавляет эти свойства профиля.
