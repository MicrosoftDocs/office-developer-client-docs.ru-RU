# <a name="wizardentry"></a>WIZARDENTRY

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Определяет функцию точки входа поставщика услуг, которую мастер профилей вызывает для получения достаточного количества сведений для отображения листов свойств конфигурации поставщика. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapiwz.h  <br/> |
|Определенные функции, реализованные в:  <br/> |Поставщики служб  <br/> |
|Определенная функция, вызванная:  <br/> |Мастер профилей MAPI  <br/> |
   
```cpp
ULONG WIZARDENTRY(
  HINSTANCE hProviderDLLInstance,
  LPSTR FAR * lpcsResourceName,
  DLGPROC FAR * lppDlgProc,
  LPMAPIPROP lpMAPIProp,
  LPMAPISUPPORTOBJECT lpMapiSupportObject
);
```

## <a name="parameters"></a>Parameters

 _hProviderDLLInstance_
  
> [in] Экземпляр обработки DLL поставщика услуг. 
    
 _lpcsResourceName_
  
> [вышел] Указатель на строку, которая содержит полное имя диалогового ресурса, которое должно отображаться мастером профилей во время настройки. Максимальный размер строки, включая терминатор NULL, — 32 символа. 
    
 _lppDlgProc_
  
> [вышел] Указатель на стандартную процедуру Windows диалоговое окно, которое будет вызвано мастером профилей, чтобы уведомить поставщика о различных событиях. 
    
 _lpMAPIProp_
  
> [in] Указатель на реализацию интерфейса свойств, который предоставляет доступ к свойствам конфигурации. 
    
 _lpMapiSupportObject_
  
> [in] Указатель на объект поддержки MAPI, применимый к этому сеансу.
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Функция **WIZARDENTRY** поставщика услуг была вызвана успешно. 
    
MAPI_E_CALL_FAILED 
  
> Ошибка неожиданного или неизвестного происхождения не позволяет завершить операцию.
    
## <a name="remarks"></a>Примечания

Мастер профилей вызывает функцию **WIZARDENTRY,** когда она готова отображать пользовательский интерфейс конфигурации поставщика услуг. Когда мастер профилей завершает настройку всех поставщиков, он записывает свойства конфигурации в профиль, вызывая [IMsgServiceAdmin::ConfigureMsgService](imsgserviceadmin-configuremsgservice.md). 
  
## <a name="notes-to-implementers"></a>Примечания для исполнителей

Имя функции **WIZARDENTRY** должно быть размещено в записи WIZARD_ENTRY_NAME MAPISVC.INF. 
  
Имя ресурса — это имя диалогового ресурса, который будет отрисовки в области мастера профилей. Ресурс, который передается обратно, должен содержать все страницы в одном диалоговом ресурсе. Когда мастер профилей получает этот ресурс, он игнорирует стиль диалогов, но не стили управления, и создает все элементы управления в качестве детей страницы Мастер профилей. Все элементы управления изначально скрыты. Поставщики должны убедиться, что координаты их элементов управления являются нулем или нулем и не превышают максимальную ширину 200 диалогов и максимальную высоту 150 диалогов. Идентификаторы управления ниже 400 зарезервированы для мастера профилей. Мастер профилей отображает название поставщика в жирном тексте над пользовательским интерфейсом поставщика. 
  
Указатель интерфейса свойств, поставленный в  _параметре lpMAPIProp,_ должен быть сохранен поставщиком для будущей ссылки. Мастер профилей имеет дело только с самым базовым набором свойств, и поставщик может использовать реализацию интерфейса свойств, чтобы включить дополнительные свойства. Во время настройки поставщики должны добавить свои свойства конфигурации в объект, реализующий интерфейс свойства. После настройки всех поставщиков мастер профилей добавляет эти свойства в профиль. 
  
Дополнительные сведения об использовании этой функции см. в дополнительных [сведениях о поддержке конфигурации службы сообщений.](supporting-message-service-configuration.md)
