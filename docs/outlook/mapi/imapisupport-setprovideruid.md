# <a name="imapisupportsetprovideruid"></a>IMAPISupport::SetProviderUID

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Регистрирует структуру [мапиуид](mapiuid.md) , которая уникально представляет поставщика услуг. 
  
```cpp
HRESULT SetProviderUID(
LPMAPIUID lpProviderID,
ULONG ulFlags
);
```

## <a name="parameters"></a>Параметры

 _Лппровидерид_
  
> возврата Указатель на структуру **мапиуид** , которая идентифицирует адресную книгу или поставщика хранилища сообщений. 
    
 _ulFlags_
  
> Резервирования должно быть равно нулю.
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Структура **мапиуид** успешно зарегистрирована. 
    
## <a name="remarks"></a>Замечания

Метод **имаписуппорт:: сетпровидеруид** реализован для адресных книг и объектов поддержки поставщиков хранилища сообщений. Эти поставщики вызывают **сетпровидеруид** , чтобы зарегистрировать уникальный идентификатор, описанный в структуре **мапиуид** , на которую указывает _лппровидерид_. Поставщики включают этот идентификатор во все создаваемые им идентификаторы записей. 
  
MAPI использует структуру **мапиуид** при отправке исходящих сообщений в Диспетчер очереди MAPI и для определения соответствующего поставщика для обработки запросов клиентов. Например, когда клиент вызывает метод [IMAPISession:: OpenEntry](imapisession-openentry.md) , MAPI проверяет часть идентификатора записи **мапиуид** , сопоставляет ее с поставщиком, который передавал его в **сетпровидеруид**, и вызывает **OpenEntry** поставщика . 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

Call **сетпровидеруид** во время входа, чтобы зарегистрировать структуру **мапиуид** . MAPI позволяет поставщикам адресных книг и хранилищ сообщений регистрировать несколько идентификаторов. При совершении нескольких вызовов **сетпровидеруид**всегда добавляется структура **мапиуид** в набор структур **Мапиуид** поставщика, даже если **мапиуид** является дубликатом. **Сетпровидеруид** не удается удалить **мапиуид**.
