# <a name="hristoragefromstream"></a>HrIStorageFromStream

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Слои интерфейс **IStorage** в объект **IStream** . 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Мапиутил. h  <br/> |
|Реализовано в:  <br/> |MAPI  <br/> |
|Вызывающая сторона:  <br/> |Клиентские приложения и поставщики услуг  <br/> |
   
```cpp
HRESULT HrIStorageFromStream(
  LPUNKNOWN lpUnkIn,
  PIID lpInterface,
  ULONG ulFlags,
  LPSTORAGE FAR * lppStorageOut
);
```

## <a name="parameters"></a>Параметры

 _Лпункин_
  
> возврата Указатель на объект **IUnknown** , реализующий **IStream**. 
    
 _Лпинтерфаце_
  
> возврата Указатель на идентификатор интерфейса (IID) для объекта Stream. В параметре _лпинтерфаце_ можно передать любое из следующих значений: NULL, Иид_истреам или иид_илоккбитес. Передача значения NULL в _лпинтерфаце_ выполняется так же, как и передача иид_истреам. 
    
 _ulFlags_
  
> возврата Битовая маска флагов, которые управляют способом создания объекта хранилища относительно потока. По умолчанию используется значение СТГСТРМ_РЕСЕТ, которое предоставляет объекту хранилища доступ только для чтения и запускает его в позиции 0 потока. Следующие флаги можно задавать в любом сочетании, за исключением отмеченных.
    
СТГСТРМ_КРЕАТЕ 
  
> Создает новый объект Storage для объекта Stream. Этот флаг не может быть установлен, если установлен флаг СТГСТРМ_РЕСЕТ. 
    
СТГСТРМ_КУРРЕНТ 
  
> Запускает хранилище в текущей позиции потока. Этот флаг не может быть установлен, если установлен флаг СТГСТРМ_РЕСЕТ. 
    
СТГСТРМ_МОДИФИ 
  
> Позволяет поставщику службы вызовов записывать в возвращенное хранилище. Этот флаг не может быть установлен, если установлен флаг СТГСТРМ_РЕСЕТ. 
    
СТГСТРМ_РЕСЕТ 
  
> Запускает хранилище в нулевой позиции. Этот флаг не может быть установлен, если задан любой другой флаг. 
    
 _Лппсторажеаут_
  
> вышли Указатель на указатель на возвращенный объект **IStorage** .

## <a name="remarks"></a>Примечания

Поставщики хранилища сообщений поддерживают функцию **христоражефромстреам** , используя интерфейс **IStorage** для вложений. Поставщики хранилища должны реализовать интерфейс **IStream** . **Христоражефромстреам** предоставляет интерфейс **IStorage** для объекта **IStream** . Можно передать либо интерфейс **ILockBytes** , либо интерфейс **IStream** в _лпункин_.
