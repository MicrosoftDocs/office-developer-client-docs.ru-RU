# <a name="mapi-form-manager"></a>Диспетчер форм MAPI

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Диспетчер форм — это объект, реализуя [интерфейс IMAPIFormMgr.](imapiformmgriunknown.md) Большинство организаций будут использовать диспетчер форм, поставляемый с MAPI, именуемой диспетчером форм по умолчанию. Однако при желании организация может заменить диспетчера форм по умолчанию настраиваемой диспетчером форм. Руководитель форм заботится о поиске форм в библиотеках форм, загрузке форм в ответ на запросы пользователей и установке форм в локализованную библиотеку форм пользователя, библиотеку форм папок или личную библиотеку форм. 
  
Чтобы пользователь взаимодействовал с сообщением, необходимо создать и активировать экземпляр сервера формы для класса сообщений для отображения сообщения и выполнения запрашиваемой операции в сообщении. Как описано в разделе [MapI Form Libraries,](mapi-form-libraries.md)реализация формы может существовать в нескольких различных расположениях (библиотеках форм), и нет никакой гарантии, что форма или ее сервер будут доступны локально или в запущенном состоянии, когда пользователь хочет взаимодействовать с ней. Диспетчер форм заботится о деталях размещения и активации формы.
  
Клиенты используют службы, предоставляемые менеджером форм, для поиска и активации форм. Интерфейс **IMAPIFormMgr** реализован менеджером форм и вызван клиентами для доступа к его службам. Менеджер формы является важным компонентом, так как он скрывает почти все сведения о поиске и активации форм от клиентов обмена сообщениями. 
  
При загрузке серверов форм диспетчер форм по умолчанию загружает форму из первой библиотеки форм, в которой находится реализация для класса сообщений формы. Менеджер форм по умолчанию ищет библиотеки форм в следующем порядке:
  
1. Локализованная библиотека форм пользователя. Сначала эта библиотека форм будет искаться, так как обеспечивает самый быстрый доступ к реализации формы, если реализация установлена в локальной библиотеке форм.
    
2. Библиотека форм папки контейнера сообщения — папка, в которой загружается сообщение.
    
3. Библиотека личной формы пользователя.
    
Настраиваемый диспетчер форм может выполнять поиск доступных библиотек форм в любом порядке или реализовать другие библиотеки форм, такие как библиотека форм для всей организации. Дополнительные сведения о библиотеках форм см. в [материале MAPI Form Libraries.](mapi-form-libraries.md)
