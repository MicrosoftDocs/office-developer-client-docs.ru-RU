# <a name="api-elements-deprecated-in-this-edition"></a>Элементы API, объявленные устаревшими в этом выпуске

  
  
**Относится к**: Outlook 
  
Следующие элементы API поддерживаются в Microsoft Outlook 2013. Больше не поддерживаются и их не следует использовать в новых проектах.
  
## <a name="deprecation-of-message-and-recipient-options"></a>Об использовании параметров получателей и сообщений

Следующие элементы API поддерживаются в этой версии из-за устаревших сообщений и получателей параметры:
  
- **IXPLogon::RegisterOptions**— подсистемы MAPI больше не вызывает этот метод, чтобы установить все значения по умолчанию для доступа к параметрам сообщений и получателей для поставщика транспорта.
    
- **OPTIONDATA**— эта структура данных, поддерживаемые свойства для доступа к параметрам сообщение и получатель является устаревшим. Подсистема MAPI больше не вызывает **IXPLogon::RegisterOptions** для получения любое сообщение или получателей параметры, поставщика транспорта для типа определенного адреса. 
    
- **OPTIONCALLBACK**— в этом прототипа функции, которая транспорта поставщика используется для объявления функции обратного вызова и который, в свою очередь, подсистемы MAPI, используемой для разрешения свойства поставщика является устаревшим. Подсистема MAPI больше не вызывает **IXPLogon::RegisterOptions** или любой функция обратного вызова, возвращаемые поставщика транспорта. 
    
- **IMAPISession::MessageOptions**— клиент и служба поставщики MAPI больше не должны вызывать этот метод для отображения свойств или пользователи могут задавать свойства, управляющие определенного типа сообщения и адрес. Метод всегда возвращает MAPI_E_NOT_FOUND, это означает, что нет способа сообщения для определенного сообщения.
    
- **IMAPISession::QueryDefaultMessageOpt**— клиент и служба поставщики MAPI больше не должны вызывать этот метод для извлечения свойств, определяющих параметры сообщения для определенного адреса типа. Метод больше не возвращает указатель на какой-либо массив значений свойств.
    
- **IAddrBook::RecipOptions**— клиент и служба поставщики MAPI больше не должны вызывать этот метод для отображения свойств или пользователи могут задавать свойства этого элемента управления обработки для получателя, адрес определенного типа. Метод всегда возвращает MAPI_W_ERRORS_RETURNED, это означает, что нет способа получателей для определенного получателя.
    
- **IAddrBook::QueryDefaultRecipOpt**— клиент и служба поставщики MAPI больше не должны вызывать этот метод для извлечения свойств, которые управляют получателей параметры для типа определенного адреса. Метод больше не возвращает указатель на какой-либо массив значений свойств.
