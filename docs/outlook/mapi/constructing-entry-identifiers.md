# <a name="constructing-entry-identifiers"></a>Создание идентификаторов записей

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Идентификаторы записей построены с помощью структуры [ENTRYID.](entryid.md) Структура **ENTRYID** состоит из флага, который описывает атрибуты идентификатора записи и фактического идентификатора записи. 
  
## <a name="entryid-structure"></a>Структура ENTRYID

Структура **ENTRYID** определяется следующим образом: 
  
```cpp
typedef struct
{
    BYTE        abFlags[4];
    BYTE        ab[MAPI_DIM];
}  ENTRYID, FAR *LPENTRYID;
 
```

MAPI_DIM это константа, заданная в файле загона MapiDefs.h. 
  
Первый byte из 4-byte **abFlags** member describes the type and use of the entry identifier and can have the following values: 
  
- MAPI_NOTRECI — указывает, что идентификатор записи нельзя использовать в качестве получателя сообщения.
    
- MAPI_NOTRESERVED — указывает, что другие пользователи не могут получить доступ к идентификатору записи.
    
- MAPI_NOW — указывает, что идентификатор записи нельзя использовать в другое время.
    
- MAPI_SHORTTERM — указывает, что идентификатор записи является краткосрочным. Все остальные значения в этом byte необходимо устанавливать, если не разрешено другое использование идентификатора записи.
    
- MAPI_THISSESSION — указывает, что идентификатор записи нельзя использовать в других сеансах.
    
- MAPI_NOTRESERVED — указывает, что идентификатор записи может использоваться другими поставщиками служб для других объектов.
    
Элемент **ab** идентификаторов записей, созданный поставщиками адресной книги и магазина сообщений, состоит из двух частей: 16-байтовой структуры [MAPIUID,](mapiuid.md) которая идентифицирует поставщика услуг, и фрагмента для идентификации объекта. **MAPIUID** — это структура, которая содержит глобальный уникальный идентификатор или GUID. GUID — это независимый от порядка в порядке использования идентификатор, который можно создать с помощью средства Microsoft Visual Studio *Create GUID**. 
  
Поставщик услуг регистрирует свою структуру **MAPIUID** с помощью MAPI во время процесса регистрации в вызове метода [IMAPISupport::SetProviderUID.](imapisupport-setprovideruid.md) Когда клиент вызывает метод **OpenEntry** для доступа к объекту, MAPI использует структуру **MAPIUID,** чтобы определить, какой поставщик услуг может предоставить такой доступ. Поставщики услуг должны использовать ту же **структуру MAPIUID** для всех версий своих DLL. Это позволяет клиентам с более новой версией реагировать на сообщения, отправленные и сохраненные в более старой версии. 
  
Остальная часть **ab-члена** после 16-byte **MAPIUID** содержит двоичные данные поставщика услуг для идентификации определенных объектов. Для этой части идентификатора записи фиксированный размер не существует. Это может быть любой размер по причине. Поставщик услуг обычно включает в эту часть идентификаторов записей следующие сведения: 
  
- Сведения о версии — так как поставщик службы часто меняет формат идентификаторов записей с версии на версию, хранение сведений о версии позволяет быстро определить способ расшифровки любого идентификатора записи.
    
- Сведения о расположении — это данные, которые дают поставщику услуг индикатор того, как найти объект, представленный идентификатором записи. Например, поставщик службы может сохранить смещение диска для последнего места в файле данных, в который был сохранен объект. Поскольку этот тип информации может со временем изменяться, поставщики услуг должны предоставлять несколько способов для размещения объектов в идентификаторах записей.
    
Хотя поставщики услуг могут перезахламить свои идентификаторы записей, следует избегать этой практики. Если необходимо повторно использовать идентификатор записи, поставщики услуг должны сделать период времени между первоначальным использованием и повторное использование как можно больше. Кроме того, идентификатор записи должен быть перена назначен другому объекту того же типа. То есть определенный идентификатор записи не должен быть связан сначала с сообщением, а затем с папкой.
