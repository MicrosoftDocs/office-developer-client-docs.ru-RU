# <a name="about-setting-the-resolution-order-for-address-lists-in-outlook"></a>Сведения о настройке порядка разрешения для списков адресов в Outlook

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Для каждого профиля Microsoft Office Outlook поддерживает несколько списков адресов, и пользователи могут вручную задавать порядок списков адресов, по которому сопоставляются получатели в сообщениях электронной почты и участников в приглашениях на собрания. Например вы можете задать порядок сопоставления таким образом, чтобы имена можно было сопоставлять сначала с адресной книгой Outlook, а затем с глобальным списком адресов. На компьютере откройте адресную книгу, нажав**Инструмента**, а затем **Параметры**, чтобы указать порядок сопоставления. В корпоративной среде IT-администраторам рекомендуется на программном уровне задать порядок списков адресов, по которым имена будут сопоставляться. Такой код можно использовать как часть сценария автоматизации запуска, который администратор развертывает для корпоративных нужд. 
  
MAPI поддерживает метод ** [SetSearchPath](iaddrbook-getsearchpath.md) ** интерфейса** [IAddrBook](iaddrbookimapiprop.md)**, в котором вы можете указать новый путь поиска в профиле, который используется при сопоставление имени. Чтобы использовать метод **IAddrBook::SetSearchPath**, вы должны указать желаемый порядок сопоставления с помощью массива, который содержит контейнеры соответствующих адресных книг в том порядке, в каком должно производиться сопоставление. Каждая запись в этот массиве также должна содержать идентификатор записи соответствующей адресной книги.
