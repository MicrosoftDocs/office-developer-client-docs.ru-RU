# <a name="synchronize-state"></a>Состояние синхронизации

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время синхронизации конечного автомата репликации.

> [!NOTE]
> Конечный автомат репликации — это детерминированный конечный автомат. Клиент, который отменяет от одного состояния к другому, должен, в конечном итоге, вернуться к предыдущему из последнего. 
  
## <a name="description"></a>Описание

Это состояние инициирует синхронизацию. Локальное хранилище может перейти к отправке или состоянию загрузки отсюда. Например, локальное хранилище может перейти в состояние иерархии отправки, чтобы отправить иерархию папок на сервер, или выполнить полную синхронизацию, предварительно отправив иерархию, а затем загрузив иерархию с сервера.
  
В этом состоянии Outlook инициализирует связанную структуру данных **синхронизации** с путем к локальному хранилищу, поэтому Outlook видит изменения в других состояниях. 
  
Клиент задает элементы **синхронизации**[in], которые сообщают Outlook, как обрабатывать другие состояния. Например, клиент может установить *ulFlags* в **упс_уплоад_онли** и **упс_сесе_фолдерс** и *PEL* в список идентификаторов записей папок, чтобы сообщить Outlook о том, что будут отправлены только эти папки. При завершении этого состояния локальное хранилище возвращается в состояние Idle.
