# <a name="mapi-architecture-overview"></a>Обзор архитектуры MAPI
 
**Область применения**: Outlook 2013 | Outlook 2016 
  
MAPI определяет модульную архитектуру, как показано на следующем рисунке.  
  
![Outlook 2010 Outlook](media/amapi_43.gif "2010")
  
Приложение MAPI известно как клиент, так как является клиентом подсистемы MAPI. Приложения, основанные на обмене сообщениями, используют обмен сообщениями в качестве центральной части обработки и предлагают широкие возможности обмена сообщениями, такие как обмен информацией различных типов в различных форматах, а также возможность локального сохранения и организации информации. Приложения электронной почты, планирования и потока работы являются примерами приложений на основе обмена сообщениями.
  
Подсистема MAPI состоит из общего пользовательского интерфейса и интерфейсов программирования. Общий пользовательский интерфейс — это набор диалогов, которые дают клиентские приложения последовательный внешний вид, а пользователям — согласованный способ работы.
  
MAPI имеет интерфейсы программирования, которые используются подсистемой MAPI, разработчиками клиентского программного обеспечения и разработчиками поставщиков услуг. Интерфейс программирования MAPI — это основной объектный интерфейс программирования. Интерфейс программирования MAPI похож на объектную модель компонентов OLE и используется клиентской подсистемой MAPI и приложениями на основе обмена сообщениями, написанными в C или C++. 
  
Как разработчик клиентского программного обеспечения вы делаете звонки MAPI непосредственно через интерфейс программирования MAPI. Вы можете реализовать обмен сообщениями с помощью одного клиентского интерфейса MAPI или сочетания интерфейсов. Одно приложение может выполнять вызовы методов или функций, принадлежащих любому из интерфейсов.
