# <a name="using-mapi-objects"></a>Использование объектов MAPI

**Относится к**: Outlook 2013 | Outlook 2016 
  
Клиенты и поставщики услуг используют объекты MAPI, вызывая методы в своих реализациях интерфейса. Это единственный способ использования объектов MAPI; методы, реализованные объектом вне интерфейса MAPI, недоступны для общего реализации. Так как все интерфейсы объекта связаны посредством наследования, пользователь объекта может вызывать методы либо в базовом интерфейсе, либо в одном из унаследованных интерфейсов, как если бы они принадлежат одному интерфейсу. 
  
Когда пользователь объекта хочет вызвать метод и этот объект реализует несколько интерфейсов, связанных с наследованием, пользователю не нужно знать, к каков интерфейсу принадлежит метод. Пользователь может вызвать любой из методов в любом из интерфейсов с одним указателем на объект. Например, на следующем рисунке показано, как клиентские приложения используют объект папки. Объекты папок реализуют интерфейс [IMAPIFolder : IMAPIContainer,](imapifolderimapicontainer.md) который наследуется от [IUnknown](https://msdn.microsoft.com/library/33f1d79a-33fc-4ce5-a372-e08bda378332%28Office.15%29.aspx) косвенно через [IMAPIProp : IUnknown](imapipropiunknown.md) и [IMAPIContainer : IMAPIProp](imapicontainerimapiprop.md). Клиент может вызвать один из методов **IMAPIProp,** например [IMAPIProp::GetProps,](imapiprop-getprops.md)и один из методов [IMAPIFolder : IMAPIContainer,](imapifolderimapicontainer.md) например [IMAPIFolder::CreateMessage,](imapifolder-createmessage.md)аналогичным образом с тем же указателем объекта. Клиент не знает или зависит от того, что эти вызовы относятся к разным интерфейсам.
  
**Использование объекта папки клиентом**
  
![Использование клиентом объекта папки.](media/amapi_40.gif "Клиент использует объект папки")
  
Эти вызовы преобразуются в код по-разному в зависимости от того, написан ли клиент, который делает вызовы, на C или C++. Прежде чем можно будет вызвать метод, необходимо получить указатель на реализацию интерфейса. Указатели интерфейса можно получить следующими способами:
  
- Вызов метода для другого объекта.
    
- Вызов функции API.
    
- Вызов метода [IUnknown::QueryInterface](https://msdn.microsoft.com/library/54d5ff80-18db-43f2-b636-f93ac053146d%28Office.15%29.aspx) для целевого объекта. 
    
MAPI предоставляет несколько методов и функций API, возвращающих указатели на реализации интерфейса. Например, клиенты могут вызвать метод [IMAPISession::GetMsgStoresTable,](imapisession-getmsgstorestable.md) чтобы получить указатель на объект таблицы, который предоставляет доступ к сведениям о поставщике хранилищ сообщений через интерфейс [IMAPITable : IUnknown.](imapitableiunknown.md) Поставщики служб могут вызвать функцию API [CreateTable,](createtable.md) чтобы получить указатель на объект данных таблицы. Если функция или метод недоступны, а клиенты или поставщики услуг уже имеют указатель на объект, они могут вызвать метод **QueryInterface** объекта, чтобы получить указатель на другую реализацию интерфейса объекта.
