# <a name="formatted-text-in-mapi"></a>Форматированный текст в MAPI

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Текст сообщения можно хранить и передавать с помощью обычного текста или форматного текста. Форматированный текст улучшает текст сообщения, изменяя его внешний вид с помощью, например, одного или более шрифтов, размеров шрифтов или цветов текста. Рекомендуется, чтобы все клиенты и по возможности все поставщики магазинов сообщений поддержали форматированный текст. Поддержка форматируемого текста в сообщениях повышает ценность за счет повышения читаемости сообщений и упростить и повысить эффективность обработки сообщений.
  
Форматированный текст можно реализовать различными способами. Наиболее распространенным способом является формат "Богатый текст" (RTF). MAPI определяет три передающихся свойства для размещения текстовых данных сообщений: **PR_BODY** [(PidTagBody)](pidtagbody-canonical-property.md)для обычного текста, **PR_HTML** [(PidTagHtml)](pidtaghtml-canonical-property.md)для HTML и **PR_RTF_COMPRESSED** [(PidTagRtfCompressed)](pidtagrtfcompressed-canonical-property.md)для сжатого текста RTF. Так как форматированная версия текста сообщения может быть в два раза больше, чем версия без форматирования, текст RTF сжимается перед его переносом с сообщением и хранением в **свойстве** PR_RTF_COMPRESSED. Когда пришло время отображать сообщение на экране, оно отпечаток с помощью функции утилиты, предоставляемой MAPI. 
  
MAPI определяет эти два свойства текста сообщения и механизмы преобразования между ними, чтобы клиенты, осведомленные о RTF, могли работать с клиентами и системами обмена сообщениями, которые не поддерживают форматированный текст.
