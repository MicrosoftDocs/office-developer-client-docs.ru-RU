# <a name="implementing-a-configuration-interface-for-message-store-providers"></a>Реализация интерфейса конфигурации для поставщиков хранилищ сообщений

  
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
Поставщики хранилища сообщений не требуется реализовать интерфейс, который позволяет пользователю настраивать поставщика хранилища сообщений на компьютере этого пользователя. Как правило при добавлении поставщика хранилища сообщений профиля пользователя в настроен поставщик хранилища сообщений. Интерфейс конфигурации поставщика хранилища сообщений обычно выполняет задачи, такие как имена пользователей и пароли для защищенного сообщения хранилищ, Выбор пути к необходимые файлы и создание базовый механизм хранения его будет использоваться, если это необходимо.
  
Интерфейс конфигурации, который можно реализовать доступен через дополнительные точки входа в DLL поставщика услуг сообщения. [Служба сообщений](configuring-a-message-service.md)для получения дополнительных сведений см. Интерфейс конфигурации поставщика хранилища сообщений — это единственный пользовательского интерфейса, который должен быть реализован поставщика хранилища сообщений.
