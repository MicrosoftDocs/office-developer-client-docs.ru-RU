# <a name="mapi-address-book-provider-overview"></a>Общие сведения о поставщике MAPI адресной книги
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
Адресная книга поставщиков маркер доступа к данным каталога. Сведения о каталоге состоит из данных для двух типов получателей сообщений: отдельные системы обмена сообщениями пользователями и группами обмена сообщениями пользователей, которые часто адресованное друг с другом в списках рассылки. В зависимости от типа получателя и доступа к адресной книге существует большое число сведения, которые могут быть доступны. Например все поставщиками адресных книг хранить имя, адрес и тип адреса получателя.
  
Каждый поставщик адресной книги организует эти данные с помощью одного или нескольких контейнеров. Количество и структура контейнеров зависит от реализации поставщик адресной книги. К примеру один поставщик адресной книги могут использовать один контейнер для хранения всех данных, другой могут использовать один верхнего уровня контейнер, в котором размещается субконтейнеров и третий могут использовать несколько контейнеров верхнего уровня, каждый субконтейнеров создаст. Иерархия контейнеров адресной книги может быть весьма вложения. Нет ограничений на номер субконтейнеров, которые могут использоваться.
  
На следующем рисунке показана типичная MAPI организация адресной книги.
  
**Организация адресной книги**
  
![Организация адресной книги] (media/amapi_04.gif "Организация адресной книги")
  
MAPI интегрирует все сведения, предоставляемые поставщиками установленных адресной книги в одном адресную книгу, презентацию объединенные представления для клиентского приложения. Встроенная список верхнего уровня контейнеров, отображаемые каждым поставщиками установленных адресной книги. Большинство поставщиками адресных книг предоставлять только несколько контейнеров (обычно один-три) на верхнем уровне для включения в верхний уровень интегрированной адресной книги MAPI. Например поставщика адресных книг могут быть недоступны «Все пользователи» и «Локальные пользователи» как два контейнера верхнего уровня.
  
Пользователи клиентских приложений можно просматривать содержимое контейнеров адресной книги и, в некоторых случаях, измените значение параметра. Контейнеры адресной книги может создаваться с использованием различные уровни доступа, в зависимости от доступа к адресной книге.
