# <a name="mapi-message-classes"></a><span data-ttu-id="5bb07-103">Классы сообщений MAPI</span><span class="sxs-lookup"><span data-stu-id="5bb07-103">MAPI Message Classes</span></span>

  
  
<span data-ttu-id="5bb07-104">**Область применения**: Outlook 2013 | Outlook 2016</span><span class="sxs-lookup"><span data-stu-id="5bb07-104">**Applies to**: Outlook 2013 | Outlook 2016</span></span> 
  
<span data-ttu-id="5bb07-105">Каждое сообщение имеет свойство класса **PR_MESSAGE_CLASS** [(PidTagMessageClass),](pidtagmessageclass-canonical-property.md)которое определяет тип, цель или содержимое сообщения.</span><span class="sxs-lookup"><span data-stu-id="5bb07-105">Every message has a message class property, **PR_MESSAGE_CLASS** ([PidTagMessageClass](pidtagmessageclass-canonical-property.md)), which identifies the type, purpose, or content of the message.</span></span> <span data-ttu-id="5bb07-106">**PR_MESSAGE_CLASS** это обязательное свойство для всех новых сообщений.</span><span class="sxs-lookup"><span data-stu-id="5bb07-106">**PR_MESSAGE_CLASS** is a required property on all new messages.</span></span> <span data-ttu-id="5bb07-107">Класс сообщения определяет форму, используемую для вручия сообщения пользователю, и папку для размещения входящих сообщений.</span><span class="sxs-lookup"><span data-stu-id="5bb07-107">A message's class determines the form that is used to present the message to the user and the folder for placing incoming messages.</span></span> 
  
<span data-ttu-id="5bb07-108">Классы сообщений — это строки символов, которые содержат символы ASCII от 32 до 127 и делимитированы по периодам, но не могут закончиться периодом.</span><span class="sxs-lookup"><span data-stu-id="5bb07-108">Message classes are case-sensitive character strings that contain ASCII characters 32 through 127 and are delimited by periods, but they cannot end with a period.</span></span> <span data-ttu-id="5bb07-109">Каждая строка представляет уровень подклассинга, и количество разрешенных уровней не ограничивается.</span><span class="sxs-lookup"><span data-stu-id="5bb07-109">Each string represents a level of subclassing, and there is no limit to the number of levels allowed.</span></span> 
  
<span data-ttu-id="5bb07-110">Например, большинство сообщений, которые клиентские приложения отправляют и получают, попадают в класс сообщений **IPM** — широкую категорию, описываемую все межличностные сообщения (то есть сообщения, которые предназначены для чтения пользователем, а не программным путем компьютером).</span><span class="sxs-lookup"><span data-stu-id="5bb07-110">For example, most messages that client applications send and receive fall into the **IPM** message class, a broad category that describes all interpersonal messages (that is, messages that are meant to be read by a human user, rather than programmatically by a computer).</span></span> <span data-ttu-id="5bb07-111">Поставщики магазина сообщений более точно описывают сообщение IPM, создав **подкласс IPM.**</span><span class="sxs-lookup"><span data-stu-id="5bb07-111">Message store providers more precisely describe an IPM message by creating an **IPM** subclass.</span></span> <span data-ttu-id="5bb07-112">Подкласс **IPM** наследует свойства класса **сообщений IPM.**</span><span class="sxs-lookup"><span data-stu-id="5bb07-112">The **IPM** subclass inherits the properties of the **IPM** message class.</span></span> <span data-ttu-id="5bb07-113">Подклассы класса **IPM** называются путем примещения других строк символов к идентификатору **IPM, например IPM. Примечание,** чтобы описать сообщение заметки и **IPM. Свяжитесь** с вами, чтобы описать контактное сообщение.</span><span class="sxs-lookup"><span data-stu-id="5bb07-113">Subclasses of the **IPM** class are named by concatenating other character strings onto the IPM identifier, such as **IPM.Note** to describe a note message and **IPM.Contact** to describe a contact message.</span></span> 
  
<span data-ttu-id="5bb07-114">Для обработки отображения и управления сообщениями IPM клиенты могут использовать стандартную форму, которую предоставляет MAPI.</span><span class="sxs-lookup"><span data-stu-id="5bb07-114">To handle the display and management of IPM messages, clients can use a standard form that MAPI provides.</span></span> <span data-ttu-id="5bb07-115">Для обработки отображения и управления новыми классами сообщений у вас как у разработчика клиентских приложений есть два варианта:</span><span class="sxs-lookup"><span data-stu-id="5bb07-115">To handle the display and management of new message classes, you as a client application developer have two options:</span></span>
  
1. <span data-ttu-id="5bb07-116">Создать новую форму можно с помощью набора интерфейсов форм, определенных MAPI, которые может использовать стандартный клиент.</span><span class="sxs-lookup"><span data-stu-id="5bb07-116">You can create a new form by using the set of MAPI-defined form interfaces that a standard client can use.</span></span>
    
2. <span data-ttu-id="5bb07-117">Вы можете написать собственный клиент, реализуя полное, самостоятельное приложение.</span><span class="sxs-lookup"><span data-stu-id="5bb07-117">You can write your own client by implementing a complete, standalone application.</span></span> 
    
<span data-ttu-id="5bb07-118">Хотя клиенты  должны PR_MESSAGE_CLASS для каждого исходяходящего сообщения в подкласс IPM или **IPC,** поставщик магазина сообщений несет конечную ответственность за его настройку. </span><span class="sxs-lookup"><span data-stu-id="5bb07-118">Although clients should set the **PR_MESSAGE_CLASS** property for every outgoing message to a subclass of either **IPM** or **IPC**, the message store provider has the ultimate responsibility for setting it.</span></span> <span data-ttu-id="5bb07-119">Поэтому если клиент отправляет сообщение без настройки класса сообщения, поставщик магазина сообщений задает ему соответствующее значение по умолчанию для соответствующего типа клиента.</span><span class="sxs-lookup"><span data-stu-id="5bb07-119">Therefore, if a client sends a message without setting its message class, the message store provider sets it to the appropriate default value for the appropriate type of client.</span></span> <span data-ttu-id="5bb07-120">Класс сообщений по умолчанию для клиентов межличностных сообщений **— IPM;** класс сообщений по умолчанию для клиентов межпроцессной связи **— это IPC.**</span><span class="sxs-lookup"><span data-stu-id="5bb07-120">The default message class for interpersonal messaging clients is **IPM**; the default message class for interprocess communication clients is **IPC**.</span></span> 
  
<span data-ttu-id="5bb07-121">Классы сообщений имеют ограничение длины в 255 символов.</span><span class="sxs-lookup"><span data-stu-id="5bb07-121">Message classes have a length restriction of 255 characters.</span></span> <span data-ttu-id="5bb07-122">Однако классы сообщений не должны превышать 127 символов для поддержки классов сообщений, используемых в отчетах.</span><span class="sxs-lookup"><span data-stu-id="5bb07-122">However, message classes should not exceed 127 characters to support the message classes used in reports.</span></span> <span data-ttu-id="5bb07-123">Классы сообщений отчетов основаны на классе исходного сообщения с двумя дополнениями: префиксом и суффиксом.</span><span class="sxs-lookup"><span data-stu-id="5bb07-123">Report message classes are based on the class of the original message, with two additions: a prefix and a suffix.</span></span> <span data-ttu-id="5bb07-124">В отчете префикса указывается, что сообщение является отчетом, а суффикс указывает тип отчета: DR (отчет о доставке), NDR (отчет о неделивстве), IPNRN (отчет о прочтениях) или IPNNRN (отчет о непрочитанстве).</span><span class="sxs-lookup"><span data-stu-id="5bb07-124">The prefix REPORT indicates that the message is a report, and the suffix indicates the type of report: DR (delivery report), NDR (nondelivery report), IPNRN (read report), or IPNNRN (nonread report).</span></span> <span data-ttu-id="5bb07-125">Обратите внимание, что эти ограничения длины даются в символах; на платформах, где используется набор символов с двойным числом, фактическое число byte может быть выше.</span><span class="sxs-lookup"><span data-stu-id="5bb07-125">Note that these length restrictions are given in characters; on platforms that use a double-byte character set, the actual byte count might be higher.</span></span> 
  
<span data-ttu-id="5bb07-126">Поставщики магазинов сообщений должны возвращать MAPI_E_INVALID_PARAMETER из реализации метода [IMAPIProp::SetProps,](imapiprop-setprops.md) когда клиент пытается назначить строку, превышаю допустимую для класса сообщений.</span><span class="sxs-lookup"><span data-stu-id="5bb07-126">Message store providers should return MAPI_E_INVALID_PARAMETER from their [IMAPIProp::SetProps](imapiprop-setprops.md) method implementations when a client attempts to assign a string that exceeds the allowable limit for their message class.</span></span>
