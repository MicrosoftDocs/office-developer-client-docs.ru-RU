# <a name="display-tables"></a>Таблицы отображения

  
  
**Относится к**: Outlook 
  
Отображение таблице описывается способ отображения определенного типа диалогового окна — один один или несколько страниц с вкладками свойство, выделенным для отображения и возможно редактирование одного или нескольких свойств. Связанные с каждой отображения таблица — [IMAPIProp: IUnknown](imapipropiunknown.md) реализация интерфейса. Реализация **IMAPIProp** поддерживает свойство данные, представленные в диалоговом окне. 
  
Строки в таблице отображения представляют элементы управления или объекта пользовательского интерфейса, которые отображаются в диалоговом окне. MAPI определяет множество типов элементов управления, некоторые из которых статических значений, и некоторые из которых динамические значения, которые пользователь может изменить. Элементы управления могут быть связаны со свойствами, задаваемые с реализацией **IMAPIProp** . Когда пользователь изменяет значение можно изменить элемент управления, обновляется соответствующего свойства. 
  
Поставщики услуг реализовать отображения таблицы и интерфейса **IMAPIProp** . Создание таблицы отображения аналогична написание программы с помощью сценариев. Поставщиков услуг можно создать таблицу отображаемое с: 
  
- Вызов функции [BuildDisplayTable](builddisplaytable.md) . 
    
    - Или -
    
- Включая пользовательского кода, которая заполняет в таблице отображения напрямую с помощью объекта данных в таблице — объект, который поддерживает [ITableData: IUnknown](itabledataiunknown.md) интерфейса. 
    
Функция **BuildDisplayTable** объединяет сведения из таблицы структуры отображения с визуальных элементов из ресурс для построения строк таблицы для отображения. Функция возвращает указатель на [IMAPITable: IUnknown](imapitableiunknown.md) интерфейса реализации и, если запрос, указатель на реализацию интерфейса **ITableData** . 
  
Создание отображения таблицы с помощью **BuildDisplayTable** проста и упрощает обслуживание при изменении визуальных элементов отображения. Тем не менее поставщики услуг, которые не будет использоваться **BuildDisplayTable** можно создать отображения таблицы с помощью пользовательского кода, который использует методы **ITableData**. Например поставщиков услуг, имеющих имеющуюся структуру шаблона для их страницы свойств может потребоваться создать пользовательский код, а не использовать **BuildDisplayTable**.
  
Существует несколько способов поставщиков услуг можно реализовать интерфейс свойство для их отображения в таблице. К ним относятся:
  
- Предоставление Стандартная [IMAPIProp: IUnknown](imapipropiunknown.md) реализации. 
    
- Предоставление оболочку реализацию **IMAPIProp** , которая включает в себя специальная обработка перед внесением стандартные вызовы. 
    
- Предоставление [IPropData: IMAPIProp](ipropdataimapiprop.md) реализации. 
    
Тип реализации зависит от характеристик данных для отображения и поставщика услуг ответственность. К примеру Если существует неявной связи между данными в два элемента управления Правка и элемента управления изменяется, реализации **IMAPIProp** необходимо изменить значение другого элемента управления соответствующим образом. 
  
Отображение таблиц имеют следующие свойства в своих набор необходимых столбцов:
  
|||
|:-----|:-----|
|**PR_XPOS** ([PidTagXCoordinate](pidtagxcoordinate-canonical-property.md))  <br/> |**PR_YPOS** ([PidTagYCoordinate](pidtagycoordinate-canonical-property.md))  <br/> |
|**PR_DELTAX** ([PidTagDeltaX](pidtagdeltax-canonical-property.md))  <br/> |**PR_DELTAY** ([PidTagDeltaY](pidtagdeltay-canonical-property.md))  <br/> |
|**PR_CONTROL_TYPE** ([PidTagControlType](pidtagcontroltype-canonical-property.md))  <br/> |**PR_CONTROL_FLAGS** ([PidTagControlFlags](pidtagcontrolflags-canonical-property.md))  <br/> |
|**PR_CONTROL_STRUCTURE** ([PidTagControlStructure](pidtagcontrolstructure-canonical-property.md))  <br/> |**PR_CONTROL_ID** ([PidTagControlId](pidtagcontrolid-canonical-property.md))  <br/> |
   
 **PR_XPOS** и **PR_YPOS** указать координаты X и Y левого верхнего угла элемента управления. Горизонтальная единицы являются 1/4 значений базового ширины диалогового окна. Вертикальная единиц измерения, 1/8 единицы высота основы диалогового окна. Windows вычисляет базовых единицах текущего диалогового окна в высоту и ширину текущего шрифта системы. Координаты указываются по отношению к источнику область страницы свойств. Размер страницы свойств ограничен приблизительно 200 с 180 единицы диалогового окна. 
  
 **PR_DELTAX** и **PR_DELTAY** — ширину и высоту элемента управления. Это ULONG значения. Единицы ширины являются 1/4 значений базового ширины диалогового окна. Высота указываются 1/8 единицы высота основы диалогового окна. Координаты указываются по отношению к источнику элемента управления. 
  
Четыре свойства описания различных характеристик элемента управления. **PR_CONTROL_TYPE** указывает тип элемента управления. MAPI определяет двенадцать типы элементов управления, каждый из которых другой набор атрибутов. Эти атрибуты, описаны в свойстве флаги **PR_CONTROL_FLAGS**. Примеры атрибутов: ли элемент управления редактирования или обязательные. 
  
Структура элемента управления, **PR_CONTROL_STRUCTURE**, содержит сведения, относящиеся к определенным типом элемента управления. С другой структурой описан каждый тип элемента управления. Например элементы управления поля ввода, описаны со структурой [DTBLEDIT](dtbledit.md) . **DTBLEDIT** структуры содержат элементы этого списка, число и определенные типы символов, которые можно поместить в элемент управления и тега свойство, которое определяет свойство, значение которого будет отображаться в элементе управления. **PR_CONTROL_STRUCTURE** сохраняется как двоичного свойства. 
  
Идентификатор элемента управления **PR_CONTROL_ID**однозначно определяет элемент управления в диалоговое окно, описанные в таблице отображения. **PR_CONTROL_ID** имеет значение из значений, помещенный в *lpbNotif* и *cbNotif* члены структуры [DTCTL](dtctl.md) , которая используется для создания таблицы отображения **BuildDisplayTable** . Так как MAPI в некоторых случаях объединяет отображать таблицы, идентификатор в **PR_CONTROL_ID** всегда должно быть уникальным. Как правило поставщики назначить структура [GUID](guid.md) **PR_CONTROL_ID** для обеспечения его уникальности. Свойство **PR_CONTROL_ID** включается в структуре [TABLE_NOTIFICATION](table_notification.md) при создании отображать уведомления в таблице. 
  
Дополнительные сведения о таблицах отображения можно [Отображения реализации таблицы](display-table-implementation.md) и [О отображение уведомлений в таблице](about-display-table-notifications.md).
