# <a name="pidtagresourceflags-canonical-property"></a>Каноническое свойство PidTagResourceFlags

  
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
Содержит битовую маску флагов для поставщиков и службы сообщений.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_RESOURCE_FLAGS  <br/> |
|Идентификатор:  <br/> |0x3009  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Распространенные MAPI  <br/> |
   
## <a name="remarks"></a>Замечания

Это свойство описывает характеристики службы сообщений, поставщик службы или состояние объекта. Флаги, заданные для этого свойства зависит от его контекста. К примеру некоторые флаги являются допустимыми только для объектов состояния и другие флаги только для столбцов в таблице службы сообщений. 
  
Флаги используются три классов: static, изменяемые и динамические. Статические флаги задаются MAPI на основе данных в файл Mapisvc.inf. INF и никогда не изменяется. Можно изменить флаги задаются MAPI в файл Mapisvc.inf. INF, но может быть изменен. Динамические флаги можно задать и сброс методами MAPI.
  
Для службы сообщений одной или нескольких из следующих флагов можно задать в этом свойстве.
  
SERVICE_CREATE_WITH_STORE 
  
> Зарезервировано. Не используйте.
    
SERVICE_DEFAULT_STORE 
  
> Динамические. Служба сообщений содержит хранилище по умолчанию. Пользовательский интерфейс должен отображаться запросом на подтверждение перед удалением или перемещением этой службы из профиля. 
    
SERVICE_NO_PRIMARY_IDENTITY 
  
> Static. Флаг уровня службы, который должен содержать значение, указывающее, что ни один из поставщиков службы сообщений можно использовать для предоставления идентификатора. Этот флаг или SERVICE_PRIMARY_IDENTITY должен быть набор, но не оба.
    
SERVICE_PRIMARY_IDENTITY 
  
> Можно изменить. Соответствующая служба сообщение содержит поставщика, используемый для основного удостоверения для данного сеанса. Используйте [IMsgServiceAdmin::SetPrimaryIdentity](imsgserviceadmin-setprimaryidentity.md) , чтобы установить этот флаг. Этот флаг или SERVICE_NO_PRIMARY_IDENTITY должен быть набор, но не оба. 
    
SERVICE_SINGLE_COPY 
  
> Static. Любая попытка создания или скопируйте этой службы сообщений в профиль, где уже существует служба завершится с ошибкой. Для создания одной копии сообщения службы добавьте свойство **PR_RESOURCE_FLAGS** в раздел службы в файл Mapisvc.inf. INF и назначьте этот флаг. 
    
Для поставщика услуг одной или нескольких из следующих флагов можно задать в **PR_RESOURCE_FLAGS**:
  
HOOK_INBOUND 
  
> Static. Обработчик очереди необходим для обработки входящих сообщений.
    
HOOK_OUTBOUND 
  
> Static. Обработчик очереди должен обработки исходящих сообщений. 
    
STATUS_DEFAULT_OUTBOUND 
  
> Можно изменить. Это удостоверение должен применяться для исходящих сообщений, если профиль содержит несколько экземпляров данного поставщика транспорта. Это может произойти, если несколько экземпляров одной транспорт отображаются в профиле.
    
STATUS_DEFAULT_STORE 
  
> Можно изменить. В этом хранилище сообщений — это хранилище по умолчанию для профиля. 
    
STATUS_NEED_IPM_TREE 
  
> Динамические. Стандартные папки хранилища, включая корневую папку электронной почты — это сообщение (IPM), не было проверено. MAPI задает и удаляет этот флаг. 
    
STATUS_NO_DEFAULT_STORE 
  
> Static. В этом хранилище сообщений не способен стать хранилище сообщений по умолчанию для профиля.
    
STATUS_NO_PRIMARY_IDENTITY 
  
> Static. Этот поставщик не предоставил удостоверения в его строке состояния. Этот флаг или STATUS_PRIMARY_IDENTITY должно иметь значение.
    
STATUS_OWN_STORE 
  
> Static. Данный транспортный протокол тесно связаны с хранилищем сообщений и предоставляющей свойство **PR_OWN_STORE_ENTRYID** ([PidTagOwnStoreEntryId](pidtagownstoreentryid-canonical-property.md)) в строке его состояния.
    
STATUS_PRIMARY_IDENTITY 
  
> Можно изменить. Этот поставщик передает основной идентификатор для сеанса; Идентификатор записи для передачи идентификатора объекта возвращается из [IMAPISession::QueryIdentity](imapisession-queryidentity.md). Это флаг или **STATUS_NO_PRIMARY_IDENTITY** должно иметь значение. 
    
STATUS_PRIMARY_STORE 
  
> Можно изменить. В этом хранилище сообщений будет использоваться при входе в клиентское приложение. После запуска этого хранилища следует задать в качестве хранилища по умолчанию для профиля. 
    
STATUS_SECONDARY_STORE 
  
> Можно изменить. Хранилища будет использоваться, если основного хранилища недоступен при входе в клиентское приложение. После запуска этого хранилища следует задать в качестве хранилища по умолчанию для профиля. 
    
STATUS_SIMPLE_STORE 
  
> Динамические. В этом хранилище сообщений используется Simple MAPI как его хранилище сообщений по умолчанию.
    
STATUS_TEMP_SECTION 
  
> Динамические. Хранилища не должны быть опубликованы в таблице хранилища сообщений и будет удален из профиля после выхода из системы. 
    
STATUS_XP_PREFER_LAST 
  
> Static. В этом транспорта ожидает последнего транспорта, установите флажок, чтобы отправить сообщение, когда несколько поставщиков транспорта существует возможность передачи сообщения.
