# <a name="pidtagresourceflags-canonical-property"></a>Каноническое свойство PidTagResourceFlags

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит битмаску флагов для служб и поставщиков сообщений.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_RESOURCE_FLAGS  <br/> |
|Идентификатор:  <br/> |0x3009  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |MAPI общие  <br/> |
   
## <a name="remarks"></a>Примечания

В этом свойстве описываются характеристики службы сообщений, поставщика услуг или объекта состояния. Флаги, заданной для этого свойства, зависят от контекста. Например, некоторые флаги допустимы только для объектов состояния, а другие — только для столбцов в таблице службы сообщений. 
  
Флаги имеют три класса: статический, модификабельный и динамический. Статические флаги устанавливаются MAPI из данных в MAPISVC. INF и никогда не менялся. Modifiable flags are set by MAPI from MAPISVC. INF, но может быть впоследствии изменен. Динамические флаги можно установить и сбросить с помощью методов MAPI.
  
Для службы сообщений в этом свойстве можно установить один или несколько следующих флагов:
  
SERVICE_CREATE_WITH_STORE 
  
> Зарезервировано. Не следует использовать.
    
SERVICE_DEFAULT_STORE 
  
> Динамический. Служба сообщений содержит хранилище по умолчанию. Перед удалением или удалением этой службы из профиля должен отображаться пользовательский интерфейс. 
    
SERVICE_NO_PRIMARY_IDENTITY 
  
> Статическое. Флаг уровня службы, который должен указывать, что ни один из поставщиков в службе сообщений не может использоваться для обеспечения удостоверения. Этот флаг или SERVICE_PRIMARY_IDENTITY должен быть заданной, но не оба.
    
SERVICE_PRIMARY_IDENTITY 
  
> Модификабель. Соответствующая служба сообщений содержит поставщика, используемого для основного удостоверения для этого сеанса. Для этого флага используйте [объект IMsgServiceAdmin::SetPrimaryIdentity.](imsgserviceadmin-setprimaryidentity.md) Этот флаг или SERVICE_NO_PRIMARY_IDENTITY должен быть установлен, но не оба. 
    
SERVICE_SINGLE_COPY 
  
> Статическое. Любая попытка создать или скопировать эту службу сообщений в профиль, в котором уже существует служба, будет неудачной. Чтобы создать единую службу копирования, **добавьте свойство PR_RESOURCE_FLAGS** в раздел службы в MAPISVC. INF и установите этот флаг. 
    
Для поставщика услуг один или несколько из следующих флагов можно установить в **PR_RESOURCE_FLAGS:**
  
HOOK_INBOUND 
  
> Статическое. Крюк шпалер должен обрабатывать входящие сообщения.
    
HOOK_OUTBOUND 
  
> Статическое. Крюк шпалер должен обрабатывать исходящие сообщения. 
    
STATUS_DEFAULT_OUTBOUND 
  
> Модификабель. Это удостоверение следует применять к исходящие сообщения, если профиль содержит несколько экземпляров этого поставщика транспорта. Это может произойти, если в профиле появится несколько экземпляров одного поставщика транспорта.
    
STATUS_DEFAULT_STORE 
  
> Модификабель. Этот магазин сообщений — хранилище по умолчанию для профиля. 
    
STATUS_NEED_IPM_TREE 
  
> Динамический. Стандартные папки в этом хранилище сообщений, включая корневую папку межличностного сообщения (IPM), еще не проверены. MAPI устанавливает и очищает этот флаг. 
    
STATUS_NO_DEFAULT_STORE 
  
> Статическое. Этот магазин сообщений не может стать хранилищем сообщений по умолчанию для профиля.
    
STATUS_NO_PRIMARY_IDENTITY 
  
> Статическое. Этот поставщик не дает удостоверение в строке состояния. Этот флаг или STATUS_PRIMARY_IDENTITY должен быть заданной.
    
STATUS_OWN_STORE 
  
> Статическое. Этот поставщик транспорта тесно соединен с хранилищем сообщений и обставирует свойство[PR_OWN_STORE_ENTRYID (PidTagOwnStoreEntryId)](pidtagownstoreentryid-canonical-property.md)в строке состояния. 
    
STATUS_PRIMARY_IDENTITY 
  
> Модификабель. Этот поставщик дает основное удостоверение сеанса; идентификатор входа для объекта, обставленного удостоверением, возвращается из [IMAPISession::QueryIdentity.](imapisession-queryidentity.md) Этот флаг или **STATUS_NO_PRIMARY_IDENTITY** должен быть заданной. 
    
STATUS_PRIMARY_STORE 
  
> Модификабель. Этот магазин сообщений должен использоваться при входе клиентского приложения. После открытия этот магазин должен быть заданной как хранилище по умолчанию для профиля. 
    
STATUS_SECONDARY_STORE 
  
> Модификабель. Этот магазин сообщений будет использоваться, если основной магазин не доступен при входе в систему клиентского приложения. После открытия этот магазин должен быть заданной как хранилище по умолчанию для профиля. 
    
STATUS_SIMPLE_STORE 
  
> Динамический. Этот хранилище сообщений будет использоваться simple MAPI в качестве магазина сообщений по умолчанию.
    
STATUS_TEMP_SECTION 
  
> Динамический. Этот магазин сообщений не должен публиковаться в таблице хранения сообщений и будет удален из профиля после входа. 
    
STATUS_XP_PREFER_LAST 
  
> Статическое. Этот транспорт будет последним транспортом, выбранным для отправки сообщения, когда несколько поставщиков транспорта могут передать сообщение.
