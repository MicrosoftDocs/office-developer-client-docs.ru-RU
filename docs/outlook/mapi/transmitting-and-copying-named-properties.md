# <a name="transmitting-and-copying-named-properties"></a>Передача и копирование именных свойств

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
При отправлении, перемещении или копировании именоваемого свойства имя остается постоянным, но идентификатор должен измениться, чтобы соответствовать сопоставлению конечного объекта. Единственным исключением из этого правила является то, что источник и назначение имеют ту же подпись сопоставления, что делает повторное сопоставление ненужным.
  
Поставщик транспорта несет ответственность за то, чтобы переназначить имена переданных именных свойств соответствующим идентификаторам, которые работают в месте назначения. Отправляя поставщик транспорта не может знать правильное сопоставление в месте назначения; он должен передавать имена и полагаться на поставщика транспорта, который получает их, чтобы соедеть их с идентификаторами, которые работают. Реализация TNEF MAPI обрабатывает повторное применение именуемого свойства для поставщиков транспорта. Поставщики транспорта могут либо обрабатывать перенапевку вручную, либо использовать реализацию TNEF. 
  
Аналогичное переначинение именуемого свойства должно происходить при копировании этих свойств между хранилищами сообщений. Однако, так как поставщики пунктов хранения сообщений могут получить имя для сопоставления идентификаторов назначения, они могут сразу же переназначить свойства и не должны полагаться на конечное хранилище сообщений.
