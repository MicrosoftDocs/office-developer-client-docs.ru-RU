# <a name="about-display-table-notifications"></a>Сведения об уведомлениях касательно таблиц отображения

**Относится к**: Outlook 2013 | Outlook 2016 
  
Уведомления в таблице отображения отправляются поставщиком услуг, отвечающим за создание таблицы отображения в MAPI. MAPI регистрирует эти уведомления, вызывая метод [IMAPITable::Advise](imapitable-advise.md) таблицы отображения и указав событие изменения таблицы. 
  
Как и все уведомления таблиц, уведомления [](table_notification.md) о таблицах включают TABLE_NOTIFICATION структуру. Важны только **члены ulTableEvent** и **propIndex** этой структуры; другие члены игнорируются. Для **члена ulTableEvent** установлено значение TABLE_ROW_MODIFIED а для члена **propIndex** — значение столбца **PR_CONTROL_ID** [(PidTagControlId)](pidtagcontrolid-canonical-property.md)в соответствующей строке. MAPI отвечает на уведомление, вызывая метод [IMAPIProp::GetProps](imapiprop-getprops.md) для свойства, отображаемого в этом объекте управления, и отображая новое значение. 
  
Поставщик услуг может использовать уведомления таблицы отображения для координации изменений связанных элементов управления в диалоговом окне. Например, если реализации интерфейса свойств требуется обновить одно или несколько полей в диалоговом окне ( возможно, в ответ на другой контроль, который установил флаг DT_SET_IMMEDIATE в свойстве **PR_CONTROL_FLAGS** ([PidTagControlFlags)](pidtagcontrolflags-canonical-property.md)), это может создать уведомление таблицы отображения. Уведомление таблицы отображения может оповещать реализацию интерфейса свойств о том, что значение одного или более элементов управления необходимо перечитать из-за внося изменения или внешнего события. 
  
Поставщик услуг может издать уведомления таблицы отображения, выдав:
  
- Вызов [ITableData::HrNotify](itabledata-hrnotify.md), если таблица отображения была построена с объектом данных таблицы.
    
    - Или -
    
- Использование собственного кода, если таблица отображения была построена с реализацией **IMAPITable поставщика.** 
    
MAPI реагирует на отображение уведомлений таблицы, когда это необходимо, перечитав значение управления из реализации интерфейса свойства. В следующей таблице описываются сведения о том, как MAPI обрабатывает уведомления для определенных типов элементов управления.
  
|**Control**|**Действие MAPI**|
|:-----|:-----|
|Кнопка  <br/> |Вызывает [IMAPIProp::OpenProperty](imapiprop-openproperty.md)для получения объекта управления с помощью свойства, представленного членом **ulPRControl** структуры [DTBLBUTTON,](dtblbutton.md) если вызов не был ранее. Вызывает [IMAPIControl::GetState](imapicontrol-getstate.md) объекта управления, чтобы определить, следует ли включить кнопку и включить или отключить кнопку соответствующим образом.  <br/> |
|Флажок  <br/> |Перечитает значение для члена **ulPRPropertyName.**  <br/> |
|Поле со списком  <br/> |Повторно открывает таблицу, связанную с членом **ulPRTableName** структуры [DTBLCOMBOBOX.](dtblcombobox.md) Перечитает все строки, включая значение для **члена ulPRPropertyName.**  <br/> |
|Поле "Список в выпадаемом списке"  <br/> |Повторно открывает таблицу, связанную с членом **ulPRTableName** структуры [DTBLDDLBX,](dtblddlbx.md) и перечитает все строки. Вызывает [IMAPIProp::GetProps,](imapiprop-getprops.md) чтобы получить значения свойств, хранимые в **членах ulPRDisplayProperty** и **ulPRSetProperty.**  <br/> |
|Редактирование  <br/> |Перечитает свойство и повторное воспроизведение.  <br/> |
|Поле группы  <br/> |Игнорирует уведомление.  <br/> |
|Метка  <br/> |Игнорирует уведомление.  <br/> |
|Список выбора нескольких вариантов  <br/> |Если один из столбцов является идентификатором записи, обновляет список. Соответствующий объект не закрывается и не перезагружается.  <br/> |
|Поле со списком одного выбора  <br/> |Считывая свойство set, пытается его идентифицировать.  <br/> |
|Поле со списком с несколькими очками  <br/> |Перечитает свойство и повторно вмеет список.  <br/> |
|Вкладка страницы  <br/> |Для этого управления нет уведомлений; все является статическим.  <br/> |
|Кнопка "Радио"  <br/> |Перечитает свойство, связанное с кнопкой и сохраненное в элементе **ulPropTag** структуры [DTBLRADIOBUTTON,](dtblradiobutton.md) и делает соответствующий выбор с помощью элементов управления.  <br/> |
