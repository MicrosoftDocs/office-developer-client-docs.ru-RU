# <a name="getting-and-setting-multiple-properties"></a>Получение и установка нескольких свойств

**Область применения**: Outlook 2013 | Outlook 2016 
  
При получении и настройке максимально возможного количества свойств с наименьшим числом вызовов удаленная активность сокращается, а накладные расходы, связанные с каждым свойством, уменьшаются. Несмотря на то, что поставщики услуг пытаются собрать свойства перед вызовом удаленной процедуры для иска или изменения, вы можете оптимизировать эти усилия, запросив несколько свойств для начала.
  
Например, если вы работаете со списками маршрутизации, описывающих будущих получателей с именами свойств, принадлежащими определенным наборам свойств, обработать всех получателей двумя вызовами. Используйте один вызов [в IMAPIProp::GetIDsFromNames](imapiprop-getidsfromnames.md) для получения идентификаторов для всех свойств получателя, а другой вызов [в IMAPIProp::GetProps](imapiprop-getprops.md) для получения всех значений. Альтернатива, вызываемая **в GetIDsFromNames** с последующим вызовом **getProps** для каждого получателя, гораздо менее эффективна.
