# <a name="getting-and-setting-multiple-properties"></a>Получение и установка нескольких свойств

**Относится к**: Outlook 2013 | Outlook 2016 
  
Получив и устанавливая как можно больше свойств с наименьшим количеством вызовов, удаленная активность — куртаилед, а нагрузка, связанная с каждым свойством, уменьшается. Несмотря на то, что поставщики услуг пытаются собрать свойства перед выполнением удаленных вызовов процедур для получения или изменения, можно оптимизировать эту работу, запросив несколько свойств.
  
Например, при работе со списками маршрутизации, в которых описаны будущие получатели с именованными свойствами, принадлежащими определенным наборам свойств, необходимо обработать всех получателей с двумя вызовами. Используйте один вызов [IMAPIProp:: жетидсфромнамес](imapiprop-getidsfromnames.md) , чтобы получить идентификаторы для всех свойств получателей и другой вызов [IMAPIProp:: Prop](imapiprop-getprops.md) для получения всех значений. Кроме того, выполнение вызова **жетидсфромнамес** , за которым следует вызов **PROPS** для каждого получателя, гораздо менее эффективно.
