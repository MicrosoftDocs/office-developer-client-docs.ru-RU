# <a name="pidtagsentrepresentingsmtpaddress-canonical-property"></a>Каноническое свойство PidTagSentRepresentingSmtpAddress

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит электронный адрес электронной почты Simple Mail Transport Protocol (SMTP) для пользователя обмена сообщениями, который представлен отправитель.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_SENT_REPRESENTING_SMTP_ADDRESS, PR_SENT_REPRESENTING_SMTP_ADDRESS_A, PR_SENT_REPRESENTING_SMTP_ADDRESS_W  <br/> |
|Идентификатор:  <br/> |0x5D02  <br/> |
|Тип данных:  <br/> |PT_UNICODE, PT_STRING8  <br/> |
|Область:  <br/> |Адрес  <br/> |
   
## <a name="remarks"></a>Примечания

Это свойство является примером свойств адресов для пользователя обмена сообщениями, который представляет отправитель. Когда клиентская заявка отправляет сообщение от имени другого клиента, оно должно установить все представленные свойства отправитель к значениям для этого клиента. Пользователь обмена сообщениями, отправляющий сообщения от своего имени, обычно оставляет представленные свойства отправитель неустановленными.
  
Исходяющий поставщик транспорта всегда должен оставить это свойство без изменений, если оно было заданной клиентом отправки. Если оно не установлено, поставщик транспорта  должен установить его в свойство[PR_SENDER_SMTP_ADDRESS (PidTagSenderSmtpAddress)](pidtagsendersmtpaddress-canonical-property.md)в исходящие копии сообщения и оставить его неустановленным в локальной копии.
