# <a name="upload-table-state"></a>Upload Состояние таблицы

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время состояния таблицы загрузки машины состояния репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояния:  <br/> |**LR_SYNC_UPLOAD_TABLE** <br/> |
|Связанная структура данных:  <br/> |**[UPTBL](uptbl.md)** <br/> |
|Из этого состояния:  <br/> |[Синхронизация состояния содержимого](synchronize-contents-state.md) <br/> |
|Для этого состояния:  <br/> |[Upload, загрузите](upload-message-state.md) [состояние](upload-delete-status-state.md) [удаления,](upload-read-status-state.md)загрузите состояние состояния чтения или синхронизуйте состояние содержимого  <br/> |
   
> [!NOTE]
> Машина состояния репликации — это детерминистический автомат состояния. Клиент, отправляющийся из одного состояния в другое, должен в конечном итоге вернуться в первое из последнего. 
  
## <a name="description"></a>Описание

Это состояние инициирует отправку содержимого папки, указанной в предыдущем состоянии синхронизации содержимого. Папка может быть почтой, календарем, контактами, задачами, заметками или папкой журнала. В этом состоянии Outlook список элементов, которые были добавлены, изменены, перемещены, удалены или помечены как прочитанный, и подготавливает соответствующие внутренние сведения для соответствующего состояния сообщения отправки, загрузки состояния удаления или загрузки состояния состояния чтения.
  
Когда это состояние закончится, Outlook пометит папку как синхронизированную с ее содержимым, чтобы содержимое не было загружено снова до тех пор, пока не будет сделано другое изменение. Локальный магазин возвращается в состояние синхронизации содержимого.
