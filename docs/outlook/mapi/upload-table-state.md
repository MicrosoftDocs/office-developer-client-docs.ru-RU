# <a name="upload-table-state"></a>Состояние отправки таблицы

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время состояния отправки таблицы конечного автомата репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояния:  <br/> |**LR_SYNC_UPLOAD_TABLE** <br/> |
|Связанная структура данных:  <br/> |**[UPTBL](uptbl.md)** <br/> |
|Из этого состояния:  <br/> |[Синхронизация состояния содержимого](synchronize-contents-state.md) <br/> |
|В этом состоянии:  <br/> |[Отправка состояния сообщения,](upload-message-state.md) [отправка состояния удаления,](upload-delete-status-state.md)отправка состояния [чтения](upload-read-status-state.md)или синхронизация состояния содержимого  <br/> |
   
> [!NOTE]
> Этот автомат репликации является детерминируемым. Клиент, выступая из одного состояния в другое, должен в конечном итоге вернуться к одному из последних. 
  
## <a name="description"></a>Описание

Это состояние инициирует отправку содержимого папки, указанной в предыдущем состоянии синхронизации содержимого. Папка может быть почтой, календарем, контактами, задачами, заметками или папкой журнала. В этом состоянии Outlook создает список элементов, которые были добавлены, изменены, перемещены, удалены или помечены как прочитанный, и подготавливает соответствующие внутренние сведения для соответствующего состояния отправки сообщения, отправки состояния удаления или отправки состояния чтения.
  
Когда это состояние закончится, Outlook пометит папку как синхронизированную с ее содержимым, чтобы содержимое не было загружено повторно, пока не будет выполнено другое изменение. Локальное хранилище возвращается в состояние синхронизации содержимого.
