# <a name="upload-table-state"></a>Состояние отправки данных таблицы

  
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описываются двоичных файлов состояния в таблице отправляемых конечного автомата репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояний:  <br/> |**LR_SYNC_UPLOAD_TABLE** <br/> |
|Структура связанных данных:  <br/> |**[UPTBL](uptbl.md)** <br/> |
|Из этого состояния:  <br/> |[Синхронизировать состояние содержимого](synchronize-contents-state.md) <br/> |

> [!NOTE]
> Конечный автомат репликации является детерминированным конечного автомата. Клиент отправления одного состояния в другое должны в конечном счете возврата на первый из последний. 
  
## <a name="description"></a>Описание

Это состояние означает инициирует отправку пользователями содержимого папки, которые были заданы в предыдущем состоянии содержимое синхронизировать. Папка может быть папка почту, календарь, контакты, задачи, заметки или журнала. Во время это состояние означает Outlook создает список элементов, которые были добавлены, изменены, перемещен, удален или помечена как чтения и подготавливает соответствующие внутренние сведения для соответствующего состояния отправить сообщения, передача удалить состояние состояние или отправить состояние чтения состояние.
  
По окончании этого состояния Outlook помечает папки как имеющий синхронизирована, вместе с контентом, чтобы содержимое не будут загружаться еще раз до другого изменения. Локальное хранилище возвращается в состояние содержимое синхронизировать.
