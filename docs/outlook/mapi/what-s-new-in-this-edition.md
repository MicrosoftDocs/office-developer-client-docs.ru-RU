# <a name="whats-new-in-this-edition"></a>Новые возможности в этом выпуске

 
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
В справочнике по MAPI для Microsoft Outlook 2013 добавлена документация по различным новым функциям. 
  
## <a name="new-content"></a>Новое содержимое

Содержимое добавлено для следующих компонентов:
  
- Раздел [Приступая к работе со справкой по MAPI для outlook 2013](getting-started-with-the-outlook-mapi-reference.md) был обновлен для получения подробных сведений о моделях программирования для функций Outlook и MAPI, которые помогут определить API и технологии, наиболее подходящие для ваших потребностей. Ссылки на технические статьи, на которые указывают ссылки, также были пересмотрены в следующих разделах: 
    
  - [Справочник по MAPI для Outlook](outlook-mapi-reference.md)
    
  - [Обзор справочника по MAPI для Outlook](outlook-mapi-reference-overview.md)
    
- **Пример поставщика хранилища сообщений**: [пример кода поставщика упакованного PST-хранилища](message-store-provider-sample.md) теперь пересмотрен для распознавания и размещения Outlook 2013. Более подробную информацию вы найдете в статье предварительно пересмотренный контент в этой статье. 
    
- **Поток автозаполнения**— это раздел [кэша псевдонимов](nickname-cache.md) , ранее **Формат файла Nk2**, который был обновлен в соответствии с изменениями в outlook 2013, а также Outlook 2010. Теперь в следующих разделах приведены сведения о правилах для разработчиков формата файлов с расширением NK2 для Microsoft Outlook 2003/Microsoft Office Outlook 2007 и синтаксического анализа двоичных файлов. Более подробную информацию вы найдете в статье предварительно пересмотренный контент в этой статье.
    
  - [Профили MAPI](mapi-profiles.md)
    
  - [Кэш псевдонимов](nickname-cache.md)
    
  - [Поток автозаполнения](autocomplete-stream.md)
    
- **Интерфейсы**— в разделе [IAddrBook:: OpenEntry](iaddrbook-openentry.md) документируется метод открытия записи адресной книги и возврата указателя на интерфейс, используемый для доступа к нему. Он ранее содержал флаг в параметре *ulFlags* , **MAPI_GAL_ONLY**, который можно использовать для открытия глобального списка адресов (GAL), и был изменен, чтобы включить его определение.
    
- **Свойства**— добавлен раздел **PR_CONVERSATION_KEY** именованное свойство ([PidTagConversationKey каноническое свойство](pidtagconversationkey-canonical-property.md)), который связан с **IPM. Мессажеманажер** сообщения только в Outlook MAPI. В приведенных ниже разделах, относящихся к этой статье и о потоке формата TNEF, были исправлены следующие темы:

## <a name="previously-revised-content"></a>Предварительно пересмотренный контент

Содержимое было добавлено в предыдущих выпусках ссылки на Outlook MAPI для следующих функций:
  
- Microsoft Outlook 2013 предоставляет нетрадиционные сценарии развертывания, такие как параллельные и "нажми и работай". Эти сценарии могут усложнить логику, используемую для загрузки правильной библиотеки MAPI. Теперь разработчики MAPI могут явно ссылаться на функции MAPI и явно ссылаться на заглушку MAPI клиента MAPI по умолчанию (например, Msmapi32. dll для Outlook), не пробегая через библиотеку MAPI и заглушку Windows MAPI. Дополнительные сведения о явной компоновке по сравнению с неявным связыванием приведены [в статье Link to MAPI functions](how-to-link-to-mapi-functions.md). **Библиотека заглушек MAPI**, размещенная на веб-сайте [CodePlex](https://mapistublibrary.codeplex.com/) , предоставляет подстановку для MAPI32. lib, которая поддерживает создание приложений для 32-и 64-разрядной версии MAPI. 
    
- **Поддержка 64-разрядной версии Microsoft Outlook**— справочные статьи по применимым элементам API, которые были обновлены в соответствии с новыми файлами заголовков, поддерживающими 64-разрядный Outlook. Эти файлы заголовков доступны для загрузки в [Outlook 2010: файлы заголовков MAPI](https://www.microsoft.com/downloads/details.aspx?FamilyID=f8d01fc8-f7b5-4228-baa3-817488a66db1). Новый пример кода представлен в разделе [Проверка версии Outlook](how-to-check-the-version-of-outlook.md) , чтобы узнать, как проверить, является ли установленная версия Outlook 64-разрядным приложением Microsoft Outlook 2010 и пересмотрена для Outlook 2013. Если существующее приложение 32-разрядное приложение MAPI будет выполняться в 64-разрядной операционной системе с установленной версией 64-разрядной версии Outlook, вам потребуется перестроить приложение 32-bit в виде 64-разрядного приложения. Дополнительные сведения о поддержке MAPI для 64 — поразрядной версии Outlook приведены [в статье Создание приложений MAPI на платформах с 32 и 64](building-mapi-applications-on-32-bit-and-64-bit-platforms.md)бит.
    
- **Пример поставщика хранилища сообщений**— [Пример поставщика упакованного PST-хранилища](message-store-provider-sample.md) ранее обновлялся для поддержки 64-разрядной архитектуры. В этом примере [Инициализация поставщика упакованного PST-хранилища](initializing-a-wrapped-pst-store-provider.md) теперь расширена для предоставления сведений об "УПАКОВАНных PST-файлах и путях в Юникоде". 
    
- **Поток автозаполнения**— это раздел [кэша псевдонимов](nickname-cache.md) , ранее **Формат файла Nk2**, обновлен с учетом изменений в outlook 2013, а также Outlook 2010. Такие сведения, как список автозаполнения, который представляет собой список имен, отображаемых в полях " **Кому**", **"копия**" и **"Скрытая копия"** , когда пользователь создает сообщение электронной почты, теперь сохраняется в [потоке автозаполнения](autocomplete-stream.md) сообщения на локальном компьютере, а не сохраняется в файл, как в Outlook 2007.

- **Поддержка быстрого завершения работы для клиентов MAPI**— клиенты MAPI теперь могут инициировать быстрое завершение работы и получать уведомления загруженных поставщиков от подсистемы MAPI для минимизации потерь данных от быстрого завершения работы.
