# <a name="imapiformmgrresolvemessageclass"></a>IMAPIFormMgr::ResolveMessageClass

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Разрешает класс сообщения в форму в контейнере формы и возвращает объект сведений о форме для этой формы.
  
```cpp
HRESULT ResolveMessageClass(
  LPCSTR szMsgClass,
  ULONG ulFlags,
  LPMAPIFOLDER pFolderFocus,
  LPMAPIFORMINFO FAR * ppResult
);
```

## <a name="parameters"></a>Параметры

 _Сзмсгкласс_
  
> возврата Строка, которая указывает имя разрешаемого класса сообщений.
    
 _ulFlags_
  
> возврата Битовая маска флагов, определяющих способ разрешения класса сообщения. Можно задать следующий флаг:
    
МАПИФОРМ_ЕКСАКТМАТЧ 
  
> Необходимо разрешить только строки класса сообщений с точным совпадением.
    
 _Пфолдерфокус_
  
> возврата Указатель на папку, в которой находится разрешенное сообщение. Параметр _пфолдерфокус_ может иметь значение null. 
    
 _Ппресулт_
  
> вышли Указатель на указатель на возвращенный объект сведений о форме.

## <a name="remarks"></a>Примечания

Средства просмотра форм вызывают метод **имапиформмгр:: ресолвемессажекласс** , чтобы разрешить класс сообщения в форму в контейнере формы. Объект сведений о форме, возвращенный в параметре _ппресулт_ , обеспечивает дополнительный доступ к свойствам формы, имеющей заданный класс сообщений. 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

Чтобы разрешить класс сообщения в форму, средство просмотра формы передает имя разрешенного класса сообщения, например " `IPM.HelpDesk.Software`". Чтобы принудительно разрешить точное решение (то есть, чтобы запретить разрешение для базового класса класса Message при недоступности точного соответствия сервера форм), можно передать флаг МАПИФОРМ_ЕКСАКТМАТЧ в параметр _ulFlags_ . Если параметр _пфолдерфокус_ имеет значение null, процесс разрешения класса Message не выполняет поиск контейнера папок. 
  
Порядок поиска контейнеров зависит от реализации поставщика библиотеки форм. Поставщик библиотеки форм по умолчанию выполняет поиск сначала в локальном контейнере, а затем в контейнере папки, в которой находится переданная папка, в контейнере личных форм и в контейнере организации.
  
Имена классов сообщений всегда являются строками ANSI, а не Юникодом.
  
Идентификатор класса для разрешенного класса сообщения возвращается в составе объекта сведений о форме. Средство просмотра форм не должно работать в предположении, что идентификатор класса существует в библиотеке OLE до тех пор, пока средство просмотра форм не вызовет метод [имапиформмгр::P репареформ](imapiformmgr-prepareform.md) или [Имапиформмгр:: креатеформ](imapiformmgr-createform.md) .
