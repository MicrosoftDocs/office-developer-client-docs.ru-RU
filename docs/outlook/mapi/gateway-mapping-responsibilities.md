# <a name="gateway-mapping-responsibilities"></a>Обязанности по сопоставлению со шлюзами

**Область применения**: Outlook 2013 | Outlook 2016 
  
Когда шлюз, поддерживающий MAPI, получает сообщение, содержащее именованные свойства, в одном из специальных наборов свойств, которые содержат свойства Gateway — сопоставляемые со, шлюз должен сопоставить все свойства с протоколом целевой системы обмена сообщениями. Хотя MAPI рекомендует шлюзам обрабатывать все именованные свойства в специальных наборах свойств, шлюзы должны обрабатывать только два: адрес электронной почты и тип адреса. Так как адрес электронной почты и свойства типа адреса непосредственно влияют на передачу сообщений, очень важно, чтобы шлюзы поддерживали сопоставление этих двух свойств. Так как ключи поиска состоят из типа адреса и адреса пользователя, они также должны быть переведены, если это возможно.
  
Идентификаторы записей не должны обрабатываться часто. Чтобы включить сопоставление идентификатора записи, созданного в одной системе обмена сообщениями, с идентификатором записи, используемым другой системой обмена сообщениями, шлюз должен иметь возможность использовать формат обеих систем. Так как большинство шлюзов не знают о форматах идентификаторов записей, преобразование идентификаторов записей очень редко.
  
Еще одно свойство сопоставляемые со, которое не требуется переводить часто, это отображаемое имя. Шлюзы должны хранить отображаемые имена и передавать их, но не обязательно преобразовывать их.
