# <a name="gateway-mapping-responsibilities"></a>Обязанности сопоставления шлюза

**Применимо к**: Outlook 
  
Получив принять во внимание MAPI шлюза сообщений с именованных свойств в одном специальные свойства наборов данных, предназначенный для хранения шлюза сопоставляемые свойства, шлюз следует сопоставить все свойства протокола назначения, системы обмена сообщениями. Несмотря на то, что MAPI рекомендует, шлюзы обрабатывать все именованные свойства в наборы специальных свойств, шлюзы должны обрабатывать только два: адрес электронной почты и тип адреса. Так как свойства типа адреса и адрес электронной почты непосредственно влияют на передачи сообщений, очень важно, что шлюзы поддерживают сопоставления эти два свойства. Так как ключи поиска состоят из тип адреса и адрес пользователя, они также преобразования, если это возможно.
  
Идентификаторы записей, не должны обрабатываться часто. Для включения сопоставления идентификатора запись, которая исходит в одной системе обмена сообщениями для записи идентификатор, который можно использовать с другой системой обмена сообщениями, шлюз должен иметь возможность использовать формат обеими системами. Поскольку большинство шлюзы не поддерживают запись идентификатор форматы, перевода идентификаторы записей очень редко.
  
Другой сопоставляемые свойства, которые не должны быть переведены часто — отображаемое имя. Шлюзы следует хранить отображаемые имена и передавать их, но переводчик не обязательно.
