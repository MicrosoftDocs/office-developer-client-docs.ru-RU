# <a name="gateway-mapping-responsibilities"></a>Обязанности по сопоставлению со шлюзами

**Область применения**: Outlook 2013 | Outlook 2016 
  
Когда шлюз mapI получает сообщение, содержащее указанные свойства в одном из специальных наборов свойств, предназначенных для размещения свойств, которые можно использовать для шлюза, шлюз должен сопозначить все свойства в протоколе системы обмена сообщениями назначения. Хотя MAPI рекомендует шлюзам обрабатывать все названные свойства в специальных наборах свойств, шлюзы должны обрабатывать только два: адрес электронной почты и тип адреса. Поскольку свойства типа адресов и адресов напрямую влияют на передачу сообщений, важно, чтобы шлюзы поддержали сопоставление этих двух свойств. Так как клавиши поиска состоят из типа и адреса пользователя, их также следует перевести, если это возможно.
  
Не ожидается, что идентификаторы записей будут обрабатываться часто. Чтобы включить сопоставление идентификатора записи, который происходит в одной системе обмена сообщениями, к идентификатору записи, который можно использовать другой системой обмена сообщениями, шлюз должен иметь возможность использовать формат обеих систем. Так как большинство шлюзов не знают о форматах идентификаторов входа, перевод идентификаторов записей встречается редко.
  
Еще одно свойство, которое не следует часто переводить, — это имя отображения. Шлюзы должны хранить имена отображения и передавать их, но не обязательно их переводить.
