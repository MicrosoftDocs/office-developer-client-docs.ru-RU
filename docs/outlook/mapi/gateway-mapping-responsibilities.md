# <a name="gateway-mapping-responsibilities"></a>Обязанности по сопоставлению со шлюзами

**Относится к**: Outlook 2013 | Outlook 2016 
  
Когда шлюз, совпав с MAPI, получает сообщение, содержащее именуемые свойства в одном из специальных наборов свойств, предназначенных для размещения свойств, которые можно использовать для mappable шлюза, шлюз должен сопоставить все свойства с протоколом конечной системы обмена сообщениями. Хотя MAPI рекомендует шлюзам обрабатывать все именуемые свойства в специальных наборах свойств, шлюзы должны обрабатывать только два адреса электронной почты и тип адреса. Поскольку свойства адреса электронной почты и типа адреса напрямую влияют на передачу сообщений, шлюзы должны поддерживать сопоставление этих двух свойств. Поскольку ключи поиска состоят из типа адреса и адреса пользователя, их также следует преобразовыть, если это возможно.
  
Ожидается, что идентификаторы записей не будут обрабатываться часто. Чтобы включить сопоставление идентификатора записи из одной системы обмена сообщениями с идентификатором записи, который можно использовать другой системой обмена сообщениями, шлюз должен иметь возможность использовать формат обеих систем. Поскольку большинство шлюзов не знают о форматах идентификаторов записей, перевод идентификаторов записей встречается редко.
  
Еще одно свойство, которое не должно часто преобразовыться, — отображаемое имя. Шлюзы должны хранить отображаемую информацию и передавать их, но не обязательно переводить их.
