## <a name="putting-a-path-in-mapisvcinf"></a><span data-ttu-id="7f242-111">Размещение пути в MapiSvc. INF</span><span class="sxs-lookup"><span data-stu-id="7f242-111">Putting a Path in MapiSvc.inf</span></span>

<span data-ttu-id="7f242-112">Большинство приложений устанавливаются в разделе Program Files, поэтому необходимо обновить переменную Path, чтобы разрешить поставщикам MAPI работать.</span><span class="sxs-lookup"><span data-stu-id="7f242-112">Most applications install under Program Files, requiring an update to the path variable to allow MAPI providers to work.</span></span> <span data-ttu-id="7f242-113">С некоторыми ограничениями Microsoft Outlook 2010 и Outlook 2013 могут поддерживать полные пути к поставщикам MAPI.</span><span class="sxs-lookup"><span data-stu-id="7f242-113">With a few restrictions Microsoft Outlook 2010 and Outlook 2013 can accommodate full paths to MAPI providers.</span></span>
  
<span data-ttu-id="7f242-114">При регистрации поставщика в MapiSvc. inf можно поместить полный путь к поставщику в свойствах MAPI **PR_SERVICE_DLL_NAME** и **PR_PROVIDER_DLL_NAME**.</span><span class="sxs-lookup"><span data-stu-id="7f242-114">When registering your provider in MapiSvc.inf, you could put the full path to the provider in the MAPI properties **PR_SERVICE_DLL_NAME** and **PR_PROVIDER_DLL_NAME**.</span></span>
  
<span data-ttu-id="7f242-115">В обоих свойствах полный путь должен быть без суффикса "32", так как MAPI продолжает добавлять его в имя файла, прежде чем искать файл.</span><span class="sxs-lookup"><span data-stu-id="7f242-115">In either property, the full path must be without the suffix "32", because MAPI continues to append that to the filename before looking for your file.</span></span> <span data-ttu-id="7f242-116">Это означает, что если вы зарегистрируете путь "к:\мипас\мипровидер.длл", MAPI предпримет попытку загрузить "c:\mypath\myprovider32.dll".</span><span class="sxs-lookup"><span data-stu-id="7f242-116">This means that if you register the path "c:\mypath\myprovider.dll", MAPI will attempt to load "c:\mypath\myprovider32.dll".</span></span>
  
<span data-ttu-id="7f242-117">Так как MAPI Outlook изначально не предназначался для размещения полных путей, он выполняет эту вставку суффикса "32", выполняя поиск первого периода в строке, что означает, что пути, содержащие другие точки, не могут работать, поэтому нельзя использовать такие пути, как "к:\ми.пас\мипровидер.длл" или "к:\мипас\ми.провидер.длл".</span><span class="sxs-lookup"><span data-stu-id="7f242-117">Because Outlook's MAPI was not originally designed to accommodate full paths, it accomplishes this insertion of the "32" suffix by looking for the first period in the string, which means that paths that contain other periods cannot work, so you cannot use paths such as "c:\my.path\myprovider.dll" or "c:\mypath\my.provider.dll".</span></span>
  
<span data-ttu-id="7f242-118">Иногда в поставщике хранилища вы будете создавать идентификаторы записей с помощью функции **врапсторинтрид** , которая принимает в качестве параметра имя поставщика.</span><span class="sxs-lookup"><span data-stu-id="7f242-118">Sometimes in a store provider you will generate entry identifiers using the **WrapStoreEntryID** function, which takes as a parameter the name of your provider.</span></span>
