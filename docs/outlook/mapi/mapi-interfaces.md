# <a name="mapi-interfaces"></a>Интерфейсы MAPI

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Документация для каждого интерфейса состоит из вводного раздела, в который входит краткое описание цели интерфейса, а затем таблица со следующей информацией.
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Файл загона, в котором определен интерфейс и который должен быть включен при компиляции исходных кодов.  <br/> |
|Подвергается:  <br/> |Объект, который предоставляет интерфейс.  <br/> |
|Реализовано в:  <br/> |Список компонентов, которые обеспечивают реализацию интерфейса.  <br/> |
|Вызывающая сторона:  <br/> |Список компонентов, которые обычно называют методами интерфейса.  <br/> |
|Идентификатор интерфейса:  <br/> |GUID идентификатора интерфейса.  <br/> |
|Тип указателя:  <br/> |Тип указателя для объекта, который предоставляет интерфейс.  <br/> |
|Модель транзакции:  <br/> |Для интерфейсов, полученных из [IMAPIProp](imapipropiunknown.md). Если они не принимаются, изменения вступает в силу немедленно; если они будут внесены, изменения не вступает в силу до тех пор, пока не будет [вызвана IMAPIProp::SaveChanges.](imapiprop-savechanges.md)  <br/> |
   
За первой таблицей следует еще одна таблица, в которую перечислены все методы этого интерфейса в порядке vtable. Vtable — это массив указателей функций, созданных компилятором, содержащим по одной указатель функции для каждого метода объекта MAPI. Методы перечислены в том же порядке, что и объявленные. Методы, унаследованные от других интерфейсов, не показаны в таблице Vtable Order, но могут использоваться так же, как описаны в интерфейсе, который их определяет.
  
После каждой темы интерфейса методы интерфейса затем документируются в алфавитном порядке. Для каждого метода документация включает краткое заявление о назначении, блок синтаксиса и следующую информацию.
  
|**Заголовок**|**Content**|
|:-----|:-----|
|Parameters  <br/> |Описание каждого параметра метода.  <br/> |
|Возвращаемое значение  <br/> |Описание уникальных значений, которые может возвращать метод. Это значения, которые должны проверяться в коде звонители.  <br/> |
|Примечания  <br/> |Описание причины и способа использования метода.  <br/> |
|См. также  <br/> |Перекрестные ссылки на другие темы в этом справочнике.  <br/> |
