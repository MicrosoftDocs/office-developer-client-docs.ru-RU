# <a name="imapisupportexpandrecips"></a>IMAPISupport::ExpandRecips

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Завершает список получателей сообщения, развертывая определенные списки рассылки.
  
```cpp
HRESULT ExpandRecips(
  LPMESSAGE lpMessage,
  ULONG FAR * lpulFlags
);
```

## <a name="parameters"></a>Параметры

 _Лпмессаже_
  
> возврата Указатель на сообщение, для которого будет обрабатываться список получателей.
    
 _Лпулфлагс_
  
> вышли Указатель на битовую маску флагов, определяющих тип выполняемой обработки. Можно задать следующие флаги:
    
НИДС_ПРЕПРОЦЕССИНГ 
  
> Перед отправкой сообщения необходимо предварительно обработать его.
    
НИДС_СПУЛЕР 
  
> Диспетчер очереди MAPI (вместо транспортного поставщика, которому вызывающий абонент тесно связан) должен отправить сообщение.
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Список получателей сообщения успешно обработан.
    
## <a name="remarks"></a>Примечания

Метод **имаписуппорт:: експандреЦипс** реализован для объектов поддержки поставщика хранилища сообщений. Поставщики хранилищ сообщений вызывают **експандреЦипс** , чтобы запросить MAPI для выполнения следующих задач: 
  
- РазВорачивание определенных личных списков рассылки для получателей их компонентов.
    
- Замените все отображаемые имена, которые были изменены исходными именами.
    
- ПоМетьте все повторяющиеся записи.
    
- РазРешите все одноразовые адреса. 
    
- Убедитесь, что для сообщения требуется предварительная обработка, и, если это так, установите флаг, указывающий на _лпулфлагс_ в нидс_препроцессинг. 
    
 **ЕкспандреЦипс** развертывает все списки рассылки, которые имеют тип адреса обмена сообщениями мапипдл. 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

Всегда вызывайте **експандреЦипс** в процессе обработки сообщения. Выполните вызов **експандреЦипс** одного из первых вызовов в реализации метода [iMessage:: субмитмессаже](imessage-submitmessage.md) .
