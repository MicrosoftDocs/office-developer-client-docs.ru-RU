# <a name="imapiformcontainerresolvemultiplemessageclasses"></a>IMAPIFormContainer::ResolveMultipleMessageClasses

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Разрешает группу классов сообщений в свои формы в контейнере формы и возвращает массив объектов данных формы для этих форм.
  
```cpp
HRESULT ResolveMultipleMessageClasses(
  LPSMESSAGECLASSARRAY pMsgClassArray,
  ULONG ulFlags,
  LPSMAPIFORMINFOARRAY FAR * ppfrminfoarray
);
```

## <a name="parameters"></a>Параметры

 _Пмсгклассаррай_
  
> возврата Указатель на массив, содержащий имена классов сообщений, которые требуется разрешить. Имена классов сообщений всегда являются строками ANSI, а не Юникодом.
    
 _ulFlags_
  
> возврата Битовая маска флагов, определяющих способ разрешения классов сообщений. Можно задать следующий флаг:
    
МАПИФОРМ_ЕКСАКТМАТЧ 
  
> Необходимо разрешить только строки класса сообщений с точным совпадением.
    
 _ппфрминфоаррай_
  
> вышли Указатель на указатель на массив объектов сведений о форме. Если клиентское приложение передает значение NULL в параметр _пмсгклассаррай_ , параметр _ппфрминфоаррай_ содержит объекты информации формы для всех форм в контейнере.

## <a name="remarks"></a>Комментарии

Клиентские приложения вызывают метод **имапиформконтаинер:: ресолвемултиплемессажеклассес** для разрешения группы классов сообщений в формы в контейнере формы. Массив объектов данных формы, возвращаемых в параметре _ппфрминфоаррай_ , обеспечивает дополнительный доступ к каждому из свойств формы. 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

Чтобы разрешить группу классов сообщений формам, передайте массив имен классов сообщений, которые требуется разрешить. Чтобы принудительно разрешить точное решение (то есть, чтобы предотвратить разрешение для базового класса класса Message), можно передать флаг МАПИФОРМ_ЕКСАКТМАТЧ в параметр _ulFlags_ . 
  
Если класс сообщения невозможно разрешить в форму, возвращается значение NULL для этого класса сообщения в массиве данных формы. Таким образом, даже если метод возвращает значение S_OK, не следует предполагать, что все классы сообщений успешно разрешены. Вместо этого проверьте значения в возвращенном массиве.
