# <a name="sync"></a>SYNC

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Сведения о запуске синхронизации между локальным хранилищем и сервером. Эти сведения используются в [состоянии синхронизации](synchronize-state.md).
  
## <a name="quick-info"></a>Краткие сведения

```cpp
struct SYNC 
{ 
    ULONG ulFlags; 
    LPWSTR pwzPath; 
    FEID Reserved1; 
    MEID Reserved2; 
    LPENTRYLIST pel; 
    ULONG const * pulFolderOptions; 
};
```

## <a name="members"></a>Элементы

 _ulFlags_
  
- [out]/[in] битовая маска следующих флагов, которые изменяют поведение во время синхронизации:
    
- UPS_UPLOAD_ONLY
    
  - возврата Клиент будет выполнять только отправку. Outlook возвращает только локальные измененные папки.
    
- UPS_DNLOAD_ONLY
    
  - возврата Клиент будет выполнять только загрузку. Outlook не должен очищать биты отправки для папок.
    
- UPS_THESE_FOLDERS
    
  - возврата Клиент будет синхронизировать заданный набор папок с указанными идентификаторами записи. Этот флаг можно сочетать с флагом **UPS_UPLOAD_ONLY** или **UPS_DNLOAD_ONLY** . 
    
- UPS_OK
    
  - вышли Синхронизация выполнена успешно. Клиент устанавливает это после отправки или завершения полной синхронизации.
    
- 
    
    > [!NOTE]
    > Несмотря на то, что клиент может загружать или полностью синхронизировать (загрузить, а затем загружать) папки и элементы с помощью API репликации, клиент указывает *ulFlags* с одним направлением репликации одновременно — либо с помощью флага **UPS_UPLOAD_ONLY** , либо **UPS_DNLOAD_ONLY** . В случае полной синхронизации клиент сначала выполняет отправку с флагом **UPS_UPLOAD_ONLY** , а затем загружается с флагом **UPS_DNLOAD_ONLY** . 
  
 _пвзпас_
  
- вышли Путь к локальному хранилищу.
    
 _Reserved1_
  
- Этот член зарезервирован для внутреннего использования в Outlook и не поддерживается.
    
 _Reserved2_
  
- Этот член зарезервирован для внутреннего использования в Outlook и не поддерживается.
    
 *выражений* 
  
- возврата Это список идентификаторов записей для синхронизации, если **UPS_THESE_FOLDERS** задано. Определение типа **лпентрилист**можно найти в файле MAPIDEFS. h. 
    
 _пулфолдероптионс_
  
- возврата Это массив свойств папки для соответствующих папок в *PEL* , если **UPS_THESE_FOLDERS** задано. Эти параметры папки используются при отправке каждой из папок, перечисленных в *PEL* , в [состояние папки отправки](upload-folder-state.md). Для получения дополнительных сведений о параметрах папки обратитесь к разделу **[упфлд](upfld.md)**.
