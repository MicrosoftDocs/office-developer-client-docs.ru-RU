# <a name="imapifoldercreatefolder"></a>IMAPIFolder::CreateFolder

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Создает новую вложенную папку.
  
```cpp
HRESULT CreateFolder(
  ULONG ulFolderType,
  LPSTR lpszFolderName,
  LPSTR lpszFolderComment,
  LPCIID lpInterface,
  ULONG ulFlags,
  LPMAPIFOLDER FAR * lppFolder
);
```

## <a name="parameters"></a>Параметры

 _Улфолдертипе_
  
> возврата Тип создаваемой папки. Можно задать следующие флаги:
    
ФОЛДЕР_ЖЕНЕРИК 
  
> Следует создать универсальную папку.
    
ФОЛДЕР_СЕАРЧ 
  
> Следует создать папку результатов поиска.
    
 _Лпсзфолдернаме_
  
> возврата Указатель на строку, содержащую имя новой папки. Это имя является основой для свойства **пр_дисплай_наме** ([PidTagDisplayName](pidtagdisplayname-canonical-property.md)) новой папки.
    
 _Лпсзфолдеркоммент_
  
> возврата Указатель на строку, содержащую комментарий, связанный с новой папкой. Эта строка становится значением свойства **пр_коммент** ([PidTagComment](pidtagcomment-canonical-property.md)) новой папки. Если передается значение NULL, то папка не имеет исходного комментария.
    
 _Лпинтерфаце_
  
> возврата Указатель на идентификатор интерфейса (IID), представляющий интерфейс, который будет использоваться для доступа к новой папке. При передаче значения NULL поставщику хранилища сообщений возвращается интерфейс стандартных папок, [IMAPIFolder: IMAPIContainer](imapifolderimapicontainer.md). Клиенты должны передавать значение NULL. Другие абоненты могут задать для параметра _лпинтерфаце_ значение Иид_иункновн, Иид_имапипроп, Иид_имапиконтаинер или иид_имапифолдер. 
    
 _ulFlags_
  
> возврата Битовая маска флагов, определяющих способ создания папки. Можно задать следующие флаги:
    
МАПИ_ДЕФЕРРЕД_ЕРРОРС 
  
> Разрешает успешное возвращение **CreateFolder** , возможно, перед тем, как новая папка будет полностью доступна вызывающему клиенту. Если новая папка недоступна, то выполнение последующих вызовов может привести к ошибке. 
    
MAPI_UNICODE 
  
> Имя папки имеет формат Юникод. Если флаг МАПИ_УНИКОДЕ не установлен, имя папки указывается в формате ANSI.
    
ОПЕН_ИФ_ЕКСИСТС 
  
> Позволяет успешно выполнить метод, даже если папка с именем, указанная в параметре _лпсзфолдернаме_ , уже существует, открыв существующую папку с таким именем. Обратите внимание, что поставщики хранилищ сообщений, которые допускают наличие одинаковых папок на одном уровне, могут не открыть существующую папку, если существует более одной папки с указанным именем. 
    
 _Лппфолдер_
  
> вышли Указатель на указатель на только что созданную папку.
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Новая папка успешно создана или открыта, если установлен флаг ОПЕН_ИФ_ЕКСИСТС.
    
МАПИ_Е_БАД_ЧАРВИДС 
  
> Установлен либо флаг МАПИ_УНИКОДЕ, либо реализация не поддерживает Юникод, или МАПИ_УНИКОДЕ не задано, а реализация поддерживает только Юникод.
    
МАПИ_Е_КОЛЛИСИОН 
  
> Папка с именем, заданной в параметре _лпсзфолдернаме_ , уже существует. Имена папок должны быть уникальными. 
    
## <a name="remarks"></a>Замечания

Метод **IMAPIFolder:: CreateFolder** создает вложенную папку в текущей папке и присваивает новой папке идентификатор записи. 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

При возвращении **CreateFolder** Обратите внимание, что идентификатор записи для новой папки может быть недоступен. Некоторые поставщики хранилищ сообщений не делают идентификаторы записей доступными до тех пор, пока не будет вызван метод [IMAPIProp:: SaveChanges](imapiprop-savechanges.md) , чтобы сохранить его окончательно. Это особенно важно, если установлен флаг МАПИ_ДЕФЕРРЕД_ЕРРОРС. 
  
Обратите внимание, что некоторые поставщики хранилищ сообщений всегда назначают параметр _лппфолдер_ стандартному интерфейсу папки независимо от значения, передаваемого в качестве параметра _лпинтерфаце_ . Так как возвращаемый указатель интерфейса может отличаться от ожидаемого типа, вызовите метод [IMAPIProp::](imapiprop-getprops.md) -PROPS новой папки, чтобы получить свойство **пр_обжект_типе** ([PidTagObjectType](pidtagobjecttype-canonical-property.md)). При необходимости перед выполнением других вызовов приведите указатель к более подходящему типу.
  
Для большинства поставщиков хранилищ сообщений необходимо, чтобы имя новой папки было уникальным по отношению к именам ее родственных папок. Возможность обрабатывать значение ошибки МАПИ_Е_КОЛЛИСИОН, которое возвращается, если это правило не выполняется. 
  
Чтобы определить идентификатор записи только что созданной папки, вызовите метод **IMAPIProp::** -PROPS новой папки, чтобы получить свойство **пр_ентрид** ([PidTagEntryId](pidtagentryid-canonical-property.md)).
