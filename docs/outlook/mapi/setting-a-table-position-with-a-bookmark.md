# <a name="setting-a-table-position-with-a-bookmark"></a>Установка положения таблицы с использованием закладки

  
  
**Относится к**: Outlook 
  
Объект bookmark представляет собой ресурсов, которое указывает, определенного расположения в таблице. Установка закладки позволяет для возврата в позицию в дальнейшем функциональная возможность, которая может значительно повысить производительность операций в таблице. MAPI определяет три стандартные закладки: 
  
|||
|:-----|:-----|
|BOOKMARK_CURRENT  <br/> |Указывает на текущей строки в таблице.  <br/> |
|BOOKMARK_BEGINNING  <br/> |Указывает первую строку в таблице.  <br/> |
|BOOKMARK_END  <br/> |Указывает на последнюю строку в таблице.  <br/> |
   
Таблица специалистов по внедрению необходимых для поддержки этих стандартных закладки и может поддерживать другим пользователям. Тем не менее так как закладки, ресурсы и ресурсы ограничены, закладка пользователи должны освободить их как можно скорее. 
  
 **Чтобы установить закладку в текущей позиции в таблице**
  
- Вызов [IMAPITable::CreateBookmark](imapitable-createbookmark.md). Иногда будет недостаточно памяти для размещения новой закладки, вызывает ошибку **CreateBookmark** для возврата значения MAPI_E_UNABLE_TO_COMPLETE об ошибках. 
    
 **Чтобы освободить место на закладку**
  
- Вызов [IMAPITable::FreeBookmark](imapitable-freebookmark.md).
    
 **Перемещение курсора в закладку позицию**
  
- Вызов [IMAPITable::SeekRow](imapitable-seekrow.md). **SeekRow** устанавливает новое значение для положения BOOKMARK_CURRENT. **SeekRow** можно использовать, например, для размещения строк таблицы десяти из текущей позиции или переходит в начале. Клиентов или поставщиков услуг можно перейти к текущему Приступая к работе, или завершение таблицы или любом другом месте, который связан с предварительно заданных закладка. Их можно переместить вперед или назад направление и ограничение операции указанное число строк. Как правило вызывающие объекты должны поиск с помощью не более 50 строк с **SeekRow**; [IMAPITable::SeekRowApprox](imapitable-seekrowapprox.md) следует использовать с большего числа строк.
