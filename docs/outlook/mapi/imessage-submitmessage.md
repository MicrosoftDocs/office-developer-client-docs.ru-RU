# <a name="imessagesubmitmessage"></a>IMessage::SubmitMessage

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Сохраняет все свойства сообщения и помечает сообщение как готовый для отправки.
  
```cpp
HRESULT SubmitMessage(
  ULONG ulFlags
);
```

## <a name="parameters"></a>Параметры

 _ulFlags_
  
> возврата Битовая маска флагов, используемых для управления способом отправки сообщения. Можно задать следующий флаг:
    
ФОРЦЕ_СУБМИТ 
  
> MAPI должен немедленно передать сообщение. Этот флаг в данный момент не используется.

> Таблица получателей сообщения пуста.
    
## <a name="remarks"></a>Примечания

Метод **iMessage:: субмитмессаже** помечает сообщение как готовый для передачи. MAPI передает сообщения в базовую систему обмена сообщениями в том порядке, в котором они помечены для отправки. Из-за этой функции сообщение может оставаться в банке сообщений в течение некоторого времени, прежде чем соответствующая система обмена сообщениями будет отвечать за нее. Порядок прихода в месте назначения находится в базовом элементе управления системы обмена сообщениями и не обязательно должен совпадать с порядком отправки сообщений. 
  
## <a name="notes-to-implementers"></a>Примечания для исполнителей

ВыЗовите метод [IMAPIProp:: SaveChanges](imapiprop-savechanges.md) , чтобы сохранить его, а затем проверьте свойство **пр_мессаже_флагс** ([PidTagMessageFlags](pidtagmessageflags-canonical-property.md)) сообщения. Если установлен флаг МСГФЛАГ_РЕСЕНД, вызовите [имаписуппорт::P репаресубмит](imapisupport-preparesubmit.md). **Препаресубмит** обновляет свойство тип получателя и свойство **пр_респонсибилити** ([PidTagResponsibility](pidtagresponsibility-canonical-property.md)) для всех получателей в сообщении о повторной отправке.
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

Когда возвращается значение **субмитмессаже** , все указатели на сообщение и связанные с ним подобъекты, папки, вложения, потоки, таблицы и т. д. больше не являются допустимыми. MAPI не разрешает дальнейшие операции с этими указателями, за исключением вызовов методов **IUnknown:: Release** . ИНТЕРФЕЙС MAPI разработан таким образом, что после вызова **субмитмессаже** необходимо освободить сообщение и все связанные с ним подобъекты. Однако если **субмитмессаже** возвращает значение ошибки, указывающее отсутствующие или недопустимые сведения, сообщение остается открытым, а указатели остаются действительными. 
  
Чтобы отменить операцию отправки, перед отправкой сообщения получите и сохраните указатель на свойство **пр_ентрид** ([PidTagEntryId](pidtagentryid-canonical-property.md)) сообщения. Так как идентификатор записи сообщения становится недействительным после отправки сообщения, необходимо сохранить его перед вызовом **субмитмессаже**. Чтобы отменить отправку, укажите параметр _лпентрид_ для этого идентификатора записи и вызовите [IMsgStore:: абортсубмит](imsgstore-abortsubmit.md).
