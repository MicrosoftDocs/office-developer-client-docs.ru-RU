# <a name="fpropcontainsprop"></a>FPropContainsProp

**Относится к**: Outlook 2013 | Outlook 2016 
  
Сравнивает два значения свойств, обычно строки или двоичные массивы, чтобы проверить, содержит ли один из них другой. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Мапиутил. h  <br/> |
|Реализовано в:  <br/> |MAPI  <br/> |
|Вызывающая сторона:  <br/> |Клиентские приложения и поставщики услуг  <br/> |
   
```cpp
BOOL FPropContainsProp(
  LPSPropValue lpSPropValueDst,
  LPSPropValue lpSPropValueSrc,
  ULONG ulFuzzyLevel
);
```

## <a name="parameters"></a>Параметры

_Лпспропвалуедст_
  
> возврата Указатель на структуру [спропвалуе](spropvalue.md) , определяющую значение свойства, которое может содержать строку поиска, на которую указывает параметр _лпспропвалуесрк_ . 
    
_Лпспропвалуесрк_
  
> возврата Указатель на структуру **спропвалуе** , определяющую строку поиска, которую **фпропконтаинспроп** ищет в значении свойства, на которое указывает параметр _лпспропвалуедст_ . 
    
_Улфуззилевел_
  
> возврата Параметры параметров, определяющие степень точности, используемую в сравнении. 

  - **Младшие 16 битов** применяются к свойствам типа ПТ_БИНАРИ и PT_STRING8. Для них должно быть задано значение, равное одному из следующих значений:
      
    - ФЛ_ФУЛЛСТРИНГ: строка поиска _лпспропвалуесрк_ должна быть равна значению свойства, определенному с помощью _лпспропвалуедст_.
        
    - ФЛ_ПРЕФИКС: строка поиска _лпспропвалуесрк_ должна находиться в начале значения свойства, идентифицируемого _лпспропвалуедст_. Эти два значения должны сравниваться только с длиной строки поиска, указанной в _лпспропвалуесрк_. 
        
    - ФЛ_СУБСТРИНГ: строка поиска _лпспропвалуесрк_ должна содержаться в любом месте в значении свойства, определенном с помощью _лпспропвалуедст_. 
      
  - **Верхние 16 битов** применяются только к свойствам типа PT_STRING8. Можно задать следующие значения в любой комбинации:
    
    - ФЛ_ИГНОРЕКАСЕ: сравнение следует производить без учета регистра. 
        
    - ФЛ_ИГНОРЕНОНСПАЦЕ: сравнение должно игнорировать заданные Юникод несамостоятельные символы, такие как диакритические знаки. 
        
    - ФЛ_ЛУСЕ: сравнение должно указывать на сопоставление, если это возможно, без учета регистра символов и несамостоятельных символов.
    
## <a name="return-value"></a>Возвращаемое значение

TRUE 
  
> Все параметры являются допустимыми, а строка поиска _лпспропвалуесрк_ указывается в значении свойства _лпспропвалуедст_ . 
    
FALSE 
  
> Сравниваемые значения свойств не относятся к типу PT_STRING8 или ПТ_БИНАРИ, значения свойств относятся к разным типам, или строка поиска _лпспропвалуесрк_ не указана в значении свойства _лпспропвалуедст_ . 
    
## <a name="remarks"></a>Примечания

Метод сравнения зависит от типов свойств, указанных в определениях свойств [спропвалуе](spropvalue.md) , и эвристического алгоритма нечеткого уровня, указанного в параметре _улфуззилевел_ . Функции [фпропкомпарепроп](fpropcompareprop.md) и **фпропконтаинспроп** можно использовать для подготовки ограничений для создания таблицы. 
  
Верхние 16 разрядов _улфуззилевел_ игнорируются для свойства пт_бинари типа. Если параметры в _улфуззилевел_ отсутствуют или являются недопустимыми, выполняется точное точное совпадение. Дополнительные сведения о вложении свойств можно найти в разделе Структура [сконтентрестриктион](scontentrestriction.md) .
