# <a name="integrating-mapi-form-server-code-with-windows-code"></a>Интеграция кода форм-сервера MAPI с Windows кодом

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Помните, что сервер формы — это приложение Win32. Таким образом, существует ряд задач, связанных с загрузкой сервера форм в память и чистой очисткой. Как и Windows приложения, точкой входа для сервера форм является **функция WinMain.** Эта функция является подходящим местом для выполнения следующих задач: 
  
- Создание и регистрация класса окна, чтобы ваш сервер формы взаимодействовал с другими компонентами OLE.
    
- Создание и регистрация класса окна или классов для пользовательских интерфейсов объектов формы.
    
- Вызов [функции MAPIInitialize.](mapiinitialize.md) **MAPIInitialize** также обрабатывает требуемую инициализацию OLE. Это должно быть сделано один раз в экземпляре сервера форм. 
    
- Регистрация глобального атома с помощью строки представления идентификатора класса сервера форм (CLSID). Этот атом должен существовать в течение всего срока службы сервера форм.
    
- Вызов функции OLE [CoRegisterClassObject](https://msdn.microsoft.com/library/ms693407.aspx) для регистрации фабрики классов вашего сервера форм с помощью OLE. 
    
- Создание основного окна для получения сообщений. Возможно, это окно не должно быть видимым, так как пользователь будет взаимодействовать с определенными окнами, связанными с отдельными объектами формы. Однако во время разработки главное окно может быть удобным местом для отладки выходных данных или управления сервером форм.
    
- Создание цикла сообщений, который выполняется в течение всего срока службы сервера форм, перевод и отправка windows-сообщений на объекты активной формы.
    
Когда сервер формы выходит, он должен выполнить следующие задачи:
  
- Вызов функции OLE [CoRevokeClassObject,](https://msdn.microsoft.com/library/ms688650%28VS.85%29.aspx) чтобы отозвать регистрацию OLE класса сообщений. 
    
- Вызов **MAPIUninitialize,** чтобы правильно закрыть подключение сервера форм к MAPI. 
    
- Удалите глобальный атом, содержащий строковую репрезентативность идентификатора класса.
