# <a name="integrating-mapi-form-server-code-with-windows-code"></a>Интеграция кода сервера форм MAPI с кодом Windows

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Помните, что сервер форм является приложением Win32. Таким образом, существуют некоторые задачи, связанные с загрузкой сервера форм в память и очисткой выхода. Как и для всех приложений Windows, точкой входа для сервера форм является функция **WinMain.** Эта функция подходит для выполнения следующих задач: 
  
- Создание и регистрация класса окна для взаимодействия сервера форм с другими компонентами OLE.
    
- Создание и регистрация класса окна или классов для пользовательских интерфейсов объектов формы.
    
- Вызов [функции MAPIInitialize.](mapiinitialize.md) **MAPIInitialize** также обрабатывает требуемую инициализацию OLE. Это необходимо сделать один раз для каждого экземпляра сервера форм. 
    
- Регистрация глобального atom в строковом представлении идентификатора класса сервера форм (CLSID). Этот atom должен существовать в течение всего времени существования сервера форм.
    
- Вызов функции OLE [CoRegisterClassObject](https://msdn.microsoft.com/library/ms693407.aspx) для регистрации фабрики классов сервера форм в OLE. 
    
- Создание главного окна для получения сообщений. Возможно, это окно не должно быть видимым, так как пользователь будет взаимодействовать с определенными окнами, связанными с отдельными объектами форм. Однако во время разработки главное окно может быть удобным местом для отладки выходных данных или управления сервером форм.
    
- Создание цикла сообщений, который выполняется на протяжении всего жизненного цикла сервера форм, перевод и отправка сообщений windows в активные объекты форм.
    
После выхода сервера форм он должен выполнить следующие задачи:
  
- Вызовите функцию OLE [CoRevokeClassObject,](https://msdn.microsoft.com/library/ms688650%28VS.85%29.aspx) чтобы отозвать регистрацию OLE класса сообщений. 
    
- Вызовите **MAPIUninitialize,** чтобы должным образом закрыть подключение сервера форм к MAPI. 
    
- Удалите глобальный atom, содержащий строку, представленную идентификатором класса.
