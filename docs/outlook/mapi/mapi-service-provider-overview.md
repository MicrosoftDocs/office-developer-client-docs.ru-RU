# <a name="mapi-service-provider-overview"></a><span data-ttu-id="46708-103">Обзор поставщика услуг MAPI</span><span class="sxs-lookup"><span data-stu-id="46708-103">MAPI Service Provider Overview</span></span>

  
  
<span data-ttu-id="46708-104">**Область применения**: Outlook 2013 | Outlook 2016</span><span class="sxs-lookup"><span data-stu-id="46708-104">**Applies to**: Outlook 2013 | Outlook 2016</span></span> 
  
<span data-ttu-id="46708-105">Между подсистемой MAPI и системами обмена сообщениями находятся различные поставщики услуг.</span><span class="sxs-lookup"><span data-stu-id="46708-105">Between the MAPI subsystem and the messaging systems are the various service providers.</span></span> <span data-ttu-id="46708-106">Поставщики услуг похожи на драйверы, которые подключают клиентские приложения MAPI к системе обмена сообщениями.</span><span class="sxs-lookup"><span data-stu-id="46708-106">Service providers are like drivers that connect MAPI client applications to an underlying messaging system.</span></span> <span data-ttu-id="46708-107">Существует три типа поставщиков: поставщики магазинов сообщений, поставщики адресной книги или каталогов и поставщики транспорта сообщений.</span><span class="sxs-lookup"><span data-stu-id="46708-107">There are three types of providers: message store providers, address book or directory providers, and message transport providers.</span></span> <span data-ttu-id="46708-108">MAPI поддерживает каждый тип служб независимо друг от друга, что позволяет поставщику предлагать один или несколько пользовательских поставщиков услуг.</span><span class="sxs-lookup"><span data-stu-id="46708-108">MAPI supports each type of service independently, enabling a vendor to offer one or more custom service providers.</span></span> <span data-ttu-id="46708-109">Например, поставщику может потребоваться создать поставщика адресных книг, который использует корпоративный каталог телефонных книг сотрудников или создать поставщика магазина сообщений, который использует существующую базу данных.</span><span class="sxs-lookup"><span data-stu-id="46708-109">For example, a vendor might want to create an address book provider that uses a corporate telephone book directory of employees or to create a message store provider that uses an existing database.</span></span>
  
<span data-ttu-id="46708-110">Поставщики услуг обычно пишутся для определенных систем обмена сообщениями разработчиками программного обеспечения, которые имеют специализированные знания или опыт работы с определенной системой.</span><span class="sxs-lookup"><span data-stu-id="46708-110">Service providers are typically written for specific messaging systems by software developers who have specialized knowledge or experience with a particular system.</span></span> <span data-ttu-id="46708-111">Например, Microsoft Outlook 2013 и Microsoft Outlook 2010, русская версия мобильные службы для разоблачения мобильной адресной книги в Outlook.</span><span class="sxs-lookup"><span data-stu-id="46708-111">For example, the Microsoft Outlook 2013 and Microsoft Outlook 2010 Mobile Services use an address book provider to expose a mobile address book in Outlook.</span></span> 
  
<span data-ttu-id="46708-112">MAPI представляет клиентские приложения с единым представлением сведений о адресной книге и поставщиках транспорта.</span><span class="sxs-lookup"><span data-stu-id="46708-112">MAPI presents client applications with a unified view of address book and transport provider information.</span></span> <span data-ttu-id="46708-113">Этот интегрированный подход не позволяет клиенту-приложению иметь карту данных соответствующему поставщику.</span><span class="sxs-lookup"><span data-stu-id="46708-113">This integrated approach prevents the client application from having to map data to the appropriate provider.</span></span> <span data-ttu-id="46708-114">Это также не позволяет пользователю вести переговоры между несколькими адресной книгой и схемами адресации поставщика транспорта.</span><span class="sxs-lookup"><span data-stu-id="46708-114">It also prevents the user from having to negotiate among multiple address book and transport provider addressing schemes.</span></span> <span data-ttu-id="46708-115">Однако информация поставщика магазина сообщений не является единой, и клиенты, которые используют несколько поставщиков магазинов сообщений, несут ответственность за их индивидуальную обработку.</span><span class="sxs-lookup"><span data-stu-id="46708-115">Message store provider information, however, is not unified, and clients that use multiple message store providers are responsible for handling them individually.</span></span>
  
<span data-ttu-id="46708-116">Поставщики услуг работают с MAPI для создания и отправки сообщений следующим образом: сообщения создаются с помощью формы, подходящей для определенного типа или класса сообщений.</span><span class="sxs-lookup"><span data-stu-id="46708-116">The service providers work with MAPI to create and send messages in the following way: messages are created by using a form that is appropriate for the specific type, or class, of message.</span></span> <span data-ttu-id="46708-117">Многие сообщения сделаны со стандартной формой заметки, которая поставляется с подсистемой MAPI либо пользователем клиентского приложения, либо программным путем без взаимодействия с пользователем.</span><span class="sxs-lookup"><span data-stu-id="46708-117">Many messages are made with the standard note form that comes with the MAPI subsystem, either by the user of a client application or programmatically without user interaction.</span></span> <span data-ttu-id="46708-118">Завершенное сообщение адресовано одному или более получателям — пользователю или группе пользователей, назначенных для получения сообщения.</span><span class="sxs-lookup"><span data-stu-id="46708-118">The completed message is addressed to one or more recipients — a user or group of users designated to receive the message.</span></span> <span data-ttu-id="46708-119">Получатель может иметь или не иметь запись в каталоге, который принадлежит одному из поставщиков установленных адресных книг.</span><span class="sxs-lookup"><span data-stu-id="46708-119">A recipient might or might not have an entry in a directory that one of the installed address book providers owns.</span></span> <span data-ttu-id="46708-120">Получатели, не связанные с поставщиком установленных адресных книг, называются настраиваемые получатели или разовьемые адреса.</span><span class="sxs-lookup"><span data-stu-id="46708-120">Recipients that are not associated with an installed address book provider are called custom recipients or one-off addresses.</span></span> <span data-ttu-id="46708-121">Разовая адресная почта может быть временной, продолжительной только до отправки сообщения.</span><span class="sxs-lookup"><span data-stu-id="46708-121">A one-off address can be temporary, lasting only until the message is submitted.</span></span> 
  
<span data-ttu-id="46708-122">Когда клиентская заявка отправляет сообщение, поставщик магазина сообщений проверяет, что у каждого получателя есть уникальный и допустимый адрес и что в сообщении есть все сведения, необходимые для передачи.</span><span class="sxs-lookup"><span data-stu-id="46708-122">When the client application sends the message, the message store provider checks that each recipient has a unique and valid address and that the message has all of the information necessary for transmission.</span></span> <span data-ttu-id="46708-123">Если возникает вопрос о получателе (например, если имеется несколько получателей с одним и тем же именем), поставщик адресных книг позаботится об устранении двусмысленности.</span><span class="sxs-lookup"><span data-stu-id="46708-123">If there is a question about a recipient (for example, when there are multiple recipients with the same name), an address book provider takes care of resolving the ambiguity.</span></span> <span data-ttu-id="46708-124">Затем сообщение помещается в исходящие очереди.</span><span class="sxs-lookup"><span data-stu-id="46708-124">The message is then placed in the outbound queue.</span></span>
