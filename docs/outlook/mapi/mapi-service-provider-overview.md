# <a name="mapi-service-provider-overview"></a><span data-ttu-id="b0a25-103">Общие сведения о поставщике услуг MAPI</span><span class="sxs-lookup"><span data-stu-id="b0a25-103">MAPI Service Provider Overview</span></span>

  
  
<span data-ttu-id="b0a25-104">**Относится к**: Outlook 2013 | Outlook 2016</span><span class="sxs-lookup"><span data-stu-id="b0a25-104">**Applies to**: Outlook 2013 | Outlook 2016</span></span> 
  
<span data-ttu-id="b0a25-105">Между подсистемой MAPI и системами обмена сообщениями находятся различные поставщики услуг.</span><span class="sxs-lookup"><span data-stu-id="b0a25-105">Between the MAPI subsystem and the messaging systems are the various service providers.</span></span> <span data-ttu-id="b0a25-106">Поставщики услуг похожи на драйверы, соединяющие клиентские приложения MAPI с системой обмена сообщениями.</span><span class="sxs-lookup"><span data-stu-id="b0a25-106">Service providers are like drivers that connect MAPI client applications to an underlying messaging system.</span></span> <span data-ttu-id="b0a25-107">Существует три типа поставщиков: поставщики store сообщений, поставщики адресной книги или каталогов, а также поставщики транспорта сообщений.</span><span class="sxs-lookup"><span data-stu-id="b0a25-107">There are three types of providers: message store providers, address book or directory providers, and message transport providers.</span></span> <span data-ttu-id="b0a25-108">MAPI поддерживает каждый тип служб независимо друг от друга, позволяя поставщику предлагать одного или несколько пользовательских поставщиков услуг.</span><span class="sxs-lookup"><span data-stu-id="b0a25-108">MAPI supports each type of service independently, enabling a vendor to offer one or more custom service providers.</span></span> <span data-ttu-id="b0a25-109">Например, поставщику может потребоваться создать поставщика адресной книги, который использует каталог корпоративной телефонной книги сотрудников, или создать поставщика банка сообщений, который использует существующую базу данных.</span><span class="sxs-lookup"><span data-stu-id="b0a25-109">For example, a vendor might want to create an address book provider that uses a corporate telephone book directory of employees or to create a message store provider that uses an existing database.</span></span>
  
<span data-ttu-id="b0a25-110">Поставщики услуг обычно написаны для определенных систем обмена сообщениями разработчиками программного обеспечения, которые имеют специальные знания или опыт работы с определенной системой.</span><span class="sxs-lookup"><span data-stu-id="b0a25-110">Service providers are typically written for specific messaging systems by software developers who have specialized knowledge or experience with a particular system.</span></span> <span data-ttu-id="b0a25-111">Например, Microsoft Outlook 2013 и Microsoft Outlook 2010, русская версия Mobile Services используют поставщика адресной книги для использования мобильной адресной книги в Outlook.</span><span class="sxs-lookup"><span data-stu-id="b0a25-111">For example, the Microsoft Outlook 2013 and Microsoft Outlook 2010 Mobile Services use an address book provider to expose a mobile address book in Outlook.</span></span> 
  
<span data-ttu-id="b0a25-112">MAPI представляет клиентские приложения с единым представлением адресной книги и сведений о поставщике транспорта.</span><span class="sxs-lookup"><span data-stu-id="b0a25-112">MAPI presents client applications with a unified view of address book and transport provider information.</span></span> <span data-ttu-id="b0a25-113">Этот интегрированный подход не позволяет клиентского приложения соотнесения данных с соответствующим поставщиком.</span><span class="sxs-lookup"><span data-stu-id="b0a25-113">This integrated approach prevents the client application from having to map data to the appropriate provider.</span></span> <span data-ttu-id="b0a25-114">Кроме того, это не позволяет пользователю согласовать несколько схем адресной книги и поставщика транспорта.</span><span class="sxs-lookup"><span data-stu-id="b0a25-114">It also prevents the user from having to negotiate among multiple address book and transport provider addressing schemes.</span></span> <span data-ttu-id="b0a25-115">Однако сведения о поставщике store сообщений не являются унифицированными, и клиенты, которые используют несколько поставщиков store сообщений, отвечают за их обработку по отдельности.</span><span class="sxs-lookup"><span data-stu-id="b0a25-115">Message store provider information, however, is not unified, and clients that use multiple message store providers are responsible for handling them individually.</span></span>
  
<span data-ttu-id="b0a25-116">Поставщики служб работают с MAPI для создания и отправки сообщений следующим образом: сообщения создаются с помощью формы, соответствующей определенному типу или классу сообщений.</span><span class="sxs-lookup"><span data-stu-id="b0a25-116">The service providers work with MAPI to create and send messages in the following way: messages are created by using a form that is appropriate for the specific type, or class, of message.</span></span> <span data-ttu-id="b0a25-117">Многие сообщения составляются с помощью стандартной формы заметок, которая поставляется с подсистемой MAPI либо пользователем клиентского приложения, либо программным путем без участия пользователя.</span><span class="sxs-lookup"><span data-stu-id="b0a25-117">Many messages are made with the standard note form that comes with the MAPI subsystem, either by the user of a client application or programmatically without user interaction.</span></span> <span data-ttu-id="b0a25-118">Завершенное сообщение адресовано одному или более получателям — пользователю или группе пользователей, назначенных для получения сообщения.</span><span class="sxs-lookup"><span data-stu-id="b0a25-118">The completed message is addressed to one or more recipients — a user or group of users designated to receive the message.</span></span> <span data-ttu-id="b0a25-119">У получателя может быть или нет записи в каталоге, владельцем одного из установленных поставщиков адресной книги.</span><span class="sxs-lookup"><span data-stu-id="b0a25-119">A recipient might or might not have an entry in a directory that one of the installed address book providers owns.</span></span> <span data-ttu-id="b0a25-120">Получатели, которые не связаны с установленным поставщиком адресной книги, называются настраиваемые получатели или одновключные адреса.</span><span class="sxs-lookup"><span data-stu-id="b0a25-120">Recipients that are not associated with an installed address book provider are called custom recipients or one-off addresses.</span></span> <span data-ttu-id="b0a25-121">Один разовая адрес может быть временным, сроком только до отправки сообщения.</span><span class="sxs-lookup"><span data-stu-id="b0a25-121">A one-off address can be temporary, lasting only until the message is submitted.</span></span> 
  
<span data-ttu-id="b0a25-122">Когда клиентская приложение отправляет сообщение, поставщик хранилище сообщений проверяет, есть ли у каждого получателя уникальный и действительный адрес, а также есть ли в сообщении все сведения, необходимые для передачи.</span><span class="sxs-lookup"><span data-stu-id="b0a25-122">When the client application sends the message, the message store provider checks that each recipient has a unique and valid address and that the message has all of the information necessary for transmission.</span></span> <span data-ttu-id="b0a25-123">Если возникает вопрос о получателе (например, если имеется несколько получателей с одинаковыми именами), поставщик адресной книги решает проблему.</span><span class="sxs-lookup"><span data-stu-id="b0a25-123">If there is a question about a recipient (for example, when there are multiple recipients with the same name), an address book provider takes care of resolving the ambiguity.</span></span> <span data-ttu-id="b0a25-124">Затем сообщение помещается в исходящие очереди.</span><span class="sxs-lookup"><span data-stu-id="b0a25-124">The message is then placed in the outbound queue.</span></span>
