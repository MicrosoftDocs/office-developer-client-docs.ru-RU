# <a name="pidtagsendrichinfo-canonical-property"></a>Каноническое свойство PidTagSendRichInfo

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Содержит значение TRUE, если получатель может получать все содержимое сообщений, в том числе объекты в формате RTF и объект OLE. 
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_SEND_RICH_INFO  <br/> |
|Идентификатор:  <br/> |0x3A40  <br/> |
|Тип данных:  <br/> |PT_BOOLEAN  <br/> |
|Область:  <br/> |Address  <br/> |
   
## <a name="remarks"></a>Примечания

Рекомендуется, чтобы объекты пользователей списка рассылки и обмена сообщениями раскрывают это свойство. 
  
Это свойство указывает, считает ли отправитель, что для получателя включена поддержка MAPI. 
  
Если для этого свойства задано значение TRUE, транспортный шлюз и шлюз могут передавать все содержимое сообщения, включая объекты RTF и OLE. Поставщик транспорта и шлюз должен использовать формат TNEF для инкапсуляции любых свойств, которые не являются собственными для всех используемых систем обмена сообщениями. 
  
Если для этого свойства задано значение FALSE, поставщик транспорта и шлюз не могут удалять содержимое сообщений, которые не могут использоваться собственными клиентами. Например, если клиенты не поддерживают формат RTF, поставщик транспорта может отправлять только обычный текст. 
  
Если это свойство не задано, поведение по умолчанию определяется реализацией поставщика транспорта, агента передачи сообщений (MTA) или шлюза. Для поддержки этого свойства поставщики адресных книг не требуются. Например, жестко связанная адресная книга и поставщик транспорта могут отправить TNEF, но никогда не использовать формат RTF. 
  
Клиент не должен предполагать, что поставщик транспорта и шлюз будут использовать TNEF в собственной инициативе. Некоторые поставщики транспорта и шлюзы, поддерживающие TNEF, передают их без учета значения этого свойства, но другие отклоняются для создания или отправки TNEF, если для него не задано значение TRUE. 
  
> [!NOTE]
> Параметр этого свойства и решения, основанные на его значении, относятся к отдельному получателю. 
  
По умолчанию MAPI устанавливает значение TRUE. Клиент вызывает [IAddrBook:: креатеонеофф](iaddrbook-createoneoff.md) или поставщик, вызывающий [Имаписуппорт:: креатеонеофф](imapisupport-createoneoff.md) может устанавливать бит **MAPI_SEND_NO_RICH_INFO** в параметре _ulFlags_ , что приводит к тому, что MAPI устанавливает для этого свойства значение false. Односторонние свойства, созданные в пользовательском интерфейсе, используют значение, заданное при создании шаблона. 
  
При вызове метода [IAddrBook:: ресолвенаме](iaddrbook-resolvename.md) , когда имя не удается разрешить, но может интерпретироваться как адрес в Интернете (SMTP), этому свойству присваивается значение false. Если вы хотите использовать адрес в Интернете, отображаемое имя неразрешенной записи должно быть в формате X@Y. Z, например "pete@pinecone.com".
