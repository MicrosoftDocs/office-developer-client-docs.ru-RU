# <a name="pidtagsendrichinfo-canonical-property"></a>Каноническое свойство PidTagSendRichInfo

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Содержит true, если получатель может получить все содержимое сообщения, в том числе объекты RTF и объекты OLE. 
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_SEND_RICH_INFO  <br/> |
|Идентификатор:  <br/> |0x3A40  <br/> |
|Тип данных:  <br/> |PT_BOOLEAN  <br/> |
|Область:  <br/> |Address  <br/> |
   
## <a name="remarks"></a>Примечания

Рекомендуется, чтобы список рассылки и объекты пользователей сообщений выдали это свойство. 
  
Это свойство указывает, считает ли отправитель получателя включенным MAPI. 
  
Если для этого свойства установлено true, транспорт и шлюз могут передавать все содержимое сообщения, включая объекты RTF и OLE. Поставщик транспорта и шлюз должны использовать формат TNEF для инкапсуляции всех свойств, которые не являются входными для всех систем обмена сообщениями. 
  
Если для этого свойства установлено false, поставщик транспорта и шлюз могут отменить содержимое сообщений, которое их собственные клиенты не могут использовать. Например, если клиенты не поддерживают RTF, поставщик транспорта может отправлять только обычный текст. 
  
Если это свойство не установлено, поведение по умолчанию определяется реализацией поставщика транспорта, агента передачи сообщений (MTA) или шлюза. Поставщики адресных книг не требуются для поддержки этого свойства. Например, тесно тесная адресная книга и поставщик транспорта могут выбрать отправку TNEF, но никогда не использовать RTF. 
  
Клиент не должен предполагать, что поставщик транспорта и шлюз будут использовать TNEF по собственной инициативе. Некоторые поставщики транспорта и шлюзы, которые поддерживают TNEF, передают его безототно в отношении значения этого свойства, но другие отклоают создание или отправку TNEF, если для него не установлено значение TRUE. 
  
> [!NOTE]
> Параметр этого свойства и решения, основанные на его значении, настраиваются для каждого получателя. 
  
По умолчанию MAPI устанавливает значение TRUE. Клиент, вызывающий [IAddrBook::CreateOneOff](iaddrbook-createoneoff.md) или поставщик, вызывающий [IMAPISupport::CreateOneOff,](imapisupport-createoneoff.md) может установить бит **MAPI_SEND_NO_RICH_INFO** в параметре  _ulFlags,_ в результате чего MAPI задает для этого свойства false. Одновключицы, созданные пользовательским интерфейсом, используют значение, указанное в шаблоне создания. 
  
При вызове метода [IAddrBook::ResolveName,](iaddrbook-resolvename.md) когда имя не может быть разрешено, но может интерпретироваться как АДРЕС Интернета (SMTP), этому свойству задается false. Чтобы его можно было рассматривать как адрес Интернета, отображаемая запись должна иметь формат X@Y. Z, например "pete@pinecone.com".
