# <a name="pidtagfolderwebviewinfo-cannonical-property"></a>Каноническое свойство PidTagFolderWebViewInfo Cannonical Property

  
  
**Относится к**: Outlook 
  
Содержит URL-адрес для домашней страницы папки в Microsoft Outlook. Это свойство содержит двоичный поток, называется **WebViewPersistenceObject**.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_FOLDER_WEBVIEWINFO  <br/> |
|Идентификатор:  <br/> |0x36DF  <br/> |
|Тип данных:  <br/> |PT_BINARY  <br/> |
|Область:  <br/> |Папки MAPI  <br/> |
   
## <a name="remarks"></a>Замечания

URL-адрес домашней страницы может использоваться для любой папки Outlook. Эти сведения можно обратиться в Outlook из вкладка " **Домашняя страница** " диалогового окна свойств папки. 
  
В зависимости от того, определенные параметры политики на домашнюю страницу может игнорируется с Outlook, если хранилище MAPI, содержащем эту папку не дает MSCAP_SECURE_FOLDER_HOMEPAGES в реализации [IMSCapabilities::GetCapabilities](pidtagfolderwebviewinfo-cannonical-property.md) . 
  
Папки **Outlook сегодня** и общей папки может иметь URL-адреса домашней страницы. Тем не менее папки **Outlook сегодня** используется другой механизм для управления его URL-адрес домашней страницы; Этот механизм не рассматриваются в данном разделе. Общедоступная папка также могут иметь определенные в нем URL домашней страницы, характерные для пользователя. Тем не менее эту возможность не описанных в данном разделе. 
  
Значение этого свойства — это двоичный поток, называется **WebViewPersistenceObject**.
  
### <a name="webviewpersistenceobject-stream-structure"></a>Структура WebViewPersistenceObject потока

Структура потока **WebViewPersistenceObject** содержит сведения об URL-адрес домашней страницы для папки. 
  
Элементы данных в эту структуру хранятся в прямом байтовом, сразу после друг с другом в следующем порядке. 
  
> [!NOTE]
> Следующее описание не список всех значений поля, поддерживаются в Outlook; Таким образом когда код считывает существующего потока, некоторые флаги, не перечисленных здесь может также найти. Тем не менее можно использовать это описание для программного создания значения для свойства **PidTagFolderWebViewInfo** , получите представление об Outlook. 
  
 _dwVersion_
  
> Значение DWORD (4 байта). Версия формата структуры. По состоянию на Microsoft Office Outlook 2007 поддерживается только значение для этого поля — следующим образом.
    
|**Value name**|**Значение**|
|:-----|:-----|
|WEBVIEW_PERSISTENCE_VERSION  <br/> |0x00000002  <br/> |
   
 _dwType_
  
> Значение DWORD (4 байта). Тип данных домашней страницы. По состоянию на Microsoft Office Outlook 2007 поддерживается только значение для этого поля — следующим образом.
    
|**Value name**|**Значение**|
|:-----|:-----|
|WEBVIEWURL  <br/> |0x00000001  <br/> |
   
 _dwFlags_
  
> Значение DWORD (4 байта). Сочетание ноль или больше помечает, значения и значения перечислены в следующей таблице.
    
|Флаг имя ***|Значение ***|****Описание****|
|:-----|:-----|:-----|
|WEBVIEW_FLAGS_SHOWBYDEFAULT  <br/> |0x00000001  <br/> |На вкладке **Домашняя страница** диалоговое окно Свойства для папки установлен флажок **Показывать домашнюю страницу по умолчанию для этой папки** .  <br/> |
   
 _dwUnused [7]_
  
> Массив элементов 7 DWORD (28 байт). Не используется.
    
cbData
  
> ULONG (4 байта). Размер в байтах, _wzURL_ элемента данных. 
    
 _wzURL_
  
> Массив элементов WCHAR. Представление UTF-16 в строке URL-адреса нулем домашней страницы.
