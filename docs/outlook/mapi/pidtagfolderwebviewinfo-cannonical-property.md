# <a name="pidtagfolderwebviewinfo-cannonical-property"></a>Каноническое свойство PidTagFolderWebViewInfo

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит URL-адрес домашней страницы папки в Microsoft Outlook. Это свойство содержит двоичный поток с именем **вебвиевперсистенцеобжект**.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |ПР_ФОЛДЕР_ВЕБВИЕВИНФО  <br/> |
|Идентификатор:  <br/> |0x36DF  <br/> |
|Тип данных:  <br/> |PT_BINARY  <br/> |
|Область:  <br/> |Папка MAPI  <br/> |
   
## <a name="remarks"></a>Замечания

URL-адрес домашней страницы можно указать для любой папки Outlook. Эту информацию можно получить в Outlook на вкладке " **домашнЯя страница** " диалогового окна Свойства для папки. 
  
В зависимости от определенных параметров политики Домашняя страница может быть проигнорирована в Outlook, если хранилище MAPI, содержащее эту папку, не сообщает о МСКАП_СЕКУРЕ_ФОЛДЕР_ХОМЕПАЖЕС в реализации [IMSCapabilities:: Capabilities](pidtagfolderwebviewinfo-cannonical-property.md) . 
  
В папке " **Outlook сегодня** " и в общедоступной папке могут быть URL-адреса домашней страницы. Однако папка " **Outlook сегодня** " использует другой механизм для управления URL-адресом домашней страницы; Этот механизм не рассматривается в этом разделе. В общедоступной папке также может быть определен URL-адрес домашней страницы, относящийся к конкретному пользователю. Однако эта возможность не описывается в этой статье. 
  
Значение этого свойства является двоичным потоком, который называется **вебвиевперсистенцеобжект**.
  
### <a name="webviewpersistenceobject-stream-structure"></a>Структура потока Вебвиевперсистенцеобжект

Структура потока **вебвиевперсистенцеобжект** содержит сведения об URL-адресе домашней страницы для папки. 
  
Элементы данных в этой структуре хранятся в обратном байтовом порядке, непосредственно следующем за другим в приведенном ниже указанном порядке. 
  
> [!NOTE]
> Следующее описание может не перечислить все значения полей, поддерживаемые в Outlook; Таким образом, когда код читает существующий поток, некоторые флаги, которые не указаны здесь, также могут быть найдены. Однако это описание можно использовать для программного создания значений для свойства **каноническое pidtagfolderwebviewinfo** , которое будет понято в Outlook. 
  
 _Двверсион_
  
> DWORD (4 байта). Версия формата структуры. По мере использования Microsoft Office Outlook 2007 для этого поля поддерживается только следующее значение.
    
|**Имя значения**|**Значение**|
|:-----|:-----|
|ВЕБВИЕВ_ПЕРСИСТЕНЦЕ_ВЕРСИОН  <br/> |0x00000002  <br/> |
   
 _Двтипе_
  
> DWORD (4 байта). Тип данных домашней страницы. По мере использования Microsoft Office Outlook 2007 для этого поля поддерживается только следующее значение.
    
|**Имя значения**|**Значение**|
|:-----|:-----|
|ВЕБВИЕВУРЛ  <br/> |0x00000001  <br/> |
   
 _dwFlags_
  
> DWORD (4 байта). Сочетание нулевого или большего числа флагов, значения и значения которых перечислены в следующей таблице.
    
|Имя флага * * * *|****Value** (Значение)**|****Описание****|
|:-----|:-----|:-----|
|ВЕБВИЕВ_ФЛАГС_ШОВБИДЕФАУЛТ  <br/> |0x00000001  <br/> |Флажок **Показывать домашнюю страницу по умолчанию для этой папки** установлен на вкладке " **Домашняя страница** " диалогового окна Свойства для папки.  <br/> |
   
 _Двунусед [7]_
  
> Массив из 7 элементов DWORD (всего 28 байт). Неиспользуемых.
    
КБДата
  
> ULONG (4 байта). Размер элемента данных _взурл_ в байтах. 
    
 _Взурл_
  
> Массив элементов типа WCHAR. Представление UTF-16 строки URL-адреса домашней страницы, завершающейся нулем.
