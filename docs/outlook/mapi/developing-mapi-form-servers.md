# <a name="developing-mapi-form-servers"></a>Разработка серверов форм MAPI

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
В этом разделе описывается процесс создания исполняемых файлов сервера форм и файлов конфигурации форм для создания настраиваемых форм MAPI. Прежде чем читать этот раздел, ознакомьтесь с информацией в [формах MAPI.](mapi-forms.md)
  
Разработка сервера форм включает следующие этапы:
  
1. Выбор сведений, которые будут содержаться в форме, и выбор набора свойств для их размещения. Дополнительные сведения [см. в выборе набора свойств формы.](choosing-a-form-s-property-set.md)
    
2. Проектирование пользовательского интерфейса, с которым пользователи могут взаимодействовать со свойствами формы.
    
3. Выбор класса сообщения и создание уникального идентификатора класса (CLSID). Обзор классов сообщений см. в сведениях [о классах сообщений MAPI.](mapi-message-classes.md) Дополнительные сведения о классах и формах сообщений см. в [подклассе "Выбор класса сообщений".](choosing-a-message-class.md)
    
4. Реализация необходимых интерфейсов форм MAPI, а также дополнительных интерфейсов, необходимых конкретному серверу форм. Дополнительные сведения см. в [записи кода сервера форм.](writing-form-server-code.md) 
    
5. Написание кода пользовательского интерфейса для обработки взаимодействия пользователя с объектом формы и свойствами, которые использует форма.
    
6. Создание файла конфигурации формы. Дополнительные сведения [см. в формате файлов конфигурации форм.](file-format-of-form-configuration-files.md)
    
7. Установка формы на компьютерах пользователей. Дополнительные сведения см. [в установке формы в библиотеку.](installing-a-form-into-a-library.md)
    
Скорее всего, вы будете выполнять шаги с 1 по 5 одновременно, а не последовательно. Процесс разработки сервера форм, как и многих проектов программирования, не является процессом, в котором имеется особенно четко определяемая последовательность. Например, создание файла конфигурации формы отображается как шаг от второго до последнего выше, но, вероятно, файл конфигурации формы будет создаваться постепенно, и он станет более полным по мере добавления компонентов на сервер форм.
