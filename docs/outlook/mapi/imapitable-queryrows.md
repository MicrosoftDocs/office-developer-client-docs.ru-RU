# <a name="imapitablequeryrows"></a>IMAPITable::QueryRows

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Возвращает одну или несколько строк из таблицы, начиная с текущего положения курсора.
  
```cpp
HRESULT QueryRows(
LONG lRowCount,
ULONG ulFlags,
LPSRowSet FAR * lppRows
);
```

## <a name="parameters"></a>Параметры

 _lRowCount_
  
> [in] Максимальное число возвращаемых строк.
    
 _ulFlags_
  
> [in] Битоваяmas флагов, которые контролируют, как возвращаются строки. Можно установить следующий флаг:
    
TBL_NOADVANCE 
  
> Предотвращает наступание курсора в результате искомой строки. Если установлен TBL_NOADVANCE, курсор указывает на первую возвращенную строку. Если флаг TBL_NOADVANCE не установлен, курсор указывает на строку после последней возвращаемой строки.
    
 _lppRows_
  
> [out] Указатель на указатель на структуру [SRowSet,](srowset.md) удерживая строки таблицы. 
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Строки были успешно возвращены.
    
MAPI_E_BUSY 
  
> В настоящее время идет другая операция, которая не позволяет начать операцию ирисовки строк. Либо операция должна быть разрешена для завершения, либо она должна быть остановлена.
    
MAPI_E_INVALID_PARAMETER 
  
> Для  _параметра IRowCount_ задан ноль. 
    
## <a name="remarks"></a>Примечания

Метод **IMAPITable::QueryRows** получает одну или несколько строк данных из таблицы. Значение параметра  _IRowCount_ влияет на отправную точку ищего. Если  _IRowCount_ положительное, строки считыются в направлении вперед, начиная с текущей позиции. Если  _IRowCount_ отрицательное, **QueryRows** сбрасывает началую точку, перемещая назад задаваемого числа строк. После сброса курсора строки считыются в порядке переадселки. 
  
Член **cRows** в структуре [SRowSet,](srowset.md) на который указывает параметр  _lppRows,_ указывает количество возвращаемой строки. Если возвращается ноль строк: 
  
- Курсор уже был в начале таблицы, а значение  _IRowCount_ отрицательное. -Or- 
    
- Курсор уже находится в конце таблицы, а  _значение IRowCount_ положительное. 
    
Количество столбцов и их порядок одинаковы для каждой строки. Если свойство не существует для строки или при чтении свойства произошла ошибка, **структура SPropValue** для свойства в строке содержит следующие значения: 
  
- PT_ERROR для типа свойства в члене **ulPropTag.** 
    
- MAPI_E_NOT_FOUND для члена **Value.** 
    
Память, используемая для структур [SPropValue](spropvalue.md) в наборе строк, на которые указывает параметр  _lppRows,_ должна быть выделена отдельно и освобождена для каждой строки. Используйте [MAPIFreeBuffer,](mapifreebuffer.md) чтобы освободить структуры значений свойств и освободить набор строк. Однако когда вызов **QueryRows** возвращает ноль, указывающий начало или конец таблицы, необходимо освободить только сам **набор SRowSet.** Дополнительные сведения о выделении и выделении памяти в структуре **SRowSet** см. в под управлением памятью [для ADRLIST и структур SRowSet.](managing-memory-for-adrlist-and-srowset-structures.md)
  
Возвращаемые строки и порядок их возврата зависят от того, были ли выполнены успешные вызовы [IMAPITable::Restrict](imapitable-restrict.md) и [IMAPITable::SortTable.](imapitable-sorttable.md) **Ограничение** строк фильтров из представления, в результате чего **QueryRows** возвращает только строки, которые соответствуют критериям, указанным в ограничении. **SortTable** устанавливает стандартный или упорядоченный порядок сортировки, влияющий на последовательность строк, возвращаемых **QueryRows.** Возвращаемые строки находятся в порядке, указанном в структуре [SSortOrderSet,](ssortorderset.md) переданном **в SortTable.**
  
Столбцы, возвращаемые для каждой строки, и порядок их возврата зависят от того, был ли выполнен успешный вызов [IMAPITable::SetColumns.](imapitable-setcolumns.md) **SetColumns** устанавливает набор столбцов, определяя свойства, которые необходимо включить в столбцы таблицы, и порядок, в котором они должны быть включены. Если был выполнен вызов **SetColumns,** определенные столбцы в каждой строке и порядок этих столбцов соответствуют набору столбцов, указанному в вызове. Если вызов **SetColumns** не выполнен, таблица возвращает набор столбцов по умолчанию. 
  
Если ни один из этих вызовов не был выполнен, **QueryRows** возвращает все строки в таблице. Каждая строка содержит набор столбцов по умолчанию в порядке по умолчанию. 
  
Когда набор столбцов, установленный в вызове [IMAPITable::SetColumns,](imapitable-setcolumns.md) включает столбцы, задаваемые как PR_NULL, массив [SPropValue в](spropvalue.md) наборе строк, возвращаемом  _в lppRows,_ будет содержать пустые слоты. 
  
## <a name="notes-to-implementers"></a>Примечания для исполнителей

Вы можете разрешить звонячему запрашивать неподдермеченный столбец, который должен быть включен в набор столбцов. В этом случае поместите PT_ERROR в часть типа свойства тега свойства, а MAPI_E_NOT_FOUND в значение свойства для неподтверганного столбца. 
  
Обрабатывайте количество строк как запрос, а не как требование. Вы можете вернуться в любом месте от нуля строк, если в направлении запроса нет строк, до запрашиваемого числа. 
  
Возвращает только строки, которые будут видеться пользователем при запросе строк из представления таблицы с классификацией, позволяя вызываемой стороне делать допустимые предположения об области данных и избегать дополнительной работы. 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

Обычно у вас будет столько строк, сколько указано в параметре _lRowCount._ Однако если в памяти или реализации не существует проблем или операция достигает начала или окончания таблицы раньше времени, **QueryRows** возвращает меньше строк, чем запрашивается. 
  
Если **QueryRows** возвращает MAPI_E_BUSY, вызовите метод [IMAPITable::WaitForCompletion](imapitable-waitforcompletion.md) и повторно выполните вызов **QueryRows** после завершения асинхронной операции. 
  
При **вызове QueryRows** следует помнить, что синхронизация асинхронных уведомлений может привести к тому, что набор строк, возвращаемой из **QueryRows,** не будет точно представлять данные, которые следует использовать. Например, вызов **QueryRows** в таблицу содержимого папки после удаления сообщения, но до получения соответствующего уведомления приведет к возвращению удаленной строки в наборе строк. Всегда дождись поступления уведомления перед обновлением представления данных пользователем. 
  
Дополнительные сведения о том, как получить строки из таблиц, см. в этой [теме.](retrieving-data-from-table-rows.md)
