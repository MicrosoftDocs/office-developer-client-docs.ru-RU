# <a name="sbitmaskrestriction"></a>SBitMaskRestriction

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Описывает ограничение битовой маски, которое используется для выполнения поразрядной операции **and** и проверки результата. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |MAPIDEFS. h  <br/> |
   
```cpp
typedef struct _SBitMaskRestriction
{
  ULONG relBMR;
  PT_LONG ulPropTag;
  ULONG ulMask;
} SBitMaskRestriction;

```

## <a name="members"></a>Members

 **Релбмр**
  
> Оператор отношения, описывающий, как маска, указанная в элементе **улмаск** , должна быть применена к тегу свойства. Возможны следующие значения: 
    
БМР_ЕКЗ 
  
> Выполните побитовую операцию **and** для маски в элементе **улмаск** со свойством, представленным элементом **улпроптаг** , и протестируйте его равным нулю. 
    
БМР_НЕЗ 
  
> Выполните побитовую операцию **and** для маски в элементе **улмаск** со свойством, представленным элементом **улпроптаг** , и проверьте, что оно не равно нулю. 
    
 **Улпроптаг**
  
> Тег свойства свойства, к которому применяется битовая маска.
    
 **Улмаск**
  
> Битовая маска, применяемая к свойству, определенному с помощью **улпроптаг**.
    
## <a name="remarks"></a>Примечания

Структура **сбитмаскрестриктион** выполняет битовую операцию **и** использует битовую маску, описанную в элементе **улмаск** , и значение свойства, описываемого элементом **улпроптаг** . Если результат равен нулю, БМР_ЕКЗ является удовлетворенным. Если это ненулевое значение, то есть, если значение свойства имеет по крайней мере один такой же бит, что и **улмаск**, то бмр_нез является удовлетворенным.
  
Для получения дополнительных сведений о структуре и ограничениях **сбитмаскрестриктион** в целом ознакомьтесь [](about-restrictions.md)с ограничениями.
