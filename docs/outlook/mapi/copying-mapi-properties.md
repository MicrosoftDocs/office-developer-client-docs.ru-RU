# <a name="copying-mapi-properties"></a>Копирование свойств MAPI

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Клиенты и поставщики услуг могут копировать одно или несколько свойств объекта с помощью следующих методов **IMAPIProp** и функций API: 
  
- Метод [IMAPIProp:: CopyTo](imapiprop-copyto.md) копирует все свойства объекта в другой объект, при необходимости исключая выбранные свойства. **CopyTo** используется для копирования или перемещения объектов любого типа. 
    
- Метод [IMAPIProp:: копипропс](imapiprop-copyprops.md) копирует выбранные свойства объекта. **Копипропс** используется в основном с сообщениями. Когда клиент создает переадресованную копию сообщения или ответа, **копипропс** обрабатывает копирование соответствующих свойств из исходного сообщения. 
    
- Функция [пропкопиморе](propcopymore.md) копирует одно значение свойства из одного расположения в другое. Используйте **пропкопиморе** с осторожностью. Возможно, при копировании одного значения за раз, чтобы выделить множество небольших блоков памяти и вызвать фрагментацию памяти. 
    
- Функция [сккопипропс](sccopyprops.md) копирует значения свойств в пакетном режиме. **Сккопипропс** может копировать значения свойств, созданных из несвязных блоков памяти. Он возвращает новый массив свойств. 
    
- Если массив свойств, возвращаемый **сккопипропс** , должен храниться на диске, используйте функцию [скрелокпропс](screlocprops.md) для изменения указателей. **Скрелокпропс** должен вызываться дважды; один раз, чтобы изменить адреса перед записью операции с данными и затем снова во время операции чтения. Функция **скрелокпропс** предполагает, что массив значений свойства был изначально выделен в виде одного выделения. 
    
Функции API, описанные в предыдущих свойствах копирования списка в памяти, а не из одного объекта в другой объект. Эти функции в настоящее время поддерживаются, но могут не поддерживаться в будущих выпусках.
