# <a name="copying-mapi-properties"></a>Копирование свойств MAPI

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Клиенты и поставщики услуг могут копировать одно или несколько свойств объекта с помощью следующих методов **IMAPIProp** и функций API: 
  
- Метод [IMAPIProp::CopyTo](imapiprop-copyto.md) копирует все свойства объекта в другой объект, дополнительно исключая выбранные свойства. **CopyTo** используется для копирования или перемещения любого типа объекта. 
    
- Метод [IMAPIProp::CopyProps](imapiprop-copyprops.md) копирует выбранные свойства объекта. **CopyProps** используется главным образом с сообщениями. Когда клиент создает переадрещенную копию сообщения или ответа, **CopyProps** обрабатывает копирование соответствующих свойств из исходного сообщения. 
    
- Функция [PropCopyMore](propcopymore.md) копирует одно значение свойства из одного расположения в другое. Используйте **PropCopyMore с** осторожностью. При копировании по одному значению можно выделить много небольших блоков памяти и привести к фрагментации памяти. 
    
- Функция [ScCopyProps](sccopyprops.md) массово копирует значения свойств. **ScCopyProps** может копировать значения свойств, построенные из несоединенных блоков памяти. Он возвращает новый массив свойств. 
    
- Если массив свойств, возвращаемого **ScCopyProps,** должен храниться на диске, используйте функцию [ScRelocProps](screlocprops.md) для настройки указателей. **ScRelocProps** должен быть вызван дважды; один раз для настройки адресов перед записью операции с данными, а затем еще раз во время операции чтения. Функция **ScRelocProps** предполагает, что массив значений свойств изначально был выделен в одном выделении. 
    
Функции API, описанные в предыдущем списке, копируют свойства в памяти, а не из одного объекта в другой. В настоящее время эти функции поддерживаются, но могут не поддерживаться в будущих выпусках.
