# <a name="copying-mapi-properties"></a>Копирование свойств MAPI

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Клиенты и поставщики услуг могут скопировать одно или несколько свойств объекта с помощью следующих методов **IMAPIProp** и функций API: 
  
- Метод [IMAPIProp::CopyTo](imapiprop-copyto.md) копирует все свойства объекта на другой объект, необязательно исключая выбранные свойства. **CopyTo** используется для копирования или перемещения любого типа объекта. 
    
- Метод [IMAPIProp::CopyProps](imapiprop-copyprops.md) копирует выбранные свойства объекта. **CopyProps** используется в основном с сообщениями. Когда клиент создает перенаправимую копию сообщения или ответа, **CopyProps** обрабатывает копирование соответствующих свойств из исходного сообщения. 
    
- Функция [PropCopyMore](propcopymore.md) копирует одно значение свойства из одного расположения в другое. Используйте **PropCopyMore с** осторожностью. При копировании одного значения можно выделить несколько небольших блоков памяти и привести к фрагментации памяти. 
    
- Функция [ScCopyProps](sccopyprops.md) массово копирует значения свойств. **ScCopyProps** может копировать значения свойств, которые были построены из несоединенных блоков памяти. Возвращается новый массив свойств. 
    
- Если массив свойств, возвращенный **ScCopyProps,** должен храниться на диске, для настройки указателей используйте функцию [ScRelocProps.](screlocprops.md) **ScRelocProps** следует дважды обзвонить; один раз для настройки адресов перед написанием операции данных, а затем снова во время операции чтения. Функция **ScRelocProps** предполагает, что массив значений свойств изначально был выделен в одном выделении. 
    
Функции API, описанные в предыдущих свойствах копирования списка в памяти, а не от одного объекта к другому объекту. Эти функции в настоящее время поддерживаются, но могут не поддерживаться в будущем выпуске.
