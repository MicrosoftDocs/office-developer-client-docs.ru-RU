# <a name="copying-mapi-properties"></a>Копирование свойств MAPI

  
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
Клиенты и поставщиков услуг можно скопировать один или несколько свойств объекта с помощью следующих методов **IMAPIProp** и функции интерфейса API: 
  
- Метод [IMAPIProp::CopyTo](imapiprop-copyto.md) копирует все свойства объекта в другой объект, при необходимости за исключением выбранных свойств. **CopyTo** используется для копирования или перемещения любой тип объекта. 
    
- Метод [IMAPIProp::CopyProps](imapiprop-copyprops.md) копирует выбранных свойств объекта. **CopyProps** используется главным образом с сообщениями. Когда клиента создает копию переадресованных сообщение или ответ, **CopyProps** значками копирование соответствующие свойства из исходного сообщения. 
    
- Функция [PropCopyMore](propcopymore.md) копирует значения одного свойства из одного места в другое. Используйте **PropCopyMore** с осторожностью. Возможно — при копировании одно значение за раз — для размещения множество небольших блоков памяти и привести к памяти для фрагментов. 
    
- Функция [ScCopyProps](sccopyprops.md) копирует значения свойств в пакетном режиме. **ScCopyProps** можно скопировать значения свойств, которые были созданы из несвязанное блоки памяти. Возвращает новый массив свойство. 
    
- Если свойство массив, возвращенный **ScCopyProps** будут храниться на диске, используйте функцию [ScRelocProps](screlocprops.md) регулировка указатели. **ScRelocProps** должен быть вызван дважды; один раз, чтобы настроить адреса перед записью операции с данными и затем снова во время операции чтения. Функция **ScRelocProps** предполагается, что массива значение свойства сначала выделенная в за одну операцию. 
    
Функции интерфейса API, описано в предыдущей копии списка свойств в памяти, а не из одного объекта в другой объект. Эти функции поддерживаются в настоящее время, но может не поддерживаться в следующем выпуске.
