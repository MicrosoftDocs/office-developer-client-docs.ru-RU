# <a name="imapiformmgrloadform"></a>IMAPIFormMgr::LoadForm

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Запускает форму, чтобы открыть существующее сообщение.
  
```cpp
HRESULT LoadForm(
  ULONG_PTR ulUIParam,
  ULONG ulFlags,
  LPCSTR lpszMessageClass,
  ULONG ulMessageStatus,
  ULONG ulMessageFlags,
  LPMAPIFOLDER pFolderFocus,
  LPMAPIMESSAGESITE pMessageSite,
  LPMESSAGE pmsg,
  LPMAPIVIEWCONTEXT pViewContext,
  REFIID riid,
  LPVOID FAR * ppvObj
);
```

## <a name="parameters"></a>Параметры

 _улуипарам_
  
> возврата Дескриптор родительского окна индикатора хода выполнения, отображаемого при открытии формы. Параметр _улуипарам_ игнорируется, если в параметре _ulFlags_ не задан флаг MAPI_DIALOG. 
    
 _ulFlags_
  
> возврата Битовая маска флагов, определяющих способ открытия формы. Можно задать следующие флаги:
    
MAPI_DIALOG 
  
> Отображает пользовательский интерфейс для предоставления сведений о состоянии или предлагает пользователю ввести дополнительные сведения. Если этот флаг не установлен, Пользовательский интерфейс не отображается.
    
MAPIFORM_EXACTMATCH 
  
> Необходимо разрешить только строки класса сообщений с точным совпадением.
    
 _лпсзмессажекласс_
  
> возврата Указатель на строку, которая присваивает имя классу сообщения, которое необходимо загрузить. Если в параметре _лпсзмессажекласс_ передается значение null, класс Message определяется на основе сообщения, на которое указывает параметр _ПМСГ_ . 
    
 _улмессажестатус_
  
> возврата Битовая маска заданных клиентом или поставщиками флагов, скопированных из свойства **PR_MSG_STATUS** ([PidTagMessageStatus](pidtagmessagestatus-canonical-property.md)) сообщения, которое содержит сведения о состоянии сообщения. Если _лпсзмессажекласс_ не имеет значение null, необходимо задать параметр _улмессажестатус_ . в противном случае _улмессажестатус_ игнорируется. 
    
 _улмессажефлагс_
  
> возврата Указатель на битовую маску флагов, скопированных из свойства **PR_MESSAGE_FLAGS** ([PidTagMessageFlags](pidtagmessageflags-canonical-property.md)) сообщения, которое показывает текущее состояние сообщения. Если _лпсзмессажекласс_ не имеет значение null, необходимо задать параметр _улмессажефлагс_ . в противном случае _улмессажефлагс_ игнорируется. 
    
 _пфолдерфокус_
  
> возврата Указатель на папку, непосредственно содержащую сообщение. Параметр _пфолдерфокус_ может иметь значение null, если такая папка не существует (например, если сообщение внедрено в другое сообщение). 
    
 _пмессажесите_
  
> возврата Указатель на сайт сообщения.
    
 _пмсг_
  
> возврата Указатель на сообщение.
    
 _пвиевконтекст_
  
> возврата Указатель на контекст представления для сообщения. Параметр _пвиевконтекст_ может иметь значение null. 
    
 _риид_
  
> возврата Идентификатор интерфейса (IID) интерфейса, который будет использоваться для возвращаемого объекта Form. Параметр _РИИД_ не должен иметь значение null. 
    
 _ппвобж_
  
> вышли Указатель на указатель на возвращенный интерфейс.

## <a name="remarks"></a>Примечания

Для просмотра форм вызывается метод **имапиформмгр:: лоадформ** , чтобы открыть форму для существующего сообщения. **Лоадформ** открывает объект Form, загружает сообщение в объект формы, настраивает соответствующий контекст представления, при необходимости и возвращает запрошенный интерфейс для объекта Form. 
  
Параметр _пфолдерфокус_ указывает папку, в которой находится сообщение. Если сообщение внедрено в другое сообщение, _пфолдерфокус_ должно иметь значение null. 
  
## <a name="notes-to-implementers"></a>Примечания для исполнителей

Если в _лпсзмессажекласс_передается значение null, то реализация получает класс сообщений, состояние и флаги сообщения из **PR_MESSAGE_CLASS** ([PidTagMessageClass](pidtagmessageclass-canonical-property.md)), **PR_MSG_STATUS** и **PR_MESSAGE_FLAGS** свойств. Если строка класса сообщения предоставлена в _лпсзмессажекласс_, реализация должна использовать значения в _улмессажестатус_ и _улмессажефлагс_.
