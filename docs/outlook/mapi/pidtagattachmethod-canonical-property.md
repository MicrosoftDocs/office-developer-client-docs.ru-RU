# <a name="pidtagattachmethod-canonical-property"></a>Каноническое свойство PidTagAttachMethod

 
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит константу, определенную MAPI, которая представляет способ доступа к содержимому вложения. 
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |ПР_АТТАЧ_МЕСОД  <br/> |
|Идентификатор:  <br/> |0x3705  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Вложение в сообщение  <br/> |
   
## <a name="remarks"></a>Замечания

Это свойство может иметь только одно из следующих значений:
  
НО_АТТАЧМЕНТ 
  
> Вложение только что было создано. 
    
АТТАЧ_БИ_ВАЛУЕ 
  
> Свойство **пр_аттач_дата_бин** ([PidTagAttachDataBinary](pidtagattachdatabinary-canonical-property.md)) содержит данные вложений. 
    
АТТАЧ_БИ_РЕФЕРЕНЦЕ 
  
> Свойство **пр_аттач_паснаме** ([PidTagAttachPathname](pidtagattachpathname-canonical-property.md)) или **пр_аттач_лонг_паснаме** ([PidTagAttachLongPathname](pidtagattachlongpathname-canonical-property.md)) содержит полный путь, определяющий вложение получателям, у которых есть доступ к общему файлу. сервер. 
    
АТТАЧ_БИ_РЕФ_РЕСОЛВЕ 
  
> Свойство **пр_аттач_паснаме** или **пр_аттач_лонг_паснаме** содержит полный путь, указывающий вложение. 
    
АТТАЧ_БИ_РЕФ_ОНЛИ 
  
> Свойство **пр_аттач_паснаме** или **пр_аттач_лонг_паснаме** содержит полный путь, указывающий вложение. 
    
АТТАЧ_ЕМБЕДДЕД_МСГ 
  
> Свойство **пр_аттач_дата_обж** ([PidTagAttachDataObject](pidtagattachdataobject-canonical-property.md)) содержит внедренный объект, который поддерживает интерфейс **iMessage** . 
    
АТТАЧ_ОЛЕ 
  
> Вложение является внедренным объектом OLE.
    
АТТАЧ_БИ_ВЕБРЕФЕРЕНЦЕ 
  
> Содержимое вложения отсутствует в сообщении. 
    
При создании все объекты вложения имеют начальное значение **пр_аттач_месод** для **но_аттачмент**. 
  
Клиентские приложения и поставщики услуг необходимы только для поддержки метода вложения, представленного значением **аттач_би_валуе** . Другие методы вложения являются необязательными. Хранилище сообщений не обеспечивает никакой согласованности между значением **пр_аттач_месод** и значениями других свойств вложения. 
  
Имена в формате UNC рекомендуются для полных путей, которые следует использовать с **аттач_би_референце** и **аттач_би_реф_онли**. В **аттач_би_реф_ресолве**абсолютный путь выполняется быстрее, так как диспетчер очереди MAPI преобразует вложение в **аттач_би_валуе**. 
  
Если задано значение **аттач_би_референце** , **пр_аттач_дата_бин** должно быть пустым. Исходящий шлюз может превратить вложение **аттач_би_референце** в вложение **аттач_би_валуе** путем копирования данных вложения в свойство **пр_аттач_дата_бин** . 
  
Если задано значение **аттач_би_реф_ресолве** , **пр_аттач_дата_бин** должно быть пустым. При отправке сообщения, содержащего вложение **аттач_би_реф_ресолве** , диспетчер очереди MAPI копирует данные вложения в вложение **аттач_би_валуе** . В этом процессе разрешения данные вложений размещаются в **пр_аттач_дата_бин**. 
  
Если задано значение **аттач_би_реф_онли** , **пр_аттач_дата_бин** должно быть пустым, а система обмена сообщениями никогда не разрешает ссылку вложения. Используйте это значение, если хотите отправить ссылку, но не данные. 
  
Если OLE-объект находится в формате **ISTORAGE** OLE 2,0, доступ к данным осуществляется с помощью **пр_аттач_дата_обж**. Когда объект OLE представлен в формате OLE 1,0 **олестреам** , доступ к данным предоставляется через **Пр_аттач_дата_бин** в виде **IStream**. Тип OLE-кодировки можно определить с помощью значения **пр_аттач_таг** ([PidTagAttachTag](pidtagattachtag-canonical-property.md)). 
  
Дополнительные сведения о интерфейсах и форматах OLE приведены в *справочнике по OLE* для программистов. 
  
## <a name="remarks"></a>Замечания

Если для параметра **пр_аттач_месод** задано значение **аттач_би_вебреференце**, содержимое вложения отсутствует в сообщении. Вместо этого свойство **пр_аттач_лонг_филенаме** содержит абсолютный URL-адрес содержимого вложений, которое хранится в сети.
