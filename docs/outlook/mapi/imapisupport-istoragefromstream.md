# <a name="imapisupportistoragefromstream"></a>IMAPISupport::IStorageFromStream

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Реализует объект Storage для доступа к потоку.
  
```cpp
HRESULT IStorageFromStream(
  LPUNKNOWN lpUnkIn,
  LPCIID lpInterface,
  ULONG ulFlags,
  LPSTORAGE FAR * lppStorageOut
);
```

## <a name="parameters"></a>Параметры

 _Лпункин_
  
> возврата Указатель на объект Stream.
    
 _Лпинтерфаце_
  
> возврата Указатель на идентификатор интерфейса (IID), представляющий интерфейс, который будет использоваться для доступа к потоку, на который указывает _лпункин_. Допустимы следующие значения: Иид_истреам, Иид_илоккбитес или **null**, что указывает на то, что для доступа к потоку должен использоваться интерфейс [IStream](https://msdn.microsoft.com/library/aa380034%28VS.85%29.aspx) . 
    
 _ulFlags_
  
> возврата Битовая маска флагов, которые управляют способом создания объекта хранилища относительно объекта Stream. По умолчанию хранилище создается с доступом только для чтения, а поток начинается с нулевой позиции в хранилище. Можно задать следующие флаги:
    
СТГСТРМ_КРЕАТЕ 
  
> Для объекта Stream необходимо создать новый объект хранилища.
    
СТГСТРМ_КУРРЕНТ 
  
> Объект Storage должен начинаться с текущей позиции в потоке.
    
СТГСТРМ_МОДИФИ 
  
> Вызывающий абонент должен иметь разрешение на чтение и запись возвращенного объекта хранилища.
    
СТГСТРМ_РЕСЕТ 
  
> Объект Storage должен начинаться с позиции 0.
    
 _Лппсторажеаут_
  
> вышли Указатель на указатель на объект хранилища.
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Объект хранилища успешно создан.
    
## <a name="remarks"></a>Замечания

Метод **имаписуппорт:: исторажефромстреам** реализован для всех объектов поддержки поставщика услуг. Поставщики услуг вызывают **исторажефромстреам** , чтобы создать объект хранилища, который будет использоваться для открытия определенных свойств. Поставщики услуг, у которых есть собственная реализация интерфейса [IStorage](https://msdn.microsoft.com/library/aa380015%28VS.85%29.aspx) , не обязаны вызывать **исторажефромстреам**. 
  
Объект хранилища, созданный с помощью **исторажефромстреам** , вызывает метод [IUnknown:: AddRef](https://msdn.microsoft.com/library/ms691379%28v=VS.85%29.aspx) для увеличения счетчика ссылок, а затем уменьшает значение счетчика при отпускании хранилища. 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

При вызове метода [IMAPIProp:: опенпроперти](imapiprop-openproperty.md) одного из объектов, чтобы открыть свойство с интерфейсом **IStorage** , выполните следующие задачи: 
  
1. Откройте объект Stream с разрешениями на чтение и запись для свойства.
    
2. Помечает поток свойств как объект хранилища.
    
3. Call **исторажефромстреам** для создания объекта Storage. 
    
4. Возвращает указатель на этот объект хранилища.
    
При реализации дополнительных интерфейсов, использующих объект Storage, создайте объект, который упаковывает объект Storage и реализует метод [IUnknown:: QueryInterface](https://msdn.microsoft.com/library/ms682521%28v=VS.85%29.aspx) более высокого уровня::. 
  
Не разрешать открытие свойства с помощью интерфейса **IStream** , если он был создан с помощью **IStorage**. И наоборот, не разрешать открытие свойства с помощью интерфейса **IStorage** , если он был создан с помощью **IStream**. 
  
С одним исключением можно использовать интерфейс **помощью istreamdocfile** для потоковой передачи объекта хранилища из одного контейнера в другой, но идентификатор интерфейса иид_истреамдокфиле должен быть передан в методе **опенпроперти** _лпинтерфаце _параметр.
