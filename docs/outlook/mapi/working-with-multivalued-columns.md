# <a name="working-with-multivalued-columns"></a>Работа с многоценными столбцами

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Многоценный столбец содержит данные многоценного свойства, которое имеет массив значений базового типа, а не одно значение. Поскольку ни одна из таблиц не содержит многоценных свойств в своих столбцах по умолчанию, многоценные свойства включаются в таблицу только в том случае, если пользователь таблицы запрашивает ее. 
  
Многоценные столбцы могут отображаться в таблицах:
  
- В одной строке все значения свойств отображаются в экземпляре одного столбца. Этот параметр используется по умолчанию.
    
    - Или -
    
- В ряде строк с одной строкой для каждого из значений свойства. Каждое уникальное значение отображается в столбце в своей строке с таким количеством строк, как в многоценном свойстве. Каждая строка имеет уникальное значение **для свойства PR_INSTANCE_KEY** [(PidTagInstanceKey),](pidtaginstancekey-canonical-property.md)но одинаковые значения для других столбцов. Если строка содержит несколько столбцов с несколькими значениями, например, два столбца со значениями  _M_ и  _N_ соответственно, в таблице отображаются экземпляры  _M \* N_ строки. 
    
Пользователь таблицы запрашивает тип отображения nondefault, назвав метод [IMAPITable::SetColumns](imapitable-setcolumns.md) с флагом MVI_FLAG в типе свойства многоценного столбца. Флаг MVI_FLAG является константой, определяемой в результате объединения флагов MV_FLAG и MV_INSTANCE с логической операцией **OR.** Помимо использования в **SetColumns,** MVI_FLAG также можно передать [IMAPITable::SortTable](imapitable-sorttable.md) в _параметре lpSortCriteria_ и [IMAPITable::Restrict](imapitable-restrict.md) в **члене ulPropTag** _параметра lpRestriction._ После MVI_FLAG, **SortTable** выполняет аналогично **SetColumns,** добавляя по одной строке для каждого значения в многоценном столбце и сортируя по одним значениям в экземплярах. Для каждого значения добавляется одна строка. 
  
 **Ограничение,** однако, не расширяет многоценный столбец на дополнительные вычислительные строки. Многоценный столбец с набором MVI_FLAG предписывает поставщику услуг использовать этот столбец при ограничении таблицы. Если в ограничении есть значение свойства, оно должно быть одним тегом свойства, идентичным тому, которое будет возвращено [IMAPITable::QueryRows](imapitable-queryrows.md) для столбца. 
  
Для поддержки типа отображения по умолчанию требуются только те, кто MAPI_E_TOO_COMPLEX, если вызываемая запрашивает другую альтернативу. Возможность поддержки обоих типов отображения является наиболее важной для поставщиков магазинов сообщений, реализующих таблицы содержимого папок.
