# <a name="pidtagsentrepresentingentryid-canonical-property"></a>Каноническое свойство PidTagSentRepresentingEntryId

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Содержит идентификатор записи для пользователя обмена сообщениями, представленного отправищиком.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_SENT_REPRESENTING_ENTRYID  <br/> |
|Идентификатор:  <br/> |0x0041  <br/> |
|Тип данных:  <br/> |PT_BINARY  <br/> |
|Область:  <br/> |Address  <br/> |
   
## <a name="remarks"></a>Примечания

Это свойство является одним из свойств адреса для пользователя обмена сообщениями, представленного отправищиком. Когда клиентская приложение отправляет сообщение от имени другого клиента, оно должно установить для всех представленных свойств отправитель значения для этого клиента. Пользователь обмена сообщениями, отправляющий сообщения от своего имени, обычно оставляет представленные свойства отправитель ненамеренными.
  
Отправляющий клиент всегда должен оставить это свойство без изменений. Если он не задан, поставщик транспорта должен установить для него PR_SENDER_ENTRYID **(** [PidTagSenderEntryId)](pidtagsenderentryid-canonical-property.md)в исходящие копии сообщения и оставить его ненастроенным в локальной копии.
