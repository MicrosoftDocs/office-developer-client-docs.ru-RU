# <a name="download-table-state"></a>Состояние таблицы загрузки

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время состояния таблицы загрузки машины состояния репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояния:  <br/> |**LR_SYNC_DOWNLOAD_TABLE** <br/> |
|Связанная структура данных:  <br/> |**[DNTBL](dntbl.md)** <br/> |
|Из этого состояния:  <br/> |[Синхронизация состояния содержимого](synchronize-contents-state.md) <br/> |
|Для этого состояния:  <br/> |Синхронизация состояния содержимого  <br/> |
   
> [!NOTE]
> Машина состояния репликации — это детерминистический автомат состояния. Клиент, отправляющийся из одного состояния в другое, должен в конечном итоге вернуться в первое из последнего. 
  
## <a name="description"></a>Описание

Это состояние инициирует загрузку папки. В этом состоянии Outlook инициализирует связанную структуру **данных DNTBL** с информацией о папке. Клиент загружает содержимое папки и обновляет папку в локальном магазине с помощью нового содержимого, изменений или удалений с сервера. Процесс загрузки принимает microsoft Exchange синхронизации изменений (ICS). Дополнительные сведения об ICS см. в статье [Критерии оценки ICS](https://msdn.microsoft.com/library/aa579252%28EXCHG.80%29.aspx).
  
После окончания этого состояния локальный магазин возвращается в состояние синхронизации содержимого.
