# <a name="download-table-state"></a>Состояние загрузки таблицы

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время загрузки состояния таблицы для конечного автомата репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояния:  <br/> |**LR_SYNC_DOWNLOAD_TABLE** <br/> |
|Связанная структура данных:  <br/> |**[DNTBL](dntbl.md)** <br/> |
|Из этого состояния:  <br/> |[Состояние синхронизации контента](synchronize-contents-state.md) <br/> |
|В это состояние:  <br/> |Состояние синхронизации контента  <br/> |
   
> [!NOTE]
> Конечный автомат репликации — это детерминированный конечный автомат. Клиент, который отменяет от одного состояния к другому, должен, в конечном итоге, вернуться к предыдущему из последнего. 
  
## <a name="description"></a>Описание

Это состояние инициирует загрузку папки. В этом состоянии Outlook инициализирует связанную структуру данных **днтбл** информацией о папке. Клиент загружает содержимое папки и обновляет папку в локальном хранилище, используя новые содержимое, изменения или удаления с сервера. В процессе загрузки выполняется добавочная синхронизация изменений Microsoft Exchange (ICS). Дополнительные сведения об ICS см. в статье [Критерии оценки ICS](https://msdn.microsoft.com/library/aa579252%28EXCHG.80%29.aspx).
  
При завершении этого состояния локальное хранилище возвращается в состояние синхронизация содержимого.
