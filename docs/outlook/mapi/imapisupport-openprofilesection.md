# <a name="imapisupportopenprofilesection"></a>IMAPISupport::OpenProfileSection

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Открывает раздел текущего профиля и возвращает указатель [ипрофсект](iprofsectimapiprop.md) для получения дальнейших прав. 
  
```cpp
HRESULT OpenProfileSection(
LPMAPIUID lpUid,
ULONG ulFlags,
LPPROFSECT FAR * lppProfileObj
);
```

## <a name="parameters"></a>Параметры

 _Лпуид_
  
> возврата Указатель на структуру [мапиуид](mapiuid.md) , которая определяет раздел профиля, который требуется открыть. При передаче значения NULL для параметра _лпуид_ открывается раздел профиля вызывающего абонента. 
    
 _ulFlags_
  
> возврата Битовая маска флагов, которые управляют способом открытия раздела профиля. Можно задать следующие флаги:
    
МАПИ_ДЕФЕРРЕД_ЕРРОРС 
  
> Разрешает успешное возвращение **опенпрофилесектион** , возможно, перед тем, как раздел профиля будет полностью доступен для вызывающего абонента. Если раздел профиля недоступен, то выполнение следующего вызова объекта может привести к ошибке. 
    
МАПИ_МОДИФИ 
  
> ЗаПрашивает разрешение на чтение и запись. По умолчанию объекты открываются как доступные только для чтения, и абоненты не должны работать в предположении, что предоставлено разрешение на чтение и запись. 
    
 _Лпппрофилеобж_
  
> вышли Указатель на указатель на открытый раздел профиля.
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Раздел профиля успешно открыт.
    
МАПИ_Е_НО_АКЦЕСС 
  
> Предпринята попытка изменить раздел профиля, предназначенный только для чтения, или доступ к объекту, для которого вызывающая сторона не имеет достаточных разрешений.
    
МАПИ_Е_НОТ_ФАУНД 
  
> Отсутствует раздел профиля, связанный с идентификатором записи, переданным в _лпентрид_.
    
МАПИ_Е_УНКНОВН_ФЛАГС 
  
> Используются заРезервированные или неподдерживаемые флаги, поэтому операция не выполнена.
    
## <a name="remarks"></a>Примечания

Метод **имаписуппорт:: опенпрофилесектион** реализован для всех объектов поддержки. Поставщики услуг и службы сообщений вызывают **опенпрофилесектион** , чтобы открыть раздел профиля и получить указатель на его реализацию интерфейса **ипрофсект** . 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

 **Опенпрофилесектион** открывает разделы профиля только для чтения, если не установлен флаг мапи_модифи в параметре _ulFlags_ , и у вас недостаточно разрешений. Установка этого флага не гарантирует разрешения на чтение и запись; предоставленные разрешения зависят от уровня доступа и объекта. 
  
Если **опенпрофилесектион** пытается открыть несуществующий раздел профиля как доступное только для чтения, он возвращает мапи_е_нот_фаунд. Если **опенпрофилесектион** пытается открыть несуществующий раздел профиля как доступный для чтения и записи, он создает раздел профиля и возвращает указатель **ипрофсект** .
