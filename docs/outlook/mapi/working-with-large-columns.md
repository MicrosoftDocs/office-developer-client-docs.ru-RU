# <a name="working-with-large-columns"></a>Работа с большими столбцами

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Столбцы со строками или двоичными данными свойств могут быть большими, возможно, длиной в несколько тысяч bytes. Поскольку включить один или несколько столбцов с сотнями bytes в представлении часто нецелесообразно, MAPI позволяет реализации таблицы усечено значение, чаще всего до 255 bytes и реже до 510 bytes. По возможности, реализутели таблицы должны включать полное значение свойства в столбец таблицы. Рекомендуется включить только первые 255 bytes.
  
Клиенты не могут заранее узнать, является ли таблица, используемая ими, усеченной большими столбцами. Они должны предполагать, что столбец представляет усеченное свойство, если длина столбца составляет 255 или 510 bytes. При необходимости клиенты могут напрямую получить полное значение усеченного столбца из объекта, позвонив по методу [IMAPIProp::GetProps](imapiprop-getprops.md) объекта. 
  
Клиенты, строив ограничения с большими свойствами, должны иметь в виду, что до того, как работают эти ограничения, должен знать только реализации таблицы. Некоторые реализутели таблиц позволяют ограничениям, которые построены с усеченным столбцом, основываться на усеченном размере, в то время как другие основывают его на всем значении.
