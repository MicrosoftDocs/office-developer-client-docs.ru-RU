# <a name="handing-address-book-notification"></a>Обработка уведомления об адресной книге
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Уведомления адресной книги позволяют клиенту узнать о событиях, происходящих с любой записью адресной книги или определенной записью. Вы можете зарегистрировать эти уведомления через адресную книгу MAPI, вызывая [IAddrBook::Advise](iaddrbook-advise.md) или с помощью иерархии контейнера адресной книги или таблицы содержимого, вызывая [IMAPITable::Advise](imapitable-advise.md). 
  
Укажите идентификатор записи контейнера адресной книги, списка рассылки или пользователя системы обмена сообщениями, если вы регистрируете уведомления по определенной записи и NULL при регистрации уведомлений во всей адресной книге. Идентификатор записи должен представлять пользователя обмена сообщениями или список рассылки в контейнере адресной книги. **IAddrBook::Advise** проверяет этот идентификатор записи, чтобы определить, какой поставщик адресной книги отвечает за соответствующий объект, и перенаадует вызов [методу IABLogon::Advise](iablogon-advise.md) соответствующего поставщика адресной книги. 
  
Клиенты могут регистрировать события следующих типов в записях адресной книги:
  
- Критическая ошибка
    
- Любые события объекта (созданные, измененные, удаленные, перемещенные или скопированные)
    
- Таблица изменена
    
Обычно регистрация происходит только для содержимого контейнера адресной книги и таблиц иерархии. Клиенты редко регистрируются в объектах пользователей сообщений нижнего уровня и списков рассылки. Это потому, что:
  
- Многие поставщики адресных книг не поддерживают уведомления для своих пользователей сообщений и списков рассылки.
    
- Уведомлений в таблице достаточно для отслеживания изменений и их отчетности пользователям.
