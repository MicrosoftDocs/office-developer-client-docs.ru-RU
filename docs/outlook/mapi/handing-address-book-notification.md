# <a name="handing-address-book-notification"></a>Передачей уведомлений адресной книги
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
Уведомления о адресной книги Разрешить клиенту для получения событий, происходящих для любой записи адресной книги или записи. Для этих уведомлений через адресной книги MAPI, вызвав [IAddrBook::Advise](iaddrbook-advise.md) или с помощью иерархии контейнер адресной книги или таблицу содержимого можно регистрировать путем вызова [IMAPITable::Advise](imapitable-advise.md). 
  
Укажите идентификатор записи контейнер адресной книги, список рассылки или обмена мгновенными сообщениями пользователя при регистрации уведомлений на конкретной записи и NULL при регистрации уведомлений на всей адресной книги. Идентификатор записи должен представлять обмена сообщениями пользователя или список рассылки в контейнер адресной книги. **IAddrBook::Advise** проверяет этот идентификатор записи, чтобы определить, какие адресной книги поставщика несет ответственность за соответствующего объекта и перенаправляет вызов метода [IABLogon::Advise](iablogon-advise.md) поставщик соответствующие адресной книги. 
  
Клиенты могут зарегистрировать для следующих типов событий в адресной книги:
  
- Критическая ошибка
    
- Какие-либо события объекта (создан, изменения, удаления, перемещения или копирования)
    
- Изменения в таблице
    
Как правило Регистрация происходит только для содержимого контейнера адресной книги и таблиц иерархии. Очень редко, что клиенты регистрируются с низким уровнем системы обмена сообщениями объекты списка пользователей и распространения. Это, так как:
  
- Многие поставщиками адресной книги не поддерживают уведомлений на их сообщениями пользователей и списки рассылки.
    
- Уведомления о таблице являются достаточными для отслеживания изменений и сообщения о них для пользователей.
