# <a name="hrdecomposeeid"></a>HrDecomposeEID

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Отделяет составной идентификатор записи объекта, обычно это сообщение в хранилище сообщений, в идентификатор записи этого объекта в хранилище и идентификатор записи хранилища.
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Мапиутил. h  <br/> |
|Реализовано в:  <br/> |MAPI  <br/> |
|Вызывающая сторона:  <br/> |Клиентские приложения  <br/> |
   
```cpp
HrDecomposeEID(
  LPMAPISESSION psession,
  ULONG cbEID,
  LPENTRYID pEID,
  ULONG FAR * pcbStoreEID,
  LPENTRYID FAR * ppStoreEID,
  ULONG FAR * pcbMsgEID,
  LPENTRYID FAR * ppMsgEID
);
```

## <a name="parameters"></a>Параметры

 _псессион_
  
> возврата Указатель на сеанс, используемый клиентским приложением. 
    
 _Кбеид_
  
> возврата Размер (в байтах) составного идентификатора операции, который должен быть отделен. 
    
 _Пеид_
  
> возврата Указатель на идентификатор составной записи, который необходимо отделить. 
    
 _Пкбсториид_
  
> вышли Указатель на возвращенный размер (в байтах) идентификатора записи для хранилища сообщений, содержащего объект. Если параметр _пеид_ указывает на несоставный идентификатор записи, параметр _пкбсториид_ указывает на нулевое значение. 
    
 _Ппсториид_
  
> вышли Указатель на указатель возвращаемого идентификатора записи хранилища сообщений, содержащего объект. Если параметр _пеид_ указывает на несоставный идентификатор записи, в параметре _ППСТОРИИД_ возвращается значение null. 
    
 _Пкбмсжеид_
  
> вышли Указатель на возвращенный размер (в байтах) идентификатора записи объекта. Если параметр _пеид_ указывает на несоставный идентификатор записи, параметр _пкбмсжеид_ равен значению параметра _кбеид_ . 
    
 _Ппмсжеид_
  
> вышли Указатель на указатель на возвращенный идентификатор записи объекта. Если параметр _пеид_ указывает на несоставной идентификатор записи, _ппмсжеид_ указывает на указатель на копию несоставного идентификатора записи. 
    
## <a name="return-value"></a>Возвращаемое значение

Нет.
  
## <a name="remarks"></a>Примечания

Если идентификатор, указанный в параметре _пеид_ , является составным, он делится на идентификатор записи объекта в его банке сообщений и идентификатор записи хранилища. Строки несоставных идентификаторов записей просто копируются. Составной идентификатор, который должен быть отделен, обычно создается функцией [хркомпосиид](hrcomposeeid.md) . 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

При успешном завершении этой функции будет выпущена память, содержащая параметр _пеид_ . Вызываемая реализация отвечает за освобождение памяти для выходных параметров.
