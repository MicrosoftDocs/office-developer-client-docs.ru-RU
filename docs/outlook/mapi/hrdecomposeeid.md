# <a name="hrdecomposeeid"></a>HrDecomposeEID

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Отделяет идентификатор сложной записи объекта, обычно сообщения в хранилище сообщений, в идентификатор входа этого объекта в магазине и идентификатор входа в хранилище.
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapiutil.h  <br/> |
|Реализовано в:  <br/> |MAPI  <br/> |
|Вызывающая сторона:  <br/> |Клиентские приложения  <br/> |
   
```cpp
HrDecomposeEID(
  LPMAPISESSION psession,
  ULONG cbEID,
  LPENTRYID pEID,
  ULONG FAR * pcbStoreEID,
  LPENTRYID FAR * ppStoreEID,
  ULONG FAR * pcbMsgEID,
  LPENTRYID FAR * ppMsgEID
);
```

## <a name="parameters"></a>Parameters

 _psession_
  
> [in] Указатель на сеанс, который используется клиентом приложения. 
    
 _cbEID_
  
> [in] Размер в bytes идентификатора сложной записи, который необходимо разделить. 
    
 _pEID_
  
> [in] Указатель на идентификатор сложной записи, который необходимо разделить. 
    
 _pcbStoreEID_
  
> [вышел] Указатель на возвращенный размер, в bytes, идентификатор входа в хранилище сообщений, который содержит объект. Если параметр  _pEID_ указывает на идентификатор некомпонентной записи, то параметр  _pcbStoreEID_ указывает на значение нуля. 
    
 _ppStoreEID_
  
> [вышел] Указатель на указатель на возвращенный идентификатор входа в хранилище сообщений, содержащий объект. Если параметр _pEID_ указывает на идентификатор некомпонентной записи, NULL возвращается в _параметре ppStoreEID._ 
    
 _pcbMsgEID_
  
> [вышел] Указатель на возвращенный размер в bytes идентификатора входа объекта. Если параметр _pEID_ указывает на идентификатор некомпонентной записи, то параметр _pcbMsgEID_ равен значению _параметра cbEID._ 
    
 _ppMsgEID_
  
> [вышел] Указатель на указатель на возвращенный идентификатор входа объекта. Если параметр  _pEID_ указывает на идентификатор некомпонентной записи,  _ppMsgEID_ указывает на указатель на копию идентификатора некомпонентной записи. 
    
## <a name="return-value"></a>Возвращаемое значение

Нет.
  
## <a name="remarks"></a>Примечания

Если идентификатор, указанный параметром  _pEID,_ является составным, он делится на идентификатор входа объекта в хранилище сообщений и идентификатор входа магазина. Строки идентификатора некомпонентной записи просто копируется. Идентификатор соединения, который необходимо разделить, обычно создается функцией [HrComposeEID.](hrcomposeeid.md) 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

Память, которая содержит параметр  _pEID,_ будет выпущена после успешного завершения этой функции. Реализация вызовов отвечает за то, чтобы освободить память для параметров вывода.
