# <a name="mapiinit0"></a>MAPIINIT_0

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Передает параметры функции [мапиинитиализе](mapiinitialize.md) . 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |МАПИКС. Высоты  <br/> |
   
```cpp
typedef struct
{
  ULONG ulVersion;
  ULONG ulFlags;
} MAPIINIT_0, FAR *LPMAPIINIT_0;

```

## <a name="members"></a>Members

 **Улверсион**
  
> Целое значение, представляющее номер версии структуры **MAPIINIT_0** . Элемент **улверсион** предназначен для будущего расширения и не представляет версию интерфейса MAPI. В настоящее время для **улверсион** должно быть задано значение мапи_инит_версион. 
    
 **ulFlags**
  
> Битовая маска флагов, используемых для управления инициализацией сеанса MAPI. Можно задать следующие флаги:
    
МАПИ_МУЛТИСРЕАД_НОТИФИКАТИОНС 
  
> MAPI должен создавать уведомления с помощью потока, предназначенного для обработки уведомлений, а не для первого потока, используемого для вызова **мапиинитиализе**.
    
МАПИ_НТ_СЕРВИЦЕ 
  
> Вызывающий абонент работает как служба Windows. Вызывающие абоненты, которые не работают как служба Windows, не должны устанавливать этот флаг; для абонентов, работающих в качестве службы, необходимо установить этот флаг.
    
МАПИ_НО_КОИНИТ
  
> Установите флаг МАПИ_НО_КОИНТ таким образом, чтобы **мапиинитиализе** не пытался инициализировать COM при вызове функции [CoInitialize](https://msdn.microsoft.com/library/0f171cf4-87b9-43a6-97f2-80ed344fe376%28Office.15%29.aspx). Если структура **MAPIINIT_0** передается в **мапиинитиализе** с параметром _ulFlags_ , равным мапи_но_коинит, то MAPI предполагает, что модель COM уже инициализирована и не будет обходить вызов **CoInitialize**.
    
## <a name="remarks"></a>Примечания

Многопотоковые клиенты должны устанавливать флаг МАПИ_МУЛТИСРЕАД_НОТИФИКАТИОНС. Если флаг не установлен, уведомления создаются в потоке, используемом для совершения первого вызова **мапиинитиализе**. 
  
Дополнительные сведения о том, когда следует устанавливать этот флаг и как реализовать потокобезопасность в клиенте, см [в разделе Threading in MAPI](threading-in-mapi.md).
