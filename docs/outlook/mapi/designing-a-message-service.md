# <a name="designing-a-message-service"></a>Проектирование службы сообщений

**Относится к**: Outlook 2013 | Outlook 2016 
  
Прежде чем приступить к написанию кода для поддержки службы сообщений, важно создать проект. У решения следующих проблем в процессе разработки:
  
1. Определите, сколько поставщиков услуг должно быть включено в службу сообщений. Включите в службу только связанных поставщиков услуг (то есть поставщиков, которые работают с одной системой обмена сообщениями). Несвязанные поставщики служб не принадлежат к одной службе сообщений. Используйте профиль для интеграции несвязанных поставщиков услуг и служб сообщений.
    
2. Определите, какой тип поставщиков услуг должен быть включен в службу сообщений. В большинстве службы единой службы содержится по одному поставщику каждого из общих типов. То есть у обычной службы сообщений есть один поставщик адресной книги, один поставщик store сообщений и один поставщик транспорта.
    
3. Определите, сколько DLL должно содержать службу сообщений. Количество DLL, которые использует служба сообщений, зависит от следующих большого количества:
    
   - Степень сложности, которую может обработать автор службы сообщений.
    
   - Тип поставщиков услуг в службе сообщений.
    
   - Связь, которую служба сообщений может иметь с другой службой сообщений.
    
   Так как MAPI хранит только одну точку входа для каждого типа поставщика, не включаем несколько поставщиков одного типа в одну DLL. Если имеет смысл включить несколько поставщиков одного типа, либо реализуйте их в отдельных DLL, либо обесполняйте функцию точки входа. Другой вариант — реализовать связанные службы сообщений или службы сообщений, которые могут использовать один и тот же код установки и конфигурации и ту же функцию точки входа DLL в одной DLL.
    
   По возможности сохраняйте простоте и используйте одну DLL-технологию, которая содержит реализацию всех поставщиков услуг в службе сообщений, а также весь код для установки и настройки службы сообщений. Если это невозможно, можно реализовать одну DLL для кода установки и конфигурации и одну DLL для всех поставщиков услуг или одну DLL для каждого поставщика.
    
4. Определите имя DLL или DLL службы сообщений.
