# <a name="download-message-header-state"></a>Скачивание состояния загона сообщений

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время состояния заголовок сообщения загрузки машины состояния репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояния:  <br/> |**LR_SYNC_DOWNLOAD_HEADER** <br/> |
|Связанная структура данных:  <br/> |**[HDRSYNC](hdrsync.md)** <br/> |
|Из этого состояния:  <br/> |[Состояние бездействия](idle-state.md) <br/> |
|Для этого состояния:  <br/> |Состояние бездействия  <br/> |
   
> [!NOTE]
> Машина состояния репликации — это детерминистический автомат состояния. Клиент, отправляющийся из одного состояния в другое, должен в конечном итоге вернуться в первое из последнего. 
  
## <a name="description"></a>Описание

В этом состоянии клиент обновляет заготку сообщения в локальном магазине. Локальный магазин вводит это состояние при **[IOSTX::SyncHdrBeg](iostx-synchdrbeg.md)** и выходит, когда **[называется IOSTX::SyncHdrEnd.](iostx-synchdrend.md)** В этом состоянии Outlook инициализирует членов связанной структуры **данных HDRSYNC** с информацией о загоне сообщения. Сначала клиент загружает полный элемент сообщения с сервера, а затем обновляет заготку элемента сообщения локально. 
  
Когда синхронизация заканчивается, клиент задает результаты скачивания. Локальный магазин возвращается в состояние простоя.
