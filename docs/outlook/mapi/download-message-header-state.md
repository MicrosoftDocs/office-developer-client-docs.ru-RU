# <a name="download-message-header-state"></a>Скачивание состояния загона сообщения

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время состояния заголовок сообщения загрузки конечного автомата репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояния:  <br/> |**LR_SYNC_DOWNLOAD_HEADER** <br/> |
|Связанная структура данных:  <br/> |**[HDRSYNC](hdrsync.md)** <br/> |
|Из этого состояния:  <br/> |[Состояние бездействия](idle-state.md) <br/> |
|В этом состоянии:  <br/> |Состояние бездействия  <br/> |
   
> [!NOTE]
> Этот автомат репликации является детерминируемым. Клиент, выступая из одного состояния в другое, должен в конечном итоге вернуться к одному из последних. 
  
## <a name="description"></a>Описание

В этом состоянии клиент обновляет заглавную часть сообщения в локальном хранилище. Локальное хранилище вступает в это состояние после **[IOSTX::SyncHdrBeg](iostx-synchdrbeg.md)** и выходит из него, когда **[вызван IOSTX::SyncHdrEnd.](iostx-synchdrend.md)** В этом состоянии Outlook инициализирует члены связанной структуры данных **HDRSYNC** с информацией о загоне сообщения. Клиент сначала скачивает полный элемент сообщения с сервера, а затем локально обновляет замещение элемента сообщения. 
  
Когда синхронизация завершается, клиент задает результаты скачивания. Локальное хранилище возвращается в состояние простоя.
