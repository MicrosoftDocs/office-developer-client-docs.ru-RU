# <a name="download-message-header-state"></a>Состояние скачивания заголовка сообщения

  
  
**Относится к**: Outlook 
  
 В этом разделе описываются двоичных файлов состояния заголовок сообщения загрузки конечного автомата репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояний:  <br/> |**LR_SYNC_DOWNLOAD_HEADER** <br/> |
|Структура связанных данных:  <br/> |**[HDRSYNC](hdrsync.md)** <br/> |
|Из этого состояния:  <br/> |[Состояние простоя](idle-state.md) <br/> |
|Это состояние:  <br/> |Состояние простоя  <br/> |
   
> [!NOTE]
> Конечный автомат репликации является детерминированным конечного автомата. Клиент отправления одного состояния в другое должны в конечном счете возврата на первый из последний. 
  
## <a name="description"></a>Описание

В этом состоянии клиента обновляет заголовка сообщения для локального хранилища. Локальное хранилище в этот режим на **[IOSTX::SyncHdrBeg](iostx-synchdrbeg.md)** и завершает работу при вызове **[IOSTX::SyncHdrEnd](iostx-synchdrend.md)** . В этом состоянии Outlook инициализирует элементы связанной структуры данных **HDRSYNC** со сведениями о заголовке сообщения. Клиент сначала загружает элемент полного сообщения с сервера и обновляет заголовка сообщения локально. 
  
По окончании о синхронизации клиента задает файл для загрузки результатов. Локальное хранилище возвращается в состояние простоя.
