# <a name="using-a-table-to-work-with-properties"></a>Использование таблицы для работы со свойствами

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Многие свойства доступны как из объектов, которые их поддерживают, так и в виде столбцов в таблицах. Когда это возможно, извлеките эти свойства через таблицу.
  
Call [IMAPITable:: метода SetColumns](imapitable-setcolumns.md) , чтобы включить все свойства, необходимые для клиента, и [IMAPITable:: QueryRows](imapitable-queryrows.md) , чтобы получить все строки таблицы. 
  
Эти два вызова обычно достаточно подходят для получения достаточной информации для отображения пользователю и, как правило, достаточно для любой необходимой внутренней обработки, делая вызов **OpenEntry** для открытия объекта ненужным. 
  
Существует только два исключения.
  
- , Если свойство превышает 255 байт. Интерфейс * * IMAPITable * * может не возвращать значение свойства целиком, а отбрасывает его в 255 байт. Тем не менее, подумайте об этом компромиссе. Если вы отображаете эти данные для пользователя, 255 байт могут быть достаточно для текстового поля, например комментария. 
    
- Если требуется определенное свойство из одной строки в таблице. В этом случае нет необходимости создавать таблицу со свойствами, которые никогда не будут использоваться. В большинстве случаев для всех строк потребуется одинаковое свойство.
