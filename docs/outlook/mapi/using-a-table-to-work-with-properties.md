# <a name="using-a-table-to-work-with-properties"></a>Использование таблицы для работы со свойствами

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Многие свойства доступны как из объектов, которые их поддерживают, так и в качестве столбцов в таблицах. По возможности извлекать эти свойства через таблицу.
  
Вызовите [IMAPITable::SetColumns,](imapitable-setcolumns.md) чтобы включить все свойства, необходимые клиенту, и [IMAPITable::QueryRows](imapitable-queryrows.md) для получения всех строк таблицы. 
  
Обычно этих двух вызовов достаточно для получения достаточной информации для отображения пользователю и часто достаточно для любой внутренней обработки, что делает вызов **OpenEntry** ненужным, чтобы открыть объект. 
  
Существует только два исключения:
  
- Если свойство больше 255 bytes. The ** IMAPITable ** interface might not return the entire property value, instead truncating it at 255 bytes. Однако подумайте об этом компромиссе. Если вы отображаете эти данные для пользователя, для текстового поля, такого как комментарий, может быть достаточно 255 bytes. 
    
- Если требуется определенное свойство из одной строки таблицы. В этом случае нет необходимости создавать таблицу со свойствами, которые никогда не будут использоваться. В большинстве раз для всех строк требуются одинаковые свойства.
