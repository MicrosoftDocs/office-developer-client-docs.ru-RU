# <a name="using-a-table-to-work-with-properties"></a>Использование таблицы для работы с свойствами

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Многие свойства доступны как из поддерживаемых объектов, так и в качестве столбцов на столах. По возможности извлекай эти свойства через таблицу.
  
Вызов [IMAPITable::SetColumns,](imapitable-setcolumns.md) чтобы включить все свойства, необходимые клиенту, и [IMAPITable::QueryRows,](imapitable-queryrows.md) чтобы получить все строки таблицы. 
  
Эти два вызова обычно достаточны для получения достаточного количества сведений для отображения пользователю и часто являются достаточными для любой необходимой внутренней обработки, что делает вызов **в OpenEntry,** чтобы открыть объект ненужным. 
  
Существует только два исключения:
  
- Если свойство более 255 bytes. Интерфейс IMAPITable ** может не возвращать все значение свойства, а усеченный на 255 bytes. Однако подумайте об этом компромиссе. Если вы отображаете эти данные пользователю, для текстового поля, например комментария, может хватить 255 битов. 
    
- Если вам нужно определенное свойство из одной строки в таблице. В этом случае нет необходимости создавать таблицу со свойствами, которые никогда не будут использоваться. В большинстве раз вам потребуется одно и то же свойства для всех строк.
