# <a name="form-configuration-file-platforms-section"></a>Раздел Конфигурация формы [Платформы]

**Область применения**: Outlook 2013 | Outlook 2016 
  
В **разделе [Платформы]** перечислены полный набор платформ, поддерживаемых этой формой. Каждая запись платформы состоит из платформы **префикса.** _строка,_  _в которой строка_ является произвольным кодом строки для платформы. Каждая строка соответствует записи **ЦП** отдельных **разделов [Платформы].** Каждая запись в **разделе [Платформы]** определяет строку  _платформы,_ которая ссылается на **последующее [Platform).** _раздел строки_ **платформы]** как показано здесь. 
  
В **разделе [Платформы]** перечислены полный набор платформ, поддерживаемых этой формой. Каждая запись платформы состоит из платформы **префикса.** _строка,_  _в которой строка_ является произвольным кодом строки для платформы. Каждая строка соответствует записи **ЦП** отдельных **разделов [Платформы].** Каждая запись в **разделе [Платформы]** определяет строку  _платформы,_ которая ссылается на **последующее [Platform).** _раздел строки_ **платформы]** как показано здесь. 
  
**[Платформы]**
  
**Платформа**. _строка_  =   _строка платформы_
  
Ниже приводится пример **раздела [Платформы].** 
  
```cpp
[Platforms]
Platform.1 = NTx86
Platform.2 = Win95

```

Каждая **[Платформа.** _Раздел строки_ **платформы ]** содержит две необходимые записи, **ЦП** и **OSVersion.** Запись **ЦП** указывает процессор, а **запись OSVersion** — операционную систему. **Допустимые значения** ЦП описаны в следующей таблице. 
  
|**Запись ЦП**|**Процессор**|
|:-----|:-----|
|Ix86  <br/> |Процессоры серии Intel 80x86 и Pentium, а также эквивалентные процессоры от amD, Cyrix, NextGen и других производителей.  <br/> |
|MIPS  <br/> |Процессоры серии MIPS R4000.  <br/> |
|AXP  <br/> |Процессор Alpha AXP корпорации цифрового оборудования.  <br/> |
|PPC  <br/> |Процессоры серии PC Power Motorola.  <br/> |
|M68  <br/> |Процессоры серии Mororola 68x00.  <br/> |
   
Допустимые значения **OSVersion** описаны в следующей таблице. 
  
|**Запись OSVersion**|**Операционная система**|
|:-----|:-----|
|Win3.1  <br/> |Windows 3.1 и Windows для workgroups 3.11.  <br/> |
|WinNT3.5  <br/> |Windows NT 3,5 или ниже.  <br/> |
|Win95  <br/> |Windows 95.  <br/> |
|WinNT4.0  <br/> |Windows NT 4.0.  <br/> |
|Mac7  <br/> |Macintosh System 7.  <br/> |
   
Кроме того, **[Платформа.** _Раздел строки_ **платформы ]** должен содержать запись **File** или **LinkTo.** В **записи** File перечислены исполняемый файл приложения-сервера форм, который библиотека форм поддерживает и загружает в новый подтекстом в кэше диска при запуске формы. Если вместо этого используется запись **LinkTo,** она содержит имя другой строки платформы, из которой принимаются сведения **о файле.** Это полезно, если одна версия формы поддерживает несколько платформ. 
  
Запись **Реестра** используется всякий раз, когда используется запись **File,** она определяет ключ реестра для библиотеки форм, где хранится исполняемый файл для приложения-сервера формы. Строки, предшествовавшие ответной строке ( \ ) размещаются в корне реестра. Строки, не предшествовавшие ответной строке, помещаются в ключ  _guid \registry HKEY_CLASSES_ROOT\CLSID\ GUID,_ где  _GUID_ является **GUID** формы. Символы "%d" можно использовать для указать имя пути каталога, из которого был прочитаен файл конфигурации формы. Это полезно для указания других файлов с именами по отношению к файлу конфигурации формы. **Несколько записей** **файла** или реестра можно указать с помощью файла или реестра в качестве префикса, за которым следует любой другой текст. Формат для **[Платформы.** _Раздел строки_ **платформы ]** является: 
  
- **[Платформа.** _строка платформы_ **]**
    
- **ЦП**  =   _строка_
    
- **OSVersion**  =   _строка_
    
- **Файл**  =   _путь_
    
- **LinkTo**  =   _строка_
    
- **Реестр**  =   _строка_
  
Ниже приводится два **примера [Platform.** _строка_ **платформы ]** разделы, один с помощью **записи File** и один с помощью **записи LinkTo.** 
  
```cpp
[Platform.NTx86]
CPU = ix86
OSVersion = WinNT3.5
File = \helpdesk.exe
Registry = Local Server = %d\helpdesk.exe
[Platform.Win95]
CPU = ix86
OSVersion = Win95
LinkTo = NTx86

```

**[Платформа.**  Раздел строки **платформы** игнорируется при добавлении формы в локализованную библиотеку форм, когда предполагается, что установщик поместил файлы, вложенные обработщиком класса сообщений, в доступное локальное хранилище, как по имени в разделе обработщика в реестре OLE, и сделал регистрацию OLE в реестре системы.
