# <a name="mapi-address-types"></a>Типы адресов MAPI

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Каждый пользователь обмена сообщениями связан с типом адреса, строкой символов, описывающей формат адреса пользователя, который хранится в свойстве **PR_ADDRTYPE** ([PidTagAddressType](pidtagaddresstype-canonical-property.md)). Типы адресов сопоставляются с форматами адресов. То есть, изучив тип адреса получателя, клиентские приложения могут определять, как форматировать адрес, соответствующий получателю. 
  
Например, тип `SMTP` адреса определяет стандартный Интернет адрес: 
  
 `username@companyname.com.`
  
И тип `EX` адреса указывает адрес сервера Exchange. 
  
Все записи адресной книги должны иметь допустимый тип адреса. Клиенты должны указать тип адреса при создании типа получателя, неподдерживаемого поставщиком адресной книги. Для тех записей, которые они поддерживают, поставщики адресных книг должны предоставлять допустимые типы адресов. 
  
MAPI определяет только один тип адреса: МАПИПДЛ, который означает личный список рассылки.
  
Чтобы получить список типов адресов, поддерживаемых всеми поставщиками транспорта в сеансе, клиентские приложения вызывают метод **IMAPISession:: енумадртипес** . Дополнительные сведения см. в статье [IMAPISession:: енумадртипес](imapisession-enumadrtypes.md).
