# <a name="dtblpage"></a>DTBLPAGE

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Описывает страницу с вкладками, которая будет использоваться в диалоговом окне, созданном из таблицы отображения. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |MAPIDEFS. h  <br/> |
|Связанный макрос:  <br/> |[SizedDtblPage](sizeddtblpage.md) <br/> |
   
```cpp
typedef struct _DTBLPAGE
{
  ULONG ulbLpszLabel;
  ULONG ulFlags;
  ULONG ulbLpszComponent;
  ULONG ulContext;
} DTBLPAGE, FAR *LPDTBLPAGE;

```

## <a name="members"></a>Members

 **Улблпсзлабел**
  
> Позиция в памяти подписи строки символов для вкладки страницы.
    
 **ulFlags**
  
> Битовая маска флагов, используемых для обозначения формата метки, на который указывает элемент **улблпсзлабелнаме** . Можно задать следующий флаг: 
    
MAPI_UNICODE 
  
> Метка имеет формат Юникод. Если флаг МАПИ_УНИКОДЕ не установлен, метка имеет формат ANSI.
    
 **Улблпсзкомпонент**
  
> Позиция в памяти строки символов, указывающей раздел **[Help File Mappings]** в Mapisvc. INF-файл конфигурации или 0. Имя файла, отображаемое в MAPISVC. Раздел INF может использоваться пользователем для доступа к расширенной справке для страницы с вкладками, нажав кнопку **Справка** в диалоговом окне. Дополнительные сведения о записях в MAPISVC. INF в разделе [Формат файлов Mapisvc. INF-файл](file-format-of-mapisvc-inf.md).
    
 **Улконтекст**
  
> Уникальный идентификатор для страницы с вкладками в строке, определенной элементом **улблпсзкомпонент** . Элемент **улблпсзкомпонент** и элемент **улконтекст** должны быть ненулевыми, чтобы кнопка **справки** работала. Если этот идентификатор равен нулю, а строка компонента имеет значение NULL, Справка, связанная со страницей, отсутствует. 
    
## <a name="remarks"></a>Комментарии

Структура **дтблпаже** описывает элемент управления с вкладками, который используется для разделения нескольких связанных диалоговых окон. Как правило, эти диалоговые окна являются страницами свойств для отображения параметров конфигурации, сообщений или получателей. Если щелкнуть вкладку, пользователь может переключаться с одного листа на другой. 
  
Строка компонента и идентификатор контекста содержат сведения о том, доступна ли расширенная Справка для вкладки. Если доступна расширенная Справка, строка компонента и идентификатор контекста будут содержать сведения о том, как получить к нему доступ. Строка компонента сопоставляется с файлом справки; Идентификатор контекста сопоставляется с исходным разделом справки. Если идентификатор контекста равен нулю, а строка компонента имеет значение NULL, расширенная Справка недоступна.
  
Общие сведения о отображаемых таблицах приведены в разделе [Display Tables](display-tables.md). Сведения о том, как реализовать таблицу отображения, можно найти [в статье реализация таблицы отображения](display-table-implementation.md).
