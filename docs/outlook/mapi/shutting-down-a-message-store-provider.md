# <a name="shutting-down-a-message-store-provider"></a>Завершение работы поставщика хранилища сообщений

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Если поставщик является поставщиком хранилища сообщений, его можно завершить одним из следующих способов:
  
- Если клиент или диспетчер очереди MAPI вызывает [IMsgStore:: сторелогофф](imsgstore-storelogoff.md). При завершении работы поставщика хранилища сообщений с **сторелогофф** происходит завершение работы в определенном порядке и управляемым образом. 
    
- Когда клиент вызывает [IMAPISession:: logoff](imapisession-logoff.md). 
    
Реализация **IMsgStore:: сторелогофф** должна начинаться с вызова [Имаписуппорт:: сторелогоффтранспортс](imapisupport-storelogofftransports.md) , чтобы уведомить MAPI о завершении работы, указывая, что все связанные транспортные поставщики должны выйти из системы. Когда **IMsgStore:: сторелогофф** возвращает значение, его вызывающий объект вызывает метод [IUnknown:: Release](https://msdn.microsoft.com/library/4b494c6f-f0ee-4c35-ae45-ed956f40dc7a%28Office.15%29.aspx) хранилища сообщений. Реализуйте этот метод **освобождения** , вызвав метод **IUnknown:: Release** объекта support. 
  
MAPI выполняет следующие задачи в своей реализации интерфейса **IUnknown:: Release** для хранилищ сообщений: 
  
1. Удаляет все структуры [мапиуид](mapiuid.md) , зарегистрированные поставщиком хранилища сообщений. 
    
2. Удаляет строку поставщика хранилища сообщений из таблицы состояния.
    
3. Calls [имслогон:: logoff](imslogon-logoff.md) для освобождения всех открытых объектов, подобъектов и объектов Status. 
    
4. Вызывает метод [IUnknown:: Release](https://msdn.microsoft.com/library/4b494c6f-f0ee-4c35-ae45-ed956f40dc7a%28Office.15%29.aspx) для освобождения объекта входа поставщика хранилища сообщений. 
    
Некоторые клиенты могут опустить вызов **IMsgStore:: сторелогофф**, инициируя завершение работы поставщика хранилища сообщений с вызовом метода **IUnknown:: Release** хранилища сообщений. Завершение работы при таких обстоятельствах без вызова **сторелогофф** менее упорядочено и управляется. Запишите метод **освобождения** хранилища сообщений, чтобы обработать эту возможность, и следит за тем, был ли вызван вызов **Имаписуппорт:: сторелогоффтранспортс** . В процессе завершения работы **сторелогоффтранспортс** должен вызываться один раз. Если вы обнаружите в методе **Release** , что **сторелогоффтранспортс** еще не вызывался, вызовите его с флагом LOGOFF_ABORT.
