# <a name="mapi-status-objects"></a>Объекты состояния MAPI

  
  
**Относится к**: Outlook 
  
Объекты состояния о сведения обо всех ресурсах MAPI. Например поставщика услуг, процесс отправки и получения MAPI или в адресной книге.
  
Имеется объект состояния, предоставление сведений о каждой отдельной службы поставщика текущего профиля. MAPI несет ответственность за внедрение объектов состояния для подсистемы, процесс отправки и получения MAPI и адресной книги. Объект состояния подсистемы предоставляет глобальные данные. Объект состояния для встроенной адресной книги предоставляет состояние всех поставщиками адресной книги в настоящее время работы.
  
Каждый объект состояния включен в таблице состояния, таблицы, задаваемые с помощью интерфейса MAPI, который предоставляет клиентам все сведения о состоянии сеанса. Для получения дополнительных сведений см [Состояние](status-tables.md). Клиенты могут доступ к объекту определенное состояние, либо из таблицы или для поставщика услуг, через возвращается объект входа в систему. Например для доступа к объекту состояние поставщика адресных книг, клиент может вызвать **IABLogon::OpenStatusEntry**. Для получения дополнительных сведений см [IABLogon::OpenStatusEntry](iablogon-openstatusentry.md).
  
Клиенты могут использовать состояние объектов:
  
- Сведения о состоянии сеанса.
    
- Мониторинг поставщика услуг.
    
- Контроль передачи сообщений.
    
- Просмотр или изменение конфигурации и состоянии ресурса.
    
Каждый объект состояния реализует интерфейс **IMAPIStatus** . Дополнительные сведения можно [IMAPIStatus: IMAPIProp](imapistatusimapiprop.md). Тем не менее не каждый объект состояния полностью поддерживает все методы **IMAPIStatus** . Поскольку вариантов в методах, поддерживаемых приложением объект состояния, клиентам требуется сведения об объекте определенное состояние перед их можно использовать. Состояние объектов необходимо опубликовать сведения о компонентах для их в следующие три свойства: 
  
 **PR_RESOURCE_METHODS** ([PidTagResourceMethods](pidtagresourcemethods-canonical-property.md)) 
  
 **PR_RESOURCE_TYPE** ([PidTagResourceType](pidtagresourcetype-canonical-property.md)) 
  
 **PR_RESOURCE_FLAGS** ([PidTagResourceFlags](pidtagresourceflags-canonical-property.md)) 
  
Дополнительные сведения о реализации объекта состояния [Реализация объекта состояния](status-object-implementation.md)см. Дополнительные сведения об использовании состояние объекта можно [таблице состояния и состояния объектов](status-table-and-status-objects.md).
