# <a name="mapi-status-objects"></a>Объекты состояния MAPI

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Объекты status сообщают сведения о ресурсах MAPI. Например, поставщик услуг, процесс отправки и получения MAPI или адресная книга.
  
В текущем профиле имеется объект состояния, поставляющий сведения о каждом отдельном поставщике услуг. MAPI отвечает за реализацию объектов состояния для подсистемы, процесса отправки и получения MAPI и адресной книги. Объект состояния подсистемы поставляет глобальную информацию. Объект состояния интегрированной адресной книги обеспечивает состояние всех действующих поставщиков адресной книги.
  
Каждый объект состояния включен в таблицу состояния, таблицу, поддерживаемую MAPI, которая предоставляет клиентам всю информацию о состоянии для сеанса. Дополнительные сведения см. в [таблице Status Tables.](status-tables.md) Клиенты могут получить доступ к определенному объекту состояния либо через таблицу, либо для поставщика услуг через его объект logon. Например, чтобы получить доступ к объекту состояния поставщика адресной книги, клиент может вызвать **IABLogon::OpenStatusEntry.** Дополнительные сведения см. [в iABLogon::OpenStatusEntry](iablogon-openstatusentry.md).
  
Клиенты могут использовать объекты состояния для:
  
- Узнайте о состоянии сеанса.
    
- Мониторинг поставщика услуг.
    
- Контроль передачи сообщений.
    
- Просмотр или изменение конфигурации и состояния ресурса.
    
Каждый объект состояния реализует **интерфейс IMAPIStatus.** Дополнительные сведения см. [в iMAPIStatus: IMAPIProp](imapistatusimapiprop.md). Однако не каждый объект состояния полностью поддерживает **каждый метод IMAPIStatus.** Поскольку существуют различия в методах, поддерживаемых объектом состояния, клиентам необходимо узнать об определенном объекте состояния, прежде чем использовать его. Объекты status должны публиковать сведения о своих свойствах в следующих трех свойствах: 
  
 **PR_RESOURCE_METHODS** [(PidTagResourceMethods)](pidtagresourcemethods-canonical-property.md) 
  
 **PR_RESOURCE_TYPE** [(PidTagResourceType)](pidtagresourcetype-canonical-property.md) 
  
 **PR_RESOURCE_FLAGS** [(PidTagResourceFlags)](pidtagresourceflags-canonical-property.md) 
  
Дополнительные сведения о реализации объекта состояния см. в дополнительных [сведениях о реализации объектов status.](status-object-implementation.md) Дополнительные сведения об использовании объекта состояния см. в таблице [status Table и Status Objects.](status-table-and-status-objects.md)
