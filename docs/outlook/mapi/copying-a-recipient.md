# <a name="copying-a-recipient"></a>Копирование получателя

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Чтобы скопировать одного или несколько получателей из одного контейнера в другой или один контейнер, сначала убедитесь, что целевой контейнер можно модификабельно. Контейнеры, которые можно модификатировать, устанавливают флаг AB_MODIFIABLE в **свойстве PR_CONTAINER_FLAGS** ([PidTagContainerFlags).](pidtagcontainerflags-canonical-property.md)
  
Чтобы скопировать одну или несколько записей в измеримый контейнер, вызовите метод [IABContainer::CopyEntries](iabcontainer-copyentries.md) контейнера назначения. Поскольку копирование записей адресной книги может быть трудоемким, **CopyEntries** принимает четыре входных параметра: массив идентификаторов записей для копирования записей, окне, индикатор хода выполнения и битовуюmask флагов. 
  
Окне окне и индикатор хода выполнения используются поставщиком адресной книги для показа состояния операции пользователю. Если необходимо отобразить ход выполнения, передав окне родительского окна индикатора хода выполнения в _параметре ulUIParam_ и не устанавливая флаг AB_NO_DIALOG в параметре _ulFlags._ Если у вас есть собственная реализация индикатора хода выполнения, передайте указатель на реализацию в _параметре lpProgress._ Если нет, передав NULL. Поставщик адресной книги будет использовать реализацию индикатора хода выполнения MAPI. 
  
Битоваяmas флагов указывает, следует ли отображать индикатор хода выполнения и как следует обрабатывать повторяющиеся проверки записей. Установите флаг AB_NO_DIALOG для подавления индикатора хода выполнения. Установите флаг CREATE_CHECK_DUP_LOOSE, чтобы поставщик адресной книги свободно проверял дубликаты или флаг CREATE_CHECK_DUP_STRICT на более строгую проверку дубликатов. Установите флаг CREATE_REPLACE, чтобы скопированные записи заменяли существующие записи, когда поставщик определяет, что имеются дубликаты. 
  
При копировании в контейнер личной адресной книги поставщик копирует некоторые или все свойства для каждой записи. Так как MAPI не устанавливает требования для поставщиков, реализующих операции копирования контейнеров, нельзя делать допущения о количестве и типе свойств, скопированные с помощью записи адресной книги.
