# <a name="dtbllabel"></a>DTBLLABEL

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Описывает метку, которая будет использоваться в диалоговом окне, построенного из таблицы отображения.
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapidefs.h  <br/> |
|Связанный макрос  <br/> |[SizedDtblLabel](sizeddtbllabel.md) <br/> |
   
```cpp
typedef struct _DTBLLABEL
{
  ULONG ulbLpszLabelName;
  ULONG ulFlags;
} DTBLLABEL, FAR *LPDTBLLABEL;

```

## <a name="members"></a>"Участники"

 **ulbLpszLabelName**
  
> Положение в памяти метки строки символов.
    
 **ulFlags**
  
> Битовая метка флагов, используемая для обозначения формата метки, на который указывает член **ulbLpszLabelName.** Можно установить следующий флаг: 
    
MAPI_UNICODE 
  
> Метка имеет формат Юникод. Если флаг MAPI_UNICODE не установлен, метка будет в формате ANSI.
    
## <a name="remarks"></a>Примечания

Структура **DTBLLABEL** описывает текст управления меткой, который отображается с помощью другого типа управления, чтобы добавить значение для этого управления. Например, большинство элементов управления редактированием находятся рядом с меткой, чтобы сообщить пользователю о типе вносяемой информации. Некоторые элементы управления, такие как групповые поля и кнопки, удерживают собственные метки. 
  
Метка может включать ускоритель Windows, обозначенный как символ, следующий за амперандом ( &amp; ). Нажатие клавиши ускорителя помещает фокус на первый элемент управления без метки, который следует за этой меткой в таблице отображения.
  
Многоуровневые метки не поддерживаются. Для показа нескольких строк требуется несколько меток.
  
Метку нельзя использовать в качестве редактируемого только для чтения. Разница заключается в том, что можно выбрать и скопировать изменение, а метку — нет. 
  
Обзор таблиц отображения см. в [таблице отображения.](display-tables.md) Сведения о реализации таблицы отображения см. в таблице ["Реализация таблицы отображения".](display-table-implementation.md)
