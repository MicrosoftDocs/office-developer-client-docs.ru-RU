# <a name="pidtagsendername-canonical-property"></a>Каноническое свойство PidTagSenderName

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Содержит отображаемую имя отправитель сообщения.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_SENDER_NAME, PR_SENDER_NAME_A, PR_SENDER_NAME_W  <br/> |
|Идентификатор:  <br/> |0x0C1A  <br/> |
|Тип данных:  <br/> |PT_UNICODE, PT_STRING8  <br/> |
|Область:  <br/> |Address  <br/> |
   
## <a name="remarks"></a>Примечания

Эти свойства являются примерами свойств адреса для отправитель сообщения. Он должен быть установлен поставщиком исходячих транспортных данных, который никогда не должен распространять ранее существовавших значений.
  
Если ни один поставщик транспорта не предоставил никаких свойств адресов отправитель, то пулер MAPI пытается заполнить их, вызывая метод [IMAPISession::QueryIdentity](imapisession-queryidentity.md) для идентификатора записи. Если идентификаторы записей не указаны, то пулер MAPI сохраняет "Unknown" во всех свойствах адреса отправитель типа PT_TSTRING.
