# <a name="pidtagextendedfolderflags-canonical-property"></a>Каноническое свойство PidTagExtendedFolderFlags
 
**Относится к**: Outlook 2013 | Outlook 2016 
  
Содержит расширенные флажки для папки.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_EXTENDED_FOLDER_FLAGS  <br/> |
|Идентификатор:  <br/> |0x36DA  <br/> |
|Тип данных:  <br/> |PT_BINARY  <br/> |
|Область:  <br/> |Контейнер MAPI  <br/> |
   
## <a name="remarks"></a>Примечания

Данное свойство представляет собой двоичный поток, который содержит закодированные подсвойства для папки. Он форматируется как ряд вложенных элементов переменной длины. Первые 8 битов дочернего элемента — это поле ID, которое указывает, какой тип флага представляет подэлемент. Второй 8 бит — это количество байтов данных, которые следует выполнить.
  
Возможные значения ID:
  
- Invalid
    
   Не используйте это значение
    
- екстендедфлагс
    
   Данные представляют собой 4-байтовые значения в формате:
    
   |**Биты**|**Описание**|
   |:-----|:-----|
   |0-1  <br/> |Резервирования.  <br/> |
   |2  <br/> |Значение 0, если приложение должно отображать описание политики.  <br/> |
   |3-5  <br/> |Резервирования.  <br/> |
   |6-7  <br/> |Управляет отображением числа сообщений в папке.  <br/> 0 — используйте параметр по умолчанию  <br/> 1 — использовать количество непрочитанных сообщений  <br/> 3 — использовать общее количество сообщений  <br/> |
   |8-31  <br/> |Резервирования.  <br/> |
   
Зарезервированные элементы можно игнорировать, но существующие значения должны быть сохранены.
    
- сеарчфолдерид
    
   Поле данных является 16-байтовым полем. Когда приложение создает папку сохраняемого поиска, оно должно задать для этого поля в папке то же значение, что и для двоичного свойства **PR_WB_SF_TAG** ([PidTagSearchFolderId)](pidtagsearchfolderid-canonical-property.md)) сообщения папки поиска.
    
- тодофолдерверсион
    
   Поле данных является 4-байтовым полем. Когда приложение создает папку поиска дел, ему необходимо задать для этого поля в папке значение с прямым порядком байтов "0x000c0000".
