# <a name="opentnefstreamex"></a>OpenTnefStreamEx

**Относится к**: Outlook 2013 | Outlook 2016 
  
Создает объект Transport-Neutral формата Инкапсуляции (TNEF), который можно использовать для кодирования или декодирования объекта сообщения в поток данных TNEF для использования транспортами, шлюзами и хранилищами сообщений. Это точка входа для доступа в TNEF. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Tnef.h  <br/> |
|Реализовано в:  <br/> |MAPI  <br/> |
|Вызывающая сторона:  <br/> |Поставщики транспорта  <br/> |
   
```cpp
HRESULT OpenTnefStreamEx(
  LPVOID lpvSupport,
  LPSTREAM lpStream,
  LPSTR lpszStreamName,
  ULONG ulFlags,
  LPMESSAGE lpMessage,
  WORD wKeyVal,
  LPADDRESSBOOK lpAddressBook,
  LPITNEF FAR * lppTNEF
);
```

## <a name="parameters"></a>Параметры

_lpvSupport_
  
> [in] Передает объект поддержки или передается в NULL. Если параметр NULL,  _параметр lpAddressBook_ должен иметь ненулевую заданную. 
    
_lpStream_
  
> [in] Указатель на объект потока хранилища, например интерфейс OLE **IStream,** предоставляющий источник или назначение для сообщения потока TNEF. 
    
_lpszStreamName_
  
> [in] Указатель на имя потока данных, который использует объект TNEF. Если вызывающая группа установила флаг TNEF_ENCODE (параметр _ulFlags)_ в вызове **OpenTnefStream,** параметр  _lpszName_ должен указывать ненулевую строку, состоящую из символов, которые считаются допустимыми для именования файла. MAPI не разрешает имена строк, включая символы "[", "]" или ":", даже если файловая система разрешает их использование. Размер строки, передаваемой для параметра  _lpszName,_ не должен превышать значение MAX_PATH максимальной длины строки, содержаной имя пути. 
    
_ulFlags_
  
> [in] Битоваяmas флагов, используемых для указать режим функции. Можно установить следующие флаги:
    
TNEF_BEST_DATA 
  
> Все возможные свойства сопоируются с их атрибутами более высокого уровня, но при возможной потере данных из-за преобразования в атрибут более высокого уровня свойство также кодируются в инкапсуляции. Обратите внимание, что это приведет к дублированию информации в потоке TNEF. TNEF_BEST_DATA по умолчанию, если другие режимы не указаны. 
    
TNEF_COMPATIBILITY 
  
> Обеспечивает обратную совместимость с старыми клиентские приложениями. Потоки TNEF, закодированные с помощью этого флага, соотят все возможные свойства с соответствующим атрибутом более высокого уровня. Этот режим также приводит к стандарту некоторых свойств, которые требуются клиентам более высокого уровня. 
    
  > [!CAUTION]
  > Этот флаг устарел и не должен использоваться. 
  
TNEF_DECODE 
  
> Объект TNEF в заявяемом потоке открывается с доступом только для чтения. Поставщик транспорта должен установить этот флаг, если функция инициализирует объект для последующего декодирования.
    
TNEF_ENCODE 
  
> Объект TNEF в заявяемом потоке открывается для разрешения на чтение и записи. Поставщик транспорта должен установить этот флаг, если функция инициализирует объект для последующей кодии.
    
TNEF_PURE 
  
> Кодирует все свойства в блоки инкапсуляции MAPI. Поэтому "чистый" TNEF-файл будет состоять не более чем из атрибутов attMAPIProps, attAttachment, attRenddata и attRecipTable. Этот режим идеально подходит для использования, когда обратная совместимость не требуется.
    
_lpMessage_
  
> [in] Указатель на объект сообщения в качестве места назначения декодированного сообщения с вложениями или источника закодированного сообщения с вложениями. Любые свойства сообщения назначения могут быть перезаписаны свойствами закодированного сообщения.
    
_wKeyVal_
  
> [in] Ключ поиска, который объект TNEF использует для поиска вложений с текстовыми тегами, вставленными в текст сообщения. Это значение должно быть относительно уникальным для всех сообщений. 
    
_lpAddressBook_
  
> [in] Указатель на объект адресной книги, используемый для получения сведений об адресов для идентификаторов записей. 
    
_lppTNEF_
  
> [out] Указатель на новый объект TNEF.

## <a name="remarks"></a>Примечания

Функция **OpenTnefStreamEx** является рекомендуемой заменой [openTnefStream](opentnefstream.md), исходной точки входа для доступа в TNEF. 
  
Объект TNEF, созданный функцией **OpenTnefStreamEx,** позднее вызывает метод OLE **IUnknown::AddRef,** чтобы добавить ссылки на объект поддержки, объект stream и объект сообщения. Поставщик транспорта может освободить ссылки для всех трех объектов одним вызовом метода OLE **IUnknown::Release** для объекта TNEF. 
  
**OpenTnefStreamEx** выделяет и инициализирует объект TNEF, который поставщик должен использовать для кодивизации сообщения MAPI в сообщение потока TNEF. Кроме того, эта функция может настроить объект поставщика для использования в последующих вызовах [ITnef::ExtractProps](itnef-extractprops.md) для декодирования сообщения потока TNEF в сообщение MAPI. Чтобы освободить объект TNEF и закрыть сеанс, поставщик транспорта должен вызвать унаследованный метод **IUnknown::Release** объекта. 
  
Базовое значение параметра _wKeyVal_ не должно быть нулем и не должно быть одинаковым для каждого вызова **OpenTnefStreamEx.** Вместо этого используйте случайные числа на основе системного времени из генератора случайных чисел библиотеки времени запуска.
