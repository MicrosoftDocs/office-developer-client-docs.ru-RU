# <a name="scontentrestriction"></a>SContentRestriction
 
**Относится к**: Outlook 
  
Описывается ограничение содержимого, которое используется для ограничения представление таблицы, чтобы только строки, которые включают столбец с соответствующими строке поиска содержимого. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapidefs.h  <br/> |
   
```cpp
typedef struct _SContentRestriction
{
  ULONG        ulFuzzyLevel;
  ULONG        ulPropTag;
  LPSPropValue lpProp;
} SContentRestriction;

```

## <a name="members"></a>Members

**ulFuzzyLevel**
  
> Определение уровня привыкли, ограничение содержимого следует применять при проверке соответствия параметров.
    
   **Нижние 16 бит** член **ulFuzzyLevel** применять для свойств типа PT_BINARY и PT_STRING8 и должен быть установлено одно из следующих значений: 
    
   - FL_FULLSTRING: В соответствии с, строка поиска **lpProp** должны быть включены в свойство, определяемое **ulPropTag**.
        
   - FL_PREFIX: В соответствии с, строка поиска **lpProp** , которые должны встречаться в начале свойство, определяемое **ulPropTag**. Только полностью заполняя длину строки поиска, указанный в параметре **lpProp**сравнения двух строк. 
        
   - FL_SUBSTRING: В соответствии с, строка поиска **lpProp** должны быть включены в любом месте в свойство, определяемое **ulPropTag**. 
        
   **Верхнем 16-разрядный** члена **ulFuzzyLevel** применяются только к свойств типа PT_STRING8 и может быть установлено в любое сочетание следующих значений: 
        
   - FL_IGNORECASE: Необходимо выполнить сравнение без учета регистр. 
        
   - FL_IGNORENONSPACE: Сравнение следует игнорировать определенные Юникод дополнительный символы, такие как диакритические знаки. 
        
   - FL_LOOSE: Сравнение должен привести соответствие по возможности без учета case и дополнительный символов. 
    
**ulPropTag**
  
> Свойство tag идентифицирующий свойство строки для проверки вхождения строки поиска. 
    
**lpProp**
  
> Указатель на структуру значение свойства, которое содержит строковое значение для использования в качестве строки поиска.
    
## <a name="remarks"></a>Замечания

Существует два свойства теги в структуре **SContentRestriction** : один в элемент **ulPropTag** , а другая — в член **ulPropTag** структуры **SPropValue** указывает **lpProp**. В обоих тегов MAPI требуется только поле типа свойства и игнорирует поле идентификатора свойства. Тем не менее должно соответствовать типы двух свойств, либо MAPI_E_TOO_COMPLEX возвращается значение при ограничении удалось вызвать [IMAPITable::Restrict](imapitable-restrict.md) или [IMAPITable::FindRow](imapitable-findrow.md). 
  
Значения FL_FULLSTRING, FL_PREFIX и FL_SUBSTRING являются взаимоисключающими. Можно задать только один из них и одно из них должно быть задано. Предопределенная их значения и поставщик должен реализовывать их так же, как определенные. Поставщик должен возвращать MAPI_E_TOO_COMPLEX, если он не может поддерживать эти значения. 
  
Значения FL_IGNORECASE, FL_IGNORENONSPACE и FL_LOOSE не зависят. В любом из нуля на все три из них можно задать. В качестве рекомендации только приведены их определения и поставщик может реализовать свой собственный особое значение каждого флага. Поставщик не должен возвращать указаний об ошибке, если он не имеет реализации указанного флага. 
  
В результате контента ограничения, связанные со свойством не определен, если свойство не существует. Когда клиент требует некорректно для подобных ограничений и не является ли свойство например существует, он не был обязательный столбец таблицы его следует создать **и** ограничения для присоединения к ограничение содержимого ограничению exist. Используйте структуру [SExistRestriction](sexistrestriction.md) , чтобы определить ограничение exist и структуру [SAndRestriction](sandrestriction.md) для определения **и** ограничение. 
  
Дополнительные сведения о структуре **SContentRestriction** и ограничения в целом, обратитесь к разделу [О ограничения](about-restrictions.md).
