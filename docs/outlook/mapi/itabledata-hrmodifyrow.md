# <a name="itabledatahrmodifyrow"></a>ITableData::HrModifyRow

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Вставляет новую строку таблицы, возможно заменив существующую.
  
```cpp
HRESULT HrModifyRow(
  LPSRow lpSRow
);
```

## <a name="parameters"></a>Параметры

 _Лпсров_
  
> возврата Указатель на структуру [сров](srow.md) , описывающую строку, которую необходимо добавить или заменить существующую строку. Одна из структур значений свойств, на которую ссылается элемент **лппропс** структуры **сров** , должна содержать столбец индекса, то же значение, которое было указано в параметре _улпроптагиндексколумн_ при вызове [креатетабле ](createtable.md)функция. 
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Строка была успешно вставлена или изменена.
    
МАПИ_Е_ИНВАЛИД_ПАРАМЕТЕР 
  
> Переданная строка не имеет столбца индекса.
    
## <a name="remarks"></a>Примечания

Метод **итабледата:: хрмодифиров** вставляет строку, описанную структурой **сров** , на которую указывает параметр _лпсров_ . Если строка, имеющая то же значение для столбца индекса, в качестве строки, в которой _лпсров_ указывает, уже существует в таблице, то существующая строка заменяется. Если строка, соответствующая одной из структур **сров** , не существует, **хрмодифиров** добавляет строку в конец таблицы. 
  
Все представления таблицы изменяются, чтобы включить строку, на которую указывает _лпсров_. Однако если в представлении есть ограничение, которое исключает строку, оно может быть невидимым для пользователя. 
  
Столбцы в строке, на которые указывает _лпсров_ , не обязательно должны находиться в том же порядке, что и столбцы в таблице. Вызывающий также может включать в себя свойства Columns, которые в настоящее время не находятся в таблице. Для существующих представлений **хрмодифиров** делает эти новые столбцы доступными, но не включают их в текущий набор столбцов. Для будущих просмотров **хрмодифиров** включает новые столбцы в наборе столбцов. 
  
После того как **хрмодифиров** добавит строку, уведомления отправляются всем клиентам или поставщикам услуг, которые имеют представление таблицы и вызывают метод [IMAPITable:: Advise](imapitable-advise.md) для регистрации уведомлений.
