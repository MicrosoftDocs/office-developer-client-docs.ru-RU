# <a name="launchwizardentry"></a>LAUNCHWIZARDENTRY

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Определяет функцию, которая запускает приложение Мастер профилей для добавления одной или более служб сообщений в профиль. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapiwz.h  <br/> |
|Определенные функции, реализованные в:  <br/> |MAPI  <br/> |
|Определенная функция, вызванная:  <br/> |Клиентские приложения  <br/> |
   
```cpp
HRESULT LAUNCHWIZARDENTRY(
  HWND hParentWnd,
  ULONG ulFlags,
  LPCSTR FAR * lppszServiceNameToAdd,
  ULONG cbBufferMax,
  LPSTR lpszNewProfileName
);
```

## <a name="parameters"></a>Parameters

 _hParentWnd_
  
> [in] Ручка родительского окна вызываемого. Если у вызываемого нет родительского окна,  _параметром hParentWnd_ должен быть NULL. 
    
 _ulFlags_
  
> [in] Bitmask флагов с указанием параметров мастера профиля. Можно установить следующие флаги:
    
MAPI_PW_ADD_SERVICE_ONLY 
  
> Мастер профилей должен добавлять только службы сообщений, перечисленные через параметр  _lppszServiceNameToAdd,_ а не отображать свою страницу для выбора служб сообщений. 
    
MAPI_PW_FIRST_PROFILE 
  
> Созданный профиль является первым для этой рабочей станции. 
    
MAPI_PW_HIDE_SERVICES_LIST 
  
> Страница мастера профилей для выбора служб сообщений не должна отображаться. 
    
MAPI_PW_LAUNCHED_BY_CONFIG 
  
> Мастер профилей был запущен приложением конфигурации панели управления. 
    
MAPI_PW_PROVIDER_UI_ONLY 
  
> Должны отображаться только диалоговые окна конфигурации поставщиков услуг, а страницы мастера профилей не отображаются. Этот флаг можно установить, только если MAPI_PW_ADD_SERVICE_ONLY заданной. 
    
 _lppszServiceNameToAdd_
  
> [in] Указатель на массив строк, содержащий имена служб сообщений, которые будут добавлены в профиль. Массив должен завершиться со значением NULL. 
    
 _cbBufferMax_
  
> [in] Размер буфера, на который указывает параметр _lpszNewProfileName._ 
    
 _lpszNewProfileName_
  
> [вышел] Указатель на буфер строки, где функция на основе **LAUNCHWIZARDENTRY** возвращает имя созданного профиля.

## <a name="remarks"></a>Примечания

Реализация прототипа функции **LAUNCHWIZARDENTRY** MAPI является точкой входа в приложение Мастер профилей MAPI. MAPI называет эту точку **входа LaunchWizard**. 
  
Когда флаг MAPI_PW_ADD_SERVICE_ONLY в параметре  _ulFlags,_ применяются следующие правила: 
  
- Флаг MAPI_PW_LAUNCHED_BY_CONFIG отобразится на странице приветствия. 
    
- Флаги MAPI_PW_HIDE_SERVICES_LIST и MAPI_PW_PROVIDER_UI_ONLY полезны только при отсутствие профиля по умолчанию. В этом случае эти флаги определяют, какая страница мастера профилей должна отображаться. 
    
- Если профиль по умолчанию существует, ни одна из страниц мастера профилей не должна отображаться. 
    
- Если профиль по умолчанию существует, через параметр  _lppszServiceNameToAdd_ перечислены только одна служба сообщений, и эта служба сообщений уже находится в профиле по умолчанию, мастер профилей возвращает S_OK, ничего не добавляя в профиль. 
    
Чтобы каждая служба сообщений была добавлена в профиль, мастер профилей вызывает функцию точки входа службы на основе прототипа [MSGSERVICEENTRY.](msgserviceentry.md) Чтобы каждый поставщик услуг, выбранный из службы сообщений, добавляемой в профиль, вызывает функцию точки входа поставщика на основе прототипа [WIZARDENTRY.](wizardentry.md) Во время интерактивной конфигурации каждое событие пользователя на страницах свойств заставляет мастера профиля вызывать функцию вызова поставщика на основе прототипа [SERVICEWIZARDDLGPROC.](servicewizarddlgproc.md) 
  
Если поставщик услуг, добавляемый в профиль, поддерживает страницы мастера профилей, он должен разрешить программную конфигурацию профиля.
