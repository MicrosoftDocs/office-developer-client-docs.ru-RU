# <a name="launchwizardentry"></a>LAUNCHWIZARDENTRY

  
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
Определяет функцию, которая запускает мастер профиля приложения для добавления одной или нескольких служб сообщения в профиль. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapiwz.h  <br/> |
|Функция реализован:  <br/> |MAPI  <br/> |
|Вызывается функция:  <br/> |Клиентские приложения  <br/> |
   
```cpp
HRESULT LAUNCHWIZARDENTRY(
  HWND hParentWnd,
  ULONG ulFlags,
  LPCSTR FAR * lppszServiceNameToAdd,
  ULONG cbBufferMax,
  LPSTR lpszNewProfileName
);
```

## <a name="parameters"></a>Параметры

 _hParentWnd_
  
> [in] Дескриптор родительского окна вызывающего абонента. Если вызывающий не имеет родительского окна, параметр _hParentWnd_ должен быть NULL. 
    
 _ulFlags_
  
> [in] Битовая маска, указывающее, параметры для мастера профиль флаги. Можно задать следующие флажки:
    
MAPI_PW_ADD_SERVICE_ONLY 
  
> Мастер профиля является добавление службы сообщений, перечисленных через параметр _lppszServiceNameToAdd_ и отображает его страницы для выбора службы сообщений. 
    
MAPI_PW_FIRST_PROFILE 
  
> Профиль будет создан является первым для данной рабочей станции. 
    
MAPI_PW_HIDE_SERVICES_LIST 
  
> Страница мастера профиля для выбора службы сообщений не должно отображаться. 
    
MAPI_PW_LAUNCHED_BY_CONFIG 
  
> В приложении настройки панели управления был запущен мастер профилей. 
    
MAPI_PW_PROVIDER_UI_ONLY 
  
> Отображать только поставщиков услуг конфигурации диалоговых окон и страницах мастера профилей не должно быть. Этот флаг могут быть установлены только в том случае, если установлен флаг MAPI_PW_ADD_SERVICE_ONLY. 
    
 _lppszServiceNameToAdd_
  
> [in] Указатель на массив строк, содержащий имена службы сообщений для добавления к профилю. Массив должен завершать значение NULL. 
    
 _cbBufferMax_
  
> [in] Размер буфера, на который указывает параметр _lpszNewProfileName_ . 
    
 _lpszNewProfileName_
  
> [out] Указатель на буфер строки, где на основании **LAUNCHWIZARDENTRY** функция возвращает имя созданного профиля.

## <a name="remarks"></a>Замечания

Реализация интерфейса MAPI прототип функции **LAUNCHWIZARDENTRY** является точка входа для приложения мастера профилей MAPI. MAPI имена эта точка входа **LaunchWizard**. 
  
Если в параметре _ulFlags_ флаг MAPI_PW_ADD_SERVICE_ONLY, применяются следующие правила: 
  
- Флаг MAPI_PW_LAUNCHED_BY_CONFIG этот параметр запрещает начальная страница отображаются. 
    
- Флаги MAPI_PW_HIDE_SERVICES_LIST и MAPI_PW_PROVIDER_UI_ONLY полезны только в том случае, если профиль не по умолчанию. В этом случае эти флаги определить, какой мастер профилей, будет отображаться страница. 
    
- Если с профилем по умолчанию, нет на страницах мастера профилей, для отображения. 
    
- Если с профилем по умолчанию, через параметр _lppszServiceNameToAdd_ указана только одно сообщение служба и служба сообщений уже профиля по умолчанию, мастер профиля возвращает значение S_OK, не добавляя все действия в профиль. 
    
Для каждой службы сообщения добавляется к профилю мастер профилей вызывает функцию точки входа службы на основании прототип [MSGSERVICEENTRY](msgserviceentry.md) . Для каждого поставщика служб, выбранные из службы сообщений для добавления к профилю мастер профилей вызывает функцию точки входа поставщика на основании прототип [WIZARDENTRY](wizardentry.md) . Во время интерактивного настройки мастер профиля для вызова функции обратного вызова поставщика, основанной на прототип [SERVICEWIZARDDLGPROC](servicewizarddlgproc.md) вызывает событие каждого пользователя в страницы свойств. 
  
Если поставщик услуг, добавляемый к профилю поддерживает на страницах мастера профилей, его необходимо разрешить программный настройки профиля.
