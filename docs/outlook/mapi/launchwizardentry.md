# <a name="launchwizardentry"></a>LAUNCHWIZARDENTRY

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Определяет функцию, которая запускает приложение мастера профилей для добавления одной или более служб сообщений в профиль. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapiwz.h  <br/> |
|Определяемая функция, реализованная в:  <br/> |MAPI  <br/> |
|Определяемая функция, вызванная:  <br/> |Клиентские приложения  <br/> |
   
```cpp
HRESULT LAUNCHWIZARDENTRY(
  HWND hParentWnd,
  ULONG ulFlags,
  LPCSTR FAR * lppszServiceNameToAdd,
  ULONG cbBufferMax,
  LPSTR lpszNewProfileName
);
```

## <a name="parameters"></a>Параметры

 _hParentWnd_
  
> [in] Handle to the caller's parent window. Если у вызываемого не имеется родительского окна, параметр  _hParentWnd_ должен иметь NULL. 
    
 _ulFlags_
  
> [in] Битоваяmas флагов, указывающих параметры для мастера профилей. Можно установить следующие флаги:
    
MAPI_PW_ADD_SERVICE_ONLY 
  
> Мастер профилей добавляет только службы сообщений, перечисленные с помощью параметра  _lppszServiceNameToAdd,_ и не отображает свою страницу для выбора служб сообщений. 
    
MAPI_PW_FIRST_PROFILE 
  
> Профиль, который необходимо создать, является первым для этой рабочей станции. 
    
MAPI_PW_HIDE_SERVICES_LIST 
  
> Страница мастера профилей для выбора служб сообщений не должна отображаться. 
    
MAPI_PW_LAUNCHED_BY_CONFIG 
  
> Мастер профилей был запущен приложением конфигурации панели управления. 
    
MAPI_PW_PROVIDER_UI_ONLY 
  
> Должны отображаться только диалоговые окна конфигурации поставщиков служб, а страницы мастера профилей не должны отображаться. Этот флаг можно установить, только если MAPI_PW_ADD_SERVICE_ONLY установлен. 
    
 _lppszServiceNameToAdd_
  
> [in] Указатель на массив строк, содержащий имена служб сообщений, добавляемого в профиль. Массив должен завершаться значением NULL. 
    
 _cbBufferMax_
  
> [in] Размер буфера, на который указывает параметр _lpszNewProfileName._ 
    
 _lpszNewProfileName_
  
> [out] Указатель на буфер строк, где функция, основанная на **LAUNCHWIZARDENTRY,** возвращает имя созданного профиля.

## <a name="remarks"></a>Примечания

Реализация MAPI прототипа функции **LAUNCHWIZARDENTRY** является точкой входа в приложение мастера профилей MAPI. MAPI именует эту точку **входа LaunchWizard.** 
  
Если флаг MAPI_PW_ADD_SERVICE_ONLY в параметре  _ulFlags,_ применяются следующие правила: 
  
- Флаг MAPI_PW_LAUNCHED_BY_CONFIG не отображает страницу приветствия. 
    
- Флаги MAPI_PW_HIDE_SERVICES_LIST и MAPI_PW_PROVIDER_UI_ONLY полезны, только если нет профиля по умолчанию. В этом случае эти флаги определяют, какая страница мастера профилей должна отображаться. 
    
- Если профиль по умолчанию существует, страницы мастера профилей не отображаются. 
    
- Если профиль по умолчанию существует, с помощью параметра  _lppszServiceNameToAdd_ указана только одна служба сообщений, а эта служба сообщений уже находится в профиле по умолчанию, мастер профилей возвращает S_OK, не добавляя ничего в профиль. 
    
Для каждой службы сообщений, добавляемой в профиль, мастер профилей вызывает функцию точки входа службы на основе прототипа [MSGSERVICEENTRY.](msgserviceentry.md) Для каждого поставщика услуг, выбранного из службы сообщений, добавляемой в профиль, мастер профилей вызывает функцию точки входа поставщика на основе прототипа [WIZARDENTRY.](wizardentry.md) Во время интерактивной настройки каждое событие пользователя на страницах свойств заставляет мастер профилей вызывать функцию вызова поставщика на основе прототипа [SERVICEWIZARDDLGPROC.](servicewizarddlgproc.md) 
  
Если добавляемая в профиль служба поддерживает страницы мастера профилей, он должен разрешить программную настройку профиля.
