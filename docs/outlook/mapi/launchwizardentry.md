# <a name="launchwizardentry"></a>LAUNCHWIZARDENTRY

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Определяет функцию, которая запускает приложение мастера профилей в целях добавления одной или нескольких служб сообщений в профиль. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Мапивз. h  <br/> |
|Определенная функция реализована следующим образом:  <br/> |MAPI  <br/> |
|Определенная функция, вызываемая:  <br/> |Клиентские приложения  <br/> |
   
```cpp
HRESULT LAUNCHWIZARDENTRY(
  HWND hParentWnd,
  ULONG ulFlags,
  LPCSTR FAR * lppszServiceNameToAdd,
  ULONG cbBufferMax,
  LPSTR lpszNewProfileName
);
```

## <a name="parameters"></a>Параметры

 _Хпарентвнд_
  
> возврата Дескриптор родительского окна вызывающего абонента. Если у вызывающего абонента нет родительского окна, параметр _хпарентвнд_ должен иметь значение null. 
    
 _ulFlags_
  
> возврата Битовая маска флагов, указывающих параметры для мастера профилей. Можно задать следующие флаги:
    
МАПИ_ПВ_АДД_СЕРВИЦЕ_ОНЛИ 
  
> С помощью мастера профилей можно добавить только службы сообщений, перечисленные с помощью параметра _лппсзсервиценаметоадд_ , и не отображать его страницу для выбора служб сообщений. 
    
МАПИ_ПВ_ФИРСТ_ПРОФИЛЕ 
  
> Профиль, который необходимо создать, является первым для этой рабочей станции. 
    
МАПИ_ПВ_ХИДЕ_СЕРВИЦЕС_ЛИСТ 
  
> Не отображается страница мастера профилей для выбора служб сообщений. 
    
МАПИ_ПВ_ЛАУНЧЕД_БИ_КОНФИГ 
  
> Мастер профилей запущен приложением настройки панели управления. 
    
МАПИ_ПВ_ПРОВИДЕР_УИ_ОНЛИ 
  
> Должны отображаться только диалоговые окна настройки поставщиков услуг, и не должны отображаться страницы мастера профилей. Этот флаг можно задать, только если установлен флаг МАПИ_ПВ_АДД_СЕРВИЦЕ_ОНЛИ. 
    
 _Лппсзсервиценаметоадд_
  
> возврата Указатель на массив строк, содержащий имена служб сообщений, которые необходимо добавить в профиль. Массив должен завершаться значением NULL. 
    
 _Кббуффермакс_
  
> возврата Размер буфера, на который указывает параметр _лпсзневпрофиленаме_ . 
    
 _Лпсзневпрофиленаме_
  
> вышли Указатель на буфер строки, в котором функция на основе **лаунчвизардентри** возвращает имя созданного профиля.

## <a name="remarks"></a>Замечания

Реализация функции MAPI для прототипа функции **лаунчвизардентри** — точка входа в приложение мастера профилей MAPI. MAPI Names указывает эту точку входа **лаунчвизард**. 
  
Если в параметре _ulFlags_ ЗАДАН флаг мапи_пв_адд_сервице_онли, применяются следующие правила. 
  
- Флаг МАПИ_ПВ_ЛАУНЧЕД_БИ_КОНФИГ запрещает отображение страницы приветствия. 
    
- Флаги МАПИ_ПВ_ХИДЕ_СЕРВИЦЕС_ЛИСТ и МАПИ_ПВ_ПРОВИДЕР_УИ_ОНЛИ можно использовать только в том случае, если профиль по умолчанию отсутствует. В этом случае эти флаги определяют, какую страницу мастера профилей следует отображать. 
    
- Если профиль по умолчанию существует, не будут отображаться никакие страницы мастера профилей. 
    
- Если профиль по умолчанию существует, только одна служба сообщений перечислена через параметр _лппсзсервиценаметоадд_ , и эта служба сообщений уже включена в профиль по умолчанию, мастер профилей ВОЗВРАЩАЕТ значение S_OK без добавления в профиль. 
    
Для каждой службы сообщений, добавляемой в профиль, мастер профилей вызывает функцию точки входа службы на основе прототипа [мсгсервицеентри](msgserviceentry.md) . Для каждого поставщика услуг, выбранного из службы сообщений, добавляемого в профиль, мастер профилей вызывает функцию точки входа поставщика на основе прототипа [визардентри](wizardentry.md) . Во время интерактивной конфигурации каждое событие пользователя на страницах свойств заставляет мастер профилей вызывать функцию обратного вызова поставщика на основе прототипа [сервицевизарддлгпрок](servicewizarddlgproc.md) . 
  
Если поставщик услуг, добавляемый в профиль, поддерживает страницы мастера профилей, он должен разрешить программную конфигурацию профиля.
