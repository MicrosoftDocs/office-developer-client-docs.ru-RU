# <a name="comparing-address-book-entries"></a>Сравнение записей адресной книги

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Реализация [IABLogon::CompareEntryIDs](iablogon-compareentryids.md) поставщика сравнивает идентификаторы записей для двух объектов поставщика. MAPI вызывает этот метод после определения того, что два идентификатора записи содержат зарегистрированный [MAPIUID](mapiuid.md)поставщика. Поэтому **методу CompareEntryIDs** не нужно проверять, принадлежат ли идентификаторы записей, переданные для _lpEntryID1_ и _lpEntryID2._ 
  
Вызов **IABLogon::CompareEntryIDs** эквивалентен PR_RECORD_KEY **свойства** [(PidTagRecordKey)](pidtagrecordkey-canonical-property.md)для каждого из двух объектов и сопоставлению их напрямую.
  
 **Реализация CompareEntryIds**
  
1. Проверьте тип идентификаторов записи, переданных в том случае, если поставщик сохраняет эти сведения. Например, один идентификатор записи может принадлежать пользователю обмена сообщениями, а другой — списку рассылки. Если типы не совпадают, установите содержимое  _параметра lpulResult_ false и возврат. 
    
2. Сравните размеры двух идентификаторов записи. Если они не одинаковые, установите содержимое  _параметра lpulResult_ false и возврат. 
    
3. Убедитесь, что размер идентификаторов записи является правильным для их типа. Если нет, установите содержимое  _параметра lpulResult_ false и верни значение ошибки MAPI_E_UNKNOWN_ENTRYID. 
    
4. Проверьте, одинаковы ли идентификаторы записи. Если они сравнивают одинаково, установите содержимое  _параметра lpulResult_ к TRUE и возврату. В противном случае установите его false перед возвращением. 
    
5. Если поставщик сравнивает краткосрочный идентификатор записи с долгосрочным идентификатором, он должен сравниваться одинаково.
