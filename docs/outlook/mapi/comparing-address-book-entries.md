# <a name="comparing-address-book-entries"></a>Сравнение записей адресной книги

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Реализация [IABLogon::CompareEntryIDs](iablogon-compareentryids.md) поставщика сравнивает идентификаторы записей для двух объектов поставщика. MAPI вызывает этот метод после определения того, что два идентификатора записи содержат зарегистрированный [MAPIUID поставщика.](mapiuid.md) Поэтому метод **CompareEntryIDs** не должен проверять, что идентификаторы записей, переданные для  _параметров lpEntryID1_ и  _lpEntryID2,_ принадлежат поставщику. 
  
Вызов **IABLogon::CompareEntryIDs** эквивалентен тому, как получить свойство **PR_RECORD_KEY** ([PidTagRecordKey)](pidtagrecordkey-canonical-property.md)для каждого из двух объектов и сравнить их напрямую.
  
 **Реализация CompareEntryIds**
  
1. Проверьте тип идентификаторов записей, переданных, если поставщик сохраняет эти сведения. Например, один идентификатор записи может принадлежать пользователю системы обмена сообщениями, а другой — списку рассылки. Если типы не совпадают, установите для  _параметра lpulResult_ параметр FALSE и вернетесь. 
    
2. Сравните размеры двух идентификаторов записей. Если они не одинаковые, установите для  _содержимого параметра lpulResult_ параметр FALSE и вернетесь. 
    
3. Убедитесь, что размер идентификаторов записей правильный для их типа. Если нет, установите для  _содержимого параметра lpulResult_ значение FALSE и вернетесь значение ошибки MAPI_E_UNKNOWN_ENTRYID. 
    
4. Проверьте, одинаковы ли идентификаторы записей. Если они сравнивают одинаково, установите для  _содержимого параметра lpulResult_ параметр TRUE и возврат. В противном случае установите для него false перед возвратом. 
    
5. Если поставщик сравнивает краткосрочный идентификатор записи с долгосрочным идентификатором, он должен сравниться одинаково.
