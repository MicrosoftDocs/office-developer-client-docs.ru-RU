# <a name="pidtagreceivedrepresentingaddresstype-canonical-property"></a>Каноническое свойство PidTagReceivedRepresentingAddressType

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Содержит тип адреса для пользователя обмена сообщениями, представленного фактически принимающим сообщение пользователем.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_RCVD_REPRESENTING_ADDRTYPE, PR_RCVD_REPRESENTING_ADDRTYPE_A, PR_RCVD_REPRESENTING_ADDRTYPE_W  <br/> |
|Идентификатор:  <br/> |0x0077  <br/> |
|Тип данных:  <br/> |PT_UNICODE, PT_STRING8  <br/> |
|Область:  <br/> |Address  <br/> |
   
## <a name="remarks"></a>Примечания

Эти свойства являются примерами свойств адреса для пользователя обмена сообщениями, представленного принимающим пользователем. Его должен установить поставщик входящих транспортных услуг, который также отвечает за авторизацию или проверку делегата. Если пользователь сообщений не представлен, этому свойству следует установить тип адреса, содержащийся в свойстве **PR_RECEIVED_BY_ADDRTYPE** ([PidTagReceivedByAddressType).](pidtagreceivedbyaddresstype-canonical-property.md)
  
Клиентская приложение, отвечая на сообщение, полученное от имени другого клиента, должно скопировать это свойство из полученного сообщения в свойство **PR_SENT_REPRESENTING_ADDRTYPE** ([PidTagSentRepresentingAddressType)](pidtagsentrepresentingaddresstype-canonical-property.md)для ответа.
