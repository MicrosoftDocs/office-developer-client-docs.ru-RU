# <a name="pidtagrecipientflags-canonical-property"></a>Каноническое свойство PidTagRecipientFlags

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Задает битовое поле, описывающее состояние получателя.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |ПР_РЕЦИПИЕНТ_ФЛАГС  <br/> |
|Идентификатор:  <br/> |0x5FFD  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Получатель транспорта  <br/> |
   
## <a name="remarks"></a>Примечания

Это свойство не является обязательным. Ниже приведены отдельные флаги, которые можно задать.
  
|**Value**|**Описание**|
|:-----|:-----|
|S (РеЦипсендабле, 0x00000001)  <br/> |Получатель — это доступный для **отправки** участник. Этот флаг используется только в свойстве **диспидапптунсендаблереЦипс** ([PidLidAppointmentUnsendableRecipients](pidlidappointmentunsendablerecipients-canonical-property.md)).  <br/> |
|O (РеЦипорганизер, 0x0000002)  <br/> |**РеЦипиентров** , на котором установлен этот флаг, представляет организатор собрания.  <br/> |
|ER (РеЦипексцептионалреспонсе, 0x00000010)  <br/> |Указывает, что участник предоставил ответ на исключение, в котором находится этот **реЦипиентров** . Этот флаг используется только в **реЦипиентров** исключения внедренного объекта Message объекта собрания организатора.  <br/> |
|ED (РеЦипексцептионалделетед, 0x00000020)  <br/> |Указывает, что несмотря на то что **реЦипиентров** существует, он должен рассматриваться так, как если бы соответствующий получатель не был. Этот флаг используется только в **реЦипиентров** исключения внедренного объекта Message объекта собрания организатора.  <br/> |
|X (зарезервировано, 0x00000040)  <br/> |Не должно быть задано.  <br/> |
|X (зарезервировано, 0x00000080)  <br/> |Не должно быть задано.  <br/> |
|G (РеЦипоригинал, 0x00000100)  <br/> |Указывает, что получатель является исходным участником. Этот флаг используется только в свойстве **диспидапптунсендаблереЦипс** .  <br/> |
|X (зарезервировано, 0x00000200)  <br/> |Резервирования.  <br/> |
