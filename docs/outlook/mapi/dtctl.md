# <a name="dtctl"></a>DTCTL

**Относится к**: Outlook 2013 | Outlook 2016 
  
Описывает элемент управления, который будет использоваться в диалоговом окне, созданном из таблицы отображения. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |MAPIDEFS. h  <br/> |
   
```cpp
typedef struct
{
  ULONG ulCtlType;
  ULONG ulCtlFlags;
  LPBYTE lpbNotif;
  ULONG cbNotif;
  LPSTR lpszFilter;
  ULONG ulItemID;
  union
  {
    LPVOID lpv;
    LPDTBLLABEL lplabel;
    LPDTBLEDIT lpedit;
    LPDTBLLBX lplbx;
    LPDTBLCOMBOBOX lpcombobox;
    LPDTBLDDLBX lpddlbx;
    LPDTBLCHECKBOX lpcheckbox;
    LPDTBLGROUPBOX lpgroupbox;
    LPDTBLBUTTON lpbutton;
    LPDTBLRADIOBUTTON lpradiobutton;
    LPDTBLMVLISTBOX lpmvlbx;
    LPDTBLMVDDLBX lpmvddlbx;
    LPDTBLPAGE lppage;
  } ctl;
} DTCTL, FAR *LPDTCTL;

```

## <a name="members"></a>Members

**Улктлтипе**
  
> Тип элемента управления, включенный в элемент **CTL** и соответствующий свойству **пр_контрол_типе** ([PidTagControlType](pidtagcontroltype-canonical-property.md)) элемента управления. Возможны следующие значения:
    
ДТКТ_ЛАБЕЛ 
  
> Элемент управления Label.
    
ДТКТ_ЕДИТ 
  
> Элемент управления Edit.
    
ДТКТ_ЛБКС 
  
> Элемент управления "список".
    
ДТКТ_КОМБОБОКС 
  
> Элемент управления "поле со списком".
    
ДТКТ_ДДЛБКС 
  
> Элемент управления "раскрывающийся список".
    
ДТКТ_ЧЕККБОКС 
  
> Элемент управления "флажок".
    
ДТКТ_ГРАУПБОКС 
  
> Элемент управления "Группа".
    
ДТКТ_БУТТОН 
  
> Элемент управления "Кнопка".
    
ДТКТ_ПАЖЕ 
  
> Элемент управления "Вкладка".
    
ДТКТ_РАДИОБУТТОН 
  
> Элемент управления "переключатель".
    
ДТКТ_МВЛИСТБОКС 
  
> Многозначный список элементов управления.
    
ДТКТ_МВДДЛБКС 
  
> Элемент управления раскрывающегося списка с несколькими значениями.
    
**Улктлфлагс**
  
> Битовая маска флагов, описывающих компоненты элемента управления и соответствующие свойству **пр_контрол_флагс** ([PidTagControlFlags](pidtagcontrolflags-canonical-property.md)) элемента управления. Эти флажки можно устанавливать только для флажков, полей со списками, списков и элементов управления редактирования. Возможны следующие значения:
    
ДТ_АКЦЕПТ_ДБКС 
  
> Принимается формат ANSI или DBCS. Этот флаг действителен только для элементов управления редактированием.
    
ДТ_ЕДИТАБЛЕ 
  
> Пользователь может изменять текст в элементе управления. 
    
ДТ_МУЛТИЛИНЕ 
  
> Элемент управления может содержать несколько строк текста. Этот флаг действителен только для элементов управления редактированием.
    
ДТ_ПАССВОРД_ЕДИТ 
  
> Элемент управления содержит пароль; Поэтому содержимое элемента управления не должно отображаться для пользователя. Этот флаг действителен только для элементов управления редактированием.
    
ДТ_РЕКУИРЕД 
  
> Элемент управления "диалоговое окно" является обязательным. Этот флаг действителен только для элементов управления "поле ввода" и "поле со списком".
    
ДТ_СЕТ_ИММЕДИАТЕ 
  
> Включает немедленный вывод значения при изменении элемента управления. Это позволяет установить отношение зависимости между двумя элементами управления. 
    
**Лпбнотиф**
  
> Указатель на структуру, состоящую из структуры [GUID](guid.md) , которая представляет поставщика услуг и идентификатор элемента управления. Члены **лпбнотиф** и **кбнотиф** соответствуют свойству **пр_контрол_ид** ([PidTagControlId](pidtagcontrolid-canonical-property.md)) элемента управления и используются для уведомления пользовательского интерфейса при необходимости обновления элемента управления.
    
**Кбнотиф**
  
> Количество байтов в структуре, на которую указывает элемент **лпбнотиф** . 
    
**Лпсзфилтер**
  
> Указатель на строку символов, описывающую символы, которые можно ввести в элемент управления "поле ввода" или "поле со списком". Для других типов элементов управления элемент **лпсзфилтер** может иметь значение null. Для элементов управления "поле со списком" и "поле со списком" это должно быть регулярное выражение, которое применяется к отдельному символу за раз. Один и тот же фильтр применяется ко всем символам в элементе управления. Строка фильтра имеет следующий формат: 
    
|**Символ**|**Описание**|
|:-----|:-----|
| `*` <br/> | Разрешен любой символ (например, `"*"`).  <br/> |
| `[ ]` <br/> |Определяет набор символов (например, `"[0123456789]"`.).  <br/> |
| `-` <br/> |Указывает диапазон символов (например, `"[a-z]"`).  <br/> |
| `~` <br/> |Указывает, что эти символы не разрешены (например, `"[~0-9]")`. <br/>|   
| `\` <br/> |Используется для заключения в кавычки любого из предыдущих символов (например `"[\-\\\[\]]"` , допустимые \, символы-, [, и]).  <br/> |
   
**Улитемид**
  
> Значение, идентифицирующее элемент управления в ресурсе диалогового окна. Для элементов управления страниц с вкладками типа ДТКТ_ПАЖЕ элемент **улитемид** (необязательно) используется для загрузки имени компонента страницы из строкового ресурса. Сведения о положении и метке считываются из диалогового окна ресурс. 
    
**доверен**
  
> Структура, содержащая данные для элемента управления и соответствующая свойству **пр_контрол_структуре** ([PidTagControlStructure](pidtagcontrolstructure-canonical-property.md)) элемента управления. Каждый тип элемента управления имеет разную структуру.
    
## <a name="remarks"></a>Примечания

Структура **дтктл** описывает один элемент управления любого типа. Большинство его членов используются для задания свойств элемента управления. 
  
Элемент **CTL** является объединением структур, которые относятся к определенному типу элементов управления. Если структура **дтктл** описывает элемент управления для редактирования, например, элемент **CTL** будет указывает на структуру [дтбледит](dtbledit.md) . Эта структура соответствует свойству **пр_контрол_структуре** элемента управления. Union имеет в качестве первого члена переменную типа ЛПВОИД, чтобы разрешить инициализацию времени компиляции структуры **дтктл** . 
  
Несмотря на то, что функция [буилддисплайтабле](builddisplaytable.md) использует структуру **дтктл** для создания таблицы отображения из ресурсов управления, структура **дтктл** никогда не отображается в самой таблице. Эта структура просто предоставляет информацию в **буилддисплайтабле**.
  
В элементе **улктлфлагс** четыре флага ДТ_АКЦЕПТ_ДБКС, Дт_едитабле, дт_мултилине_анд дт_пассворд_едит влияют только на элементы управления редактированием. Два других ДТ_РЕКУИРЕД и ДТ_СЕТ_ИММЕДИАТЕ влияют на любой редактируемый элемент управления. 
  
Элементы управления, доступные для диалогового окна: Label, текстовое поле, текстовое поле с поддержкой рукописного ввода, список, раскрывающийся список, поле со списком, флажок, группа, кнопка, переключатель и страница с вкладками.
  
Общие сведения о отображаемых таблицах приведены в разделе [Display Tables](display-tables.md). Сведения о том, как реализовать таблицу отображения, можно найти [в статье реализация таблицы отображения](display-table-implementation.md).
