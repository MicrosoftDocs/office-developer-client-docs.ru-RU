# <a name="imapicontainersetsearchcriteria"></a>IMAPIContainer::SetSearchCriteria

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Устанавливает критерии поиска для контейнера.
  
```cpp
HRESULT SetSearchCriteria(
  LPSRestriction lpRestriction,
  LPENTRYLIST lpContainerList,
  ULONG ulSearchFlags
);
```

## <a name="parameters"></a>Параметры

 _Лпрестриктион_
  
> возврата Указатель на структуру [срестриктион](srestriction.md) , которая определяет критерии поиска. Если в параметре _лпрестриктион_ переДАЕТСЯ значение null, то условия поиска, использовавшиеся последними для этого контейнера, повторно используются. ЗНАЧЕНИЕ NULL не должно передаваться в _лпрестриктион_ для первого поиска в контейнере. 
    
 _Лпконтаинерлист_
  
> возврата Указатель на массив идентификаторов записей, представляющих контейнеры, которые необходимо включить в поиск. Если клиент передает значение NULL в параметр _лпконтаинерлист_ , для нового поиска используются идентификаторы записей, использовавшиеся последними для поиска в этом контейнере. Клиент не должен передавать значение NULL в _лпконтаинерлист_ для первого поиска в контейнере. 
    
 _Улсеарчфлагс_
  
> возврата Битовая маска флагов, которые управляют выполнением поиска. Можно задать следующие флаги:
    
БАККГРАУНД_СЕАРЧ 
  
> Поиск должен выполняться с обычным приоритетом относительно других поисков. Этот флаг не может быть установлен одновременно с флагом ФОРЕГРАУНД_СЕАРЧ.
    
ФОРЕГРАУНД_СЕАРЧ 
  
> Поиск должен выполняться с высоким приоритетом, связанным с другими поисковыми операциями. Этот флаг не может быть установлен одновременно с флагом БАККГРАУНД_СЕАРЧ.
    
НОН_КОНТЕНТ_ИНДЕКСЕД_СЕАРЧ
  
> Для поиска совпадений не следует использовать индексирование контента. Этот флаг действителен только для хранилищ Exchange.
    
РЕКУРСИВЕ_СЕАРЧ 
  
> Поиск должен включать в себя контейнеры, указанные в параметре _лпконтаинерлист_ , и все дочерние контейнеры. Этот флаг не может быть установлен одновременно с флагом ШАЛЛОВ_СЕАРЧ. 
    
РЕСТАРТ_СЕАРЧ 
  
> Поиск должен быть инициирован при первом вызове **сетсеарчкритериа**или перезапуске, если поиск неактивен. Этот флаг не может быть установлен одновременно с флагом СТОП_СЕАРЧ.
    
ШАЛЛОВ_СЕАРЧ 
  
> Поиск должен искать только в контейнерах, указанных в параметре _лпконтаинерлист_ для совпадений записей. Этот флаг не может быть установлен одновременно с флагом РЕКУРСИВЕ_СЕАРЧ. 
    
СТОП_СЕАРЧ 
  
> Поиск должен быть остановлен. Этот флаг не может быть установлен одновременно с флагом РЕСТАРТ_СЕАРЧ.
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Условия поиска успешно заданы.
    
МАПИ_Е_ТУ_КОМПЛЕКС 
  
> Поставщик услуг не поддерживает указанные критерии поиска.
    
## <a name="remarks"></a>Примечания

Метод **IMAPIContainer:: сетсеарчкритериа** устанавливает критерии поиска для контейнера, поддерживающего Поиск, как правило, папка результатов поиска. В папке результатов поиска содержатся ссылки на сообщения, соответствующие условиям поиска; фактические сообщения по-прежнему хранятся в исходных расположениях. Единственными уникальными данными, которые хранятся в папке результатов поиска, является его таблица содержимого. В таблице содержимого папки результатов поиска имеется объединенное содержимое хранилища сообщений после применения ограничения поиска. 
  
Операция поиска работает только в этой таблице Объединенных оглавлений; Он не выполняет поиск в других папках результатов поиска. Результаты поиска возвращают только те сообщения, которые отвечают условиям поиска; Иерархия папок не возвращается.
  
Управление возвращается клиенту по завершении поиска.
  
## <a name="notes-to-implementers"></a>Примечания для исполнителей

Контейнеры адресных книг установите условия поиска, применяя ограничения к их таблицам содержимого. Для получения дополнительных сведений о критериях поиска и контейнерах адресной книги, ознакомьтесь со статьей [Реализация расширенНого поиска](implementing-advanced-searching.md).
  
Вы должны поддерживать операции по открытию, копированию, перемещению и удалению сообщений в папках результатов поиска, а не в самой папке результатов поиска. Не разрешать создание сообщений в папке результатов поиска или копирование в нее. 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

Чтобы найти получателей сообщения, задайте для параметра _лпрестриктион_ значение, которое указывает на ограничение на вложенный объект, с элементом **Улсубобжект** в структуре [ссубрестриктион](ssubrestriction.md) , равным **пр_мессаже_реЦипиентс** ([PidTagMessageRecipients](pidtagmessagerecipients-canonical-property.md)). Чтобы выполнить поиск вложений, установите для члена **улсубобжект** значение **пр_мессаже_аттачментс** ([PidTagMessageAttachments](pidtagmessageattachments-canonical-property.md)). ПриСвойте элементу **лпрес** ограничение свойства, которое описывает критерии поиска для получателей или вложений. 
  
Например, чтобы найти вложенные файлы с расширением MSS, задайте для параметра **улсубобжект** значение **пр_мессаже_аттачментс** и **лпрес** в ограничении свойств, которое соответствует **пр_аттач_екстенсион** ([PidTagAttachExtension](pidtagattachextension-canonical-property.md) ) с. MSS.
  
Установка флага ФОРЕГРАУНД_СЕАРЧ в параметре _улсеарчфлагс_ может привести к снижению производительности системы. 
  
Вы можете использовать **сетсеарчкритериа** , чтобы изменить условия поиска, выполняемые при поиске. Вы можете указать новые ограничения, новые списки папок для поиска и новый приоритет поиска, например обновление поиска с более высоким приоритетом. Изменения, внесенные в приоритет поиска, не приводят к перезапуску существующего поиска, но другие изменения в условиях поиска могут. 
  
При работе с папкой результатов поиска вы можете либо удалить эту папку, либо оставить ее открытой для последующего использования. Если вы удаляете папку результатов поиска, удаляются только ссылки на сообщения. Фактические сообщения хранятся в их родительских папках. 
  
Для получения дополнительных сведений о папках результатов поиска в разделе [MAPI Search Folders](mapi-search-folders.md).
