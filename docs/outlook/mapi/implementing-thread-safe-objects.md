<span data-ttu-id="ce4bf-105">При непосредственном вызове объектов, возвращаемых из метода интерфейса, поставщик несет ответственность за обеспечение безопасности потоков.</span><span class="sxs-lookup"><span data-stu-id="ce4bf-105">With objects that are returned from interface method calls directly, it is the provider's responsibility to ensure thread-safety.</span></span> <span data-ttu-id="ce4bf-106">При вызове объектов это ответственность клиентского приложения.</span><span class="sxs-lookup"><span data-stu-id="ce4bf-106">With callback objects, it is the client application's responsibility.</span></span>
  
<span data-ttu-id="ce4bf-107">Клиент может реализовать вызов, безопасный для потоковых уведомлений, позвонив в утилиту [MAPI HrThisThreadAdviseSink.](hrthisthreadadvisesink.md)</span><span class="sxs-lookup"><span data-stu-id="ce4bf-107">A client can implement a thread-safe notification callback by calling the MAPI utility [HrThisThreadAdviseSink](hrthisthreadadvisesink.md).</span></span> <span data-ttu-id="ce4bf-108">**HrThisThreadAdviseSink** преобразует непотобезопасную советую погрузиться в безопасный поток.</span><span class="sxs-lookup"><span data-stu-id="ce4bf-108">**HrThisThreadAdviseSink** transforms a non-thread-safe advise sink into a thread-safe one.</span></span> <span data-ttu-id="ce4bf-109">Для отзовов хода выполнения такой утилиты не существует.</span><span class="sxs-lookup"><span data-stu-id="ce4bf-109">For progress callbacks, there is no such utility.</span></span> <span data-ttu-id="ce4bf-110">Клиент может выбрать, использовать объект mapI, безопасный для потоков, или создать его вручную.</span><span class="sxs-lookup"><span data-stu-id="ce4bf-110">A client can choose to use the MAPI thread-safe progress object or create one manually.</span></span> 
  
<span data-ttu-id="ce4bf-111">Объект, безопасный для потоков, может быть или не знать поток.</span><span class="sxs-lookup"><span data-stu-id="ce4bf-111">A thread-safe object might or might not also be thread-aware.</span></span> <span data-ttu-id="ce4bf-112">Объект с учетом потоков поддерживает отдельный контекст для каждого используемого потока.</span><span class="sxs-lookup"><span data-stu-id="ce4bf-112">A thread-aware object maintains a separate context for every thread that is using it.</span></span> <span data-ttu-id="ce4bf-113">Поставщики услуг не обязаны поддерживать осведомленность о потоке в своих объектах, безопасных для потоков, хотя поддержка осведомленности о потоках может быть полезной в некоторых ситуациях.</span><span class="sxs-lookup"><span data-stu-id="ce4bf-113">Service providers are not required to support thread-awareness in their thread-safe objects, although supporting thread-awareness can be useful in some situations.</span></span> <span data-ttu-id="ce4bf-114">Две таблицы MAPI всегда предоставляют собственный контекст по определению.</span><span class="sxs-lookup"><span data-stu-id="ce4bf-114">Two MAPI tables always provide their own context by definition.</span></span> <span data-ttu-id="ce4bf-115">Одна таблица, используемая в разных потоках, не предоставляет и не должна предоставлять уникальный контекст.</span><span class="sxs-lookup"><span data-stu-id="ce4bf-115">One table used on different threads does not and should not provide unique context.</span></span>

[](mapiinit_0.md)</span><span class="sxs-lookup"><span data-stu-id="ce4bf-117">To ensure that notifications arrive on the same thread that was used to call **MAPIInitialize**, a client calls [MAPIInitialize](mapiinitialize.md) and passes zero in the **ulFlags** member of the [MAPIINIT_0](mapiinit_0.md) structure.</span></span> <span data-ttu-id="ce4bf-118">Затем уведомления доставляются во время основного цикла сообщений.</span><span class="sxs-lookup"><span data-stu-id="ce4bf-118">Notifications are then delivered during the main message loop.</span></span>
