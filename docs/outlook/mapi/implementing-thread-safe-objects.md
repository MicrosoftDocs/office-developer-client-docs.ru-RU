<span data-ttu-id="6f639-104">**Область применения**: Outlook 2013 | Outlook 2016</span><span class="sxs-lookup"><span data-stu-id="6f639-104">**Applies to**: Outlook 2013 | Outlook 2016</span></span> 
  
<span data-ttu-id="6f639-105">С объектами, возвращаемыми непосредственно из вызовов метода интерфейса, это ответственность поставщика на обеспечение безопасности потоков.</span><span class="sxs-lookup"><span data-stu-id="6f639-105">With objects that are returned from interface method calls directly, it is the provider's responsibility to ensure thread-safety.</span></span> <span data-ttu-id="6f639-106">С объектами обратного вызова это ответственность за клиентское приложение.</span><span class="sxs-lookup"><span data-stu-id="6f639-106">With callback objects, it is the client application's responsibility.</span></span>
  
<span data-ttu-id="6f639-107">Клиент может реализовать безопасный обратный вызов уведомлений, вызвав служебную программу MAPI [хрсиссреададвисесинк](hrthisthreadadvisesink.md).</span><span class="sxs-lookup"><span data-stu-id="6f639-107">A client can implement a thread-safe notification callback by calling the MAPI utility [HrThisThreadAdviseSink](hrthisthreadadvisesink.md).</span></span> <span data-ttu-id="6f639-108">**Хрсиссреададвисесинк** преобразует приемник уведомлений без посторонних потоков в потокобезопасный один.</span><span class="sxs-lookup"><span data-stu-id="6f639-108">**HrThisThreadAdviseSink** transforms a non-thread-safe advise sink into a thread-safe one.</span></span> <span data-ttu-id="6f639-109">Для обратных вызовов выполнения такая служебная программа отсутствует.</span><span class="sxs-lookup"><span data-stu-id="6f639-109">For progress callbacks, there is no such utility.</span></span> <span data-ttu-id="6f639-110">Клиент может выбрать, использовать ли потоковый объект степени безопасности MAPI, или создать его вручную.</span><span class="sxs-lookup"><span data-stu-id="6f639-110">A client can choose to use the MAPI thread-safe progress object or create one manually.</span></span> 
  
<span data-ttu-id="6f639-111">Потокобезопасный объект также может поддерживать поток.</span><span class="sxs-lookup"><span data-stu-id="6f639-111">A thread-safe object might or might not also be thread-aware.</span></span> <span data-ttu-id="6f639-112">Объект, поддерживающий цепочку, поддерживает отдельный контекст для каждого потока, который использует его.</span><span class="sxs-lookup"><span data-stu-id="6f639-112">A thread-aware object maintains a separate context for every thread that is using it.</span></span> <span data-ttu-id="6f639-113">Поставщики услуг не обязаны поддерживать поддержку потоков в потокобезопасных объектах, хотя поддержка обработки потоков может быть полезна в некоторых ситуациях.</span><span class="sxs-lookup"><span data-stu-id="6f639-113">Service providers are not required to support thread-awareness in their thread-safe objects, although supporting thread-awareness can be useful in some situations.</span></span> <span data-ttu-id="6f639-114">Две таблицы MAPI всегда предоставляют собственный контекст по определению.</span><span class="sxs-lookup"><span data-stu-id="6f639-114">Two MAPI tables always provide their own context by definition.</span></span> <span data-ttu-id="6f639-115">Одна таблица, используемая в разных потоках, не должна обеспечивать уникальный контекст.</span><span class="sxs-lookup"><span data-stu-id="6f639-115">One table used on different threads does not and should not provide unique context.</span></span>
  
<span data-ttu-id="6f639-116">Клиент может выбрать между получением уведомлений в том же потоке, который использовался для вызова **мапиинитиализе** , в том же потоке, который использовался для вызова **advise** , или в отдельном потоке, принадлежащем MAPI.</span><span class="sxs-lookup"><span data-stu-id="6f639-116">A client can choose between receiving notifications on the same thread that was used for the **MAPIInitialize** call, on the same thread that was used for the **Advise** call, or on a separate thread owned by MAPI.</span></span> <span data-ttu-id="6f639-117">Чтобы гарантировать, что уведомления приходятся на тот же поток, который использовался для вызова **мапиинитиализе**, клиент вызывает [мапиинитиализе](mapiinitialize.md) и передает ноль в элементе **ulFlags** структуры [MAPIINIT_0](mapiinit_0.md) .</span><span class="sxs-lookup"><span data-stu-id="6f639-117">To ensure that notifications arrive on the same thread that was used to call **MAPIInitialize**, a client calls [MAPIInitialize](mapiinitialize.md) and passes zero in the **ulFlags** member of the [MAPIINIT_0](mapiinit_0.md) structure.</span></span> <span data-ttu-id="6f639-118">Затем уведомления доставляются в рамках основного цикла сообщений.</span><span class="sxs-lookup"><span data-stu-id="6f639-118">Notifications are then delivered during the main message loop.</span></span> 
  
<span data-ttu-id="6f639-119">Чтобы получать уведомления о потоке, принадлежащем MAPI, клиент вызывает **мапиинитиализе** с элементом **ulFlags** структуры **MAPIINIT_0** , равным мапи_мултисреад_нотификатионс.</span><span class="sxs-lookup"><span data-stu-id="6f639-119">To receive notifications on the MAPI-owned thread, a client calls **MAPIInitialize** with the **ulFlags** member of the **MAPIINIT_0** structure set to MAPI_MULTITHREAD_NOTIFICATIONS.</span></span> <span data-ttu-id="6f639-120">Вызов **advise** осуществляется с использованием объекта приемника уведомлений клиента, а не упакованной версии.</span><span class="sxs-lookup"><span data-stu-id="6f639-120">The **Advise** call is made with the client's advise sink object rather than a wrapped version.</span></span> 
  
<span data-ttu-id="6f639-121">Чтобы убедиться, что уведомления приходят в том же потоке, который использовался для вызова **advise**, клиент вызывает [хрсиссреададвисесинк](hrthisthreadadvisesink.md) и передает новый созданный приемник оповещений \*\*\*\* , а не исходный приемник уведомлений.</span><span class="sxs-lookup"><span data-stu-id="6f639-121">To ensure that notifications arrive on the same thread that was used to call **Advise**, a client calls [HrThisThreadAdviseSink](hrthisthreadadvisesink.md) and passes the newly created wrapped advise sink to **Advise** rather than the original advise sink.</span></span> <span data-ttu-id="6f639-122">**Мапиинитиализе** можно вызывать с любым значением флага.</span><span class="sxs-lookup"><span data-stu-id="6f639-122">**MAPIInitialize** can be called with either flag value.</span></span>
