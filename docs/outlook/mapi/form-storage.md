<span data-ttu-id="e62f5-105">Хотя не обязательно знать все сведения о том, как физически хранятся формы, полезно понять некоторые из основных понятий.</span><span class="sxs-lookup"><span data-stu-id="e62f5-105">Although it is not necessary to know all the details of how forms are physically stored, it is useful to understand a few of the main concepts.</span></span> <span data-ttu-id="e62f5-106">Поэтому перед описанием трех типов библиотек форм, поддерживаемых менеджером форм по умолчанию, в этом разделе приводится обзор того, как хранятся формы.</span><span class="sxs-lookup"><span data-stu-id="e62f5-106">Therefore, before describing the three types of form libraries supported by the default form manager, this topic gives an overview of how forms are stored.</span></span>
  
<span data-ttu-id="e62f5-107">Определения форм можно физически хранить в папках в одном или нескольких хранилищах сообщений MAPI.</span><span class="sxs-lookup"><span data-stu-id="e62f5-107">Form definitions can be physically stored within folders in one or more MAPI message stores.</span></span> <span data-ttu-id="e62f5-108">Каждая папка MAPI может иметь две области для хранения объектов сообщений: стандартная часть и связанная часть.</span><span class="sxs-lookup"><span data-stu-id="e62f5-108">Every MAPI folder can be thought of as having two areas for storing message objects: the standard part and the associated part.</span></span> <span data-ttu-id="e62f5-109">Стандартная часть папки включает сообщения и папки, которые пользователи манипулируют.</span><span class="sxs-lookup"><span data-stu-id="e62f5-109">The standard part of the folder includes the messages and folders that users manipulate.</span></span>
  
<span data-ttu-id="e62f5-110">Связанная часть включает скрытые объекты сообщений, связанные с папкой, включая определения форм, представления, шаблоны правил, шаблоны ответов и т. д.</span><span class="sxs-lookup"><span data-stu-id="e62f5-110">The associated part includes hidden message objects that are associated with the folder, including form definitions, views, rule templates, reply templates, and so on.</span></span> <span data-ttu-id="e62f5-111">Эта альтернативная часть называется таблицей содержимого, связанной с папками, а набор сообщений в связанной таблице содержимого называется информацией, связанной с папками.</span><span class="sxs-lookup"><span data-stu-id="e62f5-111">This alternate part is called the folder-associated contents table, and the set of messages in со стандартным содержимым папки при копировании папки.</span><span class="sxs-lookup"><span data-stu-id="e62f5-112">The hidden messages are an integral part of the folder and are copied along with the standard folder contents when the folder is copied.</span></span> <span data-ttu-id="e62f5-113">Хотя физически хранятся в качестве сообщений, информация в связанной таблице содержимого папки ведет настраиваемые формы.</span><span class="sxs-lookup"><span data-stu-id="e62f5-114">Any folder object that supports an associated contents table is capable of storing custom forms.</span></span> <span data-ttu-id="e62f5-115">Метод [IMAPIContainer::GetContentsTable](imapicontainer-getcontentstable.md) может возвращать стандартное содержимое или связанное содержимое папки в зависимости от значения  _параметра ulflags_ метода.</span><span class="sxs-lookup"><span data-stu-id="e62f5-115">The [IMAPIContainer::GetContentsTable](imapicontainer-getcontentstable.md) method can return either the <span data-ttu-id="e62f5-116">Библиотека форм состоит из определений форм, хранимых в связанной таблице содержимого папки.</span><span class="sxs-lookup"><span data-stu-id="e62f5-116">A form library consists of form definitions stored in a folder's associated contents table.</span></span> <span data-ttu-id="e62f5-117">Определение формы включает свойства формы, действия, поддерживаемые формой, и даже исполняемый файл сервера форм, который хранится в виде одного или более вложений сообщений.</span><span class="sxs-lookup"><span data-stu-id="e62f5-117">The form definition includes the form's properties, the actions the form supports, and even the form server executable file, which is stored as one or more message attachments.</span></span>
  
<span data-ttu-id="e62f5-118">Кроме того, формы можно хранить в любом файле или расположении, которое поддерживает используемый менеджер форм.</span><span class="sxs-lookup"><span data-stu-id="e62f5-118">Additionally, forms can be stored in any file or location that the form manager being used supports.</span></span> <span data-ttu-id="e62f5-119">Менеджер форм по умолчанию хранит серверы форм в папках MAPI, но настраиваемый диспетчер форм может реализовать собственное хранилище для серверов форм.</span><span class="sxs-lookup"><span data-stu-id="e62f5-119">The default form manager stores form servers in MAPI folders, but a custom form manager could implement its own storage for form servers.</span></span>
  
<span data-ttu-id="e62f5-120">Форма может иметь несколько пользовательских интерфейсов, привязанных к классу сообщений.</span><span class="sxs-lookup"><span data-stu-id="e62f5-120">A form can have multiple user interfaces that are bound to its message class.</span></span> <span data-ttu-id="e62f5-121">Например, форма может иметь отдельные пользовательские интерфейсы Compose и Read.</span><span class="sxs-lookup"><span data-stu-id="e62f5-121">For example, a form can have separate Compose and Read user interfaces.</span></span> <span data-ttu-id="e62f5-122">Форма требует запроса соответствующего пользовательского интерфейса для различных запросов пользователей в зависимости от того, какой из глаголов формы называется.</span><span class="sxs-lookup"><span data-stu-id="e62f5-122">The form takes care of invoking the proper user interface for different user requests, depending on which of the form's verbs is being called.</span></span> <span data-ttu-id="e62f5-123">Например, если на сервере формы есть отдельные пользовательские интерфейсы для создания и чтения, пользовательский интерфейс Compose может быть открыт автоматически, когда пользователь создает новое сообщение класса сообщений формы, а пользовательский интерфейс Read может быть открыт автоматически, когда пользователь откроет существующее сообщение класса сообщений формы.</span><span class="sxs-lookup"><span data-stu-id="e62f5-123">For example, if your form server has separate composing and reading user interfaces, the Compose user interface can be opened automatically when the user creates a new message of the form's message class and the Read user interface can be opened automatically when the user opens an existing message of the form's message class.</span></span>
  
<span data-ttu-id="e62f5-124">Большая часть сведений, хранимых в определении формы, доступна с помощью метода [IMAPIFormInfo::IMAPIProp](imapiforminfoimapiprop.md) на **объекте IMAPIFormInfo.**</span><span class="sxs-lookup"><span data-stu-id="e62f5-124">Most of the information stored within a form definition is available by invoking the [IMAPIFormInfo::IMAPIProp](imapiforminfoimapiprop.md) method on an **IMAPIFormInfo** object.</span></span> <span data-ttu-id="e62f5-125">Интерфейс **IMAPIFormInfo** упрощает доступ к информации формы, вызывая все папки MAPI и методы сообщений, необходимые для получения информации.</span><span class="sxs-lookup"><span data-stu-id="e62f5-125">The **IMAPIFormInfo** interface simplifies access to form information by calling all the MAPI folder and message methods needed to retrieve the information.</span></span> <span data-ttu-id="e62f5-126">Объект **IMAPIFormInfo** можно получить, позвонив [методу IMAPIFormContainer::ResolveMessageClass.](imapiformcontainer-resolvemessageclass.md)</span><span class="sxs-lookup"><span data-stu-id="e62f5-126">An **IMAPIFormInfo** object can be obtained by calling the [IMAPIFormContainer::ResolveMessageClass](imapiformcontainer-resolvemessageclass.md) method.</span></span>
