<span data-ttu-id="b6dbe-105">Поставщики адресных книг несут ответственность за поддержку разрешения имен — процесс связывания идентификатора записи с отображаемым именем.</span><span class="sxs-lookup"><span data-stu-id="b6dbe-105">Address book providers are responsible for supporting name resolution — the process of associating an entry identifier with a display name.</span></span> <span data-ttu-id="b6dbe-106">Клиенты инициируют разрешение имен при вызове [IAddrBook:: ресолвенаме](iaddrbook-resolvename.md) , чтобы убедиться, что каждый участник списка получателей исходящих сообщений соответствует действительному адресу.</span><span class="sxs-lookup"><span data-stu-id="b6dbe-106">Clients initiate name resolution when they call [IAddrBook::ResolveName](iaddrbook-resolvename.md) to ensure that each member of an outgoing message's recipient list corresponds to a valid address.</span></span> 
  
<span data-ttu-id="b6dbe-107">Поставщик может поддерживать разрешение имен следующим образом:</span><span class="sxs-lookup"><span data-stu-id="b6dbe-107">Your provider can support name resolution by:</span></span>
  
- <span data-ttu-id="b6dbe-108">Поддержка ограничения свойства **пр_анр** ([PidTagAnr](pidtaganr-canonical-property.md)), необходимое для всех контейнеров адресных книг.</span><span class="sxs-lookup"><span data-stu-id="b6dbe-108">Supporting the **PR_ANR** ([PidTagAnr](pidtaganr-canonical-property.md)) property restriction, a requirement for all address book containers.</span></span>
    
- <span data-ttu-id="b6dbe-109">Реализация метода [иабконтаинер:: ResolveNames](iabcontainer-resolvenames.md) , который является параметром для всех контейнеров адресных книг.</span><span class="sxs-lookup"><span data-stu-id="b6dbe-109">Implementing the [IABContainer::ResolveNames](iabcontainer-resolvenames.md) method, an option for all address book containers.</span></span> 
    
<span data-ttu-id="b6dbe-110">Если вы решили поддерживать **иабконтаинер:: ResolveNames**, попытайтесь найти точное совпадение для каждого неразрешенного отображаемого имени в структуре [ADRLIST](adrlist.md) , переДанной с параметром _лпадрлист_ .</span><span class="sxs-lookup"><span data-stu-id="b6dbe-110">If you choose to support **IABContainer::ResolveNames**, attempt to locate an exact match for each unresolved display name in the [ADRLIST](adrlist.md) structure passed in with the  _lpAdrList_ parameter.</span></span> <span data-ttu-id="b6dbe-111">Можно идентифицировать неразрешенное отображаемое имя, так как в нем отсутствует свойство **пр_ентрид** ([PidTagEntryId](pidtagentryid-canonical-property.md)) в массиве значений свойства в его элементе **аентриес** структуры **ADRLIST** .</span><span class="sxs-lookup"><span data-stu-id="b6dbe-111">You can identifiy an unresolved display name because it is missing the **PR_ENTRYID** ([PidTagEntryId](pidtagentryid-canonical-property.md)) property in the property value array in its **aEntries** member of the **ADRLIST** structure.</span></span> <span data-ttu-id="b6dbe-112">Игнорируйте все записи, связанные с нулевыми свойствами.</span><span class="sxs-lookup"><span data-stu-id="b6dbe-112">Ignore any entries that have zero properties associated with them.</span></span> 
  
<span data-ttu-id="b6dbe-113">Сообщите результат попытки при разрешении в параметре _лпфлаглист_ , массив флагов, соответствующий массиву отображаемых имен в _лпадрлист_.</span><span class="sxs-lookup"><span data-stu-id="b6dbe-113">Report the result of your attempt at resolution in the  _lpFlagList_ parameter, an array of flags that corresponds to the array of display names in  _lpAdrList_.</span></span> <span data-ttu-id="b6dbe-114">Флаги располагаются таким образом, что первый флаг соответствует первому элементу **аентриес** в структуре **ADRLIST** , второй флаг соответствует второму элементу **аентриес** и т. д.</span><span class="sxs-lookup"><span data-stu-id="b6dbe-114">The flags are positional such that the first flag corresponds to the first **aEntries** member in the **ADRLIST** structure, the second flag corresponds to the second **aEntries** member, and so on.</span></span>

- <span data-ttu-id="b6dbe-118">Можно найти несколько совпадений, то есть несколько записей контейнера, соответствующих записи в структуре **ADRLIST** .</span><span class="sxs-lookup"><span data-stu-id="b6dbe-118">Several matches can be found, meaning that there are multiple container entries that match the entry in the **ADRLIST** structure.</span></span> <span data-ttu-id="b6dbe-119">Установите для соответствующей записи в параметре _лпфлаглист_ значение мапи_амбигуаус.</span><span class="sxs-lookup"><span data-stu-id="b6dbe-119">Set the corresponding entry in the  _lpFlagList_ parameter to MAPI_AMBIGUOUS.</span></span> <span data-ttu-id="b6dbe-120">Не изменяйте число записей в структуре **ADRLIST** .</span><span class="sxs-lookup"><span data-stu-id="b6dbe-120">Do not change the number of entries in the **ADRLIST** structure.</span></span> 
    
- <span data-ttu-id="b6dbe-121">Можно найти точное совпадение, то есть только одна запись контейнера, соответствующая записи в структуре **ADRLIST** .</span><span class="sxs-lookup"><span data-stu-id="b6dbe-121">An exact match can be found, meaning that there is only one container entry that matches the entry in the **ADRLIST** structure.</span></span> <span data-ttu-id="b6dbe-122">Задайте для соответствующего элемента в параметре _лпфлаглист_ значение мапи_ресолвед и добавьте идентификатор записи в массив свойств, связанных с записью **ADRLIST** .</span><span class="sxs-lookup"><span data-stu-id="b6dbe-122">Set the corresponding member in the  _lpFlagList_ parameter to MAPI_RESOLVED and add the entry identifier to the array of properties associated with the **ADRLIST** entry.</span></span> 
    
<span data-ttu-id="b6dbe-123">Если вы не поддерживаете **иабконтаинер:: ResolveNames**, возвращайте мапи_е_но_суппорт из реализации.</span><span class="sxs-lookup"><span data-stu-id="b6dbe-123">If you choose not to support **IABContainer::ResolveNames**, return MAPI_E_NO_SUPPORT from your implementation.</span></span>
