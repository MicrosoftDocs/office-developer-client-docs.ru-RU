# <a name="implementing-name-resolution"></a><span data-ttu-id="0e9cb-103">Реализация разрешения имен</span><span class="sxs-lookup"><span data-stu-id="0e9cb-103">Implementing Name Resolution</span></span>

  
  
<span data-ttu-id="0e9cb-104">**Область применения**: Outlook 2013 | Outlook 2016</span><span class="sxs-lookup"><span data-stu-id="0e9cb-104">**Applies to**: Outlook 2013 | Outlook 2016</span></span> 
  
<span data-ttu-id="0e9cb-105">Поставщики адресных книг отвечают за поддержку разрешения имен — процесс связывать идентификатор записи с отображающим именем.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-105">Address book providers are responsible for supporting name resolution — the process of associating an entry identifier with a display name.</span></span> <span data-ttu-id="0e9cb-106">Клиенты инициируют разрешение имен при вызове [IAddrBook::ResolveName,](iaddrbook-resolvename.md) чтобы убедиться, что каждый член списка получателей исходярия сообщения соответствует допустимым адресам.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-106">Clients initiate name resolution when they call [IAddrBook::ResolveName](iaddrbook-resolvename.md) to ensure that each member of an outgoing message's recipient list corresponds to a valid address.</span></span> 
  
<span data-ttu-id="0e9cb-107">Поставщик может поддерживать разрешение имен с помощью:</span><span class="sxs-lookup"><span data-stu-id="0e9cb-107">Your provider can support name resolution by:</span></span>
  
- <span data-ttu-id="0e9cb-108">Поддержка **ограничения свойств PR_ANR** [(PidTagAnr),](pidtaganr-canonical-property.md)необходимого для всех контейнеров адресной книги.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-108">Supporting the **PR_ANR** ([PidTagAnr](pidtaganr-canonical-property.md)) property restriction, a requirement for all address book containers.</span></span>
    
- <span data-ttu-id="0e9cb-109">Реализация метода [IABContainer::ResolveNames,](iabcontainer-resolvenames.md) параметра для всех контейнеров адресной книги.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-109">Implementing the [IABContainer::ResolveNames](iabcontainer-resolvenames.md) method, an option for all address book containers.</span></span> 
    
<span data-ttu-id="0e9cb-110">Если вы решите поддерживать **IABContainer::ResolveNames,** попытается найти точное совпадение для каждого неурегулированного имени отображения в структуре [ADRLIST,](adrlist.md) переданной с параметром _lpAdrList._</span><span class="sxs-lookup"><span data-stu-id="0e9cb-110">If you choose to support **IABContainer::ResolveNames**, attempt to locate an exact match for each unresolved display name in the [ADRLIST](adrlist.md) structure passed in with the  _lpAdrList_ parameter.</span></span> <span data-ttu-id="0e9cb-111">Вы можете фиксировать неурегулированное имя **отображения,** так как отсутствует свойство [PR_ENTRYID (PidTagEntryId)](pidtagentryid-canonical-property.md)в массиве значений свойств в его члене **aEntries** структуры **ADRLIST.**</span><span class="sxs-lookup"><span data-stu-id="0e9cb-111">You can identifiy an unresolved display name because it is missing the **PR_ENTRYID** ([PidTagEntryId](pidtagentryid-canonical-property.md)) property in the property value array in its **aEntries** member of the **ADRLIST** structure.</span></span> <span data-ttu-id="0e9cb-112">Игнорируйте любые записи с нулевыми свойствами, связанными с ними.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-112">Ignore any entries that have zero properties associated with them.</span></span> 
  
<span data-ttu-id="0e9cb-113">Сообщайте о результатах попытки разрешения в параметре  _lpFlagList_ , массиве флагов, соответствующих массиву имен отображения в  _lpAdrList_.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-113">Report the result of your attempt at resolution in the  _lpFlagList_ parameter, an array of flags that corresponds to the array of display names in  _lpAdrList_.</span></span> <span data-ttu-id="0e9cb-114">Флаги позиционны так, что первый флаг соответствует первому члену **aEntries** в структуре **ADRLIST,** второй флаг соответствует второму члену **aEntries** и так далее.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-114">The flags are positional such that the first flag corresponds to the first **aEntries** member in the **ADRLIST** structure, the second flag corresponds to the second **aEntries** member, and so on.</span></span> 
  
<span data-ttu-id="0e9cb-115">Существует три возможных результата для каждой неурегулированной записи:</span><span class="sxs-lookup"><span data-stu-id="0e9cb-115">There are three possible results for each unresolved entry:</span></span>
  
- <span data-ttu-id="0e9cb-116">Совпадение не обнаружено, что означает, что ни одна из записей в записях контейнера не совпадает с записью **в структуре ADRLIST.**</span><span class="sxs-lookup"><span data-stu-id="0e9cb-116">No match was found, meaning that none of the entries in your container entries match the entry in the **ADRLIST** structure.</span></span> <span data-ttu-id="0e9cb-117">Установите соответствующую запись в  _параметре lpFlagList_ для MAPI_UNRESOLVED.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-117">Set the corresponding entry in the  _lpFlagList_ parameter to MAPI_UNRESOLVED.</span></span> 
    
- <span data-ttu-id="0e9cb-118">Можно найти несколько совпадений, что означает, что существует несколько записей контейнеров, которые соответствуют входу в **структуре ADRLIST.**</span><span class="sxs-lookup"><span data-stu-id="0e9cb-118">Several matches can be found, meaning that there are multiple container entries that match the entry in the **ADRLIST** structure.</span></span> <span data-ttu-id="0e9cb-119">Установите соответствующую запись в  _параметре lpFlagList_ для MAPI_AMBIGUOUS.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-119">Set the corresponding entry in the  _lpFlagList_ parameter to MAPI_AMBIGUOUS.</span></span> <span data-ttu-id="0e9cb-120">Не измените количество записей в **структуре ADRLIST.**</span><span class="sxs-lookup"><span data-stu-id="0e9cb-120">Do not change the number of entries in the **ADRLIST** structure.</span></span> 
    
- <span data-ttu-id="0e9cb-121">Можно найти точное совпадение, что означает, что в структуре **ADRLIST** имеется только одна запись контейнера, которая соответствует записи.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-121">An exact match can be found, meaning that there is only one container entry that matches the entry in the **ADRLIST** structure.</span></span> <span data-ttu-id="0e9cb-122">Установите соответствующий член в параметре _lpFlagList_ для MAPI_RESOLVED и добавьте идентификатор записи в массив свойств, связанных с записью **ADRLIST.**</span><span class="sxs-lookup"><span data-stu-id="0e9cb-122">Set the corresponding member in the  _lpFlagList_ parameter to MAPI_RESOLVED and add the entry identifier to the array of properties associated with the **ADRLIST** entry.</span></span> 
    
<span data-ttu-id="0e9cb-123">Если вы решите не поддерживать **IABContainer::ResolveNames,** MAPI_E_NO_SUPPORT из своей реализации.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-123">If you choose not to support **IABContainer::ResolveNames**, return MAPI_E_NO_SUPPORT from your implementation.</span></span>
  
<span data-ttu-id="0e9cb-124">Все поставщики адресных книг должны поддерживать неоднозначное разрешение имен — ограничение PR_ANR свойств — на столах содержимого контейнеров. </span><span class="sxs-lookup"><span data-stu-id="0e9cb-124">All address book providers are required to support ambiguous name resolution — the **PR_ANR** property restriction — on their containers' contents tables.</span></span> <span data-ttu-id="0e9cb-125">Чтобы обеспечить эту поддержку, выполните PR_ANR ограничения в реализации [IMAPITable:::Restrict,](imapitable-restrict.md) выполняя тип поиска "наилучшего угадать", соотнося с одним или более определенными свойствами, которые имеют смысл для поставщика.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-125">To provide this support, handle the PR_ANR restriction in your implementation of [IMAPITable::Restrict](imapitable-restrict.md) by performing a "best guess" type of search, matching against one or more particular properties that make sense for your provider.</span></span> <span data-ttu-id="0e9cb-126">Вы можете использовать одно и то же свойство или свойства каждый **раз,** например PR_DISPLAY_NAME [(PidTagDisplayName)](pidtagdisplayname-canonical-property.md)или **PR_ACCOUNT** [(PidTagAccount),](pidtagaccount-canonical-property.md)или разрешить администратору выбирать из списка приемлемых свойств.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-126">You can choose to use the same property or properties every time, such as **PR_DISPLAY_NAME** ([PidTagDisplayName](pidtagdisplayname-canonical-property.md)) or **PR_ACCOUNT** ([PidTagAccount](pidtagaccount-canonical-property.md)), or allow an administrator to choose from a list of acceptable properties.</span></span> 
  
<span data-ttu-id="0e9cb-127">Хотя большинство поставщиков поставляют собственную реализацию таблицы контента, можно настроить реализацию, предоставленную MAPI с помощью [функции CreateTable.](createtable.md)</span><span class="sxs-lookup"><span data-stu-id="0e9cb-127">Although most providers supply their own contents table implementation, you can customize the implementation supplied by MAPI through the [CreateTable](createtable.md) function.</span></span> <span data-ttu-id="0e9cb-128">Однако, так как реализация MAPI не поддерживает ограничения любого рода, необходимо создать объект оболочки, чтобы включить настраиваемую версию **Ограничения,** которая перехватывает вызов.</span><span class="sxs-lookup"><span data-stu-id="0e9cb-128">However, because the MAPI implementation does not support restrictions of any kind, you must create a wrapper object to include a customized version of **Restrict** that intercepts the call.</span></span>
