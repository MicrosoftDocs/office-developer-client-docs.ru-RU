# <a name="tablenotification"></a>TABLE_NOTIFICATION

**Область применения**: Outlook 2013 | Outlook 2016 
  
Описание строки в таблице, которые были затронуты некоторые тип события, например, изменения или ошибка. В этом случае уведомление о таблице был создан. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapidefs.h  <br/> |
   
```cpp
typedef struct _TABLE_NOTIFICATION
{
  ULONG ulTableEvent;
  HRESULT hResult;
  SPropValue propIndex;
  SPropValue propPrior;
  SRow row;
} TABLE_NOTIFICATION;

```

## <a name="members"></a>Элементы

**ulTableEvent**
  
> Битовая маска флаги, используемые для представления типа события по таблице. Можно задать следующие флажки:
    
TABLE_CHANGED 
  
> Указывает, что что-то о таблице был изменен на высоком уровне. В таблице состояния — как до события. Это означает, что все свойства **PR_INSTANCE_KEY** ([PidTagInstanceKey](pidtaginstancekey-canonical-property.md)), закладки, текущее расположение и выбранные элементы интерфейса пользователя еще не истек. Это событие обрабатывается путем повторного считывания в таблице. Поставщики услуг, которые не требуется реализовать расширенный таблице уведомлений отправлять события TABLE_CHANGED, а не более подробные события, чтобы указать определенный тип изменения. 
    
TABLE_ERROR 
  
> Обычно во время выполнения асинхронной операции произошла ошибка. Ошибки во время обработки из следующих методов можно создать это событие: 
    
   - [IMAPITable::SortTable](imapitable-sorttable.md)
    
   - [IMAPITable::SetColumns](imapitable-setcolumns.md)
    
   - [IMAPITable::Restrict](imapitable-restrict.md)
    
   После получения события TABLE_ERROR, клиент не может полагаться на точность содержимое таблицы. Кроме того ожидающие уведомления о другие изменения могут быть потеряны. Метод [IMAPITable::GetLastError](imapitable-getlasterror.md) не может предоставить любые дополнительные сведения об ошибке, так как он был создан в некоторых предыдущих точке, не обязательно последнего вызова метода. 
    
TABLE_RELOAD 
  
> Следует перезагрузить данных в таблице. Поставщиков услуг отправить TABLE_RELOAD при, например, базовые данные хранятся в базе данных и базы данных будет заменен. Обработайте это событие, если предполагается, что ничего о таблице не остаются действительными и повторного чтения в таблице. Все закладки, ключи экземпляра, состояние и данные размещения являются недопустимыми.
    
TABLE_RESTRICT_DONE 
  
> Ограничение операции, инициированные с для вызова метода **IMAPITable::Restrict** завершена. 
    
TABLE_ROW_ADDED 
  
> В таблице и соответствующего объекта сохранен добавлен новую строку. События TABLE_ROW_ADDED создаются после вызова метода [IMAPIProp::SaveChanges](imapiprop-savechanges.md) . 
    
TABLE_ROW_DELETED 
  
> Строка удалена из таблицы. Член **propPrior** имеет значение NULL. 
    
TABLE_ROW_MODIFIED 
  
> Строка была изменена. Элемент **строки** содержит указанные свойства для строки. Несколько событий TABLE_ROW_MODIFIED, передаются в том порядке, в котором они отображаются в табличном представлении. 
    
  События TABLE_ROW_MODIFIED отправляются после изменения для соответствующего объекта, выполненные с помощью вызова метода **IMAPIProp::SaveChanges** . Если измененной строки теперь первой строки в таблице, значение свойства тега в элемент **propPrior** — **PR_NULL** ([PidTagNull](pidtagnull-canonical-property.md)).
    
TABLE_SETCOL_DONE 
  
> Завершения операции параметр столбца, инициированных с для вызова метода **IMAPITable::SetColumns** . 
    
TABLE_SORT_DONE 
  
> Таблица сортировки операции, инициированные с для вызова метода **IMAPITable::SortTable** завершена. 
    
**hResult**
  
> Значение HRESULT для произошедшей ошибки, если член **ulTableEvent** задано значение TABLE_ERROR. 
    
**propIndex**
  
> Структура [SPropValue](spropvalue.md) для свойства **PR_INSTANCE_KEY** затронутых строки. 
    
**propPrior**
  
> Структура **SPropValue** для свойства **PR_INSTANCE_KEY** строки перед затронутых проектом. При первой строки в таблице, затронутых строку **propPrior** должен иметь значение **PR_NULL** и не нулю. Нулевой не является допустимым свойством тегом. 
    
**Строка**
  
> Структура [SRow](srow.md) , описывающее затронутых строк. Эта структура заполняется для всех событий уведомлений в таблице. Для события уведомления в таблице, которые не передачи данных строки член **cValues** структуры **SRow** присваивается нулевое значение и член **lpProps** имеет значение NULL. Поскольку эта структура **SRow** доступен только для чтения; Клиенты необходимо принять его копию, если требуется внести изменения. Функция [ScDupPropset](scduppropset.md) можно использовать для создания копии. 
    
## <a name="remarks"></a>Примечания

**Таблица\_уведомления** структуры — это один из участников объединение структуры, включенные в элемент **info** структуры [УВЕДОМЛЕНИЙ](notification.md) . Член **информация** включает в себя **в ТАБЛИЦЕ\_уведомления** структуры, если член **ulEventType** структуры равен _fnevTableModified_.
  
Порядок и типы столбцов в элемент строки отражают порядок и тип, который был фактически во время создания уведомления. Порядок и тип во время создания уведомления не обязательно то же, что при доставке уведомления. 
  
Дополнительные сведения о уведомлений видеть темы, описанные в следующей таблице.

|[Обработка уведомлений](handling-notifications.md) <br/> |Обсуждение как клиенты должны обрабатывать уведомления.  <br/> |

Так как уведомления в таблице является асинхронным, клиенты могут получать уведомления о добавленной строки после обучению работе с добавлением по-другому. Существует возможность получения TABLE_ERROR события, при наличии ошибки в методе **IMAPITable::Sort**, **IMAPITable::Restrict**или **IMAPITable::SetColumns** или при основной процесс пытается обновить таблицу в организации, например, новый или измененные строки.
