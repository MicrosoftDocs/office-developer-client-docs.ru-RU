# <a name="launching-a-form-server"></a>Запуск сервера форм

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Ряд взаимодействий, возникающих при загрузке формы из постоянного хранилища (из библиотеки форм) для отображения сообщения, выглядит следующим образом:
  
1. Клиент обмена сообщениями получает класс сообщений, флаги сообщений и состояние сообщения. Это необязательный шаг; Если эти блоки данных не предоставлены на этапе 2, диспетчер формы извлечет их.
    
2. Клиент обмена сообщениями вызывает [имапиформмгр:: лоадформ](imapiformmgr-loadform.md) с целевым сообщением. 
    
3. Диспетчер форм загружает сервер форм из соответствующей библиотеки форм. Если сервер форм для целевого сообщения не установлен, диспетчер формы также устанавливает исполняемые файлы формы.
    
4. Диспетчер форм вызывает метод [IUnknown:: QueryInterface](https://msdn.microsoft.com/library/54d5ff80-18db-43f2-b636-f93ac053146d%28Office.15%29.aspx) для объекта Form, чтобы получить интерфейс [имапиформ: IUnknown](imapiformiunknown.md) и [иперсистмессаже: IUnknown](ipersistmessageiunknown.md) . 
    
5. Диспетчер форм вызывает [иперсистмессаже:: Load](ipersistmessage-load.md) с узлом сообщения и интерфейсами сообщений из объекта средства просмотра. 
    
6. Объект Form обращается обратно к методу [имапимессажесите:: жетситестатус](imapimessagesite-getsitestatus.md) клиента обмена сообщениями. 
    
7. Диспетчер форм вызывает метод [имапиформ:: сетвиевконтекст](imapiform-setviewcontext.md) объекта Form с интерфейсом контекста представления из клиента обмена сообщениями. 
    
8. Объект Form обращается обратно к методу [имапивиевконтекст:: сетадвисесинк](imapiviewcontext-setadvisesink.md) клиента обмена сообщениями. 
    
9. Объект Form обращается обратно к методу [имапивиевконтекст:: жетвиевстатус](imapiviewcontext-getviewstatus.md) клиента обмена сообщениями. 
    
10. Клиент обмена сообщениями вызывает метод [имапиформ:: Advise](imapiform-advise.md) объекта Form с интерфейсами контекста представления из объекта средства просмотра и объекта сайта сообщения. 
    
11. Клиент обмена сообщениями вызывает метод [имапиформ::D оверб](imapiform-doverb.md) объекта Form. 
    
12. При необходимости объект формы создает его пользовательский интерфейс и взаимодействует с пользователем.
