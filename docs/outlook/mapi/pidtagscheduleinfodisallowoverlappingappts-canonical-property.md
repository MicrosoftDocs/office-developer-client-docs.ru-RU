# <a name="pidtagscheduleinfodisallowoverlappingappts-canonical-property"></a>Каноническое свойство PidTagScheduleInfoDisallowOverlappingAppts

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит значение TRUE, если перекрывающиеся встречи запрещены.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |ПР_СЧДИНФО_ДИСАЛЛОВ_ОВЕРЛАППИНГ_АППТС  <br/> |
|Идентификатор:  <br/> |0x686F  <br/> |
|Тип данных:  <br/> |PT_BOOLEAN  <br/> |
|Область:  <br/> |Сведения о доступности  <br/> |
   
## <a name="remarks"></a>Комментарии

Это свойство имеет смысл только в том случае, если значение свойства **пр_счдинфо_ауто_акцепт_апптс** ([PIDTAGSCHEDULEINFOAUTOACCEPTAPPOINTMENTS](pidtagscheduleinfoautoacceptappointments-canonical-property.md)) равно true. Значение TRUE указывает, что при автоматическом ответе на приглашения на собрания клиент или сервер должен отклонять экземпляры, которые перекрывают ранее запланированные события. Значение FALSE или отсутствие этого свойства указывает на то, что перекрывающиеся экземпляры должны приниматься. Это свойство не является обязательным.
