# <a name="iprovideradmindeleteprovider"></a>IProviderAdmin::DeleteProvider

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Удаляет поставщика услуг из службы сообщений.
  
```cpp
HRESULT DeleteProvider(
  LPMAPIUID lpUID
);
```

## <a name="parameters"></a>Параметры

 _лпуид_
  
> [вход, выход] Указатель на структуру [мапиуид](mapiuid.md) , которая содержит уникальный идентификатор, представляющий поставщика, которого требуется удалить. 
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Поставщик успешно удален из службы сообщений.
    
MAPI_E_NOT_FOUND 
  
> **Мапиуид** , на который указывает параметр _лпуид_ , не распознан. 
    
## <a name="remarks"></a>Примечания

Метод **ипровидерадмин::D елетепровидер** Удаляет поставщика услуг из службы сообщений. **Делетепровидер** определяет поставщика услуг, который необходимо удалить, с помощью соответствующей структуры **мапиуид** , на которую указывает _лпуид_ с набором идентификаторов, зарегистрированных поставщиками активных служб. 
  
Большинство служб сообщений не разрешают удалять поставщиков при использовании профиля. Если поставщик для удаления используется, **делетепровидер** помечает его для удаления, а не немедленно и возвращает S_OK. Когда поставщик больше не используется, он удаляется. 
  
 **Делетепровидер** вызывает функцию точки входа службы сообщений, прежде чем поставщик будет удален из службы. Для параметра _улконтекст_ задано значение MSG_SERVICE_PROVIDER_DELETE. Функция точки входа службы сообщений выполняет следующие задачи: 
  
- Удаляет поставщика услуг.
    
- Удаляет раздел профиля поставщика услуг.
    
Функция точки входа службы сообщений не вызывается повторно после удаления поставщика.
