# <a name="imapipropgetidsfromnames"></a>IMAPIProp::GetIDsFromNames

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Предоставляет идентификаторы свойств, соответствующие одному или более именам свойств.
  
```cpp
HRESULT GetIDsFromNames(
  ULONG cPropNames,
  LPMAPINAMEID FAR * lppPropNames,
  ULONG ulFlags,
  LPSPropTagArray FAR * lppPropTags
);
```

## <a name="parameters"></a>Параметры

 _cPropNames_
  
> [in] Количество имен свойств, на которые указывает параметр _lppPropNames._ Если  _lppPropNames_ имеет NULL, параметр  _cPropNames должен_ иметь 0. 
    
 _lppPropNames_
  
> [in] Указатель на массив имен свойств или NULL. Передача NULL запрашивает идентификаторы свойств для всех имен свойств во всех наборах свойств, сведения о которых есть у объекта. Параметр _lppPropNames_ не должен иметь NULL, если MAPI_CREATE флага в _параметре ulFlags._ 
    
 _ulFlags_
  
> [in] Битоваяmas флагов, которая указывает, как должны возвращаться идентификаторы свойств. Можно установить следующий флаг:
    
MAPI_CREATE 
  
> Назначает идентификатор свойства, если он еще не назначен, одному или несколько имен, включенных в массив имен свойств, на которые указывает _lppPropNames._ Этот флаг регистрирует идентификатор в таблице сопоставления "имя-идентификатор".
    
 _lppPropTags_
  
> [out] Указатель на указатель на массив тегов свойств, содержащий существующие или только что назначенные идентификаторы свойств. Для типов свойств тегов свойств в этом массиве установлено **PT_UNSPECIFIED.**
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Идентификаторы для указанных имен свойств были успешно возвращены.
    
MAPI_E_NO_SUPPORT 
  
> Объект не поддерживает именуемые свойства.
    
MAPI_E_NOT_ENOUGH_MEMORY 
  
> Недостаточно памяти для извлечения идентификаторов.
    
MAPI_E_TOO_BIG 
  
> Операцию невозможно выполнить, так как для ее возврата требуется слишком много тегов свойств.
    
MAPI_W_ERRORS_RETURNED 
  
> Вызов в целом был успешным, но не удалось вернуть один или несколько идентификаторов свойств. Соответствующему типу свойства для каждого недоступного свойства PT_ERROR **идентификатору** — нуль. Когда возвращается это предупреждение, обрабатывают вызов как успешный. Чтобы проверить это предупреждение, используйте **макрос HR_FAILED.** См. ["Использование макроса для обработки ошибок".](using-macros-for-error-handling.md)
    
## <a name="remarks"></a>Примечания

Метод **IMAPIProp::GetIDsFromNames** извлекает массив тегов свойств, в котором идентификаторы свойств для одного или более именуемого свойства. **IMAPIProp::GetIDsFromNames** можно использовать для следующего: 
  
- Создайте идентификаторы для новых имен.
    
- Извлечение идентификаторов для определенных имен.
    
- Извлечение идентификаторов всех именуемой свойства, включенных в сопоставление объекта.
    
Именуемые свойства обычно используются поставщиками хранения сообщений для папок и сообщений. Другие объекты, например пользователи сообщений и разделы профилей, могут не поддерживать связь имен с идентификаторами свойств и возвращать MAPI_E_NO_SUPPORT из **GetIDsFromNames.**
  
При ошибке, возвращаемой идентификатором для определенного имени, **GetIDsFromNames** возвращает MAPI_W_ERRORS_RETURNED и задает тип свойства в записи массива тегов свойств, соответствующей имени **PT_ERROR,** а идентификатору — нулю. 
  
Сопоставление имени и идентификатора представлено свойством PR_MAPPING_SIGNATURE **объекта** ([PidTagMappingSignature).](pidtagmappingsignature-canonical-property.md) **PR_MAPPING_SIGNATURE** содержит структуру [MAPIUID,](mapiuid.md) которая указывает поставщика услуг, ответственного за объект. Если свойство **PR_MAPPING_SIGNATURE** одинаково для двух объектов, предположим, что эти объекты используют одно и то же сопоставление имени с идентификатором. 
  
## <a name="notes-to-implementers"></a>Примечания для исполнителей

Идентификаторы, которые вы передаете обратно в массиве тегов свойств, на которые указывает параметр  _lppPropNames,_ должны 0x8000 в 0xFFFE диапазона. Записи в этом массиве должны быть в том же порядке, что и имена, переданные в массиве имен свойств, на которые указывает _lppPropNames._ 
  
Если вы поддерживаете именуемые свойства в контейнере, используйте одинаковое сопоставление имен с идентификатором для всех объектов в контейнере (то есть не используйте отдельное сопоставление для каждой папки в хранилище сообщений или каждого сообщения в папке).
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

Поскольку типы свойств для возвращаемого идентификатора в массиве тегов свойств, на которые указывает _lppPropTags,_ имеют **PT_UNSPECIFIED,** для получения точных типов необходимо вызвать метод [IMAPIProp::SetProps.](imapiprop-setprops.md) 
  
Если перемещаются или копируют объекты с именоваными свойствами, а исходные и назначения имеют разные подписи **сопоставления,** указанные значениями их свойств PR_MAPPING_SIGNATURE, во время этих операций необходимо сохранить имена. Чтобы сохранить имена свойств, настройте соответствующие идентификаторы свойств в соответствие с сопоставлением имени и идентификатора объекта назначения. 
  
Некоторые объекты имеют ограничение на количество идентификаторов свойств, которые они могут назвать. Если вызов **GetIDsFromNames** приводит к превышению этого ограничения, метод возвращает MAPI_E_TOO_BIG. В этом случае запрос по идентификатору. 
  
Дополнительные сведения см. в [под названием MAPI.](mapi-named-properties.md)
