# <a name="providing-notifications-for-message-store-providers"></a>Предоставление уведомлений для поставщиков store сообщений

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Хотя уведомления являются необязательными, они являются очень важной частью хорошего поставщика store сообщений. Клиентские приложения и пулер MAPI используют уведомления от поставщика store сообщений, чтобы получить хорошую производительность при отправке исходящие сообщения или получении входящих сообщений. Клиенты и пулер MAPI могут работать без получения уведомлений от поставщика store сообщений, но они не смогут сообщить пользователям об изменениях в хранилище сообщений без них. Как правило, это означает, что пользователи не смогут увидеть, что новое сообщение поступило, пока их клиент не откроет папку получения в хранилище сообщений.
  
Клиенты регистрируются для получения уведомлений, вызывая метод [IMsgStore::Advise.](imsgstore-advise.md) Клиент передает интерфейс [IMAPIAdviseSink : IUnknown,](imapiadvisesinkiunknown.md) битовуюmask, которая указывает, какой тип уведомлений клиент заинтересован в получении, и  **EntryID,** который указывает, к какому объекту в хранилище сообщений применяется запрос "Сообщить". Когда в объекте происходят соответствующие события (например, когда новое сообщение поступает в папку получения в хранилище сообщений), поставщик или сам объект должны вызвать метод [IMAPIAdviseSink::OnNotify](imapiadvisesink-onnotify.md) для всех объектов **IMAPIAdviseSink,** зарегистрированных для этого типа события. 
  
Даже если ваш поставщик хранилищ сообщений никогда не сообщает другим компонентам MAPI об изменениях в хранилище сообщений, он по-прежнему должен реализовать **IMsgStore::Advise,** чтобы вернуть MAPI_E_NO_SUPPORT. Это сообщает другим компонентам о том, что не следует ожидать уведомлений от поставщика store сообщений.
