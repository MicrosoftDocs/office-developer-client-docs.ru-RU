# <a name="synchronize-contents-state"></a>Состояние синхронизации контента

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время состояния "Синхронизация контента" для конечного автомата репликации.

|Связанная структура данных:  <br/> |**[SYNCCONT](synccont.md)** <br/> |

> [!NOTE]
> Конечный автомат репликации — это детерминированный конечный автомат. Клиент, который отменяет от одного состояния к другому, должен, в конечном итоге, вернуться к предыдущему из последнего. 
  
## <a name="description"></a>Описание

Это состояние инициирует один из двух процессов репликации: Отправка содержимого указанных папок в локальном хранилище или полная синхронизация. При полной синхронизации для каждой из указанных папок содержимое загружается первыми, а затем загружается. В зависимости от набора *ulFlags* в соответствующей структуре **[синхронизации](sync.md)** в предыдущем состоянии синхронизации Outlook инициализирует члены [out] структуры **синкконт** для предоставления сведений о содержимом. 
  
С помощью той же структуры **синкконт** клиент получает количество папок, содержимое которых будет отправлено или загружено. Клиент получит доступ к каждой из этих папок, переместив локальное хранилище в состояние таблицы отправки для отправки папки или переместив локальное хранилище в состояние загрузки таблицы, чтобы скачать эту папку. 
  
Кроме того, клиент получает идентификаторы записей для папок, требующих репликации.
  
При завершении этого состояния Outlook Очищает внутреннюю информацию. Локальное хранилище возвращается в состояние Synchronize.
