## <a name="remarks"></a><span data-ttu-id="68fe9-144">Замечания</span><span class="sxs-lookup"><span data-stu-id="68fe9-144">Remarks</span></span>

<span data-ttu-id="68fe9-145">Клиенты и поставщики услуг вызовите метод **уведомлений** для регистрации для определенного типа или типов уведомлений на запись адресной книги.</span><span class="sxs-lookup"><span data-stu-id="68fe9-145">Clients and service providers call the **Advise** method to register for a particular type or types of notification on an address book entry.</span></span> <span data-ttu-id="68fe9-146">Типы уведомлений обозначены маска событие, переданное с помощью параметра _ulEventMask_ .</span><span class="sxs-lookup"><span data-stu-id="68fe9-146">The types of notification are indicated by the event mask passed in with the  _ulEventMask_ parameter.</span></span> 
  
<span data-ttu-id="68fe9-147">MAPI переадресует этот вызов **уведомлений** для доступа к адресной книге, который несет ответственность за запись, что указывает идентификатор записи в параметре _lpEntryID_ .</span><span class="sxs-lookup"><span data-stu-id="68fe9-147">MAPI forwards this **Advise** call to the address book provider that is responsible for the entry as indicated by the entry identifier in the  _lpEntryID_ parameter.</span></span> <span data-ttu-id="68fe9-148">Поставщик адресной книги обрабатывает регистрации самого или вызывает метод поддержки [IMAPISupport::Subscribe](imapisupport-subscribe.md), чтобы запрашивать MAPI для регистрации вызывающего абонента.</span><span class="sxs-lookup"><span data-stu-id="68fe9-148">The address book provider either handles the registration itself or calls the support method, [IMAPISupport::Subscribe](imapisupport-subscribe.md), to prompt MAPI to register the caller.</span></span> <span data-ttu-id="68fe9-149">Для представления успешной регистрации возвращается число ненулевое подключения.</span><span class="sxs-lookup"><span data-stu-id="68fe9-149">A nonzero connection number is returned to represent the successful registration.</span></span>
  
<span data-ttu-id="68fe9-150">При каждом изменении записи тип, указанный в параметре регистрации уведомлений, поставщик адресной книги вызывает метод [IMAPIAdviseSink::OnNotify](imapiadvisesink-onnotify.md) для объекта приемник уведомлений, указанный в параметре _lpAdviseSink_ .</span><span class="sxs-lookup"><span data-stu-id="68fe9-150">Whenever a change occurs to the entry of the type indicated by the notification registration, the address book provider calls the [IMAPIAdviseSink::OnNotify](imapiadvisesink-onnotify.md) method for the advise sink object specified in the  _lpAdviseSink_ parameter.</span></span> <span data-ttu-id="68fe9-151">Метод **OnNotify** содержит структуру [УВЕДОМЛЕНИЙ](notification.md) в качестве входного параметра, который содержит данные для описания события.</span><span class="sxs-lookup"><span data-stu-id="68fe9-151">The **OnNotify** method includes a [NOTIFICATION](notification.md) structure as an input parameter that contains data to describe the event.</span></span> 
  
<span data-ttu-id="68fe9-152">В зависимости от доступа к адресной книге вызов **OnNotify** может произойти сразу после изменения зарегистрированных объект или позже.</span><span class="sxs-lookup"><span data-stu-id="68fe9-152">Depending on the address book provider, the call to **OnNotify** can occur immediately following the change to the registered object or at a later time.</span></span> <span data-ttu-id="68fe9-153">На компьютерах, поддерживающих многопоточной среде вызов **OnNotify** может выполняться на любой поток.</span><span class="sxs-lookup"><span data-stu-id="68fe9-153">On systems that support multiple threads of execution, the call to **OnNotify** can occur on any thread.</span></span> <span data-ttu-id="68fe9-154">Клиенты могут запрашивать только такие уведомления на определенного потока с помощью вызова функции [HrThisThreadAdviseSink](hrthisthreadadvisesink.md) для создания объекта приемник уведомлений, который передается в **уведомлений**.</span><span class="sxs-lookup"><span data-stu-id="68fe9-154">Clients can request that these notifications occur on a particular thread by calling the [HrThisThreadAdviseSink](hrthisthreadadvisesink.md) function to create the advise sink object that is passed to **Advise**.</span></span> 
  
<span data-ttu-id="68fe9-155">Так как поставщика адресных книг можно освободить объект приемника уведомлений, переданного с клиентами в любое время после успешного завершения **уведомлений** звонков и перед [IAddrBook::Unadvise](iaddrbook-unadvise.md) звонок, чтобы отменить уведомление, освобождать клиентов их уведомлений приемника объектов при возврате **уведомлений** .</span><span class="sxs-lookup"><span data-stu-id="68fe9-155">Because an address book provider can release the advise sink object passed in by clients at any time after the successful completion of the **Advise** call and before an [IAddrBook::Unadvise](iaddrbook-unadvise.md) call to cancel the notification, clients should release their advise sink objects when **Advise** returns.</span></span>
