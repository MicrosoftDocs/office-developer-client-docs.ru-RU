# <a name="calling-queryrows-for-small-tables"></a>Вызов QueryRows для небольших таблиц

  
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
Во время извлечения строк из небольшая таблица, вызовите [IMAPITable::QueryRows](imapitable-queryrows.md) вместо первого построение ограничение. Создание ограничение влияет на производительность, так как поставщик необходимо сначала создать таблицу, найдите сопоставления строк в исходной таблице и затем скопируйте строки в новую таблицу. Если общее число строк в таблице меньше, чем 100, вероятно эффективнее читать все строки, а затем вызвать [IMAPITable::FindRow](imapitable-findrow.md) , чтобы найти соответствующую строку. Это особенно хорошей стратегией, если эти сведения требуются редко. 
  
Заданное время для использования с ограничением является ограниченных или отфильтрованные сведения будет использован на длительный период времени или частоты использования. Для экземпляра требуемое всегда представление с непрочитанных сообщений, ограничение — это вызов соответствующих прав для использования.
