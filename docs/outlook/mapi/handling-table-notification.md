# <a name="handling-table-notification"></a>Обработка уведомления о таблице

**Область применения**: Outlook 2013 | Outlook 2016 
  
В качестве альтернативы регистрации напрямую с исходным объектом advise, например папкой или пользователем обмена сообщениями, клиент может зарегистрироваться для получения уведомлений по содержимому или таблице иерархии. Отслеживание изменений записей адресной книги, папок и сообщений с помощью таблицы контента или иерархии может быть проще и проще, чем через отдельные объекты. 

Например, вы можете вызвать метод [IMAPITable:: Advise](imapitable-advise.md) для таблицы иерархии папки, чтобы определить, когда изменения происходят в одной из вложенных в нее папок. Если вы поддерживаете Просмотр удаленных сообщений, зарегистрируйте их в таблице состояния, чтобы наблюдать за действиями от поставщиков транспорта и диспетчера очереди MAPI. 
  
Тем не менее, не всегда рекомендуется использовать уведомления таблиц вместо уведомлений объектов. Наблюдение за изменениями в количестве сообщений в папке — это пример того, что клиенту может понадобиться зарегистрировать уведомления объектов в папке, а не в таблице, реализуемой папкой.
