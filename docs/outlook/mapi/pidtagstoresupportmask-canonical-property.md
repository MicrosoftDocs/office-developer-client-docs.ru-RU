# <a name="pidtagstoresupportmask-canonical-property"></a>Каноническое свойство PidTagStoreSupportMask

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит битмаску флагов, которые клиентские приложения запрашивают для определения характеристик магазина сообщений. 
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_STORE_SUPPORT_MASK  <br/> |
|Идентификатор:  <br/> |0x340D  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Разное  <br/> |
   
## <a name="remarks"></a>Примечания

Это свойство раскрывает возможности магазина сообщений для клиентских приложений, которые планируют отправить ему сообщение. Флаги могут поддерживать решения клиента или другого магазина,  например отправку PR_BODY[(PidTagBody)](pidtagbody-canonical-property.md)или только PR_RTF_COMPRESSED[(PidTagRtfCompressed).](pidtagrtfcompressed-canonical-property.md)  Клиент никогда не должен **PR_STORE_SUPPORT_MASK;** попытка установить этот флаг возвращает MAPI_E_COMPUTED. 
  
Один или несколько из следующих флагов  можно установить для PR_STORE_SUPPORT_MASK bitmask: 
  
STORE_ANSI_OK
  
> (131072, 0x00020000) Хранилище сообщений поддерживает свойства, содержащие символы ANSI (8-bit).
    
STORE_ATTACH_OK 
  
> (32, 0x00000020) Хранилище сообщений поддерживает вложения (OLE или non-OLE) для сообщений. 
    
STORE_CATEGORIZE_OK 
  
> (1024, 0x00000400) Хранилище сообщений поддерживает классифицируются представления таблиц. 
    
STORE_CREATE_OK 
  
> (16, 0x00000010) Хранилище сообщений поддерживает создание новых сообщений. 
    
STORE_ENTRYID_UNIQUE 
  
> (1, 0x00000001) Идентификаторы входа для объектов в хранилище сообщений уникальны, то есть никогда не будут повторноиспользовать в течение всего хранения. 
    
STORE_HTML_OK 
  
> (65536, 0x00010000) Хранилище сообщений поддерживает HTML-сообщения, хранимые в **свойстве PR_BODY_HTML** [(PidTagBodyHtml).](pidtagbodyhtml-canonical-property.md) Если в среде разработки используется MAPIDEFS. Вместо H-файла, не STORE_HTML_OK, используйте значение 0x00010000. 
    
STORE_ITEMPROC
  
> (2097152, 0x00200000) В завернутом магазине PST указывается, что при прибытии нового сообщения в магазин магазин выполняет правила и обработку фильтра нежелательной почты в сообщении отдельно. Магазин вызывает [IMAPISupport:::Notify,](imapisupport-notify.md)задав **fnevNewMail** в структуре [УВЕДОМЛЕНий,](notification.md) которая передается в качестве параметра, а затем передает сведения о новом сообщении прослушиваемому клиенту. Впоследствии, когда прослушивающий клиент получает уведомление, к сообщению не применяются правила. 
    
STORE_LOCALSTORE
  
> (524288, 0x00080000) Этот флаг зарезервирован и не должен использоваться.
    
STORE_MODIFY_OK 
  
> (8, 0x00000008) Хранилище сообщений поддерживает изменение существующих сообщений. 
    
STORE_MV_PROPS_OK 
  
> (512, 0x00000200) Хранилище сообщений поддерживает многооценные свойства, гарантирует стабильность порядка значений в многоценном свойстве на протяжении всей операции сохранения и поддерживает моментацию многоценных свойств в таблицах. 
    
STORE_NOTIFY_OK 
  
> (256, 0x00000100) Хранилище сообщений поддерживает уведомления. 
    
STORE_OLE_OK 
  
> (64, 0x00000040) Хранилище сообщений поддерживает вложения OLE. Доступ к данным OLE можно получить с помощью интерфейса **IStorage,** например с помощью **свойства** [PR_ATTACH_DATA_OBJ (PidTagAttachDataObject).](pidtagattachdataobject-canonical-property.md) 
    
STORE_PUBLIC_FOLDERS 
  
> (16384, 0x00004000) Папки в этом магазине являются общедоступными (несколькими пользователями), а не частными (возможно, много экземплярами, но не несколькими пользователями). 
    
STORE_PUSHER_OK
  
> (8388608, 0x00800000) Обработщик протокола MAPI не будет обходить магазин, и магазин несет ответственность за то, чтобы внести изменения через уведомления в индексер для индексации сообщений.
    
STORE_READONLY 
  
> (2, 0x00000002) Все интерфейсы для магазина сообщений имеют уровень доступа только для чтения. 
    
STORE_RESTRICTION_OK 
  
> (4096, 0x00001000) Хранилище сообщений поддерживает ограничения. 
    
STORE_RTF_OK 
  
> (2048, 0x00000800) Хранилище сообщений поддерживает сообщения формата rich Text (RTF), как правило, сжатые, а сам магазин PR_BODY и **PR_RTF_COMPRESSED** синхронизированы.  
    
STORE_RULES_OK
  
> (268435456, 0x10000000) Указывает, что правила должны храниться в этом хранилище PST, даже если это не хранилище по умолчанию. Если **STORE_RULES_OK** используется совместно с **NON_EMS_XP_SAVE,** правила могут запускаться в магазинах PST, не завернутых по умолчанию.
    
STORE_SEARCH_OK 
  
> (4, 0x00000004) Хранилище сообщений поддерживает папки результатов поиска. 
    
STORE_SORT_OK 
  
> (8192, 0x00002000) Хранилище сообщений поддерживает сортировку представлений таблиц. 
    
STORE_SUBMIT_OK 
  
> (128, 0x00000080) Хранилище сообщений поддерживает маркировку сообщения для отправки. 
    
STORE_UNCOMPRESSED_RTF 
  
> (32768, 0x00008000) Хранилище сообщений поддерживает хранение сообщений RTF в некомпрессивной форме. Uncompressed RTF stream is identified by the value **dwMagicUncompressedRTF** in the stream header. Значение **dwMagicUncompressedRTF** определяется в RTFLIB. H-файл. 
    
STORE_UNICODE_OK
  
> (262144, 0x00040000) Указывает, что хранилище сообщений поддерживает хранилище Unicode. Клиент можно проверить наличие флага, чтобы принять решение о запросе или сохранении сведений Юникода в хранилище. 
    
Версия сообщения RTF всегда может храниться, даже если хранилище сообщений не является RTF-известно. Если STORE_RTF_OK не установлен для определенного магазина, клиент, который поддерживает версии RTF, должен сам вызывать функцию  [RTFSync,](rtfsync.md) чтобы синхронизировать версии PR_BODY и **PR_RTF_COMPRESSED** для текстового контента. RTF всегда хранится в **PR_RTF_COMPRESSED,** является ли он фактически сжатым или нет.
