# <a name="form-verbs"></a>Команды форм

**Область применения**: Outlook 2013 | Outlook 2016 
  
Пользовательский интерфейс формы обычно предлагает элементы меню или элементы управления, которые позволяют пользователям принимать какие-то действия с формой. Обработка этих действий пользователя — задача сервера форм. Этот интерфейс реализуется с помощью стандартных API Win32; написание одного так же, как написание других интерфейсов для регулярных программ Win32.
  
Часто действия пользователей связаны с глаголами. Глагол — это имя действия, определенного определенному классу сообщений. Например, **Reply** — это глагол, реализуемый многими серверами форм, каждый из которых может иметь другую интерпретацию этого глагола. Глаголы иногда называют командами. 
  
> [!NOTE]
> Не все элементы меню и элементы управления в форме соответствуют глаголу. Например, кнопка **Отмена** не соответствует глаголу Отмена на сервере форм. Обычно глаголы связаны с действиями, определенными определенному классу сообщений или набору классов сообщений. Хотя различные классы сообщений могут поддерживать различные наборы глаголов, все они поддерживают по крайней мере глагол Open, который отображает пользовательский интерфейс формы и загружает его со значениями свойств сообщения. 
  
Глаголы могут не принимать параметров. Формы, которые экспортируют команды с переменными параметрами, должны использовать механизмы автоматизации.
  
Клиенты могут определить, какие глаголы поддерживаются определенным классом сообщений с помощью метода [IMAPIFormInfo::CalcVerbSet,](imapiforminfo-calcverbset.md) реализуемого менеджером форм MAPI. Руководитель формы получает эти сведения из файла конфигурации формы. Набор глаголов, возвращаемый этим методом, используется клиентом, чтобы показать пользователю, какие команды можно выполнять в сообщении. Например, клиент может позволить пользователям щелкнуть правой кнопкой мыши сообщение для отображения глаголов, применимых к этому сообщению.
