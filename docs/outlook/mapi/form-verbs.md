# <a name="form-verbs"></a>Команды форм

**Относится к**: Outlook 2013 | Outlook 2016 
  
Пользовательский интерфейс формы обычно предлагает элементы меню или элементы управления, которые позволяют пользователям делать какие-либо действия с формой. Обработка этих действий пользователей на сервере форм является задачей сервера форм. Этот интерфейс реализуется с помощью стандартных API Win32; Написание одной такой же, как и написание других интерфейсов для обычных программ Win32.
  
Часто действия пользователя связаны с глаголами. Глагол — это имя действия, которое является специфическим для определенного класса сообщений. Например, **reply** — это глагол, реализованный многими серверами форм, каждый из которых может иметь разную интерпретацию этой команды. Глаголы иногда называют командами. 
  
> [!NOTE]
> Не все элементы меню и элементы управления в форме соответствуют глаголу. Например, кнопка **"Отмена"** не соответствует глаголу "Отмена" на сервере формы. Обычно глаголы связаны с действиями, характерными для определенного класса сообщений или набора классов сообщений. Хотя разные классы сообщений могут поддерживать разные наборы глаголов, все они поддерживают по крайней мере глагол Open, который отображает пользовательский интерфейс формы и загружает его со значениями свойств сообщения. 
  
Глаголы не могут принимать параметры. Формы, которые экспортируют команды с переменными параметрами, должны использовать механизмы автоматизации.
  
Клиенты могут определить, какие команды поддерживаются определенным классом сообщений, с помощью метода [IMAPIFormInfo::CalcVerbSet,](imapiforminfo-calcverbset.md) который реализуется диспетчером форм MAPI. Диспетчер форм получает эти сведения из файла конфигурации формы. Набор глаголов, возвращаемого этим методом, используется клиентом, чтобы показать пользователю, какие команды можно выполнять в сообщении. Например, клиент может позволить пользователям нажать на сообщение правую кнопку мыши, чтобы отобразить команды, применимые к этому сообщению.
