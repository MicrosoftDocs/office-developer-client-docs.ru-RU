# <a name="dtbllbx"></a>DTBLLBX

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Описывает список, который будет использоваться в диалоговом окне, созданном из таблицы отображения.
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |MAPIDEFS. h  <br/> |
   
```cpp
typedef struct _DTBLLBX
{
  ULONG ulFlags;
  ULONG ulPRSetProperty;
  ULONG ulPRTableName;
} DTBLLBX, FAR *LPDTBLLBX

```

## <a name="members"></a>Элементы

 **ulFlags**
  
> Битовая маска флагов, используемая для удаления горизонтальной или вертикальной полосы прокрутки из списка. Можно задать следующие флаги:
    
МАПИ_НО_ХБАР 
  
> В списке не должно отображаться горизонтальная полоса прокрутки.
    
МАПИ_НО_ВБАР 
  
> В списке не должно отображаться вертикальная полоса прокрутки.
    
 **Улпрсетпроперти**
  
> Тег Property для свойства любого типа. Это свойство является одним из столбцов таблицы, определяемой элементом **улпртаблетабле** . 
    
 **Улпртабленаме**
  
> Тег свойства для свойства Table типа ПТ_ОБЖЕКТ, которое можно открыть с помощью вызова **опенпроперти** . Количество столбцов, которые должна иметь таблица, зависит от того, является ли список одним или несколькими вариантами выбора. Если для члена **улпрсетпроперти** задано значение **пр_нулл** ([PidTagNull](pidtagnull-canonical-property.md)), в списке можно выбрать несколько вариантов.
    
## <a name="remarks"></a>Примечания

Структура **дтбллбкс** описывает список элементов управления, которые используются для отображения нескольких элементов и позволяют пользователю выбрать один или несколько элементов. 
  
Члены **улпрсетпроперти** и **улпртабленаме** работают совместно; когда из таблицы выбирается одно значение, оно записывается обратно в **улпрсетпроперти** при закрытии диалогового окна. 
  
Значение Flags указывает, должна ли отображаться в списке горизонтальная или вертикальная полоса прокрутки. По умолчанию при необходимости отображаются типы полос прокрутки. Поставщики услуг могут настроить МАПИ_НО_ХБАР для отключения горизонтальной полосы прокрутки и МАПИ_НО_ВБАР для подавления вертикальной полосы прокрутки. 
  
Два элемента тега свойства работают вместе, чтобы отображать значения в списке и устанавливать соответствующие свойства при выборе элемента в списке. При первом отображении списка MAPI вызывает метод **опенпроперти** реализации **IMAPIProp** для получения таблицы, указанной в элементе **улпртабленаме** . Количество столбцов в таблице зависит от значения элемента **улпрсетпроперти** . Если для **улпрсетпроперти** задано значение **пр_нулл**, список является списком множественного выбора на основе объекта, содержащего получателей, например, контейнера адресной книги, таблицы получателей сообщения или таблицы содержимого списка рассылки. 
  
Таблица для списка множественного выбора должна содержать следующие столбцы:
  
 **Пр_дисплай_наме** ([PidTagDisplayName](pidtagdisplayname-canonical-property.md))
  
 **Пр_ентрид** ([PidTagEntryId](pidtagentryid-canonical-property.md))
  
 **Пр_инстанце_кэй** ([PidTagInstanceKey](pidtaginstancekey-canonical-property.md))
  
 **Пр_дисплай_типе** ([PidTagDisplayType](pidtagdisplaytype-canonical-property.md)), а также можно отобразить не более пяти других многозначных строковых свойств с тремя обязательными столбцами. 
  
Если для члена **улпрсетпроперти** не задано значение **пр_нулл**, список является одним списком выбора. Начальное значение **улпрсетпроперти** определяет первую выбранную строку. Когда пользователь выбирает одну из строк, для элемента **улпрсетпроперти** задается выбранное значение, а это значение записывается обратно в реализацию интерфейса свойства с вызовом [IMAPIProp:: SetProps](imapiprop-setprops.md). 
  
Общие сведения о отображаемых таблицах приведены в разделе [Display Tables](display-tables.md). Сведения о том, как реализовать таблицу отображения, можно найти [в статье реализация таблицы отображения](display-table-implementation.md).
