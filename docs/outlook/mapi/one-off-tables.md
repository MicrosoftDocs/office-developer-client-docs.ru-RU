<span data-ttu-id="537d0-105">Одноразовая таблица содержит сведения о шаблонах, которые поддерживаются поставщиком адресных книг для создания новых получателей.</span><span class="sxs-lookup"><span data-stu-id="537d0-105">A one-off table contains information about the templates that an address book provider supports for creating new recipients.</span></span> <span data-ttu-id="537d0-106">Одноразовые таблицы реализуются поставщиками адресных книг, отдельными адресными книгами и MAPI и могут быть постоянными или временными.</span><span class="sxs-lookup"><span data-stu-id="537d0-106">One-off tables are implemented by address book providers, individual address book containers, and by MAPI, and can be persistent or temporary.</span></span> 
  
> [!NOTE]
> <span data-ttu-id="537d0-107">Не путайте шаблоны в одноразовых таблицах с идентификаторами шаблонов; так как их назначения похожи, их конструкции кода ничего не представляют.</span><span class="sxs-lookup"><span data-stu-id="537d0-107">Do not confuse the templates in one-off tables with template identifiers; while their purposes are similar, their code constructs are nothing alike.</span></span> <span data-ttu-id="537d0-108">Шаблоны используются для создания получателей определенного типа, когда идентификаторы используются для связывания данных одного получателя, принадлежащего поставщику узла, с кодом для поддержки другого получателя, принадлежащего внешнему поставщику.</span><span class="sxs-lookup"><span data-stu-id="537d0-108">Templates are used to create recipients of a particular type while template identifiers are used to bind the data of one recipient that belong to a host provider with code to support another recipient that belong to a foreign provider.</span></span> 
  
<span data-ttu-id="537d0-109">Клиенты создают новых получателей:</span><span class="sxs-lookup"><span data-stu-id="537d0-109">Clients create new recipients:</span></span>
  
- <span data-ttu-id="537d0-110">Для добавления в список получателей исходящего сообщения.</span><span class="sxs-lookup"><span data-stu-id="537d0-110">To add to the recipient list of an outgoing message.</span></span>
    
- <span data-ttu-id="537d0-111">Для добавления в один из контейнеров в адресной книге.</span><span class="sxs-lookup"><span data-stu-id="537d0-111">To add to one of the containers in the address book.</span></span>
    
<span data-ttu-id="537d0-112">В обоих сценариях поставщику адресной книги предлагается возврат одноразовой таблицы.</span><span class="sxs-lookup"><span data-stu-id="537d0-112">In both scenarios, an address book provider is asked to return a one-off table.</span></span> <span data-ttu-id="537d0-113">Поставщики адресных книг могут реализовать одну одноразовую таблицу, которая будет использоваться в обеих ситуациях, или отдельную одноразовую таблицу для каждой ситуации.</span><span class="sxs-lookup"><span data-stu-id="537d0-113">Address book providers can implement either a single one-off table to be used in both situations or a unique one-off table for each situation.</span></span> 
  
<span data-ttu-id="537d0-114">Когда получатель будет включен в исходящее сообщение, MAPI вызывает метод [иаблогон:: жетонеоффтабле](iablogon-getoneofftable.md) поставщика адресной книги для получения ее одноразовой таблицы.</span><span class="sxs-lookup"><span data-stu-id="537d0-114">When the recipient will be included with an outgoing message, MAPI calls the address book provider's [IABLogon::GetOneOffTable](iablogon-getoneofftable.md) method to retrieve its one-off table.</span></span> <span data-ttu-id="537d0-115">В таблице одноразовых адресов содержатся шаблоны, которые позволяют пользователю вводить данные, что приводит к созданию получателя с допустимым адресом.</span><span class="sxs-lookup"><span data-stu-id="537d0-115">The one-off table includes templates which enable a user to enter information resulting in the creation of a recipient with a valid address.</span></span> <span data-ttu-id="537d0-116">Регистрирует регистры MAPI для уведомлений в этой таблице, не открывая ее, поэтому изменения могут быть отражаться у пользователя.</span><span class="sxs-lookup"><span data-stu-id="537d0-116">MAPI registers for notifications on this table, keeping it open so that changes can be reflected to the user.</span></span> <span data-ttu-id="537d0-117">MAPI освобождает таблицу только при вызове метода [имапистатус:: валидатестате](imapistatus-validatestate.md) в подсистеме или объекте состояния адресной книги.</span><span class="sxs-lookup"><span data-stu-id="537d0-117">MAPI releases the table only when its subsystem or address book status object's [IMAPIStatus::ValidateState](imapistatus-validatestate.md) method is called.</span></span> 
  
<span data-ttu-id="537d0-118">Когда получатель добавится в контейнер, MAPI выполняет другой вызов, вызывая метод контейнера [IMAPIProp:: опенпроперти](imapiprop-openproperty.md) для получения свойства **пр_креате_темплатес** ([PidTagCreateTemplates](pidtagcreatetemplates-canonical-property.md)).</span><span class="sxs-lookup"><span data-stu-id="537d0-118">When the recipient will be added to a container, MAPI makes a different call, invoking the container's [IMAPIProp::OpenProperty](imapiprop-openproperty.md) method to retrieve its **PR_CREATE_TEMPLATES** ([PidTagCreateTemplates](pidtagcreatetemplates-canonical-property.md)) property.</span></span> <span data-ttu-id="537d0-119">Набор шаблонов, включенных в эту односторонние таблицы, представляет типы получателей, которые могут быть добавлены в контейнер.</span><span class="sxs-lookup"><span data-stu-id="537d0-119">The set of templates included in this one-off table represents the types of recipients that can be added to the container.</span></span> <span data-ttu-id="537d0-120">Например, почтовые серверы часто предоставляют один контейнер для каждого установленного шлюза, чтобы каждый контейнер содержал только адреса соответствующего шлюза.</span><span class="sxs-lookup"><span data-stu-id="537d0-120">For example, mail servers often expose one container for every gateway that is installed so that each container only holds addresses specific to the corresponding gateway.</span></span>
  
<span data-ttu-id="537d0-121">MAPI предоставляет одноразовую таблицу, включающую собственные шаблоны, а также шаблоны из каждого поставщика адресных книг в сеансе.</span><span class="sxs-lookup"><span data-stu-id="537d0-121">MAPI provides a one-off table that includes its own templates as well as templates from each of the address book providers in the session.</span></span> <span data-ttu-id="537d0-122">MAPI предоставляет универсальный шаблон, который можно использовать для создания нового получателя для любого типа адреса, предполагая, что пользователь знает его формат.</span><span class="sxs-lookup"><span data-stu-id="537d0-122">MAPI provides a generic template that can be used to create a new recipient for any address type, assuming that the user knows its format.</span></span> <span data-ttu-id="537d0-123">Поставщики адресных книг используют эту одноразовую таблицу, вызывая [имаписуппорт:: жетонеоффтабле](imapisupport-getoneofftable.md).</span><span class="sxs-lookup"><span data-stu-id="537d0-123">Address book providers use this one-off table by calling [IMAPISupport::GetOneOffTable](imapisupport-getoneofftable.md).</span></span> <span data-ttu-id="537d0-124">Каждый из шаблонов, включенных в таблицу одноразовых MAPI, приводит к созданию получателей с действительными адресами получателей.</span><span class="sxs-lookup"><span data-stu-id="537d0-124">Each of the templates included in the MAPI one-off table results in the creation of recipients with valid recipient addresses.</span></span>
  
<span data-ttu-id="537d0-125">Поставщики адресных книг обычно предоставляют один шаблон для каждого поддерживаемого им типа адреса.</span><span class="sxs-lookup"><span data-stu-id="537d0-125">Address book providers typically supply one template for every address type they support.</span></span> <span data-ttu-id="537d0-126">Однако поддержка шаблонов не является обязательной.</span><span class="sxs-lookup"><span data-stu-id="537d0-126">However, support for templates is not required.</span></span> <span data-ttu-id="537d0-127">Поставщики адресных книг, которые не разрешают создавать новые адреса, могут возвращать МАПИ_Е_НО_СУППОРТ при вызовах MAPI для запроса односторонней таблицы.</span><span class="sxs-lookup"><span data-stu-id="537d0-127">Address book providers that do not allow the creation of new addresses can return MAPI_E_NO_SUPPORT when MAPI calls to request a one-off table.</span></span> <span data-ttu-id="537d0-128">Поставщики адресных книг, которые допускают создание нового адреса, но не предоставляют шаблоны, могут вызывать **имаписуппорт:: жетонеоффтабле** для использования шаблонов, перечисленных в таблице одноразовой MAPI.</span><span class="sxs-lookup"><span data-stu-id="537d0-128">Address book providers that do allow new address creation but do not supply any templates can call **IMAPISupport::GetOneOffTable** to use the templates listed in the MAPI one-off table.</span></span>
