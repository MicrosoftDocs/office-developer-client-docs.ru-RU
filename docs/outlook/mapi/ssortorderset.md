# <a name="ssortorderset"></a>SSortOrderSet

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Определяет коллекцию ключей сортировки для таблицы, используемой для стандартной или категоризизированной сортировки.
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapidefs.h  <br/> |
|Связанные макрос:  <br/> |[CbNewSSortOrderSet](cbnewssortorderset.md), [CbSSortOrderSet](cbssortorderset.md), [SizedSSortOrderSet](sizedssortorderset.md) <br/> |
   
```cpp
typedef struct _SSortOrderSet
{
  ULONG cSorts;
  ULONG cCategories;
  ULONG cExpanded;
  SSortOrder aSort[MAPI_DIM];
} SSortOrderSet, FAR *LPSSortOrderSet;

```

## <a name="members"></a>"Участники"

 **cSorts**
  
> Количество [структур SSortOrder,](ssortorder.md) включенных в **состав aSort.** 
    
 **cCategories**
  
> Количество столбцов, назначенных в качестве столбцов категорий. Возможные значения варьируются от нуля, что указывает на не категории или стандартный сорт, до числа, указанных членом **cSorts.** 
    
 **cExpanded**
  
> Количество категорий, которые начинаются в расширенном состоянии, где все строки, применимые к категории, видны в представлении таблицы. Возможные значения варьируются от 0 до числа, указанных **cCategories.**
    
 **aSort**
  
> Массив **структур SSortOrder,** каждый из которых определяет порядок сортировки. 
    
## <a name="remarks"></a>Примечания

Структура **SSortOrderSet** используется для определения нескольких заказов сортировки для стандартной и категоризизированной сортировки. 
  
Каждая **структура SSortOrderSet** содержит по крайней мере одну **структуру SSortOrder,** определяя направление сортировки и столбец, который будет использоваться в качестве ключа сортировки. Для сортировки по категориям этот столбец используется в качестве категории. Когда значение члена **cSorts** превышает значение члена **cCategories,** из столбцов, которые отображаются первыми в массиве **SSortOrder,** создается больше ключей сортировки, чем категорий. 
  
Например, если **cSorts** задают 3, а **cCategories** — 2, столбцы, описанные членом **ulPropTag** из первых двух записей в массиве **SSortOrder,** используются в качестве столбцов категорий. Первая запись выполняет группу категорий верхнего уровня; вторая запись в качестве вторичной группировки. Все строки, которые соответствуют столбцам двух категорий, сортироваться с помощью ключа сортировки, определенного в третьей записи. 
  
Член **cExpanded** указывает количество категорий, которые сначала расширяются. При нескольких категориях реализация таблицы начинается с первого столбца, который будет назначен в качестве категории, и продолжается последовательно с последующими столбцами категорий до превышения количества **cCategories.** Если столбцов категорий больше, чем расширенных, столбцы категории рухнут. Если **cExpanded** равен нулю, для отображения пользователю таблицы доступна только строка заголовка верхнего уровня. Если **cExpanded** равен одному меньше числа категорий, то доступны все строки заголовка и ни одна из строк листа. Если **cExpanded** равен числу категорий, таблица полностью расширена. 
  
Дополнительные сведения о стандартной и категоризации сортировки см. в [рубрике Сортировка и классификация.](sorting-and-categorization.md)
