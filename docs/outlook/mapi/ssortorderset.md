# <a name="ssortorderset"></a>SSortOrderSet

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Определяет коллекцию ключей сортировки для таблицы, используемой для стандартной или классификации сортировки.
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapidefs.h  <br/> |
|Связанные макросы:  <br/> |[CbNewSSortOrderSet](cbnewssortorderset.md), [CbSSortOrderSet](cbssortorderset.md), [SizedSSortOrderSet](sizedssortorderset.md) <br/> |
   
```cpp
typedef struct _SSortOrderSet
{
  ULONG cSorts;
  ULONG cCategories;
  ULONG cExpanded;
  SSortOrder aSort[MAPI_DIM];
} SSortOrderSet, FAR *LPSSortOrderSet;

```

## <a name="members"></a>"Участники"

 **cSorts**
  
> Количество структур [SSortOrder,](ssortorder.md) включенных в **член aSort.** 
    
 **cCategories**
  
> Количество столбцов, обозначенных в качестве столбцов категорий. Диапазон возможных значений: от 0, что указывает на не категории или стандартную сортировку, до числа, задаваемого членом **cSorts.** 
    
 **cExpanded**
  
> Количество категорий, которые начинаются в расширенном состоянии, где все строки, применимые к категории, видны в представлении таблицы. Диапазон возможных значений : от 0 до числа, на который указывают **cCategories.**
    
 **aSort**
  
> Массив структур **SSortOrder,** каждый из которых определяет порядок сортировки. 
    
## <a name="remarks"></a>Примечания

Структура **SSortOrderSet** используется для определения нескольких заказов сортировки для стандартной и классификационной сортировки. 
  
Каждая **структура SSortOrderSet** содержит по крайней мере одну **структуру SSortOrder,** определяющие направление сортировки, и столбец, который будет использоваться в качестве ключа сортировки. Для сортировки по категориям этот столбец используется в качестве категории. Если значение члена **cSorts** превышает значение члена **cCategories,** ключей сортировки больше, чем категорий, и категории создаются из столбцов, которые отображаются первыми в массиве **SSortOrder.** 
  
Например, если **cSorts** имеет 3, а **cCategories** — 2, столбцы, описанные членом **ulPropTag** первых двух записей в **массиве SSortOrder,** используются в качестве столбцов категории. Первая запись служит в качестве группировки категорий верхнего уровня; вторая запись в качестве вторичной группировки. Все строки, которые соответствуют двум столбцам категории, сортироваться с помощью ключа сортировки, определенного в третьей записи. 
  
Член **cExpanded** указывает количество категорий, которые были сначала расширены. Если имеется несколько категорий, реализация таблицы начинается с первого столбца, который назначается в качестве категории, и продолжается в последовательном порядке с последующими столбцы категорий до тех пор, пока количество **cCategories** не будет превышено. Если столбцов категорий больше, чем расширенных, столбцы категории свернуты. Если **cExpanded** равно нулю, пользователю таблицы доступна только строка заголовка верхнего уровня для отображения. Если **cExpanded** на один меньше числа категорий, то все строки заголовков и ни одна из листовых строк не будут доступны. Если **cExpanded** равно числу категорий, таблица будет полностью расширена. 
  
Дополнительные сведения о стандартной и категоризации сортировки см. в этой [теме.](sorting-and-categorization.md)
