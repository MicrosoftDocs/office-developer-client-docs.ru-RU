# <a name="writing-an-automated-client"></a>Написание автоматизированного клиента

  
  
**Относится к**: Outlook 
  
Автоматическая отправка клиентского приложения — это приложение, на котором выполняется в автоматическом режиме, пользовательский интерфейс не отображался.
  
 По умолчанию множество методов интерфейса MAPI отображения пользовательского интерфейса. Все эти методы имеют флаги, разрешающие клиента разрешить или отключить этот отображения. Хотя MAPI ожидает поставщиков услуг принять следующие флаги, существуют некоторые поставщики, которые не всегда соответствовать этим требованиям. Допустимым Причина не учитывая флаги — это расчет поставщика услуг на другой службы, которая не позволяет подавления пользовательского интерфейса. При разработке автоматической клиента Обратите пристальное внимание для поставщиков услуг, которую вы используете и как они были настроены. Не следует считать, что все звонки подавления пользовательского интерфейса будет выполнена успешно. 
  
Автоматическая отправка клиенты должны иметь сведения, необходимые для правильной настройки каждого из службы сообщений в профиле. Существует два способа для предоставления сведений о конфигурации во время входа:
  
- Поставщик услуг может получить сведения из профиля.
    
- Поставщик услуг может запрашивать у пользователя сведения. 
    
Так как второй параметр недоступен для автоматической клиентов, эти клиенты должны использовать первый вариант. Клиенты необходимо настроить свой профиль, чтобы убедиться, что этот параметр всегда работает.
  
Автоматическая отправка клиентов всегда устанавливать флаг MAPI_NO_MAIL в вызове функции [MAPILogonEx](mapilogonex.md) для начала сеанса MAPI.
