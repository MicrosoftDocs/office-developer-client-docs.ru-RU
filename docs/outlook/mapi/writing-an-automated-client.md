# <a name="writing-an-automated-client"></a>Написание автоматизированного клиента

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Автоматическое клиентские приложения — это приложение, которое запускается автоматически и не отображает пользовательский интерфейс.
  
 По умолчанию во многих методах интерфейса MAPI имеется пользовательский интерфейс. Все эти методы имеют флаги, которые позволяют клиенту разрешить или подавить этот дисплей. Хотя MAPI ожидает, что поставщики услуг будут соблюдать эти флаги, некоторые поставщики не всегда соответствуют этим ожиданиям. Допустимой причиной невыполнения флагов является зависимость поставщика службы от другой службы, которая не разрешает подавления пользовательского интерфейса. При разработке автоматизированного клиента обратите особое внимание на поставщиков услуг, которые вы используете, и их настройку. Не следует предполагать, что все ваши вызовы для подавления пользовательского интерфейса будут успешными. 
  
Автоматические клиенты должны иметь необходимую информацию для правильной настройки каждой службы сообщений в профиле. Существует два способа предоставить сведения о конфигурации во время логотипа:
  
- Поставщик службы может получить сведения из профиля.
    
- Поставщик услуг может затмить пользователя в запросе информации. 
    
Так как второй вариант недоступен для автоматизированных клиентов, эти клиенты должны использовать первый вариант. Клиенты должны тщательно настраивать свои профили, чтобы гарантировать, что этот параметр всегда работает.
  
Автоматические клиенты всегда устанавливают флаг MAPI_NO_MAIL в вызове функции [MAPILogonEx,](mapilogonex.md) чтобы начать сеанс MAPI.
