# <a name="writing-an-automated-client"></a>Написание автоматизированного клиента

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Автоматизированное клиентские приложения — это приложение, которое работает без присмотра, не отображая пользовательского интерфейса.
  
 По умолчанию многие методы интерфейса MAPI показывают пользовательский интерфейс. Все эти методы имеют флаги, которые позволяют клиенту разрешить или подавить этот дисплей. Хотя MAPI ожидает, что поставщики услуг будут соблюдать эти флаги, существуют некоторые поставщики, которые не всегда соответствуют этим ожиданиям. Законной причиной невыполнения флагов является зависимость поставщика услуг от другой службы, которая не позволяет подавления пользовательского интерфейса. Если вы разрабатываете автоматизированного клиента, обратите внимание на поставщиков услуг, которые вы используете и как они настроены. Не следует предполагать, что все ваши вызовы для подавления пользовательского интерфейса будут успешными. 
  
Автоматизированные клиенты должны иметь необходимую информацию для правильной конфигурации каждой из служб сообщений в профиле. Существует два способа поставки сведений о конфигурации во время логотипа:
  
- Поставщик служб может получать сведения из профиля.
    
- Поставщик служб может подсказыть пользователю информацию. 
    
Так как второй вариант недоступен для автоматизированных клиентов, эти клиенты должны использовать первый вариант. Клиенты должны тщательно настраивать свои профили, чтобы убедиться, что этот параметр всегда работает.
  
Автоматические клиенты всегда задают флаг MAPI_NO_MAIL в [вызове функции MAPILogonEx](mapilogonex.md) для начала сеанса MAPI.
