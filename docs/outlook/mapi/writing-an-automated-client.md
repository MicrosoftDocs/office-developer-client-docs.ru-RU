# <a name="writing-an-automated-client"></a>Написание автоматизированного клиента

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Автоматизированное клиентское приложение — это приложение, которое выполняется в автоматическом режиме без пользовательского интерфейса.
  
 По умолчанию многие методы интерфейса MAPI демонстрируют пользовательский интерфейс. Все эти методы имеют флаги, позволяющие клиенту разрешить или запретить это отображение. Несмотря на то, что служба MAPI ожидает, что эти флаги поддерживаются поставщиками, существует несколько поставщиков, которые не всегда удовлетворяют этим ожиданиям. Допустимая причина отсутствия флагов — зависимость от поставщика услуг в другой службе, которая не допускает запрета пользовательского интерфейса. При разработке автоматизированного клиента следует уделять особое внимание используемым поставщикам услуг и способам их настройки. Не следует предполагать, что все звонки, поблокирующие пользовательский интерфейс, будут успешными. 
  
Для автоматизированных клиентов должна быть доступна необходимая информация для правильной настройки каждой службы сообщений в профиле. Существует два способа предоставления сведений о конфигурации во время входа:
  
- Поставщик услуг может получить сведения из профиля.
    
- Поставщик услуг может запросить сведения у пользователя. 
    
Так как второй параметр недоступен для автоматических клиентов, эти клиенты должны использовать первый параметр. Клиенты должны тщательно настраивать их профили, чтобы убедиться, что этот вариант всегда работает.
  
Автоматические клиенты всегда устанавливают флаг MAPI_NO_MAIL в вызове функции [мапилогонекс](mapilogonex.md) для запуска сеанса MAPI.
