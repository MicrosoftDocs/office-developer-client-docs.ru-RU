# <a name="property-entries-in-mapisvcinf-message-service-sections"></a>Записи свойств в разделах службы сообщений MapiSvc. INF

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Записи, которые задают свойства, используют следующий формат:
  
 **тег Property** **=** значение свойства 
  
Тег Property может быть стандартным тегом свойства MAPI, если данные конфигурации представляют одно из свойств, предопределенных MAPI, или нестандартный тег, если данные не представляют собой свойство MAPI. Нестандартный тег состоит из сочетания значения идентификатора свойства с типом свойства. Результатом является 8-значное шестнадцатеричное число. Значение свойства может иметь любой смысл для тега Property. 
  
Разделы службы сообщений могут содержать различные записи в зависимости от настроенной службы сообщений. Следующие свойства MAPI обычно включаются в раздел "службы сообщений" в указанном формате:
  
 **** =  _Строка_ пр_дисплай_наме
  
 **Пр_сервице_длл_наме** =  _имя файла DLL_
  
 **Пр_сервице_ентри_наме** =  _имя функции точки входа_
  
 **Пр_сервице_суппорт_филес** =  _список файлов_
  
 **Пр_сервице_делете_филес** =  _список файлов_
  
 **** =  _Битовая маска_ пр_ресаурце_флагс
  
Строка **пр_дисплай_наме** ([PidTagDisplayName](pidtagdisplayname-canonical-property.md)) — это имя службы сообщений, отображаемое в пользовательском интерфейсе, имя, которое пользователь свяжет со службой сообщений. Отображаемое имя является необязательной записью в Mapisvc. INF. Иногда отображаемое имя будет состоять из двух частей; часть, назначенная службой сообщений, и часть, назначенная пользователем. Если пользователь несет ответственность за назначение одной из частей, обычно это правило обрабатывается специальным диалоговым окном, которое называется листом свойств, предоставленным службой сообщений в элементе управления клиентского приложения. 
  
Сведения, предоставленные для записи **пр_сервице_длл_наме** ([PidTagServiceDllName](pidtagservicedllname-canonical-property.md)), это имя библиотеки DLL, содержащей службу сообщений. Сведения, предоставляемые для записи **пр_сервице_ентри_наме** ([PidTagServiceEntryName](pidtagserviceentryname-canonical-property.md)), это имя функции точки входа в этой библиотеке DLL, с помощью которой вызовы MAPI настраивают службу сообщений. 
  
Файлы, перечисленные в записи **пр_сервице_суппорт_филес** ([PidTagServiceSupportFiles](pidtagservicesupportfiles-canonical-property.md)), — это файлы, которые необходимо установить вместе со службой сообщений. Аналогично, при удалении службы сообщений необходимо удалить файлы из записи **пр_сервице_делете_филес** ([PidTagServiceDeleteFiles](pidtagservicedeletefiles-canonical-property.md)). 
  
Запись **пр_ресаурце_флагс** ([PidTagResourceFlags](pidtagresourceflags-canonical-property.md)) представляет собой коллекцию параметров, определенных для службы сообщений. Например, бит СЕРВИЦЕ_СИНГЛЕ_КОПИ задается, когда служба сообщений может появиться только один раз в заданном профиле. Бит СЕРВИЦЕ_НО_ПРИМАРИ_ИДЕНТИТИ задается, если служба сообщений не предоставляет сведения об удостоверениях. 
  
Ниже приведены два примера нестандартных записей свойств. Первая запись указывает путь к файлу, используемому адресной книгой по умолчанию в качестве значения свойства; Вторая запись указывает значение числового свойства. Обе записи имеют значение, характерное для службы сообщений АДРЕСНОЙ книги.
  
```cpp
6600001e=full path to file
66040003=integer

```
