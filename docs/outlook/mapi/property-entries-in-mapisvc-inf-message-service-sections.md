# <a name="property-entries-in-mapisvcinf-message-service-sections"></a>Записи свойств в разделах службы сообщений MapiSvc. INF

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Записи, которые задают свойства, используют следующий формат:
  
 значение свойства **тега свойства** **=** 
  
Тег Property может быть стандартным тегом свойства MAPI, если данные конфигурации представляют одно из свойств, предопределенных MAPI, или нестандартный тег, если данные не представляют собой свойство MAPI. Нестандартный тег состоит из сочетания значения идентификатора свойства с типом свойства. Результатом является 8-значное шестнадцатеричное число. Значение свойства может иметь любой смысл для тега Property. 
  
Разделы службы сообщений могут содержать различные записи в зависимости от настроенной службы сообщений. Следующие свойства MAPI обычно включаются в раздел "службы сообщений" в указанном формате:
  
 **PR_DISPLAY_NAME** =  _Строка_ PR_DISPLAY_NAME
  
 **PR_SERVICE_DLL_NAME** =  _имя файла DLL_
  
 **PR_SERVICE_ENTRY_NAME** =  _имя функции точки входа_
  
 **PR_SERVICE_SUPPORT_FILES** =  _списка файлов_
  
 **PR_SERVICE_DELETE_FILES** =  _списка файлов_
  
 **PR_RESOURCE_FLAGS** =  _Битовая маска_ PR_RESOURCE_FLAGS
  
Строка **PR_DISPLAY_NAME** ([PidTagDisplayName](pidtagdisplayname-canonical-property.md)) — это имя службы сообщений, отображаемое в пользовательском интерфейсе, имя, которое пользователь свяжет со службой сообщений. Отображаемое имя является необязательной записью в Mapisvc. INF. Иногда отображаемое имя будет состоять из двух частей; часть, назначенная службой сообщений, и часть, назначенная пользователем. Если пользователь несет ответственность за назначение одной из частей, обычно это правило обрабатывается специальным диалоговым окном, которое называется листом свойств, предоставленным службой сообщений в элементе управления клиентского приложения. 
  
Сведения, предоставленные для записи **PR_SERVICE_DLL_NAME** ([PidTagServiceDllName](pidtagservicedllname-canonical-property.md)), это имя библиотеки DLL, содержащей службу сообщений. Сведения, предоставленные для записи **PR_SERVICE_ENTRY_NAME** ([PidTagServiceEntryName](pidtagserviceentryname-canonical-property.md)), это имя функции точки входа в этой библиотеке DLL, которая вызывает MAPI для настройки службы сообщений. 
  
Файлы, перечисленные в записи **PR_SERVICE_SUPPORT_FILES** ([PidTagServiceSupportFiles](pidtagservicesupportfiles-canonical-property.md)), — это файлы, которые необходимо установить вместе со службой сообщений. Аналогично, при удалении службы сообщений необходимо удалить файлы из записи **PR_SERVICE_DELETE_FILES** ([PidTagServiceDeleteFiles](pidtagservicedeletefiles-canonical-property.md)). 
  
Запись **PR_RESOURCE_FLAGS** ([PidTagResourceFlags](pidtagresourceflags-canonical-property.md)) представляет собой коллекцию параметров, определенных для службы сообщений. Например, бит SERVICE_SINGLE_COPY задается, когда служба сообщений может появиться только один раз в заданном профиле. Бит SERVICE_NO_PRIMARY_IDENTITY задается, если служба сообщений не предоставляет сведения об удостоверениях. 
  
Ниже приведены два примера нестандартных записей свойств. Первая запись указывает путь к файлу, используемому адресной книгой по умолчанию в качестве значения свойства; Вторая запись указывает значение числового свойства. Обе записи имеют значение, характерное для службы сообщений адресной книги.
  
```cpp
6600001e=full path to file
66040003=integer

```
