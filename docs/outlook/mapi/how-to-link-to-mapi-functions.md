### <a name="to-call-mapi-api-elements-without-the-mapi-library-and-the-windows-mapi-stub"></a><span data-ttu-id="1d38e-121">Чтобы вызвать элементы API-интерфейса MAPI без библиотеки MAPI и заглушка Windows MAPI</span><span class="sxs-lookup"><span data-stu-id="1d38e-121">To call MAPI API elements without the MAPI library and the Windows MAPI stub</span></span>

1. <span data-ttu-id="1d38e-122">В файле программы создания глобального списка указатели на функции для каждого элемента MAPI API, которое используется.</span><span class="sxs-lookup"><span data-stu-id="1d38e-122">In your program file, create a global list of function pointers for each MAPI API element that you are using.</span></span>

    LPMAPIUNINITIALIZE pfnMAPIUninitialize = NULL;
   ```

2. <span data-ttu-id="1d38e-124">Создание функции, инициализирует функции MAPI для ссылки на библиотеку DLL MAPI клиент MAPI по умолчанию (например, Msmapi32.dll из Microsoft Outlook).</span><span class="sxs-lookup"><span data-stu-id="1d38e-124">Create a function that initializes MAPI functions to link to the MAPI DLL of the default MAPI client (for example, Msmapi32.dll of Microsoft Outlook).</span></span> <span data-ttu-id="1d38e-125">В этой функции выполните следующие действия.</span><span class="sxs-lookup"><span data-stu-id="1d38e-125">In this function, do the following:</span></span> 
    
    1. <span data-ttu-id="1d38e-126">Загрузите mapi32.dll из каталога, соответствующую систему.</span><span class="sxs-lookup"><span data-stu-id="1d38e-126">Load mapi32.dll from the appropriate system directory.</span></span>

       |:-----|:-----|
       |<span data-ttu-id="1d38e-127">x64 или x86 изначально</span><span class="sxs-lookup"><span data-stu-id="1d38e-127">x64 or x86 natively</span></span>  <br/> |<span data-ttu-id="1d38e-128">**%windir%\system32\mapi32.dll**</span><span class="sxs-lookup"><span data-stu-id="1d38e-128">**%windir%\system32\mapi32.dll**</span></span> <br/> |

    2. <span data-ttu-id="1d38e-131">Вызовите функцию [FGetComponentPath](fgetcomponentpath.md) для получения путь и имя библиотеки DLL, который реализует подсистемы MAPI.</span><span class="sxs-lookup"><span data-stu-id="1d38e-131">Call the [FGetComponentPath](fgetcomponentpath.md) function to get the path and DLL name that implements the MAPI subsystem.</span></span> <span data-ttu-id="1d38e-132">Дополнительные сведения можно [выбрать определенные версии MAPI для загрузки](how-to-choose-a-specific-version-of-mapi-to-load.md).</span><span class="sxs-lookup"><span data-stu-id="1d38e-132">For more information, see [Choose a Specific Version of MAPI to Load](how-to-choose-a-specific-version-of-mapi-to-load.md).</span></span>

    <span data-ttu-id="1d38e-135">В следующем примере показано предыдущие действия:</span><span class="sxs-lookup"><span data-stu-id="1d38e-135">The following example shows the previous steps:</span></span>

3. <span data-ttu-id="1d38e-136">И, наконец вызовите функцию, созданный на шаге 2 в приложение для обмена сообщениями прежде чем выполнять вызовы к элементам API-интерфейса MAPI.</span><span class="sxs-lookup"><span data-stu-id="1d38e-136">Finally, call the function that you created in step 2 in your messaging application before you make calls to MAPI API elements.</span></span> 
    
   > [!CAUTION]

    int main()

        HRESULT hr;
        InitializeMapiFunctions();
        // Initialize the MAPI subsystem.
        hr = (*pfnMAPIInitialize)(NULL);
        if (hr!= S_OK)
        {
            // Handle the error case.
        }
        // Here is where you make calls to other MAPI interfaces.
        // Uninitialize the MAPI subsystem.

    return (0);
    }
   ```
