# <a name="imapisessionopenaddressbook"></a>IMAPISession::OpenAddressBook

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Открывает интегрированную адресную книгу MAPI, возвращая указатель [IAddrBook](iaddrbookimapiprop.md) для получения доступа. 
  
```cpp
HRESULT OpenAddressBook(
  ULONG_PTR ulUIParam,
  LPCIID lpInterface,
  ULONG ulFlags,
  LPADRBOOK FAR * lppAdrBook
);
```

## <a name="parameters"></a>Параметры

 _улуипарам_
  
> возврата Дескриптор родительского окна для диалогового окна общего адреса и других связанных дисплеев.
    
 _лпинтерфаце_
  
> возврата Указатель на идентификатор интерфейса (IID), представляющий интерфейс, который будет использоваться для доступа к адресной книге. При передаче **значения NULL** возвращается указатель на стандартный интерфейс адресной книги [IAddrBook: IMAPIProp](iaddrbookimapiprop.md). 
    
 _ulFlags_
  
> возврата Битовая маска флагов, управляющих открытием адресной книги. Можно задать следующий флаг:
    
AB_NO_DIALOG 
  
> Подавляет отображение диалоговых окон. Если флаг AB_NO_DIALOG не установлен, то поставщики адресных книг, которые вносят вклад в интегрированную адресную книгу, могут запрашивать у пользователя необходимые сведения. 
    
 _лппадрбук_
  
> вышли Указатель на указатель на адресную книгу.
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Адресная книга успешно открыта.
    
MAPI_W_ERRORS_RETURNED 
  
> Вызов выполнен успешно, но не удается открыть контейнеры одного или нескольких поставщиков адресной книги. При возвращении этого предупреждения вызов должен обрабатываться как успешный. Чтобы проверить это предупреждение, используйте макрос **HR_FAILED** . Дополнительные сведения см. [в разделе Использование макросов для обработки ошибок](using-macros-for-error-handling.md).
    
## <a name="remarks"></a>Примечания

Метод **IMAPISession:: опенаддрессбук** открывает интегрированную адресную книгу MAPI, коллекцию контейнеров верхнего уровня для всех поставщиков адресной книги в профиле. Указатель, возвращенный в параметре _лппадрбук_ , предоставляет дополнительный доступ к содержимому адресной книги. Это позволяет вызывающему абоненту выполнять такие задачи, как открытие отдельных контейнеров, Поиск пользователей обмена сообщениями и отображение общих диалоговых окон адресов. 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

 **Опенаддрессбук** возвращает MAPI_W_ERRORS_RETURNED, если не удается загрузить одного или нескольких поставщиков адресной книги в профиле. Это значение является предупреждением, а не значением ошибки; Обработайте ее так же, как S_OK. **Опенаддрессбук** всегда возвращает допустимый указатель в параметре _лппадрбук_ , независимо от того, сколько поставщиков адресной книги не удалось загрузить. Таким образом, перед выходом из системы необходимо всегда вызывать метод [IUnknown:: Release](https://msdn.microsoft.com/library/ms682317%28v=VS.85%29.aspx) адресной книги в некоторый момент. 
  
Когда **опенаддрессбук** возвращает MAPI_W_ERRORS_RETURNED, вызовите [IMAPISession:: GetLastError](imapisession-getlasterror.md) , чтобы получить структуру [мапиеррор](mapierror.md) , содержащую сведения о неисправном поставщике. Возвращается одна структура **мапиеррор** , содержащая сведения, предоставленные всеми поставщиками.
