# <a name="imapipropdeleteprops"></a>IMAPIProp::DeleteProps

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Удаляет одно или несколько свойств объекта. 
  
```cpp
HRESULT DeleteProps(
  LPSPropTagArray lpPropTagArray,
  LPSPropProblemArray FAR * lppProblems
);
```

## <a name="parameters"></a>Параметры

 _Лппроптагаррай_
  
> возврата Указатель на массив тегов свойств, указывающий свойства, которые требуется удалить. Элемент **квалуес** структуры [спроптагаррай](sproptagarray.md) , на который указывает _лппроптагаррай_ , не должен иметь значение 0, а сам параметр _лппроптагаррай_ не должен иметь значение null. 
    
 _Лпппроблемс_
  
> [вход, выход] На входе указатель на указатель на структуру [спроппроблемаррай](spropproblemarray.md) ; в противном случае — значение NULL, которое указывает, что нет необходимости в сведениях об ошибке. Если _лпппроблемс_ является допустимым указателем на входе, **делетепропс** возвращает подробные сведения об ошибках при удалении одного или нескольких свойств. 
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Свойства были успешно удалены.
    
МАПИ_Е_НО_АКЦЕСС 
  
> Вызывающий не имеет достаточных разрешений на удаление свойств.
    
## <a name="remarks"></a>Комментарии

Метод **IMAPIProp::D елетепропс** удаляет одно или несколько свойств из текущего объекта. 
  
## <a name="notes-to-implementers"></a>Примечания для исполнителей

Не нужно разрешать удаление свойств из всех объектов. Если объект не является изменяемым, возвращайте МАПИ_Е_НО_АКЦЕСС из метода **делетепропс** . 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

Нет необходимости задавать тип свойства для каждого тега свойства в массиве тегов свойств, на который указывает параметр _лппроптагаррай_ . Типы свойств игнорируются; используются только идентификаторы свойств. 
  
Обратите внимание, что некоторые объекты не допускают изменения и что эти объекты возвращают МАПИ_Е_НО_АКЦЕСС из метода **делетепропс** . Другие объекты позволяют удалять некоторые свойства, но не другие. При возникновении проблем с удалением только некоторых свойств **делетепропс** ВОЗВРАЩАЕТ значение S_OK. Если вы передали допустимый указатель в параметре _лпппроблемс_ , **делетепропс** установит указатель на структуру **спроппроблемаррай** , содержащую подробные сведения о проблемах с каждым свойством. Например, если удаляются все свойства сообщения и возникла проблема с одним или несколькими вложениями, структура **спроппроблемаррай** будет содержать запись для **пр_мессаже_аттачментс** ([PidTagMessageAttachments ](pidtagmessageattachments-canonical-property.md)). 
  
Структура, на которую указывает _лпппроблемс_ , допустима только в том случае, если **ДЕЛЕТЕПРОПС** возвращает значение S_OK. Если **делетепропс** возвращает ошибку, не пытайтесь использовать структуру **спроппроблемаррай** . Вместо этого вызовите метод объекта [IMAPIProp:: GetLastError](imapiprop-getlasterror.md) , чтобы получить дополнительные сведения об ошибке. 
  
Освободите возвращаемую структуру **спроппроблемаррай** , вызвав функцию [мапифрибуффер](mapifreebuffer.md) .
