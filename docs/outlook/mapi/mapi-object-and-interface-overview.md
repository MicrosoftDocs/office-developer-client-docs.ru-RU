# <a name="mapi-object-and-interface-overview"></a>Общие сведения об объекте и интерфейсе MAPI

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Объект MAPI — это класс объекта C++ или структура данных C, унаследованных от одного или нескольких интерфейсов MAPI, или коллекций связанных функций. Эти коллекции связанных функций известны разработчикам C++ как чистые виртуальные функции. Для чисто виртуальной функции MAPI предоставляет только прототип функции, а не реализацию. Ожидается, что клиентское приложение, поставщик услуг или MAPI предоставит эту реализацию, создавая класс объекта, наследуемый от интерфейса и соответствие описанию функции API обмена сообщениями. Для интерфейса MAPI можно создавать экземпляры только с помощью унаследованного класса.
  
Существует множество различных объектов MAPI, каждый из которых наследует от интерфейса, который в конечном счете наследуется от интерфейса [IUnknown](https://msdn.microsoft.com/library/33f1d79a-33fc-4ce5-a372-e08bda378332%28Office.15%29.aspx) . **IUnknown** является базовым интерфейсом модели COM-компонентов OLE. Он предоставляет объекты MAPI стандартным механизмом связи и управления. COM определяет, как разработчики объектов обрабатывают проблемы, такие как управление памятью, Управление параметрами и многопоточность. В соответствии с этой моделью объект, реализующий объект, соответствует контракту, указанному в интерфейсах, включенных в объект. 
  
Многие интерфейсы MAPI наследуются напрямую от **IUnknown**, в то время как другие наследуются косвенно через один из двух других базовых интерфейсов: [IMAPIProp: IUnknown](imapipropiunknown.md) для управления свойствами и [IMAPIContainer: IMAPIProp](imapicontainerimapiprop.md) для папки и доступ к адресной книге. Базовые интерфейсы никогда не реализуются как отдельные, автономные объекты; они всегда реализуются как часть других объектов, объекты, реализующие производные интерфейсы. 
  
MAPI определяет множество типов объектов, каждый из которых реализуется одним или несколькими компонентами MAPI. Объекты, реализуемые клиентами, используются MAPI, поставщики услуг и настраиваемые компоненты форм. Объекты, реализуемые поставщиками услуг, обычно используются MAPI и клиентами. Объекты, реализуемые поставщиками библиотек форм и серверами форм, используются другими компонентами форм и клиентами.
