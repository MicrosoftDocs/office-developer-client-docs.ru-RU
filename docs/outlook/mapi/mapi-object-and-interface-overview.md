# <a name="mapi-object-and-interface-overview"></a><span data-ttu-id="b5fb2-103">Общие сведения об объекте и интерфейсе MAPI</span><span class="sxs-lookup"><span data-stu-id="b5fb2-103">MAPI Object and Interface Overview</span></span>

  
  
<span data-ttu-id="b5fb2-104">**Область применения**: Outlook 2013 | Outlook 2016</span><span class="sxs-lookup"><span data-stu-id="b5fb2-104">**Applies to**: Outlook 2013 | Outlook 2016</span></span> 
  
<span data-ttu-id="b5fb2-105">Объект MAPI — это класс объекта C++ или структура данных C, унаследованных от одного или нескольких интерфейсов MAPI, или коллекций связанных функций.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-105">A MAPI object is a C++ object class or C data structure inherited from one or more MAPI interfaces, or collections of related functions.</span></span> <span data-ttu-id="b5fb2-106">Эти коллекции связанных функций известны разработчикам C++ как чистые виртуальные функции.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-106">These collections of related functions are known to C++ developers as pure virtual functions.</span></span> <span data-ttu-id="b5fb2-107">Для чисто виртуальной функции MAPI предоставляет только прототип функции, а не реализацию.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-107">For a pure virtual function, MAPI supplies only the function prototype, not an implementation.</span></span> <span data-ttu-id="b5fb2-108">Ожидается, что клиентское приложение, поставщик услуг или MAPI предоставит эту реализацию, создавая класс объекта, наследуемый от интерфейса и соответствие описанию функции API обмена сообщениями.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-108">It is expected that a client application, a service provider, or MAPI will provide this implementation by creating an object class that inherits from the interface and conforms to the function descriptions of the Messaging API.</span></span> <span data-ttu-id="b5fb2-109">Для интерфейса MAPI можно создавать экземпляры только с помощью унаследованного класса.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-109">A MAPI interface can be instantiated only through an inherited class.</span></span>
  
<span data-ttu-id="b5fb2-110">Существует множество различных объектов MAPI, каждый из которых наследует от интерфейса, который в конечном счете наследуется от интерфейса [IUnknown](https://msdn.microsoft.com/library/33f1d79a-33fc-4ce5-a372-e08bda378332%28Office.15%29.aspx) .</span><span class="sxs-lookup"><span data-stu-id="b5fb2-110">There are many different MAPI objects, each object inheriting from an interface that is ultimately inherited from the [IUnknown](https://msdn.microsoft.com/library/33f1d79a-33fc-4ce5-a372-e08bda378332%28Office.15%29.aspx) interface.</span></span> <span data-ttu-id="b5fb2-111">**IUnknown** является базовым интерфейсом модели COM-компонентов OLE.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-111">**IUnknown** is the OLE Component Object Model (COM) base interface.</span></span> <span data-ttu-id="b5fb2-112">Он предоставляет объекты MAPI стандартным механизмом связи и управления.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-112">It provides MAPI objects with a standard mechanism for communication and control.</span></span> <span data-ttu-id="b5fb2-113">COM определяет, как разработчики объектов обрабатывают проблемы, такие как управление памятью, Управление параметрами и многопоточность.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-113">COM dictates how object implementers handle issues such as memory management, parameter management, and multithreading.</span></span> <span data-ttu-id="b5fb2-114">В соответствии с этой моделью объект, реализующий объект, соответствует контракту, указанному в интерфейсах, включенных в объект.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-114">By conforming to this model, an object implementer adheres to a contract as specified by the interfaces included in the object.</span></span> 
  
<span data-ttu-id="b5fb2-115">Многие интерфейсы MAPI наследуются напрямую от **IUnknown**, в то время как другие наследуются косвенно через один из двух других базовых интерфейсов: [IMAPIProp: IUnknown](imapipropiunknown.md) для управления свойствами и [IMAPIContainer: IMAPIProp](imapicontainerimapiprop.md) для папки и доступ к адресной книге.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-115">Many MAPI interfaces are inherited directly from **IUnknown**, while others are inherited indirectly through one of two other base interfaces: [IMAPIProp : IUnknown](imapipropiunknown.md) for property management and [IMAPIContainer : IMAPIProp](imapicontainerimapiprop.md) for folder and address book access.</span></span> <span data-ttu-id="b5fb2-116">Базовые интерфейсы никогда не реализуются как отдельные, автономные объекты; они всегда реализуются как часть других объектов, объекты, реализующие производные интерфейсы.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-116">Base interfaces are never implemented as separate, standalone objects; they are always implemented as part of other objects, objects that implement derived interfaces.</span></span> 
  
<span data-ttu-id="b5fb2-117">MAPI определяет множество типов объектов, каждый из которых реализуется одним или несколькими компонентами MAPI.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-117">MAPI defines many types of objects, each implemented by one or more MAPI components.</span></span> <span data-ttu-id="b5fb2-118">Объекты, реализуемые клиентами, используются MAPI, поставщики услуг и настраиваемые компоненты форм.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-118">Objects implemented by clients are used by MAPI, by service providers, and by custom form components.</span></span> <span data-ttu-id="b5fb2-119">Объекты, реализуемые поставщиками услуг, обычно используются MAPI и клиентами.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-119">Objects implemented by service providers are typically used by MAPI and by clients.</span></span> <span data-ttu-id="b5fb2-120">Объекты, реализуемые поставщиками библиотек форм и серверами форм, используются другими компонентами форм и клиентами.</span><span class="sxs-lookup"><span data-stu-id="b5fb2-120">Objects implemented by form library providers and form servers are used by other form components and by clients.</span></span>
