# <a name="mapi-object-and-interface-overview"></a>Обзор объектов и интерфейсов MAPI

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Объект MAPI — это объектный класс C++ или структура данных C, унаследованные от одного или более интерфейсов MAPI или коллекций связанных функций. Эти коллекции связанных функций известны разработчикам C++ как чистые виртуальные функции. Для чисто виртуальной функции MAPI поставляет только прототип функции, а не реализацию. Ожидается, что клиентские приложения, поставщик услуг или MAPI обеспечат эту реализацию, создав объектный класс, унаследованный от интерфейса и соответствующий описаниям функций API обмена сообщениями. Интерфейс MAPI можно мгновенно использовать только через унаследованный класс.
  
Существует множество различных объектов MAPI, каждый из которых наследуется от интерфейса, который в конечном счете наследуется [интерфейсом IUnknown.](https://msdn.microsoft.com/library/33f1d79a-33fc-4ce5-a372-e08bda378332%28Office.15%29.aspx) **IUnknown —** базовый интерфейс OLE Component Object Model (COM). Он обеспечивает объекты MAPI стандартным механизмом связи и управления. COM диктует, как реализующие объекты обрабатывают такие проблемы, как управление памятью, управление параметрами и многоуровневая реализация. В соответствии с этой моделью реализующий объект придерживается контракта, указанного интерфейсами, включенными в объект. 
  
Многие интерфейсы MAPI наследуются напрямую от **IUnknown,** другие — косвенно через один из двух других базовых интерфейсов: [IMAPIProp: IUnknown](imapipropiunknown.md) для управления свойствами и [IMAPIContainer : IMAPIProp](imapicontainerimapiprop.md) для доступа к папкам и адресной книге. Базовые интерфейсы никогда не реализуются как отдельные автономные объекты; они всегда реализуются как часть других объектов, объектов, реализуют производные интерфейсы. 
  
MAPI определяет множество типов объектов, каждый из которых реализуется одним или более компонентами MAPI. Объекты, реализуемые клиентами, используются MAPI, поставщиками услуг и пользовательскими компонентами форм. Объекты, реализуемые поставщиками услуг, обычно используются MAPI и клиентами. Объекты, реализованные поставщиками библиотек форм и серверами форм, используются другими компонентами формы и клиентами.
