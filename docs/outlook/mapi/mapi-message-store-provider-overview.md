# <a name="mapi-message-store-provider-overview"></a>Общие сведения о поставщике хранилища сообщений MAPI
  
**Относится к**: Outlook 
  
Поставщики хранилища сообщений обрабатывать хранения и извлечения сообщений и других сведений для пользователей из клиентских приложений. Сведения о сообщении организован с помощью иерархической системы, известных как хранилища сообщений. Хранилище сообщений реализована в несколько уровней контейнеры содержат сообщения различных типов папок. Нет ограничений на количество уровней в хранилище сообщений; может содержать множество вложенные папки. 
  
На следующем рисунке показана архитектура хранилища иерархических сообщений.
  
**Архитектура хранилища сообщений**
  
![Архитектура хранилища сообщений] (media/amapi_03.gif "Архитектура хранилища сообщений")
  
На рисунке показаны две папки с вложенной папке. Пользователей клиентского приложения можно получить доступ к сводное представление сообщений, содержащихся в каждой папке или просматривать их по отдельности с формой. Отображает ли клиент стандартной формы, которая предоставляет MAPI или настраиваемую форму, которая предоставляет разработчик формы зависит от типа или класса сообщения. Первая папка содержит примечаний и использует стандартные Примечание формы MAPI. Второй папка содержит сообщения запроса на инвентаризации и используется форма настраиваемой инвентаризации. Сведения о обе формы представляет свойства сообщения.
  
Можно использовать данные хранилища сообщений в различными способами. С помощью данных для традиционных электронной почты, можно использовать папки форум для общих обсуждений, как репозиторий документов с ссылку или контейнером для голосовой почты, календарь, контакты или задачи, например. Хранилище одно сообщение может содержать различные типы данных. Несколько клиентов можно установить из одного хранилища сообщений. Таким образом, общий доступ к данным легко и быстро. 
  
Папки хранилища сообщений позволяет сортировать и фильтровать сообщения и для настройки отображения сообщения в пользовательском интерфейсе. Ссылки на отфильтрованные сообщения хранятся в специальных папках, имя папки результатов поиска. Клиентское приложение пользователем критерии фильтрации, которые MAPI называется ограничение и критерии применяется к сообщениям, хранящиеся в одной или нескольких папок. Например пользователь может потребоваться просматривать только сообщения, которые относятся к конкретной темы и датой поступления, более ранние, чем прошлую неделю. Справочные материалы, которые соответствуют критериям перечислены в папке поиска, и реальные сообщения остаются в своих папок, регулярные.
  
Сообщения, единиц измерения данных, которые переносятся из одного пользователя или приложения для другого пользователя или приложения. Каждое сообщение содержит некоторые текста сообщения, с помощью простых и сложных форматирование и сведения о конверт сообщения, используемый для передачи данных. Некоторые сообщения включать одно или несколько вложений и дополнительные данные связанные с и транспортируются сообщение в виде файла, другое сообщение или объекта OLE. 
  
В зависимости от сообщения хранилища поставщика, пользователь может сохранить новое сообщение, в настоящее время записывается в дополнение к сообщениям, которые отправку и получение. Сообщения можно скопировать или переместить из одной папки в другую для каждой копии становится отдельное сообщение, который можно скопировать, удалить или изменить по отдельности. Другого компонента, что некоторые сообщения хранения enable поставщиков является возможность изменять сообщение после его получения и сохраните ее обратно в его папки. Пользователь может использовать эту функцию для вращающимся факсов сообщение, которое было доставлено сверху вниз. Правильный режим можно сохранять в папке для последующего просмотра.
