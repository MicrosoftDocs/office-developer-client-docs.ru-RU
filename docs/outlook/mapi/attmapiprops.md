# <a name="attmapiprops"></a>attMAPIProps

  
  
**Относится к**: Outlook 
  
Атрибут **attMAPIProps** особые, в том, что они могут использоваться для кодирования любого свойства MAPI, для которого не аналога в наборе существующие атрибуты TNEF. Атрибут данных является инвентаризации набор свойств MAPI расположения начала до конца. Формат данной кодировки, что позволяет для любого набора свойств MAPI, выглядит следующим образом:  
  
 _Property_Seq:_
  
> свойство count _Property_Value..._
    
Должно быть любое количество элементов _Property_Value_ указывает значение свойства count. 
  
 _Property_Value:_
  
> Свойство tag _Property_property тега _Proptag_Name свойство_
    
Тег свойства — это просто значение, связанное с идентификатором свойства, такие как 0x0037001F для **PR_SUBJECT** ([PidTagSubject](pidtagsubject-canonical-property.md)).
  
 _Свойство:_
  
>  Количество значений _значение_ , _значение..._
    
 _Значение:_
  
> данные значения значение данных размер значения отступов внутренние поля данных значение ИД размер значения значение интерфейса
    
 _Proptag_Name:_
  
> Имя guid вида имя идентификатора имени имя guid имя типа длина строки имя строки с названием внутренние поля
    
Инкапсуляция каждое свойство в зависимости от идентификатор свойства и типа свойства. Свойство теги, идентификаторы и типы определяются в файлах заголовка Mapitags.h и Mapidefs.h.
  
Если свойство имеет именованного свойства, а затем свойство tag немедленно следуют имя свойства MAPI, состоящий из глобальный уникальный идентификатор (GUID), тип и идентификатор или строка Юникода.
  
Многозначные свойства и свойства с помощью переменной длины значения, такие как типы свойств PT_BINARY, PT_STRING8, PT_UNICODE или PT_OBJECT обрабатывается следующим образом. Сначала количество значений, помеченные как неподписанные длинные, 32-разрядного переводится в поток TNEF, а затем отдельных значений. Данные значения каждого свойства закодирован его размер в байтах, следуют значение данных самого. Значение параметра дополняется выходной параметр на границе 4-байтных, несмотря на то, что отступ не включено в размер значения.
  
Если свойство имеет тип PT_OBJECT, размер значения следует интерфейс идентификатор объекта. Текущая реализация TNEF поддерживает только идентификаторы интерфейса IID_IMessage, IID_IStorage и IID_Istream. Размер идентификатор интерфейса включается в размер значения.
  
Если объект является внедренных сообщений (то есть, он имеет тип свойства PT_OBJECT и идентификатор интерфейса IID_Imessage), данные значения кодировке внедренных поток TNEF. Фактическая кодировка внедренных сообщений в реализации TNEF выполняется, открыв второй объект TNEF для исходного потока и обработки встроенного потока.
