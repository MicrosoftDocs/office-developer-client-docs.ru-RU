# <a name="imapitableseekrowapprox"></a>IMAPITable::SeekRowApprox

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
ПереМещает курсор к приблизительному расположению в таблице. 
  
```cpp
HRESULT SeekRowApprox(
ULONG ulNumerator,
ULONG ulDenominator
);
```

## <a name="parameters"></a>Параметры

 _Улнумератор_
  
> возврата Указатель на числитель дробной части, представляющий положение таблицы. Если значение параметра _улнумератор_ равно нулю, курсор размещается в начале таблицы независимо от значения знаменателя. Если _улнумератор_ равен параметру _улденоминатор_ , курсор помещается после последней строки таблицы. 
    
 _Улденоминатор_
  
> возврата Указатель на знаменатель дроби, представляющий положение таблицы. Параметр _улденоминатор_ не может иметь значение 0. 
    
## <a name="return-value"></a>Возвращаемое значение

S_OK 
  
> Операция поиска выполнена успешно.
    
МАПИ_Е_БУСИ 
  
> Выполняется другая операция, которая не позволяет запустить операцию поиска строк. Выполнение выполняемой операции должно быть разрешено или остановлено.
    
## <a name="remarks"></a>Комментарии

Положение курсора в таблице после вызова метода **IMAPITable:: сикроваппрокс** является эвристическим по отношению к дробным и может быть неточным. Например, определенные поставщики могут реализовать таблицу поверх двоичного дерева, рассматривая положение таблицы в верхней части дерева по соображениям производительности. Если дерево не сбалансировано, то используемая половина точки может не распоровнося на середине таблицы. 
  
## <a name="notes-to-callers"></a>Примечания для вызывающих методов

ВыЗовите **сикроваппрокс** , чтобы предоставить данные для реализации полосы прокрутки. Например, если пользователь позиционирует бегунок 2/3 вниз на полосу прокрутки, это действие можно смоделировать, вызвав **сикроваппрокс** и передав эквивалентное дробное значение с помощью _улнумератор_ и _улденоминатор_. Поиск **сикроваппрокс** всегда является абсолютным относительно начала таблицы. Чтобы перейти к концу таблицы, значения в _улнумератор_ и _улденоминатор_ должны быть одинаковыми. 
  
Используйте любую нужную схему номеров. То есть, чтобы перейти к позиции на середине по таблице, можно указать 1/2, 10/20 или 50/100.
