# <a name="pidtagsentrepresentingaddresstype-canonical-property"></a>Каноническое свойство PidTagSentRepresentingAddressType

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит тип адреса для пользователя обмена сообщениями, представленного отправивщиком.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_SENT_REPRESENTING_ADDRTYPE, PR_SENT_REPRESENTING_ADDRTYPE_A, PR_SENT_REPRESENTING_ADDRTYPE_W  <br/> |
|Идентификатор:  <br/> |0x0064  <br/> |
|Тип данных:  <br/> |PT_UNICODE, PT_STRING8  <br/> |
|Область:  <br/> |Адрес  <br/> |
   
## <a name="remarks"></a>Примечания

Эти свойства являются примерами свойств адресов для пользователя обмена сообщениями, представленного отправителю. Когда клиентская заявка отправляет сообщение от имени другого клиента, оно должно установить все представленные свойства отправитель к значениям для этого клиента. Пользователь обмена сообщениями, отправляющий сообщения от своего имени, обычно оставляет представленные свойства отправитель неустановленными.
  
Исходяющий поставщик транспорта всегда должен оставить это свойство без изменений, если оно было заданной клиентом отправки. Если оно не установлено, поставщик транспорта должен установить его в **свойство PR_SENDER_ADDRTYPE** [(PidTagSenderAddressType)](pidtagsenderaddresstype-canonical-property.md)в исходящие копии сообщения и оставить его неустановленным в локальной копии.
