# <a name="pidtagsenderemailaddress-canonical-property"></a>Каноническое свойство PidTagSenderEmailAddress

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Содержит адрес электронной почты отправителя сообщения.
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_SENDER_EMAIL_ADDRESS, PR_SENDER_EMAIL_ADDRESS_A PR_SENDER_EMAIL_ADDRESS_W  <br/> |
|Идентификатор:  <br/> |0x0C1F  <br/> |
|Тип данных:  <br/> |PT_UNICOIDE PT_STRING8  <br/> |
|Область:  <br/> |Address  <br/> |
   
## <a name="remarks"></a>Примечания

Эти свойства являются примерами свойств адреса для отправителя сообщения. Они должны быть заданы поставщиком исходящей транспортировки, который никогда не должен распространять какие бы то ни было существующие значения.
  
Если поставщик транспорта не предоставил свойства адреса отправителя, то диспетчер очереди MAPI пытается заполнить их, вызвав метод [IMAPISession:: куеридентити](imapisession-queryidentity.md) для идентификатора записи. Если не указаны никакие идентификаторы записей, диспетчер очереди MAPI сохраняет значение "Unknown" во всех свойствах адреса отправителя типа PT_TSTRING.
