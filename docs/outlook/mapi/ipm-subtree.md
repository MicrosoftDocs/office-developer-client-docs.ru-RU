# <a name="ipm-subtree"></a>Поддерево IPM

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
MAPI создает дерево папок под корневой папкой хранилища сообщений для всех клиентов, которые отправляют и получают сообщения от человека, а не компьютера, получателя. Сообщения, которыми обмениваются получатели людей, называются взаимосвязанными сообщениями, а это дерево называется межпользовательским сообщением или IPM, поддеревом. 
  
Поддерево IPM для хранилища доставки состоит из по крайней мере следующих папок:
  
- Inbox;
    
- Исходящие
    
- Отправленные
    
- Удаленные
    
Это имена и роли по умолчанию для каждой из этих папок; Клиент может указать собственные имена, если имена по умолчанию не подходят. MAPI назначает имена по умолчанию и сопоставления для этих папок, чтобы предотвратить непреднамеренное исчезновение сообщений, если клиент не устанавливает получение папок для сообщений. 
  
В контексте Microsoft Office Outlook поддерево IPM состоит из дополнительных папок по умолчанию для календаря, контактов, задач, заМеток и журнала.
  
Обычно в папке "Входящие" хранятся входящие сообщения, а в папке "исХодящие" находятся исходящие сообщения (то есть сообщения, ожидающие отправки). Папка "Отправленные" содержит копию каждого отправленного сообщения, если клиент установил для свойства **пр_сентмаил_ентрид** ([PidTagSentMailEntryId](pidtagsentmailentryid-canonical-property.md)) идентификатор записи этой папки. Папка "Удаленные" содержит сообщения, помеченные для удаления.
