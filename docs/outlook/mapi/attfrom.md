# <a name="attfrom"></a>attFrom

**Применимо к**: Outlook 2013 | Outlook 2016 
  
Атрибут **attFrom** закодирован **BIOS** структуру, которая кодирует отображаемое имя и адрес электронной почты адрес отправителя, следуют отображаемое имя и адрес отправителя, следуют любые необходимые внутренние поля. Формат для **attFrom** выглядит следующим образом: 
  
**attFrom**: заполнение _ адрес отправителя отправителя отображаемое имя _ _Структура BIOS._ 
    
Отправитель отображаемое имя — строка символом null, заполняется дополнительные символ null, если это необходимо для связи с 2-байтовое границы. Внутренние поля в конце кодирования **attFrom** состоит из числа символов null, чтобы сделать доступными **sizeof(TRP)** границы. 
  
_BIOS структура:_ Сертификация cch cbgrtrp **trpidOneOff** 
    
Для элемента **attFrom** **BIOS.**-Структура — это всегда одноразовых кодирования, поэтому trpid off **BIOS.**-поле структуры всегда является **trpidOneOff**. Cbgrtrp, cch и сертификация элементов соответствуют остальные поля структуры **BIOS.** . 
  
В поле cbgrtrp рассчитывается как сумма **(sizeof(TRP) \*2)**, длина символом null-display имя отправителя его заполнения и длина символом null адрес отправителя.
  
В поле cch рассчитывается как длина символом null названия его заполнения.
  
В поле сертификация рассчитывается как длина символом null адрес отправителя.
  
_адрес отправителя:_ **:** адрес типа адреса **«\0»**
    
Адрес отправителя — это строка, которая состоит из четырех частей, тип адреса, литерал двоеточие (:), сам по себе адрес и конечный символ null. Например, строка `fax:1-909-555-1234\0` может иметь значение юридический адрес отправителя.
