# <a name="determining-a-tables-end"></a>Определение конца таблицы

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
 Распространенной ошибкой является предполагается, что конец таблицы достигнут в следующих случаях: 
  
- [IMAPITable:: QueryRows](imapitable-queryrows.md) вызывается в цикле, а конец цикла определяется числом строк, которое возвращается методом [IMAPITable::.](imapitable-getrowcount.md) Число, которое **** возвращает функция GetRows, не всегда представляет точное количество строк в таблице; это примерное количество. 
    
- **QueryRows** вызывался с фиксированным количеством строк и возвращается меньшее количество строк. Пока **QueryRows** не возвратит набор строк, в котором число строк равно нулю, что больше нет строк для получения. 
    
> [!IMPORTANT]
> Единственный момент, когда вызывающий может предположить, что курсор находится в конце таблицы для положительного числа строк или в начале таблицы для отрицательного числа строк, когда возвращаются значения S_OK и 0. Значение МАПИ_Е_НОТ_ФАУНД никогда не возвращается.
