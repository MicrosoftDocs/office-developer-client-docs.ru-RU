# <a name="transport-provider-overview"></a>Обзор поставщиков транспорта

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Поставщика транспорта — это библиотека динамической компоновки (DLL), которая выполняет роль посредника между подсистемы MAPI и один или несколько базовых систем обмена сообщениями. Системы обмена сообщениями — некоторые определенные механизм, с помощью которого отправки и получения сообщений. Приведены некоторые примеры систем обмена сообщениями.
  
- Файловая система общей сетевой, поставщика транспорта записывает сообщения, чтобы напрямую.
    
- Сетевой интерфейс TCP/IP, поставщика транспорта для подключения к серверу обмена сообщениями.
    
- Интерактивная служба, который пользователи подключаются к.
    
- На основе узла обмена мгновенными сообщениями или office автоматизации система.
    
- Набор удаленный вызов процедур в сервер обмена сообщениями.
    
- Другие данные, которые можно использовать для передачи данных с одного компьютера на другой.
    
DLL-Библиотеки поставщика транспорта должен соответствовать типу интерфейса, указанная в MAPI. Разработчик, поставщика транспорта следует реализовать этот интерфейс с точки зрения функциональности в системе обмена сообщениями.
