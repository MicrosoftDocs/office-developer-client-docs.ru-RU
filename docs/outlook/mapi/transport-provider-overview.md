# <a name="transport-provider-overview"></a>Обзор поставщика транспорта

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Поставщик транспорта — это библиотека динамической связи (DLL), которая выступает в качестве посредника между подсистемой MAPI и одной или более систем обмена сообщениями. Система обмена сообщениями — это определенный механизм, с помощью которого отправляются и получаются сообщения. Вот несколько примеров систем обмена сообщениями:
  
- Общая файловая система сети, в которую поставщик транспорта записывает сообщения напрямую.
    
- Сетевой интерфейс TCP/IP, который поставщик транспорта использует для подключения к серверу обмена сообщениями.
    
- Веб-служба, к которую подключаются пользователи.
    
- Система обмена сообщениями на основе хост-приложений или системы автоматизации Office.
    
- Набор удаленных вызовов процедур на сервер обмена сообщениями.
    
- Все, что можно использовать для передачи данных с одного компьютера на другой.
    
DLL поставщика транспорта должен соответствовать интерфейсу, указанному MAPI. Как разработчик поставщика транспорта вы будете реализовывать этот интерфейс с точки зрения функций, присутствующих в системе обмена сообщениями.
