# <a name="component-object-model-and-mapi"></a>Модель COM и MAPI

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
Документация по Windows SDK содержит полное обсуждение правил реализации объектов, которые соответствуют модели COM. Эти правила обуяют следующее:
  
- Проектирование интерфейсов и объектов.
    
- Реализуют [интерфейс IUnknown.](https://msdn.microsoft.com/library/ms680509%28VS.85%29.aspx) 
    
- Управление памятью.
    
- Обработка подсчета ссылок.
    
- Реализуют объекты с потоковой реализацией в многоядровом окнах.
    
Хотя все объекты MAPI считаются com-основанными, так как они реализуют интерфейсы, наследуемые от [IUnknown,](https://msdn.microsoft.com/library/ms680509%28VS.85%29.aspx)MAPI в некоторых случаях отклоняется от стандартных правил COM. Это отклонение позволяет разработчикам более гибко реализации. Например, интерфейс MAPI, как и любой интерфейс COM, описывает контракт между реализуемой и вызываемой. После создания и публикации интерфейса его определение не может изменяться и не меняется. MAPI не отклоняться от этого описания, но он немного ослабляет описание. Реализующие средства могут не реализовывать определенные методы, возвращая звонячему одно из следующих значений ошибки: 
  
- MAPI_E_NO_SUPPORT
    
- MAPI_E_TOO_COMPLEX
    
- MAPI_E_BAD_CHARWIDTH
    
- MAPI_E_TYPE_NO_SUPPORT
    
Другие отклонения от стандартных правил COM описаны в следующей таблице.
  
|**Правило программирования COM**|**Вариант MAPI**|
|:-----|:-----|
|Все строковые параметры в методах интерфейса должны быть Юникод.  <br/> |Интерфейсы MAPI определяются для разрешения параметров строки Юникод или ANSI. Многие методы с параметром string также имеют параметр **ulFlags;** Ширина строки указывается значением флага MAPI_UNICODE в **ulFlags.** Некоторые интерфейсы MAPI не поддерживают Юникод и возвращают MAPI_E_BAD_CHARWIDTH при MAPI_UNICODE флага.  <br/> |
|Все методы интерфейса должны иметь возвращаемую тип HRESULT.  <br/> |MAPI имеет по крайней мере один метод, который возвращает значение, не относящеся к HRESULT: [IMAPIAdviseSink::OnNotify](imapiadvisesink-onnotify.md).  <br/> |
|Вызыватели и реализутели должны выделять и освободить память для параметров интерфейса с помощью стандартных компонующих задач COM.  <br/> |Все методы MAPI используют связанные средства [MAPIAllocateBuffer,](mapiallocatebuffer.md) [MAPIAllocateMore](mapiallocatemore.md)и [MAPIFreeBuffer](mapifreebuffer.md) для управления памятью параметров интерфейса. Все реализации интерфейсов MAPI, определенные OLE, например [IStream,](https://msdn.microsoft.com/library/aa380034%28VS.85%29.aspx)используют стандартные композиторы задач COM.  <br/> |
|При сбойе метода все параметры указателя на выходе должны быть явно заданы как NULL.  <br/> |Интерфейсы MAPI требуют, чтобы для параметров указателя на выходе было установлено NULL или они оставались неизменными при сбойе метода. Во всех реализациях интерфейсов MAPI, определенных OLE, в случае сбоя явно заданы параметры NULL.  <br/> |
|Реализуют агрегируемые объекты, когда это возможно.  <br/> |Интерфейсы MAPI не агрегируемы.  <br/> |
