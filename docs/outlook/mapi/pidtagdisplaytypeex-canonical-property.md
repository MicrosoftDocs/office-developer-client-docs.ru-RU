# <a name="pidtagdisplaytypeex-canonical-property"></a>Каноническое свойство PidTagDisplayTypeEx

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит тип записи, по отношению к способу отображения записи в строке таблицы для глобального списка адресов. 
  
|||
|:-----|:-----|
|Связанные свойства:  <br/> |ПР_ДИСПЛАЙ_ТИПЕ_ЕКС  <br/> |
|Идентификатор:  <br/> |0x3905  <br/> |
|Тип данных:  <br/> |PT_LONG  <br/> |
|Область:  <br/> |Адресная книга MAPI  <br/> |
   
## <a name="remarks"></a>Примечания

Это свойство определяет тип записи по отношению к тому, как она должна отображаться в глобальном списке адресов. Он предоставляет дополнительные сведения, которые не могут быть представлены в **пр_дисплай_типе** ([PidTagDisplayType](pidtagdisplaytype-canonical-property.md)).
  
> [!NOTE]
> Значения обоих **пр_дисплай_типе** и этого свойства начинаются с "дт_" и определяются в мапитагс. h. Все значения, которые не задокументированы, зарезервированы для MAPI. Клиентские приложения не должны определять новые значения и должны быть готовы к работе с недокументированным значением. 
  
Существует несколько макросов, которые могут помочь определить атрибуты объекта, например, будь это локальная, удаленная или контроль безопасности. К этим макросам относятся: 
  
|**Macro**|**Значение**|
|:-----|:-----|
|ДТЕ_ФЛАГ_РЕМОТЕ_ВАЛИД  <br/> |0x80000000  <br/> |
|ДТЕ_ФЛАГ_АКЛ_КАПАБЛЕ  <br/> |0x40000000  <br/> |
|ДТЕ_МАСК_РЕМОТЕ  <br/> |0x0000FF00  <br/> |
|ДТЕ_МАСК_ЛОКАЛ  <br/> |0x000000ff  <br/> |
|ДТЕ_ИС_РЕМОТЕ_ВАЛИД (v)  <br/> |(!! ((v) &amp; дте_флаг_ремоте_валид)  <br/> |
|ДТЕ_ИС_АКЛ_КАПАБЛЕ (v)  <br/> |(!! ((v) &amp; дте_флаг_акл_капабле))  <br/> |
|ДТЕ_РЕМОТЕ (v)  <br/> |(((v) &amp; дте_маск_ремоте) \> \> 8)  <br/> |
|ДТЕ_ЛОКАЛ (v)  <br/> |((v) &amp; дте_маск_локал)  <br/> |
|ДТ_РУМ  <br/> |((ULONG) 0x00000007)  <br/> |
|ДТ_ЕКУИПМЕНТ  <br/> |((ULONG) 0x00000008)  <br/> |
|ДТ_СЕК_ДИСТЛИСТ  <br/> |((ULONG) 0x00000009)  <br/> |
   
Ниже приведены несколько заметок о том, как использовать эти макросы. 
  
- Чтобы проверить, является ли запись удаленной записью в другом лесу, примените макрос ДТЕ_ИС_РЕМОТЕ_ВАЛИД к значению этого свойства, чтобы проверить, установлен ли флаг ДТЕ_ФЛАГ_РЕМОТЕ_ВАЛИД в записи. Если это удаленная запись, можно узнать тип удаленной записи с помощью макроса ДТЕ_РЕМОТЕ. 
    
- В конфигурациях с одним лесом и между лесами, когда **пр_дисплай_типе** имеет значение дт_дистлист, а дте_ис_ремоте_валид имеет значение false, применение макроса дте_локал к значению этого свойства позволяет определить тип объекта как ДТ_ДИСТЛИСТ (список рассылки) или ДТ_СЕК_ДИСТЛИСТ (список рассылки по безопасности). 
    
- Если вы применяете макрос ДТЕ_ЛОКАЛ к значению **пр_дисплай_типе_екс** и ВОЗВРАЩАЕТ тип дт_ремоте_маилусер, то запись будет удалена. 
    
- В отдельном лесу или в конфигурации с несколькими лесами, где управление репликацией осуществляется со списками управления доступом (ACL), можно использовать макрос ДТЕ_ИС_АКЛ_КАПАБЛЕ, чтобы определить, является ли запись субъектом безопасности.
    
В конфигурации с несколькими лесами **пр_дисплай_типе** имеет значение дт_ремоте_маилусер. Применение макроса ДТЕ_РЕМОТЕ к значению этого свойства позволяет получить тип удаленной записи. Возможны следующие типы удаленной записи: 
  
|**Тип удаленной записи**|**Value**|**Описание**|
|:-----|:-----|:-----|
|ДТ_АЖЕНТ  <br/> |((ULONG) 0x00000003)  <br/> |Динамический список рассылки.  <br/> |
|ДТ_ДИСТЛИСТ  <br/> |((ULONG) 0x00000001)  <br/> |Список рассылки.  <br/> |
|ДТ_ЕКУИПМЕНТ  <br/> |((ULONG) 0x00000008)  <br/> |Оборудование, например принтер или проектор.  <br/> |
|ДТ_МАИЛУСЕР  <br/> |((ULONG) 0x00000000)  <br/> |Пользователь с почтовым ящиком.  <br/> |
|ДТ_РЕМОТЕ_МАИЛУСЕР  <br/> |((ULONG) 0x00000000)  <br/> |Запись списка адресов в глобальном списке адресов.  <br/> |
|ДТ_РУМ  <br/> |((ULONG) 0x00000007)  <br/> |Конференц-зал.  <br/> |
|ДТ_СЕК_ДИСТЛИСТ  <br/> |((ULONG) 0x00000009)  <br/> |Список рассылки по безопасности.  <br/> |
   
Как в случае с одним лесом, так и в конфигурации с несколькими лесами, когда **пр_дисплай_типе** имеет значение дт_дистлист, а дте_ис_ремоте_валид — false, применение макроса дте_локал к значению этого свойства позволит получить тип списка рассылки. . Ниже перечислены возможные типы списка рассылки. 
  
|**Тип списка рассылки**|**Value**|**Описание**|
|:-----|:-----|:-----|
|ДТ_ДИСТЛИСТ  <br/> |((ULONG) 0x00000001)  <br/> |Список рассылки.  <br/> |
|ДТ_СЕК_ДИСТЛИСТ  <br/> |((ULONG) 0x00000009)  <br/> |Список рассылки по безопасности.  <br/> |
