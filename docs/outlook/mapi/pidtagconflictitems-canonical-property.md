# <a name="pidtagconflictitems-canonical-property"></a>Каноническое свойство PidTagConflictItems

  
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Содержит один или несколько входных ИД элементов, которые были вовлечены в автоматическое разрешение конфликтов.
  
## 

|||
|:-----|:-----|
|Связанные свойства:  <br/> |PR_CONFLICT_ITEMS  <br/> |
|Идентификатор:  <br/> |0x1098  <br/> |
|Тип свойства:  <br/> |PT_MV_BINARY  <br/> |
|Область:  <br/> |ICS  <br/> |
   
## <a name="remarks"></a>Примечания

Типы стандартных элементов Microsoft Outlook, которые поддерживают автоматическое разрешение конфликтов, включают следующие стандартные типы элементов: пункты назначения, контактные элементы, элементы журнала, элементы почты, элементы собраний, липкие элементы заметки и элементы задач. Элемент, принадлежащий классу сообщений, который происходит из одного из этих стандартных типов элементов, также поддерживает автоматическое разрешение конфликтов. В Microsoft Outlook 2003 и Microsoft Office Outlook 2007 г., когда Outlook синхронизирует элементы и считает, что существует вероятность того, что результатовая копия может содержать не все необходимые данные, Outlook хранит конфликтующие копии в папке **Conflicts,** в папке **Sync Issues.** 
  
> [!NOTE]
> **Проблемы синхронизации и** его подмостки скрыты до тех пор, пока вы не нажмете список папок **в** меню **Go.** 
  
Элемент предоставляет свойство **PR_CONFLICT_ITEMS,** если это один из типов элементов, которые поддерживают автоматическое разрешение конфликтов, выиграли в разрешении конфликтов или были помещены в папку **Conflicts** из-за разрешения конфликтов. Папка, в которой размещен элемент, определяет содержимое **PR_CONFLICT_ITEMS.** Если элемент находится в какой-либо папке, кроме папки **Conflicts,** и элемент предоставляет свойство **PR_CONFLICT_ITEMS,** элемент должен  был выиграть разрешение конфликтов, а PR_CONFLICT_ITEMS будет содержать один или несколько входных ИД тех элементов, которые были потеряны в разрешении конфликтов. Если элемент находится в папке **Conflict** и элемент предоставляет свойство **PR_CONFLICT_ITEMS,** этот элемент должен был потерять  разрешение конфликта, и PR_CONFLICT_ITEMS будет содержать входной ID элемента, выигранного в разрешении конфликтов.
