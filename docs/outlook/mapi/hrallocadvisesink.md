# <a name="hrallocadvisesink"></a>HrAllocAdviseSink

  
  
**Применимо к**: Outlook 
  
Создает объект приемника уведомлений, задавая контекст, указанного идентификатором вызывающего реализации и функции обратного вызова, чтобы инициировать, уведомления о событии. 
  
|||
|:-----|:-----|
|Файл заголовка:  <br/> |Mapiutil.h  <br/> |
|Реализованный:  <br/> |MAPI  <br/> |
|Вызывается:  <br/> |Клиентские приложения и поставщиков услуг  <br/> |
   
```cpp
STDAPI HrAllocAdviseSink(
  LPNOTIFCALLBACK lpfnCallback,
  LPVOID lpvContext,
  LPMAPIADVISESINK FAR * lppAdviseSink
);
```

## <a name="parameters"></a>Parameters

 _lpfnCallback_
  
> [in] Указатель на функцию обратного вызова, на основании прототип [NOTIFCALLBACK](notifcallback.md) , который будет вызываться при возникновении события уведомления для только что созданный является MAPI уведомить приемника. 
    
 _lpvContext_
  
> [in] Указатель на данные вызывающего передается функции обратного вызова, когда она вызывается средствами MAPI. Вызывающий объект данных может представлять адрес значение для клиента или поставщика. Обычно для кода C++ параметр _lpvContext_ представляет указатель на адрес объекта. 
    
 _lppAdviseSink_
  
> [out] Указатель на указатель на объект приемник уведомлений.
    
## <a name="return-value"></a>Возвращаемое значение

Нет.
  
## <a name="remarks"></a>Замечания

Использовать функцию **HrAllocAdviseSink** , клиентское приложение или поставщик услуг создает объекта, который будет получать уведомления, создает функцию обратного вызова уведомления на основании прототипа функции [NOTIFCALLBACK](notifcallback.md) , что происходит с помощью этого объекта и передает указатель на объект функции **HrAllocAdviseSink** , что значение _lpvContext_ . Таким образом выполняется уведомления; и в ходе процесса уведомления MAPI вызывает функцию обратного вызова с помощью указателя на объект контекста. 
  
MAPI асинхронно реализует его обработчиком уведомлений. В C++ обратного вызова для уведомления может быть метода объекта. Если объект, создание уведомления не представить, клиента или поставщика, запрос уведомлений о необходимо поддерживать количество отдельных ссылку для этого объекта для объектов уведомить приемника. 
  
> [!CAUTION]
> **HrAllocAdviseSink** следует использовать с осторожностью; он является более безопасным для клиентов создать собственные приемники уведомлений.
