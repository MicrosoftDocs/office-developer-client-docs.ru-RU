# <a name="mapi-client-objects"></a>Объекты клиента MAPI
  
**Область применения**: Outlook 2013 | Outlook 2016 
  
Стандартные клиентские приложения для обмена сообщениями реализуют только один объект — раковину рекомендации. Советы по раковинам наследуются интерфейсом [IMAPIAdviseSink: IUnknown](imapiadvisesinkiunknown.md) и используются MAPI и поставщиками услуг для уведомления о событиях. Некоторые клиенты также реализуют объекты прогресса для поддержки отображения диалогов прогресса. 
  
Более сложные клиенты, поддерживающий настраиваемые формы, реализуют другой объект раковины и несколько других объектов, например объект сайта сообщений, наследуемый интерфейсом [IMAPIMessageSite: IUnknown](imapimessagesiteiunknown.md) и объектом контекста представления, который наследуется от [интерфейса IMAPIViewContext: IUnknown.](imapiviewcontextiunknown.md) Дополнительный объект рекомендации по раковине наследует от [интерфейса IMAPIViewAdviseSink: IUnknown.](imapiviewadvisesinkiunknown.md) 
  
В следующей таблице подводятся итоги объектов MAPI, реализованных стандартными клиентами обмена сообщениями и клиентами, поддерживаюными просмотр пользовательских форм.
  
|**Объект клиента**|**Описание**|
|:-----|:-----|
|Советы по раковине  <br/> |Предоставляет функцию вызова для событий, происходящих в хранилище сообщений, адресной книге или сеансе.  <br/> |
|Сайт сообщений  <br/> |Обрабатывает манипуляции с объектами форм.  <br/> |
|Ход выполнения  <br/> |Отображает диалоговое окно, чтобы показать ход операции.  <br/> |
|Просмотр рекомендации по раковине  <br/> |Предоставляет функции вызова для событий, происходящих в форме.  <br/> |
|Просмотр контекста  <br/> |Поддерживает команды для печати и сохранения форм и для навигации между формами.  <br/> |
   
На следующем рисунке показана связь между этими различными объектами клиента, интерфейсами, из которых они наследуют, и компонентами MAPI, которые их используют. 
  
![Объекты клиента и соответствующие интерфейсы](media/amapi_65.gif "Объекты клиента и соответствующие интерфейсы")
  
Клиенты используют гораздо больше объектов, чем реализуют. Все клиенты используют объект сеанса, чтобы получить доступ к широкому кругу объектов и объектов поставщика услуг, реализуемых MAPI. Клиенты взаимодействуют с поставщиками услуг через сеанс, адресную книгу или объекты состояния, которые поставляет MAPI, или непосредственно через различные объекты, которые реализуют конкретные поставщики услуг. Для прямого контакта с поставщиками адресных книг клиенты используют контейнеры адресных книг, пользователей сообщений и списки рассылки. Чтобы напрямую получить доступ к поставщику хранения сообщений, клиенты используют объект магазина сообщений, папки, сообщения и вложения. Когда поставщики услуг поддерживают объект состояния, клиенты могут использовать объект состояния для мониторинга состояния поставщика услуг.
  
Клиенты, которые поддерживают поставщика услуг и конфигурацию служб сообщений, используют три объекта, реализуемые MAPI: объект администрирования службы сообщений, объект администрирования профилей и объект администрирования поставщика. Клиенты, отображаемые в настраиваемой форме, используют несколько объектов форм, реализуемые поставщиком библиотеки форм или сервером форм.
