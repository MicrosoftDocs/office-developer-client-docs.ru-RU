# <a name="mapi-client-objects"></a>Клиентские объекты MAPI
  
**Относится к**: Outlook 
  
Стандартный клиентских приложений для обмена сообщениями реализовать только один объект — приемника уведомления. Уведомить приемники наследование от [IMAPIAdviseSink: IUnknown](imapiadvisesinkiunknown.md) интерфейса и используемых MAPI службы и поставщиков для уведомлений о событиях. Некоторые клиенты также реализовать хода выполнения объекты для поддержки отображения хода выполнения диалоговых окон. 
  
Более сложная клиентов, что поддержка настраиваемых форм реализовать другой уведомлений приемник объект и некоторые другие объекты, такие как объект сайта сообщения, который наследует от [IMAPIMessageSite: IUnknown](imapimessagesiteiunknown.md) интерфейс и объект контекст представления, который наследует от [IMAPIViewContext: IUnknown](imapiviewcontextiunknown.md) интерфейса. Объект приемника дополнительных уведомлений наследует от [IMAPIViewAdviseSink: IUnknown](imapiviewadvisesinkiunknown.md) интерфейса. 
  
В следующей таблице приведены объекты MAPI, реализуемый стандартных обмена сообщениями клиентов и клиентов, которые поддерживают просмотра настраиваемых форм.
  
|**Клиентского объекта**|**Описание**|
|:-----|:-----|
|Уведомить приемника  <br/> |Предоставляет функцию обратного вызова для событий, возникающих в хранилище сообщений, адресной книги или сеанса.  <br/> |
|Сообщение сайта  <br/> |Обработка манипуляции объектам формы.  <br/> |
|Выполнение  <br/> |Отображает диалоговое окно для отображения хода выполнения операции.  <br/> |
|Просмотр рекомендаций приемника  <br/> |Предоставляет функции обратного вызова для событий, возникающих в форме.  <br/> |
|Контекст представления  <br/> |Поддерживает команды для печати и сохранении формы и перемещения между формами.  <br/> |
   
На следующем рисунке показана связь между эти различные клиентские объекты, интерфейсы, с которых они наследуют и компоненты MAPI, их использования. 
  
![Клиентские объекты и соответствующие интерфейсы] (media/amapi_65.gif "Клиентские объекты и соответствующие интерфейсы")
  
Клиенты используют много больше объектов, чем их реализации. Все клиенты используют объект сеанса для получения доступа к широкий набор объектов поставщика службы и объекты, внедряемые MAPI. Взаимодействия клиентов с поставщиками услуг косвенно, через сеанс, адресной книги или состояние объектов, которые предоставляет MAPI, либо непосредственно через различные объекты, реализуемые поставщиками конкретной службы. Чтобы контакты с поставщиками адресных книг, клиенты используют контейнеров адресной книги, система обмена сообщениями пользователей и списки рассылки. Для доступа к поставщика хранилища сообщений напрямую, клиенты используют объекта хранилища сообщений, папок, сообщений и вложений. Когда поставщиков услуг поддерживает состояние объекта, клиенты могут использовать объект состояния для отслеживания состояния поставщика услуг.
  
Клиенты, которые поддерживают поставщика услуг и конфигурации службы сообщений используется три объекты, внедряемые MAPI: объект администрирования службы сообщений, объект администрирования профилей и объект администрирования поставщика. Клиенты, которые отображают настраиваемые формы используйте несколько объектов формы, которые реализует поставщика библиотеки форм или на сервере формы.
