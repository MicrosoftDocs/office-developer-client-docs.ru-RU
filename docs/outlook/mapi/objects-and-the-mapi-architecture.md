# <a name="objects-and-the-mapi-architecture"></a>Объекты и архитектура MAPI

**Относится к**: Outlook 2013 | Outlook 2016 
  
Все объекты, определенные MAPI, попадают в один или несколько уровней архитектуры MAPI. Уровень клиентского интерфейса содержит все объекты, которые может реализовать клиентские приложения, средства просмотра форм или сервер форм. Уровень интерфейса поставщика услуг содержит объекты, которые может реализовать поставщик служб любого типа. Этот уровень включает объекты, реализованные адресными книгами, хранилищами сообщений, поставщиками транспорта и библиотеками форм. Уровень, который представляет подсистему MAPI, находится между уровнями интерфейса клиента и поставщика услуг. Уровень MAPI содержит все объекты, которые mapI реализует для клиентов или поставщиков услуг, которые необходимо использовать. 
  
На следующем рисунке показано, где каждый объект MAPI соответствует архитектуре MAPI. Объекты представлены именами их производных интерфейсов. Например, объект-источник консультации отображается как [IMAPIAdviseSink : IUnknown](imapiadvisesinkiunknown.md), интерфейс, который является потомком [IUnknown](https://msdn.microsoft.com/library/33f1d79a-33fc-4ce5-a372-e08bda378332%28Office.15%29.aspx) и который реализует каждый объект-источник рекомендации. Интерфейсы, мосты уровней используются или реализуются несколькими компонентами. Хотя уровень MAPI отделяет уровни клиента и поставщика, подразумевая, что все коммуникации должны проходить через MAPI, это не так. Клиенты могут взаимодействовать напрямую с объектами поставщика услуг.
