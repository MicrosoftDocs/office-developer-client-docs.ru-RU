<span data-ttu-id="9784f-110">Можно реализовать поставщик хранилища сообщений с помощью любого вида базовый механизм хранения продолжать.</span><span class="sxs-lookup"><span <span data-ttu-id="9784f-123">Диспетчер очереди MAPI и клиентов MAPI общаться с поставщиком хранилища сообщений, чтобы создать новые сообщения.</span><span class="sxs-lookup"><span data-stu-id="9784f-123">Both the MAPI spooler and MAPI clients communicate with the message store provider to create new messages.</span></span> <span data-ttu-id="9784f-124">Клиентские приложения для этого при создании сообщения пользователям.</span><span class="sxs-lookup"><span data-stu-id="9784f-124">Client applications do this when users compose a message.</span></span> <span data-ttu-id="9784f-125">Диспетчер очереди MAPI выполняется при получении входящего сообщения.</span><span class="sxs-lookup"><span data-stu-id="9784f-125">The MAPI spooler does this when it receives an incoming message.</span></span> <span data-ttu-id="9784f-126">В любом случае нового сообщения обычно создается в папке "Входящие" хранилища сообщений, если он существует.</span><span class="sxs-lookup"><span data-stu-id="9784f-126">In either case, the new message is usually created in the Inbox folder of the message store, if there is one.</span></span>
