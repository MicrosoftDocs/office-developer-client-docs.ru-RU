# <a name="upload-read-status-state"></a>Состояние отправки статуса чтения

  
  
**Применимо к**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время загрузки, прочитайте состояния состояние конечного автомата репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояний:  <br/> |**LR_SYNC_UPLOAD_MESSAGE_READ** <br/> |
|Структура связанных данных:  <br/> |**[UPREAD](upread.md)** <br/> |
|Из этого состояния:  <br/> |[Отправка состояний в таблице](upload-table-state.md) <br/> |
|Это состояние:  <br/> |Отправка состояний в таблице  <br/> |
   
> [!NOTE]
> Конечный автомат репликации является детерминированным конечного автомата. Клиент отправления одного состояния в другое должны в конечном счете возврата на первый из последний. 
  
## <a name="description"></a>Описание

Это состояние означает инициирует отправку состояние чтения элементов в папке, указанной в предыдущего состояния в таблице отправить. В этом состоянии Outlook инициализирует связанной структуры данных **UPREAD** со сведениями для этих элементов в папке чтения, состояние которого был изменен. Клиент затем обновляет состояние чтения этих элементов на сервере как прочитан или не прочитан. 
  
По окончании этого состояния Outlook очищает внутренние сведения о состоянии чтение элемента, предотвращения выгружаемого еще раз состояние чтения элемента. Локальное хранилище возвращается в состояние таблицы отправить.
