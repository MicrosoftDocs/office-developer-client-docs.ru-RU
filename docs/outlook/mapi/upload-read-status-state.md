# <a name="upload-read-status-state"></a>Состояние отправки состояния чтения

  
  
**Относится к**: Outlook 2013 | Outlook 2016 
  
 В этом разделе описывается, что происходит во время состояния отправки состояния чтения конечного автомата репликации. 
  
## <a name="quick-info"></a>Краткие сведения

|||
|:-----|:-----|
|Идентификатор состояния:  <br/> |**LR_SYNC_UPLOAD_MESSAGE_READ** <br/> |
|Связанная структура данных:  <br/> |**[UPREAD](upread.md)** <br/> |
|Из этого состояния:  <br/> |[Состояние отправки таблицы](upload-table-state.md) <br/> |
|В этом состоянии:  <br/> |Состояние отправки таблицы  <br/> |
   
> [!NOTE]
> Этот автомат репликации является детерминируемым. Клиент, выступая из одного состояния в другое, должен в конечном итоге вернуться к одному из последних. 
  
## <a name="description"></a>Описание

Это состояние инициирует отправку состояния чтения элементов в папке, указанной в предыдущем состоянии отправки таблицы. В этом состоянии Outlook инициализирует связанную структуру данных **UPREAD** с информацией для тех элементов в папке, состояние чтения которых изменилось. Затем клиент обновляет состояние чтения этих элементов на сервере как прочитанные или непрочитанные. 
  
Когда это состояние закончится, Outlook очищает внутренние сведения о состоянии чтения элемента, не мешая повторной отправке состояния чтения элемента. Локальное хранилище возвращается в состояние отправки таблицы.
