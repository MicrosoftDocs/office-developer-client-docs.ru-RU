# <a name="outlook-social-connector-provider-error-codes"></a>Коды ошибок поставщика Outlook Social Connector

Поставщики должны возвращать ошибки с абонентом, используя один из кодов ошибок, приведенные в следующей таблице. 
  
|**Error**|**Код ошибки (шестнадцатеричный)**|**Описание**|
|:-----|:-----|:-----|
|OSC_E_AUTH_ERROR  <br/> |0x80041404  <br/> |Ошибка при проверке подлинности в сети сайтов социальных сетей.  <br/> |
|OSC_E_COULDNOTCONNECT  <br/> |0x80041402  <br/> |Подключение недоступно для подключения к сайту социальных сетей.  <br/> |
|OSC_E_FAIL  <br/> |0x80004005  <br/> |Общий сбой.  <br/> |
|OSC_E_INTERNAL_ERROR  <br/> |0x80041400  <br/> |Внутренняя ошибка из-за недопустимой операции.  <br/> |
|OSC_E_INVALIDARG (E_INVALIDARG)  <br/> |0x80070057  <br/> |Указан недопустимый аргумент передан функции.  <br/> |
|OSC_E_NO_CHANGES  <br/> |0x80041406  <br/> |Изменения не были выполнены с момента последней синхронизации.  <br/> |
|OSC_E_NOT_FOUND  <br/> |0x80041405  <br/> |Не удается найти ресурс.  <br/> |
|OSC_E_NOT_IMPLEMENTED (E_NOTIMPL)  <br/> |0x80004001  <br/> |Запрос на сайт социальной сети указано правильно, но не был реализован сайтов социальных сетей.  <br/> |
|OSC_E_OUT_OF_MEMORY (E_OUTOFMEMORY)  <br/> |0x8007000E  <br/> |Произошла ошибка нехватки памяти.  <br/> |
|OSC_E_PERMISSION_DENIED  <br/> |0x80041403  <br/> |Поставщика OSC отказано в разрешении для ресурса.  <br/> |
|OSC_E_SERVER_VERSION_NOT_SUPPORTED  <br/> |0x80041406  <br/> |Версия сервера для настройки учетной записи социальной сети не поддерживается.  <br/> |
|OSC_E_VERSION  <br/> |0x80041401  <br/> |Поставщик не поддерживает эту версию возможности расширения поставщика OSC.  <br/> |
   
## <a name="remarks"></a>Замечания

Успех, предупреждение и ошибочные значения возвращаются с помощью 32-разрядное число, который будет вызываться дескриптор результатов или **HRESULT**. Значение **HRESULT** не маркер на что-либо; Это просто значение 32-разрядная версия, имеет несколько полей в значения кодировке. Положительных означает успешное выполнение с состоянием, нулевой результат означает успешное выполнение без состояния (S_OK) и отрицательным указывает на ошибку.
