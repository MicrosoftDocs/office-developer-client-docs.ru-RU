# <a name="outlook-social-connector-provider-error-codes"></a>Коды ошибок поставщика Outlook Social Connector

Поставщики должны возвращать вызываемой службе ошибки, используя один из кодов ошибок, показанных в следующей таблице. 
  
|**Ошибка**|**Код ошибки (hexadecimal)**|**Описание**|
|:-----|:-----|:-----|
|OSC_E_AUTH_ERROR  <br/> |0x80041404  <br/> |Не удалось проверить подлинность в сети сайта социальной сети.  <br/> |
|OSC_E_COULDNOTCONNECT  <br/> |0x80041402  <br/> |Подключение к сайту социальной сети отсутствует.  <br/> |
|OSC_E_FAIL  <br/> |0x80004005  <br/> |Общая ошибка сбоя.  <br/> |
|OSC_E_INTERNAL_ERROR  <br/> |0x80041400  <br/> |Возникла внутренняя ошибка из-за недопустимой операции.  <br/> |
|OSC_E_INVALIDARG (E_INVALIDARG)  <br/> |0x80070057  <br/> |Недопустимый аргумент передан функции.  <br/> |
|OSC_E_NO_CHANGES  <br/> |0x80041406  <br/> |С момента последней синхронизации изменений не было.  <br/> |
|OSC_E_NOT_FOUND  <br/> |0x80041405  <br/> |Не удается найти ресурс.  <br/> |
|OSC_E_NOT_IMPLEMENTED (E_NOTIMPL)  <br/> |0x80004001  <br/> |Запрос к сайту социальной сети действителен, но не реализован сайтом социальной сети.  <br/> |
|OSC_E_OUT_OF_MEMORY (E_OUTOFMEMORY)  <br/> |0x8007000E  <br/> |Произошла ошибка из-за потери памяти.  <br/> |
|OSC_E_PERMISSION_DENIED  <br/> |0x80041403  <br/> |Поставщик OSC отказано в разрешении для ресурса.  <br/> |
|OSC_E_SERVER_VERSION_NOT_SUPPORTED  <br/> |0x80041406  <br/> |Версия сервера для настройки учетной записи социальной сети не поддерживается.  <br/> |
|OSC_E_VERSION  <br/> |0x80041401  <br/> |Поставщик не поддерживает эту версию возможности extensibility поставщика OSC.  <br/> |
   
## <a name="remarks"></a>Примечания

Значения успешности, предупреждения и ошибки возвращаются с помощью 32-битного номера, называемого десналом результатов, или **HRESULT.** **HRESULT не** является дескультом для чего-либо; это просто 32-битное значение, в значении закодированное несколько полей. Положительный результат указывает на успешность с состоянием, нулевой результат означает успех без состояния (S_OK), а отрицательный результат указывает на сбой.
