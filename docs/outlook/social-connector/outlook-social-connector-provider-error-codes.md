# <a name="outlook-social-connector-provider-error-codes"></a>Outlook Коды ошибок поставщика социальных соединители

Поставщики должны возвращать ошибки звонячему с помощью одного из кодов ошибок, показанных в следующей таблице. 
  
|**Ошибка**|**Код ошибки (hexadecimal)**|**Описание**|
|:-----|:-----|:-----|
|OSC_E_AUTH_ERROR  <br/> |0x80041404  <br/> |В сети сайта социальной сети не удалось проверить подлинность.  <br/> |
|OSC_E_COULDNOTCONNECT  <br/> |0x80041402  <br/> |Подключение к сайту социальной сети не доступно.  <br/> |
|OSC_E_FAIL  <br/> |0x80004005  <br/> |Общая ошибка сбоя.  <br/> |
|OSC_E_INTERNAL_ERROR  <br/> |0x80041400  <br/> |Внутренняя ошибка произошла из-за недействительной операции.  <br/> |
|OSC_E_INVALIDARG (E_INVALIDARG)  <br/> |0x80070057  <br/> |Недействительный аргумент был передан функции.  <br/> |
|OSC_E_NO_CHANGES  <br/> |0x80041406  <br/> |После последней синхронизации изменений не произошло.  <br/> |
|OSC_E_NOT_FOUND  <br/> |0x80041405  <br/> |Ресурс не может быть найден.  <br/> |
|OSC_E_NOT_IMPLEMENTED (E_NOTIMPL)  <br/> |0x80004001  <br/> |Запрос на сайт социальной сети действителен, но не реализован сайтом социальной сети.  <br/> |
|OSC_E_OUT_OF_MEMORY (E_OUTOFMEMORY)  <br/> |0x8007000E  <br/> |Произошла ошибка из памяти.  <br/> |
|OSC_E_PERMISSION_DENIED  <br/> |0x80041403  <br/> |Поставщик OSC отказал в разрешении для ресурса.  <br/> |
|OSC_E_SERVER_VERSION_NOT_SUPPORTED  <br/> |0x80041406  <br/> |Версия сервера для настройки учетной записи социальной сети не поддерживается.  <br/> |
|OSC_E_VERSION  <br/> |0x80041401  <br/> |Поставщик не поддерживает эту версию extensibility поставщика OSC.  <br/> |
   
## <a name="remarks"></a>Примечания

Значения успешности, предупреждения и ошибок возвращаются с помощью 32-битного номера, называемого обработкой результатов, **или HRESULT.** **HRESULT** — это не ручка для чего-либо; это всего лишь 32-битное значение, в значении закодированное несколько полей. Положительный результат указывает на успех со статусом, нулевой результат указывает на успех без состояния (S_OK), а отрицательный результат указывает на сбой.
