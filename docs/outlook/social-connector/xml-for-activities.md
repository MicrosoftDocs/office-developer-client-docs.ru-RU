<span data-ttu-id="0f134-113">В приведенном ниже сценарии OSC динамически синхронизирует действия для пользователя, выбранного в области людей, и получает сведения об этом лице:</span><span class="sxs-lookup"><span data-stu-id="0f134-113">In the following scenario, the OSC dynamically synchronizes activities for a person selected in the People Pane, and gets details about that person:</span></span>
  
1. <span data-ttu-id="0f134-114">Поставщик OSC, поддерживающий синхронизацию действий по запросу, указывает, что для OSC используются элементы **динамикактивитиеслукупекс** **и.**</span><span class="sxs-lookup"><span data-stu-id="0f134-114">An OSC provider that supports on-demand synchronization of activities indicates that to the OSC by using the **getActivities** and **dynamicActivitiesLookupEx** elements.</span></span> <span data-ttu-id="0f134-115">Поставщик OSC также задает элемент **хашфунктион** .</span><span class="sxs-lookup"><span data-stu-id="0f134-115">The OSC provider also sets the **hashFunction** element.</span></span> <span data-ttu-id="0f134-116">Все три элемента являются дочерними элементами **возможностей**.</span><span class="sxs-lookup"><span data-stu-id="0f134-116">All three elements are child elements of **capabilities**.</span></span> 
    
2. <span data-ttu-id="0f134-117">Поставщик OSC реализует методы **исоЦиалпровидер:: Capabilities** и [ISocialSession2:: жетактивитиесекс](isocialsession2-getactivitiesex.md) .</span><span class="sxs-lookup"><span data-stu-id="0f134-117">The OSC provider implements the **ISocialProvider::GetCapabilities** and [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) methods.</span></span> 
    
3. <span data-ttu-id="0f134-118">OSC вызывает метод **исоЦиалпровидер::** и проверяет значение функций **динамикактивитиеслукупекс** и **getActivities** , чтобы убедиться, что поставщик OSC поддерживает синхронизацию действий по требованию.</span><span class="sxs-lookup"><span data-stu-id="0f134-118">The OSC calls **ISocialProvider::GetCapabilities** to check the value of **getActivities** and **dynamicActivitiesLookupEx** to verify that the OSC provider supports on-demand synchronization of activities.</span></span> <span data-ttu-id="0f134-119">В файле OSC также заменяется значение элемента **хашфунктион** , поддерживаемого поставщиком OSC.</span><span class="sxs-lookup"><span data-stu-id="0f134-119">The OSC also notes the value of the **hashFunction** element supported by the OSC provider.</span></span> 
    
4. <span data-ttu-id="0f134-120">OSC обновляет область пользователей или карточку контакта, чтобы позволить пользователю видеть последние действия выбранного человека.</span><span class="sxs-lookup"><span data-stu-id="0f134-120">The OSC refreshes the People Pane or Contact Card to let the user see the latest activities of the selected person.</span></span> <span data-ttu-id="0f134-121">OSC шифрует SMTP-адрес пользователя с помощью хэш-функции, указанной в элементе **хашфунктион** , формируя XML-строку, которая соответствует определению схемы XML для элемента **хашедаддрессес** .</span><span class="sxs-lookup"><span data-stu-id="0f134-121">The OSC encrypts the person's SMTP address by using the hash function specified in the **hashFunction** element, forming an XML string that conforms to the XML schema definition for the **hashedAddresses** element.</span></span> 
    
5. <span data-ttu-id="0f134-122">OSC вызывает метод **ISocialSession2:: жетактивитиесекс**, предоставляющую эту строку XML в качестве параметра _хашедаддрессес_ , чтобы получить текущую коллекцию действий для этого человека в параметре _activitys_ .</span><span class="sxs-lookup"><span data-stu-id="0f134-122">The OSC calls **ISocialSession2::GetActivitiesEx**, providing this XML string of the hashed address as the  _hashedAddresses_ parameter, to get a current collection of activities for that person in the  _activities_ parameter.</span></span> <span data-ttu-id="0f134-123">Строка параметра _activitys_ соответствует определению схемы XML элемента **activityFeed** .</span><span class="sxs-lookup"><span data-stu-id="0f134-123">The  _activities_ parameter string complies with the XML schema definition of the **activityFeed** element.</span></span> 
    
6. <span data-ttu-id="0f134-124">В зависимости от определения схемы XML для **activityFeed**, OSC выполняет синтаксический анализ строки _действий_ , чтобы узнать тип, дату публикации и другие сведения о каждом действии, а также как отобразить действие.</span><span class="sxs-lookup"><span data-stu-id="0f134-124">Based on the XML schema definition for **activityFeed**, the OSC further parses the  _activities_ string to find out the type, publish date, and other information about each activity, and how to display the activity.</span></span> 
    
7. <span data-ttu-id="0f134-125">Чтобы получить сведения о выбранном пользователе, OSC вызывает метод [ISocialSession2:: жетпеопледетаилс](isocialsession2-getpeopledetails.md), предоставляя ту же строку XML, что и аргумент для параметра _персонсаддрессес_ .</span><span class="sxs-lookup"><span data-stu-id="0f134-125">To get details about the selected person, the OSC calls [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), providing the same XML string of hashed addresses as the argument for the  _personsAddresses_ parameter.</span></span> <span data-ttu-id="0f134-126">Сведения о лице возвращаются в параметре _персонсколлектион_ .</span><span class="sxs-lookup"><span data-stu-id="0f134-126">The details about the person are returned in the  _personsCollection_ parameter.</span></span> <span data-ttu-id="0f134-127">Эти сведения могут включать **имя**, **фамилию**и **EmailAddress**.</span><span class="sxs-lookup"><span data-stu-id="0f134-127">These details can include **firstName**, **lastName**, and **emailAddress**.</span></span> <span data-ttu-id="0f134-128">Параметр _персонсколлектион_ соответствует определению схемы XML для элемента **Person** .</span><span class="sxs-lookup"><span data-stu-id="0f134-128">The  _personsCollection_ parameter conforms to the XML schema definition for the **person** element.</span></span>
