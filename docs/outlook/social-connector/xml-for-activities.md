<span data-ttu-id="dba8f-113">В следующем сценарии OSC динамически синхронизирует действий для человека, выбранного в области пользователей и возвращает подробные сведения об этом человеке:</span><span class="sxs-lookup"><span data-stu-id="dba8f-113">In the following scenario, the OSC dynamically synchronizes activities for a person selected in the People Pane, and gets details about that person:</span></span>

2. <span data-ttu-id="dba8f-117">Поставщика OSC реализует методы **ISocialProvider::GetCapabilities** и [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) .</span><span class="sxs-lookup"><span data-stu-id="dba8f-117">The OSC provider implements the **ISocialProvider::GetCapabilities** and [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) methods.</span></span> 
    
3. <span data-ttu-id="dba8f-118">Вызовов OSC **ISocialProvider::GetCapabilities** для проверки значения **getActivities** и **dynamicActivitiesLookupEx** чтобы убедиться, что поставщика OSC поддерживает синхронизацию по требованию действий.</span><span class="sxs-lookup"><span data-stu-id="dba8f-118">The OSC calls **ISocialProvider::GetCapabilities** to check the value of **getActivities** and **dynamicActivitiesLookupEx** to verify that the OSC provider supports on-demand synchronization of activities.</span></span> <span data-ttu-id="dba8f-119">OSC также указывается значение элемента **hashFunction** , поддерживаемых поставщиком OSC.</span><span class="sxs-lookup"><span data-stu-id="dba8f-119">The OSC also notes the value of the **hashFunction** element supported by the OSC provider.</span></span> 
    
4. <span data-ttu-id="dba8f-120">OSC обновляет области пользователей или карточки контакта, чтобы позволить пользователю видеть последние действия выбранного пользователя.</span><span class="sxs-lookup"><span data-stu-id="dba8f-120">The OSC refreshes the People Pane or Contact Card to let the user see the latest activities of the selected person.</span></span> <span data-ttu-id="dba8f-121">OSC шифрует его SMTP-адреса с помощью хэш-функции, заданная в элементе **hashFunction** формирование XML-строка, которая соответствует определение схемы XML для элемента **hashedAddresses** .</span><span class="sxs-lookup"><span data-stu-id="dba8f-121">The OSC encrypts the person's SMTP address by using the hash function specified in the **hashFunction** element, forming an XML string that conforms to the XML schema definition for the **hashedAddresses** element.</span></span> 
    
5. <span data-ttu-id="dba8f-122">OSC вызывает **ISocialSession2::GetActivitiesEx**, предоставление XML-строка хэш адреса в качестве параметра _hashedAddresses_ , чтобы получить текущий набор операций этого контакта с помощью параметра _действия_ .</span><span class="sxs-lookup"><span data-stu-id="dba8f-122">The OSC calls **ISocialSession2::GetActivitiesEx**, providing this XML string of the hashed address as the  _hashedAddresses_ parameter, to get a current collection of activities for that person in the  _activities_ parameter.</span></span> <span data-ttu-id="dba8f-123">Строки _действий_ параметра соответствует определение схемы XML в элемент **activityFeed** .</span><span class="sxs-lookup"><span data-stu-id="dba8f-123">The  _activities_ parameter string complies with the XML schema definition of the **activityFeed** element.</span></span> 
    
6. <span data-ttu-id="dba8f-124">На основании определения схемы XML для **activityFeed**, OSC дальнейшей анализирует строку _действия_ , которую нужно получить тип, публикация Дата и другие сведения о каждой операции и способ отображения действия.</span><span class="sxs-lookup"><span data-stu-id="dba8f-124">Based on the XML schema definition for **activityFeed**, the OSC further parses the  _activities_ string to find out the type, publish date, and other information about each activity, and how to display the activity.</span></span> 
    
7. <span data-ttu-id="dba8f-125">Чтобы получить подробные сведения о выбранного пользователя, OSC вызывает [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), предоставляя же XML-строка хэш адресов в качестве аргумента для параметра _personsAddresses_ .</span><span class="sxs-lookup"><span data-stu-id="dba8f-125">To get details about the selected person, the OSC calls [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), providing the same XML string of hashed addresses as the argument for the  _personsAddresses_ parameter.</span></span> <span data-ttu-id="dba8f-126">Подробные сведения об этом человеке возвращаются в параметре _personsCollection_ .</span><span class="sxs-lookup"><span data-stu-id="dba8f-126">The details about the person are returned in the  _personsCollection_ parameter.</span></span> <span data-ttu-id="dba8f-127">Эти сведения могут включать **firstName**, **lastName**и **emailAddress**.</span><span class="sxs-lookup"><span data-stu-id="dba8f-127">These details can include **firstName**, **lastName**, and **emailAddress**.</span></span> <span data-ttu-id="dba8f-128">Параметр _personsCollection_ соответствует определение схемы XML для элемента **контакта** .</span><span class="sxs-lookup"><span data-stu-id="dba8f-128">The  _personsCollection_ parameter conforms to the XML schema definition for the **person** element.</span></span>
