<span data-ttu-id="fcba3-113">В следующем сценарии OSC динамически синхронизирует действия для человека, выбранного в области "Люди", и получает сведения о нем:</span><span class="sxs-lookup"><span data-stu-id="fcba3-113">In the following scenario, the OSC dynamically synchronizes activities for a person selected in the People Pane, and gets details about that person:</span></span>

4. <span data-ttu-id="fcba3-120">OsC обновляет области пользователей или карточки контакта, чтобы пользователь видел последние действия выбранного пользователя.</span><span class="sxs-lookup"><span data-stu-id="fcba3-120">The OSC refreshes the People Pane or Contact Card to let the user see the latest activities of the selected person.</span></span> <span data-ttu-id="fcba3-121">OSC шифрует SMTP-адрес человека с помощью функции hashFunction, указанной в элементе **hashFunction,** образуя строку XML, которая соответствует определению схемы XML для элемента **hashedAddresses.**</span><span class="sxs-lookup"><span data-stu-id="fcba3-121">The OSC encrypts the person's SMTP address by using the hash function specified in the **hashFunction** element, forming an XML string that conforms to the XML schema definition for the **hashedAddresses** element.</span></span> 
    
5. <span data-ttu-id="fcba3-122">OSC вызывает **ISocialSession2::GetActivitiesEx,** предоставляя эту строку XML-адреса в качестве параметра _hashedAddresses,_ чтобы получить текущую коллекцию действий для этого человека в параметре _activities._</span><span class="sxs-lookup"><span data-stu-id="fcba3-122">The OSC calls **ISocialSession2::GetActivitiesEx**, providing this XML string of the hashed address as the  _hashedAddresses_ parameter, to get a current collection of activities for that person in the  _activities_ parameter.</span></span> <span data-ttu-id="fcba3-123">_Строка_ параметра activities соответствует определению схемы XML элемента **activityFeed.**</span><span class="sxs-lookup"><span data-stu-id="fcba3-123">The  _activities_ parameter string complies with the XML schema definition of the **activityFeed** element.</span></span> 
    
6. <span data-ttu-id="fcba3-124">На основе определения схемы XML для **activityFeed** OSC  дополнительно синтаксирует строку действий, чтобы узнать тип, дату публикации и другие сведения о каждом действии, а также способ отображения действия.</span><span class="sxs-lookup"><span data-stu-id="fcba3-124">Based on the XML schema definition for **activityFeed**, the OSC further parses the  _activities_ string to find out the type, publish date, and other information about each activity, and how to display the activity.</span></span> 
    
7. <span data-ttu-id="fcba3-125">Чтобы получить сведения о выбранном человеке, OSC вызывает [ISocialSession2::GetPeopleDetails,](isocialsession2-getpeopledetails.md)предоставляя ту же строку XML-адресов, что и аргумент _параметра personsAddresses._</span><span class="sxs-lookup"><span data-stu-id="fcba3-125">To get details about the selected person, the OSC calls [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), providing the same XML string of hashed addresses as the argument for the  _personsAddresses_ parameter.</span></span> <span data-ttu-id="fcba3-126">Сведения о человеке возвращаются в _параметре personsCollection._</span><span class="sxs-lookup"><span data-stu-id="fcba3-126">The details about the person are returned in the  _personsCollection_ parameter.</span></span> <span data-ttu-id="fcba3-127">Эти сведения могут включать **firstName,** **lastName** и **emailAddress.**</span><span class="sxs-lookup"><span data-stu-id="fcba3-127">These details can include **firstName**, **lastName**, and **emailAddress**.</span></span> <span data-ttu-id="fcba3-128">Параметр _personsCollection_ соответствует определению схемы XML для элемента **person.**</span><span class="sxs-lookup"><span data-stu-id="fcba3-128">The  _personsCollection_ parameter conforms to the XML schema definition for the **person** element.</span></span>
