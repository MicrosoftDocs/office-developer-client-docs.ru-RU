# <a name="xml-for-activities"></a>XML для действий

В этой статье приведен пример сценария, в котором показаны вызовы API расширения поставщика Outlook Social Connector (OSC), реализуемые поставщиком OSC, а также OSC для получения сведений о действиях. Сведения выражаются в XML-строках, соответствующих схеме XML поставщика OSC.
  
Схема XML поставщика OSC позволяет поставщику OSC определять действия. Сведения об активности могут включать социальные сети, в которых созданы элементы веб-канала активности, сведения о каждом элементе веб-канала активности (например, дату владельца, тип и время публикации действия), а также шаблон для отображения действия. Чтобы обеспечить поддержку отображения действий в области пользователей или карточке контакта, поставщик OSC в социальной сети должен реализовать и возвратить правильные XML-файлы действий. Пример XML-кода канала активности представлен в статье [Пример XML-канала активности](activity-feed-xml-example.md). Дополнительные сведения о синхронизации действий друзей приведены в статье [Синхронизация друзей и действий](synchronizing-friends-and-activities.md). Полное определение XML-схемы поставщика OSC, в том числе элементов, обязательных или необязательных, можно найти в статье [схема XML поставщика социальных соединителЕй Outlook](outlook-social-connector-provider-xml-schema.md). 
  
В приведенном ниже сценарии OSC динамически синхронизирует действия для пользователя, выбранного в области людей, и получает сведения об этом лице:
  
1. Поставщик OSC, поддерживающий синхронизацию действий по запросу, указывает, что для OSC используются элементы **динамикактивитиеслукупекс** и. **** Поставщик OSC также задает элемент **хашфунктион** . Все три элемента являются дочерними элементами **возможностей**. 
    
2. Поставщик OSC реализует методы **исоЦиалпровидер:: Capabilities** и [ISocialSession2:: жетактивитиесекс](isocialsession2-getactivitiesex.md) . 
    
3. OSC вызывает метод **исоЦиалпровидер::** и проверяет значение функций **динамикактивитиеслукупекс и** , чтобы убедиться, что поставщик OSC поддерживает синхронизацию действий по требованию. **** В файле OSC также заменяется значение элемента **хашфунктион** , поддерживаемого поставщиком OSC. 
    
4. OSC обновляет область пользователей или карточку контакта, чтобы позволить пользователю видеть последние действия выбранного человека. OSC шифрует SMTP-адрес пользователя с помощью хэш-функции, указанной в элементе **хашфунктион** , ФОРМИРУЯ XML-строку, которая соответствует определению схемы XML для элемента **хашедаддрессес** . 
    
5. OSC вызывает метод **ISocialSession2:: жетактивитиесекс**, предоставляющую эту строку XML в качестве параметра _хашедаддрессес_ , чтобы получить текущую коллекцию действий для этого человека в параметре _activitys_ . Строка __ параметра activitys соответствует определению схемы XML элемента **activityFeed** . 
    
6. В зависимости от определения схемы XML для **activityFeed**, OSC выполняет синтаксический анализ строки _действий_ , чтобы узнать тип, дату публикации и другие сведения о каждом действии, а также как отобразить действие. 
    
7. Чтобы получить сведения о выбранном пользователе, OSC вызывает метод [ISocialSession2:: жетпеопледетаилс](isocialsession2-getpeopledetails.md), предоставляя ту же строку XML, что и аргумент для параметра _персонсаддрессес_ . Сведения о лице возвращаются в параметре _персонсколлектион_ . Эти сведения могут включать **имя**, **фамилию**и **EmailAddress**. Параметр _персонсколлектион_ соответствует определению схемы XML для элемента **Person** .
