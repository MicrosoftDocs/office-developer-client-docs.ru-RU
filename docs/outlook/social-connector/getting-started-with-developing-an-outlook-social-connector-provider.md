# <a name="getting-started-with-developing-an-outlook-social-connector-provider"></a>Начало работы с разработкой Outlook поставщика социальных соединители

Ссылка Outlook поставщика социального соединителя (OSC) описывает, как разработать поставщика OSC с помощью extensibility поставщика OSC. Если вы не разрабатываете решения для Outlook, см. в примере Выбор API или технологии для разработки Outlook решений для определения [API](https://msdn.microsoft.com/library/8295da20-e567-4d08-b8e4-5c9b4498edd4%28Office.15%29.aspx) и технологий, наиболее подходящих для ваших потребностей. 

В этом разделе приводится обзор osC, как поставщик OSC может быть полезен, быстрые шаги для обучения разработке поставщика, технические требования, лучшие практики для разработки поставщика и что нового в этом выпуске. 
  
## <a name="in-this-section"></a>В этом разделе

- [Что нового](what-s-new-for-providers.md)для поставщиков: сравнивает функции OSC в предыдущем и текущем выпуске и описывает участников интерфейса и элементы схемы XML, которые были добавлены, изменены или обескровлены в этом выпуске. 
    
- [Почему разработка поставщика Outlook](why-develop-an-outlook-social-connector-provider.md)социальных соединители: описывает, как поставщик OSC может быть полезен для общих сайтов социальных сетей и других внутренних сетевых инструментов.
    
- [Связь OSC](relating-the-osc-with-outlook-and-social-networks.md)с Outlook и социальными сетями: предоставляет архитектурное представление о том, как OSC соединяет Outlook пользователей с социальными сетями. Кроме того, определяет способ использования общих терминов, таких как "социальные сети", "друзья", "не-друзья" и "контакты".
    
- [Быстрые шаги по обучению разработке поставщика:](quick-steps-for-learning-to-develop-a-provider.md)предоставляет сводку действий, чтобы научиться разрабатывать поставщика OSC.
    
- [Технические требования.](technical-requirements.md)Описывает поддерживаемые языки программирования, требования к видимости COM, требования к типу возврата метода и сведения о DLL поставщика extensibility osC.
    
- [Лучшие практики для разработки поставщика:](best-practices-for-developing-a-provider.md)предоставляет список наиболее оптимальных методов, которые следует соблюдать при разработке поставщика OSC.
