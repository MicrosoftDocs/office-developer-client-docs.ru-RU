# <a name="xml-for-friends"></a>XML для друзей

Элемент **друзей** в XML-схеме поставщика Social Connector Microsoft Outlook (OSC) позволяет поставщика OSC для указания сведений список лиц, связанных с пользователей Outlook в социальных сетях. Если поставщика OSC поддерживает кэширования данных синхронизации, этот список пользователя будет содержать только друзей пользователя Outlook в социальных сетях. Если OSC поддерживает синхронизации по запросу или комбинированным, этот список может содержать как друзья и не-пользователя Outlook. 

Каждого пользователя в списке представлен как элемент **человека** в XML-схему, которая поддерживает подробные сведения, такие как имя, Фамилия и адреса электронной почты. Поставщики OSC используйте элементы **друзей** и **пользователя** независимо от того, каким образом требуется OSC, чтобы синхронизировать данные friend из социальных сетей. Обратите внимание, что дочерние элементы **человека** похожи на некоторые свойства контакта Outlook, которое упрощает хранение друзей в папке контактов Outlook определенного социальные сети, если кэширования для поддержки социальной сети или гибридного Синхронизация друзей в папке контактов Outlook. 

## <a name="example-scenarios"></a>Примеры сценариев использования

В следующих сценариях примере показано поставщика OSC вызовов API расширяемости, реализует поставщика OSC и OSC делает для получения информации из friend. Сведения представлены в строки XML, которые соответствуют XML-схему поставщика OSC.
  
В качестве примера друзья XML видеть [Друзья примере XML-кода](friends-xml-example.md). Дополнительные сведения о синхронизации сведений о друзьях можно [Синхронизация друзей и действия](synchronizing-friends-and-activities.md).

### <a name="scenario-1-get-a-list-of-friends"></a>Сценарий 1: получение списка друзей

Сценарий 1 — OSC возвращает список друзей и объект [ISocialPerson](isocialpersoniunknown.md) и изображения для каждого friend: 
    
1. Поставщика OSC, обеспечивающие отображение друзья с сайта социальной сети и позволяя OSC кэширования данных friend указывает, что для OSC с помощью элементы **getFriends** и **cacheFriends** , которые являются дочерними элементами ** возможности** элемент. 
    
2. Поставщика OSC также реализует [ISocialProvider::GetCapabilities](isocialprovider-getcapabilities.md), [ISocialSession::GetPerson](isocialsession-getperson.md), [ISocialPerson::GetFriendsAndColleagues](isocialperson-getfriendsandcolleagues.md)и [ISocialPerson::GetPicture](isocialperson-getpicture.md) методы. 
    
3. OSC вызывает **ISocialProvider::GetCapabilities** для проверки значения из следующих элементов: **getFriends** , чтобы убедиться, что для поддержки поставщика OSC, отражающая друзей из социальных сетей и **cacheFriends** чтобы убедиться, что поставщик поддерживает кэширование друзей. 
    
4. OSC вызывает **ISocialSession::GetPerson** для получения объекта **ISocialPerson** для пользователя Outlook. 
    
5. OSC вызывает **ISocialPerson::GetFriendsAndColleagues** для получения Outlook пользователя в список друзей, возвращаемые в строке параметров _personCollection_ . Строка _personCollection_ стандарту определение схемы XML для элемента **друзей** в XML-схему. 
    
6. Для каждого друга в XML-строка _personCollection_ OSC получает значение элемента **идентификатор пользователя** для вызова **ISocialSession::GetPerson** для получения объекта **ISocialPerson** для этого friend. 
    
7. Для каждого друга в XML-строка **personCollection** OSC вызывает [ISocialPerson::GetPicture](isocialperson-getpicture.md) для получения ресурсов изображения для этого friend. 
    
   OSC внести вызовы объекта **ISocialPerson** для получения сведений о (например, адреса электронной почты) и действия для этого friend. 
    
### <a name="scenario-2-synchronize-friends"></a>Сценарий 2: синхронизация друзей 

Сценарий 2 — OSC синхронизирует друзья динамически:
    
1. Поставщика OSC, которая поддерживает синхронизацию по требованию друзья и не указывает, что для OSC с помощью элементов **getFriends** и **dynamicContactsLookup** . Поставщика OSC также задает элемент **hashFunction** . Все три элемента являются дочерними элементами **возможностей**. 
    
2. Поставщика OSC также реализует метод [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md) . 
    
3. OSC вызывает **ISocialProvider::GetCapabilities** для проверки значения **getFriends** и **dynamicContactsLookup** , чтобы убедиться, что поставщика OSC поддерживает друзей и синхронизации по требованию друзья и не. OSC делает Примечание значение **hashFunction** поддерживается поставщиком OSC. 
    
4. Для каждого пользователя, отображаемые в области пользователей OSC собирает адрес электронной почты пользователя и шифрует его с помощью хэш-функции, указанной в **hashFunction**. Это форм XML-строка, которая соответствует определение схемы XML для элемента **hashedAddresses** . 
    
5. OSC вызывает **ISocialSession2::GetPeopleDetails**, предоставляя XML-строка хэш адресов как параметр _personAddresses_ динамически получить обновленные сведения для пользователей с помощью параметра _personsCollection_ . Строка параметра _personsCollection_ стандарту определение схемы XML для элемента **друзей** в XML-схему.
