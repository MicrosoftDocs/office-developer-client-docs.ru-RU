# <a name="synchronizing-friends-and-activities"></a>Синхронизация друзей и действий

Outlook Social Connector (OSC) поддерживает отображение данных из социальных сетей о человеке в карточке контакта или в области пользователей Outlook. SharePoint Server, SharePoint Workspace, клиент Lync и все клиентские приложения Office, которые поддерживают сведения о присутствии поддерживает карточки контакта.
  
Карточки контакта можно использовать в сценарии совместной работы в приложениях Office, чтобы узнать больше о людей, с которыми совместная работа с. Эти сценарии примеры системы обмена сообщениями в Outlook и совместного редактирования документов в Word. Если щелкнуть вкладку **новые возможности** в карточке контакта отображаются сведения об этом человеке. 
  
Области пользователей Outlook отображается информация о человека, который может быть отправителя или получателя выбранного элемента Outlook. В случае выберите другой пользователь в области пользователей или другой элемент в обозревателе Outlook или открытие элемента Outlook в инспекторе Outlook Social Connector (OSC) обновляет области пользователей. 
  
Для карточки контакта или области пользователей, для отображения текущих сведений для выбранного пользователя OSC синхронизирует такие данные, поставщики OSC и некоторые формы кэширования. В этом синхронизации зависит от поставщики OSC, установленных на клиентском компьютере, социальные сети, что вы вошли на через их поставщики OSC и режим синхронизации, что каждый из поставщиков OSC для этих социальных сетей поддерживает.
  
OSC поддерживает синхронизации друзей, не являющиеся друзей и действия для друзья и не различными способами: кэширования данных синхронизации, синхронизация по запросу и гибридного синхронизации. — Это основное различие между эти режимы синхронизации, где OSC хранит данные, будет ли это в папке в хранилище Outlook по умолчанию, или в памяти на компьютере пользователя. В обоих случаях как было указано в этом разделе имеется по умолчанию минимальное время данные остаются в папке или объем памяти до обновления данных. В некоторых случаях минимальное количество времени, можно настроить с помощью групповой политики. Дополнительные сведения о групповых политик, которые управляют поведением OSC Узнайте, [как управление Outlook Social Connector с помощью групповой политики](https://support.microsoft.com/default.aspx?scid=kb%3Ben-US%3B2020103).
  
Обратите внимание, что если выбранного пользователя не должна быть членом социальных сетей, OSC не отображает какие-либо лицо или активности сведения об этом контакте в карточке контакта или области пользователей.
  
## <a name="cached-synchronization"></a>Кэшированные синхронизации

Поставщика OSC можно хранить сведения о друзьях в социальных сетях в определенной папке в хранилище Outlook по умолчанию и периодически обновлять этот кэш по истечении указанного периода времени. Кэширование данных в папке имеет преимущество сокращения трафика в социальной сети.
  
> [!NOTE]
> Начиная с Outlook Social Connector 2013 OSC больше не поддерживает кэшированные синхронизации действий. 
  
### <a name="cached-synchronization-of-friends"></a>Кэшированные синхронизации друзей

Если поставщика OSC поддерживает кэшированные синхронизация для друзей, OSC кэширует сведений о друзьях пользователя, вошедшего в систему на социальных сетей. Сведения о кэшируются в папке контактов Outlook, относящуюся к этой социальной сети в хранилище Outlook по умолчанию. Имя папки Контакты основано на имя социальной сети, который получает OSC с помощью свойства [ISocialProvider::SocialNetworkName](isocialprovider-socialnetworkname.md) . 
  
В кэшированной синхронизации OSC хранятся сведения только друзей при входе пользователя в социальных сетях. OSC не осуществляет доступ к информации, не являющиеся друзей.
  
Интервал по умолчанию для OSC обновить папку «Контакты» в социальных сетях сведения друзей — один раз в день (или один раз в 1440 минут). В этом интервал обновления также можно установить с помощью групповой политики, как описано в начале этого раздела.
  
При возникновении ошибки во время обновления, OSC повторов периодичностью, который указан с помощью элемента **contactSyncRestartInterval** в **возможности** XML. В этом интервал повтора имеет значение по умолчанию 30 минут, а также можно установить с помощью групповой политики. 
  
Когда пользователь открывает карточки контакта и Выбор вкладки **новые** , обновляет **новые** вкладки. Аналогично когда пользователь Outlook reselects элемента в Outlook или reselects человека в области пользователей, обновляет области пользователей. Если не истек интервал обновления кэша, OSC переходит в кэш, чтобы получить какие-либо сведения для выбранного пользователя. Это позволяет избежать затрат на использование возможности расширения поставщика OSC для доступа к социальных сетей. Если истек интервал обновления, OSC вызывает метод [ISocialPerson::GetFriendsAndColleagues](isocialperson-getfriendsandcolleagues.md) для получения сведений о текущем друзьях для пользователя, вошедшего в систему и обновляет кэш в папке «Контакты». 
  
Поставщика OSC информирует OSC, что он поддерживает кэшированные синхронизации друзья, указав следующие элементы в **возможности** XML: 
  
- **getFriends** = **значение true**
    
- **cacheFriends** = **значение true**
    
- **dynamicContactsLookup** = **значение false**
    
## <a name="on-demand-synchronization"></a>Синхронизация по запросу

Если пользователь выбирает вкладку **новые возможности** в карточке контакта или выбирает другой элемент Outlook или другое лицо в области пользователей в Outlook, OSC обновляет карточки контакта или области пользователей соответственно. Если поставщика OSC поддерживает синхронизацию по требованию лиц или действия, OSC синхронизируется с кэша в памяти и обновляет сведений, таких как имя, заголовок, рисунков и активности потоков, в карточке контакта или в области пользователей. Для синхронизации по запросу, в отличие от кэшированных синхронизации OSC пытается обновить данные о для пользователя независимо от того, является ли он friend или не friend пользователя, вошедшего в систему в социальных сетях. 
  
По запросу пользователя (или действия) данные сохраняются только в памяти. Данные в памяти будет очищен при клиентского приложения Office завершает работу, или пользователь приводит к обновлению из карточки контакта или области пользователей и данных остается в памяти для больше, чем интервал обновления. Обратите внимание, что обновление из социальных сетей всегда инициируется пользователь обновлять карточки контакта или области пользователей (например, путем выбора другого пользователя в области пользователей или Выбор другой элемент в окне проводника Outlook). 

Тем не менее, не наоборот всегда — не все обновления из карточки контакта или области пользователей обязательно приводит к обновления из социальных сетей. Если пользователь обновляет карточки контакта или области пользователей и человека (или активности) данных оставалась в памяти для больше, чем интервал обновления, OSC вызовы [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md) (или [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md)) обновление данных в памяти из социальных сетей. Допустимый период friend и не являющиеся friend сведения в памяти — 24 часов и для мероприятий, 30 минут. 
  
Один важные различия между синхронизации кэширования и по запросу —, что синхронизация по запросу извлечь сведения человека и активности друзей и не являющиеся друзья в сети. Если выбранного пользователя не друга, OSC обновляет сведения и данные об этом контакте Если любое из следующих требований: 
  
- Человека — это пользователь, социальные сети и разрешает общедоступных просмотра сведения профиля и действия.
    
- Пользователь находится в сети, в которой вошедшего в систему пользователя, социальные сети (например, в одной сети для выпускникам университета).
    
Синхронизация по запросу результатов лиц и действия в нескольких вызовов поставщика из ядра OSC. Социальные сети должен быть способен обрабатывать требования к увеличению пропускной способности синхронизации по запросу.
  
### <a name="specifying-xml-elements-for-on-demand-synchronization"></a>Указание XML-элементы для синхронизации по запросу

Поставщика OSC информирует OSC он поддерживает синхронизацию по требованию друзей и не являющиеся друзья, указав следующие элементы в **возможности** XML: 
  
- **getFriends** = **значение true**
    
- **cacheFriends** = **значение false**
    
- **dynamicContactsLookup** = **значение true**
    
Поставщика OSC информирует OSC его поддерживает синхронизацию по требованию действий, указав следующие элементы в **возможности** XML: 
  
- **getActivities** = **значение true**
    
- **cacheActivities** = **значение false**
    
- **dynamicActivitiesLookupEx** = **значение true**
    
## <a name="hybrid-synchronization"></a>Гибридные синхронизации

Поставщика OSC можно обеспечить поддержку синхронизации гибридного друзья и не. Это можно оптимизировать вызовы между ядра OSC и поставщика OSC, вызовы социальной сети для синхронизации по запросу друзей и валюты друзей своих данных. Минимальное время данные могут оставаться в папке или памяти, где это возможно, — это то же, что ограничения в режимах синхронизации кэшированных или по запросу.
  
> [!NOTE]
> Начиная с Outlook Social Connector 2013 OSC поддерживает только по запросу синхронизации действий и больше не поддерживает гибридные синхронизации действий. 
  
### <a name="hybrid-synchronization-of-friends-and-non-friends"></a>Синхронизация гибридного друзья и не

Если поставщика OSC поддерживает синхронизацию гибридного друзья и не, OSC выполняет следующие действия: 
  
- OSC хранятся сведения для друзей пользователя, вошедшего в систему социальные сети конкретных папки "Контакты".
    
- OSC хранятся сведения не друзей пользователя, вошедшего в систему в памяти.
    
Поставщика OSC информирует OSC он поддерживает гибридные синхронизации друзей и не являющиеся друзья, указав следующие элементы в **возможности** XML: 
  
- **getFriends** = **значение true**
    
- **cacheFriends** = **значение true**
    
- **dynamicContactsLookup** = **значение true**
