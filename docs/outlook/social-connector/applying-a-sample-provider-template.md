### <a name="to-apply-an-osc-provider-template"></a><span data-ttu-id="4970e-108">Применение шаблона поставщика OSC</span><span class="sxs-lookup"><span data-stu-id="4970e-108">To apply an OSC provider template</span></span>

1. <span data-ttu-id="4970e-109">В меню " **Пуск** " щелкните правой кнопкой мыши элемент **Microsoft Visual Studio 2010** и выберите команду **Запуск от имени администратора** .</span><span class="sxs-lookup"><span data-stu-id="4970e-109">On the **Start** menu, right-click **Microsoft Visual Studio 2010** and click the **Run as administrator** command.</span></span> <span data-ttu-id="4970e-110">При появлении соответствующего запроса нажмите кнопку **Да** , чтобы запустить Visual Studio от имени администратора.</span><span class="sxs-lookup"><span data-stu-id="4970e-110">When prompted, click **Yes** to run Visual Studio as an administrator.</span></span> 
    
2. <span data-ttu-id="4970e-111">Измените имя проекта и пространство имен в шаблоне на имя проекта и идентификаторы пространства имен.</span><span class="sxs-lookup"><span data-stu-id="4970e-111">Change the project name and namespace in the template to your project name and namespace identifiers.</span></span>
    
3. <span data-ttu-id="4970e-112">Измените класс **AssemblyInfo** , указав соответствующие сведения о сборке.</span><span class="sxs-lookup"><span data-stu-id="4970e-112">Modify the **AssemblyInfo** class to specify the appropriate assembly information.</span></span> 
    
4. <span data-ttu-id="4970e-113">Реализуйте элементы интерфейса, помеченные как **задачи** , и добавляйте дополнительные зависимости и ссылки, если необходимо.</span><span class="sxs-lookup"><span data-stu-id="4970e-113">Implement the interface members marked as **To-Do** and add more dependencies and references, as required.</span></span> 
    
5. <span data-ttu-id="4970e-114">Выполните построение проекта.</span><span class="sxs-lookup"><span data-stu-id="4970e-114">Build the project.</span></span>
    
6. <span data-ttu-id="4970e-115">Убедитесь, что идентификатор ProgID сборки поставщика указан как ключ `HKEY_CURRENT_USER\Software\Microsoft\Office\Outlook\SocialConnector\SocialProviders`.</span><span class="sxs-lookup"><span data-stu-id="4970e-115">Ensure that the provider assembly ProgID is listed as a key under  `HKEY_CURRENT_USER\Software\Microsoft\Office\Outlook\SocialConnector\SocialProviders`.</span></span>
    
7. <span data-ttu-id="4970e-116">Чтобы распространить проект установки, создайте проект установки в Visual Studio или средство установки.</span><span class="sxs-lookup"><span data-stu-id="4970e-116">To distribute the setup project, create a setup project in Visual Studio or a setup tool of your choice.</span></span>
    
8. <span data-ttu-id="4970e-117">Проект установки должен завершить регистрацию COM для сборки, а также создать ключ ProgID, как указано в шаге 5.</span><span class="sxs-lookup"><span data-stu-id="4970e-117">Your setup project should complete COM registration for your assembly and also create the ProgID key as listed in step 5.</span></span>
