<span data-ttu-id="679a2-104">Outlook Поставщики социальных соединителов (OSC) могут установить элементы **getActivities** и **dynamicActivitiesLookupEx** для хранения элементов активности OSC в памяти.</span><span class="sxs-lookup"><span data-stu-id="679a2-104">Outlook Social Connector (OSC) providers can set the **getActivities** and **dynamicActivitiesLookupEx** elements to have the OSC store activity items in memory.</span></span> <span data-ttu-id="679a2-105">В этом разделе описывается API-API-службы поставщика extensibility OSC, которые osC вызывает для получения или обновления сведений владельца действий и действий, если поставщик OSC поддерживает синхронизацию каналов активности из социальной сети.</span><span class="sxs-lookup"><span data-stu-id="679a2-105">This topic describes the OSC provider extensibility APIs that the OSC calls to get or refresh activities and activity owner details, if the OSC provider supports synchronizing activity feeds from the social network.</span></span> <span data-ttu-id="679a2-106">В этой теме также перечислены несколько детских элементов элемента **activityFeed,** который поставщик OSC должен установить для OSC для правильного отображения действий в Office контактной карточке или Outlook области людей.</span><span class="sxs-lookup"><span data-stu-id="679a2-106">The topic also lists a few child elements of the **activityFeed** element that the OSC provider should set for the OSC to display activities properly in the Office Contact Card or Outlook People Pane.</span></span> 
  
- <span data-ttu-id="679a2-107">OsC вызывает [метод ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) для получения и хранения действий в папке Ленты новостей для зарегистрированного пользователя.</span><span class="sxs-lookup"><span data-stu-id="679a2-107">The OSC calls the [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) method to get and store activities in the News Feed folder for the logged-on user.</span></span> <span data-ttu-id="679a2-108">Поставщик OSC должен реализовать **GetActivitiesEx,** чтобы вернуть строку _XML_ действий, которая соответствует определению схемы XML-схемы поставщика OSC элемента **activityFeed.**</span><span class="sxs-lookup"><span data-stu-id="679a2-108">The OSC provider must implement **GetActivitiesEx** to return an  _activities_ XML string that complies with the OSC provider XML schema definition of the **activityFeed** element.</span></span> 
    
- <span data-ttu-id="679a2-109">Поставщик OSC должен установить **элемент ownerID,** который является детским элементом **элемента activityDetails.**</span><span class="sxs-lookup"><span data-stu-id="679a2-109">The OSC provider must set the **ownerID** element, which is a child element of the **activityDetails** element.</span></span> <span data-ttu-id="679a2-110">**ownerID** — непрозраченная строка, определяемая владельцем действия в социальной сети.</span><span class="sxs-lookup"><span data-stu-id="679a2-110">**ownerID** is an opaque string that identifies the owner of the activity on the social network.</span></span> 
    
- <span data-ttu-id="679a2-111">Поставщик OSC должен установить элементы **nameHint** и **emailAddress** в **узле publisherVariable** элемента **templateVariables.**</span><span class="sxs-lookup"><span data-stu-id="679a2-111">The OSC provider should set the **nameHint** and **emailAddress** elements in the **publisherVariable** node of the **templateVariables** element.</span></span> 
    
   <span data-ttu-id="679a2-112">Обратите внимание, что в схеме XML поставщика OSC элемент **nameHint** является необязательным элементом.</span><span class="sxs-lookup"><span data-stu-id="679a2-112">Note that per the OSC provider XML schema, the **nameHint** element is an optional element.</span></span> <span data-ttu-id="679a2-113">OsC использует его, чтобы соответствовать отображаемой фамилии пользователя, выбранного в карточке контакта или области людей.</span><span class="sxs-lookup"><span data-stu-id="679a2-113">The OSC uses it to match the display name of the user selected in the Contact Card or People Pane.</span></span> <span data-ttu-id="679a2-114">Аналогичным образом элемент **emailAddress** является необязательным элементом схемы XML.</span><span class="sxs-lookup"><span data-stu-id="679a2-114">Similarly, the **emailAddress** element is an optional element in the XML schema.</span></span> <span data-ttu-id="679a2-115">OsC использует его для совпадения SMTP-адреса пользователя, выбранного в карточке контакта или в области людей.</span><span class="sxs-lookup"><span data-stu-id="679a2-115">The OSC uses it to match the SMTP address of the user selected in the Contact Card or People Pane.</span></span> 
    
   <span data-ttu-id="679a2-116">Если указан только элемент **ownerID,** но одно или оба **имениHint** и **emailAddress** не указаны, OSC вызывает [метод ISocialSession2::GetPeopleDetails,](isocialsession2-getpeopledetails.md) а затем метод [ISocialPerson::GetDetails,](isocialperson-getdetails.md) чтобы получить дополнительные сведения о человеке, идентифицированного **ownerID**.</span><span class="sxs-lookup"><span data-stu-id="679a2-116">If only the **ownerID** element is specified, but one or both of **nameHint** and **emailAddress** are not specified, the OSC calls the [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md) method and then the [ISocialPerson::GetDetails](isocialperson-getdetails.md) method to get more information about the person identified by the **ownerID**.</span></span> <span data-ttu-id="679a2-117">Когда OSC вызывает **ISocialPerson::GetDetails,** поставщик должен вернуть человеку **XML,** который указывает элементы **fullName** и **emailAddress.**</span><span class="sxs-lookup"><span data-stu-id="679a2-117">When the OSC calls **ISocialPerson::GetDetails**, the provider must return **person** XML that specifies the **fullName** and **emailAddress** elements.</span></span>
