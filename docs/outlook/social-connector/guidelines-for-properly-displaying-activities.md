# <a name="guidelines-for-properly-displaying-activities"></a><span data-ttu-id="aee6a-103">Рекомендации по правильному отображению действий</span><span class="sxs-lookup"><span data-stu-id="aee6a-103">Guidelines for properly displaying activities</span></span>

<span data-ttu-id="aee6a-104">Поставщики Outlook Social Connector (OSC) могут настроить элементы **getActivities** и **dynamicActivitiesLookupEx** для хранения элементов активности OSC в памяти.</span><span class="sxs-lookup"><span data-stu-id="aee6a-104">Outlook Social Connector (OSC) providers can set the **getActivities** and **dynamicActivitiesLookupEx** elements to have the OSC store activity items in memory.</span></span> <span data-ttu-id="aee6a-105">В этом разделе описываются API-aPI для обеспечения возможности работы поставщика OSC, которые OSC вызывает для получения или обновления действий и сведений владельца действия, если поставщик OSC поддерживает синхронизацию веб-каналов активности из социальной сети.</span><span class="sxs-lookup"><span data-stu-id="aee6a-105">This topic describes the OSC provider extensibility APIs that the OSC calls to get or refresh activities and activity owner details, if the OSC provider supports synchronizing activity feeds from the social network.</span></span> <span data-ttu-id="aee6a-106">В этом разделе также перечислены несколько элементов элемента **activityFeed,** которые поставщик OSC должен настроить для OSC для правильного отображения действий в карточке контакта Office или в области людей Outlook.</span><span class="sxs-lookup"><span data-stu-id="aee6a-106">The topic also lists a few child elements of the **activityFeed** element that the OSC provider should set for the OSC to display activities properly in the Office Contact Card or Outlook People Pane.</span></span> 
  
- <span data-ttu-id="aee6a-107">OsC вызывает метод [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) для получения и хранения действий в папке "Канал новостей" для пользователя, выполнимого при входе в систему.</span><span class="sxs-lookup"><span data-stu-id="aee6a-107">The OSC calls the [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) method to get and store activities in the News Feed folder for the logged-on user.</span></span> <span data-ttu-id="aee6a-108">Поставщик OSC должен реализовать **GetActivitiesEx,** чтобы вернуть  XML-строку действий, которая соответствует определению XML-схемы поставщика OSC элемента **activityFeed.**</span><span class="sxs-lookup"><span data-stu-id="aee6a-108">The OSC provider must implement **GetActivitiesEx** to return an  _activities_ XML string that complies with the OSC provider XML schema definition of the **activityFeed** element.</span></span> 
    
- <span data-ttu-id="aee6a-109">Поставщик OSC должен установить элемент **ownerID,** который является ключевым элементом **элемента activityDetails.**</span><span class="sxs-lookup"><span data-stu-id="aee6a-109">The OSC provider must set the **ownerID** element, which is a child element of the **activityDetails** element.</span></span> <span data-ttu-id="aee6a-110">**ownerID** — непрозраченная строка, идентифицирует владельца действия в социальной сети.</span><span class="sxs-lookup"><span data-stu-id="aee6a-110">**ownerID** is an opaque string that identifies the owner of the activity on the social network.</span></span> 
    
- <span data-ttu-id="aee6a-111">Поставщик OSC должен установить **элементы nameHint** и **emailAddress** в узле **publisherVariable** элемента **templateVariables.**</span><span class="sxs-lookup"><span data-stu-id="aee6a-111">The OSC provider should set the **nameHint** and **emailAddress** elements in the **publisherVariable** node of the **templateVariables** element.</span></span> 
    
   <span data-ttu-id="aee6a-112">Обратите внимание, что в XML-схеме поставщика OSC элемент **nameHint** является необязательным элементом.</span><span class="sxs-lookup"><span data-stu-id="aee6a-112">Note that per the OSC provider XML schema, the **nameHint** element is an optional element.</span></span> <span data-ttu-id="aee6a-113">OsC использует его для совпадения с отображаемым именем пользователя, выбранного в карточке контакта или области "Люди".</span><span class="sxs-lookup"><span data-stu-id="aee6a-113">The OSC uses it to match the display name of the user selected in the Contact Card or People Pane.</span></span> <span data-ttu-id="aee6a-114">Аналогичным образом элемент **emailAddress** является необязательным элементом в схеме XML.</span><span class="sxs-lookup"><span data-stu-id="aee6a-114">Similarly, the **emailAddress** element is an optional element in the XML schema.</span></span> <span data-ttu-id="aee6a-115">OsC использует его для связи с SMTP-адресом пользователя, выбранного в карточке контакта или области "Люди".</span><span class="sxs-lookup"><span data-stu-id="aee6a-115">The OSC uses it to match the SMTP address of the user selected in the Contact Card or People Pane.</span></span> 
    
   <span data-ttu-id="aee6a-116">Если указан только элемент **ownerID,** но один или оба элемента **nameHint** и **emailAddress** не указаны, OSC вызывает метод [ISocialSession2::GetPeopleDetails,](isocialsession2-getpeopledetails.md) а затем метод [ISocialPerson::GetDetails,](isocialperson-getdetails.md) чтобы получить дополнительные сведения о человеке, определенном **по ownerID.**</span><span class="sxs-lookup"><span data-stu-id="aee6a-116">If only the **ownerID** element is specified, but one or both of **nameHint** and **emailAddress** are not specified, the OSC calls the [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md) method and then the [ISocialPerson::GetDetails](isocialperson-getdetails.md) method to get more information about the person identified by the **ownerID**.</span></span> <span data-ttu-id="aee6a-117">Когда OSC вызывает **ISocialPerson::GetDetails,** поставщик должен вернуть  XML-код пользователя, который указывает элементы **fullName** и **emailAddress.**</span><span class="sxs-lookup"><span data-stu-id="aee6a-117">When the OSC calls **ISocialPerson::GetDetails**, the provider must return **person** XML that specifies the **fullName** and **emailAddress** elements.</span></span>
