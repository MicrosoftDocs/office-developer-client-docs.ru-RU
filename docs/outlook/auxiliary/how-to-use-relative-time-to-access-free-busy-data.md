# <a name="use-relative-time-to-access-freebusy-data"></a>Использование относительного времени для доступа к свободным и загруженным данным

Интерфейс [IFreeBusyData](ifreebusydata.md) в API Free/Busy использует понятие относительного времени, которое является количеством минут с 1 января 1601 г., выраженным в универсальном времени (UTC), и является значением типа **LONG**. 
  
Ниже приводится ряд часто используемых относительных значений времени:
  
- `ULONG ulrtmMax = 1525252319L`
    
- `ULONG ulrtmMin = 0L`
    
Используйте предшествующие максимальные и минимальные значения относительного времени для проверки допустимых значений относительного времени.
  
Поскольку время записей файлов NTFS в формате [FILETIME](https://msdn.microsoft.com/library/9baf8a0e-59e3-4fbd-9616-2ec9161520d1%28Office.15%29.aspx) было бы удобно использовать следующий пример кода для преобразования относительного времени в FILETIME и из **НЕГО.** 
  
```cpp
static const LONGLONG UnitsPerMinute = 600000000; 
static const LONGLONG UnitsPerHalfMinute = 300000000; 
void RTimeToFileTime(LONG rtime, FILETIME *pft) 
{ 
    Assert(pft != NULL); 
    ULARGE_INTEGER *puli = (ULARGE_INTEGER *)pft; 
    puli->QuadPart = rtime * UnitsPerMinute; 
} 
  
void FileTimeToRTime(FILETIME *pft, LONG* prtime) 
{ 
    Assert(pft != NULL); 
    Assert(prtime != NULL); 
 
    ULARGE_INTEGER uli = *(ULARGE_INTEGER *)pft; 
  
    uli.QuadPart += UnitsPerHalfMinute; 
    uli.QuadPart /= UnitsPerMinute; 
    *prtime = uli.LowPart; 
} 

```
