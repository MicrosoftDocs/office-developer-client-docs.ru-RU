# <a name="about-registering-a-new-domain-for-automatic-configuration"></a>Сведения о регистрации нового домена для автоматической настройки

Outlook предоставляет способ указать новый домен службы сообщений для автоматической настройки и разрешить поставщику службы сообщений настраивать учетную запись.
  
При разработке поставщика службы сообщений можно использовать следующий ключ в реестре Windows, чтобы указать новый домен, который автоматически настраивается соответствующим поставщиком службы сообщений: 
  
`HKLM\Software\Microsoft\Office\Outlook\AutoConfigDomains\<domain name>\`
  
В разделе Key `<domain name>` — домен для автоматической настройки. Это доменное имя поддерживает подстановочный знак \* только в начале. В следующей таблице приведены значения, поддерживаемые этим ключом. 
  
| Значение | Тип | Описание |
|:-----|:-----|:-----|
|Понятное имя  <br/> |REG_SZ  <br/> |Доменное имя, отображаемое пользователю во время автоматической настройки.  <br/> |
|Имя службы  <br/> |REG_SZ  <br/> |Служба сообщений, зарегистрированная в Mapisvc. INF, которая поддерживает этот домен.  <br/> |
|Расположение установки  <br/> |REG_SZ  <br/> |URL-адрес расположения для установки поставщика службы сообщений (если он еще не установлен).  <br/> |
|Минимальная версия  <br/> |REG_DWORD  <br/> |Минимальная версия DLL поставщика службы сообщений, которая является обязательной. Это значение является необязательным.  <br/> |
   
Когда Outlook начинает автоматическую настройку учетной записи электронной почты, он проверяет реестр Windows на регистрацию домена, указанного в адресе электронной почты. Если домен уже указан в реестре Windows, Outlook проверяет, зарегистрирована ли служба сообщений в Mapisvc. INF. Outlook не может продолжить работу с автоматической настройкой домена, если она не указана в реестре Windows.
  
Если указанная служба сообщений в настоящее время не зарегистрирована в Mapisvc. INF или поставщик службы сообщений установлена, но версия DLL-файла более ранняя, чем у указанного минимума, Outlook использует указанное понятное имя и предлагает пользователю установить поставщика. Если пользователь принимает данные, Outlook перенаправляет пользователя в указанное место установки, чтобы пользователь мог установить поставщика. Установка поставщика регистрирует службу сообщений в Mapisvc. INF.
  
Если служба сообщений в настоящее время зарегистрирована в Mapisvc. INF, а версия provider. dll является подходящей, Outlook создает службу сообщений с помощью [имсгсервицеадмин:: креатемсгсервице](https://msdn.microsoft.com/library/0135f049-0311-45e5-9685-78597d599a4e%28Office.15%29.aspx), а затем настраивает ее с помощью [Имсгсервицеадмин:: конфигуремсгсервице](https://msdn.microsoft.com/library/a08f5905-2585-49ca-abb7-a77f2736f604%28Office.15%29.aspx). В автоматической конфигурации Outlook используются следующие три свойства, позволяющие поставщику настроить учетную запись: [PidTagAutoConfigurationUserName](https://msdn.microsoft.com/library/05dfa0e2-4ab1-4f57-9009-6a815aca87bd%28Office.15%29.aspx), [PidTagAutoConfigurationUserEmail](https://msdn.microsoft.com/library/845140c8-5454-4b47-acec-ab5aff00b768%28Office.15%29.aspx)и [PidTagAutoConfigurationUserPassword](https://msdn.microsoft.com/library/d33e7c45-55d8-4dc1-ade9-605542d87e61%28Office.15%29.aspx).
