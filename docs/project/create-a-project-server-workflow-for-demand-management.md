# <a name="create-a-project-server-workflow-for-demand-management"></a>Создание рабочего процесса Project Server для управления запросами

В этой статье описывается создание простого рабочего процесса с помощью SharePoint Designer 2013. Вы можете экспортировать рабочий процесс в Visio 2013 для визуализации и редактирования или использовать Visio 2013 для разработки рабочего процесса Project Server 2013 и импорта дизайна в SharePoint Designer 2013 для публикации в Project Web App. Дополнительные сведения о платформе рабочих процессов SharePoint и создании рабочих процессов с помощью Visio 2013 и SharePoint Designer 2013 см. в статьях о рабочих процессах [в SharePoint 2013](https://msdn.microsoft.com/library/jj163986%28office.15%29.aspx) в документации для разработчиков SharePoint 2013. 
  
Сведения о подготовке Project Server к работе см. в подпроекте [" Start: Set up and configure SharePoint 2013 Workflow Manager".](https://msdn.microsoft.com/library/jj163276%28office.15%29.aspx)

<a name="pj15_CreateWorkflowSPD_General"> </a>

## <a name="creating-a-general-workflow"></a>Создание общего рабочего процесса

Чтобы создать рабочий процесс Project Server 2013 с помощью SharePoint Designer 2013, с помощью следующих действий. Рабочий процесс разрабатывается для управления запросами проектных инициатив.
  
Дополнительные сведения см. в разделе ["Создание ветвяющихся рабочих процессов".](#pj15_CreateWorkflowSPD_Detailed) 
  
### <a name="to-create-a-project-server-workflow-general-procedure"></a>Создание рабочего процесса Project Server (общая процедура)

1. Определите требования, а затем разработайте рабочий процесс. Организуйте его этапы и стадии, а также определите настраиваемые поля, которые будет использовать этот рабочий процесс.
    
2. В Project Web App создайте сущности, необходимые для рабочего процесса:
    
    1. Просмотрите существующие этапы рабочего процесса; создайте этапы по мере необходимости.
        
    2. Создайте корпоративные настраиваемые поля, которые будет использовать рабочий процесс. Чтобы настраиваемое поле было доступно в стадии рабочего процесса, оно должно управляться рабочим процессом.
        
    3. Отредактируйте или создайте страницы сведений о проекте (PDP), которые будут использоваться стадиями рабочего процесса для сбора сведений для проекта. В данном примере стадии используют страницы сведений о проекте по умолчанию, которые редактируются, чтобы включить в них новое настраиваемое поле.
        
    4. Создайте необходимые стадии рабочего процесса, а затем сопоставьте каждую стадию рабочего процесса с правильным этапом.
    
3. В SharePoint Designer 2013 с помощью декларативных заявлений в текстовом **конструкторе:**
    
    > [!NOTE]
    > Вы также можете переключиться на **визуальный** конструктор в SharePoint Designer 2013 или импортировать существующий рабочий процесс из Visio 2013. Чтобы воспользоваться **текстовым конструктором**, выполните следующие действия. 
    > 
    > 1. Откройте сайт Project Web App, а затем создайте рабочий процесс сайта, использующий рабочий процесс **SharePoint 2013 —** платформа рабочих процессов Project Server. 
    > 2. Добавьте стадии, которые будет использовать рабочий процесс.
    > 3. Вставьте шаги, условия, действия и циклы рабочего процесса, которые требуются в каждой стадии.
    > 4. Проверьте наличие каких-либо ошибок рабочего процесса и исправьте все найденные ошибки.
    > 5. (Необязательно) Переключить представление в **визуальный конструктор** или экспортировать рабочий процесс в файл Visio 2013. Можно изменить представление Visio и сохранить изменения текущего рабочего процесса. Вы можете изменить файл Visio и импортировать его в SharePoint Designer 2013, чтобы создать другие рабочие процессы.
    > 6. Опубликуйте рабочий процесс. После публикации рабочий процесс отображает в списке рабочий процесс для Project Web App сайта.
    
4. В Project Web App используйте рабочий процесс для управления запросами предложений по проектам:
    
    1. Создайте шаблон корпоративного проекта (EPT), использующий рабочий процесс.
        
    2. На странице центра проектов создайте проект, который использует шаблон EPT для рабочего процесса, а затем пройдите по стадиям рабочего процесса.
        
    3. Тщательно протестируйте рабочий процесс.
        
    4. Разверните рабочий процесс на рабочем сервере.

## <a name="creating-a-branching-workflow"></a>Создание ветвящегося рабочего процесса

Прежде чем использовать SharePoint Designer 2013 для создания рабочего процесса Project Server, необходимо настроить службу Workflow Manager Client 1.0 для использования действий рабочих процессов Project Server 2013. Сведения о настройке workflow Manager Client 1.0 см. в статьях ["Workflows in SharePoint 2013"](https://msdn.microsoft.com/library/jj163986%28office.15%29.aspx) документации для разработчиков SharePoint Server 2013. 
  
В следующей подробной процедуре описаны те же действия, что и в разделе "Создание [общего рабочего процесса".](#pj15_CreateWorkflowSPD_General) 
  
### <a name="to-create-a-project-server-branching-workflow-detailed-procedure"></a>Создание ветвящегося рабочего процесса Project Server (подробная процедура)

#### <a name="1-plan-and-design-the-workflow"></a>1. Планирование и проектирование рабочего процесса.

Рабочий процесс Project Server может интегрироваться с несколькими стадиями и этапами в процессе управления запросами. Поскольку рабочие процессы могут быть сложными, необходимо понимать бизнес-требования и тщательно планировать рабочий процесс. В качестве простого примера сконструируйте рабочий процесс, который использует оценочную стоимость проектной инициативы, чтобы определить, принимается ли эта инициатива. Если оценочная стоимость больше 25 000 долларов США, инициатива отклоняется; в противном случае инициатива принимается, и проект создается.
    
Так как вы можете использовать Visio 2013 и SharePoint Designer 2013 для проектирования и создания рабочих процессов для Project Server 2013, вы можете проще экспериментировать с рабочими процессами, чем это возможно в Project Server 2010. Пример проекта рабочего процесса в этой статье такой [](https://msdn.microsoft.com/library/a02cafdc-d881-4271-b446-d8b2cd456a52%28Office.15%29.aspx) же, как в статье "Создание ветвяющихся рабочих процессов" в SDK Project 2010. Вы можете разработать и создать тестовый рабочий процесс на удаленном компьютере с помощью тестового экземпляра Project Web App не нужно создавать рабочие процессы непосредственно на компьютере Project Server 2013. 
    
#### <a name="2-create-the-entities-that-your-workflow-requires"></a>2. Создайте сущности, необходимые для рабочего процесса.

В Project Web App просмотрите доступные этапы и стадии рабочего процесса, а также доступные корпоративные настраиваемые поля. При необходимости создайте сущности, которые требуются вашему рабочему процессу, как показано в следующих действиях.
    
1. **Этапы рабочего процесса** По умолчанию установка Project Web App включает этапы создания, выбора, планирования, управления и завершения. В примере с ветвящимся рабочим процессом другие этапы создавать не требуется. 
        
2. **Корпоративные настраиваемые поля** Ветвяющийся рабочий процесс требует настраиваемого поля затрат проекта, которое управляется процессом. Значение управляемого рабочим процессом настраиваемого поля устанавливается на странице сведений о проекте, которая используется рабочим процессом. Например, выберите  значок "Параметры" в верхней правой части страницы Project Web App страницы, выберите параметры **PWA,** а затем выберите корпоративные настраиваемые поля и таблицы **подбора.**
        
   Создайте настраиваемое поле с именем Стоимость инициативы для сущности **Проект** и выберите тип **Cost** (Стоимость). В качестве описания введите Оценочная стоимость проектной инициативы. В разделе **Поведение** выберите **Поведение управляется рабочим процессом**.
        
3. **Страницы подробных проектов** Изменение или создание PDP, которые будут использовать стадии рабочего процесса. Например, выполните следующие действия. 
        
    1. На странице параметров сервера выберите **Страницы сведений о проекте**, а затем выберите PDP **ProjectInformation**. 
            
    2. На вкладке **СТРАНИЦА** ленты в группе **Правка** выберите команду **Изменить страницу**.
            
    3. Choose the down arrow at the top-right of the **Basic Info** web part, and then choose Edit **web part**. Или на вкладке **"ВЕБ-ЧАСТЬ"**  ленты в  группе "Свойства" выберите "Свойства веб-части", чтобы отдемонстрировать веб-часть редактора. 
            
    4. В разделе **Отображаемые поля проекта** веб-части "Редактор" (см. рисунок 1) выберите **Изменить**.
            
    5. Добавьте **настраиваемые поля "Стоимость** предложения", переместийте его над полем "Владелец" в списке "Выбранные поля проекта" и выберите "ОК" **(см.** рис. 1).  
      
    6. В веб-части "Редактор" нажмите кнопку **ОК**, а затем на вкладке **СТРАНИЦА** ленты в группе **Правка** нажмите **Stop Editing** (Завершить редактирование). На рисунке 2 показано настраиваемое поле **Proposal Cost**, добавленное на страницу сведений о проекте информации по проекту.

#### <a name="3-construct-the-workflow-in-the-text-based-designer"></a>3. Создание рабочего процесса в конструкторе Text-Based конструктора.

В SharePoint Designer 2013 с помощью декларативных Text-Based Designer. Вы можете начать вводить текст в оранжевую строку вставки, чтобы получить контекстно-чувствительные для логики  и действий автоза завершение рабочего процесса, или вставить логику и действия с помощью элементов управления в группе "Вставка" на вкладке **WORKFLOW** ленты. 
    
1. В представлении Backstage SharePoint Designer 2013 выберите "Открыть **сайт".** Например, откройте  `https://ServerName/pwa` . В области **навигации** выберите **"Рабочий процесс".** Затем на вкладке **"WORKFLOWS"** ленты в группе "Новая" выберите **"Рабочий процесс сайта".**  В этом примере назовем рабочий процесс ветвяющийся рабочий процесс. Убедитесь, что в выпадаемом списке "Тип  платформы" выбран рабочий процесс **SharePoint 2013 Project Server** (см. рис. 5). 
    
    **Рис. 5. Создание рабочего процесса сайта Project Server**

    ![Создание рабочего процесса сайта Project Server]Создание рабочего процесса сайта Project(media/pj15_CreateWorkflowSPD_CreateSiteWorkflow.gif "Server")
  
2. Перейдите на вкладку **Ветвящийся рабочий процесс**. Затем на вкладке **РАБОЧИЙ ПРОЦЕСС** ленты в группе **Manage** (Управление) в раскрывающемся списке **Представления** выберите **Текстовый конструктор**. Чтобы отобразить представление с мигающей оранжевой строкой вставки (см. рисунок 6), щелкните мышью где-нибудь внутри представления.
    
    **Рис. 6. Использование представления текстового конструктора для рабочего процесса**

    ![Использование представления Text-Based с](media/pj15_CreateWorkflowSPD_TextBasedDesigner.gif "помощью Text-Based конструктора")
  
3. В представлении **Текстовый конструктор** добавьте стадии, используемые рабочим процессом. На вкладке **РАБОЧИЙ ПРОЦЕСС** ленты в группе **Вставка** в раскрывающемся списке **Стадия** в разделе **Create** (Создание) выберите **Начальные сведения об инициативе**.
    
    Аналогично, поместите оранжевую строку вставки под полем **Stage: Initial Proposal Details** (Стадия: начальные сведения об инициативе) и добавьте другие стадии, используемые рабочим процессом: **Сведения о проекте**, **Автоматическое отклонение** и **Выполнение** (см. рисунок 7). 
    
    **Рис. 7. Добавление стадии в рабочий процесс в SharePoint Designer**

    ![Добавление стадии в рабочий процесс]в SPD Добавление стадии в рабочий(media/pj15_CreateWorkflowSPD_AddStageInSPD.gif "процесс в SPD")
  
4. В каждой стадии добавьте шаги и логику рабочего процесса. 
    
    1. В стадии **Начальные сведения об инициативе** поместите оранжевую строку вставки вверху области стадии. В группе **Вставка** нажмите **Действие**, прокрутите вниз до раздела **Действия Project Web App** и выберите **Дождаться события проекта**. Нажмите **это событие проекта**, а затем выберите в раскрывающемся списке пункт **Событие: при отправке проекта**. 
    
    2. В разделе **Переход к стадии** для стадии **Начальные сведения об инициативе** вставьте **Если любое значение равно указанному значению**. Можно начать вводить эту инструкцию или воспользоваться элементом управления **Условие** в группе **Вставка** на ленте. 
    
    3. Выберите первый элемент управления **value**, а затем выберите **fx**, чтобы отобразить диалоговое окно **Определение подстановки в рабочем процессе** (см. рисунок 8). В раскрывающемся списке **Источник данных** выберите **Данные проекта**. В раскрывающемся списке **Поле из источника** выберите **Стоимость инициативы**.
    
       **Рис. 8. Определение значения подстановки в рабочем процессе**

       ![Определение значения подбора]в рабочий процесс Определение значения(media/pj15_CreateWorkflowSPD_DefineWorkflowLookup.gif "подбора в рабочий процесс")
  
    4. Выполните это заявление, чтобы  `If` он отображал следующее: **если project Data:Proposal Cost больше 25000**
    
       > [!NOTE]
       > Вместо этого можно было бы создать переменную рабочего процесса, установить для этой переменной значение настраиваемого поля, а затем сравнивать переменную со значением. Например, в раскрывающемся списке **Локальные переменные** на ленте создайте переменную с именем **TotalCost** (без пробелов) и типом **Number**. В диалоговом окне **Определение подстановки в рабочем процессе** выберите **Переменные и параметры рабочего процесса** для источника данных, а затем выберите **Переменная: TotalCost** в качестве поля. Инструкция **Если** может быть такой: **Если Переменная: TotalCost больше 25000**

## <a name="importing-a-workflow-from-visio"></a>Импорт рабочего процесса из Visio

Чтобы изменить рабочий процесс, можно создать или изменить управляемые рабочим процессом настраиваемые поля и создать или изменить этапы и стадии рабочего процесса. Вы можете использовать SharePoint Designer 2013 для добавления условий, действий, циклов и стадий, а затем сохранить и повторно опубликовать рабочий процесс. Чтобы повторно использовать или сохранить резервную копию рабочего процесса, вы можете экспортировать его в файл Visio 2013. 
  
Вы также можете создать или изменить рабочий процесс в Visio 2013 и импортировать файл в SharePoint Designer 2013 для использования Project Web App. Чтобы использовать неизмененнный рабочий процесс, экземпляр Project Web App должен включать свойства стадии рабочего процесса, которые такие же, как и в исходном Project Web App экземпляре. Дополнительные сведения об использовании Visio для создания рабочих процессов см. в разработке рабочих процессов в [SharePoint Designer 2013 и Visio 2013.](https://msdn.microsoft.com/library/jj163272%28office.15%29.aspx)
  
> [!NOTE]
> При импорте файла Visio 2013 в другой экземпляр Project Web App, у стадий имеются разные GUID стадии, даже если имена стадий одинаковы. После импорта рабочего процесса необходимо настроить свойства стадии и действия для использования значений, характерных для Project Web App экземпляра. 
> 
> При создании рабочего процесса в Visio 2013 у стадий и действий нет свойств, характерных для экземпляра Project Web App, так как Visio не подключается к Project Web App. При подключении SharePoint Designer 2013 с Project Web App, создании рабочего процесса и импорте VSDX-файла необходимо переписать активный рабочий процесс. Затем необходимо настроить свойства стадии и действия в соответствие со значениями, которые SharePoint Designer 2013 получает из Project Web App. 
  
### <a name="to-import-a-workflow-from-visio-to-sharepoint-designer"></a>Импорт рабочего процесса из Visio в SharePoint Designer

1. В Visio 2013 создайте простой рабочий процесс. Например, выполните следующие действия.
    
   1. Откройте Visio и создайте рабочий процесс. Выберите область **КАТЕГОРИИ** для нового рабочего процесса, нажмите **Блок-схема**, выберите в области **Создать** шаблон **Рабочий процесс Microsoft SharePoint 2013** и нажмите **Создать**. Рабочий процесс откроется с фигурой Stage с именем **Stage 1**. Рабочий процесс содержит компонент Start и фигуры Enter и Exit как часть фигуры Stage.

      > [!NOTE]
      >  Вы должны использовать только наборы элементов фигуры, которые имеются в списке фигур блок-схемы: 
      > - **Действия — рабочий процесс SharePoint 2013**
      > - **Компоненты — рабочий процесс SharePoint 2013**
      > - **Условия — рабочий процесс SharePoint 2013**
  
   2. В области **фигур** нажмите **Экспресс-фигуры**, а затем перетащите фигуру Condition с именем **Если любое значение равно указанному значению** справа от фигуры Stage. 
    
   3. На вкладке **ГЛАВНАЯ** ленты выберите инструмент **Соединитель**, а затем соедините фигуру Exit в стадии с фигурой Condition (см. рисунок 19).

      ![Создание схемы рабочего процесса в Visio]Создание схемы рабочего процесса(media/pj15_CreateWorkflowSPD_NewVisioWorkflow.gif "в Visio")

   7. В области **"Фигуры"** выберите "Действия — рабочий процесс **SharePoint 2013"** и перетащите действие "Дождаться события проекта" в середину фигуры для стадии **1** (см. рис. 20).  
    
      **Рис. 20. Оформление рабочего процесса в Visio**

      ![Завершение рабочего процесса в Visio]Завершение рабочего процесса(media/pj15_CreateWorkflowSPD_CompletedVisioWorkflow.gif "в Visio")

   В **диалоговом окне** "Создание рабочего процесса сайта" убедитесь, что типом платформы является рабочий процесс **SharePoint 2013 Project Server.** Choose **Create**, and SharePoint Designer opens the **Text-Based Designer pane** for the new workflow.

Выберите **стадию "Начальные сведения о предложении"** в выпадаемом списке **стадии** проекта и выберите "ОК".  SharePoint Designer переименовает стадию.

   ![Установка свойств в импортируемых свойствах параметра]рабочего процесса в(media/pj15_CreateWorkflowSPD_ImportFromVisio1.gif "импортируемом рабочий процесс")
  
   Для второй стадии установите свойство **Стадия проекта** в значение **Автоматическое отклонение**. Для третьей стадии установите свойство **Стадия проекта** в значение **Выполнение**.
    
7. Аналогично, для действия **Дождаться события проекта** установите свойство **Имя события** в значение **Событие: при отправке проекта**.
    
8. Аналогичным образом установите свойства условия **"Если любое значение равно условию** значения". Например, установите для первого свойства **Value** значение **Project Data:Proposal Cost.** Установите для **свойства Operator** **меньше.** Установите для второго **свойства Value** значение 5000.
    
9. Проверьте рабочий процесс на наличие ошибок, а затем сохраните его. Если ошибки отсутствуют, можно изменить представление на **Текстовый конструктор** (см. рисунок 22). 
    
   **Рис. 22. Просмотр импортированного рабочего процесса в текстовом конструкторе**

   ![Просмотр импортируемого рабочего процесса](media/pj15_CreateWorkflowSPD_WorkflowFromVisio.gif "Просмотр импортируемого рабочего процесса")
  
10. Опубликуйте рабочий процесс. Если сохранить рабочий процесс, но не опубликовать его, то он не будет доступен при создании типа корпоративного проекта.
    
11. Чтобы протестировать импортируемый простой рабочий процесс из **Visio** в Project Web App, создайте EPT, использующий рабочий процесс, а  затем создайте проекты, использующие новый EPT, как это было в примере ветвяющихся рабочих процессов. Однако в данном случае отклоняются проекты со стоимостью меньше $5000.

Вы также создали схему для еще более простого ветвяшегося рабочего процесса с помощью Visio 2013. Вы импортировали схему рабочего процесса Visio в SharePoint Designer 2013, где вы установили свойства каждого этапа, условия и действия из подключения к Project Web App.
  
Visio 2013 и SharePoint Designer предоставляют разработчикам, руководителям проектов, разработчикам рабочих процессов и тестировщикам удобные способы создания, предоставления и настройки макетов рабочих процессов для различных установок Project Server 2013 и Project Online. Для бизнес-процессов, которые требуют программного доступа к Project Server, который SharePoint Designer не предоставляет, можно использовать Visual Studio 2012 с клиентской объектной моделью (CSOM).
