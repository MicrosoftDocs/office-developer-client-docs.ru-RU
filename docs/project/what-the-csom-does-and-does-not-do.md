# <a name="what-the-csom-does-and-does-not-do"></a>Какие задачи CSOM выполняет, а какие — нет

Клиентская объектная модель (CSOM) — это набор API-интерфейсы для Project Server 2013, предназначенных как для Интернет-версия и локальной использовать в приложениях, которые могут быть разработаны для ПК, мобильных устройств и планшетных компьютеров. В этой статье включает в себя некоторые типичные сценарии использования CSOM, а также приведены ограничения CSOM.
  
|||
|:-----|:-----|
|||
   
CSOM позволяет разработка приложений для Project Server 2013 и интеграции Project Server с другими приложениями. Приложения может быть разработан для запуска на ПК, мобильных устройств, такие как Windows Phone 7.5, планшетные ПК, такие как устройства Windows 8 и операций ввода-вывода и устройствами Android. CSOM предоставляет интерфейсы API, которые охватывают функциональных возможностей двенадцать наиболее часто используемых служб PSI в Project Server. API-интерфейсы CSOM упорядочены по-разному и существенно упрощает использование чем службы PSI на основе ASMX и на основе WCF. CSOM не использует наборов данных ADO.NET и доступен через протокол OData. Вы можете создавать с помощью CSOM при помощи библиотек платформы .NET Framework 4, JavaScript, или запросы представлений состояния (REST).
  
Обзор CSOM и статьи, в которых показано, как использовать JavaScript и .NET Framework 4 с помощью CSOM в разделе [со стороны клиента объектной модели (CSOM) для Project Server](client-side-object-model-csom-for-project-2013.md). Дополнительные сведения о сборках, классы и члены CSOM видеть ссылку на пространство имен [Microsoft.ProjectServer.Client](https://msdn.microsoft.com/library/Microsoft.ProjectServer.Client.aspx) . 
  
## <a name="usage-scenarios-for-the-csom"></a>Сценарии использования для CSOM
<a name="pj15_WhatTheCSOM_UsageScenarios"> </a>

Ниже приведены примеры некоторые виды приложений, которые поддерживает CSOM. CSOM можно использовать вместо PSI для многих сценариев:
  
- **Разработка приложений, которые расширение Project Server** Основным назначением CSOM — разработки приложений для Project Server 2013, где приложения могут быть созданы для самых разнообразных устройств, включая компьютеры, мобильные устройства и планшетные ПК. Приложения могут быть территориально в в частном каталоге приложений или в общедоступном магазине Office. 
    
- **Автоматизация создания или управления сущностей в Project Server** Операции CRUD для сущностей, таких как проекты, задач, назначений, корпоративные ресурсы, настраиваемые поля, таблицы подстановки, расписания, обработчиков событий и этапы рабочего процесса и этапов можно выполнить CSOM. Часто имеется случаев, где пользовательское приложение экономии времени с неполным или повторяющихся заданий. 
    
- **Получение данных в опубликованных таблицах базы данных Project** Так как прямой базы данных, доступ к черновиков, опубликованных проектов и архивировать таблицы не поддерживается, CSOM можно использовать для считывания данных, недоступны в отчетности таблицы или представления. Например получение сведений о стадии рабочего процесса, этапов и действий. Для чтения данных в таблицах отчетов, можно использовать запросов OData. 
    
- **Проверять данные на отчеты о состоянии и расписаний** Для проверки состояния или табеля учета рабочего времени данные назначения, введите пользователей, перед сохранением данных в Project Web App используйте CSOM в обработчики событий на локальном или удаленных приемников событий для события перед. 
    
- **Создать финансовые проекты** Создание проектов для захвата времени через табеля учета рабочего времени для интеграции с финансовой системы. Создание иерархии финансовые коды, чтобы отразить структуры декомпозиции стоимость финансовой системы. Финансовые проекты не требуют планирования и состояние обновления. 
    
- **Интеграция с системами бухгалтерского учета.** Запись ресурса затраты и расходы, связанные с проектами для веб-канала системы финансовые и выставления счетов и для сравнения бюджета. Синхронизация задач, ресурсов и назначений между системами. Запись данных табеля учета рабочего времени в одной системе для веб-канала другое (какие расписания используется зависит от потребностей организации или отдельных проектов). 
    
- **Автоматизация обновлений от участников группы** Для проектов, которые не управляются активно автоматически обновите проекты на сервере с помощью хода выполнения и другие изменения членов группы проекта. Проекты можно обновить и повторной публикации без руководителя проектов с помощью изучения результатов и выполнение корректировок в план. 
    
    > [!NOTE]
    > CSOM поддерживает отправка обновлений состояния, но в настоящее время не поддерживает утверждения состояния. 
  
- **Оценка Project Server данных в удаленные приемники событий** Удаленного приемника событий для события до операции **ProjectCreating** можно использовать данные Project Server из CSOM можно определить, следует ли отменить событие. Например перед созданием проекта, сравните предложения проекта с существующими проектами. 
    
- **Поддержка декларативных рабочих процессов Project Server** CSOM позволяет рабочих процессов Project Server, созданных в SharePoint Designer 2013. CSOM поддерживает определений рабочих процессов, использующих Windows Workflow Foundation версии 4 (WF4). (PSI не поддерживает рабочие процессы WF4.) 
    
- **Создание сложных рабочих процессов Project Server** При разработке рабочих процессов с помощью Visual Studio 2012, можно использовать CSOM для сложных действий в стадии рабочего процесса или создать настраиваемые действия рабочих процессов. 
    
## <a name="what-the-csom-does-not-do"></a>Что делать CSOM
<a name="pj15_WhatTheCSOM_DoesNotDo"> </a>

CSOM не завершена заменой PSI. Так как CSOM во внутренней сети использует службы PSI, CSOM имеет многие из те же функциональные ограничения, для которых есть PSI. В дополнение к ограничения PSI, например, не имевшими доступа к данным в локальных проектов (MPP-файлы) CSOM не включает возможности администрирования, который обычно обрабатывает Project Web App. Например создание группы безопасности могут быть обработаны в настройках веб - странице разрешения для Project Web App. 
  
Список действий, которые обрабатывают PSI, ни CSOM обратитесь к разделу *PSI что действия не выполняются* в [PSI что делает и действия не выполняются](what-the-psi-does-and-does-not-do.md).
  
### <a name="psi-services-that-the-csom-does-not-cover"></a>Службы PSI, которые не охватывает CSOM
<a name="pj15_WhatTheCSOM_PSIServices"> </a>

CSOM не включает функциональные возможности ниже службы PSI:
  
- **Службы администрирования** Для управления операций в Project Server и связанных проектов сайтов, таких как создание финансовых периодов и отображение параметров расписания и административных параметров используйте методы PSI в классе [WebSvcAdmin.Admin](https://msdn.microsoft.com/library/WebSvcAdmin.Admin.aspx) . Project Web App самого использует методы **администрирования** во многих страниц, привязанные к странице "Параметры сервера" (https:// *имя_сервера*  /  *ProjectServerName* /_layouts/15/pwa/Admin/Admin.aspx). 
    
- **Архив службы** Чтобы сохранить и управление сущностей, таких как проекты, ресурсы и настраиваемые поля в таблицах архива, используйте методы PSI в классе [архива](https://msdn.microsoft.com/library/WebSvcArchive.Archive.aspx) . 
    
- **Cubeadmin задание таймера службы** Создание и управление ими кубов OLAP для локальных установок, используйте методы PSI в классе [WebSvcCubeAdmin.CubeAdmin](https://msdn.microsoft.com/library/WebSvcCubeAdmin.CubeAdmin.aspx) или страница управления базой данных OLAP (https:// *имя_сервера*  /  *ProjectServerName* /_layouts/15/pwa / CubeAdmin/CubeAnalysisAdmin.aspx) в Project Web App. 
    
    > [!NOTE]
    > Project Online не поддерживает кубов OLAP. 
  
- **Драйвер службы** Для создания и управления бизнес-факторами для анализа портфеля проектов, используйте методы PSI в классе [WebSvcDriver.Driver](https://msdn.microsoft.com/library/WebSvcDriver.Driver.aspx) . 
    
- **Службы LoginForms и LoginWindows** Проверка подлинности в CSOM выполняется во время инициализации объекта **ProjectContext** с OAuth или проверки подлинности Windows. Для создания приложений для многоуровневой проверки подлинности, где проверка подлинности через формы и проверка подлинности Windows, можно использовать локальное приложение с полным доверием, используйте методы PSI в класс [WebSvcLoginForms.LoginForms](https://msdn.microsoft.com/library/WebSvcLoginForms.LoginForms.aspx) и [ WebSvcLoginWindows.LoginWindows](https://msdn.microsoft.com/library/WebSvcLoginWindows.LoginWindows.aspx) класса. 
    
- **Служба уведомлений** Создание и Управление оповещениями и напоминаниями, используйте методы PSI в классе [WebSvcNotifications.Notifications](https://msdn.microsoft.com/library/WebSvcNotifications.Notifications.aspx) . 
    
- **ObjectLinkProvider службы** Создание и управление веб-объекты и ссылки на документы и элементы списков SharePoint, используйте методы PSI в классе [WebSvcObjectLinkProvider.ObjectLinkProvider](https://msdn.microsoft.com/library/WebSvcObjectLinkProvider.ObjectLinkProvider.aspx) . 
    
- **Служба PortfolioAnalyses** Создание и Управление анализом портфелей проектов, включая планировщик работы решений и решений оптимизатора используйте методы PSI в классе [WebSvcPortfolioAnalyses.PortfolioAnalyses](https://msdn.microsoft.com/library/WebSvcPortfolioAnalyses.PortfolioAnalyses.aspx) . 
    
- **Служба QueueSystem** CSOM можно получить основные сведения о заданиями в очереди Project Server и включает в себя метод [ProjectContext.WaitForQueue](https://msdn.microsoft.com/library/Microsoft.ProjectServer.Client.ProjectContext.WaitForQueue.aspx) . Для более тщательного управления Система очередей Project Server используйте методы PSI в классе [WebSvcQueueSystem.QueueSystem](https://msdn.microsoft.com/library/WebSvcQueueSystem.QueueSystem.aspx) . 
    
- **Службы безопасности** Создание и управление группами безопасности Project Server, шаблоны и категорий и проверить разрешения для текущего пользователя, используйте методы PSI в классе [WebSvcSecurity.Security](https://msdn.microsoft.com/library/WebSvcSecurity.Security.aspx) . 
    
- **Служба WssInterop** Для получения сведений о и управления сайтами проектов, используя методы PSI в классе [WebSvcWssInterop.WssInterop](https://msdn.microsoft.com/library/WebSvcWssInterop.WssInterop.aspx) . 
    
    > [!NOTE]
    > CSOM можно использовать в SharePoint Server 2013. Сайты проектов, сайты SharePoint. 
  
CSOM не включает расширения, такие как PSI может иметь. Например при создании расширения PSI для локального использования, CSOM нельзя изменить для использования расширения PSI. Расширение сценариев можно реализовать другими способами:
  
- Агрегированные CSOM вызывает в локальном компонент или компонент, который работает в Microsoft Azure.
    
- Использование запросов OData данных отчетов вместо прямой доступ к отчетности таблиц в базе данных Project Server.
    
- Интеграция вызовы CSOM с приложений сторонних производителей с помощью проверки подлинности OAuth с Project Online или серверные компоненты для локального использования.
    
- Приложения, использующие CSOM можно также использовать настраиваемые баз данных в локальной или с SQL Azure.
    
### <a name="request-limits-of-the-csom"></a>Ограничения запроса CSOM
<a name="pj15_WhatTheCSOM_RequestLimits"> </a>

CSOM в Project Server 2013 создан на основе CSOM реализации в SharePoint Server 2013 и наследует ограничения для максимального размера запроса. Ограничение 2 МБ для запроса операций и 50 МБ предельный размер отправленных двоичного объекта из SharePoint. Размер запроса ограничено для защиты сервера из очередей слишком много операций и от обработки задержки для больших двоичных объектов.
  
Например, при использовании CSOM для создания проекта и затем изменить проект, чтобы добавить 252 задач с минимальный объем сведения, такие как краткое имя, GUID задачи и длительность 1 d общий объем данных в запросе на **DraftProject.Update** будет меньше, чем 2 МБ. Но при попытке добавить 253 такие задачи, пустой проект превысила ограничение 2 МБ, и вы получаете следующее исключение: **Microsoft.SharePoint.Client.ServerException: запрос использует слишком много ресурсов**
  
Для записи данных в запросе CSOM по протоколам HTTP и HTTPS, можно использовать веб-узел отладки средства, такие как [Fiddler](https://www.fiddler2.com) (https://www.fiddler2.com). Пример кода, который реализует тестового размер запроса, а также решения, разрывов больших запроса в небольших групп в разделе [DraftProject.Update](https://msdn.microsoft.com/library/Microsoft.ProjectServer.Client.DraftProject.Update.aspx) .
