# <a name="from-0-to-60-with-project-online"></a>Быстрое начало разработки для Project Online

Разработчик приложений может настраивать сайт Project Online (с хостингом SharePoint) с помощью автономных приложений и/или надстройки Project. Существует множество приложений, которые могут быть в диапазоне от решения потребностей тех, кто участвует в проекте, до функций поддержки PMO, таких как любое из следующих:
  
- Упрощенная запись данных системы времени для сотрудников
- Эффективное утверждение систем времени для руководителей
- Надзор за разрешениями (закупками и состоянием), необходимыми для проекта
- Проверка состояния и состояния активных проектов
- Отчет о проблемах
- Отчет о состоянии управления изменениями
    
Project Online включает поддержку API для реализации следующих сценариев:
  
- Для надстройки, которая находится в Project (SharePoint):
    
  - Код (JavaScript, HTML, CSS), который находится в SharePoint Online
  - Ресурсы, загруженные в браузер и выполненные в SharePoint Online.  
  - Бизнес-логика в JavaScript   
  - Доступ к данным, хранимым в Project Online или SharePoint, например (но не ограничивается):  
  - настраиваемые поля;  
  - Списки
    
- Для надстройки Project (SharePoint), которая находится у поставщика:
    
  - Код, который существует на сайте, внешнем от сайта Project Online 
  - Внешний сайт, который может быть (но не ограничен):  
  - Другой сайт SharePoint  
  - Веб-приложение или служба, построенные на любой платформе  
  - Внешний сайт содержит бизнес-логику  
  - Браузер перенаправляется из Project Online на внешний сайт с маркерами доступа к Project Online  
  - Внешний сайт может звонить в SharePoint и Project Online
    
- Для внешней или независимой надстройки:
    
  - Пользователь выполняет приложение на своем устройстве
  - Приложение непосредственно аутентификация и вызов API Project Online
    

|Тип приложения|Реализация API|Целевая среда|Примеры приложений|
|:-----|:-----|:-----|:-----|
|Project hosted  <br/> |JSOM (объектная модель Java Script)  <br/> REST  <br/> |Браузер  <br/> |Запись с картой времени  <br/> Утверждение с картой времени  <br/> Состояние проекта  <br/> Отчет по вопросам  <br/> |
|Поставщик project Hosted  <br/> |Клиентская библиотека CSOM  <br/> |Веб-сайт или приложение Azure  <br/> Среда, не относягая к Windows (LAMP и т. д.)  <br/> |Внешнее проверка времени  <br/> Импортер проектов  <br/> |
|Внешняя/standalone  <br/> |REST  <br/> CSOM  <br/> |REST — любая платформа  <br/> CSOM — любая поддерживаемая платформа .NET  <br/> |Запись с картой времени  <br/> Перенос проектов на новый сайт  <br/> Изменение состояния управления.  <br/> |
   
## <a name="what-does-it-take-to-start-developing-applications-for-project-online"></a>Что необходимо для начала разработки приложений для Project Online?

Общие элементы, необходимые для разработки приложений Project Online учетной записи Project Online и тестовые данные — проекты и сведения, связанные с проектами, которые включают назначения, задачи, ресурсы и настраиваемые поля. Кроме того, необходима среда разработки, но ее особенности зависят от типа приложения и интерфейса API, необходимого для приложения. В следующих разделах описываются потребности в разработке для трех интерфейсов API.
  
В справочной документации описывается объектная модель, которая является общей для всех трех интерфейсов, а также карта объектов, которая показывает отношения между компонентами объектной модели.
  
## <a name="project-hosted-add-in-development-environment"></a>Среда разработки надстройки, в которой находится project

Надстройка, которая находится на сервере и загружается в браузер для выполнения во время выполнения, — это надстройка, которая находится на сервере. Надстройки с hosted могут использовать интерфейсы JSOM или REST и написаны на JavaScript. Project Online предоставляет ссылки на библиотеку JSOM для выполнения во время выполнения. Предположим, что разработка находится на платформе Windows, необходимые ресурсы следуют:
  
- Visual Studio 2015 (предпочтительный) или Visual Studio 2013
    
- Средства разработки Office для Visual Studio
    
- Язык JavaScript
    
Посетите https://github.com/OfficeDev/Project-JSOM-Copy-Work-Packages пример приложения. 
  
Вы можете скачать и запустить пример в несколько простых действий:
  
1. Загрузка и открытие примера приложения
    
2. Обновление SiteURL в окне "Свойства"
    
   Project Online проверяет область приложения надстройки и разрешения пользователя на управление доступом к информации на хост-сайте Project Online. Если доступ явно отказано в любом из параметров или в обоих параметрах, Project Online отказано в доступе к информации. В противном случае доступ предоставляется.
    
3. Включить [загрузку неогрузки](https://docs.microsoft.com/office/dev/add-ins/testing/create-a-network-shared-folder-catalog-for-task-pane-and-content-add-ins) на сайте.  
    
4. Выполните построение проекта.
    
5. Запустите проект.
    
## <a name="project-provider-hosted-add-in-development-environment"></a>Среда разработки надстройки с хостингом у поставщика Project

Надстройки с хостингом поставщика — это приложения, написанные и которые находятся на любой веб-платформе. Они могут подключаться и выполнять операции с данными с помощью API REST (или CSOM для платформ Майкрософт). Для разработки можно использовать любой язык и среду, поддерживают интерфейс REST. 
  
Пример среды разработки Windows для этого типа приложений включает следующие элементы:
  
-  Visual Studio 2015 (предпочтительный) или Visual Studio 2013 
    
- Microsoft Office средств разработки для Visual Studio (поставляются с выпусками Visual Studio 2015 профессиональный и корпоративный)
    
- .NET Framework 4.0 или более новой
    
- [Пакет CSOM SharePointOnline](https://www.nuget.org/packages/Microsoft.SharePointOnline.CSOM) (для вызовов CSOM) 
    
- Язык программирования, например C # 
    
Посетите https://github.com/OfficeDev/Project-Add-in-REST-BasicDataOperations сайт для работы с примерами сценариев. 
  
Пример можно запустить в несколько этапов:
  
1. Загрузка и открытие примера приложения
    
2. Обновление SiteURL в окне "Свойства"
    
   Project Online проверяет область приложения надстройки и разрешения пользователя на управление доступом к информации на хост-сайте Project Online. Если доступ явно отказано в любом из параметров или в обоих параметрах, Project Online отказано в доступе к информации. В противном случае доступ предоставляется.
    
3. Включить [загрузку неогрузки](https://docs.microsoft.com/office/dev/add-ins/testing/create-a-network-shared-folder-catalog-for-task-pane-and-content-add-ins) на сайте. 
    
4. Выполните построение проекта.
    
5. Запустите проект.
    
## <a name="externalstandalone-application-development-environment"></a>Среда разработки внешних и автономных приложений

Автономные приложения могут вызывать Project Online с помощью клиентской объектной модели (CSOM) или REST для связи с Project Online для создания, извлечения, обновления и удаления информации, которая находится на сервере. Это приложение является автономным клиентом, которое зависит от уровня доступа пользователя для запуска. 
  
Пример среды разработки Windows для этого типа приложений включает следующие элементы:
  
- Visual Studio 2015 (предпочтительный) или Visual Studio 2013 
    
- Microsoft Office средств разработки для Visual Studio (поставляются с выпусками Visual Studio 2015 профессиональный и корпоративный)
    
- .NET Framework 4.0 или более новой
    
- [Пакет CSOM SharePointOnline](https://www.nuget.org/packages/Microsoft.SharePointOnline.CSOM) (для вызовов CSOM) 
    
- Язык программирования, например C # 
    
Посетите https://github.com/OfficeDev/Project-CSOM-Read-Enterprise-CustomFields пример приложения. 
  
Пример можно запустить в несколько этапов:
  
1. Скачивание примера приложения
    
2. Внести несколько изменений для доступа к сайту Project Online — имя сайта, учетную запись пользователя и пароль.
    
   Убедитесь, что у пользователя есть доступ ко всем проектам. Project Online использует разрешения пользователей для управления доступом к информации в хранилище данных.
    
3. Добавьте сборку SharePoint в ссылки с помощью консоли Nuget диспетчер пакетов, доступной в меню "Инструменты", введя в консоли Nuget следующую информацию: 
    
   `Install-Package Microsoft.SharePointOnline.CSOM`

4. Выполните построение проекта.
    
5. Запустите проект.
