# <a name="from-0-to-60-with-project-online"></a>Быстрое начало разработки для Project Online

Разработчик приложений может настроить сайт Project Online (SharePoint Hosted), используя автономные приложения и/или надстройки Project. Множество приложений может быть в диапазоне от требований к адресации, задействованных в проекте, для функций поддержки PMO, таких как следующие:
  
- Упрощенная запись данных табеля для работников
- Утверждение эффективного табеля для руководителей
- Обзор разрешений (закупаемой продукции и состояния), необходимых для проекта
- Проверка состояния и работоспособности активных проектов
- Отчет о проблемах
- Изменение отчета о состоянии управления
    
Project Online включает поддержку API для следующих сценариев:
  
- Для размещенной надстройки Project (SharePoint):
    
  - Код (JavaScript, HTML, CSS), размещенный в SharePoint Online
  - Ресурсы, загруженные в браузер и выполняемые в SharePoint Online.  
  - Бизнес-логика в JavaScript   
  - Доступ к данным, хранящимся и хранящимся в Project Online или SharePoint (но не ограничен):  
  - Настраиваемые поля   
  - Списки
    
- Для надстройки, размещаемой у поставщика Project (SharePoint):
    
  - Код, который существует на сайте, который является внешним по отношению к сайту Project Online 
  - Внешний сайт, который может быть (но не ограничен):  
  - Другой сайт SharePoint  
  - Веб-приложение или служба, созданные на любой платформе  
  - Внешний сайт содержит бизнес-логику  
  - Браузер перенаправляется из Project Online на внешний сайт с маркерами доступа в Project Online  
  - Внешний сайт может совершать вызовы в SharePoint и Project Online
    
- Для внешней или автономной надстройки:
    
  - Пользователь выполняет приложение на своем устройстве
  - Приложение выполняет проверку подлинности и вызывает API Project Online напрямую
    

|Тип приложения|Реализация API|Целевая среда|Примеры приложений|
|:-----|:-----|:-----|:-----|
|Размещено в проекте  <br/> |JSOM (объектная модель скрипта Java)  <br/> REST  <br/> |Браузер  <br/> |Запись табеля  <br/> Утверждение табеля  <br/> Состояние проекта  <br/> Отчет о проблемах  <br/> |
|Размещенный у поставщика проект  <br/> |Клиентская библиотека CSOM  <br/> |Веб-сайт или приложение Azure  <br/> Среда, отличная от Windows (лампа и т. д.)  <br/> |Средство проверки внешних расписаний  <br/> Средство импорта проектов  <br/> |
|Внешний или автономный  <br/> |REST  <br/> CSOM  <br/> |REST — любая платформа  <br/> CSOM — любая платформа, поддерживаемая .NET  <br/> |Запись табеля  <br/> Миграция проектов на новый сайт  <br/> Изменение состояния управления.  <br/> |
   
## <a name="what-does-it-take-to-start-developing-applications-for-project-online"></a>Что нужно сделать, чтобы начать разработку приложений для Project Online?

Общие элементы, необходимые для разработки приложений Project Online, — это учетная запись Project Online и тестовые данные проектов и связанные с проектом сведения, включающие назначения, задачи, ресурсы и настраиваемые поля. Среда разработки также необходима, но особенности среды разработки зависят от типа приложения и интерфейса API, необходимого для приложения. В следующих разделах описываются потребности разработки для трех интерфейсов API.
  
В справочной документации описывается объектная модель, которая является общей для всех трех интерфейсов, а также сопоставление объектов, которое показывает отношения между компонентами объектной модели.
  
## <a name="project-hosted-add-in-development-environment"></a>Среда разработки надстройки, размещаемой в проекте

Размещенная надстройка — это надстройка, которая размещается на сервере и загружается в браузер для выполнения среды выполнения. Размещаемые надстройки могут использовать интерфейсы JSOM или REST и написаны на JavaScript. Project Online предоставляет ссылки на библиотеку JSOM для выполнения среды выполнения. Если разработка разрабатывается на платформе Windows, необходимы следующие ресурсы:
  
- Visual Studio 2015 (предпочтительно) или Visual Studio 2013
    
- Средства разработки Office для Visual Studio
    
- Язык JavaScript
    
Посетите https://github.com/OfficeDev/Project-JSOM-Copy-Work-Packages пример приложения. 
  
Загрузить и запустить пример можно несколькими простыми действиями:
  
1. Загрузка и открытие примера приложения
    
2. Обновление SiteURL в окне "Свойства"
    
   Project Online проверяет как область приложения надстройки, так и разрешения пользователя, чтобы управлять доступом к сведениям на узле Project Online. Если доступ явным образом запрещен в обоих или обоих параметрах, Project Online запрещает доступ к данным. В противном случае предоставляется доступ.
    
3. Включите загрузку неопубликованных [приложений](https://docs.microsoft.com/office/dev/add-ins/testing/create-a-network-shared-folder-catalog-for-task-pane-and-content-add-ins) на сайте.  
    
4. Выполните построение проекта.
    
5. Запустите проект.
    
## <a name="project-provider-hosted-add-in-development-environment"></a>Среда разработки надстройки, размещаемой у поставщика проекта

Надстройки, размещаемые у поставщика, — это приложения, написанные и размещенные на любой веб-платформе. Они могут подключаться и выполнять операции с данными с помощью API REST (или CSOM для платформ Майкрософт). Для разработки можно использовать любой язык и среду, поддерживающие интерфейс REST. 
  
Пример среды разработки Windows для этого типа приложения состоит из следующих элементов:
  
-  Visual Studio 2015 (предпочтительно) или Visual Studio 2013 
    
- Средства разработки Microsoft Office для Visual Studio (поставляется с Visual Studio 2015 Professional и Enterprise Editions)
    
- .NET Framework 4,0 или более поздняя версия
    
- [Пакет CSOM SharePointOnline](https://www.nuget.org/packages/Microsoft.SharePointOnline.CSOM) (для вызовов CSOM) 
    
- Язык программирования, например C# 
    
Посетите https://github.com/OfficeDev/Project-Add-in-REST-BasicDataOperations страницу Работа с примерами сценариев. 
  
Пример можно выполнить несколькими шагами:
  
1. Загрузка и открытие примера приложения
    
2. Обновление SiteURL в окне "Свойства"
    
   Project Online проверяет как область приложения надстройки, так и разрешения пользователя, чтобы управлять доступом к сведениям на узле Project Online. Если доступ явным образом запрещен в обоих или обоих параметрах, Project Online запрещает доступ к данным. В противном случае предоставляется доступ.
    
3. Включите загрузку неопубликованных [приложений](https://docs.microsoft.com/office/dev/add-ins/testing/create-a-network-shared-folder-catalog-for-task-pane-and-content-add-ins) на сайте. 
    
4. Выполните построение проекта.
    
5. Запустите проект.
    
## <a name="externalstandalone-application-development-environment"></a>Внешняя или автономная среда разработки приложений

Автономное приложение может вызывать Project Online с помощью клиентской объектной модели (CSOM) или REST для связи с Project Online для создания, получения, обновления и удаления данных, находящихся на сервере. Это автономное клиентское приложение, зависящее от уровня доступа пользователя, который необходимо запустить. 
  
Пример среды разработки Windows для этого типа приложения состоит из следующих элементов:
  
- Visual Studio 2015 (предпочтительно) или Visual Studio 2013 
    
- Средства разработки Microsoft Office для Visual Studio (поставляется с Visual Studio 2015 Professional и Enterprise Editions)
    
- .NET Framework 4,0 или более поздняя версия
    
- [Пакет CSOM SharePointOnline](https://www.nuget.org/packages/Microsoft.SharePointOnline.CSOM) (для вызовов CSOM) 
    
- Язык программирования, например C# 
    
Посетите https://github.com/OfficeDev/Project-CSOM-Read-Enterprise-CustomFields пример приложения. 
  
Пример можно выполнить несколькими шагами:
  
1. Скачать пример приложения
    
2. Выполните два изменения, чтобы получить доступ к сайту Project Online — имя сайта, учетная запись пользователя и пароль.
    
   Убедитесь, что у пользователя есть доступ ко всем проектам. В Project Online для управления доступом к данным в хранилище данных используются разрешения пользователя.
    
3. Добавьте сборку SharePoint в ссылки с помощью консоли диспетчера пакетов NuGet, доступной в консоли NuGet, введя следующую команду в консоли NuGet: 
    
   `Install-Package Microsoft.SharePointOnline.CSOM`

4. Выполните построение проекта.
    
5. Запустите проект.
