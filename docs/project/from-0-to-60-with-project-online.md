# <a name="from-0-to-60-with-project-online"></a>Быстрое начало разработки для Project Online

Разработчик приложения может настроить веб-Project Online (SharePoint) с помощью автономных приложений и/или Project надстройки. Возможно множество приложений, которые варьируются от решения потребностей тех, кто участвует в проекте, до функций поддержки PMO, таких как любое из следующих ниже:
  
- Упрощение записи данных с картой времени для сотрудников
- Эффективное утверждение системы времени для руководителей
- Надзор за разрешениями (закупками и состоянием), необходимыми для проекта
- Проверка состояния и здоровья активных проектов
- Отчет о проблемах
- Отчет о состоянии управления изменениями
    
Project Online включает поддержку API для размещения следующих сценариев:
  
- Для Project (SharePoint) надстройки:
    
  - Код (JavaScript, HTML, CSS), который находится в SharePoint Online
  - Активы, которые загружаются в браузер и выполняются SharePoint Online.  
  - Бизнес-логика, которая находится в JavaScript   
  - Доступ к данным, хранимым в Project Online или SharePoint, например (но не ограничивается):  
  - настраиваемые поля;  
  - Списки
    
- Для надстройки Project (SharePoint) поставщика:
    
  - Код, который существует на сайте, внешнем Project Online сайте 
  - Внешний сайт, который может быть (но не ограничен):  
  - Другой SharePoint сайт  
  - Веб-приложение/служба, созданная на любой платформе  
  - Внешний сайт содержит бизнес-логику  
  - Браузер перенаправляется с Project Online на внешний сайт с маркерами доступа к Project Online  
  - Внешний сайт может звонить на SharePoint и Project Online
    
- Для внешней или автономных надстройки:
    
  - Пользователь выполняет приложение на своем устройстве
  - Проверка подлинности приложения и вызовы Project Online API напрямую
    

|Тип приложения|Реализация API|Целевая среда|Примеры приложений|
|:-----|:-----|:-----|:-----|
|Project хостинга  <br/> |JSOM (объектная модель Java Script)  <br/> REST  <br/> |Браузер  <br/> |Запись с картой времени  <br/> Утверждение timecard  <br/> Состояние проекта  <br/> Отчет о проблемах  <br/> |
|Project Provider Hosted  <br/> |Клиентская библиотека CSOM  <br/> |Веб-сайт Azure/App  <br/> Нестандартная Windows (LAMP и т.д.)  <br/> |Валидатор внешнего времени  <br/> Project Импортер  <br/> |
|External/Standalone  <br/> |REST  <br/> CSOM  <br/> |REST — любая платформа  <br/> CSOM — любая поддерживаемая платформа .NET  <br/> |Запись с картой времени  <br/> Перенос проектов на новый сайт  <br/> Изменение состояния управления.  <br/> |
   
## <a name="what-does-it-take-to-start-developing-applications-for-project-online"></a>Что нужно для начала разработки приложений для Project Online?

Общими элементами, необходимыми для разработки Project Online приложений, являются Project Online учетные записи и тестовые данные — проекты и сведения, связанные с проектами, которые включают назначения, задачи, ресурсы и настраиваемые поля. Необходима также среда разработки, но специфика среды разработки зависит от типа приложения и интерфейса API, необходимых для приложения. В следующих нескольких разделах описываются потребности в разработке для трех интерфейсов API.
  
В справочной документации описывается общая для всех трех интерфейсов объектная модель, а также карта объектов, на которых показаны связи между компонентами объектной модели.
  
## <a name="project-hosted-add-in-development-environment"></a>Project среды разработки надстройки

Расположенная надстройка — это надстройка, которая находится на сервере и загружается в браузер для выполнения. Хост-надстройки могут использовать интерфейсы JSOM или REST и написаны на JavaScript. Project Online содержит ссылки на библиотеку JSOM для выполнения. Если разработка находится на платформе Windows, необходимые ресурсы следуют следующим образом:
  
- Visual Studio 2015 (предпочтительно) или Visual Studio 2013
    
- Office разработки для Visual Studio
    
- Язык JavaScript
    
Посетите https://github.com/OfficeDev/Project-JSOM-Copy-Work-Packages пример приложения. 
  
Вы можете скачать и запустить пример в нескольких простых шагах:
  
1. Скачайте и откройте пример приложения
    
2. Обновление siteURL в окне Свойства
    
   Project Online рассматривает область применения надстройки и разрешения пользователей для управления доступом к информации на Project Online хост. Если доступ явно отказано в любом или обоих параметрах, Project Online не будет доступа к этой информации. В противном случае доступ предоставляется.
    
3. Включить [побную загрузку](https://docs.microsoft.com/office/dev/add-ins/testing/create-a-network-shared-folder-catalog-for-task-pane-and-content-add-ins) на вашем сайте.  
    
4. Выполните построение проекта.
    
5. Запустите проект.
    
## <a name="project-provider-hosted-add-in-development-environment"></a>Project среды разработки надстройки с хостингом поставщика

Надстройки поставщика — это приложения, написанные и проживающие на любой веб-платформе. Они могут подключать и выполнять операции данных с помощью API REST (или CSOM для платформ Майкрософт). Для разработки можно использовать любой язык и среду, которая поддерживает интерфейс REST. 
  
Пример среды разработки Windows этого типа приложения включает следующие элементы:
  
-  Visual Studio 2015 (предпочтительно) или Visual Studio 2013 
    
- Microsoft Office Средства разработки для Visual Studio (поставляются с Visual Studio 2015 Professional и Enterprise выпусками)
    
- платформа .NET Framework 4.0 или более новые
    
- [Пакет CSOM SharePointOnline](https://www.nuget.org/packages/Microsoft.SharePointOnline.CSOM) (для вызовов CSOM) 
    
- Язык программирования, например C # 
    
Посетите https://github.com/OfficeDev/Project-Add-in-REST-BasicDataOperations рабочие примеры сценариев. 
  
Пример можно выполнить в нескольких шагах:
  
1. Скачайте и откройте пример приложения
    
2. Обновление siteURL в окне Свойства
    
   Project Online рассматривает область применения надстройки и разрешения пользователей для управления доступом к информации на Project Online хост. Если доступ явно отказано в любом или обоих параметрах, Project Online не будет доступа к этой информации. В противном случае доступ предоставляется.
    
3. Включить [побную загрузку](https://docs.microsoft.com/office/dev/add-ins/testing/create-a-network-shared-folder-catalog-for-task-pane-and-content-add-ins) на вашем сайте. 
    
4. Выполните построение проекта.
    
5. Запустите проект.
    
## <a name="externalstandalone-application-development-environment"></a>Среда разработки внешних и автономных приложений

Автономные приложения могут вызывать Project Online с помощью клиентской объектной модели (CSOM) или REST для связи с Project Online для создания, получения, обновления и удаления сведений, проживающих на сервере. Это автономные клиентские приложения, которые зависят от уровня доступа пользователей к запуску. 
  
Пример среды разработки Windows этого типа приложения включает следующие элементы:
  
- Visual Studio 2015 (предпочтительно) или Visual Studio 2013 
    
- Microsoft Office Средства разработки для Visual Studio (поставляются с Visual Studio 2015 Professional и Enterprise выпусками)
    
- платформа .NET Framework 4.0 или более новые
    
- [Пакет CSOM SharePointOnline](https://www.nuget.org/packages/Microsoft.SharePointOnline.CSOM) (для вызовов CSOM) 
    
- Язык программирования, например C # 
    
Посетите https://github.com/OfficeDev/Project-CSOM-Read-Enterprise-CustomFields пример приложения. 
  
Пример можно выполнить в нескольких шагах:
  
1. Скачайте пример приложения
    
2. Внести несколько изменений для доступа к Project Online сайта, имени сайта, учетной записи пользователя и пароля.
    
   Убедитесь, что пользователь имеет доступ ко всем проектам. Project Online для управления доступом к информации в хранилище данных с помощью пользовательских разрешений.
    
3. Добавьте сборку SharePoint ссылки с помощью консоли Nuget диспетчер пакетов, доступной в меню Tools, введя в консоль Nuget следующее: 
    
   `Install-Package Microsoft.SharePointOnline.CSOM`

4. Выполните построение проекта.
    
5. Запустите проект.
