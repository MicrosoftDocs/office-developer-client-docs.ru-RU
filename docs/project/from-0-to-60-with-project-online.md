# <a name="from-0-to-60-with-project-online"></a>Быстрое начало разработки для Project Online

Разработчик приложения можно настроить Project Online сайта (SharePoint hosted) с помощью автономных приложений и/или надстройки Project. Богатый набор приложений — это возможно, в диапазоне от-адресации потребностей пользователей, участвующих в проекте функции поддержки отдела управления Проектами, таких как одно из следующих:
  
- Упрощенный карточками табельного учета ввода данных для сотрудников
- Утверждение эффективно карточками табельного учета для руководителей
- Надзор за получение разрешений (Закупка и состояние), необходимых для проекта
- Проверка состояния и состояния активных проектов
- Отчет о проблемы
- Изменить отчет о состоянии управления
    
Project Online включает поддержку API для учета в следующих сценариях:
  
- Для Project (SharePoint), размещенные надстройки:
    
  - Код (JavaScript, HTML, CSS), которая размещается в SharePoint Online
  - Средства, которые загружаются в браузере и выполняются в SharePoint Online.  
  - Бизнес-логику, которая находится в JavaScript   
  - Доступ к данным, такие как / сохранены в Project Online и SharePoint (но не ограничиваясь):  
  - настраиваемые поля;  
  - Списки
    
- Для Project (SharePoint) у поставщика надстройки:
    
  - Код, существующий на сайте, внешними по отношению к сайт Project Online 
  - Внешний сайт, который может быть (но не ограничиваясь):  
  - Другой сайт SharePoint  
  - Веб-приложения и службы основаны на любой платформы  
  - Внешний узел содержит бизнес-логики  
  - Веб-обозреватель перенаправляется из Project Online для внешних сайтов с помощью маркеров доступа для Project Online  
  - Внешних сайтов могут выполнять вызовы SharePoint и Project Online
    
- Для внешних/автономных надстройке:
    
  - Пользователь выполняет приложение на мобильном устройстве
  - Приложение выполняет проверку подлинности и напрямую вызывает Project Online API-интерфейсы
    

|Тип приложения|Реализация интерфейса API|Целевая среда|Примеры приложений|
|:-----|:-----|:-----|:-----|
|Размещенные проекта  <br/> |JSOM (объектная модель)  <br/> REST  <br/> |Веб-браузер  <br/> |Запись карточками табельного учета  <br/> Утверждение карточками табельного учета  <br/> Состояние проекта  <br/> Отчет о проблемы  <br/> |
|Проект размещением у поставщика  <br/> |Клиентская библиотека CSOM  <br/> |Веб-сайт Azure/приложения  <br/> Среда не Windows (LAMP, и т.д.)  <br/> |Средства проверки внешних табеля учета рабочего времени  <br/> Средства импорта проектов  <br/> |
|Внешний/автономных  <br/> |REST  <br/> CSOM  <br/> |REST - любой платформы  <br/> CSOM - любой платформы .NET поддерживается  <br/> |Запись карточками табельного учета  <br/> Перенос проектов для нового сайта  <br/> Изменение состояния управления.  <br/> |
   
## <a name="what-does-it-take-to-start-developing-applications-for-project-online"></a>Что требуется быстро приступить к разработке приложений для Project Online?

Общие элементы, необходимые для разработки проекта веб-приложений являются учетной записи Project Online и тестовых данных--проектов и сведения, связанными с проектом, которые включают в себя назначения, задачи, ресурсы и настраиваемые поля. Также требуются среду разработки, но особенности среды разработки зависят от типа приложения и интерфейс API, необходимый для приложения. В следующих разделах описываются потребностей разработки для трех API-интерфейсов.
  
Справочная документация Описание объектной модели, которые являются общими для всех трех интерфейсов, а также сопоставление объектов, показаны отношения между компонентами объектной модели.
  
## <a name="project-hosted-add-in-development-environment"></a>Проект размещенные среды разработки надстройки

Размещенной надстройки — это надстройка, в которой находится на сервере и загружаются в браузер для выполнения. Размещенной надстройки могут использовать интерфейсы JSOM и REST и записывается в JavaScript. Project Online содержит ссылки на библиотеку JSOM для выполнения. Предполагается применение разработки — на платформе Windows, выполните необходимые ресурсы:
  
- Visual Studio 2015 (рекомендуется) или Visual Studio 2013
    
- Средства разработки Office для Visual Studio
    
- Язык JavaScript
    
Посетите https://github.com/OfficeDev/Project-JSOM-Copy-Work-Packages для примера приложения. 
  
Можно загрузить и запустить образец в несколько этапов:
  
1. Загрузите и откройте пример приложения
    
2. Обновите свойство SiteURL в окне свойств
    
   Project Online проверяет приложения области надстройки и разрешения пользователя для управления доступа к данным на узле Project Online. Если явно отказано в доступе в один или оба параметров Project Online не дает доступ к данным. В противном случае доступа.
    
3. Включите sideloading на вашем сайте. В статье [Настройка Project Online для разработки приложений ](http://nearbaseline.com/2013/12/configuring-project-online-for-app-development/.aspx)для получения дополнительных сведений. 
    
4. Выполните построение проекта.
    
5. Запустите проект.
    
## <a name="project-provider-hosted-add-in-development-environment"></a>Среда размещением у поставщика разработки надстройки Project

Надстройки размещением у поставщика, приложений и находящихся на любой веб-платформы. Они подключения и выполнения операций с данными с помощью REST (или CSOM для платформ Майкрософт, обеспечивающий) API. Любой язык и среда, поддерживающая интерфейс REST можно использовать для разработки. 
  
Пример среды разработки Windows для этого типа приложений включает в себя следующие элементы:
  
-  Visual Studio 2015 (рекомендуется) или Visual Studio 2013 
    
- Средства разработки Microsoft Office для Visual Studio (поставляется с помощью Visual Studio 2015 Professional и корпоративные выпуски)
    
- .NET framework 4.0 или более поздней версии
    
- [Пакет SharePointOnline CSOM](https://www.nuget.org/packages/Microsoft.SharePointOnline.CSOM/.aspx) (для CSOM звонков) 
    
- Язык программирования, таких как C# 
    
Посетите https://github.com/OfficeDev/Project-Add-in-REST-BasicDataOperations для работы примеры сценариев. 
  
Пример можно запустить в несколько этапов:
  
1. Загрузите и откройте пример приложения
    
2. Обновите свойство SiteURL в окне свойств
    
   Project Online проверяет приложения области надстройки и разрешения пользователя для управления доступа к данным на узле Project Online. Если явно отказано в доступе в один или оба параметров Project Online не дает доступ к данным. В противном случае доступа.
    
3. Включите sideloading на вашем сайте. В статье [Настройка Project Online для разработки приложений ](http://nearbaseline.com/2013/12/configuring-project-online-for-app-development/.aspx)для получения дополнительных сведений. 
    
4. Выполните построение проекта.
    
5. Запустите проект.
    
## <a name="externalstandalone-application-development-environment"></a>Внешний/автономного приложения среды разработки

Автономного приложения можно вызвать Project Online с помощью клиента со стороны объектной модели (CSOM) и REST для взаимодействия с Project Online в создание, получение, обновление и удаление сведений, находящихся на сервере. Это изолированная клиентского приложения, которое зависит от уровня доступа пользователя для запуска. 
  
Пример среды разработки Windows для этого типа приложений включает в себя следующие элементы:
  
- Visual Studio 2015 (рекомендуется) или Visual Studio 2013 
    
- Средства разработки Microsoft Office для Visual Studio (поставляется с помощью Visual Studio 2015 Professional и корпоративные выпуски)
    
- .NET framework 4.0 или более поздней версии
    
- [Пакет SharePointOnline CSOM](https://www.nuget.org/packages/Microsoft.SharePointOnline.CSOM/.aspx) (для CSOM звонков) 
    
- Язык программирования, таких как C# 
    
Посетите https://github.com/OfficeDev/Project-CSOM-Read-Enterprise-CustomFields для примера приложения. 
  
Пример можно запустить в несколько этапов:
  
1. Загрузить пример приложения
    
2. Внести несколько изменений для доступа к Project Online сайт — имя сайта, учетная запись пользователя и пароль.
    
   Убедитесь, что у пользователя есть доступ ко всем проектам. Project Online использует разрешения пользователей для контроля доступа к данным в хранилище данных.
    
3. Добавьте ссылки, с помощью консоли диспетчер пакетов Nuget из меню "Сервис", введя следующее в командной консоли Nuget сборки SharePoint: 
    
   `Install-Package Microsoft.SharePointOnline.CSOM`

4. Выполните построение проекта.
    
5. Запустите проект.
